**WHAT happened:**  
The code threw an `AttributeError` when attempting to call the `upper()` method on an element within the `to_class` method.

**WHY it happened:**  
In Python, `None` does not have an `upper()` method. The code line `array[i].upper()` triggers this error if `array[i]` is `None`. This behavior mimics Java's attempt to call a method on `null`, which results in a `NullPointerException`.

**WHERE it occurred:**  
This error occurs on line 186 inside the `to_class` method of the `ClassUtils` class.

**Fix Instructions:**  

1. **Remove the Faulty Line:**
   Replace the problematic line (`_ = array[i].upper()`) with a safer approach.

2. **Check for None:**
   Before calling any methods on `array[i]`, check if it is `None`. If it is, append `type(None)` to the `classes` list directly.

3. **Fixed Code for `to_class`:**
   ```python
   @classmethod
   def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
       if array is None:
           return None
       if len(array) == 0:
           return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
       
       classes = []
       for i in range(len(array)):
           if array[i] is None:
               classes.append(type(None))
           else:
               classes.append(type(array[i]))
       
       return classes
   ```

This alteration prevents the method from trying to call `upper()` on `None` and instead directly collects class types, including handling `None` appropriately as `type(None)`.
