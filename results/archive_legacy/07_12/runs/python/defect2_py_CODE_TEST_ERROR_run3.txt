#### WHY it happened:
The issue arises because the method uses the uncapped `value` instead of the clamped `v`. Consequently, if the input value is below the lower bound, it produces negative RGB values.

#### WHERE it occurred:
The problem appears on **Line 73** in the `get_paint` method of the `GrayPaintScale` class.
