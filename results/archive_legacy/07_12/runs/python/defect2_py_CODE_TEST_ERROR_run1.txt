### What Happened
The code raised a `ValueError` stating that the color parameter is outside the expected RGB range when trying to get a paint color for a value of `-0.5`.

### Why It Happened
This error occurred because the calculation of the grayscale value is using the original `value` instead of the clamped `v` within the `get_paint` method. When `value` is less than `lower_bound`, it leads to a negative result for `g` (the grayscale value), which is not allowed in RGB color values.

### Where It Occurred
The error happened on **line 73** in the `get_paint` method of the `GrayPaintScale` class:
```python
g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
```

### How to Fix It
1. **Change** the calculation to use `v` instead of `value`:
   ```python
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
   ```
   - This ensures `g` is calculated using the clamped value, preventing negative values.

By making this change, you will ensure that the grayscale calculation uses the valid, clamped range, thus avoiding negative or out-of-range RGB values.
