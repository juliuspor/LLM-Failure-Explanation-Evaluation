### What Happened
The code incorrectly rejects when negative minutes are provided alongside negative hours for time offsets. This creates a problem for offsets like `-02:15`, where `hours_offset` is `-2` and `minutes_offset` is `-15`.

### Why It Happened
The validation logic in `for_offset_hours_minutes` method only allows `minutes_offset` values between `0` and `59`. It doesn't account for scenarios where hours are negative, making negative minutes valid.

### Where It Occurred
The bug is located in the `for_offset_hours_minutes` method of the `DateTimeZone` class, specifically at line 109:
```python
if minutes_offset < 0 or minutes_offset > 59:  # Line 109: Bug is here
    raise ValueError(f"Minutes out of range: {minutes_offset}")
```
