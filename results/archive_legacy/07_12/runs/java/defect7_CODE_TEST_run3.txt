### Problem: Assertion in Test Fails

**WHAT happened:**

The assertion `assertTrue(Arrays.equals(new Class[] { String.class, null, Double.class }, ClassUtils.toClass(new Object[] { "Test", null, 99d })));` in the test fails.

**WHY it happened:**

The method `ClassUtils.toClass(Object[] array)` incorrectly attempts to access the class of `null` values, causing a `NullPointerException`.

**WHERE it occurred:**

The error occurs in the method `toClass` within `ClassUtils` on this line:

```java
classes[i] = array[i].getClass();
```

When `array[i]` is `null`, calling `getClass()` results in an exception.

### Solution Instructions

**Fix the bug in method `toClass` as follows:**

1. **Add a Null Check:**

Modify the loop within the `toClass` method to handle `null` values correctly by checking if each element is `null` before calling `getClass()`.

**Here's how:**

```java
public static Class<?>[] toClass(Object[] array) {
    if (array == null) {
        return null;
    } else if (array.length == 0) {
        return ArrayUtils.EMPTY_CLASS_ARRAY;
    }
    Class<?>[] classes = new Class[array.length];
    for (int i = 0; i < array.length; i++) {
        classes[i] = (array[i] != null) ? array[i].getClass() : null;
    }
    return classes;
}
```

- **Add Null Check**: Check if `array[i]` is `null`.
- **Set to `null`**: If it’s `null`, set `classes[i]` to `null`.
- **Otherwise**: Use `getClass()` on non-`null` elements.

### Summary

- Problem was a `NullPointerException` in `toClass`.
- Fixed by adding a null check before `getClass()`.
- Implement the fix by updating the `for` loop in `toClass`.
