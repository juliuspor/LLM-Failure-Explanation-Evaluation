### Problem Identification

**Root Cause**: There's a specific error that might occur when calculating offsets due to improper handling of negative numbers in the `forOffsetHoursMinutes` method.

### Explanation of the Issue

#### WHAT Happened
An exception can be thrown if `forOffsetHoursMinutes(int hoursOffset, int minutesOffset)` is called with positive `hoursOffset` and negative `minutesOffset`, such as `(2, -15)`.

#### WHY It Happened
The current logic does not handle the case where `hoursOffset` is positive and `minutesOffset` is negative. Specifically, the following code incorrectly modifies `minutesOffset`:

```java
if (hoursInMinutes < 0) {
    minutesOffset = hoursInMinutes - minutesOffset; // Incorrect handling
} else {
    minutesOffset = hoursInMinutes + minutesOffset;
}
```

This incorrectly allows scenarios where a positive `hoursOffset` with a negative `minutesOffset` leads to a valid offset, which is logically inconsistent. Moreover, in cases where minutes cross boundaries with negative values (e.g., 0:00 crossing into -0:15), this logic fails.

#### WHERE It Occurred
The problematic function, `forOffsetHoursMinutes`, is defined at or around line 229 in the given class `DateTimeZone`.

### Actionable Fix Instructions

1. **Fix Negative Handling Logic**:
   - Change the handling of `minutesOffset` when `hoursOffset` is positive. Do not allow negative `minutesOffset` if `hoursOffset` is positive.

2. **Block Invalid Range**:
   - Add a check at the start of `forOffsetHoursMinutes` to block combinations of positive hours and negative minutes. Here's the corrected logic:

   ```java
   public static DateTimeZone forOffsetHoursMinutes(int hoursOffset, int minutesOffset) throws IllegalArgumentException {
       // Prevent positive hours with negative minutes
       if (hoursOffset > 0 && minutesOffset < 0) {
           throw new IllegalArgumentException("Minutes cannot be negative when hours are positive");
       }
       if (hoursOffset == 0 && minutesOffset == 0) {
           return DateTimeZone.UTC;
       }
       if (hoursOffset < -23 || hoursOffset > 23) {
           throw new IllegalArgumentException("Hours out of range: " + hoursOffset);
       }
       if (minutesOffset < -59 || minutesOffset > 59) {
           throw new IllegalArgumentException("Minutes out of range: " + minutesOffset);
       }
       ...
   }
   ```

Incorporating this validation makes the function properly handle invalid input combinations, improving both functionality and robustness.
