**What Happened**

A test using the `GrayPaintScale` class failed. It produced an `IllegalArgumentException` saying: "Color parameter outside of expected range: Red Green Blue".

**Why It Happened**

The problem is due to incorrect calculation of the color's gray scale value in the `getPaint` method. When the method receives a value that is below the `lowerBound`, the calculation results in a negative number. This causes an `IllegalArgumentException` since the `Color` constructor expects values between 0 and 255 for RGB.

**Where It Occurred**

The issue is located in the `getPaint` method (lines 65-73 in your code):

```java
int g = (int) ((value - this.lowerBound) / (this.upperBound - this.lowerBound) * 255.0);
```

If `value < lowerBound`, `g` becomes negative because `value - this.lowerBound` results in a negative number.

1. **Adjust Range Calculation:** Ensure `g` is calculated using the clamped value `v` rather than `value`.

2. **Change Code:** Modify the `getPaint` method to use `v` for calculations.

```java

    double v = Math.max(value, this.lowerBound);
    v = Math.min(v, this.upperBound);
    int g = (int) ((v - this.lowerBound) / (this.upperBound - this.lowerBound) * 255.0);
    return new Color(g, g, g);
}
```

This ensures that `g` is always in the range of 0 to 255, thus fixing the exception.
