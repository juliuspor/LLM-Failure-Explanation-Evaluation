**Problem Explanation**

### What Happened:
There was an error when trying to create a `Color` object with the color components (red, green, and blue) being outside the valid range of 0 to 255.

### Why It Happened:
The method `getPaint(double value)` incorrectly calculates the grayscale value, `g`. When the input `value` is lower than `lowerBound` (like -0.5 in the test), `g` becomes negative, leading to an invalid color value.

### Where It Occurred:
The problem is within the `getPaint` method (lines 82-86). Specifically, the calculation of `g` does not take into account the clamping done for `v` at lines 83-84 and still runs the risk of being negative.

**Solution**

### Instructions:
1. Fix the `getPaint` method to use the clamped value `v` rather than the original input `value` when calculating `g`.

2. Change line 86 of `getPaint` from:

   ```java
   int g = (int) ((value - this.lowerBound) / (this.upperBound - this.lowerBound) * 255.0);
   ```

   To:
   
   ```java
   int g = (int) ((v - this.lowerBound) / (this.upperBound - this.lowerBound) * 255.0);
   ```

### Outcome
By using the clamped value `v`, `g` will always be within the valid range (0-255) for color components, preventing the `IllegalArgumentException`.
