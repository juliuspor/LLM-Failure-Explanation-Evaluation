What happened (root cause)
- The loop at while mantissa % 10 == 0 (inside add_number) can never terminate when value == 0. 
- For x == 0 you set value = int(x) = 0 and mantissa = 0. Then:
  - 0 % 10 == 0 → loop enters
  - check_val = (0 // 10) * (10 ** (exp + 1)) = 0
  - check_val == value → True, so you do mantissa //= 10 → mantissa stays 0, exp increments
  - Next iteration it's the same: mantissa is still 0, check_val is still 0, loop never breaks.
- So the code hangs (infinite loop) inside add_number while trying to trim trailing zeros for the integer 0. This occurs in add_number, at the while loop that begins with while mantissa % 10 == 0.

Why this is the problem
- The algorithm assumes mantissa will shrink to a non-multiple-of-10 and the loop will end. That assumption fails for mantissa == 0, because integer division by 10 keeps it 0 forever, so the loop condition stays true.

How to fix it (concrete steps)
1. Short-circuit value == 0 before the loop:
   - Immediately output "0" (or handle it as you intend) and return.
   Example:
     if value == 0:
         self.add("0")
         return

2. Or change the loop condition to avoid iterating when mantissa is zero:
   Replace
     while mantissa % 10 == 0:
   with
     while mantissa != 0 and mantissa % 10 == 0:

Either change prevents the infinite loop by ensuring the loop doesn't run when mantissa is 0.

Where to change (line references)
- In add_number, inside the integer branch (the block beginning with if x == int(x) ...), modify the code just before or at the while mantissa % 10 == 0 line as shown above.
