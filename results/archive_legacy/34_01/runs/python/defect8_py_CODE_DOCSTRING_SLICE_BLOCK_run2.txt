What happens
- The check at lines 239–240:
  if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
    raise ValueError(f"Invalid locale format: {locale_str}")
  is rejecting some valid inputs because it compares characters by direct character ordering (ch3 < 'A', etc.). In Python those comparisons use Unicode codepoint order, not an ASCII-only class test. Characters outside the plain A–Z range (for example accented letters or other Unicode letters) will compare as "less than 'A'" (or "greater than 'Z'") even when your intent was just to check whether the characters are uppercase ASCII letters.

Why that causes a failure (concrete)
- Suppose locale_str is "en_ÑA" or contains any country-letter that is a Unicode uppercase letter but not in the ASCII A–Z range. At line 239 you do ch3 < 'A' which will be True for some non-ASCII characters, so the code raises ValueError even though the real intent was to allow only ASCII uppercase letters — or at least to test uppercase using a character-class test. The failure therefore happens at the check of ch3/ch4 (lines 239–240), causing false negatives.

Where it occurs
- The problematic check is in to_locale, after the underscore separator is validated (the block starting with ch3 = locale_str[3], ch4 = locale_str[4]) — specifically lines 239–240 in your slice block.

Correct, reliable fixes (step-by-step)
1. Use explicit ASCII checks (preferred if you want strictly A–Z):
   - Replace the comparisons with:
       if not ('A' <= ch3 <= 'Z') or not ('A' <= ch4 <= 'Z'):
           raise ValueError(...)
     This is equivalent in intent but clearer. However it still relies on codepoint ordering — it works for ASCII range checks because the comparisons operate correctly for ASCII.
2. Better: use str.isupper() and ensure ASCII only if required:
   - If you accept any Unicode uppercase letters:
       if not (ch3.isupper() and ch4.isupper()):
           raise ValueError(...)
   - If you need strictly ASCII A–Z:
       import re
       if not re.match(r'^[A-Z]{2}$', locale_str[3:5]):
           raise ValueError(...)
3. Prefer a single regex to validate the whole format (clearer and less error-prone):
   - Example pattern for strict ASCII rules:
       ^[a-z]{2}(?:_[A-Z]{2}(?:_.+?)?)?$
     Steps:
     - Validate with the regex.
     - If matched, extract language, country (if present), variant (if present), and call Locale(...).
