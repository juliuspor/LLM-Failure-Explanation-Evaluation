What happened (where it fails)
- The exception is raised at lines 74–75 inside get_paint (see slice_union). The code computes
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)  # line 71
  then immediately checks
  if g < 0 or g > 255:
      raise ValueError(f"Color parameter outside of expected range: Red Green Blue")  # lines 74–75
- So the ValueError is thrown because g ended up < 0 or > 255.

Why it happened (root cause, precisely)
- g is a linear mapping of value into the 0–255 color range. g < 0 occurs when value < self._lower_bound. g > 255 occurs when value > self._upper_bound.
- In short: get_paint received a value outside the configured range [self._lower_bound, self._upper_bound], so the mapping produced an out-of-range 8‑bit color component and the check at lines 74–75 raised the ValueError.
- Note: this is not a division-by-zero error. If self._lower_bound == self._upper_bound you would get a ZeroDivisionError at line 71 instead; the raised ValueError means the division produced a finite g but out of 0–255 bounds.

  That will show whether value < lower or value > upper (and by how much).
