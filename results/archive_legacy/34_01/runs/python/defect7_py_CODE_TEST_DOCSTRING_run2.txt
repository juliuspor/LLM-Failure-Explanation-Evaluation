What failed
- The exception happens inside the for-loop on this line:
      _ = array[i].upper()
  When the second element is None that executes as None.upper() and Python raises:
      AttributeError: 'NoneType' object has no attribute 'upper'
- Because the loop stops on that statement, the code never reaches the type-building line:
      classes.append(type(array[i]))
  so the function never returns the expected [str, type(None), float].

Why it fails (root cause)
1. The call to array[i].upper() assumes every element is a string (has an upper() method). It is not true for None or non-string objects (float, int, etc.). Calling .upper() on those values raises AttributeError.
2. The code’s behavior contradicts the docstring/test: the docstring says None elements should be converted to a None element in the output array (and the test expects type(None) in the result). Instead, the code raises when it encounters None.
3. The offending statement is unnecessary for computing the element types — it serves no purpose for building the classes list.

Where it occurs (specific variables/lines)
- Inside the for loop over array:
    - array[i].upper()  <-- this causes the AttributeError when array[i] is None (or any non-string)
    - classes.append(type(array[i]))  <-- the intended action, not reached for the failing element

How to fix it (concrete steps)
1. Remove the .upper() call entirely (it is not needed to determine the element’s class).
2. Explicitly handle None values so you append type(None) (or None) as the docstring and test require.
3. Optionally use Pythonic iteration rather than indexing.

Fixed implementation examples

Option A — return list of types, with None -> type(None)
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for elem in array:
            if elem is None:
                classes.append(type(None))
            else:
                classes.append(type(elem))
        return classes

Option B — if you want None in the result list (not type(None)), adjust accordingly:
    ...
            if elem is None:
                classes.append(None)
            else:
                classes.append(type(elem))
    ...

Notes
- Remove any stray .upper() call unless you actually intended to validate or normalize string elements; if validation is required, check types first:
      if isinstance(elem, str):
          s = elem.upper()    # safe
- Update the docstring to match behavior. The current "Raises: AttributeError: If any element is None" is wrong for the intended behavior and should be removed or corrected.

With the fixed code above, the test
    result = ClassUtils.to_class(['Test', None, 99.0])
will produce [str, type(None), float] as expected.
