What happens (where)
- The failure occurs at line 766: `_ = array[i].upper()`.
- That line calls the string method upper() on whatever is in array[i].

Why it fails (root cause)
- Not every item in array is guaranteed to be a str. If array[i] is None, an int, a dict, or any non-string object, it does not have an upper() method and Python raises an AttributeError (e.g. "'int' object has no attribute 'upper'").
- The code then uses type(array[i]) on the same non-string value and returns its type, so the call to upper() is both meaningless and unsafe for non-strings.

Concrete fixes (step-by-step)
1. Decide the intended input:
   - If array is supposed to contain strings, enforce/validate that and raise a clear error when an element is not a string.
   - If array may contain other types and you only want to uppercase strings, only call upper() on strings and skip or coerce other values.
