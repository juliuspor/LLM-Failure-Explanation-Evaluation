What happened (where it failed)
- The exception is raised at the validation on line 396:

      raise ValueError(f"Minutes out of range: {minutes_offset}")
- You called for_offset_hours_minutes(...) with minutes_offset == -15, so the method throws ValueError("Minutes out of range: -15") immediately at that check.

Why this check causes the failure (root cause)
- The code treats minutes_offset as an unsigned quantity (0..59) at line 396, rejecting any negative minute values.
- Later, at lines 402–404, the code assumes it can compute a signed total minutes by combining hours_in_minutes and minutes_offset:
    hours_in_minutes = hours_offset * 60
    if hours_in_minutes < 0:
        minutes_offset = hours_in_minutes - minutes_offset
    else:
        minutes_offset = hours_in_minutes + minutes_offset
  That computation expects minutes_offset to be a non-negative minute magnitude when hours_in_minutes is negative (so it subtracts the minute magnitude), or to be non-negative when hours_in_minutes is positive (so it adds).
- If the caller passed a negative minutes_offset (e.g. hours_offset = -5, minutes_offset = -15), the initial validation rejects it. In other words: the code both rejects negative minutes up-front and then later uses minutes_offset as a sign-less magnitude. The validation and the later arithmetic are inconsistent with callers who pass negative minute values.
