What happened
- The test calls ArrayUtils.add(None, None, expected_type=str).
- The implementation checks expected_type against the element before it handles the special case "element is None".
- Because isinstance(None, str) is False, the code raises TypeError instead of returning [None].
- That contradicts the docstring, which says: "If the input list is None, a new one element list is returned whose component type is the same as the element, unless the element itself is None, in which case the return type is List[object]."

Why it failed (root cause)
- The root cause is ordering of checks inside add(). The code enforces the expected_type constraint unconditionally (or too early), even when element is None. The docstring explicitly describes a different branch: when element is None, the method should accept it and produce a List[object] result, regardless of expected_type.
- In short: element == None should be handled before (or should bypass) the expected_type isinstance check. As written, the isinstance check finds None is not a str, so it raises TypeError.

Where this occurs (what to change)
- In add(array, element, expected_type): move the "element is None" handling ahead of any type-check that would reject None. Concretely:
  1. If array is None and element is None: return [None] (component type object).
  2. Otherwise, if expected_type is provided and element is not None, enforce isinstance(element, expected_type).
- Alternatively, if you want a single place to normalize expected_type: if element is None, set expected_type = object (or skip checking), then continue.

Suggested concrete fixes (exact steps)
1. Locate the type-check block where you do:
     if expected_type and not isinstance(element, expected_type):
         raise TypeError(...)
2. Change the order so that you handle element is None before that block:
     if element is None:
         # when array is None: return [None]
         # when array exists: append None (no type error)
         ...
     if expected_type and not isinstance(element, expected_type):
         raise TypeError(...)
3. Or, set expected_type to object when element is None:

         expected_type = object
     if expected_type is not None and not isinstance(element, expected_type):
         raise TypeError(...)

Why this fix is correct
- It implements the docstring behavior: when the element is None, the method must not reject it just because expected_type is non-object. Returning a single-element list containing None (with logical component type object) satisfies the documented contract.
