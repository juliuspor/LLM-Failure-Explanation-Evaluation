What happened (where it fails)
- The bug is in the lines that compute the total minutes from hours_offset and minutes_offset:
  - Line computing hours_in_minutes: hours_in_minutes = hours_offset * 60
  - The conditional branch:
    if hours_in_minutes < 0:
        minutes_offset = hours_in_minutes - minutes_offset
    else:
        minutes_offset = hours_in_minutes + minutes_offset
  - For hours_offset = -2 and minutes_offset = -15 this computes:
    hours_in_minutes = -120
    minutes_offset = -120 - (-15) = -105
  - That yields -105 minutes (-01:45) instead of the expected -135 minutes (-02:15).
- In short: subtracting minutes_offset when hours_in_minutes is negative undoes the negative minutes rather than combining them, so two negative parts become less negative.

Why it happens (root cause)
- The code treats the minutes sign incorrectly when the hour part is negative. The expression hours_in_minutes - minutes_offset is wrong when minutes_offset is already negative. The correct arithmetic for total minutes is simply:
  total_minutes = hours_in_minutes + minutes_offset
  regardless of sign.
- The current code effectively flips the sign of minutes_offset when hours are negative, producing the wrong total.

Where to change (exact variables/lines)
- Change the branch that assigns minutes_offset after computing hours_in_minutes:
  Replace both branches (the if/else that sets minutes_offset) with a single line:
    total_minutes = hours_in_minutes + minutes_offset
  Use total_minutes (or overwrite minutes_offset) for the subsequent call to FieldUtils.safe_multiply.
