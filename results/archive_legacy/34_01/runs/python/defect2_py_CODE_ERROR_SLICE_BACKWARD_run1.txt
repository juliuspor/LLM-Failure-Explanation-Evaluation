What happened (where it failed)
- At lines 57–59 you clamp the incoming value into the local variable v:
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
- At line 71 you compute g using the original value, not the clamped v:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- Because value can be outside the bounds, g can be < 0 or > 255, so the check at line 74 triggers:
  if g < 0 or g > 255:
      raise ValueError(...)

Why that causes the ValueError
- You intended to clamp value but then ignored the clamped result and used the un-clamped value to compute g. If value < _lower_bound you compute a negative fraction; if value > _upper_bound you compute a fraction > 1. Both yield g outside 0–255 and cause the raised ValueError.

Exact fixes (step-by-step)
1. Use the clamped variable v when computing g:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
2. Add a guard against identical bounds to avoid division by zero:
   if self._upper_bound == self._lower_bound:
       g = 0  # or 255 or whatever makes sense for your use case
3. (Optional) Improve the raised message to include the offending value and bounds for easier debugging:
   raise ValueError(f"Color parameter out of range: g={g}, value={value}, bounds=({self._lower_bound},{self._upper_bound})")
