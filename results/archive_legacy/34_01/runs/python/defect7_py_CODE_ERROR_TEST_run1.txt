What happened (root cause)
- The exception comes from this line inside the loop:
  _ = array[i].upper()
- When array[i] is None (or when array[i] is not a str), calling .upper() raises AttributeError because NoneType (and float) do not have an upper() method.
- In your failing test input ['Test', None, 99.0], the second element is None, so array[1].upper() triggers the AttributeError.

Where it occurs (exact spot)
- In the for loop (the third block of the function). Concretely it's the invocation of .upper() on array[i] right before you append its type.

Why the .upper() call is wrong here
- You only need the element's type (you use type(array[i]) immediately after), so calling .upper() is unnecessary.
- .upper() assumes the element is a string; that assumption isn't true for None or numbers, so it will fail for those values.

How to fix it (clear steps)
1. Remove the .upper() call entirely if all you want is the element type.
2. Prefer iterating over elements directly (for element in array) instead of indexing.
3. If you do need to treat strings specially, check isinstance(element, str) before calling .upper().

Corrected function (one clear fix)
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for element in array:
            classes.append(type(element))
        return classes

This will make ClassUtils.to_class(['Test', None, 99.0]) return [str, type(None), float] as your test expects.

Alternative if you intended to normalize string values first
    for element in array:
        if isinstance(element, str):
            _ = element.upper()  # safe now
        classes.append(type(element))

But only add that if you actually need the uppercased value for some side effect; otherwise remove it.
