What happened (what failed)
- The loop in ClassUtils.to_class tries to call .upper() on every array element at lines 765–766:
  - 765 |         for i in range(len(array)):
  - 766 |             _ = array[i].upper()
- When you call ClassUtils.to_class(['Test', None, 99.0]) that code executes array[1].upper() and array[2].upper():
  - array[1] is None — NoneType has no attribute upper → AttributeError.
  - array[2] is 99.0 — float has no attribute upper → AttributeError.
- So the method raises AttributeError during the loop instead of returning the expected [str, type(None), float].

Why it fails (root cause)
- The code assumes every element is a string and unconditionally calls .upper() on array[i]. It does not:
  - check for None,
  - check for non-string types, or
  - use type() / __class__ to get the element’s class.
- The docstring even says "If any of these objects is None, a None element will be inserted into the array" but the implementation contradicts that (and the "Raises" section is inconsistent). The unconditional .upper() is the precise mismatch between intended behavior and implementation.

Where it occurs
- In ClassUtils.to_class, lines 765–766 are the immediate point of failure because they call array[i].upper() without validation.

How to fix it (concrete steps)
1. Add a None check and early return for a None input array:
   - if array is None: return None
2. For each element, do not call .upper() unconditionally. Instead:
   - elem = array[i]
   - if elem is None:
       result[i] = None
     else:
       result[i] = type(elem)         # or elem.__class__
3. (Optional) If the method should accept string class names (e.g., "java.lang.String") and map them to classes, perform that mapping only when elem is a string:
   - if isinstance(elem, str):
       result[i] = lookup_class_from_name(elem)   # implement safe lookup
     else:
       result[i] = type(elem)
4. Remove or correct the misleading "Raises" section in the docstring so it matches actual behavior.

Minimal corrected loop example
- Replace lines 765–766 with something like:
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          result[i] = None
      else:
          result[i] = type(elem)

That change prevents AttributeError for None and non-string objects and will make the test pass (['Test', None, 99.0] → [str, type(None), float]).
