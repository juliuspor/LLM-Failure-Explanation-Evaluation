What happened (where it fails)
- At line 126 you defined method add(self, newcode).
- At line 138 that method calls self.append(newcode).
- The code fails because self.append is not the built-in list.append you expected; instead it resolves to a method that ultimately calls add again, producing infinite recursion (or else append doesn’t exist and you get AttributeError). The crash therefore happens in add at line 138 when it calls self.append.

Why that occurs (root cause)
- If the class either:
  - defines append elsewhere as a thin wrapper that calls add (for example append = add or append method that delegates back to add), then add -> append -> add -> … repeats until Python raises RecursionError; or
  - does not define append and the object is not a list or does not inherit from a class that provides append, then self.append does not exist and you get AttributeError at line 138.
- In short, add calls append, but append is not the independent, lower-level storage operation you intended — it is either missing or delegates back to add, so you get a loop or a missing-attribute error.
