The bug stems from mixing UTF-16 code unit indexing with Python string codepoint indexing. I will rewrite CharSequenceTranslator.translate to work in Python code points: build a list of codepoints from input_seq, pass that list and codepoint indexes to translate_codepoint, and when no translation occurs append UTF-16 code units (surrogate pair for codepoints > 0xFFFF) to out. This keeps translator implementations simple and ensures indices are consistent. The function will raise on None out and return on None input_seq, matching previous behavior.
