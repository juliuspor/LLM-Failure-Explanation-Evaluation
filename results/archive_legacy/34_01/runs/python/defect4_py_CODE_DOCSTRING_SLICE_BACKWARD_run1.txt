What happened (what fails)
- In the "max middle" section you test self._max_middle_index (if self._max_middle_index >= 0) but then read the period using self._min_middle_index:

- That is the wrong variable. As a result the code compares the current period's middle to the middle of the period at _min_middle_index instead of the period at _max_middle_index.

Why it fails (root cause)
- Logic bug: a copy/paste (or typo) made the code use _min_middle_index where it must use _max_middle_index.
- Two concrete problems follow:
  1. Wrong comparisons: the max-middle tracking never really compares against the existing max entry; it compares against the min entry instead, so _max_middle_index can end up incorrect.
