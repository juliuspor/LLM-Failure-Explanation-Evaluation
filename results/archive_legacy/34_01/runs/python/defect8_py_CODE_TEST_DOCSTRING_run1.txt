Why it fails (root cause)
- The code assumes that when a variant is present the country part (characters at indexes 3 and 4) is always a two-letter uppercase country code.
- It does not handle the valid case where the country is empty and the variant is present. That format uses two consecutive underscores: language + "_" + "_" + variant (e.g. "fr__POSIX").
- Specifically, the code never checks for the "empty country" pattern (locale_str[3] == '_') and therefore incorrectly enforces uppercase letters at positions 3 and 4.

- The problematic variables/indices: locale_str[2], locale_str[3], locale_str[4], ch3, ch4.
- The error is raised by the block that validates ch3 and ch4 as uppercase letters immediately after verifying locale_str[2] == '_'.
