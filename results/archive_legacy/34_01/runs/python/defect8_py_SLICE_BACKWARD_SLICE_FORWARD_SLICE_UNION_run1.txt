- Where it happened: inside to_locale(cls, locale_str: str) around lines 231 and 237–240. Specifically, line 231 checks length == 2, then lines 237–238 do ch3 = locale_str[3], ch4 = locale_str[4].
- Why it happened: the length check is wrong / mismatched with the indices that follow. Indexing uses 0-based offsets, so reading characters 3 and 4 requires length >= 5, but the code only guaranteed length == 2.

- It prevents accessing locale_str[3] and locale_str[4] unless length >= 5, eliminating the IndexError.
- It validates underscores at indices 2 and 5 before slicing, matching the expected format positions.
- It keeps explicit checks and clear failure points so you won't guess where the error came from.
