What happened (where it failed)
- In GrayPaintScale.get_paint(value) (method starting at line 54), the code computes g on line 71:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- For value = -0.5 that expression produces a negative g.
- On lines 74–75 the code checks "if g < 0 or g > 255:" and raises ValueError. That is why the test fails instead of returning (0, 0, 0).

Why it failed (root cause)
- The method assumes g will always be inside [0,255] and raises an exception if it is not.
- The test expects out-of-range inputs to be clamped to the nearest valid color (value below the lower bound should map to black (0,0,0)), but the current code does not clamp — it raises.
- In short: negative input yields negative g, the code raises at lines 74–75 instead of returning a clamped color.

Exactly where to change it and how to fix it
Replace the raising behavior with clamping. Concretely:
- Keep line 71 as the computed g.
- Replace lines 74–75 with explicit clamping to [0,255], for example:

  if g < 0:
      g = 0
  elif g > 255:
      g = 255

- Then return the gray tuple (g, g, g).

Why this fix works
- For value = -0.5 the computed g becomes negative, the clamping sets g = 0, and get_paint returns (0, 0, 0) as the test expects.
- For values above the upper bound the same clamp maps them to (255,255,255) instead of raising.
