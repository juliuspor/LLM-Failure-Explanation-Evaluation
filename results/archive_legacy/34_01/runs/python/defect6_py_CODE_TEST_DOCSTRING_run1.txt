What happened (root cause)
- The code treats -0.0 as an integer zero because of this test:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
  For x = -0.0, x == int(x) is True (int(-0.0) == 0). So execution enters the integer branch.
- Inside that branch you do:
  value = int(x)  # value becomes 0
  ...
  self.add(str(value))  # becomes "0"
  That converts the numeric value 0 to the string "0", which loses the sign bit. The result is "0", not "-0.0".
- The earlier check that tries to handle negatives for spacing:
  if x < 0 and prev == '-':
  does not trigger for -0.0 because in Python -0.0 < 0 is False. So the code never notices the sign of -0.0 before the conversion.

Where it occurs in your code
- The check if x < 0 and prev == '-' (first conditional) fails for -0.0.
- The integer-branch test if x == int(x) (the next conditional) is True for -0.0 and drives the code into converting to int and then to "0", which drops the negative sign.

How to fix it (concrete steps)
1. Detect negative zero early, before the integer branch, using math.copysign. Example insertion (right after prev = self.get_last_char()):
   import math  # if not already imported
   if x == 0.0 and math.copysign(1.0, x) < 0:
       # preserve the sign of -0.0; add its normal float text and return
       self.add(str(x))
       return

   This preserves "-0.0" exactly the same way str(-0.0) prints it, and stops the rest of the logic from converting it to 0.

2. Optionally adjust the spacing check if you intended to treat -0.0 as negative for spacing:
   Replace
       if x < 0 and prev == '-':
   with
       if (x < 0 or (x == 0.0 and math.copysign(1.0, x) < 0)) and prev == '-':
   so the spacing rule applies to -0.0 too.

Why these fixes work
- math.copysign(1.0, x) < 0 is a reliable way to tell whether x carries a negative sign bit, even when x == 0.0.
- Handling negative zero before converting to int prevents losing the sign when value = int(x) yields 0.

Minimal code change (combined)
Right after prev = self.get_last_char(), add:

    if x == 0.0 and math.copysign(1.0, x) < 0:
        self.add(str(x))
        return

This makes cc.add_number(-0.0) produce "-0.0" and keeps the rest of your formatting logic unchanged.
