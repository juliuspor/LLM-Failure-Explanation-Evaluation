Bug arises because translate uses UTF-16 code_units and advances pos by len(c) (UTF-16 units) after copying fallback, but when a translator consumes codepoints it returns count of codepoints, and loop increments pos by char_count(code_point_at(code_units,pos)) per consumed. The error occurs when pos becomes equal to length and then code_point_at is called. Fix by changing loop to: while pos < length: consumed = translate_codepoint(...); if consumed == 0: ... pos += len(c); else: for i in range(consumed): if pos >= length: break; cp = code_point_at(code_units,pos); pos += char_count(cp); This avoids calling code_point_at when pos==length.
