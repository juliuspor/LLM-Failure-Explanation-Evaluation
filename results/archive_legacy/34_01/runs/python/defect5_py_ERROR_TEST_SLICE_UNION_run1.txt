What happened (root cause)
- The method ArrayUtils.add tries to build a typed Java array from the existing array and the new element.
- When every element seen so far is None, the code never sets a concrete element type. At line 337 the code initializes inferred_type = object and line 340 only changes inferred_type when element is not None. Because both the existing element and the element being added are None, inferred_type stays java.lang.Object.
- Later the code converts/casts the underlying Object[] into a String[] because the caller passed expected_type=str. That cast fails at runtime: an Object[] cannot be cast to a String[] (ClassCastException), producing the error message you saw: "Cannot cast object list to str list".
- The check at lines 346–347 (if inferred_type == object and expected_type != object: raise TypeError) is reached only in some code paths. In your failing run the conversion/cast happened before or instead of that check, so you got the ClassCastException rather than the intended TypeError.

Why this is wrong (why it fails)
- None values give no runtime element type, so inferred_type remains object.
- The implementation assumes it can cast the internal Object[] to the caller-requested array type (String[]). That assumption is false when the runtime array is actually Object[]; Java does not allow casting Object[] -> String[].
- The code does not consistently use expected_type as the array component type when inferred_type is still object and expected_type was provided. Instead it attempts a direct cast, causing the ClassCastException.

- The problematic cast happens after type inference and before/around the check at lines 346–347 that would have raised TypeError; the cast triggers the ClassCastException instead of the explicit TypeError.
- The failing variable names: inferred_type (stays object), expected_type (equals str), and the internal array (Object[]) that is being cast to String[].

     - if inferred_type == object and expected_type is not None and expected_type != object: set inferred_type = expected_type
   - This makes the method create an array of the requested component type instead of trying to cast an Object[] into that type.

3. Keep a clear error path:
