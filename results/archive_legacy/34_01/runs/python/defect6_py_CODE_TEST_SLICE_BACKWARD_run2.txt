What happened (where it fails)
- The failure occurs inside add_number (the branch that starts at "if x == int(x) ...").
- For x = -0.0, the test x == int(x) is true, so the code executes the integer branch. On the next line you do value = int(x). int(-0.0) becomes 0, losing the negative sign and fractional form.
- The code then calls self.add(str(value)) and writes "0" into the code buffer. Your test expected "-0.0", so the assertion fails.

Why it happens (root cause)
- Negative zero is equal to zero (x == int(x) is true) but still has a distinct sign bit in IEEE floats. Converting to int discards that sign bit and any fractional form.
- The code treats every float equal to an integer as a plain integer by doing value = int(x) and then str(value), which cannot represent -0.0 correctly.

Exactly where (variables and lines)
- The problematic logic begins at the condition: if x == int(x) and not math.isinf(x) and not math.isnan(x):
- The specific line that loses the sign is: value = int(x)
- The final write that produces the wrong output is: self.add(str(value))

Concrete fixes (step-by-step)
1. Detect negative zero before converting to int and preserve its string form. For example, at the top of add_number, add:
   - if x == 0.0 and math.copysign(1.0, x) < 0:
       self.add(str(x))
       return
   This uses math.copysign to detect the negative sign on zero and writes "-0.0" as a float would print.

2. Alternatively, change the integer-branch condition so it excludes signed zero:
   - Replace the condition
       if x == int(x) and not math.isinf(x) and not math.isnan(x):
     with
       if x == int(x) and not math.isinf(x) and not math.isnan(x) and not (x == 0.0 and math.copysign(1.0, x) < 0):
     and then proceed as before; negative zero will fall through to the float branch and be written as "-0.0".

3. If you want to keep the rest of the integer-compression logic but still preserve "-0.0", you could add the negative-zero check immediately after computing value and before any modification:
   - if value == 0 and math.copysign(1.0, x) < 0:
       self.add(str(x))
       return

Pick one of these fixes and add the math import if not already present.

Why these fixes work
- They detect the distinct sign of negative zero without losing it by converting to int, and write the original float representation (str(x)) which preserves "-0.0".
