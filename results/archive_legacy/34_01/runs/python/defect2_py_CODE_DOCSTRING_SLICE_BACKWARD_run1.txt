What happened (where and what failed)
- In get_paint (lines 54–75) you clamp the input into v:
  - Line 56: v = max(value, self._lower_bound)
  - Line 57: v = min(v, self._upper_bound)
- But on line 71 you compute g using the original value, not the clamped v:
  - g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If value is outside [self._lower_bound, self._upper_bound] that calculation can produce g < 0 or g > 255, so the check at line 74 raises the ValueError.
- Also, if self._upper_bound == self._lower_bound, the division on line 71 will raise a ZeroDivisionError.

Why it happened (root cause)
- You intended to clamp the input, but you never used the clamped variable v when computing g. The calculation still uses the un-clamped value, so clamping has no effect.
- There is no protection against a zero-width range (upper == lower), so the denominator can be zero.

How to fix it (explicit steps and exact changes)
1. Use the clamped value v when computing g.
2. Guard against a zero-length range before dividing.
3. Make the error message useful (optional).

Example corrected code (showing the exact line changes):
- Replace line 71 with:
  g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- Add a check for zero-range before that line, e.g.:
  if self._upper_bound == self._lower_bound:
      # Decide desired behavior: return 0, 255, middle, or raise. Example: return middle gray
      return (127, 127, 127)
- Improve the ValueError message on line 75 to include g and bounds, for example:
  raise ValueError(f"Computed color component out of range: g={g} (expected 0..255)")

Minimal corrected function example:

    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)

        # Guard against zero division
        if self._upper_bound == self._lower_bound:
            return (127, 127, 127)  # or choose another sensible default/raise

        # Use clamped value v (was using 'value' incorrectly)
        g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

        if g < 0 or g > 255:
            raise ValueError(f"Computed color component out of range: g={g} (expected 0..255)")

        return (g, g, g)

That fixes the root cause (using the clamped input) and prevents division-by-zero.
