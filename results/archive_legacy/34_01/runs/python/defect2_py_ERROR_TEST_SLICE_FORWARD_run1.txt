What happened
- The test calls gps.get_paint(-0.5). The grayscale routine tries to turn that input into RGB channels and passes those channels to a color constructor (or validator) that enforces valid channel ranges.
- That color constructor raised the ValueError: "Color parameter outside of expected range: Red Green Blue" because at least one of the RGB channel values was outside its allowed range (negative in this case). The -0.5 input produced a negative channel, so the constructor rejected it.

Where it happened (specific names)
- Caller: gps = GrayPaintScale(); color = gps.get_paint(-0.5)
- Failing code: inside GrayPaintScale.get_paint(...) when it converts the scalar to RGB and calls the color constructor/validator (e.g., Color(r, g, b) or a library helper like matplotlib.colors.to_rgb). The negative input -0.5 is the root cause.

Why it failed (root cause)
- The function assumes inputs are within an expected interval (usually 0..1 or 0..255). It does not clamp or validate the input before creating the RGB triple.
- Passing -0.5 produces an invalid channel value (negative), so the color constructor raises the ValueError complaining that the Red, Green, or Blue parameter is out of range.

Concrete fixes (step-by-step)
1. Decide the valid input range for get_paint (pick one: 0..1 or 0..255). For tests like assert color == (0,0,0) you want negative values to map to black.
2. Add input clamping in GrayPaintScale.get_paint:
   - Example (for 0..1 expected):
     - v = max(0.0, min(value, 1.0))
     - r = g = b = v
     - return (r, g, b)  or convert to integers if the API expects 0..255
3. If the color constructor expects integer 0..255, convert after clamping:
   - v = int(round(255 * max(0.0, min(value, 1.0))))
   - return (v, v, v)
4. Add a unit test for out-of-range inputs:
   - assert gps.get_paint(-0.5) == (0,0,0)
   - assert gps.get_paint(1.5) == (255,255,255)  (or (1.0,1.0,1.0) depending on chosen convention)
5. Optionally, if you want to reject out-of-range inputs rather than clamp, validate and raise a clearer error message from get_paint, e.g. raise ValueError(f"get_paint: value {value} must be between 0 and 1").

Short actionable summary
