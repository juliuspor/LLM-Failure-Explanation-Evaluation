The bug stems from inferring the wrong 'inferred_type' using type(array) which returns list type rather than element component type; when array contains only None or is empty and element is None the inferred type becomes object. Fix by determining the component type from the array's existing elements: find the first non-None element and use its type; if none, fall back to element's type or object. Keep behavior of expected_type check same but compare against object as fallback. Use existing _copy_list_grow1 to create new list and append element.
