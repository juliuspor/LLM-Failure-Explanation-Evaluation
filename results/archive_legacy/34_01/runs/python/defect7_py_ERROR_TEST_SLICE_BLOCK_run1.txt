What happened (where and which variable)
- At line 766 you call array[i].upper(). The value of array[i] for i == 1 is None (from the test input ['Test', None, 99.0]).
- None has no method upper(), so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.

Why it happened (root cause)
- The code assumes every element in array is a string and calls .upper() unconditionally.
- The test includes a non-string (None), so that assumption breaks. The code never checks the element’s type or presence before calling upper().

Exact location in the snippet
- Line 765: for i in range(len(array)):
- Line 766: _ = array[i].upper()  <-- raises AttributeError when array[i] is None
- Line 767: classes.append(type(array[i]))  <-- never reached for that i because exception occurs on 766

What to do (concrete fixes with steps)
Pick one of these fixes depending on your intent:

1) If you only wanted to record the type of each element (no string processing needed)
- Remove the .upper() call entirely.
- Change lines 765–767 to:
  for i in range(len(array)):
      classes.append(type(array[i]))

2) If you intended to normalize only string elements before using them
- Check the element is a string before calling .upper():
  for i in range(len(array)):
      if isinstance(array[i], str):
          _ = array[i].upper()
      classes.append(type(array[i]))

3) If you intended to safely call .upper() when present (tolerate non-strings)
- Use getattr with a default or a try/except:
  for i in range(len(array)):
      _ = getattr(array[i], 'upper', lambda: None)()
      classes.append(type(array[i]))

4) If you intended to convert all elements to strings before uppercasing
- Convert None to "None" explicitly (but note this changes resulting types):
  for i in range(len(array)):
      s = str(array[i])
      _ = s.upper()
      classes.append(type(array[i]))

- For the test you showed, option 1 or 2 is appropriate. The test expects [str, type(None), float], so you must not change element types. The simplest correct change is to remove or guard the .upper() call so None doesn’t cause an exception.
