What happened (where it fails)
- At lines 396–397 the code unconditionally rejects any negative minutes:
  - Code: if minutes_offset < 0 or minutes_offset > 59: raise ValueError(...)
  - This throws ValueError whenever minutes_offset is negative, regardless of hours_offset.

Why that is wrong (root cause)
- The docstring explicitly allows certain negative-minute combinations:
  - (0, -15) -> -00:15 is valid
  - (-2, -15) -> -02:15 is valid
- The unconditional rejection at lines 396–397 contradicts the documented behavior. Because the code never accepts negative minutes, any call that intends to create a negative-offset zone with negative minutes (hours == 0 or hours < 0 and minutes < 0) will fail immediately at those lines instead of letting the later logic (lines 402, 404, etc.) adjust/convert the minutes into a total offset.

Exactly where to change it
- Replace the unconditional check at lines 396–397 with logic that:
  1. still enforces the absolute bounds of minutes (|minutes_offset| <= 59), and
  2. rejects negative minutes only when hours_offset is positive (which the docstring disallows).
- Concretely:
  - Remove or change: if minutes_offset < 0 or minutes_offset > 59: raise ValueError(...)
  - Use instead (pseudo-code using your variable names and line numbers):
    - if minutes_offset > 59 or minutes_offset < -59:
        raise ValueError(f"Minutes out of range: {minutes_offset}")  # enforces absolute bound
    - if hours_offset > 0 and minutes_offset < 0:
        raise ValueError(f"Minutes must be non-negative when hours is positive: {minutes_offset}")
  - Or equivalently:
    - if minutes_offset < 0 and hours_offset > 0: raise ValueError(...)
    - if minutes_offset > 59 or minutes_offset < -59: raise ValueError(...)

Why this fix works
- It allows the valid combinations listed in the docstring:
  - Positive hours with zero or positive minutes
  - Zero hours with positive or negative minutes
  - Negative hours with positive or negative minutes
- It still prevents out-of-range minute values (>59 or <-59) and invalid sign combinations (minutes negative when hours positive).

Extra note about later code
