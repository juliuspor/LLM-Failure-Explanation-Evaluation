- add_number calls self.add(...) (e.g. lines in your [CODE]).
- add (lines 126–138 in [SLICE_BLOCK]) calls self.append(" ") and self.append(newcode) (lines 136 and 138).

Why that happens (root cause)
- add is intended as the higher-level method that enforces spacing rules and then writes text.
- append should be the low-level routine that writes characters into the buffer without applying spacing rules.
- Instead, append calls add (or otherwise triggers add's logic) so the two functions call each other in a cycle. The cycle is triggered any time add calls append, including from add_number.
