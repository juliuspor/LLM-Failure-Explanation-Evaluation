What happened (where it fails)
- The method get_paint computes g on this line:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  but earlier you clamped to v:
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
- You never use v in the g calculation, so g is computed from the original value instead of the clamped value. If value is outside the bounds, g can be < 0 or > 255, which triggers the ValueError on the next lines:
  if g < 0 or g > 255:
      raise ValueError(...)

Why that causes the error
- The code intends to clamp value to the allowed range and then map it to 0–255. Because you computed using value (not v), the clamping step is ignored; therefore values outside the bounds produce g outside 0–255 and hit the explicit range check, raising the exception.

Also note a second possible failure mode
- If self._upper_bound == self._lower_bound the expression divides by zero. That will raise ZeroDivisionError before the ValueError check.

Concrete fixes (step-by-step)
1. Use the clamped variable v when computing g:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

2. Protect against zero range (upper == lower). Decide the intended behavior when the range has zero width—for example, return 0 or 255 or a middle value. Example:
   if self._upper_bound == self._lower_bound:
       g = 0  # or 127 or 255 depending on requirement

3. Ensure final g is clamped to [0,255] before returning (defensive):
   g = max(0, min(255, g))

4. Improve the error message if you keep the explicit check so it includes the offending values (optional).

Corrected example
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)
        
        # Avoid division by zero
        if self._upper_bound == self._lower_bound:
            g = 0  # or choose appropriate constant
        else:
            g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
        
        # Final clamp to [0,255]
        g = max(0, min(255, g))
        
        return (g, g, g)
