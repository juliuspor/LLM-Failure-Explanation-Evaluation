The original to_locale rejected the valid double-underscore form language__variant (e.g., fr__POSIX) because it always expected a two-letter uppercase country after the first underscore. I will modify parsing to first validate the two-letter lowercase language, then handle three cases: plain language (len==2), double-underscore variant (language__variant), and language_country[_variant]. The double-underscore branch will accept a non-empty variant. The country branch will validate two uppercase letters; if a variant is present it must be preceded by a single underscore and be non-empty. This preserves strict validation while accepting the empty-country variant form.
