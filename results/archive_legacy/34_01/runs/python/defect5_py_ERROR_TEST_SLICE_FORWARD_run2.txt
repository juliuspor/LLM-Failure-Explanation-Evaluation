What happened (root cause)
- In ArrayUtils.add(array, element, expected_type) the code builds a Java List (named list in the method), then returns that list converted to an array by calling list.toArray() and casting the result to expected_type[]:
  (expected_type[]) list.toArray()
- list.toArray() with no arguments returns Object[].
- Casting an Object[] to String[] (or any other concrete component type) is illegal at runtime and throws the exception you saw:
  ClassCastException: [Ljava.lang.Object; cannot be cast to [Ljava.lang.String;
- In your test you called ArrayUtils.add(None, None, expected_type=str), so:
  - array == None, element == None, expected_type == String.class (or equivalent)
  - the method created a List, added a single null element, then tried to cast the Object[] returned by list.toArray() to String[], which caused the failure.

Where it occurs
- Inside the method ArrayUtils.add(...) at the line that does the unchecked cast:
  (expected_type[]) list.toArray()
  (or equivalently: return (String[]) list.toArray();)
- Variables involved: array (the input array, here null), element (the item to add, here null), expected_type (String.class), list (the temporary java.util.List used to collect elements), and the return value (expected_type[]).

Why the cast fails (simple explanation)
- list.toArray() returns Object[].
- Java does not allow runtime casting from Object[] to String[] because the JVM tracks array component types.
- Even if every element is null, the array object is still an Object[] and cannot be reinterpreted as a String[].

How to fix it — concrete steps and code examples
Option A — use the typed toArray overload (recommended)
Replace the bad cast line with:
  return list.toArray((T[]) java.lang.reflect.Array.newInstance(expected_type, list.size()));
If you have expected_type as Class<T>:

  @SuppressWarnings("unchecked")
  public static <T> T[] add(T[] array, T element, Class<T> expected_type) {
      List<T> list = new ArrayList<>();
      if (array != null) {
          Collections.addAll(list, array);
      }
      list.add(element);
      // Create a correctly typed array and return the list contents into it
      T[] typed = (T[]) java.lang.reflect.Array.newInstance(expected_type, list.size());
      return list.toArray(typed);
  }

Why this works
- java.lang.reflect.Array.newInstance(expected_type, size) creates a real String[] (or T[]) at runtime.
- list.toArray(typed) fills and returns an array whose runtime component type is expected_type, so no ClassCastException occurs.

Option B — use Arrays.copyOf (if you already have Object[] from toArray)
If you have an Object[] result = list.toArray(); and expected_type is Class<T>:

  @SuppressWarnings("unchecked")
  T[] typed = (T[]) java.lang.reflect.Array.newInstance(expected_type, result.length);
  System.arraycopy(result, 0, typed, 0, result.length);
  return typed;

Or using Arrays.copyOf (Java 6+):

  Object[] result = list.toArray();
  T[] typed = Arrays.copyOf(result, result.length, (Class<? extends T[]>) java.lang.reflect.Array.newInstance(expected_type, 0).getClass());
  return typed;

- If expected_type is already provided (as in your test), use it to allocate the array as shown above.
