What happened
- In the "max middle" section you fetch the wrong data item:
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  That uses self._min_middle_index instead of self._max_middle_index.
- Because you compute max_middle from the min-middle item, the comparison
  if middle > max_middle:
  can be evaluated against the wrong baseline. That lets a later item (index 3 in your failing test) become marked as the max-middle when it should not, which produces the failed assertion (expected max_middle_index=1, got 3).

Where it occurs (exact place)
- In the block that starts with:
  if self._max_middle_index >= 0:
  the call to get_data_item(...) incorrectly passes self._min_middle_index. Replace that with self._max_middle_index.

Why that causes the failure (step-by-step)
1. You intended to compare the current item's middle to the stored maximum middle (the item at self._max_middle_index).
2. Because you used self._min_middle_index, you computed max_middle from the min-middle item instead.
3. If the min-middle value is lower than the true max-middle, the comparison middle > max_middle may be true when it should be false, so self._max_middle_index gets updated wrongly to the current index (3).
4. Test expected the max index to remain 1, but it was overwritten to 3 — hence the assertion failure.

Exact fix (patch)
Replace this line in the max-middle block:
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
with:
  max_middle_period = self.get_data_item(self._max_middle_index).get_period()

Suggested additional checks
- Add a unit test that exercises both min and max middle tracking with at least three items so you detect this regression.
- Optionally assert inside the block (during development) that self._max_middle_index != self._min_middle_index only when appropriate, or log values when updating to make debugging easier.

Example corrected block
(if you want the whole corrected snippet)
if self._max_middle_index >= 0:
    max_middle_period = self.get_data_item(self._max_middle_index).get_period()
    if isinstance(max_middle_period, SimpleTimePeriod):
        s = max_middle_period.get_start_millis()
        e = max_middle_period.get_end_millis()
    else:
        s = int(max_middle_period.get_start().timestamp() * 1000)
        e = int(max_middle_period.get_end().timestamp() * 1000)
    max_middle = s + (e - s) // 2
