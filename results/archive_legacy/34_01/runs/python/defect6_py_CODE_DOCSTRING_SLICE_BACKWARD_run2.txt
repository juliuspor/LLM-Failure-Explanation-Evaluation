What fails
- When this method runs it raises NameError: name 'math' is not defined at the line
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- Root cause: the code calls math.isinf and math.isnan but the math module is never imported in the file, so the name math is undefined.

Where it happens (in your code)
- In add_number(...), at the conditional that uses math.isinf and math.isnan.
- The relevant names are x, value, mantissa, exp, and the methods get_last_char() and add() called inside add_number; the error is raised before any of the integer-cleanup loop is reached because Python cannot resolve math.

Why that causes failure
- NameError is raised immediately when Python evaluates that boolean expression and encounters the unknown name math. Execution stops and the method never completes or calls self.add(...).

How to fix it (concrete steps)
1. At the top of the module (before add_number is defined) add:
   import math
   or, if you prefer only the two functions:
   from math import isinf, isnan
2. If you used the second form, update the condition accordingly:
   if x == int(x) and not isinf(x) and not isnan(x):
3. Save and rerun your code. The NameError will be gone and add_number will proceed to the integer-handling branch.

Optional safety suggestions (explicit)
- If there is any chance x can be a numpy scalar, you may want to use numpy.isinf/numpy.isnan or coerce x to a float first; otherwise math.isinf/math.isnan will work for built-in floats.
- Consider guarding int(x) with a try/except if you expect non-float inputs that int(...) might reject.
