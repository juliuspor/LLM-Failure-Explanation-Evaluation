What happened (where): on line 766 (inside to_class's loop) the code does
    _ = array[i].upper()
which runs for every element in array.

Why it fails (root cause): calling .upper() assumes array[i] is a string object. If array[i] is None (or any non-string that does not define .upper()), Python raises AttributeError: 'NoneType' object has no attribute 'upper' (or "'X' object has no attribute 'upper'"). That is exactly what the docstring warns about ("If any of these objects is None, a None element will be inserted into the array") but the current code does the opposite: it unconditionally calls .upper() and will crash before you can append a None to classes.

Where it occurs in the code:
- to_class, loop for i in range(len(array)):
  - line 766: _ = array[i].upper() — this is the direct cause of the AttributeError
  - line 767 (next): classes.append(type(array[i])) is never reached for a crashing element

Extra notes:
- The assignment to _ is unused, so the .upper() call serves no purpose except to crash on non-strings.
- If array elements can be arbitrary objects (including None), you must not call string methods without checking the element's type or None-ness first.

Concrete fixes (steps you can apply)
1. If you only need to record types and allow None elements:
   - Remove the .upper() call.
   - Append None when the element is None, otherwise append type(elem).

   Example replacement loop:
       for i in range(len(array)):
           elem = array[i]
           if elem is None:
               classes.append(None)
           else:
               classes.append(type(elem))

2. If you intended to verify that elements are strings and normalize them:
   - Check the element type before calling .upper(), and handle non-strings explicitly.

   Example:
       for i in range(len(array)):
           elem = array[i]
           if elem is None:
               classes.append(None)
           elif isinstance(elem, str):
               _ = elem.upper()   # safe to call
               classes.append(str)
           else:
               raise TypeError(f"array[{i}] must be str or None, got {type(elem).__name__}")

Pick the approach that matches the intended behavior. The immediate fix to stop the AttributeError is to remove or guard the .upper() call (line 766) and explicitly handle None (and non-string) elements before calling any string methods.
