What happened
- The code that inserts a separator space (lines 132–137) is inserting a space between a minus sign and the following digits. That turns a token like "-0.0" into "- 0.0". The parser later interprets that as a positive zero (0) instead of a negative zero (-0.0), so the assertion "Expected -0.0, got 0" fails.

Why it happens (root cause)
- The decision to insert a space lives in add(newcode) at line 133:
  if (self.is_word_char(c) or c == '\\') and self.is_word_char(self.get_last_char()):
- That code assumes characters classified as "word chars" should be separated when adjacent.
- The bug is that is_word_char('-') returns True (or otherwise treats '-' as a word character). When the existing last character is '-' and the next character c is a digit (e.g. '0'), the condition becomes true and add() appends a space (line 136). So "-0.0" becomes "- 0.0" before being consumed by the parser/consumer, and the sign information is lost.

Where it occurs
- Method: add(self, newcode)
- Lines: 132–137 (the check that decides to append " ")
- Relevant variables/methods: newcode, c = newcode[0], self.get_last_char(), self.is_word_char(...), self.append(" ")

How to reproduce quickly

- Observe the stored string contains "- 0.0" (space inserted between '-' and digits), and downstream code produces 0 instead of -0.0.
