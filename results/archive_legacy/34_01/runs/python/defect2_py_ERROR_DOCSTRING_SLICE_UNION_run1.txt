What happened (where it fails)
- The failure comes from get_paint at line 71 and the check at lines 74–75:
  - Line 71 computes g:
    g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  - Line 74 checks:
    if g < 0 or g > 255:
        raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- The code raises the ValueError because the computed g is outside 0–255. That is, the expression for g produced a number < 0 or > 255, so the guard at line 74 triggered and raised the error at line 75.

Why it happened (root causes)
- value outside the expected range:
  - If value is less than self._lower_bound or greater than self._upper_bound, the normalized ratio (value - lower) / (upper - lower) will be < 0 or > 1, so multiplying by 255 produces g < 0 or g > 255. In that case get_paint deliberately raises the ValueError at line 75.
- denominator problem (wrong bounds ordering):
  - If self._upper_bound <= self._lower_bound, the denominator (self._upper_bound - self._lower_bound) is zero or negative:
    - If it is zero, the expression will raise ZeroDivisionError (not the ValueError shown). If you are not seeing ZeroDivisionError, your _upper_bound and _lower_bound are probably not equal.
    - If it is negative (upper < lower), the ratio sign is inverted and g can easily fall outside 0–255 for valid values, causing the ValueError on line 75.
- Misleading error message:
  - The raised message "Color parameter outside of expected range: Red Green Blue" does not include which variable failed, nor the actual numbers. That makes diagnosing harder.
