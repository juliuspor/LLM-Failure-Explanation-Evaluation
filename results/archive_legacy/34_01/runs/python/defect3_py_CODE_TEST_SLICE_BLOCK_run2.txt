Why it failed (root cause)
- The code advances pos incorrectly after a translator reported it consumed code units.
- Specifically, inside the while loop you do:
  for pt in range(consumed):
      pos += char_count(code_point_at(code_units, pos))
- That calls code_point_at(code_units, pos) while pos may already be at or beyond length (because you increment pos inside the loop and do multiple iterations). As a result code_point_at receives an invalid index and raises the IndexError at lines 35–36.
- In short: you call code_point_at with a bad pos because the loop mutates pos and uses it to compute the next increment, instead of using the initial codepoint or simply adding the known number of code units consumed.

How to fix it (concrete steps)
You have two correct, explicit options depending on what consumed represents:

Option A — If consumed is the number of UTF‑16 code units consumed by the translator:
- Replace the entire for loop with a single increment by consumed:
    pos += consumed
- Rationale: consumed already counts code units; advancing pos by consumed moves to the next unread code unit without calling code_point_at.

Option B — If you want to advance by codepoints (using char_count of the codepoint at the start):
- Read the codepoint once at the starting pos, compute its char_count, then advance by that:
    codepoint = code_point_at(code_units, pos)
    pos += char_count(codepoint)
- Do not call code_point_at repeatedly while mutating pos.
