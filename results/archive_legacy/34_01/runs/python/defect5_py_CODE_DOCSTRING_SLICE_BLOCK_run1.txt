What failed
- Root cause: on line 2 you call inferred_type = type(array). That returns the Python/Java class of the list object itself (e.g. list / java.lang.Object[] / [Lmy.pkg.MyClass;) rather than the component (element) type you need. The rest of the code and ArrayUtils._copy_list_grow1 expect a component/element type, not the list object type.
- Where it happens: when array is not None (any non-empty or empty list), line 2 sets inferred_type incorrectly. That wrong inferred_type then flows into ArrayUtils._copy_list_grow1 (call at new_list = ArrayUtils._copy_list_grow1(array, inferred_type)), and later to the type-check on lines 346–351. Because you passed the list class rather than an element class, the copy routine and the later type checks can produce a ClassCast/TypeError (or fail to detect the actual element-type mismatch).
- Observable symptom: the function raises an unexpected TypeError/ClassCast-like message or fails to allow valid element additions because inferred_type does not represent the element type. The special-case check on lines 346–351 (inferred_type == object) will also not behave as intended because inferred_type will rarely be object; instead it will be a list/array class.

Why that is wrong (concrete example)
- Suppose array is a List[int] (or a Java Integer[]). type(array) is list (or Integer[] runtime class), not int (or Integer). Passing the list/array class to _copy_list_grow1 makes the copy routine think the component type is the list class, so adding an int/Integer element later will either be rejected or produce an incorrect cast message. If array is empty, you should look at the element argument to infer the component type — not the list object itself.

How to fix (step-by-step)
1. Use the element type, or the element type from the existing array's contents, not type(array).
   - If array is not None and has elements, infer the component type from an element: inferred_type = type(array[0]) (or examine element types consistently).
   - Else if element is not None, inferred_type = type(element).
   - Else inferred_type = object.
2. Handle empty arrays specially: an empty array gives no element to inspect, so fall back to the element argument. If both are missing/None, use object.
3. Keep the existing expected_type check but it will now behave as intended because inferred_type will be an element/component type.
