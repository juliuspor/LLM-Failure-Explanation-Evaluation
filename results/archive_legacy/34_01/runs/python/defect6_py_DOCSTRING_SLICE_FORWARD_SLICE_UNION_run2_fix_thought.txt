The bug arises because add_number calls self.add which eventually calls self.append; if a subclass lacks append an AttributeError occurs. To avoid relying on append, implement add_number to directly manipulate the internal code buffer (self.code) while still honoring maybe_end_statement behavior and spacing rules. The new implementation calls maybe_end_statement(), inspects the previous character via get_last_char(), handles spacing for negative numbers, computes integer formatting with mantissa/exp logic, and finally appends the resulting string directly into self.code.
