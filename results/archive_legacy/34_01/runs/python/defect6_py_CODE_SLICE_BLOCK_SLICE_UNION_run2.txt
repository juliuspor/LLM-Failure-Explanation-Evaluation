What happens
- The code goes into an infinite loop when x is 0.0 (so value == 0).
- In add_number you set value = int(x) and mantissa = value (line numbers as in your snippet).
- Then you enter the loop at:
  while mantissa % 10 == 0:
Because mantissa is 0, mantissa % 10 == 0 is true, and mantissa //= 10 leaves mantissa at 0. The computed check_val is also 0, so the loop body always takes the mantissa //= 10; exp += 1 branch and never changes mantissa or makes the condition false. That produces an infinite loop and the program “hangs”.

Where it occurs
- In add_number(), inside the block starting at "if abs(x) >= 100:" — specifically the while mantissa % 10 == 0: loop.
- The symptom is the program never returns from add_number when x == 0.0 (value == 0).

Why it happens (root cause)
- For mantissa == 0, mantissa % 10 == 0 is true, mantissa //= 10 keeps mantissa at 0, and check_val == value stays true. There is no code that breaks the loop for the zero case, so exp increments indefinitely and the loop never terminates.

Concrete fixes (steps you can apply)
1. Short-circuit zero before the loop:
   - Right after value = int(x), add:
     if value == 0:
         self.add("0")
         return
   This avoids entering the reduction logic at all for zero.

2. Or change the while condition to prevent iterating on zero:
   - Replace the loop header with:
     while mantissa != 0 and mantissa % 10 == 0:
   That keeps the existing behavior for other integers but exits immediately for mantissa == 0.

Either fix addresses the infinite loop. The simplest, safest change is option 1 (handle value == 0 immediately), because it makes the intent explicit and avoids surprising behavior for zero.
