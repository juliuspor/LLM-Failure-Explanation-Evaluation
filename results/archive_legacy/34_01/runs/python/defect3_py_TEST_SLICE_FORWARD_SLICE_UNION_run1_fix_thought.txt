The translate bug arises from inconsistent construction of UTF-16 code units. Rebuilding code_units using a reliable utf-16-le encoding ensures surrogate pairs are represented as two 16-bit units. The new translate implementation encodes the input string to UTF-16 little-endian, converts bytes to 16-bit code units, and then proceeds with the previous translation loop unchanged (preserving behavior). This fixes mismatches between to_utf16_code_units and consumers without modifying other helper functions.
