What happened (root cause)
- At line 766 you call array[i].upper(). That assumes array[i] is a string (has an upper() method).
- The exception AttributeError: 'NoneType' object has no attribute 'upper' means array[i] was None for at least one index. Calling .upper() on None fails.
- The problem occurs inside the loop at lines 765–767:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()    ← raises when array[i] is None
  - 767:     classes.append(type(array[i]))

Why this happened (specific)
- The code never checks for None elements inside the input list array. The method only checks if array itself is None (line that returns None) and if it is empty; it doesn’t guard against entries inside array being None.
- Because array contains at least one None element, calling .upper() on that element triggers the AttributeError.

How to fix it (concrete steps and code)
Choose one of these fixes depending on intent.

1) If you only wanted the classes and the .upper() call is accidental or unnecessary:
- Remove the .upper() call entirely.
- Replace the loop with:

    for i in range(len(array)):
        classes.append(type(array[i]))
    return classes

Or more Pythonic:

    return [type(x) for x in array]

2) If you intended to call .upper() on strings but want to skip None elements:
- Check each element before calling .upper():

    for i in range(len(array)):
        if array[i] is None:
            classes.append(type(None))
            continue
        _ = array[i].upper()   # safe now because not None
        classes.append(type(array[i]))
    return classes

3) If you intended to validate that every element is a string and raise a clearer error:
- Validate and raise ValueError with context:

    for i, v in enumerate(array):
        if v is None or not isinstance(v, str):
            raise ValueError(f"array[{i}] must be a str, got {type(v).__name__}")
        _ = v.upper()
        classes.append(type(v))
    return classes

4) If you intended to treat None as a legitimate value but do not need .upper():
- Use explicit handling for None and preserve its type:

    for v in array:
        if v is None:
            classes.append(type(None))
        else:
            classes.append(type(v))
    return classes

Summary
- The error is raised because array[i] is None and .upper() is being called at line 766.
- Fix by removing or guarding the .upper() call, or by validating elements before calling methods on them.
