What happened (where it failed)
- The failure happens inside add_number(), in the integer branch beginning at the line:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- For x == -0.0 that condition is True (because -0.0 == int(-0.0) == 0).
- Then you do value = int(x) which produces 0 (losing the negative sign).
- Later you do self.add(str(value)) which adds "0". The test expected "-0.0", so AssertionError: Expected -0.0, got 0.

Why it happens (root cause)
- IEEE floating-point has a negative zero value (-0.0) that compares equal to 0.0 and to the integer 0. The conversion int(-0.0) yields 0 with no sign.
- Your code detects negativity with x < 0 earlier, but -0.0 < 0 is False (so that check does not catch negative zero).
- Thus the code reaches the integer branch and converts -0.0 to the unsigned integer 0, losing the sign information.

Exactly where to change (lines and variables)
1. The problematic test and conversion:
   - if x == int(x) ...  (the branch that starts the integer handling)
   - value = int(x)      (this line discards the negative-zero sign)
2. The earlier attempt to handle a leading '-':
   - prev = self.get_last_char()
   - if x < 0 and prev == '-': ...
   That x < 0 check does not detect -0.0.

Concrete fixes (step-by-step)
Option A — special-case negative zero before integer branch (recommended):
1. After computing prev = self.get_last_char(), add a check:
   if x == 0.0 and math.copysign(1.0, x) < 0:
       # preserve negative zero as "-0.0" (or whatever format you expect)
       self.add("-0.0")
       return
2. Then keep the rest of add_number unchanged. This ensures -0.0 never reaches the int conversion.

Option B — detect and preserve sign when converting to int:
1. Before value = int(x), detect negative zero:
   if x == 0.0 and math.copysign(1.0, x) < 0:
       # choose how you want to print it; if you need "-0.0" use:
       self.add("-0.0")
       return
2. Otherwise proceed with value = int(x).

Option C — use string conversion that preserves sign:
1. For the non-exponential branch, instead of value = int(x) then str(value), use:
   s = repr(x)  # or format(x, ...) that preserves sign for -0.0
   self.add(s)
2. But be careful: repr(x) will give "-0.0" for -0.0 and also may produce "1.0" for integers; choose format consistent with your intended pretty-printing.
