What happened
- At runtime you tried to cast an Object[] to a String[]. The JVM error is telling you exactly that: an array whose runtime component type is Object (Object[]) cannot be cast to an array whose component type is String (String[]).
- In your function the variables from the docstring are: array, element, expected_type. The failure occurs where the code does something like:
  (String[]) array.toArray()
  or
  (String[]) someObject /* where someObject is actually an Object[] */.
  In other words: you called toArray() (or otherwise produced an Object[]) and then cast that result to a more specific array type.

Why it happened (root cause)
- Java arrays are covariant but the runtime type matters. toArray() with no argument returns Object[]. If the original list contents were heterogeneous or the list variable type was List<Object> or the element was None/Null and you used reflective creation that produced Object[], the runtime component type is Object.
- You cannot cast an Object[] to a String[] even if every element is a String at that moment. The JVM enforces the runtime component type of the array, not just the declared generic type.
- That is why you get: ClassCastException: [Ljava.lang.Object; cannot be cast to [Ljava.lang.String; which the Python wrapper reports as a TypeError.

Where it occurs in your code
- The cast happens at the line where you convert the list to an array and immediately cast its result to a specific array type. Look for code like:
  - array.toArray() cast to (String[]) or (T[]) or
  - (String[]) Arrays.copyOf(someArray, newLength) when someArray is Object[]
  - or Array.newInstance usage where the component type passed in was Object.class
- Specifically check the line that returns the new array/list. That’s the line throwing the ClassCastException.

How to fix it (step-by-step, concrete)
1. If you know the target element type (String), create the correctly typed array when calling toArray:
   - Use toArray(T[] a): list.toArray(new String[list.size() + 1])
   - Example: String[] result = list.toArray(new String[list.size() + 1]);
   This ensures the returned array’s runtime component type is String.

2. If you are creating a new array by copying an existing array, ensure the source array’s runtime type is correct:
   - Use Arrays.copyOf(original, newLength, String[].class)
   - Example: String[] copy = Arrays.copyOf((String[]) original, original.length + 1, String[].class);
   But only do that if original is actually String[] at runtime; otherwise construct a new String[] and copy elements manually.

3. If you must handle unknown element types (expected_type may be None):
   - Use reflection to create an array with the intended component type:
     Object newArray = java.lang.reflect.Array.newInstance(componentClass, newLength);
     Then copy elements into newArray and return it as Object[] or cast to T[] only when you can guarantee the componentClass.
   - If element is null and expected_type is not provided, leave the result as Object[].

4. Simpler and safer: operate on Lists instead of arrays:
   - Create a new ArrayList<>(originalList); newList.add(element); return newList;
   - Avoids array casting issues and is idiomatic if you don’t need a plain array.

Checklist to fix your specific error
- Find the line casting the result of toArray() (or Arrays.copyOf / Array.newInstance) to String[] — change it to toArray(new String[...]) or construct with the right component type.
- If element can be null and expected_type is None, accept returning Object[] (don’t cast to String[]).
- If you must return T[] and expected_type is supplied, use Array.newInstance(expected_type, size) to create the result array, then copy elements.
