Why this is the root cause (clear, simple)
- Both inputs are None, so the routine cannot determine a concrete element type and uses object as the inferred_type placeholder.
- The code treats object as “I cannot cast this to a typed list” and refuses to continue when the caller requested a concrete expected_type (str). The method therefore raises instead of accepting a None element into a str-typed list.
