What went wrong (root cause)
- At line 336 you set inferred_type = type(array) when array is not None. type(array) returns the Python class of the container (e.g. list), not the element/component type of the list. That is the wrong thing to pass to ArrayUtils._copy_list_grow1 and the wrong thing to compare to object later.
- Because inferred_type becomes the list class (e.g. list), the check at lines 346–350:
  if inferred_type == object and expected_type != object:
never fires for the common case where you meant “unknown element type” — inferred_type is list, not object — so the code proceeds using the wrong type value.
- Passing the container class (list) instead of the element type into ArrayUtils._copy_list_grow1 causes the copy/grow operation to create/expect the wrong array component type. That leads to a runtime failure (ClassCastException / TypeError) when you later assign element into new_list (line 344) or when the array-copy routine tries to allocate/convert arrays.
