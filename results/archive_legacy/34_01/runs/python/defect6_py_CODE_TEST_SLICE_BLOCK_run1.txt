What happened (root cause)
- The function add_number treats -0.0 as an integer 0 and thereby loses the negative sign.
  - The early check uses if x < 0 to decide whether to insert a separating space when the previous character is '-'. For x == -0.0, the expression x < 0 is False in Python, so that branch is skipped.
  - Later the code checks if x == int(x) and then does value = int(x). For x == -0.0 this becomes value = 0. Converting to int removes the negative sign, so self.add(str(value)) appends "0".
- Result: the output becomes "0" rather than the expected "-0.0". The negative zero sign was never detected (and was lost by int()).

Where it occurs (specific names/places)
- In add_number:
  - The test if x < 0 fails for x == -0.0, so you don't treat it as negative when you intended to.
  - The integer branch (value = int(x); self.add(str(value))) converts -0.0 to 0 and loses the sign.
- The add() method (lines 132–138 in your slice) only affects spacing decisions; the bug is in how add_number detects negative-ness and when it converts to int.

How to fix it (concrete steps and code)
Option A — treat negative zero as special (recommended minimal change)
1. Detect negative zero using math.copysign or same-sign test before converting to int.
2. For the integer-handling branch, do not convert -0.0 to int (or preserve the sign there).

Change these checks in add_number:

- Replace the x < 0 check with a sign test that preserves negative zero:

    import math
    is_negative = math.copysign(1.0, x) < 0

    prev = self.get_last_char()
    if is_negative and prev == '-':
        self.add(" ")

- Prevent integer branch from converting -0.0 to int (add a guard):

    if x == int(x) and not math.isinf(x) and not math.isnan(x) and not (x == 0.0 and is_negative):
        ...

- Optionally, explicitly output negative zero as "-0.0" when appropriate:

    else:
        # for non-integer numbers or negative zero we want the original float string
        self.add(str(x))

Concrete minimal snippet showing the checks together:

    import math
    def add_number(self, x: float):
        is_negative = math.copysign(1.0, x) < 0
        prev = self.get_last_char()
        if is_negative and prev == '-':
            self.add(" ")

        # don't treat negative zero as an integer 0
        if x == int(x) and not math.isinf(x) and not math.isnan(x) and not (x == 0.0 and is_negative):
            value = int(x)
            ...
            self.add(str(value))
        else:
            self.add(str(x))

Option B — simpler: special-case negative zero up front
1. At the top of add_number, add:

    if x == 0.0 and math.copysign(1.0, x) < 0:
        # make sure negative zero prints as -0.0
        self.add("-0.0")
        return

Either option preserves the negative sign for -0.0 and prevents converting it to integer 0.

Why this fixes it
- math.copysign(1.0, x) < 0 reliably detects the negative sign bit for -0.0 even though x < 0 is False. Preventing conversion of -0.0 to int preserves the negative sign and lets str(x) produce "-0.0" (or you explicitly output "-0.0").
