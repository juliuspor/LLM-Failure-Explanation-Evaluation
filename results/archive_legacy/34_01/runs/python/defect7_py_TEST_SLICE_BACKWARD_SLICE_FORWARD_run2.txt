What happened (where and why)
- The loop in ClassUtils.to_class iterates over array (lines 765–767). On each iteration it does:
  - line 766: _ = array[i].upper()
  - line 767: classes.append(type(array[i]))
- With your input ['Test', None, 99.0], the first iteration (i = 0) works. On the second iteration (i = 1) array[1] is None. None has no .upper() method, so line 766 raises an AttributeError: "'NoneType' object has no attribute 'upper'".
- Because the exception is raised before line 767 runs for that element, the function never reaches classes.append(type(array[1])) and the test fails.

Root cause
- The code assumes every element in array is a string (calls .upper() unconditionally), but array contains non-strings (None and a float). Calling .upper() on a non-string raises an exception.

Concrete fixes (step-by-step)
Option A — Only call .upper() for strings
1. Change the loop to check the element type before calling .upper():

         if isinstance(array[i], str):
             _ = array[i].upper()
2. Leave line 767 unchanged (it will append the type for every element).
3. Result: .upper() runs only on strings; types are appended for all elements.

         if array[i] is not None:

Option C — Use try/except to skip non-string objects

         try:
             _ = array[i].upper()

- If you only want to uppercase actual strings and treat other items differently, use Option A.
- If you want a fallback by converting non-None values to string, use Option B.
