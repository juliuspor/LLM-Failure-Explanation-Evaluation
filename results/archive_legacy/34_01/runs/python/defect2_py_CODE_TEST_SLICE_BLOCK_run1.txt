What happened
- In get_paint you clamp the input into v (lines 2–4), but you never use v when computing g.
- You compute g from the original value (line with g = int(...)), so when value is -0.5 the calculation produces a negative g and the check at lines 74–75 raises ValueError.

Where it failed (exact spots)
- Method: get_paint
- Variables: value (original), v (clamped), g (computed color component)
- Faulty line: the g assignment uses value instead of v.
- Failure point: the ValueError raised at lines 74–75 when g is outside 0..255.

Why that causes the test to fail
- Test calls gps.get_paint(-0.5). The code sets v = max(-0.5, self._lower_bound) so v becomes the lower bound (e.g. 0). But because g is computed from value = -0.5, g becomes negative. The subsequent check sees g < 0 and raises ValueError, so the function never returns (0,0,0) as the test expects.

Exact fix (concrete steps)
1. Use the clamped variable v when computing g:
   Replace
     g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
   with
     g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

2. Guard against division by zero if lower and upper bounds are equal:
   - If self._upper_bound == self._lower_bound, decide and return either 0 or 255 (or some defined value). For example return (0,0,0) or raise a clear error.

3. Optionally clamp g to [0,255] after conversion to int to avoid off-by-one rounding issues:
   - g = max(0, min(255, g))

Minimal corrected code snippet
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)
        denom = (self._upper_bound - self._lower_bound)
        if denom == 0:
            # decide behavior; here return black
            return (0, 0, 0)
        g = int((v - self._lower_bound) / denom * 255.0)
        g = max(0, min(255, g))
        return (g, g, g)

That change ensures gps.get_paint(-0.5) uses the clamped value and returns (0, 0, 0) instead of raising ValueError.
