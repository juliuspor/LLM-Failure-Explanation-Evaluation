Empty-country locale strings like 'fr__POSIX' were rejected because the original code always treated positions 3 and 4 as country letters. The fix detects the empty-country encoding by checking for two consecutive underscores at positions 2 and 3 and handles the variant starting at position 4. For non-empty country, it validates that positions 3 and 4 are uppercase letters and proceeds as before. The function preserves previous validations and returns None for None input.
