Translators in this codebase (e.g., CsvEscaper) return the number of UTF-16 code units they consumed (they return 2 for a surrogate pair). The original translate() method treated the return value as a count of codepoints and incremented position by calculating char_count(code_point_at(...)) repeatedly, which caused incorrect pos advancement and IndexError for non-BMP characters. Simplest robust fix: treat the consumed value as number of UTF-16 code units and advance pos by consumed directly when consumed>0. Keep other behavior unchanged.
