translate previously used to_utf16_code_units which produced wrong values for supplementary characters. Instead, translate will construct UTF-16 code units reliably by encoding the input string to utf-16-le bytes and splitting into 16-bit units. The rest of the algorithm remains unchanged: repeatedly call translate_codepoint and handle consumed==0 by copying the codepoint. This avoids dependency on the buggy helper and ensures surrogate pairs are represented as two 16-bit units.
