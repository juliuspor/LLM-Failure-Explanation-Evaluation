- The actual value at failure was minutes_offset == -15, so the code saw a negative minute value and raised the ValueError.

Why it happened (root cause)
- The code applies a sign (negative) to minutes_offset before the range check runs. That means when parsing a signed timezone like "-02:15" the parser set minutes_offset to -15 and then immediately checked whether minutes_offset is between 0 and 59. Because -15 < 0, the check fails and the ValueError is raised.
- Lines later in the same function (402 and 404) adjust minutes_offset based on hours (they show code that manipulates minutes_offset: minutes_offset = hours_in_minutes - minutes_offset and minutes_offset = hours_in_minutes + minutes_offset). Those later lines indicate the function expects to combine hour and minute pieces with a sign, but the sign is being applied too early relative to the validation at 396–397.
