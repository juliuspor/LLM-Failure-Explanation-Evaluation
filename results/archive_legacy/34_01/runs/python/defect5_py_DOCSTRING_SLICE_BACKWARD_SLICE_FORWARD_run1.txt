What happened (root cause)
- At line 337 the code enters the branch for "element is not None".
- But at line 340 the code incorrectly sets inferred_type = object even though the element is present.
- Later, at line 346 the code checks if inferred_type == object and expected_type != object and raises a TypeError (line 347). Because inferred_type was wrongly set to object, that check is true whenever an expected_type other than object was passed in, so the code raises a TypeError even though there is a concrete, non-None element that should have defined the type.

Why that fails (plain explanation)
- The code treats a real element as if it were None. That causes the routine to think the element's type is unknown (object) and then to reject any non-object expected_type as a mismatch.
- The bug is a logic/assignment error: inferred_type should be derived from the actual element when element is not None, not forced to object.

Where it occurs (precise lines)
- Entry to the wrong assignment: line 337 (branch condition) and line 340 (inferred_type assignment).
- The erroneous check that raises the exception: lines 346–347.

How to reproduce quickly
- Call the function with element != None and expected_type set to something specific (not object). For example:
  - array = [1, 2]; element = 3; expected_type = int
  - The code will raise a TypeError at line 347 even though element is an int.

Exact minimal fixes (step‑by‑step)
1. On line 340, set inferred_type from the element, not to object. Replace:
   inferred_type = object
   with:
   inferred_type = type(element)   # or element.__class__
2. Ensure the "object" assignment is used only in the branch where element is None. If there is a branch later for element is None, it should set inferred_type = object there.
3. Optionally, tighten the mismatch check on lines 346–347 to only run when expected_type is provided (if expected_type may be None). For example:
   if inferred_type == object and expected_type is not None and expected_type != object:
       raise TypeError(...)

Example patch (contextual—apply to the shown lines)
- Replace the block around lines 337–347 with:
  if element is not None:
      inferred_type = type(element)
  else:
      inferred_type = object
  if inferred_type == object and expected_type is not None and expected_type != object:
      raise TypeError(...)

Why this fix works
- When element is present you now infer its actual type (so expected_type checks will compare to the real type).
- When element is None you keep inferred_type == object and only fail when an explicit incompatible expected_type is provided.

That directly fixes the incorrect assignment at line 340 that caused the false TypeError at line 347.
