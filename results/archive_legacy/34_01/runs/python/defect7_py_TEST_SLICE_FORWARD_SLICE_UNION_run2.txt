What happened (where and when)
- The failure happens inside ClassUtils.to_class when processing the input array ['Test', None, 99.0].
- Specifically, the code at line 766 calls array[i].upper() for every element (see SLICE_UNION lines 765–767). That call runs before the append on line 767.
- When i reaches the second element (None) or the third element (99.0), calling .upper() on a NoneType or a float raises an AttributeError because None and float do not have an .upper() method. The exception prevents execution from reaching line 767 where classes.append(type(array[i])) would run.

Why it fails (root cause)
- The code assumes every element in array is a string and unconditionally calls the string method .upper() on array[i] (line 766).
- The actual input contains non-string values (None and 99.0), so the assumption is false and .upper() raises an error.
- The type-checking (appending type(array[i]) at line 767) is never reached because the exception is thrown first.

Exactly where to look in the source
- Line 766: array[i].upper() — this is the immediate source of the exception.
- Line 767: classes.append(type(array[i])) — this is where the desired behavior (collecting types) would occur if the .upper() call did not fail.

Concrete fixes (step-by-step)
1. Only call .upper() on strings:
   - Replace the unconditional call with a guarded call, for example:
     - if isinstance(array[i], str): array[i] = array[i].upper()
     - then proceed to classes.append(type(array[i]))
2. Or catch the AttributeError around the .upper() call:
   - try:
       array[i] = array[i].upper()
     except AttributeError:
       pass
   - then append type(array[i])
3. A simpler approach: avoid modifying array elements at all if you only need types:
   - Remove the .upper() line entirely and just do classes.append(type(array[i])) for each element.
4. If canonicalizing strings is required, handle only str values and leave others untouched:
   - if isinstance(array[i], str):
       normalized = array[i].upper()
       classes.append(str)  # or append(type(normalized))
     else:
       classes.append(type(array[i]))

Which fix to choose
- If the method’s purpose is only to return the classes/types (as the test expects [str, type(None), float]), use step 3 or 4: do not call .upper() on non-strings.
- If you must normalize string values before type collection, use step 1.
