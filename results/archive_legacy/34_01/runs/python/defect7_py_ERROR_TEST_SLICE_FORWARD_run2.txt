What happened (root cause)
- In ClassUtils.to_class you iterate over the input list named array and at some point you call array[i].upper().
- The test passes None as the second element (array[1] is None). None has no .upper() method, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- That exception happens before you reach the append on line 767 (the code path that does classes.append(type(array[i]))). In other words, the .upper() call on the None value breaks the loop earlier.

Where it occurred
- Variable: array
- Index: i (the second element in the test, i == 1)
- Line(s): the call to array[i].upper() (in the to_class loop), which executes before the append at line 767

Why it happened
- The code assumes every element in array is a string (so .upper() exists). The test gives a None, so that assumption is false and the method call fails.

Concrete fixes (explicit steps and exact checks)
1. Guard against None before calling .upper():
   - Replace any direct calls like array[i].upper() with a check for None first.

Example change (explicit replacement)
- Current (problematic):
  if array[i].upper() == 'NONE':
      classes.append(type(None))
  # ... later ...
  classes.append(type(array[i]))  # line 767

- Replace with:
  value = array[i]
  if value is None:
      classes.append(type(None))
      continue
  if isinstance(value, str) and value.upper() == 'NONE':
      classes.append(type(None))
      continue
  # handle string class names or other values here
  classes.append(type(value))  # equivalent to existing line 767

2. Alternatively, convert only if the value is a string:
   - Use isinstance(value, str) before calling .upper().

3. If you need to treat empty/whitespace strings as None, trim and then check:
   value = array[i]
   if value is None:
       classes.append(type(None))
       continue
   if isinstance(value, str):
       v2 = value.strip().upper()
       if v2 == 'NONE':
           classes.append(type(None))
           continue
       # handle other string cases

Why these fixes work
- They avoid calling .upper() on a NoneType by explicitly checking the type/value first, so you only call .upper() on actual strings. After those checks you can safely call classes.append(type(value)) as your code already does on line 767.
