- At to_locale (start around line 188) the code tries to parse a language, country and variant using fixed index positions.
- At lines 237–240 it does:

Why it fails (root cause)
- For "fr__POSIX" the bytes are indexed as:
  - [0]='f', [1]='r', [2]='_', [3]='_', [4]='P', ...
- ch3 is the second underscore ('_'), which is not between 'A' and 'Z', so the check on line 239–240 triggers and raises ValueError.
- The code assumes positions 3 and 4 always contain two uppercase letters for a country code. That assumption is false when the country is empty and the string uses the double-underscore form "language__variant" (language '_' '' '_' variant), which is exactly "fr__POSIX".
