What happened
- The function is parsing locale_str at lines 239–247 and expects a two-letter language, then an underscore, then a two-letter country (positions 3–4), optionally another underscore and a variant.
- The failing input is "fr__POSIX". That string has two underscores in a row: language "fr", empty country, and variant "POSIX".
- At line 239 the code reads ch3 and ch4 (the characters at locale_str[3] and locale_str[4]) and checks
  if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
which rejects any character that is not an uppercase A–Z.
- For "fr__POSIX", locale_str[3] == '_' so the condition is true and the code raises ValueError("Invalid locale format: fr__POSIX"). That is exactly the error you see.

Why this is the root cause (no guessing)
- The code assumes a non-empty uppercase 2-letter country code at positions 3 and 4. It does not handle an empty country (double underscore) or lowercase letters.
- The branch that would parse a variant (lines 245–247) is never reached because the earlier check already raised the ValueError for the underscore at position 3.
