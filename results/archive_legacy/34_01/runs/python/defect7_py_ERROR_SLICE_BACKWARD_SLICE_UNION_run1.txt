What happened (where)
- The error happened on line 766: `_ = array[i].upper()`.
- At that moment `array[i]` was None, so `array[i]` is a NoneType and has no `.upper()` method. Python raises AttributeError: 'NoneType' object has no attribute 'upper'.

Why it happened (root cause)
- The code assumes every element in the list `array` is a string (has `.upper()`), but at least one element is `None`.
- That could be because:
  - `array` was built from external input that can include missing values,
  - some earlier code appended None into `array`,
  - or a function that should return a string returned None for some cases.
- The failure is deterministic: when the loop reaches the index i where `array[i] is None`, calling `.upper()` fails.

Concrete ways to fix it (choose one and follow the steps)
1) Reject None early and raise a clear error
- At the start of the function or before the loop, check for None and raise a descriptive exception.
  Steps:
  - Add:
    ```
    for j, v in enumerate(array):
        if v is None:
            raise ValueError(f"array[{j}] is None; expected a string")
    ```
  - This tells you exactly which index is bad instead of failing later.

2) Skip None values explicitly
- If None values are acceptable and should be ignored:
  Steps:
  - Replace the loop with:
    ```
    for i in range(len(array)):
        if array[i] is None:
            continue
        _ = array[i].upper()
        classes.append(type(array[i]))
    ```
  - This leaves None entries out of processing.

3) Coerce items to strings before calling .upper()
- If you want to treat None as an empty string (or string "None"):
  Steps:
  - Replace the call with:
    ```
    s = "" if array[i] is None else array[i]
    _ = str(s).upper()
    classes.append(type(array[i]))
    ```
  - Or explicitly convert all items first: `array = ["" if v is None else str(v) for v in array]`

4) Validate input where `array` is created
- Fix the source of None values so that `array` contains only strings.
  Steps:
  - Track back to where `array` is filled and add checks there (e.g., don't append None, or append a default string).
  - Add unit tests that include missing/None cases.

Suggested quick debug step (to find offending indexes)
- Add this before line 766:
  ```
  for j, v in enumerate(array):
      if v is None:
          print("Found None at index", j)
  ```
- That identifies exactly where the None came from.

Summary
- Root cause: calling `.upper()` on a None value at line 766 because `array[i]` is None.
- Fix by (a) validating/rejecting None early, (b) skipping None, (c) coercing to string, or (d) fixing the producer of `array`.
