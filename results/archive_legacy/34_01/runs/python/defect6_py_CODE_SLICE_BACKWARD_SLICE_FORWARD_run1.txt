What happens
- The code enters an infinite loop when x is 0. That makes add_number hang (never returns) and the program appears to "fail".

Why it happens (step-by-step, pointing to the exact variables and lines)
1. In add_number, when x is an integer and abs(x) >= 100 the code tries to remove trailing zeros by:
   - setting mantissa = value (value == int(x))
   - running while mantissa % 10 == 0:
       check_val = (mantissa // 10) * (10 ** (exp + 1))
       if check_val == value: mantissa //= 10; exp += 1
2. If x == 0 then value == 0 and mantissa == 0.
3. The while condition mantissa % 10 == 0 is true for mantissa == 0.
4. check_val becomes (0 // 10) * (10 ** (exp + 1)) which is 0.
5. check_val == value is true (0 == 0), so the body does mantissa //= 10 (mantissa stays 0) and exp += 1.
6. Nothing in the loop changes mantissa away from 0, so mantissa % 10 == 0 remains true forever. The loop never terminates.

Where it occurs in the code
- Inside add_number, in the block:
  if abs(x) >= 100:
      while mantissa % 10 == 0:
          check_val = (mantissa // 10) * (10 ** (exp + 1))
          if check_val == value:
              mantissa //= 10
              exp += 1
          else:
              break

How to fix it (concrete changes you can make)
Option A — simplest explicit guard for zero
- Before entering that loop, bail out for zero:
    if abs(x) >= 100:
        if value == 0:
            # nothing to shorten; use "0"
            self.add("0")
            return
        while mantissa % 10 == 0:
            ...
  This prevents the infinite loop when x == 0.

Option B — change the loop condition to require mantissa != 0
- Replace the while condition so it never runs if mantissa is 0:
    while mantissa != 0 and mantissa % 10 == 0:
        check_val = (mantissa // 10) * (10 ** (exp + 1))
        ...
  That avoids the pathological case where mantissa stays 0.

Option C — simplify the trimming logic (recommended for correctness)
- Work directly on an integer copy and divide until it no longer matches:
    if abs(x) >= 100:
        m = value
        e = 0
        if m != 0:
            while m % 10 == 0:
                m //= 10
                e += 1
                # optional sanity cap to avoid accidental infinite loops
                if e > 20:
                    break
        mantissa, exp = m, e
  Then use mantissa and exp in the later formatting. This avoids the subtle check_val == value comparison.

Why these fixes work
- They stop the loop from running when mantissa == 0, which is the root of the infinite loop. Option C also avoids the fragile check_val == value logic and is easier to reason about.

One-line test recommendation
- Add a unit test for x == 0 to ensure add_number("0") behaves and that the function returns (doesn't hang). Also test a few large multiples of 10 (e.g., 1000, -100000) to ensure trimming still behaves as you expect.
