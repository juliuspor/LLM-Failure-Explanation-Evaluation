What failed
- Where: lines 765–767 (the loop).
- What happened: the code calls array[i].upper() on every element (line 766). If array[i] is None this raises AttributeError: 'NoneType' object has no attribute 'upper'. If an element is not a string (e.g. an int), calling .upper() will raise an AttributeError for the same reason.
- Why it happened: the call to .upper() assumes every element is a str and non-None. That contradicts the docstring which says None elements should be preserved in the output. The unconditional .upper() is the root cause.

Concrete locations:
- line 766: _ = array[i].upper() — this triggers the AttributeError when array[i] is None (or not a str).
- line 767: classes.append(type(array[i])) — correct for extracting the class, but it never runs for elements that made line 766 fail.

Fixes (choose one depending on intended behavior)

1) If you want to preserve None elements (match the docstring: "If any of these objects is None, a None element will be inserted"):
- Remove the .upper() call and append None for None elements:
