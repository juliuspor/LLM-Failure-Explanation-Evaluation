What went wrong (root cause)
- Line 334: when array is not None the code does inferred_type = type(array). That returns the Python type of the whole list object (e.g. builtins.list), not the element/component type you actually need.
- _copy_list_grow1(array, inferred_type) expects inferred_type to describe the element/component type (the array’s component class), but you passed the list class instead. That makes the copy/grow routine create an array/list whose element type is wrong, so the later assignment new_list[len(new_list) - 1] = element fails with a class/containment mismatch (the stack trace text you show is a Java-style ClassCastException message).
- Because inferred_type is wrong, the safeguard at lines 346–347 (if inferred_type == object and expected_type != object: raise TypeError) either does not trigger when it should, or triggers incorrectly. In short: the code confuses "type of the container" with "type of the contained elements", so the new array/list has an incompatible component type.

Exactly where it fails
- At line 334 (if array is not None: inferred_type = type(array)) you pick the wrong type.
- At line 344 (new_list = ArrayUtils._copy_list_grow1(array, inferred_type)) the wrong inferred_type is used to create the new list/array.
- At line 345 (new_list[len(new_list) - 1] = element) the assignment fails because the list/array was created with an incompatible component type.

Why that produces the ClassCast/Type error
- _copy_list_grow1 constructs a new array that enforces a component type based on inferred_type. Passing the list class (type(array)) makes the new array expect list instances as elements. When you then assign element (not a list), the runtime rejects it, producing the ClassCast-like failure you saw.

How to fix it (concrete steps)
1. When array is provided, use the element type, not the container type. If array has elements:
   - inferred_type = type(array[0])
2. If array is provided but empty, fall back to other information in this order:
   - if element is not None: inferred_type = type(element)
   - elif expected_type is not None: inferred_type = expected_type
   - else: inferred_type = object
3. Keep the existing expected_type check, but ensure inferred_type has been derived from element/expected_type/object as above.
