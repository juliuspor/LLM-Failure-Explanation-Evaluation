Why it happened (root cause)
- Somewhere else in the code you compute and assign self._max_middle_index incorrectly. The usual concrete mistakes that produce the symptom "expected 1 but got 3" are:
  - You computed the index relative to a different list or a different window than the one used by the test. For example, you kept an index into a full-history list but later sliced that list; you then returned the stale absolute index (3) instead of the index in the current slice (1).
  - You failed to recompute _max_middle_index after removing or rotating elements. If elements are removed from the start of the list, the stored index must be decremented or recomputed.
  - You computed the index using the wrong expression (e.g., using enumerate incorrectly or using index() on a value that appears multiple times and expecting a different tie-breaking).
  - Off-by-one error when converting between positions and indices (start vs inclusive/exclusive boundaries).
