- The if at lines 133–136 decides whether to insert a separating space before appending newcode:

- Because your is_word_char implementation treats '-' as a word character, the condition evaluates true when the previous character (self.get_last_char()) is also a word character (for example a digit). The code therefore inserts a space (line 136), producing "- 0.0" instead of "-0.0".
- The wrong output comes from that inserted space; the test expects "-0.0" but got "- 0.0".

Why it fails (root cause)
- The logic on lines 132–136 is trying to decide whether to insert a space to separate two "words". But '-' is not a word separator in this context: a minus sign attached to a numeric literal must not be separated from the number. Because is_word_char('-') returns True, the code incorrectly treats '-' as something that should be space-separated from the preceding token, and inserts a space before appending the negative number.
