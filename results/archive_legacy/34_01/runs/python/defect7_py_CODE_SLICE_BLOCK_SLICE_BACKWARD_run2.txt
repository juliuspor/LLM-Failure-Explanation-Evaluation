Why it failed (root cause)
- If any array element is not a string (for example None or an int or a class object), that element has no .upper() method.
- When a non-string (most commonly None) is encountered, Python raises AttributeError: 'NoneType' object has no attribute 'upper()' (or a similar attribute error for another non-string type).
- That exception is raised at line 766 before you append anything to classes, so the method aborts.

Concrete examples that trigger the error
- array = ["foo", None, "bar"] → array[1].upper() raises AttributeError on line 766.
- array = ["foo", 123] → array[1].upper() raises AttributeError on line 766.

How to fix it (explicit steps)
1. Decide what you meant by calling .upper():
   - If you wanted to check/normalize string names, only call .upper() when the element is a str.
   - If you only need the element's type, remove the .upper() call entirely.
2. Implement a clear, explicit check before calling .upper(). For example:
   - If you want to accept only strings and treat non-strings as an error:
     - raise TypeError with a helpful message mentioning the array index and value.
   - If you want to skip non-strings or convert them to strings:
     - use isinstance(array[i], str) before calling .upper() or call str(array[i]) safely.
