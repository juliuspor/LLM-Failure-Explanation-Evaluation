What happened (where it fails)
- The code fails on line 766, in the loop:
  _ = array[i].upper()
- In your test call array[1] is None and array[2] is 99.0. Neither NoneType nor float has an upper() method, so Python raises AttributeError at that line before the code ever appends types. The failure is caused by calling .upper() unconditionally on every element.

Why it fails (root cause)
- The code assumes every element is a string and therefore has .upper(). That assumption is false for None and floats. Because the .upper() call runs before classes.append(type(array[i])), the method never completes and you never get the expected [str, type(None), float].

How to fix it (concrete steps and code options)
Choose one of these fixes depending on intent:

1) If you don't need .upper() at all (it looks unnecessary):
- Remove the line that calls .upper().

Replace the loop with:
for i in range(len(array)):
    classes.append(type(array[i]))

2) If you only want to call .upper() for string elements (keep effect only for strings):
- Guard the call with isinstance(..., str) so non-strings aren't touched.

Replace the loop with:
for i in range(len(array)):
    if isinstance(array[i], str):
        _ = array[i].upper()
    classes.append(type(array[i]))

3) If you intended to treat None specially or raise a controlled error:
- Check for None explicitly and handle it (return None type, skip, or raise a clear error).

Example (explicit None handling + string guard):
for i in range(len(array)):
    if array[i] is None:
        classes.append(type(None))
        continue
    if isinstance(array[i], str):
        _ = array[i].upper()
    classes.append(type(array[i]))

Notes
- In all cases, make the .upper() call conditional before calling it, or remove it if it's not needed.
- The problematic variables/lines are array[i] at line 766; change there to avoid calling .upper() on non-strings.
