What happened (where it fails)
- The code in add_number (the snippet you gave) tries to detect integers with "if x == int(x) ..." and then uses value = int(x) (lines in your snippet).
- For x = -0.0, x == int(x) is True and value = int(-0.0) becomes 0 (positive zero). The code then runs the integer branch and eventually does self.add(str(value)) which produces "0".
- The test expects "-0.0", so the result "0" fails the assertion.

Why it fails (root cause)
- IEEE floating-point has a distinct negative zero bit pattern, but most numeric comparisons and int() conversion do not preserve that sign: -0.0 == 0.0 and int(-0.0) == 0.
- Your code uses x < 0 to detect negative numbers when deciding whether to add a separating space (prev == '-'), but x < 0 is False for -0.0 (because -0.0 is equal to 0.0, not less). So the code neither preserves the sign via the integer path nor uses the special handling earlier.
- In short: the integer branch discards the negative-zero sign and produces "0". The negative-zero test using x < 0 never fires for -0.0, so you lose the sign entirely.

Exactly where (which variables / lines)
- Condition: if x == int(x) ... decides the integer branch (this is where -0.0 enters).
- Assignment: value = int(x) (turns -0.0 into 0).
- Final output: self.add(str(value)) (adds "0").
- Earlier check: if x < 0 and prev == '-': ... — this check does not run for -0.0 because x < 0 is False.

Concrete fixes (step-by-step, exact changes)
You need to detect negative zero and handle it explicitly before converting to int or relying on x < 0. Two clear options:

Option A — Treat negative zero as non-integer so the float formatting preserves "-0.0"
1. Change the integer-branch condition at the start of that block:
   Replace
     if x == int(x) and not math.isinf(x) and not math.isnan(x):
   with
     if x == int(x) and not math.isinf(x) and not math.isnan(x) and not (x == 0.0 and math.copysign(1.0, x) < 0):
2. No other change needed — negative zero will fall through to the else branch and self.add(str(x)) will produce "-0.0".

Option B — Keep integer branch but preserve sign for zero
1. Immediately after value = int(x), add a check:
     if value == 0 and math.copysign(1.0, x) < 0:
         # x was -0.0, preserve sign and desired format
         self.add("-0.0")
         return
2. Then the rest of the integer branch can proceed unchanged for other integers.

Which to pick
- Option A is simplest and ensures the standard float str(x) is used for -0.0, yielding "-0.0" as your test expects.
- Option B is OK if you must keep the integer-branch logic for other values and want explicit control of the "-0.0" formatting.

Minimal example patch (Option A) — replace the if line:
- from:
    if x == int(x) and not math.isinf(x) and not math.isnan(x):
- to:
    if x == int(x) and not math.isinf(x) and not math.isnan(x) and not (x == 0.0 and math.copysign(1.0, x) < 0):

After this change, add_number(-0.0) will go to the else branch and self.add(str(x)) will append "-0.0", so your test will pass.
