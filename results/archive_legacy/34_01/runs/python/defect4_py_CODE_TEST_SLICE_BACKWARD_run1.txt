What happened
- In the block that updates the maximum-middle index you accidentally fetch the period for self._min_middle_index instead of self._max_middle_index.
- Concretely, inside the if self._max_middle_index >= 0: branch you call
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  when you should call
  max_middle_period = self.get_data_item(self._max_middle_index).get_period()

Why this breaks the test
- The code compares the new period's middle against max_middle computed from the wrong existing period (the minimum-middle entry). That yields an incorrect comparison and can either prevent updating _max_middle_index when it should change or set it incorrectly.
- In your test sequence the expected max-middle index is 1, but because the code reads the wrong existing period it computes the wrong reference middle and ends up with the wrong _max_middle_index.

- In the block starting with:

- The offending line is the call to get_data_item(...) that passes self._min_middle_index instead of self._max_middle_index.
