  The exception message shows minutes_offset == -15, so the code is rejecting a minutes value of -15 as out of range.

Why it happened (root cause)
- The minutes part was parsed as a negative number (minutes_offset == -15) instead of as a positive 15 with a separate sign for the hours. The code expects minutes_offset to be in the range 0..59 and then apply the sign using the hours-based logic later (lines 402 and 404), but the parser left the minus sign on the minutes token. As a result the early range check at line 396 fails before the sign-handling code at lines 402/404 ever runs.

- The failing check: line 396 (minutes range validation).
- The sign/adjustment code that is supposed to combine hours and minutes: lines 402 and 404:

  These lines assume minutes_offset is non-negative and that the sign is carried by hours_in_minutes; they will not run successfully because the function has already thrown at 396.
