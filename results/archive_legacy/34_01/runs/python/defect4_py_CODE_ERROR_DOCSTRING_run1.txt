What happened
- In the "max middle" block the code fetches the period using the wrong index:
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  This is inside the if self._max_middle_index >= 0: block, so it should be using self._max_middle_index, not self._min_middle_index.
- Because you compare the candidate middle value to the middle of the period at _min_middle_index instead of the current _max_middle_index, the comparison logic is wrong and can cause _max_middle_index to be set incorrectly. That explains the failing assertion (expected max_middle_index=1 but got 3): the code compared against the wrong stored period and therefore updated the wrong index.

Where it occurs
- In the block beginning with:
    if self._max_middle_index >= 0:
  the line
    max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  is the bug.

Why this causes the observed result
- The method is intended to keep track of the index whose period has the largest middle value. By reading the period for _min_middle_index, the comparison middle > max_middle uses an unrelated middle value. This can make the code believe the new middle is larger (or smaller) incorrectly and update _max_middle_index to the wrong index (3 in your failure) instead of keeping it at the true max (1).

How to fix (concrete steps)
1. Change the incorrect line to use _max_middle_index:
   max_middle_period = self.get_data_item(self._max_middle_index).get_period()
2. Optionally, reduce duplication and risk of similar errors:
   - Factor the start/middle/end extraction into a small helper like _period_start_ms(period), _period_middle_ms(period), _period_end_ms(period).
   - Use that helper consistently for min/max blocks.
3. Add a unit test that feeds multiple periods whose middles are known and asserts _min_middle_index and _max_middle_index remain correct after updates.

Minimal code change (one-line patch)
- Replace:
    max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  with:
    max_middle_period = self.get_data_item(self._max_middle_index).get_period()

That single change will fix the incorrect comparison and stop _max_middle_index from being set to the wrong value.
