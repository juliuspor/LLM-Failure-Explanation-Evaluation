What happened (where it fails)
- The code in ClassUtils.to_class iterates with:
  - line 765: for i in range(len(array)):
  - line 766: _ = array[i].upper()
  - line 767: classes.append(type(array[i]))
- With your input ['Test', None, 99.0], the loop does array[0].upper() (fine), then when i == 1 it executes array[1].upper().
- array[1] is None, so at line 766 Python raises AttributeError: 'NoneType' object has no attribute 'upper'. The method never reaches line 767 for index 1 or 2, so no classes are appended and the function fails.

Why it fails (root cause)
- The code assumes every element has an upper() method (i.e., is a string) before it records its type.
- That assumption is false for None and float. Calling upper() on a non-string value triggers an AttributeError and stops the function.

Concrete fixes (step-by-step)
1. Decide the intended behavior:
   - If you only want to call upper() for strings, skip non-strings.
   - If you never need upper() to determine the type, remove the call entirely.

2. Fix options you can apply (pick one):

Option A — only call upper() for strings
- Replace line 766 with a type check:
  if isinstance(array[i], str):
      _ = array[i].upper()
  classes.append(type(array[i]))

Option B — remove the unnecessary upper() call
- Delete line 766 and just do:
  classes.append(type(array[i]))

Option C — handle exceptions explicitly
- Wrap the call in try/except if upper() is optional:
  try:
      _ = array[i].upper()
  except AttributeError:

  classes.append(type(array[i]))

- Using Option B (simplest), ClassUtils.to_class(['Test', None, 99.0]) will produce [str, type(None), float] as expected.

Summary
- Root cause: calling array[i].upper() on elements that are not strings (line 766) raises AttributeError, preventing the code from reaching classes.append(type(array[i])) at line 767. Fix by guarding the upper() call or removing it.
