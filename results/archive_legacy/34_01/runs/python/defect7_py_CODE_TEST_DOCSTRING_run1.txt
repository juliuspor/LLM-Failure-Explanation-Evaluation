What happened (root cause)
- The failure occurs inside the loop at this line:
  _ = array[i].upper()
- That line assumes array[i] is a string and has an upper() method. In the test call ClassUtils.to_class(['Test', None, 99.0]) the second element is None and the third is a float:
  - When array[i] is None, None.upper() raises AttributeError.
  - Even if you removed None, 99.0.upper() would raise AttributeError because float has no upper().
- So the method raises AttributeError and never reaches the code that appends types. That violates the behavior expected by the test and the docstring (which itself is inconsistent — it both promises to insert None for None elements and claims it raises AttributeError for None).

Exactly where it happens
- Method: to_class
- Faulty line: "_ = array[i].upper()" inside the for loop
- Cause: calling .upper() on non-string objects (None and floats) raises AttributeError

Why that line is there is unnecessary
- The method's purpose (per docstring/test) is to produce a list of types for each element. You do not need to call .upper() on elements to determine their type. That call serves no type-detection purpose and only causes failures for non-strings.

Concrete fixes (step-by-step)
1. Remove the line "_ = array[i].upper()".
2. Iterate over elements directly (for element in array) instead of using index i.
3. Append type(element) to the result list, and if the element is None, append type(None) (which type(None) already returns).
4. Fix docstring inconsistency: remove "Raises: AttributeError" or change it to say the method will not raise for None elements.

Corrected code (drop-in replacement)

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        
        classes = []
        for element in array:
            classes.append(type(element))
        return classes

Notes
- This implementation returns [str, type(None), float] for ['Test', None, 99.0], matching the test.
- If you want None elements to be represented specifically as None instead of type(None), replace classes.append(type(element)) with:
      classes.append(None if element is None else type(element))
  and adjust the expected return accordingly.
- Also update the docstring so it correctly describes the behavior (whether None elements are converted to type(None) or left as None) and remove the incorrect "Raises: AttributeError" line.
