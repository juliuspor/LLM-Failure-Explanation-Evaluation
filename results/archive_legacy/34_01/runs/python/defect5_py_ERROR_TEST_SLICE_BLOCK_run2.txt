  Since inferred_type == object and expected_type is str, that condition is true and the code raises the TypeError you saw:

Why that is wrong for your call
- You passed None as the element and expected_type=str. There is nothing in the input array to prove a conflicting element type; all we have is a None element and no existing array. There is no actual object[] that needs casting to String[], so rejecting the call is incorrect. The code assumes object means "incompatible", but object can also mean "unknown because array was None or all elements are None".
- In short: inferred_type == object is being treated as an incompatible concrete type instead of "unknown/unspecified", so the function throws when it should allow creating a new list of the requested expected_type (and accept None as a valid value).
