Why it happened (root cause)
- The code looked at the list contents and set inferred_type = object (line 340). That means "I cannot prove all elements share a specific concrete type."
- expected_type is not object (so the caller expects a list/array of strings).
- On the Java side (or wherever a JVM array type is required), the code attempted to hand over or cast a Python/list-backed Object[] to a java.lang.String[].
- In the JVM an Object[] is not the same type as String[] and cannot be cast even if every element happens to be a java.lang.String at runtime. The JVM requires the actual array object to be a String[] to treat it as such.
- Because inferred_type stayed object, the code raised the TypeError at line 347 instead of performing a safe conversion. The underlying cast would also fail if attempted, which is why the code refuses to proceed.
