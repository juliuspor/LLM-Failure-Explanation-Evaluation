What happened (root cause)
- The code treats numbers that are "equal to an int" with the branch starting at
  if x == int(x) ...
  For x == -0.0 this condition is True (because -0.0 == 0.0).
- value = int(x) converts -0.0 to 0, losing the negative-zero sign.
- The earlier check if x < 0 and prev == '-': ... does not run for -0.0 because in Python -0.0 < 0 is False. So nothing adds the '-' back.
- Result: you call self.add(str(value)) where value is 0, so the output becomes "0" (or 0), not "-0.0". That produces the failed assertion.

Where it occurs in the code
- The critical lines are:
  - if x == int(x) ...  (the branch selection)
  - value = int(x)     (conversion that drops the sign)
  - if x < 0 and prev == '-': ... (this check does not catch -0.0)
  - self.add(str(value)) (final addition that yields "0")

Why -0.0 behaves this way in Python
- -0.0 compares equal to 0.0, and int(-0.0) == 0. The sign bit is preserved in IEEE floats but comparisons and int() drop that sign. x < 0 is False for -0.0, so code paths that depend on x < 0 won't see it as negative.

Concrete fixes (step-by-step)
1. Detect negative zero before you convert to int or rely on x < 0. Use math.copysign or struct to test the sign bit:
   - math.copysign(1.0, x) == -1.0  (or math.copysign(1.0, x) < 0)
2. If x is negative zero, emit the exact string you need and return early, e.g. "-0.0".
3. Then proceed with the existing integer branch for other values.

Minimal patch example (insert before the if x == int(x) ... line):
    # preserve negative zero
    if x == 0.0 and math.copysign(1.0, x) < 0:
        self.add("-0.0")
        return

Alternate approach
- Instead of an explicit special case, you can avoid converting -0.0 to int at all: when x == int(x) keep using float formatting that preserves sign (e.g. format(x, '.1g') or repr(x)), or detect sign with math.copysign and prepend '-' to the formatted absolute value. But the explicit special-case above is the simplest and most direct fix.

Summary
- The code fails because int(x) and comparison x == int(x) drop the sign of negative zero, and the earlier x < 0 check does not detect -0.0. Fix by checking sign with math.copysign before converting to int (or before emitting), and emit "-0.0" explicitly.
