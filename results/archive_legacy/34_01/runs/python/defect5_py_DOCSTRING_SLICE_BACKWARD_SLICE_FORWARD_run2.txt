What happens
- At line 340 the code sets inferred_type = object whenever array is None and element is not None.
- Later, at line 346 the code checks if inferred_type == object and expected_type != object and raises TypeError if that is true (line 347).

Why that causes the failure
- If the caller passes a concrete element (e.g. element = 5) but also passes an expected_type (e.g. expected_type = int), the code incorrectly marks inferred_type as object (line 340) instead of the element’s actual type.
- Because inferred_type is object, the conditional at line 346 becomes true (object == object and expected_type != object) and the function raises a TypeError even though the element actually matches expected_type. The root cause is the wrong value assigned to inferred_type when element is not None.

Where to fix it (exact lines)
- Replace the assignment at line 340:
  inferred_type = object
  with:
  inferred_type = type(element)
