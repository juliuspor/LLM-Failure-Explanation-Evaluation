  - array contains only None entries.
  - array contains mixed types so the best common ancestor found was object.
- With inferred_type == object the function assumes the underlying runtime list would be an Object[] (hence the "[Ljava.lang.Object;" message).

- The code treats this as an unsafe cast and raises the TypeError rather than attempting to coerce or accept the conversion.
