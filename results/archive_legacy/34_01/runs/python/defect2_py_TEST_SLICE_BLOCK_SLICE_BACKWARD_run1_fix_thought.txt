The bug arises because g is computed from the raw input value and then validated, causing negative g for values below lower_bound. Fix by clamping the value to bounds first (use the already computed clamped v) and computing ratio from v so g will always be within 0..255. Remove the g-range check so get_paint returns (0,0,0) for values below lower bound and (255,255,255) for values above upper bound.
