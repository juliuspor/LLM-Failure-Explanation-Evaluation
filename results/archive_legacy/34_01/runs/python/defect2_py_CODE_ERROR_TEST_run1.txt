What happened (root cause)
- You clamp the input into v on lines 2–3:
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
- But on the next line you compute g using the original value variable instead of the clamped v:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If value is below self._lower_bound (as in your test value = -0.5), g becomes negative and fails the range check:
  if g < 0 or g > 255:
      raise ValueError(...)
- So the code raises ValueError because you never used the clamped v when computing g.

Where it occurs
- In get_paint, the bug is on the line that computes g (uses value) immediately after the two clamping lines that set v.
- The ValueError is raised on the if-check that follows the g computation.

Why it fails for your test
- With value = -0.5 and lower_bound (presumably 0.0), v will be set to 0.0 but g is computed from -0.5, producing a negative g and triggering the ValueError. That’s why gps.get_paint(-0.5) fails instead of returning (0,0,0).

Concrete fixes (step-by-step)
1. Use the clamped variable v when computing g:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
2. Guard against division by zero if lower and upper bounds are equal:
   denom = (self._upper_bound - self._lower_bound)
   if denom == 0:
       raise ValueError("Paint scale has equal lower and upper bounds")
   g = int((v - self._lower_bound) / denom * 255.0)
3. Improve the ValueError message to include the offending value and bounds for easier debugging:
   if g < 0 or g > 255:
       raise ValueError(f"Computed color component {g} outside range 0..255 for value {value} with bounds {self._lower_bound}..{self._upper_bound}")
4. Return the grayscale triple using g (as you already do).

Minimal corrected function
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)

        denom = (self._upper_bound - self._lower_bound)
        if denom == 0:
            raise ValueError("Paint scale has equal lower and upper bounds")

        g = int((v - self._lower_bound) / denom * 255.0)

        if g < 0 or g > 255:
            raise ValueError(f"Computed color component {g} outside range 0..255 for value {value} with bounds {self._lower_bound}..{self._upper_bound}")

        return (g, g, g)

With this change, gps.get_paint(-0.5) will use v = self._lower_bound for the calculation and return (0, 0, 0) as expected.
