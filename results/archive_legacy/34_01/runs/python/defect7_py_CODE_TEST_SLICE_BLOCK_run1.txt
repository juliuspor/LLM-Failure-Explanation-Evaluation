What happened
- At line 766 the code calls array[i].upper(). That call fails when array[i] is not a string.
- In your test array, the second element is None and the third is 99.0 (a float). Neither NoneType nor float have an upper() method.
- Python raises AttributeError: 'NoneType' object has no attribute 'upper' (and would raise a similar error for float), so the function crashes before it can build the types list.

Where it occurs
- Exact failing line: line 766 in the slice block
    766 |             _ = array[i].upper()
- The loop starts at line 765 and the next line 767 appends type(array[i]) — but control never reaches 767 for the bad element because of the exception at 766.

Why it was done wrong
- The call to .upper() is unnecessary for determining types. It both assumes every element is a string and provokes an exception for non-string values. The function’s goal (returning the element types) only needs type(array[i]) and should not call string methods on the elements.

How to fix it (concrete steps)
1. Remove the .upper() call entirely.
2. Handle None explicitly if you want type(None) rather than failing (type(None) is the built-in for None).
3. Prefer iterating the elements directly instead of indexing, e.g. for item in array:.

Corrected implementation (drop-in replacement)
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    
        classes = []
        for item in array:
            # no .upper() call — just record the type (None -> type(None))
            classes.append(type(item))
        return classes

Result with your test
- With the corrected function, ClassUtils.to_class(['Test', None, 99.0]) returns [str, type(None), float] as asserted.
