What happened (where it fails)
- The code calls array[i].upper() inside the loop (the line with "_ = array[i].upper()").
- If array[i] is None, that call raises AttributeError: "'NoneType' object has no attribute 'upper'".
- Because that exception occurs before classes.append(type(array[i])), the method fails instead of inserting a None class element as the docstring promises.

Why it fails (root cause)
- The call to .upper() assumes every element is a string-like object with an upper() method. That is not true for None (and not true for many non-string objects).
- The docstring says "If any of these objects is None, a None element will be inserted", but the .upper() call prevents that by immediately raising an AttributeError for None values. In short: the unnecessary .upper() check is the bug.

Exact location to fix
- Remove the line "_ = array[i].upper()" inside the for loop. That line is the cause of the AttributeError when array[i] is None.

Concrete fix (steps)
1. Delete the upper() call.
2. Handle None explicitly: if array[i] is None, append None to classes; otherwise append the element's type.
3. (Optional) Use array[i].__class__ or type(array[i]) consistently.

Fixed code example:
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    
        classes = []
        for i in range(len(array)):
            if array[i] is None:
                classes.append(None)
            else:
                classes.append(type(array[i]))
        return classes

Notes
- This preserves the behavior described in the docstring (None input element -> None in result).
- If you intended to enforce that elements must be strings, replace the None check with an explicit type check and raise a clear exception instead of calling .upper().
