What happened (root cause)
- At line 766 you call array[i].upper(). If array[i] is None, that expression tries to call upper on a NoneType and Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The failing test passes array = ['Test', None, 99.0]. On the second iteration array[1] is None, so the code fails exactly at line 766.

Where it occurs
- File slice shown, lines 765–767:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))
- The error occurs at line 766 when array[i] is None.

Why that code was fragile
- The code assumes every element is a string (has .upper()). It does not check types before calling .upper().
- After the .upper() call it appends type(array[i]). For None you want type(None), but you never reach the append because the .upper() call fails first.

How to fix it (concrete steps, with example)
Pick one of these explicit fixes depending on intent:

1) If you only meant to normalize strings but must handle non-strings safely:
- Replace the call to .upper() with a type check before calling it.

Example replacement in the loop (replace lines 765–767):
for i in range(len(array)):
    val = array[i]
    if isinstance(val, str):
        _ = val.upper()         # only call upper for strings
        classes.append(str)
    else:
        classes.append(type(val))

2) If you intended to accept class-name strings (e.g. "int" -> int) and also accept actual values:
- First check type, then map string names to classes safely.

Example:
for val in array:
    if isinstance(val, str):
        name = val.strip().lower()
        # map known names to classes
        if name == 'str' or name == 'string' or name == 'test':
            classes.append(str)
        elif name == 'int':
            classes.append(int)
        else:
            classes.append(str)   # or raise for unknown
    else:
        classes.append(type(val))

3) Quick robust approach using try/except (less preferred because it hides logic):
for val in array:
    try:
        _ = val.upper()
        classes.append(str)
    except AttributeError:
        classes.append(type(val))

What to change at specific lines
- Replace line 766 with a check (isinstance(array[i], str)) or wrap the call in try/except.
- Ensure classes.append happens after the type-safe check so you don't try to call .upper() on None.

Why these fixes work
- They prevent calling .upper() on None (or other non-string objects), so the AttributeError cannot occur.
- They make the function return the expected [str, type(None), float] for ['Test', None, 99.0].
