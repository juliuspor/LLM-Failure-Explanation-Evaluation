- Later, at line 346 the code checks:

  Because inferred_type was set to object even though element is not None, that condition becomes true whenever expected_type is not object, so the function always raises a TypeError in that case.
- In short: the code treats a non-None element as if its element type were object, then immediately rejects any non-object expected_type. That is why the function fails.
