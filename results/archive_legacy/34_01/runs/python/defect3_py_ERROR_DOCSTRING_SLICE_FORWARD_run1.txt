What happened (where and which line)

Why it happened (root cause)
- The code checks bounds only in the second conditional: `if 0xD800 <= high <= 0xDBFF and index + 1 < len(code_units):` (line 39). That check protects access to `code_units[index + 1]` but it does not protect the initial access to `code_units[index]` on line 38.
- If `index` equals or exceeds `len(code_units)` (for example `index == 2` when `len(code_units) == 2`), the read on line 38 fails before any bounds test runs. In short: you are indexing into the list before you ensure the index is valid.
