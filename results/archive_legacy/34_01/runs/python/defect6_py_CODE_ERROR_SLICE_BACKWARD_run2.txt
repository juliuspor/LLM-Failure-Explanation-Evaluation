What happened (root cause)
- The code treats any float that compares equal to an integer the same way: the branch at
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
  then does value = int(x) and eventually self.add(str(value)).
- For x == -0.0, x == int(x) is True and int(-0.0) yields 0. Converting that to str produces "0". That loses the negative sign on negative zero.
- You therefore get "0" where the test expects "-0.0". The loss occurs at the line value = int(x) and later self.add(str(value)).
- A second related point: the earlier test if x < 0 and prev == '-': is intended to avoid misparsing x- -4, but -0.0 < 0 is False in Python, so that branch does not catch negative zero either.

Where it occurs (variables / lines)
- Method: add_number(self, x: float)
- Problem lines:
  - the guard: if x == int(x) and not math.isinf(x) and not math.isnan(x):
  - the conversion: value = int(x)
  - the output: self.add(str(value))
  - the sign check: if x < 0 and prev == '-': (doesn't detect -0.0)

Why this is wrong (plain language)
- Converting -0.0 to int loses the sign. The code assumes integer equality implies it can safely use int(x) and string that result, but that assumption fails for signed zero. The sign test x < 0 also fails for -0.0, so you neither detect nor preserve the negative zero.

Concrete fixes (step-by-step, including exact code you can paste)

Option A — Special-case negative zero early (recommended)
1. Add a check before the integer branch to preserve "-0.0":
    import math
    ...
    prev = self.get_last_char()
    if x == 0.0 and math.copysign(1.0, x) < 0:
        # preserve negative zero exactly
        if prev == '-':
            self.add(" ")
        self.add("-0.0")
        return
2. Then the existing integer branch can follow unchanged.

Option B — Preserve sign when formatting integers (if you want e.g. "-0" instead of "-0.0")
1. Replace value = int(x) and later self.add(str(value)) with code that preserves sign:
    value = int(abs(x))
    sign = "-" if math.copysign(1.0, x) < 0 else ""
    ...
    self.add(sign + str(value))
2. (Adjust rest of mantissa/exp logic to operate on abs(x) or abs(value) as appropriate.)

Option C — Avoid int conversion for zeros
1. If x == 0.0, always format with float formatting that keeps sign:
    if x == 0.0:
        sign = "-" if math.copysign(1.0, x) < 0 else ""
        self.add(sign + "0.0")
        return

Notes
- Use math.copysign(1.0, x) < 0 to detect negative zero. Tests like x < 0 or x == -0.0 do not reliably detect -0.0.
- Choose Option A if you must output exactly "-0.0" as the test expects; choose B/C if you prefer other sign-preserving output.
