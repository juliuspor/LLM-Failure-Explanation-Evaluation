What happened
- In the block that updates self._max_middle_index the code reads the period for the wrong index:
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  — it uses self._min_middle_index instead of self._max_middle_index.
- Because you compare middle against the middle value computed from the min-middle item, the comparison can be wrong and self._max_middle_index can be left as a later (wrong) index. That is why the test expected max_middle_index == 1 but got 3: the code compared to the wrong reference period and failed to update the max correctly.

Where it occurs
- The bug is in the "if self._max_middle_index >= 0:" block (the max-middle update). The offending line is the get_data_item call shown above; later code computes s, e and max_middle from that wrong period and then uses it in:
  if middle > max_middle:
      self._max_middle_index = index

Why that causes the failing assertion
- You intended to compare the new item's middle to the current maximum-middle item (self._max_middle_index), but instead you compared it to the minimum-middle item (self._min_middle_index). That yields incorrect comparisons and leaves self._max_middle_index at an incorrect value (3) instead of the expected (1).

How to fix it (concrete steps)
1. Change the mistaken line to use _max_middle_index:
   max_middle_period = self.get_data_item(self._max_middle_index).get_period()
2. Run the unit test that triggered the assertion to verify it now passes.
3. Optional additional checks:
   - Add a unit test that inserts periods with known starts/middles/ends and asserts min/max indices after each insert.
   - Consider extracting the repeated period-to-millis conversion into a helper to reduce copy/paste errors.
   - Add an assertion or a debug log at the top of the _max_middle_index block to ensure the index used is the intended one (temporary aid while running tests).

Minimal code change example
Replace this line:

With:

That single-line change fixes the wrong reference and should resolve the assertion failure.
