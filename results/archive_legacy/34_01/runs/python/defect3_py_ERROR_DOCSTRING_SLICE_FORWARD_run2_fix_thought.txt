The original translate method called code_point_at which could raise IndexError due to that function accessing code_units[index] without a prior bounds check. To fix we avoid calling code_point_at and instead implement local safe extraction of the codepoint and char count logic, ensuring we never index out of range. The method preserves behavior: validate out not None, return on input_seq None, convert to UTF-16 code units, loop advancing pos by the correct number of UTF-16 code units, calling translate_codepoint and writing untranslated code units to out using to_chars. For consumed>0 we increment pos by the UTF-16 length of each consumed codepoint, using a safe local reader that doesn't read past the end.
