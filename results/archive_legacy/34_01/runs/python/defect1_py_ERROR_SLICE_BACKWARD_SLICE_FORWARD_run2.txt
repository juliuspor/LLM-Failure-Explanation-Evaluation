What happened (where it failed)
- The ValueError is raised at line 396:
  - if minutes_offset < 0 or minutes_offset > 59:
  - raise ValueError(f"Minutes out of range: {minutes_offset}")
- The actual minutes_offset value was -15, so the check failed and the exception was thrown.

Why it happened (root cause)
- minutes_offset at the time of that check contains a signed value (−15), not the raw minute component (0–59) the check expects.
- Later code (lines 402–405) shows the intended normalization of minutes with hours and sign:
  - line 402: minutes_offset = hours_in_minutes - minutes_offset
  - line 404: minutes_offset = hours_in_minutes + minutes_offset
  Those lines indicate the code intends to combine hours_in_minutes and the minutes component after accounting for sign.
- But the minutes value had been given a negative sign before the range check. Because the check runs before that normalization, it sees a negative minute (−15) and rejects it, even though the minute component itself was legal (15).

Concrete example
- If you parse an offset like "-0:15":
  - The minute component parsed from the string is 15.
  - A sign of − is applied too early so minutes_offset becomes −15.
  - The code then hits the range check at line 396 and raises "Minutes out of range: -15".
  - The intended handling at lines 402–405 (combine hours and minutes with sign) never gets to fix this because the exception happened first.

How to fix it (specific steps and exact places to change)
1. Keep the sign separate from the minute component until after the range check:
   - Introduce a variable sign (e.g. sign = 1 or -1) based on the parsed sign character.
   - Keep minutes_component = absolute minute value parsed from the string (0..59).
2. Validate minutes_component (not minutes_offset) at line ~396:
   - Replace the existing check with:
     - if minutes_component < 0 or minutes_component > 59:
         raise ValueError(f"Minutes out of range: {minutes_component}")
3. Apply the sign only when combining hours and minutes (the logic already at lines 402–405):
   - if sign < 0:
       minutes_offset = hours_in_minutes - minutes_component
     else:
       minutes_offset = hours_in_minutes + minutes_component
4. Then compute offset milliseconds and proceed as currently done.
