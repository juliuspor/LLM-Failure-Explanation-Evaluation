What happened
- The test calls LocaleUtils.to_locale('fr__POSIX') (locale_str = 'fr__POSIX').
- At lines 237–240 the code does:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - then checks that ch3 and ch4 are uppercase A..Z and raises ValueError if not.
- For 'fr__POSIX' the characters are:
  - index 0='f', 1='r', 2='_', 3='_' , 4='P', ...
- ch3 is '_' which is less than 'A', so the check at line 239 fails and the method raises ValueError("Invalid locale format: fr__POSIX").
- Root cause: the code at lines 231 and 237–240 assumes a two-letter country code exists at positions 3–4. It does not handle the valid form where the country is empty and the variant follows a second underscore (the "fr__POSIX" form). In other words, it treats the second underscore as an invalid country character instead of recognizing "double underscore => empty country, variant starts after it".

Where it happens
- Method: LocaleUtils.to_locale
- Variable: locale_str
- Problematic lines: 231 (branch leading here) and 237–240 (ch3/ch4 access and validation), which raise the ValueError.

Exact fix (steps)
1. Detect the "empty country" case before reading ch3/ch4: check for two consecutive underscores at positions 2 and 3 (i.e. locale_str[2] == '_' and locale_str[3] == '_').
2. If that case is true, set country = '' and set variant = locale_str[4:] (or the correct slice for your parsing logic) and skip the uppercase-country validation.
3. Otherwise continue with the existing two-letter country validation (ch3/ch4 are checked).

Minimal patch sketch (insert before the ch3/ch4 checks):
- if locale_str[2] == '_' and locale_str[3] == '_':
    country = ''
    variant = locale_str[4:]
    # proceed to construct Locale('fr', '', 'POSIX')
  else:
    ch3 = locale_str[3]
    ch4 = locale_str[4]
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(...)

Why this fixes it
- It explicitly recognizes the valid "language + empty country + variant" form (double underscore) and avoids treating the second underscore as a bad country character.
- After this change 'fr__POSIX' will parse to Locale('fr', '', 'POSIX') instead of raising ValueError.
