What happened (where it failed)
- At line 71 you compute g using the original parameter value:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- But at lines 56–59 you already clamped value into v:

  v = min(v, self._upper_bound)
- Because you never use v, g can be computed from an unclamped value outside the bounds and produce a number <0 or >255. That triggers the ValueError at line 74:

Why it happened (root cause)
- You intended to clamp the input but forgot to use the clamped variable v in the scaling formula. The formula still uses the original value, so out-of-range inputs produce out-of-range g and raise the exception.

Additional edge to check
- If self._upper_bound == self._lower_bound the division is a zero-division error; handle that too.

Precise fixes (step-by-step)
1. Use v (the clamped value) in the formula instead of value.
2. Guard against division by zero when upper and lower bounds are equal.
3. Optionally clamp g after conversion to ensure it is within 0–255 and improve the error message.
