What happened (root cause)
- The code calls array[i].upper() on every element at the line
    _ = array[i].upper()
  without checking what the element is.
- When array[i] is None (the second element in your test), None.upper() raises
  AttributeError: 'NoneType' object has no attribute 'upper'.
- That exception is why your test fails. The error occurs in the for-loop of to_class, on the line that tries to call .upper().

Why that line triggers the error
- None is not a string, so it has no upper() method. The code assumes every element is a string and calls .upper() unconditionally.
- The method then still appends type(array[i]) to classes, but the exception prevents reaching that line for a None element.

Where to fix it (concrete steps)
1. Remove or guard the call to .upper() so it runs only for string elements.
2. For None, append type(None) (or whatever behavior you want).
3. For non-string objects (like float 99.0), append their type.

Two concrete fixes you can apply

Minimal fix (preserves original intent of upper() only for strings):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()

        classes = []
        for i in range(len(array)):
            item = array[i]
            if isinstance(item, str):
                _ = item.upper()        # safe now, only called for strings
                classes.append(str)
            else:
                classes.append(type(item))
        return classes

Robust and clearer version (no unused upper call; explicit handling):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()

        classes = []
        for item in array:
            if item is None:
                classes.append(type(None))
            elif isinstance(item, str):
                classes.append(str)
            else:
                classes.append(type(item))
        return classes

Why these fixes work
- They avoid calling .upper() on None (or non-strings), so the AttributeError won't occur.
- They explicitly decide what type to append for each kind of element (None => type(None), strings => str, numbers => float, etc.), matching your test expectation: ClassUtils.to_class(['Test', None, 99.0]) -> [str, type(None), float].

Additional note
- If you were using .upper() to normalize strings before type detection, that’s unnecessary because type('Test') is str regardless of case. Remove the .upper() call unless you need it for some other logic.
