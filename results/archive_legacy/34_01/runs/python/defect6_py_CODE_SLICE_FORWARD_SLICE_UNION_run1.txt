What fails
- At runtime you get NameError: name 'math' is not defined.
- It happens on this line inside add_number:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- The code uses math.isinf and math.isnan but never imports the math module, so math is undefined.

Why it happens (root cause)
- The variable name math refers to the standard math module. Python only provides that name if you import it (for example with import math or from math import isinf, isnan).
- Because there is no import math anywhere visible in the shown code, evaluating math.isinf triggers the NameError immediately when add_number runs.

Exactly where to fix it
- Add an import at module top (outside the class/function), for example:
  import math
  or, if you prefer to import only what you need:
  from math import isinf, isnan

Concrete fixes (pick one)
1) Import the whole module (recommended if you use other math functions):
   - At the top of the file (line before class/function definitions) add:
     import math

2) Import just the functions used:
   - At the top add:
     from math import isinf, isnan
   - Then change the test to:
     if x == int(x) and not isinf(x) and not isnan(x):

3) Use an alternative single check:
   - If you only need to ensure x is a finite number, replace both checks with:
     from math import isfinite
     ...
     if x == int(x) and isfinite(x):
   - This is slightly cleaner and avoids two function calls.

Extra note (optional to consider)
- The equality check x == int(x) can behave unexpectedly for very large floats because of float precision; if you rely on exact integer detection for huge values, review whether you want int(x) or math.floor/round logic. This is not the immediate crash cause, but could be a logic edge case later.
