What happened (where it fails)

Why that is the root cause (simple, concrete)
- The function treats "inferred_type == object" as meaning "this list is an Object[] that cannot be cast to a more specific array type", and so it forbids creating a list of the requested expected_type.
- But when both array and element are None there is no actual concrete element to constrain the type. It's perfectly valid to return a list that is intended to hold str values with a None entry. The code's check is therefore too strict: it blocks a legitimate case where no runtime type information exists and the expected_type should be allowed.

- expected_type parameter equals str (call site).
- The TypeError is raised at lines 346–347 because inferred_type == object and expected_type != object.
