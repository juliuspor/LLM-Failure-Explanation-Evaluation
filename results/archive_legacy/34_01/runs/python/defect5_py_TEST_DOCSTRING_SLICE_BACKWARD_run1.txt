What happened (where and why)
- In ArrayUtils.add, the code that decides inferred_type is reversed. Look at these lines:
  - Line 337: `elif element is not None:`
  - Line 340: `inferred_type = object`
  - Line 346: `if inferred_type == object and expected_type != object:`
  - Line 347: `raise TypeError(...)`
- Because `element is None` in your test, the branch that should set `inferred_type = object` is not taken. Instead the code sets `inferred_type` to something else (or fails to set it as intended), and then reaches the check at line 346 which raises TypeError when `inferred_type == object` and `expected_type != object`.
- The net effect: when you call ArrayUtils.add(None, None, expected_type=str) the function raises TypeError even though the docstring says "if the element itself is None, the return type is List[object]" (i.e. element None should lead to inferred_type == object and that should not conflict with an expected_type of str in this way).

Root cause (simple)
- The assignment of inferred_type is done in the wrong branch. The code currently sets `inferred_type = object` inside the `elif element is not None:` branch (line 340). It should set `inferred_type = object` when `element is None`. This incorrect branching causes the wrong type-check logic at line 346 and produces the TypeError in this test.

How to fix it (explicit steps and exact change)
1. Replace the inverted logic so that inferred_type is set to object when element is None, and to type(element) when element is not None.
2. Concretely, change the code around lines 337–346 from something like (current, incorrect):
   - 337: `elif element is not None:`
   - 340: `    inferred_type = object`
   - ... other branches ...
   - 346: `if inferred_type == object and expected_type != object:`
to the correct form:
   - if element is None:
       inferred_type = object
   - else:
       inferred_type = type(element)
   - then keep the consistency/type-check logic (line 346) but note it will now behave correctly because inferred_type will be object for element==None.
