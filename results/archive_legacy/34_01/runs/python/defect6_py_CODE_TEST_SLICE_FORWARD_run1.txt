What happened (where it fails)
- The test calls add_number(-0.0).
- The code reaches the branch if x == int(x) ... and treats -0.0 as an integer. int(-0.0) produces 0, and -0.0 == 0.0 is True, so value = int(x) becomes 0 and mantissa = value is also 0.
- Later the code enters the trimming loop:
  while mantissa % 10 == 0:
      check_val = (mantissa // 10) * (10 ** (exp + 1))
      if check_val == value:
           mantissa //= 10
           exp += 1
      else:
           break
  Because mantissa is 0, mantissa % 10 == 0 is True, mantissa // 10 is 0, and check_val == value is 0 == 0 True every iteration. mantissa never changes from 0, so exp keeps increasing and the loop never terminates (infinite loop). The function never returns, so the assertion fails.

Why it happened (root causes)
1. Converting x to int loses the sign of negative zero: int(-0.0) -> 0, so the code can’t preserve the "-0.0" representation.
2. The while condition does not guard against mantissa == 0, so the trimming loop becomes infinite when value is 0.

Exact locations in your code:
- The problem begins at value = int(x) / mantissa = value (the sign loss).
- The infinite loop occurs in the while mantissa % 10 == 0: block inside the if abs(x) >= 100: section.

Concrete fixes (explicit steps)

Option A — simplest: skip integer-branch for signed zero
1. Immediately after computing prev and before the integer branch, detect signed zero and format normally:
   if x == 0.0 and math.copysign(1.0, x) < 0:
       self.add(str(x))
       return
2. Keep the rest of the function unchanged.

This preserves "-0.0" and avoids the integer path that loses the sign.

Option B — prevent the infinite loop (and also preserve sign)
1. After value = int(x); mantissa = value; exp = 0, change the trimming loop condition to avoid running when mantissa == 0:
   if abs(x) >= 100:
       while mantissa != 0 and mantissa % 10 == 0:
           ...
2. Also handle negative zero before converting to int, same as Option A, if you want to guarantee "-0.0" output:
   if x == 0.0 and math.copysign(1.0, x) < 0:
       self.add(str(x))
       return

Option C — avoid int conversion when you need exact string form
1. Only use the integer branch for values that are both integral and not a signed zero: use x.is_integer() and math.copysign to exclude -0.0:
   if x.is_integer() and not (x == 0.0 and math.copysign(1.0, x) < 0):
       value = int(x)
       ...
   else:
       self.add(str(x))

Recommended minimal change
- Add the mantissa != 0 guard in the while condition and special-case signed zero before the integer branch. Concretely:
   if x == 0.0 and math.copysign(1.0, x) < 0:
       self.add(str(x)); return
   ...
   if abs(x) >= 100:
       while mantissa != 0 and mantissa % 10 == 0:
           ...

That fixes the infinite loop and preserves "-0.0".
