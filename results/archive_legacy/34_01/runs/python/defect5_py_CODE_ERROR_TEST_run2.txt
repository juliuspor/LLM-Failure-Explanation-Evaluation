- Immediately afterward, the code treats that sentinel as an actual incompatible type and refuses to proceed when expected_type is specified and not object.
- In short: inferred_type == object (unknown) causes the TypeError even though the only element you’re adding is None and it is safe to make a list typed as str containing None.

- The problematic logic is the combination of (a) setting inferred_type = object to mean “unknown” and (b) immediately treating that as an actual incompatible type against expected_type.
