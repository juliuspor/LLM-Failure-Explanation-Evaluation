What happened (where)
- The test calls LocaleUtils.to_locale('fr__POSIX').
- Execution reaches the code shown in the slice (lines 243, 245–247).
- At line 243 the code already handled some other case and then reaches the code that assumes a two-letter country is present:
  - It uses locale_str[3:5] as the country and then checks locale_str[5] for an underscore (line 245).
- For 'fr__POSIX' the string characters are:
  - index 0='f', 1='r', 2='_', 3='_', 4='P', 5='O', ...
- The code therefore uses locale_str[3:5] == '_P' as the country, then checks locale_str[5] which is 'O' and not '_', so line 246 raises ValueError: "Invalid locale format: fr__POSIX".

Why it failed (root cause)
- The code assumes the country is always two characters long and starts at index 3 (locale_str[3:5]). That assumption is false for locale strings with an empty country and a variant, which use a double underscore: 'language' + '_' + ''(country) + '_' + 'variant' (e.g. 'fr__POSIX').
- For 'fr__POSIX' the country should be the empty string ''. Instead the code slices locale_str[3:5] (giving '_P') and then looks for an underscore at locale_str[5], which does not exist in this layout. The wrong index arithmetic causes the ValueError.

Exactly where to change (concrete steps)
1. Detect the empty-country pattern before using locale_str[3:5]. Concretely, check if locale_str[2] == '_' and locale_str[3] == '_'.
2. If so, return Locale(locale_str[0:2], '', locale_str[4:]) — variant starts at index 4 in 'fr__POSIX'.
3. Otherwise keep the existing logic that expects a two-letter country at locale_str[3:5] and an underscore at the expected position.

Minimal patch (replace lines 243–247)
- Before using locale_str[3:5], add:
  if locale_str[2] == '_' and locale_str[3] == '_':
      return Locale(locale_str[0:2], '', locale_str[4:])
- Then leave the current branch that checks locale_str[5] and returns Locale(locale_str[0:2], locale_str[3:5], locale_str[6:]).

Example outcome after fix
- to_locale('fr__POSIX') -> Locale('fr', '', 'POSIX')
- to_locale('fr_FR_POSIX') -> Locale('fr', 'FR', 'POSIX') (unchanged)

This fixes the index mismatch and stops the ValueError for the empty-country, variant-containing format.
