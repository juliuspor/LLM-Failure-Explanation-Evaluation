What happened (root cause)
- At line 35 the code checks "if index < 0 or index >= len(code_units):" but "index" is an index into the input string (the parameter named s), not into the list code_units.
- code_units starts out empty (line 12: code_units = []), so len(code_units) is 0 while the function is still iterating over the input string. As soon as index is 0 or larger the check "index >= len(code_units)" is true and you raise IndexError, which gives the message "String index out of range: 2" when index == 2.
- In short: the bound check uses the wrong length. It compares the string index to the length of code_units instead of to the length of the input string, so valid string indices are treated as out-of-range.

- Function: to_utf16_code_units(s: str) (line 10)
- code_units initialized at line 12
- The failing check is at line 35 (you raise IndexError using len(code_units) there)

- Because code_units is still empty (or shorter than s while being built), index >= len(code_units) becomes true and triggers the IndexError with the index value (e.g., 2).
