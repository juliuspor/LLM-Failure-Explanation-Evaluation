What happened (root cause)
- At the line `_ = array[i].upper()` (inside your for loop) you call the string method upper() on array[i].
- The exception AttributeError: 'NoneType' object has no attribute 'upper' means array[i] was None for some index i, so None.upper() was executed and failed.
- The stack-slice shows the subsequent line classes.append(type(array[i])) at file/line 767 — that confirms the failing iteration is the one where array[i] is None.

Why it happened (concise)
- The function accepts Optional[List[Any]] (so the list can contain any values, including None). You only check whether array itself is None or empty, not whether any element inside array is None.
- When an element is None, calling .upper() on it raises AttributeError because None has no upper() method.

Where it occurs
- In the for loop:
  - `_ = array[i].upper()` — fails when array[i] is None.
  - Immediately after, `classes.append(type(array[i]))` would be reached for the same i if the previous line did not raise.
