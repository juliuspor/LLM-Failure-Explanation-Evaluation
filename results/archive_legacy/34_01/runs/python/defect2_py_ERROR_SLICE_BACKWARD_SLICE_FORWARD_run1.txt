Why it failed (root cause)
- The code assumes the normalized channel value g will always land between 0 and 255, but it does not ensure that the input value or the bounds guarantee that.
- If value < self._lower_bound, then (value - self._lower_bound) is negative, so g becomes negative and triggers the check at line 74.
- If value > self._upper_bound, then the numerator is larger than the denominator, so g can exceed 255 and also triggers the check.
- The code also does not validate the bounds: if self._upper_bound == self._lower_bound the division will be invalid (ZeroDivisionError) at the calculation on line 71 — that is a separate crash mode you should guard against.
- The error message is generic and gives no data about which variable caused the out-of-range result, so you get no clue whether value, lower_bound, or upper_bound was wrong.

- Or set self._upper_bound == self._lower_bound and call get_paint (this will raise ZeroDivisionError at line 71).
