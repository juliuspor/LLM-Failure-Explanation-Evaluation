What happened (where)
- The test input is "fr__POSIX" and the code fails inside LocaleUtils.to_locale.
- The failure comes from the block around lines 231–247. In particular lines 237–240 raise a ValueError because the code unconditionally treats positions 3 and 4 as a two-letter country code:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(...)

Why it fails (root cause)
- "fr__POSIX" uses an empty country and a variant: language = "fr", country = "" (empty), variant = "POSIX".
- In that form there are two underscores in a row: index 2 == '_' and index 3 == '_', and the variant starts at index 4.
- The code assumes there is always a two-letter country at positions 3 and 4, so ch3 becomes '_' (not an uppercase letter), ch4 becomes 'P', and the check ch3 < 'A' triggers the ValueError on line 239.
- Also related: line 245 checks locale_str[5] != '_' to validate the separator for the variant. That index is correct only when a two-letter country is present (so the second underscore would be at index 5). For the empty-country case the variant separator is at index 3 (the second underscore), so that check is also wrong for "fr__POSIX".

Consequence
- The code does not allow the valid "language__variant" form (empty country), so LocaleUtils.to_locale rejects "fr__POSIX" even though the docstring shows it should accept that and return Locale('fr', '', 'POSIX').

How to fix it (concrete steps)
1. Before treating positions 3 and 4 as a country, detect the empty-country pattern (double underscore) and parse the variant accordingly. For example, insert at or before line 231:
   - if locale_str[2] == '_' and locale_str[3] == '_':
         return Locale(locale_str[0:2], '', locale_str[4:])
   This returns Locale("fr", "", "POSIX") for "fr__POSIX".

2. Only if there is not a double underscore, continue to validate the two-character country at locale_str[3:5] (the existing ch3/ch4 checks).

3. Keep the existing variant-separator check (locale_str[5] != '_') but only in the branch where a two-letter country exists; it must not run for the empty-country branch.

Summary (one-line)
The code treats positions 3–4 as a mandatory two-letter country and rejects the double-underscore empty-country form; detect and handle the "language__variant" case (double underscore) before validating ch3/ch4 to fix the failure.
