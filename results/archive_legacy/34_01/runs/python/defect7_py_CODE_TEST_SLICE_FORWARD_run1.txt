What went wrong
- The code calls array[i].upper() for every element (line just before classes.append). For the test input ['Test', None, 99.0] that fails on the second element because None has no method upper — Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- Because that AttributeError is raised before execution reaches classes.append(type(array[i])), the program crashes there. The trace slice you provided (line with classes.append) is the next line after the failing call, which is why the error appears around that area.

Where it happens (specific names and lines)
- In to_class, within the for loop:
  - array[i].upper() — this is the offending call when array[i] is None (or any non-string that has no upper()).
  - classes.append(type(array[i])) is not reached for that element because the exception stops execution first.

Why the programmer likely wrote .upper()
- It looks like the code was trying to normalize or verify strings, but it does that unconditionally for every element, even when the element is None or a float.

How to fix it (concrete steps)
Option A — only call upper() for strings
1. Replace the unconditional .upper() call with a type check:
   val = array[i]
   if isinstance(val, str):
       _ = val.upper()
2. Then append the type:

Option B — treat None explicitly (if you need to preserve None handling)
1. val = array[i]
2. if val is None:

       continue
   if isinstance(val, str):

   classes.append(type(val))

Option C — simpler if you only need the types (no .upper() side effects)
1. Remove the .upper() call entirely and just build the types list:

Pick the option that matches intent. If .upper() was only intended for validating or converting strings, use Option A or B. If you only need the element types (as your test expects), use Option C.
