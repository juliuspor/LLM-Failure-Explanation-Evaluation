What happened
- The failing test calls cc.add_number(-0.0) and expects the string "-0.0".
- The code that actually turns numbers into text (add_number) treats zero values with a branch like if number == 0: ... which collapses -0.0 to the same output as 0.0 because in Python -0.0 == 0.0 is True.
- That equality check loses the negative sign of negative zero before the text is emitted, so cc.get_code() returns "0.0" (or "0") instead of "-0.0".

Where it occurs
- The provided slice shows add(self, newcode) (lines 126–138). That method is used to append the string produced by add_number.
- The root cause is not in add() itself but in add_number: before add() is called it decided "this number is zero" via number == 0 and emitted the non-negative text. Because -0.0 compares equal to 0.0, the negative sign was discarded there.

Why it happens (plain)
- Python floating-point equality treats -0.0 equal to 0.0. Any code that branches on number == 0 (or number == 0.0) and then produces a fixed zero string will remove the sign for negative zero.
- This is why your test expecting "-0.0" failed: add_number normalized negative zero into the zero case and emitted the non-negative string.

Concrete fixes (step-by-step)
1. Detect negative zero explicitly before you collapse the zero case. Use math.copysign or check 1.0/number:
   - Preferred: import math and use math.copysign(1.0, number) < 0 to tell negative sign even for -0.0.
   - Alternative: test number == 0.0 and math.copysign(1.0, number) < 0 or test number == 0.0 and (1.0/number) == float('-inf').

2. Change add_number so it preserves the sign when number is zero. Example (conceptual):
   - if number == 0.0:
       if math.copysign(1.0, number) < 0:
           text = "-0.0"   # or format with needed precision
       else:
           text = "0.0"
   - else:
       text = the existing non-zero formatting

3. Keep the rest the same so add(self, newcode) (lines 126–138) receives "-0.0" and appends it.

Why this fixes it
- You stop relying solely on equality to detect zero and instead inspect the sign bit. That preserves the negative sign for -0.0 and makes the test cc.add_number(-0.0); result == '-0.0' pass.
