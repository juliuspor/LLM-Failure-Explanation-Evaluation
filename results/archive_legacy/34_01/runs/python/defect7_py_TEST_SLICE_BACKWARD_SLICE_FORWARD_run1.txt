What happened (where it failed)
- The loop in ClassUtils.to_class iterates with i over the input array.
- On line 766 the code calls array[i].upper().
- On your input array = ['Test', None, 99.0] the second element is None. None has no method upper(), so Python raises an AttributeError at line 766.
- Because the exception occurs at line 766, execution never reaches line 767 where classes.append(type(array[i])) would run, so the method never returns [str, type(None), float].

Why it happened (root cause)
- The code assumes every element in array is a string and unconditionally calls the string method upper().
- That assumption is false for your second element (None) and third element (99.0). NoneType and float do not implement .upper(), so the call fails as soon as it encounters a non-string element.

How to fix it (concrete steps and example code)
1. Decide the intended behavior for non-strings: treat strings case-insensitively, but for other types just return their type (so the expected [str, type(None), float]).
2. Replace the unconditional .upper() call with a type check or a safe getattr. Example replacement for lines 765–767:

    for i in range(len(array)):
        if isinstance(array[i], str):
            # use upper only for strings if you need normalized strings
            classes.append(str)
        else:
            classes.append(type(array[i]))

3. Alternative (minimal change) using getattr to avoid AttributeError:

    for i in range(len(array)):
        val = array[i]
        if getattr(val, 'upper', None) is not None:
            # val has .upper, assume it's a string-like object
            classes.append(str)
        else:
            classes.append(type(val))

4. If you truly want to call .upper() on strings but still preserve original value for other types, place the .upper() call inside the isinstance branch:

    for i in range(len(array)):
        val = array[i]
        if isinstance(val, str):
            _ = val.upper()        # safe: val is a str
            classes.append(str)
        else:
            classes.append(type(val))

What to expect after the fix
- With one of the fixes above, calling ClassUtils.to_class(['Test', None, 99.0]) will no longer raise an AttributeError and will return [str, type(None), float] as you expect.
