What happened (what failed)
- Input: "fr__POSIX"
- The code raises ValueError at this block:

      ch3 = locale_str[3]
      ch4 = locale_str[4]
      if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
          raise ValueError(f"Invalid locale format: {locale_str}")

- For "fr__POSIX" these characters are ch3 = '_' and ch4 = 'P'. ch3 ('_') is not between 'A' and 'Z', so the code raises ValueError("Invalid locale format: fr__POSIX").

Why it failed (root cause)
- The code assumes that when length != 2 you will always have a two-letter uppercase country in positions 3 and 4.
- It does not handle the valid locale form that has an empty country field and a variant after a double underscore, e.g. language '_' '_' variant (examples: "fr__POSIX").
- In "fr__POSIX" the country is empty, so position 3 is '_' instead of an uppercase letter. The explicit uppercase check on ch3 causes the rejection.

Where in the code it happens
- After the initial language checks and the underscore at locale_str[2], this exact check of ch3 and ch4 rejects empty-country forms. That is lines:

      if locale_str[2] != '_':
          raise ...
      ch3 = locale_str[3]
      ch4 = locale_str[4]
      if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
          raise ...

How to fix it (exact changes to make)
1. Recognize and accept the empty-country case when locale_str[3] == '_'. In that case:
   - country should be set to "".
   - variant should be locale_str[4:] (ensure it's non-empty).
   - return Locale(language, "", variant).

2. Otherwise keep the existing two-uppercase-letters country validation and behavior.

Concretely, replace the country/variant parsing block with logic like:

    # after confirming locale_str[2] == '_'
    ch3 = locale_str[3]
    if ch3 == '_':
        # empty country, variant starts at index 4
        variant = locale_str[4:]
        if not variant:
            raise ValueError(f"Invalid locale format: {locale_str}")
        return Locale(locale_str[0:2], "", variant)
    else:
        # require two uppercase letters for country
        ch4 = locale_str[4]
        if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
            raise ValueError(f"Invalid locale format: {locale_str}")
        if length == 5:
            return Locale(locale_str[0:2], locale_str[3:5])
        if locale_str[5] != '_':
            raise ValueError(f"Invalid locale format: {locale_str}")
        return Locale(locale_str[0:2], locale_str[3:5], locale_str[6:])

Notes
- The initial length-check logic is fine for allowing 2, 5, or >=7 characters. The real bug is the missing handling of the double-underscore empty country form.
- After applying the change above, "fr__POSIX" will be accepted and produce Locale("fr", "", "POSIX").
