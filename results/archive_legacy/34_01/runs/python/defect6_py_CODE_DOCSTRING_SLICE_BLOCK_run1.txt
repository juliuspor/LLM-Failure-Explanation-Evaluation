What fails
- The code calls math.isinf(...) and math.isnan(...) in add_number (the "if x == int(x) and not math.isinf(x) and not math.isnan(x):" line) but there is no import math anywhere in the shown code. That raises a NameError: name 'math' is not defined when add_number runs.

Where it happens
- The failure occurs inside add_number at the conditional that references math (the line that checks math.isinf and math.isnan). The exception is raised as soon as that line executes.

Why this is the root cause
- In Python you must import a module before using its names. If math was never imported at module top-level, referencing math.* triggers NameError. None of the other shown code (add, get_last_char, etc.) causes that particular exception.

How to fix (concrete steps)
1. At the top of the file (or at least before add_number is used) add:
   import math
2. Alternatively, import only the functions you need:
   from math import isinf, isnan
   and then change the test to:
   if x == int(x) and not isinf(x) and not isnan(x):
3. Run your unit/functional tests to confirm the NameError is gone and the rest of add_number behaves as expected.

Optional note
- If you did already import math elsewhere (in another module or under a different name), make sure that import is in the same module namespace as add_number; module-level imports are required for names used in that module.
