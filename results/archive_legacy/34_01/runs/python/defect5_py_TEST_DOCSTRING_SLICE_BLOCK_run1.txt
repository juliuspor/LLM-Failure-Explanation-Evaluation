What happened (what failed)
- Test: result = ArrayUtils.add(None, None, expected_type=str) expects [None].
- Where it failed: the code at lines 346–351 raised a TypeError instead of returning [None].
- Why it failed: inferred_type is object (because array is None and element is None), expected_type is str, so the condition on line 346 is true:
  if inferred_type == object and expected_type != object:
and that causes the TypeError on lines 347–351. The code treats any expected_type other than object as a mismatch when the inferred element type is object — even when the element itself is None and, per the docstring, the function should return a List[object] in that case.

Root cause (concise)
- The guard at line 346 does not consider that element is None. When element is None the function should allow returning a List[object] even if an expected_type was supplied. The current logic raises an error whenever inferred_type == object and expected_type != object, which incorrectly blocks the documented case (array None, element None).

Suggested fix (precise code change)

For non-None elements, the previous type-check behavior remains unchanged.
