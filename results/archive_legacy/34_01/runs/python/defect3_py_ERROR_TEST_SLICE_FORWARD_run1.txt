- The exception message "String index out of range: 2" means index == 2 but len(code_units) == 2, so code_units[index] is out of range.
- In other words, the caller called slice_forward with index equal to the length of code_units; slice_forward never checked that index < len(code_units) before doing code_units[index].

Why it happened (root cause)
- slice_forward assumes the caller will only call it with a valid index. It only checks index+1 < len(code_units) before accessing the next unit (low), but it does not check index < len(code_units) before accessing the current unit (high).
- Because the caller increments index elsewhere (likely to skip a surrogate pair) and then calls slice_forward again, the caller can pass index == len(code_units). When that happens, line 38 tries to read code_units[index] and triggers the IndexError.

That processing causes the caller to advance the index and eventually call slice_forward at the exact end of the code_units array (index == len), producing the out-of-range read.

- In slice_forward, at line 38: high = code_units[index] — there is no boundary check for index < len(code_units).
