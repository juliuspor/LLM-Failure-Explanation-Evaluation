What happens
- The while loop in add_number that strips trailing zeros never terminates when x == 0.
- Walk-through for x == 0:
  - value = int(x) → 0, mantissa = value → 0, exp = 0.
  - The code enters: while mantissa % 10 == 0:
    - mantissa % 10 == 0 is True for mantissa == 0.
    - check_val = (mantissa // 10) * (10 ** (exp + 1)) evaluates to 0.
    - check_val == value is True, so mantissa //= 10 leaves mantissa == 0 and exp increments.
    - Next loop iteration is identical because mantissa stays 0, so loop never ends (infinite loop).
- Where it occurs: inside add_number, in the loop that begins with "while mantissa % 10 == 0:" (the code block you pasted). The infinite loop causes the program to hang before it ever reaches add() (shown at lines 126–138 in your slice block).

Why it fails (root cause)
- The loop assumes mantissa will shrink to a non-multiple-of-10 and exit. That assumption is false when mantissa == 0: dividing 0 by 10 stays 0, so the loop condition remains true forever.
- The code only checks divisibility (mantissa % 10 == 0) and a reconstruction test (check_val == value) but does not handle the special-case mantissa == 0.

How to fix it (concrete steps)
1. Handle zero explicitly before the loop:
   - Add a check after computing value/mantissa/exp:
     - if mantissa == 0: self.add("0"); return
   This immediately outputs "0" and avoids the loop.

OR

2. Break the loop when mantissa becomes 0:
   - Inside the while, after computing check_val but before repeating, if mantissa == 0: break
   - Or change the loop condition to guard against zero: while mantissa != 0 and mantissa % 10 == 0:

Either fix prevents the infinite loop.

Example patch (option 2, minimal change)
Replace:
    if abs(x) >= 100:
        while mantissa % 10 == 0:
            ...
with:
    if abs(x) >= 100:
        while mantissa != 0 and mantissa % 10 == 0:
            ...

What to watch for
- If you choose option 2, make sure the later code that formats mantissa/exp handles the case mantissa == 0 correctly (you probably want to output "0" rather than "0E..." or an empty string).
- The rest of add_number's logic (checking negative signs with get_last_char, and calling self.add) can remain unchanged.
