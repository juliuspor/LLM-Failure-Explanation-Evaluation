What happened (root cause)
- In the "max middle" block you compare the new period's middle against the wrong stored period. Specifically, inside
  if self._max_middle_index >= 0:
you call
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
instead of using self._max_middle_index. That is a copy/paste bug: you fetch the period for the minimum-middle index rather than the maximum-middle index.

Why that breaks the test
- Your test adds four periods with middles: 50, 150, 25, 75. The expected max middle is 150 (index 1).
- Because the code uses _min_middle_index to compute max_middle, it compares the new middle values against the middle of the min-middle element instead of the true max-middle element. That causes the wrong comparisons and lets index 3 become the recorded _max_middle_index, so the assertion fails.

Where it occurs in the code
- In the block:
    if self._max_middle_index >= 0:
        max_middle_period = self.get_data_item(self._min_middle_index).get_period()
        ...
You should be using self._max_middle_index on the get_data_item call.

Exact fix (concrete steps)
1. Change that single line to use _max_middle_index:
   max_middle_period = self.get_data_item(self._max_middle_index).get_period()
2. Run the existing test:
   s = TimePeriodValues('Test')
   s.add(SimpleTimePeriod(0, 100), 1.0)
   s.add(SimpleTimePeriod(100, 200), 2.0)
   s.add(SimpleTimePeriod(0, 50), 3.0)
   s.add(SimpleTimePeriod(50, 100), 4.0)
   assert s.get_max_middle_index() == 1
   — it should then pass.
