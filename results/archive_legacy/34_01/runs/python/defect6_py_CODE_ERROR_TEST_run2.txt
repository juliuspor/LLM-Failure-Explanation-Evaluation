What happened (root cause)
- The code treats -0.0 as an ordinary zero because the check x == int(x) is true for -0.0 (int(-0.0) == 0), and then you do value = int(x). That converts -0.0 to the integer 0 and drops the negative sign. Later you call self.add(str(value)), which produces "0", so the output is "0" not "-0.0".
- The earlier sign check also misses negative zero: the line if x < 0 and prev == '-': does not detect -0.0 because (-0.0 < 0) is False in IEEE-754 comparison. So you neither preserve the sign there nor later.

Where it happens (specific lines)
- The sign-detection line: prev = self.get_last_char(); if x < 0 and prev == '-': ... — this will not catch -0.0.
- The integer branch start: if x == int(x) and not math.isinf(x) and not math.isnan(x): — this is true for -0.0.
- Then value = int(x) (loses the sign) and finally self.add(str(value)) produces "0".

Why this matters
- IEEE-754 negative zero is distinct from positive zero in its sign bit and when you need to preserve the exact printed representation (tests expect "-0.0"), you must not coerce -0.0 into 0 or into an int without keeping its sign.

How to fix it (concrete steps)
1. Detect negative zero using math.copysign before you do integer conversion. Add an explicit special case so you preserve "-0.0":
   - Example check: if x == 0.0 and math.copysign(1.0, x) < 0:
       self.add(str(x))
       return
   Put this before the integer branch.

2. Use math.copysign for the earlier sign-space logic so it recognizes -0.0:
   - Replace if x < 0 and prev == '-': with if math.copysign(1.0, x) < 0 and prev == '-': so you handle -0.0 correctly there too.

3. (Optional) Alternatively, skip the integer-conversion branch for any value that's exactly 0.0 (positive or negative) and format using str(x) so the sign is preserved.

Concrete minimal change to insert near the top of add_number (showing where to add):
- After prev = self.get_last_char(), add:
    if x == 0.0 and math.copysign(1.0, x) < 0:
        # Preserve negative zero representation
        self.add(str(x))
        return
- And replace the existing sign check:
    if math.copysign(1.0, x) < 0 and prev == '-':
        self.add(" ")

Result
- With those changes, calling cc.add_number(-0.0) will produce "-0.0" and the test assert result == '-0.0' will pass.
