What happened (root cause)
- The loop at slice_union.py lines 765–767 iterates the input list (variable name array).
- On line 766 the code does: _ = array[i].upper()
- When array[i] is None that statement raises AttributeError: 'NoneType' object has no attribute 'upper' because None does not have an .upper() method.
- That is why calling ClassUtils.to_class(['Test', None, 99.0]) fails: the second element is None, so the unconditional .upper() call crashes before the method can append types (line 767).

Where it occurred
- File/section: SLICE_UNION (you showed the excerpt)
- Exact lines: 765–767. The problematic line is 766: array[i].upper()
- Variables involved: array (the input list), i (index), array[i] (element), classes (the list being built)

Why the code likely did this
- The code assumes every element is a string and calls .upper() for normalization, but the input can contain non-strings (None, float, etc.). There is no guard to skip non-string values.

Concrete fixes (pick one that fits your style)
1) Only call .upper() for strings
- Replace line 766 with a type check so non-strings are not touched.

Example change:
- Before:
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))
- After:
  for i in range(len(array)):
      if isinstance(array[i], str):
          _ = array[i].upper()
      classes.append(type(array[i]))

2) Handle None explicitly (if you want special behavior for None)
- If you need to treat None differently, check for it first.

Example:
  for i in range(len(array)):
      if array[i] is None:
          classes.append(type(None))
          continue
      if isinstance(array[i], str):
          _ = array[i].upper()
      classes.append(type(array[i]))

3) Use try/except around .upper() (less preferred because it hides other bugs)
  for i in range(len(array)):
      try:
          _ = array[i].upper()
      except AttributeError:
          pass
      classes.append(type(array[i]))

What to change in practice
- Edit slice_union.py and replace the unconditional array[i].upper() on line 766 with one of the guarded variants above.
- Run the test: result = ClassUtils.to_class(['Test', None, 99.0]) should now return [str, type(None), float].

Why these fixes work
- They avoid calling .upper() on None (or other non-string types), preventing the AttributeError and allowing the code to append the correct type objects (str, type(None), float) at line 767.
