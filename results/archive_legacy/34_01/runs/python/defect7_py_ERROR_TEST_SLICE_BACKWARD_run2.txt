What happened (root cause)
- At line 766 your code does array[i].upper(). That assumes array[i] is a string.
- In the test input the second element is None (array[1] == None). None has no .upper() method, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The error occurs in the loop starting at line 765 — the call that fails is exactly the expression on line 766.

Why it fails for this test
- ClassUtils.to_class(['Test', None, 99.0]) expects a list of types [str, type(None), float].
- Your implementation tries to call .upper() on every element of the input array (line 766) before deciding the type, so it fails as soon as it reaches the None entry. The float entry (99.0) would also fail in the same way if the loop reached it because float objects don’t have .upper() either.

How to fix it (concrete steps; show exactly what to change)
1. Do not call .upper() unconditionally. Check the element’s value/type first.
2. For None, return type(None).
3. For strings, you can use .upper() if you need normalized content; for type detection you can simply use type(item) or isinstance checks.
4. For other values, use type(item).
