What went wrong
- Where: in add_number, inside the while loop that starts with "while mantissa % 10 == 0:".
- What happened: when x is 0.0 (so value == 0 and mantissa == 0) the condition mantissa % 10 == 0 is true, but dividing mantissa by 10 leaves mantissa at 0. That means mantissa never changes, exp keeps increasing forever, and the loop never terminates (infinite loop / hang).
- Why it happens: 0 has arbitrarily many trailing zeros, and the loop assumes mantissa will shrink to a non-zero number. For any non-zero integer with trailing zeros this works because integer division by 10 reduces the number; for 0 it does not.

Exact place in code
- Method: add_number(self, x: float)
- Variables: value = int(x), mantissa = value, exp = 0
- Faulty loop: while mantissa % 10 == 0: ... mantissa //= 10; exp += 1
- Trigger: value == 0 (x == 0.0)

How to fix it (concrete steps)
1. Add a guard so the loop runs only when mantissa is non-zero:
   - Change the loop condition to:
     while mantissa != 0 and mantissa % 10 == 0:
   This prevents the infinite loop for value == 0.

2. Alternatively, handle zero early:
   - After computing value, add:
     if value == 0:
         self.add("0")
         return
   This avoids the trimming logic entirely when x is zero.

3. Add tests to cover the case:
   - Call add_number(0.0) and confirm it returns "0" (or whatever formatting you expect).
   - Also test a few non-zero cases like 1000.0, -100.0, and 12.0 to ensure previous behavior is preserved.

Minimal concrete patch
Replace:
    while mantissa % 10 == 0:
with:
    while mantissa != 0 and mantissa % 10 == 0:

Why this is safe
- For non-zero integers with trailing zeros, mantissa != 0 is true and the loop behaves exactly the same.
- For zero, the loop is skipped and the code proceeds (or you can explicitly return "0" earlier), avoiding the infinite loop.
