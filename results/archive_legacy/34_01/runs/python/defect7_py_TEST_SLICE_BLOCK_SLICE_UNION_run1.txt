What happened (where and what error occurs)
- The loop in ClassUtils.to_class raises an exception at line 766:
  - Line 765: for i in range(len(array)):
  - Line 766:     _ = array[i].upper()
  - Line 767:     classes.append(type(array[i]))
- If array contains a non-string element (your test uses None at index 1 and 99.0 at index 2), the call array[i].upper() fails.
  - With None at index 1 you get: AttributeError: 'NoneType' object has no attribute 'upper' (raised on line 766). The float at index 2 would give: AttributeError: 'float' object has no attribute 'upper' if execution reached it.
- Because the code tries to call .upper() on every element unconditionally, it crashes before appending the element types. That is why result never becomes [str, type(None), float] — the exception prevents the loop from finishing.

Why it happens (root cause)
- The code assumes every array element is a string and therefore has an upper() method. This assumption is false for None and float values. The attempted method call on objects that don't implement upper() triggers an AttributeError at the exact location line 766.

Exact fixes (replace the loop and how to change lines)
Option A — explicit type checks (recommended)
Replace lines 765–767 with:

for i in range(len(array)):
    val = array[i]
    if isinstance(val, str):
        _ = val.upper()               # still OK for strings
        classes.append(str)
    elif val is None:
        classes.append(type(None))
    else:
        classes.append(type(val))

Why this fixes it:
- You only call .upper() when val is a str, avoiding method calls on None or float. You append the correct types for each element.

Option B — use hasattr or try/except
Replace lines 765–767 with either:

# hasattr approach
for i in range(len(array)):
    val = array[i]
    if hasattr(val, "upper"):
        _ = val.upper()
    classes.append(type(val))

or

# try/except approach
for i in range(len(array)):
    val = array[i]
    try:
        _ = val.upper()
    except AttributeError:
        pass
    classes.append(type(val))

Why these work:
- hasattr prevents calling upper on objects that don't have it.
- try/except catches the AttributeError so the loop continues to append types.

Additional small improvements
- Iterate directly over elements instead of index-based access:
  for val in array:
      ...
- Append the canonical types (str, type(None), float) rather than type(array[i]) if you want normalized class names (your test expects str, not e.g. builtins.str vs something else — using isinstance checks ensures you append str for string-like values).

Summary
- The code fails at line 766 because it calls .upper() on elements that are not strings (None and float). Fix by checking the element's type before calling .upper(), or by using hasattr/try-except, and then append the correct type (lines shown above).
