What happened (what failed)
- The test calls GrayPaintScale.get_paint(-0.5) expecting black (0, 0, 0).
- At line 73 the code computes g from the fraction (value - lower) / (upper - lower). With the defaults (lower = 0.0, upper = 1.0) and value = -0.5 that fraction is negative, so g becomes a negative integer.
- Immediately after, lines 76–77 check g and raise ValueError when g < 0 or g > 255. That raise is why the test fails: instead of returning (0, 0, 0) the method raises an exception.

Where it occurred (exact places)
- __init__ sets default bounds (line 32).
- get_paint computes g at line 73:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- get_paint then raises if g is out of [0,255] at lines 76–77:

      raise ValueError(f"Color parameter outside of expected range: Red Green Blue")

Why that is the root cause
- The test expects out-of-range inputs to be clamped to the range (so values < lower -> black, > upper -> white). Instead, the code treats any computed channel outside 0–255 as an error and raises, so negative input ends execution with ValueError rather than returning a color.
