What happened (where and why)
- Where it failed: in Locale.to_locale at the checks around lines 236–252. The exception was raised at line 244–245:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
      raise ValueError(f"Invalid locale format: {locale_str}")

- Why it failed for the input "fr__POSIX":
  - The string "fr__POSIX" has two underscores in a row: lowercase language "fr", no country code, then variant "POSIX".
  - The code assumes that when an underscore is present after the language it must be followed by a two-letter uppercase country code located at indices 3 and 4. For "fr__POSIX", locale_str[3] is '_' (not an uppercase letter), so the check fails and raises the ValueError.
  - In short: the function does not accept the pattern language + empty country + variant (ll__VARIANT). It expects either just language ("fr"), language+country ("fr_FR"), or language+country+variant ("fr_FR_VARIANT"). It does not handle a missing country with a present variant, hence the exception.

Summary
The code raises ValueError because it assumes a two-letter country will appear after the first underscore and explicitly checks characters at indices 3 and 4. "fr__POSIX" has an empty country (double underscore), so those characters are not uppercase letters and the check fails. Fix by parsing with split and allowing an empty middle field (or otherwise explicitly rejecting "ll__VARIANT" if you intend not to support it).
