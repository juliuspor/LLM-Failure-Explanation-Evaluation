What happens

- At line 236 the code tests if length == 2, but immediately inside that branch (lines 242–244 and later) it reads characters at indices 3, 4 and 5:
  - ch3 = locale_str[3] (line 242)

Why it fails (root cause)
- The branch condition at line 236 is wrong: it uses length == 2 for code that expects a longer string (country and optional variant). In other words, the code that validates/reads the country and variant is placed under the wrong length check. Because the branch condition does not match the code’s expectations, short inputs reach code that accesses indexes that don't exist.

Where it occurs
- Method: to_locale
- Lines: The mismatch occurs at line 236 (the if length == 2) and the later index accesses at lines 242–244 and 250–252.
