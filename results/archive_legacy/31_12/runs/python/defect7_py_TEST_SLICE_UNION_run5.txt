What happened (where it failed)
- The failure occurs inside SLICE_UNION at lines 768–770:
  - Line 769 calls array[i].upper() unconditionally.
  - Line 770 appends type(array[i]) to classes.
- In your test, array contains ['Test', None, 99.0]. When the loop reaches the None element, line 769 tries to run None.upper(), which raises AttributeError "'NoneType' object has no attribute 'upper'". The code never reaches line 770 for that element, so the function never returns the expected types [str, type(None), float].

Why it happened (root cause)
- The code assumes every element in array is a string and calls .upper() without checking the element’s type first.
- None and float do not implement .upper(), so an exception is raised when a non-str element is encountered.

Exactly where to change (what to edit)
- File: SLICE_UNION
- Lines to edit: 768–770
  - Current:
    for i in range(len(array)):
        _ = array[i].upper()
        classes.append(type(array[i]))
  - Problem: line 769 calls .upper() on any element; this is the direct cause of the AttributeError.

Concrete fixes (choose one)

1) Minimal, explicit type guard (recommended)
- Replace lines 768–770 with:
  for i in range(len(array)):
      val = array[i]
      if isinstance(val, str):
          _ = val.upper()
      classes.append(type(val))

- Why: only calls .upper() when val is a str; types for None and float are still appended.

2) Use getattr to avoid AttributeError
- Replace with:
  for i in range(len(array)):
      val = array[i]
      upper_fn = getattr(val, 'upper', None)
      if callable(upper_fn):
          _ = upper_fn()
      classes.append(type(val))

- Why: checks presence of upper method at runtime and only calls it if available.

3) Try/except (less preferred because it hides other bugs)
- Replace with:
  for i in range(len(array)):
      try:
          _ = array[i].upper()
      except AttributeError:
          pass
      classes.append(type(array[i]))

- Why: catches the specific AttributeError, but is less explicit about intent.
