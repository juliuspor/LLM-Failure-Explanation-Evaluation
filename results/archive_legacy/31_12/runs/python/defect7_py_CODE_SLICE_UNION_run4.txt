What happened (root cause, where it happened)
- The failure occurs inside to_class, specifically at the loop on lines 768–770:
  - Line 769 calls array[i].upper().
  - Line 770 then calls type(array[i]).
- The crash is caused because array[i] is not always a string. Calling .upper() on a non-string (for example a type object or None) raises an AttributeError like:
  "'type' object has no attribute 'upper'" or "'NoneType' object has no attribute 'upper'".
- In short: you are calling the string method upper() on values that are not strings.

Why this is happening (clear explanation)
- The method name to_class suggests the input array might already contain class objects (types) or a mix of strings and types.
- If an element is already a class (e.g., SomeClass), array[i].upper() is invalid because class objects do not implement upper().
- If an element is None, array[i].upper() is also invalid.
- The code does not check the element type before calling .upper(), so any non-string element causes the error.

Concrete resolve_class_by_name example (simple):

      module = importlib.import_module(module_name)
      return getattr(module, class_name)
