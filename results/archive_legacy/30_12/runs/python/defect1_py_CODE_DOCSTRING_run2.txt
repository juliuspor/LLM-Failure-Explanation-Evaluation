What happened (root cause)
- In for_offset_hours_minutes the minutes validation is wrong:
  - The code currently does
    if minutes_offset < 0 or minutes_offset > 59:
        raise ValueError(...)
  - That rejects any negative minute value. According to your docstring minutes_offset is allowed to be negative (range -59..+59). Because of this the allowed inputs like (0, -15) and (-2, -15) are incorrectly rejected.

Where it occurs
- In the method for_offset_hours_minutes, the minutes-range check line is incorrect (the line that reads "if minutes_offset < 0 or minutes_offset > 59:").

Why it fails (examples)
- Input (0, -15) should produce +00:15? (doc says (0,-15) -> -00:15) but the code raises ValueError because minutes_offset < 0.
- Input (-2, -15) should be treated as -02:15 (total -135 minutes), but the code raises ValueError for the same reason.

Also missing a required check
- Your docstring forbids some sign combinations: specifically "positive hours with negative minutes" should raise ValueError (e.g. (2, -15)). The current code does not explicitly check that; it would accept negative minutes if you fix the range check, so you must add that check to enforce the rule.

How to fix it (concrete steps)
1. Change the minutes-range validation to allow -59..+59:
   Replace
     if minutes_offset < 0 or minutes_offset > 59:
         raise ValueError(f"Minutes out of range: {minutes_offset}")
   with
     if minutes_offset < -59 or minutes_offset > 59:
         raise ValueError(f"Minutes out of range: {minutes_offset}")

2. Add an explicit sign-combination check to reject positive hours with negative minutes (per your docstring):
   Insert after the hours range check:
     if hours_offset > 0 and minutes_offset < 0:
         raise ValueError("Positive hours with negative minutes not allowed")

3. (Optional) Keep the special-case (0,0) -> UTC and the rest of the computation as-is. The rest of the code that combines hours and minutes works correctly once the validations are fixed.

Minimal corrected version (showing only the replaced/added checks)
    if hours_offset == 0 and minutes_offset == 0:
        return cls.UTC
    if hours_offset < -23 or hours_offset > 23:
        raise ValueError(f"Hours out of range: {hours_offset}")

    # disallow positive hours with negative minutes per docstring
    if hours_offset > 0 and minutes_offset < 0:
        raise ValueError("Positive hours with negative minutes not allowed")

Why this fixes it
- It matches the documented range for minutes (-59..59).
- It enforces the documented forbidden sign combination (positive hour + negative minutes).
- After these checks, the existing combination logic and final conversion to milliseconds will produce the intended offsets.
