Why it happens (root cause)
- Earlier in the method the code obtains character values with ord(), e.g.:

  so ch3 is an int (Unicode code point).
- The code then compares those integers to string literals 'A' and 'Z':
    ch3 < 'A'
  Python cannot compare an int to a str, so the comparison raises a TypeError before the ValueError branch can be reached.
- In short: mismatched types in comparison — ints vs. strings — cause the crash at line 244.
