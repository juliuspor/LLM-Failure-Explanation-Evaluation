What happened (where it fails)
- The test calls LocaleUtils.to_locale('fr__POSIX').
- In LocaleUtils.to_locale, the input is split on '_' into three parts: ['fr', '', 'POSIX'].
- The method then runs the country-code validation branch and rejects the empty country because the code requires the country part to have length 2. That validation raises ValueError and the test fails instead of returning Locale('fr', '', 'POSIX').

Why it fails (root cause)
- The implementation assumes that when there are three parts (language, country, variant) the middle part must be a 2-letter country. It does not allow the case where the country part is intentionally empty (length 0) but a variant is present (the canonical form "fr__POSIX").
- In other words, the validation is too strict: it enforces country length == 2 unconditionally for the 3-part form, rejecting the legitimate "language__variant" form.

Exactly where to look in the code
- Inspect LocaleUtils.to_locale, in the branch that handles the 3-part case after splitting on '_' (the block that validates the country string). That is the country-validation check that currently does something like:
  if len(country) != 2 or not country.isupper(): raise ValueError(...)
- This check is what causes the failure for country == ''.

How to fix it (concrete steps)
1. Open LocaleUtils.to_locale and find the branch that handles three parts (language, country, variant).
2. Replace the strict country-length check with logic that allows an empty country when a variant is present. Concretely:
   - Accept country == '' (length 0) OR country length == 2.
   - If country is non-empty, still require it to be uppercase and alphabetic.
3. Keep the language and separator validations as-is (language must be lowercase, separators must be underscores).
4. Add/adjust tests to ensure:
   - LocaleUtils.to_locale('fr__POSIX') returns Locale('fr', '', 'POSIX')
   - LocaleUtils.to_locale('fr_FR_POSIX') still returns Locale('fr', 'FR', 'POSIX')
   - LocaleUtils.to_locale('fr_POSIX') (2-part) behaves as intended (if that form is supported)

Minimal code change (illustrative)
- Replace the strict check on country with:

    # after splitting: language, country, variant
    if country == '':
        # empty country is allowed when a variant is present
        parsed_country = ''
    else:
        if len(country) != 2 or not country.isupper():
            raise ValueError("country must be 2 uppercase letters or empty when variant present")
        parsed_country = country

- Then construct and return Locale(language, parsed_country, variant).

Why this fix is correct
- It exactly matches the documented behavior: "en_GB_xxx = Locale('en','GB','xxx')" and also allows the documented "fr__POSIX = Locale('fr', '', 'POSIX')".
- It does not weaken country validation for the usual two-letter country case, only permits the documented empty-country-with-variant form.
