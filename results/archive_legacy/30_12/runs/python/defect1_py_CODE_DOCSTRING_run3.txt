What happened (where it fails)
- The bug is in for_offset_hours_minutes, inside the try block where you combine hours and minutes into a single minutes count:
  if hours_in_minutes < 0:
      minutes_offset = hours_in_minutes - minutes_offset
  else:
      minutes_offset = hours_in_minutes + minutes_offset
- Those two lines reuse the minutes_offset input variable and perform the wrong arithmetic when both hours and minutes are negative. For example:
  - Input: hours_offset = -2, minutes_offset = -15
  - hours_in_minutes = -120
  - Your code computes minutes_offset = -120 - (-15) = -105 → total -1:45, but the expected total is -2:15 = -135 minutes.
- Root cause: when hours are negative you must subtract the absolute value of the minutes (treat minute magnitude as additional negative minutes). Subtracting a negative minutes value (i.e. -(-15)) unintentionally adds minutes instead of making the offset more negative. In short: sign handling for negative hours/minutes is wrong, and you also overwrite the input minutes_offset which makes the logic harder to inspect.

Other issues the code omits (contradicting the docstring)
- The docstring says "ValueError: ... minutes are negative when the hours are positive", but the method does not check that hours>0 and minutes<0 and raise. That invalid combination currently is not rejected.
- Overwriting minutes_offset hides intent; use a new variable total_minutes for clarity.

How to fix it (concrete steps)
1. Do not overwrite minutes_offset. Introduce total_minutes.
2. Enforce the doc rule: if hours_offset > 0 and minutes_offset < 0 -> raise ValueError.
3. When hours_offset < 0, subtract the absolute minutes: total_minutes = hours_in_minutes - abs(minutes_offset).
4. Otherwise (hours >= 0), add minutes: total_minutes = hours_in_minutes + minutes_offset.
5. Use total_minutes when converting to milliseconds.

Corrected code (drop-in replacement for the try block and following lines)

        if hours_in_minutes < 0:

        offset = FieldUtils.safe_multiply(total_minutes, DateTimeConstants.MILLIS_PER_MINUTE)
    except ArithmeticError:

Why this fixes it
- Using abs(minutes_offset) for negative hours makes -2 hours and -15 minutes become -120 - 15 = -135 (correct).
- Using a separate total_minutes variable avoids accidental reuse/misinterpretation of the input variable.
- Checking the +hours / -minutes combination enforces the docstring contract and avoids ambiguous or invalid offsets.
