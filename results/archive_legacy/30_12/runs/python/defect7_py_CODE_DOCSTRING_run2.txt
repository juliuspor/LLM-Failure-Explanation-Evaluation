What fails
- The failure occurs inside to_class at the line:
  _ = array[i].upper()
  (this is the third line inside the for loop).
- That line calls .upper() on every element of array. If array[i] is None (or any object that does not have an upper method, e.g. an int, list, custom object), Python raises AttributeError: 'NoneType' object has no attribute 'upper' (or similar). That is the immediate exception you see.

Why it fails (root cause)
- The code calls a string method (.upper()) unconditionally on each element. The docstring says NULL/None elements should become None in the output and that the function converts objects to their Class (type). Those requirements do not require calling .upper() at all. Calling .upper() is incorrect because many valid elements (None or non-string objects) don’t implement upper(), so the call throws AttributeError before you can return the list of types.

Where it happens
- In method to_class, inside the for loop:
  for i in range(len(array)):
      _ = array[i].upper()    <-- raises AttributeError here for None / non-strings
      classes.append(type(array[i]))

What actually happens versus what you likely intended
- Actual: iteration stops with AttributeError when encountering a None or non-str element.
- Intended (per docstring): a None element should cause a None entry in the returned list; non-None elements should be replaced by their type (Class) — no .upper() needed.

Concrete fixes (step-by-step)
Decide which behavior you want (two options):

A) If you want None elements preserved (match the docstring)
1. Remove the .upper() call entirely.
2. Check for None and append None; otherwise append type(...).

Example corrected code:
def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
    if array is None:
        return None
    elif len(array) == 0:
        return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    classes = []
    for item in array:
        if item is None:
            classes.append(None)
        else:
            classes.append(type(item))
    return classes

B) If you intended to validate that all elements are strings and raise on None
1. Replace the unconditional .upper() with an explicit check and raise a clear error.
2. If you want to preserve the AttributeError behavior, raise it with a clear message:

Example:
def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
    if array is None:
        return None
    elif len(array) == 0:
        return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    classes = []
    for item in array:
        if item is None:
            raise AttributeError("Element is None at to_class; expected non-None string")
        if not isinstance(item, str):
            raise TypeError(f"Expected str, got {type(item).__name__}")
        # if you actually wanted to use .upper() for validation, do so explicitly:
        _ = item.upper()
        classes.append(type(item))
    return classes

Which fix to pick
- If you follow your docstring (None => None in result), use option A.
- If the .upper() was meant as validation for strings, use option B but make the intent explicit and raise informative errors instead of letting implicit AttributeError occur.

Summary
- Root cause: calling .upper() on elements that are None or not strings causes AttributeError.
- Fix: remove the .upper() call, or add explicit checks and raise clearer errors depending on the intended behavior.
