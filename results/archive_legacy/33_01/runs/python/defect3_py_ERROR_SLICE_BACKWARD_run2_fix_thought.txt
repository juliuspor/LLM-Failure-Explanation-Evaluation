The bug arises from mixing codepoint vs code unit indexing leading to IndexError. The translate function should iterate over code_units and advance pos by char_count of codepoint consumed. Current implementation uses translate_codepoint which returns number of codepoints consumed; but later code increases pos by char_count(code_point_at(code_units,pos)) in a loop which may re-read pos after it was advanced. Simpler fix: when consumed>0, advance pos by the number of UTF-16 code units corresponding to the codepoints consumed. Compute total_units=0 and for i in range(consumed): cp=code_point_at(code_units,pos+total_units); total_units+=char_count(cp). Then pos+=total_units. Also need to handle consumed==0 as before. Ensure out not None handled. Return type None.
