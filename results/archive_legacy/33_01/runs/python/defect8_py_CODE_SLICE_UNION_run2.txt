What fails
- The method to_locale (defined at line 188) is written with a first parameter named cls, but it is not marked with @classmethod. When you call Locale.to_locale("en") Python does not automatically bind the class to the first parameter, so the call ends up with the wrong arguments and raises a TypeError (missing the locale_str argument) before any of the parsing code runs.

Why it happens (root cause)
- In Python, functions defined inside a class behave differently depending on whether they are instance methods, class methods, or static methods:
  - Instance methods expect an instance (self) as the first argument and are bound when accessed via an instance.
  - Class methods expect the class (cls) as the first argument and must be decorated with @classmethod to be bound when accessed via the class.
  - Static methods expect no automatic first argument and must be decorated with @staticmethod.
- Your signature def to_locale(cls, locale_str: str) -> Optional[Locale]: indicates you intended cls to be the class, but without @classmethod Python treats it as a plain function (an unbound function when accessed on the class). Calling Locale.to_locale("en") therefore supplies "en" to the cls parameter and leaves locale_str missing, causing the TypeError.

- The failure happens at the call site when calling Locale.to_locale(...). The parsing lines (222 onward, and checks at 231, 237–247) are never reached because the method binding fails first.
