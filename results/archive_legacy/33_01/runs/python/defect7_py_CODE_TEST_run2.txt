What happens (root cause), where it happens, and why:
- In the for loop you call array[i].upper() on this line:
  _ = array[i].upper()
- That line runs for every element. When array[i] is None there is no .upper method, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'. The same would happen if an element is a float (99.0) because float has no .upper either.
- Because that exception is raised, execution never reaches the next line that appends the element's type:
  classes.append(type(array[i]))
- So the method fails at the first non-string element instead of returning [str, type(None), float] as the test expects.

How to fix it (concrete steps and exact replacement code):
Option A — remove the unnecessary .upper() call (simple and correct for the shown intent):
Replace the whole for loop with:
for i in range(len(array)):
    classes.append(type(array[i]))

Or more Pythonic:
for val in array:
    classes.append(type(val))

Option B — preserve .upper() only for strings (if you intended to normalize strings before using them):
for val in array:
    if isinstance(val, str):
        _ = val.upper()   # only called for strings
    classes.append(type(val))

Option C — tolerate non-string objects via try/except (less preferred for control flow):
for val in array:
    try:
        _ = val.upper()
    except AttributeError:
        pass
    classes.append(type(val))

Notes:
- The failing line is exactly "_ = array[i].upper()" inside the for loop; remove or guard that call.
- Using type(None) is the correct way to represent None's type; your append line is fine.
- After fixing the .upper call, the test result = ClassUtils.to_class(['Test', None, 99.0]) will produce [str, type(None), float] as expected.
