What happened (symptom)
- The code can hang (infinite loop) when add_number is called with x == 0.0. You never return or append anything in that case because the while loop never terminates.

Where it happens (exact lines / variables)
- In add_number, when x is an integer-valued float the code sets:
  - value = int(x) (line under "if x == int(x) ...")
  - mantissa = value
  - exp = 0
- Then this loop runs (the while on the next lines):
  while mantissa % 10 == 0:
      check_val = (mantissa // 10) * (10 ** (exp + 1))
      if check_val == value:
           mantissa //= 10
           exp += 1
      else:
           break
- For x == 0.0 you have value == 0 and mantissa == 0. mantissa % 10 == 0 is true, and:
  check_val = (0 // 10) * (10 ** (exp + 1)) == 0
  check_val == value is true
  mantissa //= 10 leaves mantissa == 0
  exp += 1
  The loop condition remains true forever, so the method never finishes.

Why it happens (root cause)
- The loop intends to strip trailing zeros from an integer (e.g. 1000 -> 1E3), but it has no guard against mantissa being zero. Zero has infinite trailing zeros mathematically, so the loop never makes progress for value == 0 (mantissa stays 0), producing an infinite loop.

Concrete fixes (exact steps you can apply)
Option A — simplest and explicit: handle zero before the loop
1. Right after setting value = int(x) add:
     if value == 0:
         self.add("0")
         return
2. This immediately writes "0" and exits before the problematic loop.

Option B — add a loop guard so mantissa must be nonzero
1. Change the while condition to require mantissa != 0:
     while mantissa != 0 and mantissa % 10 == 0:
         ...
2. This prevents the infinite loop; if mantissa becomes 0 the loop stops. (You should still decide how you want to format 0 — probably "0" — because with this guard you'd fall through to self.add(str(value)) and get "0".)

Which to choose
- Option A is explicit and clear: handle the special value zero immediately.
- Option B is minimal and corrects the loop condition; it leaves the existing fall-through behavior (self.add(str(value))) which returns "0" anyway.

One-line patch (Option B)
- Replace
     while mantissa % 10 == 0:
  with
     while mantissa != 0 and mantissa % 10 == 0:
