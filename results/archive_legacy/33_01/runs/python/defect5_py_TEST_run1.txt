What happens
- The test call is: result = ArrayUtils.add(None, None, expected_type=str)  (test file, call site).
- That call fails because ArrayUtils.add treats the value and/or the array as if they must already be a concrete type and it rejects None. In other words, either the method:
  - tries to call list operations on the first argument (arr.append(...)) when arr is None, causing an AttributeError, or
  - enforces expected_type immediately with isinstance(value, expected_type) and treats False for None as a type-mismatch (so it either raises a TypeError or refuses to add the value).

Why this is the root cause (where it occurs)
- In ArrayUtils.add (the implementation), there are two problematic assumptions:
  1. The parameter named arr is assumed to be a list instance and used directly (arr.append(...)). If arr is None (as in your test), arr.append raises AttributeError: 'NoneType' object has no attribute 'append'.
  2. The parameter named value is checked against expected_type with a plain isinstance(value, expected_type). For expected_type=str, isinstance(None, str) is False. If the code treats that result as an error (for example: if not isinstance(value, expected_type): raise TypeError(...)), the method will reject None even if you want to allow None entries.

Concrete fixes (step-by-step)
1. Make the method accept arr being None by initializing it:
   - Replace the start of ArrayUtils.add with:
     if arr is None:
         arr = []
   This prevents AttributeError when you call arr.append(...).

2. Allow None values when expected_type is provided (if that is the intended behavior). Change the type check so None bypasses it:
   - Replace:
       if not isinstance(value, expected_type):
           raise TypeError("...")
     with:
       if value is not None and not isinstance(value, expected_type):
           raise TypeError("...")

3. Return the array (existing behavior), so the test result will become [None].

Minimal patch example (inside ArrayUtils.add):
- Before:
    def add(arr, value, expected_type=None):
        if expected_type and not isinstance(value, expected_type):
            raise TypeError("wrong type")
        arr.append(value)
        return arr
- After:
    def add(arr, value, expected_type=None):
        if arr is None:
            arr = []
        if value is not None and expected_type and not isinstance(value, expected_type):
            raise TypeError("wrong type")
        arr.append(value)
        return arr

Why this fix works
- Initializing arr when it is None prevents attribute errors on arr.append.
- Skipping the type-check for value is None lets callers intentionally add None into the list while still enforcing types for non-None values.
