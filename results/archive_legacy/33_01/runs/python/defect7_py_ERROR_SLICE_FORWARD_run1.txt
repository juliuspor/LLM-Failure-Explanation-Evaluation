What happened (root cause)
- The AttributeError "'NoneType' object has no attribute 'upper'" means code tried to call .upper() on a value that is None.
- In the slice you showed, line 767 does classes.append(type(array[i])). That line reveals the runtime type of array[i]. The presence of NoneType there tells us array[i] is None at that index.
- Therefore somewhere earlier in the code you call array[i].upper() (or otherwise call .upper() on a variable that holds array[i]). Because that element is None, Python raises the AttributeError.

Where it occurred
- The failing call is the .upper() call performed on the array element that is None.
- Line 767 (classes.append(type(array[i]))) is useful diagnostic evidence: it shows the type for the problematic element is NoneType. The actual exception stack trace will point to the exact line where .upper() was invoked; use that to confirm the call site.

Why it happened
- Your code assumes every element of array (or whatever variable you used) is a string (or has an upper() method), but at least one element is None.
- Common causes:
  - Missing or failed initialization that left a slot as None
  - An earlier operation that sets values to None on certain branches
  - Unexpected input data containing null/None values
