The original to_locale function incorrectly rejects strings with an empty country (format lang__variant like "fr__POSIX") because it always expects two uppercase letters at positions 3 and 4. The fix is to detect the second underscore at position 3 indicating an empty country, and extract the variant accordingly. Also validate separators and character cases for other forms.
