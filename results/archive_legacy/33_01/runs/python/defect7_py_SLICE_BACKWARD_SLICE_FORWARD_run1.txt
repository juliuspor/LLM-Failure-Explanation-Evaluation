What happened (where it failed)
- The code fails on line 766: `_ = array[i].upper()`.
- The exception you see will be an AttributeError like: "'int' object has no attribute 'upper'" (or "'NoneType' object has no attribute 'upper'").
- Because of that exception, execution never reaches line 767 (`classes.append(type(array[i]))`).

Why it failed (root cause)
- Not every element in `array` is a string. The `.upper()` method exists for str objects only.
- When an element like an int, None, or some other non-str value is encountered at `array[i]`, calling `.upper()` raises AttributeError and stops the loop.

How to confirm quickly
- Print the type and value just before calling `.upper()`:
  - Insert before line 766:
    `print(i, type(array[i]), repr(array[i]))`
  - Run; when you see a non-str type, that is the failing element.

Concrete fixes (step‑by‑step options you can choose from)

1) Only call .upper() for str elements (preferred when non-strings should be ignored)
- Replace lines 765–767 with:
  ```
  for i in range(len(array)):
      if isinstance(array[i], str):
          _ = array[i].upper()
          classes.append(type(array[i]))
      else:
          # handle or skip non-strings
          classes.append(type(array[i]))
  ```
- This prevents the AttributeError and still records the element type.

2) Convert everything to string before uppercasing (when you want an uppercase representation of every item)
- Use str(...) instead of assuming a str:
  ```
  for i in range(len(array)):
      _ = str(array[i]).upper()
      classes.append(type(array[i]))
  ```
- This will not fail, but note it changes the value (e.g., 123 -> "123").

3) Use try/except to continue on errors (when failure cases are rare and you want to skip them)
- Replace with:
  ```
  for i in range(len(array)):
      try:
          _ = array[i].upper()
      except AttributeError:
          # optional: log or handle array[i]
          continue
      classes.append(type(array[i]))
  ```

4) If you intended to collect types first and then use uppercase on strings, change the order
- Move the append before calling .upper():
  ```
  for i in range(len(array)):
      classes.append(type(array[i]))
      if isinstance(array[i], str):
          _ = array[i].upper()
  ```

Which to pick
- If non-strings are expected and should be skipped: option 1.
- If you want a string representation for every element: option 2.
- If you want to skip only when .upper is missing and not stop the whole loop: option 3.
- If you simply wanted types collected before doing string work: option 4.

Summary
- Root cause: calling `.upper()` on a non-str at line 766 raises AttributeError, preventing line 767 from running.
- Fix by checking `isinstance(..., str)`, converting with `str(...)`, using try/except, or reordering the operations depending on intended behavior.
