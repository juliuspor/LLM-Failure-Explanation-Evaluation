What failed
- Root cause: on this line you call type(array) (first branch of the if) to set inferred_type:
  inferred_type = type(array)
  That returns the Python list class (e.g. <class 'list'>), not the element (component) type stored in the list. The rest of the code treats inferred_type as the element/component type and passes it into ArrayUtils._copy_list_grow1 and compares it to object — so you end up using the wrong type everywhere.
- Where it fails: the bad value is computed on the first branch (the if array is not None branch) and then used on this line:
  new_list = ArrayUtils._copy_list_grow1(array, inferred_type)
  and later when assigning:
  new_list[len(new_list) - 1] = element
  Because inferred_type is the list class, _copy_list_grow1 and any type-checking/casting logic that expects an element type will be given a wrong type, which causes incorrect copies or a ClassCastException-like situation when storing element into the new list.
- Why this produces the observed error: you intended inferred_type to represent the element type (e.g. int, str, MyClass). Using type(array) gives you list — so inferred_type == object checks will be wrong, and the created list's component type will be wrong, causing a runtime type mismatch when assigning element into new_list.

Exactly what to change (step-by-step)
1. If array is not None and non-empty, use the element type of an element inside the list, not type(array). For example:
   if array is not None and len(array) > 0:
       inferred_type = type(array[0])
2. If array is not None but empty, fall back to element or expected_type:
   elif element is not None:
       inferred_type = type(element)
   elif expected_type is not None:
       inferred_type = expected_type
   else:
       inferred_type = object
3. Keep the rest of the function the same, but now _copy_list_grow1 will receive a proper element type.

Concrete replacement for the problematic block
Replace:
    if array is not None:
        inferred_type = type(array)
    elif element is not None:
        inferred_type = type(element)
    else:
        inferred_type = object

With:
    if array is not None and len(array) > 0:
        inferred_type = type(array[0])
    elif element is not None:
        inferred_type = type(element)
    elif expected_type is not None:
        inferred_type = expected_type
    else:
        inferred_type = object

Why this fix works
- inferred_type will now represent an element/component class (e.g. int, str, MyClass) when possible.
- _copy_list_grow1 receives the correct element type and will create a list with the right component expectations.
- The assignment new_list[len(new_list) - 1] = element will not fail due to a mismatched component type, and the inferred_type == object checks will behave as intended.

Edge cases to be aware of
- If array is non-empty but contains mixed types, type(array[0]) may not reflect all elements. Decide whether that is acceptable or whether you should examine all elements or use expected_type as authoritative.
- If array is an instance of a subclass of list with different semantics, you still want the element type, not type(array).
