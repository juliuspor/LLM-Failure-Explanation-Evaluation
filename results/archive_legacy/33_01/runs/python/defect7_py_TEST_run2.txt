What happened
- The test calls ClassUtils.to_class(['Test', None, 99.0]) and asserts the result should be [str, type(None), float].
- The test fails because the element for the None input is the value None, not the class object type(None). In other words result is [..., None, ...] instead of [..., type(None), ...].

Why it failed (root cause)
- Inside ClassUtils.to_class there is a special-case branch that handles None values and appends the literal None into the result list instead of the NoneType class object.
- Concretely, there is a line like:
  if value is None:
      classes.append(None)
  That appends the value None, not the class type(None). The test expects the class object type(None), so the equality comparison fails.

Where it occurred (exact places to check)
- Inspect ClassUtils.to_class. Look for the branch that checks for None (for example a line containing "if value is None:" or "if val is None:").
- Check the code immediately under that condition: it likely does classes.append(None) or returns None for that element.

How to fix it (clear actionable steps)
1. Open the ClassUtils.py (or the file that defines ClassUtils) and find the to_class method.
2. Locate the None handling branch (the "if value is None:" block).
3. Replace the append of the bare None with the NoneType class. Two safe ways:
   - Use type(value): classes.append(type(value))  # works for all values, including None
   - Or explicitly append type(None): classes.append(type(None))
4. Ensure the method returns a list of class objects for all inputs (use type(x) for non-class inputs).
5. Run the test again.

Minimal suggested patch (example)
- Replace:
    if value is None:
        classes.append(None)
  with either:
    if value is None:
        classes.append(type(None))
  or better, remove the special-case entirely and use:
    classes.append(type(value))

Why that fix works
- type(None) is the class object representing the NoneType. The test expects that class object, not the literal None value.
- Using type(value) makes the function consistent: every element maps to its class object (str for "Test", type(None) for None, float for 99.0).

What to verify after the fix
- Run the test: ClassUtils.to_class(['Test', None, 99.0]) should return [str, type(None), float].
- Also add a unit test for a list containing None to prevent regression.
