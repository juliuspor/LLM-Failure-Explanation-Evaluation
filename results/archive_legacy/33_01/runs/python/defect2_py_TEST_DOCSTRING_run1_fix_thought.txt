The original method used the raw input value when computing the gray level instead of the clamped value and raised ValueError when the computed channel was out of 0-255. I will clamp the input to the bounds, compute the normalized ratio using the clamped value, compute the gray channel using rounding, then clamp the final channel to the 0-255 range and return the RGB tuple. This ensures inputs outside the bounds map to black or white appropriately instead of raising an exception.
