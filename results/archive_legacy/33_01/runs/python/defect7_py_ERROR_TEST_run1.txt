What happened (root cause)
- In ClassUtils.to_class you call value.upper() to test or normalize items that are strings.
- When the list contains None (the variable name is value inside the method), Python tries to evaluate None.upper() and raises AttributeError: 'NoneType' object has no attribute 'upper'.
- That exact call is where the crash occurs (inside ClassUtils.to_class on the line that reads value.upper()) because value can be None but the code assumes it's always a str.

Why that causes the failing test
- Your test calls ClassUtils.to_class(['Test', None, 99.0]).
- For the first element 'Test' the code likely succeeds and returns str.
- For the second element None the code calls None.upper() and fails before it can return type(None) as the expected result.

Precise place to change
- Open the method ClassUtils.to_class and find the line that reads something like:
  value_upper = value.upper()
  or
  if value.upper() == 'SOMETHING': ...
- That line must be guarded because value may be None or any non-str object.

Concrete fixes (step-by-step)
1. Check the type before calling .upper():
   - Use isinstance(value, str) to detect strings.
   - If value is None, return type(None) (or append type(None) to the result list).
   - For other non-string values, use type(value).

2. If the method intends to accept class-name strings (e.g., 'Test' -> builtins.Test or mapped class), only call .upper() inside an isinstance(value, str) branch.

3. Example corrected implementation (replace the body of ClassUtils.to_class with this):

   def to_class(values):
       results = []
       for value in values:
           if value is None:
               results.append(type(None))         # preserve None as type(None)
               continue
           if isinstance(value, str):
               # normalize and map string names to classes (example uses builtins)
               name = value.strip()
               # map common names -> builtins; adjust mapping to your needs
               mapping = {'str': str, 'int': int, 'float': float, 'NoneType': type(None)}
               results.append(mapping.get(name, getattr(__import__('builtins'), name, None)))
               continue
           # for actual objects (like 99.0) return their type
           results.append(type(value))
       return results

4. Add tests for None explicitly:
   - result = ClassUtils.to_class(['Test', None, 99.0])
   - assert result == [str, type(None), float]

Notes and pitfalls
- Don’t assume every input supports string methods like .upper(). Always check with isinstance(..., str) or handle None first.
- If your code expects class objects or names, be explicit about which inputs are allowed and document behavior for None and for unknown names.
- If you want unknown string names to raise a clear error, raise ValueError with a message instead of returning None from getattr.
