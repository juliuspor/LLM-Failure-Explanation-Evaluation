What happened (where and what raised the error)
- The ValueError is raised inside to_locale at the check that validates the two-character country code:
  - Specifically at the lines:
    ch3 = locale_str[3]
    ch4 = locale_str[4]
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(f"Invalid locale format: {locale_str}")
- For the input "fr__POSIX" those variables are:
  - ch3 == '_' and ch4 == 'P'
- The condition finds ch3 ('_') is not between 'A' and 'Z', so it raises ValueError: Invalid locale format: fr__POSIX.

Why it fails (root cause)
- The code assumes that after the language and a single underscore (locale_str[2] == '_') there will always be a two-letter uppercase country code at positions 3 and 4. It does not handle the valid pattern where the country is empty and represented by a second underscore (language + "__" + variant), e.g. "fr__POSIX".
- In "fr__POSIX" the second underscore occupies locale_str[3], so the code incorrectly treats that underscore as the first letter of the country code and therefore rejects it.

Exactly where to change and how (concrete steps)
1. After confirming locale_str[2] == '_' handle the special case of an empty country:
   - If locale_str[3] == '_' then the country is the empty string and the variant starts at index 4. Accept that and return Locale(language, "", locale_str[4:]) (or validate that variant exists if you want).
2. Otherwise keep the current flow that validates two uppercase letters at locale_str[3] and locale_str[4], and for a variant require locale_str[5] == '_' and variant = locale_str[6:].
3. Also, re-check the initial length check if you intended to accept long variants; but the immediate bug is the missing handling of the double-underscore case.

Minimal example fix (illustrative)
Replace the block that begins after checking locale_str[2] == '_' with:

    # after locale_str[2] == '_'
    # handle empty country: language + "__" + variant
    if locale_str[3] == '_':
        # variant starts at index 4
        return Locale(locale_str[0:2], "", locale_str[4:])  # or validate length/variant present
    # otherwise validate 2-letter country at 3..4
    ch3 = locale_str[3]
    ch4 = locale_str[4]
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(...)
    # proceed to return Locale(language, country) or with variant as before

Why this fix works
- It explicitly recognizes the valid "language + __ + variant" form (empty country) and sets country = "" with the variant sliced from the correct start index (4), avoiding the failed uppercase-letter check that currently raises the ValueError.

Additional note
- If you expect variants to always exist when there are two underscores, validate that locale_str[4:] is non-empty and raise a clear error if it's empty. Also confirm the intended index arithmetic for your Locale constructor (the current code assumes variant starts at index 6 only when a country is present).
