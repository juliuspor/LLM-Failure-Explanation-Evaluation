What happened (where & why)
- The error comes from Locale.to_locale at the check around lines 237–240.
- The code does:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - then checks that ch3 and ch4 are between 'A' and 'Z', otherwise raises ValueError(f"Invalid locale format: {locale_str}").
- For the input "fr__POSIX":
  - indexes: 0='f', 1='r', 2='_', 3='_', 4='P', ...
  - ch3 is '_' (locale_str[3]), not an uppercase letter, so the test fails and the ValueError is raised.
- Root cause: the function assumes that when a locale string has a third character (index 2) of '_' then positions 3–4 contain an uppercase two-letter country code. It does not handle the case where the country code is empty and a variant follows (double underscore, like "fr__POSIX"). The code treats the '_' at index 3 as invalid because it expects letters there.

What to change (concrete steps)
1. Parse the locale into parts instead of hard indexing:
   - Use split with a max of 2 parts: parts = locale_str.split('_', 2)
   - parts[0] = language, parts[1] = country (may be ''), parts[2] = variant (if present)
2. Validate each part explicitly:
   - language: 2-letter lowercase alphabetic
   - country: either empty or 2 uppercase letters
   - variant: optional, non-empty when present (allow uppercase/lowercase/digits/underscore as appropriate)
3. Replace the hard index checks at lines 237–240 with those validations.

Minimal example fix (pseudo-Python you can drop into to_locale)
- parts = locale_str.split('_', 2)
- if not (len(parts[0]) == 2 and parts[0].islower() and parts[0].isalpha()):
    raise ValueError(...)
- if len(parts) >= 2:
    country = parts[1]
    if country and not (len(country) == 2 and country.isupper() and country.isalpha()):
        raise ValueError(...)
- if len(parts) == 3:
    variant = parts[2]
    if not variant:
        raise ValueError(...)

Why this fixes it
- Using split handles "fr__POSIX" as ["fr", "", "POSIX"] so you can accept an empty country and a present variant.
- Explicit checks avoid misinterpreting the underscore at index 3 as invalid characters.

Summary
The code fails because it expects a non-empty two-letter country at positions 3–4 but the input has an empty country and a variant (double underscore). Replace the fixed-index checks with a split-and-validate approach so "fr__POSIX" is parsed as language="fr", country="", variant="POSIX" and accepted.
