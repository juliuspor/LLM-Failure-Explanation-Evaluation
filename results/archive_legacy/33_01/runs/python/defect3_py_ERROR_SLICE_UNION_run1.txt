- The IndexError means the caller asked for index = 2 but code_units does not contain a code unit at position 2.
- Two concrete reasons this can happen:
  1. code_units was not populated as the caller expects. For example, the code that is supposed to append code units to code_units (the loop that converts s into UTF-16 code units) either did not run or appended fewer items than expected.   2. The caller and this function disagree on what index means: the caller may be passing a character index (code point index) instead of a UTF-16 code unit index. Non-BMP characters use two UTF-16 code units (a surrogate pair). If the caller counts characters but then asks for a code unit by that same index, they can be off by 1 for each surrogate pair present, causing an index to be out of range. The code around lines 38–43 deals with surrogates, so this function expects code_units to be a UTF-16 code unit list, not a list of code points.
