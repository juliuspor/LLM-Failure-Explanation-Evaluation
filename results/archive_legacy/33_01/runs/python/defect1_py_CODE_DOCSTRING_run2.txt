What went wrong (where it fails)
- On lines that validate minutes (the third if-block):
  if minutes_offset < 0 or minutes_offset > 59:
      raise ValueError(f"Minutes out of range: {minutes_offset}")

  This rejects any negative minutes. The docstring explicitly allows negative minutes in some cases (e.g. (0, -15) and (-2, -15)). So valid inputs are incorrectly refused here.

- On the lines that combine hours and minutes into a total minute offset:
  hours_in_minutes = hours_offset * 60
  if hours_in_minutes < 0:
      minutes_offset = hours_in_minutes - minutes_offset
  else:
      minutes_offset = hours_in_minutes + minutes_offset

  This logic treats minutes_offset as if it were always a positive magnitude when hours_in_minutes is negative. For a negative hour with a negative minute the code subtracts a negative value (hours_in_minutes - (-15)) and produces the wrong total. Example:
  - Input: hours_offset = -2, minutes_offset = -15
  - hours_in_minutes = -120
  - Code computes minutes_offset = -120 - (-15) = -105
  - Resulting total = -105 minutes = -1:45 (wrong). The desired total is -135 minutes = -2:15.

- Example B (allowed by current validation but computed incorrectly): (-2, -15)
  - Passes the current checks because minutes are between -59 and 59? (actually current code rejects negatives, so this specific example would already be rejected; if you remove the first bug, the computation below still miscomputes)
  - Incorrect computation: yields -1:45 instead of -2:15.

Why this is the root cause (short)
- The code enforces minutes to be non-negative, contradicting the docstring.
- The arithmetic that combines hours and minutes mishandles the sign of minutes when hours are negative: it subtracts minutes_offset rather than subtracting the absolute minutes magnitude, so negative minutes end up reducing the magnitude of the negative offset.

     - If hours_offset < 0 and minutes_offset < 0: allow it (but handle sign correctly).

Summary

Fix the minutes range check to allow -59..+59 and enforce the sign rule for positive hours, then compute total minutes using abs(minutes) with the correct sign depending on hours.
