What fails
- At line 71 (in get_paint) you divide by (self._upper_bound - self._lower_bound). If self._upper_bound equals self._lower_bound that denominator is 0 and Python raises ZeroDivisionError at that line. The code never gets to the check at line 74 because the division already blew up.

Why that happens (root cause, tied to the code)
- Variables involved: value, self._lower_bound, self._upper_bound.
- If self._upper_bound == self._lower_bound, the expression
  (value - self._lower_bound) / (self._upper_bound - self._lower_bound)
  attempts a /0 division. That is the direct cause of the failure at line 71.

Secondary problems you should fix while you are here
- The check at lines 74–75 (if g < 0 or g > 255: raise ValueError(...)) only runs after the division, so it does not protect you from division-by-zero.
- The error message raised at line 75 is unhelpful: f"Color parameter outside of expected range: Red Green Blue" does not include which variable, what g was, or which bounds caused the error.

Concrete steps to fix (exact changes and where)
1. Guard the denominator before line 71:
   - Insert a check before the division, e.g. right before line 71:
     if self._upper_bound == self._lower_bound:
         raise ValueError(f"get_paint: upper_bound equals lower_bound ({self._upper_bound}) — cannot map value {value}")
   This makes the failure explicit and occurs before any division.

2. Clamp the computed ratio so g cannot go out of 0..255 (prevents the ValueError at lines 74–75 or makes it unlikely):
   - Replace the division and g calculation around line 71 with:
     denom = self._upper_bound - self._lower_bound
     ratio = (value - self._lower_bound) / denom
     ratio = max(0.0, min(1.0, ratio))   # clamp to [0.0, 1.0]
     g = int(round(ratio * 255.0))
   This prevents g from being <0 or >255 due to values outside the bounds or floating point rounding.

3. Improve or remove the manual range check at lines 74–75:
   - Either keep it but make the message informative:
     if g < 0 or g > 255:
         raise ValueError(f"get_paint: computed green {g} out of range for value {value} with bounds "
                          f"{self._lower_bound}-{self._upper_bound}")
   - Or remove the check if you apply the clamping in step 2.

Summary
- Immediate cause: division by zero at line 71 when self._upper_bound == self._lower_bound.
- Fix by checking the denominator before dividing, clamp the ratio to [0,1], and replace the generic error message with one that includes the offending value and bounds.
