The bug arises because translate advances pos incorrectly after translators consume codepoints. It uses char_count(code_point_at(code_units, pos)) inside a loop over consumed, but code_point_at will raise IndexError if pos advances beyond length; also when consumed represents codepoints count, we should advance by code units corresponding to those codepoints. Simpler approach: after a translator returns consumed > 0, advance pos by the total UTF-16 code units of the consumed codepoints by repeatedly reading code_point_at at current pos and adding char_count, but ensure we don't call code_point_at with pos==length. Also handle case consumed larger than remaining codepoints: clamp. Implement loop that for i in range(consumed): if pos>=length: break; cp = code_point_at(code_units,pos); pos += char_count(cp).
