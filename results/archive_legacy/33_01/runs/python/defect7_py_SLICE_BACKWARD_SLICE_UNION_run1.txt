What failed
- The crash happens at line 766: the call array[i].upper().
- The exception is an AttributeError like: "'int' object has no attribute 'upper'" (or "'NoneType' object has no attribute 'upper'").
- This occurs when array contains a value that is not a string (for example an int or None). Non-string values do not have an .upper() method, so the call fails immediately.

Why it happens (root cause)
- Your code assumes every element of array is a str and calls .upper() unconditionally.
- That assumption is false: at least one element in array is a different type. When Python evaluates array[i].upper() at line 766, it tries to call upper() on whatever object is found there; if that object has no upper method, Python raises AttributeError and the loop stops.

Where it occurs
- File/section shown: lines 765–766 (both variants). The failure is raised exactly at line 766 during the first iteration that encounters a non-string element.
- In the [SLICE_UNION] variant you append type(array[i]) on line 767 only after calling .upper(), so that debug information is never reached for the failing element.

How to fix it (concrete steps)
1. Decide how you want to treat non-strings (skip, convert, or fail with clearer error).
2. Implement one of these explicit checks before calling .upper().
