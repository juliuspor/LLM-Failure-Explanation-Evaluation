What happened (where it failed)
- Call: DateTimeZone.for_offset_hours_minutes(-2, -15)
- The code path that turns hours + minutes into total minutes has incorrect sign handling at these lines:
  - line 402: minutes_offset = hours_in_minutes - minutes_offset
  - line 404: minutes_offset = hours_in_minutes + minutes_offset
- Because both hour and minute arguments are negative, the code ends up combining the two signs inconsistently and computes the wrong total minutes. That wrong total gets converted to millis at line 405 and passed to for_offset_millis at line 408, so you either get the wrong offset or an exception later.

Why it failed (root cause, exact variables and logic)
- The code treats minutes_offset inconsistently depending on sign. It appears to assume minutes_offset is always positive in one branch and negative in another, and it adds/subtracts hours_in_minutes incorrectly.
- For the specific input hours = -2 and minutes = -15:
  - hours_in_minutes should represent -120 (or an absolute 120 combined with an overall sign).
  - minutes_offset should contribute -15 to the total minutes.
  - Instead, the code branches in a way that either subtracts a negative minutes_offset (turning -15 into +15) or adds it when it should subtract, so the minutes get the wrong sign relative to hours.
- Result: you get +2:15 (or otherwise wrong total) instead of -2:15. The bug is a sign-mismatch when combining hours_in_minutes and minutes_offset at lines 402 and 404.

How to fix it (concrete steps)
1. Make the sign handling explicit and unified. Compute total_minutes with a single expression that respects the sign of hours and then applies minutes with the same sign, for example:
   - total_minutes = hours * 60 + (1 if hours >= 0 else -1) * abs(minutes)
   Or equivalently:
   - if hours >= 0:
       total_minutes = hours * 60 + abs(minutes)
     else:
       total_minutes = hours * 60 - abs(minutes)
2. Replace the logic around lines 402–404 with that single calculation so you don’t mix addition/subtraction in different branches.
3. Keep the subsequent steps unchanged: convert total_minutes to millis (line 405) and call for_offset_millis (line 408).

Minimal code change example (pseudo-Python to drop into those lines)
- Assume hours and minutes are the input arguments, and hours_in_minutes = hours * 60:
  minutes_abs = abs(minutes)
  if hours >= 0:
      minutes_offset = hours * 60 + minutes_abs
  else:
      minutes_offset = hours * 60 - minutes_abs

This guarantees that for hours = -2, minutes = -15 you get minutes_offset = -120 - 15 = -135, which becomes -02:15 after conversion to milliseconds.
