What happens
- The infinite loop happens inside the while loop that tries to strip trailing zeros from mantissa:
  while mantissa % 10 == 0:
      check_val = (mantissa // 10) * (10 ** (exp + 1))
      if check_val == value:
           mantissa //= 10
           exp += 1
      else:
          break

Why it fails (root cause)
- If value (and therefore mantissa) is 0, the loop condition mantissa % 10 == 0 is true.
- Then check_val is computed as (0 // 10) * (10 ** (exp + 1)) = 0, so check_val == value is true.
- mantissa //= 10 sets mantissa = 0 // 10 = 0 — mantissa does not change.
- exp increments, but mantissa remains 0, so the next loop iteration repeats the same steps forever.
- In short: for value == 0 the code never makes progress because integer division of 0 by 10 stays 0, so the while loop never terminates.

Where it occurs (specific variables / lines)
- value = int(x) and mantissa = value (top of the integer branch).
- The problematic loop is the while mantissa % 10 == 0: block that computes check_val and does mantissa //= 10.
- The failing case is triggered when x == 0.0 (so value == 0 and mantissa == 0).

How to fix it (specific steps)
Option A (preferred, minimal change):
- Change the loop condition to prevent running when mantissa == 0:
  while mantissa != 0 and mantissa % 10 == 0:
      ...
This prevents the infinite loop for value == 0.

Option B (explicit early return for zero):
- After computing value, add:
  if value == 0:
      self.add("0")
      return
- This handles zero explicitly and avoids entering the trailing-zero stripping code.

Why these fixes work
- They prevent the loop from executing when mantissa is 0, so mantissa will never remain unchanged inside the loop causing an infinite loop.

Notes
- No other cases produce the infinite loop because for any nonzero mantissa dividing by 10 eventually changes mantissa and the loop can terminate either by mantissa % 10 != 0 or by check_val != value.
