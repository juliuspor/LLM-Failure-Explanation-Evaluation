Why it happened (root cause)
- Negative zero is a distinct floating-point sign bit: -0.0 is a float with a negative sign. Converting that float to an integer or otherwise coercing it to a non-float canonical form will remove the sign and produce 0.
- The common causes inside append are things like:
  - calling int(value) (int(-0.0) -> 0),
  - using value == 0 or value * 0 to normalize,
  - converting to a string form that drops the sign, then parsing back as int,
  - or otherwise storing values as ints rather than floats.
- Because append is dropping the float sign, the caller/test sees 0 instead of -0.0.
