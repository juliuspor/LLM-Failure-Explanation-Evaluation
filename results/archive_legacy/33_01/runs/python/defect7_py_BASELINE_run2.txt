What happened (where it fails)
- In to_class (lines 765–767) the code loops over the input array:
  - Line 766 calls array[i].upper().
  - When the input is ['Test', None, 99.0] the second element is None.
  - None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- So the failure occurs at line 766 when array[i] is None.

Why it fails (root cause)
- The code assumes every element is a string and calls .upper() without checking for None or non-string types.
- None does not have an upper attribute, so calling .upper() on None raises AttributeError.
- The docstring says "If any of these objects is None, a None element will be inserted into the array" but the implementation contradicts that by calling .upper() unconditionally and thus crashing before inserting type(None).

Concrete fixes (specific changes and steps)
Option A — simplest, follow docstring: stop calling .upper() at all
- Replace the loop (lines 765–767) with:
  for elem in array:
      classes.append(type(elem))
- Why: you only need the element's class; calling .upper() is unnecessary and causes the error.

Option B — if the .upper() call was intended to validate strings, check for None and strings first
- Replace the loop with:
  for elem in array:
      if elem is None:
          classes.append(type(None))
      elif isinstance(elem, str):
          _ = elem.upper()   # only call upper if elem is a str
          classes.append(str)
      else:
          classes.append(type(elem))
- Why: this avoids calling .upper() on None or non-strings and preserves any string-processing intent.
