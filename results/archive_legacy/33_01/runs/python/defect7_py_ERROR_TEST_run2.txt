What happened (root cause)
- The error came from ClassUtils.to_class when it tried to call value.upper(). The exception AttributeError: 'NoneType' object has no attribute 'upper' means value was None at runtime, so None.upper() failed.
- In your test you passed the list ['Test', None, 99.0]. When the method iterates that list it reaches the second element (the variable/value that is None) and then calls .upper() on it. That is where the failure occurs.

Where it occurred
- Inside ClassUtils.to_class, on the line that does something like:
  value.upper()
  (or uses value.upper() inside an expression). This is the exact place the AttributeError is raised because value is None for the second element.

Why it happened
- The code assumes every element is a string (or another object that implements upper()), so it unconditionally calls .upper() to decide which class to return. None does not have an upper method, so the assumption is wrong for lists that contain None (or other non-string objects, e.g. numbers).

How to fix it (concrete steps)
1. Decide the intended behavior for None and non-string inputs:
   - If None should map to type(None) (as your test expects), handle None explicitly.
   - If non-strings like floats should be handled by type detection, handle them explicitly too.

2. Change the code to check the value before calling .upper(). Example fixes you can apply inside to_class:

Option A — explicit None and non-string handling
- Replace the line that calls value.upper() with code like:
  if value is None:
      result_type = type(None)
  elif isinstance(value, str):
      name = value.upper()
      # existing string-based logic using name
  else:
      result_type = type(value)
  Append result_type to the result list.

Option B — do a safe string conversion only for string inputs
- If you only want to treat strings as type names and otherwise return the object's type:
  if isinstance(value, str):
      name = value.upper()
      # map name to a type (e.g., 'TEST' -> str class)
  else:
      result_type = type(value)

Option C — defensive try/except (less preferred because it hides logic)
  try:
      name = value.upper()
  except AttributeError:
      if value is None:
          result_type = type(None)
      else:
          result_type = type(value)

3. Add unit tests that cover these cases:
   - ClassUtils.to_class(['Test', None, 99.0]) should return [str, type(None), float]
   - ClassUtils.to_class([None]) should return [type(None)]
   - ClassUtils.to_class([99.0]) should return [float]
   - ClassUtils.to_class(['test']) should still return [str]

Summary
- The method called value.upper() without checking that value is a string. When value was None the call failed. Fix by checking value is not None and/or is a str before calling .upper(), and explicitly return type(None) for None values (and use type(value) for non-string objects if that’s the intended behavior).
