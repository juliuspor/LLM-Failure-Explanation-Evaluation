What happened (root cause)
- In ClassUtils.to_class you iterate over the list with the element variable but then treat that element as an index. The failing line is:
  - line 767: classes.append(type(array[i]))
- If the loop is written like "for i in array:" then i is the element (for the first element i == 'Test'), so array[i] tries to index the list with a string. That raises TypeError: "list indices must be integers or slices, not str" and makes the test fail.

Why this explains your failing test
- Your test calls ClassUtils.to_class(['Test', None, 99.0]).
- On the first loop iteration i == 'Test'. The code does array['Test'] which is invalid, so the function never returns [str, type(None), float] and the assertion fails (or you get an exception before the comparison).

Where it occurs
- Method: ClassUtils.to_class
- Exact line: 767 (classes.append(type(array[i])))

How to fix it (concrete steps)
1. If the loop is "for i in array:" then use the element directly:
   - Replace classes.append(type(array[i])) with classes.append(type(i))
2. If you intended to loop by index, change the loop to use a range:
   - for i in range(len(array)):
       classes.append(type(array[i]))
3. Alternatively, use enumerate if you need the index and the value:
   - for idx, val in enumerate(array):
       classes.append(type(val))

Example corrected code (one-line change)
- Change:
  for i in array:
      classes.append(type(array[i]))
- To:
  for i in array:
      classes.append(type(i))

Why the fixes work
- Using type(i) uses the actual element (string, None, float) so you get str, type(None), float.
- Using range(len(array)) makes i an integer index, so array[i] is valid.

This gives the expected output for your test: [str, type(None), float].
