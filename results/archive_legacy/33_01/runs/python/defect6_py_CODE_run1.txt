What fails: a NameError at the line

    if x == int(x) and not math.isinf(x) and not math.isnan(x):

Why it fails (root cause)
- The code calls math.isinf and math.isnan but never imports the math module. Python raises NameError: name 'math' is not defined when it reaches those names.
- The failure occurs at that line (the first use of math in add_number), so execution stops before the integer-trimming while loop or any later code runs.

Exactly where to fix it
- Add an import for the math module at the top of the file (or otherwise make math available in the function scope).

Concrete steps to fix
1. At the top of the module add:
   import math
2. Optionally, replace the two calls with the single clearer test using math.isfinite:
   if x == int(x) and math.isfinite(x):
   (math.isfinite covers both not isinf and not isnan)
3. Run the unit(s) that exercise add_number to confirm the NameError is gone and the rest of the function behaves as expected.

If you prefer not to import math, you can use:
- float('inf') comparisons (less recommended), or
- from math import isfinite, isnan, isinf and call those names directly.

That is the direct, single root cause; no change to the mantissa/exp logic is required to fix the NameError.
