The bug is an IndexError when advancing pos after a translator reports it consumed codepoints. The existing loop calls code_point_at(pos) without ensuring pos is still within bounds for each iteration, which can raise IndexError. Fix by guarding access: while consumed_count > 0 and pos < length, read codepoint_at, advance by its char_count; if pos >= length break. This avoids out-of-range calls and correctly advances over remaining code units. Also keep existing behavior for consumed==0 branch unchanged.
