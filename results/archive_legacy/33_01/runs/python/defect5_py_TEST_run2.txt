What happened (root cause)
- The call is: result = ArrayUtils.add(None, None, expected_type=str)
- ArrayUtils.add treats the second parameter as the element to add (call it element or value) and does a type check because expected_type is provided.
- Inside ArrayUtils.add there is a check similar to:
  if expected_type is not None and not isinstance(element, expected_type):
      raise TypeError("element has wrong type")
- For your call, element is None, expected_type is str, and isinstance(None, str) is False. The type check fails and ArrayUtils.add raises an exception before returning [None], so the assertion is never reached.

Where it occurs
- In ArrayUtils.add, at the line that does the type validation (the line that calls isinstance or otherwise enforces expected_type on element). That is the exact spot that raises the error.

Why that is the problem
- None is not an instance of str, so the strict type check rejects the intended value None. The code assumes expected_type excludes None unless the caller explicitly permits it.

Clear fixes (choose one)
1) Remove expected_type when you want to allow None:
   result = ArrayUtils.add(None, None)  # do not pass expected_type=str

       raise TypeError("element has wrong type")
   # change to

       raise TypeError("element has wrong type")

4) Alternatively, document that expected_type=str means "no None allowed" and make callers choose accordingly.

Which to pick
- If you want None to be a valid element, use (2) or (3).
- If you want expected_type to mean "must be a str", use (1) or keep current behavior and avoid passing None.
