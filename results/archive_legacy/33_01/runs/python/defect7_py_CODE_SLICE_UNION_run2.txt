What failed
- At lines 765–767 you call array[i].upper() and then do classes.append(type(array[i])).
- If array contains a non-string (for example an int, None, or a custom object), the call array[i].upper() raises an AttributeError like:
  AttributeError: 'int' object has no attribute 'upper'
- The exception originates exactly at line 766 (the call to .upper()), so the loop never reaches the append on line 767 for that element.

Why it fails (root cause)
- .upper() is a string method. You are calling it unconditionally on every element of array, but not every element is necessarily a str. That mismatch (calling a string method on a non-string) is the direct cause of the crash.

Where it occurred
- In the for loop at lines 765–767:
  - array[i].upper() — line 766 — triggers the AttributeError when array[i] is not a string.
  - classes.append(type(array[i])) — line 767 — is never reached for that element because of the exception.

Clear fixes (concrete steps)
1. If you don’t need .upper() at all:
   - Remove the line array[i].upper() entirely. The method classes.append(type(array[i])) already gives you the element types.
   - Replacement:
     for i in range(len(array)):
         classes.append(type(array[i]))

2. If you only want to call .upper() for string elements and ignore/accept non-strings:
   - Check the element type before calling .upper():
     for i in range(len(array)):
         if isinstance(array[i], str):
             _ = array[i].upper()
         classes.append(type(array[i]))

3. If you require all elements to be strings and want a clear error when they are not:
   - Validate and raise a TypeError with a helpful message:
     for i in range(len(array)):
         if not isinstance(array[i], str):
             raise TypeError(f"array[{i}] must be str, got {type(array[i]).__name__}")
         _ = array[i].upper()
         classes.append(type(array[i]))

4. If you want to tolerate objects that implement .upper() (duck typing):
   - Use try/except to handle the absence of .upper():
     for i in range(len(array)):
         try:
             _ = array[i].upper()
         except AttributeError:
             pass
         classes.append(type(array[i]))

Any of these make the cause explicit and avoid the AttributeError. Choose the fix that matches your intended contract for array (strings-only vs mixed types).
