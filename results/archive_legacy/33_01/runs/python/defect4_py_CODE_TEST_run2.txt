What happened
- The code that updates self._max_middle_index uses the wrong index to fetch the comparison period. On this line:
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  you fetch the period at self._min_middle_index instead of at self._max_middle_index.
- Because of that copy/paste error, the routine compares the new middle value against the period currently stored at _min_middle_index, not the one at _max_middle_index. That makes the max-middle update logic use the wrong reference value and therefore fail to pick the true maximum middle index.

Where it occurs (exact lines)
- In the "max middle" section:
  if self._max_middle_index >= 0:
      max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  ...
- That single line should use self._max_middle_index, not self._min_middle_index.

Why that triggers your failing test
- Your test adds the following SimpleTimePeriods (start,end): (0,100), (100,200), (0,50), (50,100).
- The true middle values are: 50, 150, 25, 75. The maximum middle is 150 at index 1, so s.get_max_middle_index() should be 1.
- Because the code compares against the period at _min_middle_index, it may never update _max_middle_index correctly (it compares against a smaller middle or a wrong value), causing get_max_middle_index() to return the wrong index and the assert to fail.

How to fix it (precise changes)
- Replace the incorrect line:
    max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  with:
    max_middle_period = self.get_data_item(self._max_middle_index).get_period()

- Full context of the corrected block (only changed variable):
    if self._max_middle_index >= 0:
        max_middle_period = self.get_data_item(self._max_middle_index).get_period()
        if isinstance(max_middle_period, SimpleTimePeriod):
            s = max_middle_period.get_start_millis()
            e = max_middle_period.get_end_millis()
        else:
            s = int(max_middle_period.get_start().timestamp() * 1000)
            e = int(max_middle_period.get_end().timestamp() * 1000)
        max_middle = s + (e - s) // 2
        if middle > max_middle:
            self._max_middle_index = index
    else:
        self._max_middle_index = index

Root cause summary
- A single copy/paste typo: using _min_middle_index where _max_middle_index was intended. Fixing that one variable reference resolves the incorrect behavior.
