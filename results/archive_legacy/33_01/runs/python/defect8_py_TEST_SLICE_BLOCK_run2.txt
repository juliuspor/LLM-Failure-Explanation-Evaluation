What happened
- The input string is "fr__POSIX" (language "fr", empty country, variant "POSIX").
- The code at lines 239–240 reads two characters ch3 and ch4 where it expects a 2-letter country and then validates them with:
  if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
      raise ValueError(f"Invalid locale format: {locale_str}")
- For "fr__POSIX" the characters at those positions are underscores (index 2 and 3 are '_' and '_'), so ch3 is '_' which is less than 'A'. The condition is true and the method raises ValueError instead of accepting an empty country and parsing the variant.

Why this is the root cause
- The code assumes a country code is always present and always two uppercase letters. It does not handle the valid case where the country is omitted and the locale string uses a double underscore to separate language and variant ("language__variant").
- The validation at lines 239–240 runs unconditionally and therefore incorrectly rejects the double-underscore form.

Where it occurs
- Lines 239–240 (the country-letter validation) are the precise place the failure is triggered.

Concrete fixes (steps)
Pick one of these clear fixes and implement it:

1) Explicitly detect empty-country form before validating
   - Before reading/validating ch3 and ch4, check if locale_str[2] == '_'. If so:
     - Set country = ''.
     - Set variant = locale_str[3:].
     - Skip the 2-letter country validation.
   - Otherwise perform the existing 2-letter validation.

2) Adjust the validation to allow underscores at those positions
   - Replace the unconditional letter-range check with a branch:
     - If ch3 == '_' and ch4 == '_': interpret as empty country and treat the rest as variant.
     - Else validate ch3 and ch4 are between 'A' and 'Z'.

3) Robust split-based parsing (recommended for clarity)
   - Split the string on '_' into components:

     - parts[0] => language
     - If there are 3 parts and parts[1] == '' then country = '' and variant = parts[2]
     - If there are 3 parts and parts[1] is 2 letters, country = parts[1]

   - This avoids index-based assumptions and makes handling empty fields explicit.
