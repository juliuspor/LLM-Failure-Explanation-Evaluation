What goes wrong
- The code rejects valid locales when the country (or region) part is lowercase.
- At lines 239–240 you check ch3 and ch4 like this:
  if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
    raise ValueError(f"Invalid locale format: {locale_str}")
- That forces the country code (characters at index 3 and 4) to be uppercase ASCII A–Z. If the input is "en_us" (lowercase country) or any valid locale where the country letters are not uppercase ASCII, the function raises an error even though "en_us" is a common input.

Why this matters (concrete variables / locations)
- The failing comparison happens at lines 239–240 (ch3 and ch4 checks).
- The variables involved are ch3 = locale_str[3] and ch4 = locale_str[4].
- The root cause is a strict ASCII uppercase-only test for ch3/ch4; it does not accept lowercase country codes or non-ASCII letters that can appear in some locale identifiers.

How to reproduce
- Call to_locale with "en_us":
  - length = 5 so code reaches the ch3/ch4 checks.
  - ch3 == 'u', ch4 == 's' → both are < 'A' so the condition is true and a ValueError is raised.

Recommended fixes (explicit steps)
1. Decide on the allowed case convention. If you accept either case for the country part, normalize before validating:
   - Replace ch3 = locale_str[3]; ch4 = locale_str[4] with ch3 = locale_str[3].upper(); ch4 = locale_str[4].upper()
   - Then perform the existing A–Z checks on the uppercased chars.
2. Or, if you want to allow both ASCII upper and lower without changing the input, change the test to accept a–z as well:
   - if (ch3 < 'A' or ch3 > 'Z') and (ch3 < 'a' or ch3 > 'z'): raise …
   - same for ch4.
3. If you want to be robust for Unicode letters (not just ASCII), use a Unicode-aware test:
   - import str.isalpha() and check ch3.isalpha() and ch4.isalpha() (and optionally ch3.isupper() if you require uppercase after normalization).
4. After normalizing or validating, when you construct the Locale, use a consistent case (e.g., language lowercase, country uppercase):
   - return Locale(locale_str[0:2].lower(), locale_str[3:5].upper(), ...)

Minimal concrete code change example (normalize country to uppercase):
    ch3 = locale_str[3].upper()
    ch4 = locale_str[4].upper()
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(...)

This makes the function accept "en_US" and "en_us" while preserving the intended validation on lines 239–240.
