What happened (where it failed)
- The failing line is at ClassUtils.to_class, line 767: classes.append(type(array[i]))
- The test called ClassUtils.to_class(['Test', None, 99.0]) and expected [str, type(None), float] but got a different result (the assertion failed).

Root cause (why it failed)
- The code is using array[i] but i is not an index at that point. Instead i is a tuple produced by enumerate(array) (or otherwise an element, not an integer index). When you iterate with enumerate incorrectly, i becomes (index, value) rather than the index alone. Using that tuple as a list index either raises an exception or produces wrong behavior.
- Concretely, a common incorrect loop looks like:
    for i in enumerate(array):
        classes.append(type(array[i]))
  In that loop i is (index, value), so array[i] is invalid / nonsensical.
- Because type() is being called on the wrong thing (or the lookup fails), the produced classes list does not match [str, type(None), float], so the assert fails.

Exactly where to look
- Open ClassUtils.to_class and inspect the loop around line 767 where classes.append(type(array[i])) appears.
- Check how that loop is declared: is it "for i in enumerate(array):", "for i in array:", or "for i, v in enumerate(array):"? The behavior differs and is the source of the bug.

How to fix it (step-by-step)
1. Decide whether you want the index or the value. You want the element value to get its type.
2. Replace the loop with one of these correct forms:
   - If you only need the value:
       for v in array:
           classes.append(type(v))
   - If you need the index and the value:
       for i, v in enumerate(array):
           classes.append(type(v))
   - If you prefer index-based access (less Pythonic but explicit):
       for i in range(len(array)):
           classes.append(type(array[i]))
3. Update the code at/around line 767 accordingly so classes.append receives type(v) (or type(array[i]) when i is an integer index).
4. Run the test again:
   result = ClassUtils.to_class(['Test', None, 99.0])
   assert result == [str, type(None), float]
   It should pass.

Why these fixes work
- type(v) returns the actual class of the element (str, NoneType, float).
- Using v (the element) avoids misusing enumerate’s tuple or treating a value as an index.

Extra note
- If you see a TypeError complaining about list indices must be integers or slices, that’s a clear sign i is not an int. If you see wrong class values or an assertion failure without an exception, printing i and v inside the loop will quickly show whether you’re handling index vs. value correctly.
