What happened
- The test calls ClassUtils.to_class(array) with array = ['Test', None, 99.0].
- The method raises AttributeError while processing the None element instead of returning [str, type(None), float].

Why it failed (root cause)
- Inside ClassUtils.to_class you iterate the input array and call a member method on each element to get its class. Example problematic code pattern:
  - element = array[i]
  - classes[i] = element.getClass()   (or element.get_class(), or element.__class__.something)
- When element is None the code attempts to call a method or attribute on None. None has no such attribute, so Python raises AttributeError at that line.
- The docstring is inconsistent: the top says “If any of these objects is None, a None element will be inserted into the array” but the Raises section lists AttributeError for None. The implementation followed the latter behavior (attempting to call a method on None) instead of the former.

Where it occurs (what line/variable to inspect)
- In method: ClassUtils.to_class
- Variables: array (input), element (array[i]), classes (output array), i (loop index)
- Exact failing operation: the call that reads a class from element (e.g. element.getClass() or element.get_class() or any attribute access on element) when element is None — that line raises AttributeError.

What to change (clear, ordered steps)
1. At the start of to_class, keep the existing check for a None input array:
   - if array is None: return None
2. When building the output list, do not call methods/attributes on elements without checking for None. Use Python’s built-in type() (or element.__class__) safely:
   - for i, element in enumerate(array):
       - if element is None:
           - classes[i] = type(None)   # or classes[i] = None if you intend the output to contain None (pick one consistent behavior)
       - else:
           - classes[i] = type(element)
3. Make the docstring and Raises section consistent with the actual behavior you choose:
   - If you will insert type(None) for None elements, remove “Raises: AttributeError: If any element is None”.
   - If you prefer to raise, explicitly raise a clearer exception (ValueError or TypeError) with a helpful message rather than letting an AttributeError happen.
4. Add a unit test reflecting the chosen behavior. For the test you showed, to pass you must implement step 2 so None becomes type(None).

Minimal example fix (conceptual)
- Replace any element.getClass() or similar with:
    if element is None:
        classes[i] = type(None)
    else:
        classes[i] = type(element)

That removes the AttributeError and makes the return value match the expected [str, type(None), float].
