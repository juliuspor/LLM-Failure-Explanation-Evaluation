What failed
- The Python locale parser raised: ValueError: Invalid locale format: fr__POSIX
- That exception comes from the locale module when you pass an incorrectly formed locale string into locale.setlocale(...) (or to locale.normalize(...)). The parser rejected the string "fr__POSIX".

Why it failed (root cause, concrete)
- Locale identifiers must follow this shape: language[_TERRITORY][.codeset][@modifier]. Examples: "fr", "fr_FR", "fr_FR.UTF-8", or "fr_FR@POSIX".
- Your string "fr__POSIX" contains two underscores in a row: that decodes to language="fr", empty territory="", modifier="POSIX" but with the wrong separator placement. In other words, the territory portion is empty but you still used the underscore for it, producing an invalid format. The locale parser does explicit format checking and rejects "fr__POSIX".
- In typical code this happens where you call locale.setlocale or locale.normalize. For example:
  - locale_name = os.environ.get("LANG")  # or similar
  - locale.setlocale(locale.LC_ALL, locale_name)   # raises ValueError here
  The exception will be raised on the line that calls locale.setlocale (or locale.normalize) with that bad string.

Exactly where to look in your code
- Find the call to locale.setlocale(...) or locale.normalize(...). That is the line that raises ValueError. If you don’t call those directly, search for code that reads environment variables LANG, LC_ALL, LC_MESSAGES, or a project variable named locale_name or language_code and then passes it into the locale API.
- Example locations to search:
  - any line like: locale.setlocale(locale.LC_ALL, locale_name)
  - any place you do: lang = os.environ.get("LANG") or lang = config["locale"]

How to reproduce the problem locally
- In a Python REPL:
  1) import locale
  2) locale.setlocale(locale.LC_ALL, "fr__POSIX")
  You will get the same ValueError.

   - Replace "fr__POSIX" with one of the valid forms above.
