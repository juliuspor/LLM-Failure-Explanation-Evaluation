Why it fails (root cause)
- The validation at line 396 forbids any negative minute values. The test intends to express a negative total offset of -2 hours and -15 minutes (i.e. -02:15), but the validator only accepts minute values in [0, 59]. It does not allow negative minutes, so the API rejects the input rather than computing the combined negative offset.
