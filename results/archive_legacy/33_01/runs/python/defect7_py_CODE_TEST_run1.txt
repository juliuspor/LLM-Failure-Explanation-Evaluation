What went wrong
- The failure happens inside to_class, on the line:
  _ = array[i].upper()
- That line assumes array[i] is a string and calls .upper() on it.
- In the test you pass ['Test', None, 99.0]. When the loop reaches the second element (array[1] == None), calling None.upper() raises AttributeError because NoneType has no attribute upper. The same would occur for the float 99.0: float has no upper() method.
- Because the exception is raised before you append type(array[i]) for those elements, the function never returns the expected list [str, type(None), float].

Where exactly it fails
- Method: to_class
- Problematic statement: _ = array[i].upper()
- Happens for any non-string element (None, float, int, etc.) at the iteration that encounters that element.

Why the line was likely there
- It looks like the code intended to treat strings specially (maybe to normalize case) but calling .upper() has no effect on computing the type and is unsafe for non-strings. The result of .upper() is assigned to _ and not used, so it is unnecessary for the goal of returning types.

Concrete fixes (step-by-step)
1. Remove the unnecessary .upper() call.
2. For each element elem = array[i]:
   - If elem is None, append type(None).
   - Otherwise append type(elem).
3. Optionally, if you really need to treat str values specially, check isinstance(elem, str) before calling str methods.

Corrected implementation (minimal):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for i in range(len(array)):
            elem = array[i]
            if elem is None:
                classes.append(type(None))
            else:
                classes.append(type(elem))
        return classes

Why this fixes it
- No string-only method is invoked without checking type, so None and numeric values won't cause AttributeError.
- The returned list will be [str, type(None), float] for your test input.

Edge cases to consider
- If the input contains custom objects, type(elem) will return their class (probably what you want).
- If you actually need to map strings to a specific class regardless of content, do that with an explicit isinstance(elem, str) check before calling string methods.
