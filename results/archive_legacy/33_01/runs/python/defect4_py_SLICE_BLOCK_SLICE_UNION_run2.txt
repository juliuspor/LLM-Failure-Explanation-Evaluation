- Python requires an indented statement under a function header. With the docstring line removed the interpreter sees:
  - a def line at 449 with no indented block following it, then
  - a return line at 451 that is not in the function body (dedented).
- That produces an error like "IndentationError: expected an indented block" pointing at line 449 (the def) or at the return at 451, because the function header has no indented body.

- The merge/patch that produced SLICE_UNION removed the docstring line but did not keep the indented return line as part of the function body. In other words, the edit removed the only indented line that immediately followed the def, leaving the function with no body.
