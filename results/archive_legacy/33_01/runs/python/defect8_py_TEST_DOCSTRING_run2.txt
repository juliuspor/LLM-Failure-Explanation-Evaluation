What happened
- The test calls LocaleUtils.to_locale('fr__POSIX').
- The string splits into three pieces: ['fr', '', 'POSIX'] (language='fr', country='', variant='POSIX').
- The implementation currently treats an empty second piece as invalid and raises ValueError (or fails a length/format check) instead of accepting an empty country with a variant.

Why it fails (root cause)
- The code enforces "country must be length 2" or uses a pattern that requires the country part to be either absent or exactly two uppercase letters.
- It does not allow the specific but valid format lang__variant (language + empty country + variant). The docstring explicitly shows that a locale with an empty country and a variant is valid (example: Locale("en", "GB", "xxx") and the intended form "fr__POSIX" -> Locale("fr", "", "POSIX")).
- Concretely, when locale_str is split into parts:
  - variable locale_str contains 'fr__POSIX'
  - parts = locale_str.split('_') yields ['fr', '', 'POSIX']
  - language = parts[0], country = parts[1], variant = parts[2]
  - the code then runs a check like if len(country) != 2: raise ValueError (or a regex that requires [A-Z]{2} for the country) which fails because country == ''.
- This validation occurs inside LocaleUtils.to_locale when handling the second piece (the country). That is where the code incorrectly disallows an empty country when there are three parts.

Where in the code to look
- In LocaleUtils.to_locale: the branch that handles 3-part locale strings (after splitting on '_') and the subsequent validation of the country variable.
- Look for checks like:
  - len(country) == 2
  - country.isupper()
  - a regex that requires the country group to be exactly two uppercase letters
  - any code that treats an empty string as "missing" and then rejects it instead of treating it as a valid empty country

How to fix it (concrete steps)
1. When splitting: parts = locale_str.split('_')
2. If len(parts) == 3:
   - Assign language = parts[0]; country = parts[1]; variant = parts[2]
   - Change the country validation to allow an empty country:
     - If country != '':
         - require len(country) == 2 and country.isupper()
     - Else (country == ''):
         - accept it and proceed to accept variant as parts[2]
3. If you use a regex, update it so the country group can be an empty string when the variant group is present (i.e. allow lang(_([A-Z]{2}))?(_(.*))? but also support lang__variant). Concretely: ^([a-z]{2,3})(?:_([A-Z]{2})|_)?(?:_(.*))?$ is one way, or handle splitting as above which is simpler.
4. Keep the language validation unchanged: language must be lowercase and the correct length.
5. Return Locale(language, country, variant) where country may be ''.
