What happened (root cause)
- LocaleUtils.to_locale receives the string "fr__POSIX".
- The method splits the input on '_' into parts and then validates/assigns language, country and variant.
- Because the code uses the ordinary String.split("_") (or similar split behavior that drops empty tokens), the empty country token between the two underscores is lost or treated as invalid.
- The method then either:
  - sees an unexpected number of parts and raises ValueError("Invalid locale format: " + localeString), or
  - validates the country token and rejects an empty country (e.g. requires length 2), and raises the same ValueError.
- Result: "fr__POSIX" is treated as an invalid format instead of language="fr", country="" (empty), variant="POSIX", so the test fails.

Where it occurs (what to look for)
- In LocaleUtils.to_locale:
  - The string splitting line, e.g. parts = localeString.split("_")
  - The subsequent validation/assignment lines where it does something like:
    - if len(parts) not in (1,2,3): raise ValueError(...)
    - language = parts[0]; country = parts[1] (or validate parts[1].length()==2) ; variant = parts[2]
  - The ValueError is constructed from the input (the message shows the input), so the exception is raised inside that method.

Why the code is wrong (concrete reason)
- The input "fr__POSIX" is a valid locale representation meaning language="fr", empty country, variant="POSIX".
- Typical String.split("_") behavior removes empty tokens (especially trailing or consecutive separators), so the code never sees the empty country slot and either gets the wrong number of tokens or a token in the wrong position.
- The validation code treats an empty country as invalid instead of accepting an empty string for country when a variant is present.

Exact changes to fix it (step-by-step)
1. Preserve empty tokens when splitting:
   - In Java, use String.split("_", -1) so consecutive separators produce empty strings:
     - parts = localeString.split("_", -1);
2. Allow an empty country token when a variant is present:
   - After splitting, handle cases explicitly:
     - if parts.length == 1:
         language = parts[0]; country = ""; variant = "";
     - else if parts.length == 2:
         language = parts[0]; country = parts[1]; variant = "";
     - else if parts.length == 3:
         language = parts[0]; country = parts[1]; variant = parts[2];
     - else: throw ValueError/IllegalArgumentException(...)
3. Validate language and variant correctly:
   - Ensure language is non-empty and matches whatever rule you have (e.g. 2-3 letters).
   - Do not require country to be non-empty when parts.length == 3; empty country "" is valid (the test expects Locale("fr", "", "POSIX")).
4. Optional: normalize/truncate country if you require exactly 2 chars, but only when country is non-empty.

Minimal code example for Java-like implementation
- Replace:

  with:
