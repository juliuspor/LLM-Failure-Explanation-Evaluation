What happened (where it fails)
- On these two lines you clamp the input and store the result in v:
  ```
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
  ```
- But on the next line you compute g using the original value instead of the clamped v:
  ```
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  ```
  That means if value was outside the bounds, g can be outside 0..255 and trigger the ValueError check:
  ```
  if g < 0 or g > 255:
      raise ValueError(...)
  ```
- So the failure happens because you clamped into v but then ignored it and used value when computing g. The error is raised at the g check because the unclamped value produced a g outside the valid range.
