What happened
- The code can hang (infinite loop) when x is 0.0 (or -0.0).
- In the branch where x == int(x) you set value = int(x) and mantissa = value, then enter
  while mantissa % 10 == 0:
  This condition is true for mantissa == 0, and the body reduces mantissa with mantissa //= 10 — which leaves mantissa at 0. The test check_val == value also stays true because check_val stays 0, so the loop never breaks and exp keeps increasing forever.

Where it occurs (exact variables/places)
- In add_number, inside the integer-handling branch:
  - value = int(x)
  - mantissa = value
  - exp = 0
  - while mantissa % 10 == 0:
    -> infinite loop if value == 0 because mantissa stays 0 and check_val == value is always true.

Why it happens (root cause)
- The loop is intended to strip trailing decimal zeros from an integer (e.g., turn 1000 into mantissa 1 with exp 3). That logic assumes mantissa will eventually become non-multiple-of-10. For value == 0 this never happens: 0 % 10 == 0 and 0 // 10 == 0, so the trimming never progresses and the exit condition never occurs.

How to fix it (clear, actionable steps)
1. Short-circuit zero before the loop:
   - Immediately handle value == 0 and avoid the stripping loop. Example:
     if value == 0:
         self.add("0")
         return

2. Or change the loop condition to avoid processing zero:
   - Replace while mantissa % 10 == 0: with while mantissa != 0 and mantissa % 10 == 0:
     This prevents the loop from running for zero and preserves the intended behavior for other integers.

Either fix can be applied at the start of the integer branch (right after value and mantissa are set). Example minimal change:

    if value == 0:
        self.add("0")
        return

Or change the loop:

    while mantissa != 0 and mantissa % 10 == 0:
        ...

Either approach prevents the infinite loop for x == 0.
