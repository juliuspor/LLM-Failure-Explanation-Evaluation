What happened (exact place and variables)
- The exception comes from line 38: high = code_units[index]. The IndexError says index out of range: 2, which means index == 2 but len(code_units) == 2 (valid indices are 0 and 1).
- The code only checks index + 1 < len(code_units) later on line 39, but it reads code_units[index] before any bounds check. If index is equal to len(code_units) (or otherwise >= len(code_units)), that first read fails and raises the IndexError.

Why this occurs (root cause)
- The code assumes index is valid and tries to read code_units[index] unconditionally at line 38.
- The later condition on line 39 protects the access to the next element (index + 1) but does not protect the access to the current element (high).
- Therefore when the caller passes an out-of-range index (e.g., index == 2 for a length-2 sequence), you get the IndexError at line 38.

Where it occurs (lines and variables)
- Line 38: high = code_units[index] — direct cause of the exception.
- Line 39: if 0xD800 <= high <= 0xDBFF and index + 1 < len(code_units): — this checks the second code unit, but it’s too late because high has already been read.
