Fix: accept minutes in range -59..59 and enforce the rule that if hours positive minutes cannot be negative. Calculate total minutes = hours_offset*60 + minutes_offset, then validate resulting minutes is within -1439..1439 (i.e. +/- 23:59). Then compute millis via safe_multiply. Return for_offset_millis.
