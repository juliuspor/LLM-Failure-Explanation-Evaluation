What happened (where it fails)
- At runtime your call ArrayUtils.add(None, None, expected_type=str) hits the check at line 346–347:
  - 346: if inferred_type == object and expected_type != object:
  - 347:     raise TypeError(...)
- The variable inferred_type is object (set at line 340), and expected_type is str, so the code raises TypeError. That is why the test fails.

Why this happens (root cause)
- The code treats None values as "unknown" and leaves inferred_type as object (line 340 sets inferred_type = object when it encounters no concrete element type).
- The code then assumes "inferred_type == object" means a real type conflict when expected_type is provided, so it raises a TypeError (lines 346–347).
- In your case all inputs are None, so there is no actual conflicting concrete element type; None should be compatible with any expected_type, but the code confuses "no concrete type seen" with "type conflict".

          raise TypeError(...)
  - Result: if all elements were None (saw_non_none == False), you will not raise, and you can accept expected_type.
