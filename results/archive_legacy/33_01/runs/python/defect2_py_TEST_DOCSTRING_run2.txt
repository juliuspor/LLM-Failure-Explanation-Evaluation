What happened
- The test calls gps.get_paint(-0.5) and expects color == (0, 0, 0).
- Instead of returning a black tuple, get_paint raised ValueError because the method treats the input value as producing RGB components outside the allowed 0–255 range.

Why it happened (root cause)
- In GrayPaintScale.get_paint there is input validation that raises ValueError when the computed RGB would fall outside 0–255. That code runs for value = -0.5, detects that the computed grayscale channel would be negative, and raises ValueError rather than clamping or converting negative values to 0.
- The docstring says the function "Returns a paint (RGB tuple) for the specified value" and documents a Raises: ValueError case, but the test expects the method to convert out-of-range inputs to the nearest valid color (i.e. -0.5 -> 0 -> black). The method's current behavior is raising an exception instead of returning (0,0,0).
