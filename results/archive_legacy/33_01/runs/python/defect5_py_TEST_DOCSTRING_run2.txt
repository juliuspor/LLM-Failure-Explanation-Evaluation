What happened
- The test calls ArrayUtils.add(array=None, element=None, expected_type=str) and expects the function to return [None].
- Instead the function raises a TypeError (test fails).

Why it fails (root cause)
- Inside ArrayUtils.add the code enforces the expected_type by doing an isinstance check on element without first verifying element isn't None.
- Concretely, the function does something like:
  if expected_type is not None and not isinstance(element, expected_type):
      raise TypeError("element type mismatch")
- When element is None, isinstance(None, str) returns False, so the TypeError is raised even though the docstring says "unless the element itself is None" we should allow a None element and return a one-element list of None.
- In short: the check is applied to None when it should be skipped for None values.

Where in the code to change
- In ArrayUtils.add, locate the type-checking line that uses isinstance(element, expected_type).
- Replace that check with a check that only runs when element is not None.

Exact replacement to make (pseudo-Python showing variable names you have)
- Replace this incorrect check:
    if expected_type is not None and not isinstance(element, expected_type):
        raise TypeError("element type mismatch")
- With this corrected check:
    if expected_type is not None and element is not None and not isinstance(element, expected_type):
        raise TypeError("element type mismatch")

Why this fix works
- The new condition allows element == None through without raising, which matches the docstring ("unless the element itself is None ... return type is List[object]"). It still enforces type checking for non-None elements when expected_type is provided.
- With this change, ArrayUtils.add(None, None, expected_type=str) will not raise and will return [None] as the test expects.
