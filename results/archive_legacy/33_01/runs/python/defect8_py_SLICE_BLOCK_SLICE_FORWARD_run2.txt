and that comparison raises a TypeError when this function is called with locale_str being a bytes object. In Python 3 indexing a bytes object returns an int, not a one-character bytes or str. So ch3 and ch4 are ints but the code compares them to string literals 'A' and 'Z'. The runtime error is something like:
  TypeError: '<' not supported between instances of 'int' and 'str'

Why it happened (root cause): locale_str is bytes (or some code path produced ints for ch3/ch4) but the code assumes characters (str). The comparison mixes types: int < str, which Python disallows.
