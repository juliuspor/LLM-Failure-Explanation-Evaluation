What happened
- The factory validated the two arguments hours_offset and minutes_offset and raised ValueError("Minutes out of range: -15").
- The root cause is that you passed a positive hours_offset together with a negative minutes_offset (hours_offset > 0 and minutes_offset < 0). The docstring explicitly disallows that combination: a positive hour must have a non‑negative minute. Example the table in the docstring shows (2, -15) → ValueError.

Why the code enforces this
- The factory treats hours and minutes as two parts of a signed offset. It requires the minute component to carry the same sign as the hour component (or the hour to be zero and the minute to carry the sign).
- The method checks minutes_offset is between -59 and +59 and then enforces the sign rule; when it finds minutes_offset == -15 while hours_offset == +2 it raises the shown ValueError.

Where it occurs
- The exception is thrown during the argument validation inside the timezone factory (the code that receives hours_offset and minutes_offset and constructs the DateTimeZone). The check that raises ValueError is triggered before any timezone object is created.

How to fix it (concrete steps)
1) If you intended a positive offset, make minutes non‑negative:
   - If you meant +02:15, call the factory with hours_offset = 2, minutes_offset = 15 (not -15).

2) If you intended a negative offset, make hours negative (or zero) and minutes negative:
   - If you meant -02:15, call with hours_offset = -2, minutes_offset = -15.

3) If your caller sometimes builds offsets by adding/subtracting parts and may produce mismatched signs, normalize the total offset before calling the factory. Example normalization you can run before the factory call:

   total_minutes = hours_offset * 60 + minutes_offset
   if abs(total_minutes) > (23 * 60 + 59):
       raise ValueError("Total offset exceeds +/- 23:59")
   sign = 1 if total_minutes >= 0 else -1
   abs_minutes = abs(total_minutes)
   normalized_hours = (abs_minutes // 60) * sign
   normalized_minutes = (abs_minutes % 60) * sign
   # pass normalized_hours, normalized_minutes into the factory

   This produces hour and minute components that have the required matching sign (or zero hour with signed minutes).

Examples
- You called (2, -15). total_minutes = 2*60 + (-15) = 105 -> normalize to (1, 45) and call with hours_offset=1, minutes_offset=45.
- To represent -00:15 call (0, -15) (allowed by the docstring).

Summary
The ValueError arises because the factory forbids positive hours with negative minutes; either pass minutes with the same sign as hours or compute a normalized pair (hours, minutes) from the total offset before calling the factory.
