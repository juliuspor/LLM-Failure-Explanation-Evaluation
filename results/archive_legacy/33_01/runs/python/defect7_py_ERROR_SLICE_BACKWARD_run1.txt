What happened (root cause)
- The exception AttributeError: 'NoneType' object has no attribute 'upper' means you tried to call .upper() on a value that is None, not on a string.
- In your slice trace the failing line is 766:
  766 |             _ = array[i].upper()
  This shows array[i] is None for at least one i in the loop, so None.upper() raises the error.

Why it happened (specifics)
- Variable: array
- Index: some i in range(len(array)) where array[i] == None
- Method: .upper() expects a str. None has no .upper() method, so Python raises AttributeError.
- The code does not check the element type or value before calling .upper(), so the problem appears as soon as a None element is reached.

1. Re-run the loop with logging to find the index and value:

Concrete fixes (pick one depending on intention)

- If None is expected and should be skipped, use A.
- If None should be treated as empty, use B.
