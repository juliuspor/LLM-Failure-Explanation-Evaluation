What failed
- At lines 35–36 you check "if index < 0 or index >= len(code_units):" but there is no variable named index defined inside to_utf16_code_units. The function only defines s and code_units (lines 10 and 12) and never receives or creates index. That causes a NameError (or, if the code was rearranged, incorrect behavior because the check belongs somewhere else).

Why that is the root cause
- The bounds check (lines 35–36) and the following surrogate-pair logic (lines 38–44) expect an index into an already-built list of UTF-16 code units. But those lines are inside to_utf16_code_units, which is supposed to build and return code_units. They use the local variable code_units (which does exist) but refer to index (which does not). The check and subsequent use of index must run in a function that actually receives an index or after you call to_utf16_code_units and pass its result plus an index to a separate function. As written, the code is mixing two responsibilities and refers to an undeclared variable.

Where it occurs
- In the function to_utf16_code_units (def at line 10). code_units is created at line 12. The problematic check is at lines 35–36. Surrogate handling using index is at 38–44.

Clear fixes (pick one)

1) Keep to_utf16_code_units as a builder and move the index logic into a separate function
- Remove lines 35–44 from to_utf16_code_units.
- Add a new function:
  def code_point_from_units(code_units: List[int], index: int) -> int:
      if index < 0 or index >= len(code_units):
          raise IndexError(f"String index out of range: {index}")
      high = code_units[index]
      if 0xD800 <= high <= 0xDBFF and index + 1 < len(code_units):
          low = code_units[index + 1]
          if 0xDC00 <= low <= 0xDFFF:
              return 0x10000 + ((high - 0xD800) << 10) + (low - 0xDC00)
      return high
- Call it like:
  units = to_utf16_code_units(s)
  cp = code_point_from_units(units, index)

2) Make to_utf16_code_units accept an index and return a code point (if that was intended)
- Change signature to def to_utf16_code_units(s: str, index: int) -> int (or -> Tuple[List[int], int> if you want both).
- Ensure index is validated inside the function (lines 35–36 make sense then).
- But this conflates two behaviors (building units and resolving a code point); prefer option 1.

3) If the intent was to check indexes while building the list, change the name and parameter:
- If index was supposed to be a loop variable inside the builder, declare and use it properly in that scope. But from the surrounding lines, this is unlikely.

Summary
The code fails because lines 35–44 use a variable named index that is never provided to to_utf16_code_units. Move that logic into a function that takes code_units and index (or change the function signature to accept index) so index is defined where it is used.
