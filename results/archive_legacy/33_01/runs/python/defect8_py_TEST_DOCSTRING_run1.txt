What happened
- The test calls LocaleUtils.to_locale('fr__POSIX'). That string is intended to mean:
  - language = "fr"
  - country  = ""   (empty)
  - variant  = "POSIX"
- The implementation of to_locale parses the string correctly into those three parts, but then rejects it during its validation step because of how the country token is checked.

Why it fails (root cause)
- Inside LocaleUtils.to_locale (the parsing/validation block that runs after splitting locale_str), there is a check that enforces the country token to be uppercase (e.g. country.isupper()) or to have length 2. Because the parsed country token is the empty string (""), calling country.isupper() returns False and/or the length check fails. The code treats that as a validation error and raises ValueError instead of accepting an empty country when a variant is present.
- In short: the validator does not allow an empty country token even though the format "language__variant" (double underscore) is valid and documented.

Where it occurs (variables and method)
- Method: LocaleUtils.to_locale(locale_str)
- Relevant variables: locale_str (input), tokens (result of split), language, country, variant
- Failing check: the country validation step immediately before constructing and returning new Locale(language, country, variant). That check currently assumes country must be a two-letter uppercase string and does not special-case country == "".

Concrete fix (step-by-step)
1. Parse the string with a split that preserves an empty middle token, e.g.:
   - tokens = locale_str.split('_', 2)
   - Then set language = tokens[0], country = tokens[1] if len(tokens) > 1 else "", variant = tokens[2] if len(tokens) > 2 else ""
2. Change the country validation to allow an empty country:
   - Only apply the uppercase / length check when country is non-empty.
   - Example logic:
     - if country:
         - if len(country) != 2 or not country.isupper():
             - raise ValueError("invalid country")
3. Keep the existing language validation (language must be lowercase and correct length) and variant handling intact.
4. After those checks, construct and return Locale(language, country, variant).

Example corrected condition (pseudocode)
- if country != "":
    - if len(country) != 2 or not country.isupper():
        - raise ValueError("invalid country")
- (then proceed to construct Locale(language, country, variant))

Why this fix resolves the failure
- It allows the documented and expected "fr__POSIX" form where the country is intentionally empty but a variant exists. The empty string will pass the new conditional (no uppercase/length check applied), so the method will return Locale('fr', '', 'POSIX') as the test expects.
