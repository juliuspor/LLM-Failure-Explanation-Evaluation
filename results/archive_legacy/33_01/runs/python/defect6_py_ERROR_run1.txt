What happened
- Your test expected a floating value that has a negative sign bit (the IEEE “-0.0”), but the function under test returned a value that is +0 (printed or represented as 0).
- Many languages and test frameworks print “-0.0” vs “0” when they preserve the sign bit in the textual representation, even though -0.0 == 0.0 compares equal in most languages.
- The assertion failed because the test checks for the signed-zero form (negative zero) but the code returned positive zero (or an integer 0).

Why that happens (root cause)
- The sign bit of zero is lost or never set by the code path that produced the result. This can happen in several common ways:
  - A branch or arithmetic wiped the sign (for example: -0.0 + 0.0 → +0.0; or multiplying by 0.0).
  - The function converted the float to an integer or to a string that normalized away the sign.
  - The test compared textual representations or used equality that compares representation rather than raw bits.
  - The function explicitly returned an integer 0 rather than a float -0.0.

2. From that test, trace the call to the function under test (the call site and the function name). Note the variable name where the returned value is stored (for example value = my_func(...)).
3. Insert a quick inspect at that point to show exactly what the value is at runtime (see detection code below).

How to reproduce and inspect the sign bit (examples)
- In Python:
  - Show the runtime value and its sign bit:
    import math, struct
    v = my_func(...)
    print(repr(v))                # shows -0.0 vs 0.0
    print(math.copysign(1.0, v))  # will be -1.0 for -0.0, +1.0 for +0.0
    print(struct.pack('!d', float(v)).hex())  # raw IEEE-754 bits

- In Java:
    double v = myFunc(...);
    System.out.println(Double.toString(v));            // prints -0.0 vs 0.0
    System.out.println(Double.doubleToRawLongBits(v)); // shows sign bit in bits

- In C/C++:
    double v = my_func(...);
    printf("%.17g\n", v);
    uint64_t bits;
    memcpy(&bits, &v, sizeof(bits));
    printf("%016" PRIx64 "\n", bits);

- Change the test assertion to not require sign of zero. For example:

If you need to assert the sign bit in tests (explicit check):
- Use a bit-level or sign-check helper instead of plain equality. Examples:

  Python (precise check):

    assert value == 0.0
    assert math.copysign(1.0, value) == -1.0  # ensures negative zero
