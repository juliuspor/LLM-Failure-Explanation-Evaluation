What happened
- The test calls ClassUtils.to_class(['Test', None, 99.0]) and expects [str, type(None), float].
- The failure happens inside the loop shown at lines 765–767:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))

Why it fails (root cause)
- On line 766 the code always calls array[i].upper() for every element.
- array[1] is None. None has no method upper(), so Python raises:
  AttributeError: 'NoneType' object has no attribute 'upper'
- Because that exception occurs before line 767, the code never appends type(None) to classes and the function crashes.
- The same problem would happen for array[2] (99.0) because a float also has no upper().

Where exactly it happens
- The AttributeError is raised at line 766 when array[i] is not a string (for the test input, at i == 1).

Concrete fixes (step-by-step)
You have three clear options. Pick the one that matches intended behavior.

1) If you only wanted to call upper() for strings:
- Replace line 766 with a type check so upper() is only called on str:
  - 766:     if isinstance(array[i], str):
  - 767:         _ = array[i].upper()
  - 768:     classes.append(type(array[i]))
- This keeps the rest of your logic and avoids calling upper() on None/float.

2) If upper() was unnecessary (you only need the element type):
- Remove the call to upper() and append the type directly:
  - 766:     classes.append(type(array[i]))
- Simplest fix; for the test input this produces [str, type(None), float].

3) If you intended to coerce everything to string first (not recommended for this test):
- Convert safely and then inspect:
  - 766:     text = array[i] if isinstance(array[i], str) else str(array[i])
  - 767:     _ = text.upper()
  - 768:     classes.append(type(array[i]))
- Note: this will still append the original element type, but it forces an uppercase string for non-strings (which may be unintended).

Other small improvements

Summary
- The code fails because line 766 unconditionally calls .upper() on every array element, but some elements (None, float) do not have upper(), causing an AttributeError before the type is appended. Fix by checking isinstance(..., str) before calling upper(), or remove the upper() call if it is not needed.
