- At line 54 you clamp the incoming parameter into a local variable v:

  - v = min(v, self._upper_bound)
- But at line 71 you compute g using the original value variable, not the clamped v:

- Because you used value instead of v the clamp has no effect on the computed g. If value is outside the [self._lower_bound, self._upper_bound] range g can fall outside 0..255 and trigger the ValueError raised at lines 74–75.

- Root cause 2: no guard against equal bounds, so the code can divide by zero at line 71.

    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)

        # Handle zero-width range to avoid division by zero
        span = self._upper_bound - self._lower_bound

        g = max(0, min(255, g))
