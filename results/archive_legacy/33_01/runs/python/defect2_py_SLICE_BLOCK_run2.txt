What fails
- At line 74 the code only checks the variable g: "if g < 0 or g > 255:". That means r and b can be out of range and the check will not catch them.
- When the condition is true the code raises a ValueError at line 75 with the message f"Color parameter outside of expected range: Red Green Blue". That message is misleading for two reasons:
  1. The f-string contains no { } placeholders, so it does not show which variable or what value caused the error.
  2. The text claims “Red Green Blue” as if all three were validated, but only g was checked.

Why that causes the failure you see
- Root cause: the code only validates g, not r and b. If r or b are out of range, no exception is raised and invalid values slip through. If g is out of range, the exception raised does not tell you which variable or value triggered it, so debugging is hard.
- Where: the logic error and the unhelpful error message are at lines 74–75.

Concrete fixes (step‑by‑step)
1. Validate all three channels instead of only g. For example:
   - Replace line 74 with a check that includes r, g, b:
     if (r < 0 or r > 255) or (g < 0 or g > 255) or (b < 0 or b > 255):
2. Produce a clear error message that names the offending channel and its value. Use f-string placeholders:
   - Example replacement for line 75:
     raise ValueError(f"Color parameter outside of expected range: r={r}, g={g}, b={b} (expected 0..255)")
   - Or, if you prefer to report the single offending channel, check each separately and raise with its name and value:
     if r < 0 or r > 255:
         raise ValueError(f"Red out of range: r={r} (expected 0..255)")
     if g < 0 or g > 255:
         raise ValueError(f"Green out of range: g={g} (expected 0..255)")
     if b < 0 or b > 255:
         raise ValueError(f"Blue out of range: b={b} (expected 0..255)")

Why these fixes solve it
- Checking r, g, and b prevents invalid red/blue values from being accepted.
- Including {r}, {g}, {b} in the f-string gives you exact values in the exception so you immediately know which parameter failed and why.
