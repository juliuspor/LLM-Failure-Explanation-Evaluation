What happened
- The failure is an infinite loop in add_number, inside this block:
  while mantissa % 10 == 0:
      check_val = (mantissa // 10) * (10 ** (exp + 1))
      if check_val == value:
           mantissa //= 10
           exp += 1
      else:
           break
- It occurs when value (the integer version of x) is 0. With value = 0:
  - mantissa starts as 0, so mantissa % 10 == 0 is true.
  - mantissa //= 10 leaves mantissa as 0.
  - check_val computes to 0 every iteration, so check_val == value stays true.
  - mantissa never changes to a non-zero value, exp keeps increasing, and the loop never breaks.

Where it occurs
- Method: add_number
- Variables involved: x, value, mantissa, exp, check_val
- The problematic line is the while condition: while mantissa % 10 == 0:, and the loop body that depends on mantissa changing.

Why it happens (root cause)
- The code assumes repeatedly dividing mantissa by 10 will eventually remove trailing zeros. That’s false when mantissa is 0 because 0 divided by 10 is still 0, so the loop never terminates. The check_val == value branch therefore always executes and exp grows without bound.

How to fix it (concrete steps)
Option A — simplest, explicit zero guard:
1. Right after value = int(x) add:
     if value == 0:
         self.add("0")
         return
   This handles the zero case separately and exits.

Option B — make the loop skip when mantissa is zero:
1. Change the while condition to require mantissa != 0:
     while mantissa != 0 and mantissa % 10 == 0:
         ...
   This prevents entering the loop for mantissa == 0.

Option C — extra safety check inside loop:
1. Keep the loop but add a break if mantissa == 0 before recomputing check_val:
     while mantissa % 10 == 0:
         if mantissa == 0:
             break
         check_val = ...
   (Less clean than A or B.)

Recommended fix: Use Option A (explicit zero handling) or B (require mantissa != 0). Both are small, clear, and testable.

Tests to run after fix
- x = 0.0 (should add "0")
- x = 1000.0 (should produce "1E3" or the intended formatting)
- x = -1000.0 (ensure negative formatting and the earlier space logic still behaves)
- x = 10.0, x = 20.0, x = 1.0 (various powers-of-10)
- Large integers and non-integer floats (ensure previous branches still work)
