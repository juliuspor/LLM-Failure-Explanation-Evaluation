What happened (root cause)
- At lines 765–766 you loop over array and unconditionally call array[i].upper():
  - 765 | for i in range(len(array)):
  - 766 |     _ = array[i].upper()
- In your failing test you passed array = ['Test', None, 99.0].
- When i reaches the element None, array[i] is NoneType, and None has no .upper() method.
- That causes an AttributeError (NoneType object has no attribute 'upper') at line 766 — the code never reaches the assert.

Why this is the problem (clearly tied to variables/lines)
- The code assumes every element in array is a string and calls .upper() on array[i] without checking its type. The variable name _ at line 766 receives the result of that call, but the error occurs before assignment because .upper() does not exist on None (and would also not exist on a float like 99.0).
- So the failure happens in the loop that processes array (lines 765–766), not in the assert.

Exact fixes (concrete steps and exact code you can apply)
Option A — guard by type (recommended for your test)
Replace the body of the loop (lines 765–766) with explicit type checks so non-strings are handled:

for i in range(len(array)):
    val = array[i]
    if val is None:
        array[i] = type(None)        # maps None -> type(None)
    elif isinstance(val, str):
        array[i] = str               # maps any string value -> str
    else:
        array[i] = type(val)         # maps numbers, objects -> their class

This will convert ['Test', None, 99.0] to [str, type(None), float], satisfying the assert.

Option B — use hasattr / safe call
If you intended to operate on string-like objects with .upper(), check first:

for i in range(len(array)):
    val = array[i]
    if hasattr(val, 'upper'):
        _ = val.upper()
        # handle string case...

        # handle non-string case

But this still requires you decide what to set array[i] to when val is None or a float.

Option C — try/except around .upper()
Not recommended for regular control flow, but if you prefer:

for i in range(len(array)):
    try:
        _ = array[i].upper()
        array[i] = str
    except AttributeError:
        array[i] = type(array[i])

Notes and why I recommend Option A
- Option A is explicit and maps each input to the class you expect: strings -> str, None -> type(None), numbers -> their numeric class. It fixes the AttributeError by not calling .upper() on non-string values.
- Options B/C still require explicit decisions about how to convert non-strings to classes.
