What happened (what failed)
- The test calls GrayPaintScale().get_paint(-0.5) and expects (0, 0, 0).
- Instead get_paint raised a ValueError at line 75:
  raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
  (see the slice showing lines 54, 71, 74–75).

Why it failed (root cause)
- At line 71 the code computes g with:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- For value = -0.5 that expression produces a negative g (because value is below self._lower_bound), so the next check at line 74:
  if g < 0 or g > 255:
triggers and the function raises a ValueError (line 75). The function never clamps out-of-range inputs to black or white; it rejects them instead. The test expected out-of-range input to map to black (0) but the method rejects it.

Where it occurred
- Calculation and validation at lines 71 and 74–75 inside get_paint (method start shown at line 54).

How to fix it (concrete steps)
Choose one of these fixes and apply it to get_paint:

Option A — Clamp result to valid range (recommended)
1. After computing g, replace the raise with clamping:
   g = max(0, min(255, g))
2. Return (g, g, g) as the gray color.
This makes get_paint(-0.5) return (0, 0, 0) and preserves behavior for valid inputs.

Option B — Clamp ratio before scaling (preserves rounding)
1. Compute ratio = (value - self._lower_bound) / (self._upper_bound - self._lower_bound)
2. Clamp ratio = max(0.0, min(1.0, ratio))
3. Compute g = int(ratio * 255.0)
4. Return (g, g, g)

Summary
