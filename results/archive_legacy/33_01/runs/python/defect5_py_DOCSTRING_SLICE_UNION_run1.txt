- The code sets inferred_type = object when element is present (element is not None). That is backwards: when you have a concrete element you should infer its concrete type (e.g. type(element)), not assign object.
- As a result, when an expected_type was provided (expected_type != object), the code thinks the element's inferred type is the generic object, treats that as a type mismatch, and raises TypeError incorrectly.
- Concretely: the condition at line 337 and the assignment at line 340 are inconsistent. The branch that runs for a non-None element assigns the generic object type, which triggers the later check at line 346–347 and causes the failure.
