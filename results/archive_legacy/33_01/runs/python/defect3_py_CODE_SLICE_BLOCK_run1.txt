What failed

- Why it crashed (root cause): translate_codepoint(...) and the code that advances pos disagree about what "consumed" counts. The loop treats consumed as a number of codepoints to skip and calls code_point_at(code_units, pos) that expects pos to point at a valid code unit. But consumed is actually larger than the number of remaining codepoints or is measured in code units, so the loop runs too many iterations and calls code_point_at with pos beyond the end of code_units. That makes code_point_at raise IndexError for an out-of-range index.

  The code calls code_point_at(code_units, pos) consumed times.
- If translate_codepoint returned a value measured in UTF-16 code units (or otherwise differs from "how many codepoints remain here"), the for-loop will iterate the wrong number of times and eventually call code_point_at with pos == len(code_units) (or larger), causing the IndexError shown in your slice.

Which to choose
