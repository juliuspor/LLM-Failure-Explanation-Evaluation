What happened
- You clamp the input value to v on lines 2–3:
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
- But when you compute g you still use the original value instead of the clamped v:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

Because get_paint(-0.5) passes a value below self._lower_bound, using value in that expression produces a negative numerator, so g becomes negative and the subsequent range check raises ValueError. The failure occurs at the line that computes g (the third code line shown) and then at the range-check/raise that follows.

Why that is the root cause
- The clamp assigns the safe value to v, but the computation ignores it. The code is therefore inconsistent: clamping is useless because the clamped value is never used in the grayscale calculation.

Precise fix (step-by-step)
1. Use the clamped variable v when computing g:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
2. Guard against zero division if lower and upper bounds are equal (optional but recommended):
   denom = (self._upper_bound - self._lower_bound)
   if denom == 0:
       g = 0  # or 255, or raise — decide based on desired behavior
   else:
       g = int((v - self._lower_bound) / denom * 255.0)
3. Ensure g is an integer in [0,255] (you can clamp as a final safety):
   g = max(0, min(255, g))
4. Improve the ValueError message to include actual values (optional):
   raise ValueError(f"Color parameter outside expected range: g={g}, value={value}, lower={self._lower_bound}, upper={self._upper_bound}")

Minimal corrected version
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)
        
        denom = self._upper_bound - self._lower_bound
        if denom == 0:
            g = 0
        else:
            g = int((v - self._lower_bound) / denom * 255.0)
        
        g = max(0, min(255, g))
        return (g, g, g)

That change fixes the failing test: get_paint(-0.5) now uses the clamped v and returns (0, 0, 0).
