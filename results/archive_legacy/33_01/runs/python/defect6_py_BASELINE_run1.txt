What happened
- The code treats -0.0 as an integer and ends up printing "0" instead of "-0.0".
- Why: at the start of add_number you test if x is an integer via
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
  For x == -0.0 that condition is True (because -0.0 == 0.0 and int(-0.0) == 0).
  You then set value = int(x) which becomes 0, and later do self.add(str(value)) which produces "0". That loses the negative sign.
- Also, the earlier code intended to detect an existing trailing '-' to avoid producing "--" uses x < 0:
  prev = self.get_last_char()
  if x < 0 and prev == '-':
  But x < 0 is False for -0.0, so you don't take the special-space branch either. So nothing preserves the negative zero sign.

Where it occurs (lines from your slice)
- In add_number:
  - The sign check using x < 0 happens near the top (lines shown in your snippet).
  - The integer branch test (if x == int(x) ...) is what incorrectly handles -0.0 and leads to self.add(str(value)).
- In add (lines 126–138) nothing is wrong; it's just the method used to append the string that eventually gets "0".

Concrete, minimal fixes (explicit steps and exact changes)
1) Detect negative zero early:
   Add
     is_neg_zero = (x == 0.0 and math.copysign(1.0, x) < 0)
   near the top of add_number (right after prev = self.get_last_char()).

2) Use that detection in both places:
   - Replace the existing space-insertion test
       if x < 0 and prev == '-':
     with
       if (x < 0 or is_neg_zero) and prev == '-':
     so you handle a preceding '-' when x is negative zero.

   - Prevent treating negative zero as an integer branch:
     Change the integer branch condition
       if x == int(x) and not math.isinf(x) and not math.isnan(x):
     to
       if not is_neg_zero and x == int(x) and not math.isinf(x) and not math.isnan(x):
     This ensures -0.0 falls through to the else branch that uses str(x), which yields "-0.0".

Alternative (single-line) fixes
- Instead of the second change you can early-return for negative zero:
    if is_neg_zero:
        self.add(str(x))
        return
  Place that after computing is_neg_zero. That immediately prints "-0.0" as desired and avoids the integer branch entirely.

Why these fixes work
- math.copysign preserves sign-bit information for zero. is_neg_zero becomes True only for -0.0.
- Using is_neg_zero fixes both the mistaken integer detection and the x < 0 check that fails for negative zero, so the output will be "-0.0" and your test will pass:
    cc.add_number(-0.0) -> cc.get_code() == '-0.0'
