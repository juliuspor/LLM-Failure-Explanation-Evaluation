What happened (where)
- The failure occurs in add_number, at the line:
  value = int(x)
  This is inside the branch that runs when if x == int(x) and not math.isinf(x) and not math.isnan(x).
- The test that failed expected the string for -0.0, but the method produced "0". The assertion error shows: Expected -0.0, got 0.

Why it happened (root cause)
- In Python a float of -0.0 compares equal to 0.0: (-0.0) == 0 is True. That makes the branch if x == int(x) take the integer-handling path.
- Converting the float to int with value = int(x) turns -0.0 into the integer 0 and loses the sign bit. From that moment the code only sees value == 0, so it prints "0" (via self.add(str(value))) instead of preserving the negative zero.
- The earlier check if x < 0 and prev == '-' does not catch -0.0 because (-0.0) < 0 is False in Python, so no special handling is done before conversion.

Exactly where to fix it
- The sign loss happens at the value = int(x) line. Detect negative zero before that conversion (or avoid int() for the -0.0 case).

Concrete fixes (step-by-step)
1) Import math if not already present (you already use math elsewhere).
2) Right after computing prev = self.get_last_char(), add a check for negative zero:
   if x == 0.0 and math.copysign(1.0, x) < 0:
       # preserve the -0.0 representation
       self.add(str(x))
       return
   This uses math.copysign to detect the sign bit even when the numeric value is zero.
3) Alternatively, if you prefer handling it later in the integer branch, change the integer-path to preserve sign before int():
   if x == int(x) and not math.isinf(x) and not math.isnan(x):
       if x == 0.0 and math.copysign(1.0, x) < 0:
           self.add(str(x))
           return
       value = int(x)
       ...
4) Remove reliance on int(x) to detect an integer when you must preserve -0.0; always check math.copysign for zero first.

Why this fix works
- math.copysign(1.0, x) < 0 tests the sign bit directly, so it distinguishes -0.0 from +0.0 even though they compare equal numerically.
- Handling -0.0 before int(x) prevents losing the sign when you convert to int or later format the value.
