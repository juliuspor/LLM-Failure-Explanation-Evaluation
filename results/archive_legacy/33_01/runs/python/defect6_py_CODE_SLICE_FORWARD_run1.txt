What fails
- The code calls math.isinf(x) and math.isnan(x) on this line:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- If the math module was not imported in the file, that line raises NameError: name 'math' is not defined and the method crashes at that point.

Why it fails (root cause)
- The failure is a missing import: the name math is not defined in the current scope. The code is trying to call functions on the math module but never imported it, so Python cannot resolve the identifier math.

Exactly where it happens
- At the if statement shown above (the line that uses math.isinf and math.isnan). Execution never reaches the integer-trimming loop because the NameError occurs before that.

How to fix (concrete steps)
1. At the top of the module (or before this method is used), add:
   import math
2. Alternatively, if you only need those two predicates you can import them explicitly:
   from math import isinf, isnan
   and then change the line to:
   if x == int(x) and not isinf(x) and not isnan(x):
3. After adding the import, run the unit(s) or a quick manual test:
   - Call add_number(1000.0) expecting "1E3" (because exp > 2).
   - Call add_number(-4.0) and add_number(3.14) to check other branches.

Note
- If you already have import math elsewhere, ensure it is in the same module (not only imported inside a conditional or a different scope) so math is defined when add_number runs.
