The bug is caused by treating isolated UTF-16 surrogate halves as valid code points when falling back in translate(). To fix, the translate method should detect unpaired high or low surrogates and raise an error instead of passing them through. Implement translate to examine code_units at pos: if a valid surrogate pair, copy both units; if an unpaired high or any low surrogate at start, raise ValueError; otherwise copy single code unit. Preserve existing behavior for None checks and out argument validation.
