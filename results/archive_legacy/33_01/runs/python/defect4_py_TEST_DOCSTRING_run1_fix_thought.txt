The root issue is stale stored index values when items are inserted before existing elements. The fix increments any stored index that is >= the new element's index to account for the shift, then computes start/end/middle for the new period and compares against current extreme periods to update indices. Also fixed a bug where max_middle_period used the wrong stored index variable. The implementation handles SimpleTimePeriod and generic TimePeriod consistently and preserves existing behavior when indices are -1.
