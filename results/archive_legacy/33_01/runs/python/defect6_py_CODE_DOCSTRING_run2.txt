What goes wrong
- The failure occurs inside add_number, at the while loop that starts
  while mantissa % 10 == 0:
  (this is inside the branch where x == int(x) …).
- If value == 0 (i.e. x is 0.0), then mantissa is 0. The loop condition mantissa % 10 == 0 is always true for mantissa == 0, and check_val is computed as 0 every iteration, so the if check_val == value always succeeds, mantissa //= 10 leaves mantissa at 0, exp increments, and the loop never terminates. That causes an infinite loop (hang) in add_number when x is 0.0.

Why that happens (root cause)
- The code assumes you can repeatedly divide mantissa by 10 until a non-zero last digit appears. That assumption fails for mantissa == 0 because dividing 0 by 10 stays 0 forever. There is no guard against mantissa being zero before entering the loop.

Precise location
- Method: add_number
- Line: the while loop beginning with while mantissa % 10 == 0:
- Variables: value (== int(x)), mantissa (initialized to value), exp (initialized to 0)

Concrete fixes (pick one)
1) Add a zero check before the loop:
   if value == 0:
       self.add("0")
       return
   Then the existing while loop is safe for non-zero mantissa.

2) Change the loop condition to avoid running when mantissa is zero:
   while mantissa != 0 and mantissa % 10 == 0:
       ...
   This prevents the infinite loop; afterwards, handle exp and mantissa as you already do.

3) Explicitly break when mantissa becomes zero inside the loop:
   while mantissa % 10 == 0:
       check_val = ...
       if check_val == value:
           if mantissa == 0:
               break
           mantissa //= 10
           exp += 1
       else:
           break

Recommendation
- Option 2 (while mantissa != 0 and mantissa % 10 == 0) is the smallest safe change. If you also want to display 0 in a compact form (not "0E..."), add the simple special-case from option 1 to return "0" immediately for value == 0.

No other parts of the method need to change for this bug; the infinite loop is entirely due to unguarded mantissa == 0 behavior in that while loop.
