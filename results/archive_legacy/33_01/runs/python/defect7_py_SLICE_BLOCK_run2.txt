What happened (where it fails)
- The failure happens at line 766: array[i].upper().
- That call assumes array[i] is a string, but at least one element in array is not a string.
- Python raises an AttributeError like: 'int' object has no attribute 'upper' (or similar for whatever non-string type is present).
- Because the exception is raised, the loop stops and classes.append(type(array[i])) on line 767 may never run for that item.

Why it fails (root cause)
- .upper() is a string method. Only objects of types that implement upper (str, maybe custom classes) support it.
- The code does not check the element type before calling .upper(), so a non-string element causes the AttributeError.
- Assigning the result to _ (line 766) masks that the returned value is ignored; that doesn’t prevent the error.

Exact location to look:
- Line 766: calling array[i].upper() without a type check
- Line 767: classes.append(type(array[i])) — this line shows you are interested in element types, which is why a non-string element can appear in array.

Simple concrete fixes (pick one)

1) Only call upper() for strings
- Replace lines 765–767 with:
  if isinstance(array[i], str):
      val = array[i].upper()
  else:
      val = array[i]
  classes.append(type(array[i]))
- This prevents calling .upper() on non-strings and preserves the element for later use.

2) Convert everything to string first (if you want uppercase string form)
- If your goal is to get an uppercase representation for every element:
  val = str(array[i]).upper()
  classes.append(type(array[i]))
- This never raises AttributeError because str(...) works for any object.

3) Catch the error explicitly
- Surround the call with try/except to handle unexpected types:
  try:
      val = array[i].upper()
  except AttributeError:
      val = array[i]   # or val = str(array[i]).upper()
  classes.append(type(array[i]))

4) If you only wanted the type names (and not uppercasing), remove the .upper() call
- Replace line 766 with nothing (or remove the line) if .upper() was unnecessary.

Which fix to choose
- Use option 1 if you only want to uppercase actual strings.
- Use option 2 if you want a string uppercase representation for every element.
- Use option 3 if you expect mixed types and need a fallback.
- Remove .upper() if it was accidental.
