Why it happened (root cause)
- The code unconditionally calls array[i].upper() for every element in the input array:
    _ = array[i].upper()
- For the second element in your test input, array[1] is None. Calling None.upper() raises AttributeError because None has no attribute upper.
- That is exactly the message you see: 'NoneType' object has no attribute 'upper'.

Why the code was likely written this way (intent vs reality)
- It appears the author wanted to ensure each element was a string-like value, maybe to validate or normalize before calling type(array[i]). However:
  - Calling .upper() makes no sense for non-string elements (like None or floats).
  - The method is supposed to accept arbitrary objects and return their classes (or None when the input element is None), per its docstring. Running .upper() breaks that contract.

What you should change (concrete fixes with steps)
1. Remove the unnecessary .upper() call entirely. The method only needs to get the element's type.
   - Replace the loop body:
       for i in range(len(array)):
           _ = array[i].upper()
           classes.append(type(array[i]))
     with:
       for i in range(len(array)):
           if array[i] is None:
               classes.append(None)
           else:
               classes.append(type(array[i]))
2. Or, more Pythonically:
       for elem in array:
           classes.append(None if elem is None else type(elem))
3. Update the docstring or behavior to match what you want the method to do:
   - If you want to raise on None elements, explicitly check and raise with a clear message.
   - If you want None entries to map to None in the result (as your tests expect), implement as step 1/2.

One-line patch example
