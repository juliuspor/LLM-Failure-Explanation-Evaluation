What happened
- Failure occurs in ClassUtils.to_class when called with array = ['Test', None, 99.0].
- The traceback shows AttributeError: 'NoneType' object has no attribute 'upper'.
- The offending line is classes.append(type(array[i])) at source line shown, but the real problem comes just before that: the code does "_ = array[i].upper()" unconditionally (line ~770 in your slice), which tries to call .upper() on the element at index 1 (which is None). None.upper() raises AttributeError, which is what you see.

Why it happened (root cause)
- In to_class you have:
    for i in range(len(array)):
        _ = array[i].upper()
        classes.append(type(array[i]))
- That first statement assumes every array element is a string-like object supporting .upper(). For None (or other non-string objects like float), this fails.
- The intent appears to be to perform some check or normalization on string elements, but calling .upper() on every element with no type check causes the crash when an element is None (or not a str).

Where it occurs
- Method: ClassUtils.to_class
- Exact place: inside the for loop, the line calling .upper() on array[i] (your "RELEVANT_SLICE" shows the next line with classes.append, but the AttributeError is raised at the .upper() call immediately above it).
- Test that triggers it: ClassUtils.to_class(['Test', None, 99.0])

Concrete fixes (clear steps)
1. If the .upper() call is not needed, remove it. That will make the method simply return types for each element (None -> None? see next point).
   - Replace the loop with:
        for item in array:
            classes.append(type(item) if item is not None else type(None))
   - Or, to preserve the original behavior of inserting None for original None elements, use:
        classes.append(None if item is None else type(item))

2. If the intent was to normalize only string elements (e.g., to treat type names), guard the call:
        for item in array:
            if isinstance(item, str):
                _ = item.upper()
            # then append type or None as appropriate
            classes.append(None if item is None else type(item))

3. Decide how to represent None in the returned list:
   - The test expects [str, type(None), float], i.e. None element should produce type(None), not None. So implement:
        classes.append(type(item) if item is not None else type(None))

4. Replace eval usages elsewhere if any similar assumptions exist; but for this specific test, changing to one of the patterns above fixes the crash and satisfies the expected output.

Minimal corrected implementation for to_class (drop the erroneous .upper() and handle None):
    if array is None:
        return None
    elif len(array) == 0:
        return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    classes = []
    for item in array:
        classes.append(type(item) if item is not None else type(None))
    return classes

Why this fix works
- It removes the unsafe .upper() call that raises on None.
- It produces type(None) for None elements, matching your test expectation.
- It avoids assumptions about element types and thus won't fail for floats or other objects.
