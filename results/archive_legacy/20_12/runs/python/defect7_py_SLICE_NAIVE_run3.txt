What happened (root cause)
- The exception AttributeError: 'NoneType' object has no attribute 'upper' occurs on line 769 in method to_class():
  - Code: _ = array[i].upper()
- The test calls ClassUtils.to_class(['Test', None, 99.0]). The second element is None.
- When the loop reaches i == 1, array[1] is None, so array[1].upper() attempts to call upper() on None and raises the AttributeError.

Why this line exists and why it is wrong
- The comment and docstring for to_class say: "If any of these objects is None, a None element will be inserted into the array." But the code instead unconditionally calls .upper() on every element before using type(array[i]).
- That .upper() call serves no meaningful purpose here (perhaps a leftover check) and is invalid for non-string values (None, numbers, objects) — hence the crash.

Where it occurs (specific references)
- Method: ClassUtils.to_class
- File lines: 767–771 of the provided slice; failing statement at line 769:
    _ = array[i].upper()
- Test: result = ClassUtils.to_class(['Test', None, 99.0])

What to do (concrete fixes)
1. Remove the erroneous .upper() call — it is unnecessary. Replace the loop with:
   for i in range(len(array)):
       classes.append(type(array[i]))
   This will correctly append None's type as type(None) for None entries and float for 99.0.

2. If the intended behavior was to preserve None as None in the result (the docstring might mean "None element will be inserted" meaning result should contain None rather than type(None)), change the loop to explicitly handle None:
   for elt in array:
       if elt is None:
           classes.append(type(None))   # or append(None) if you want None in result
       else:
           classes.append(type(elt))

   Decide whether the expected output for None is type(None) (i.e. <class 'NoneType'>) or literal None. The failing test expects type(None) (see assert result == [str, type(None), float]), so use type(None).

3. Optional: add input validation and clearer docstring:
   - Update the docstring to state that None elements become type(None).
   - If you want to raise a helpful error for unexpected values, raise a ValueError with a clear message instead of letting an AttributeError bubble up.
