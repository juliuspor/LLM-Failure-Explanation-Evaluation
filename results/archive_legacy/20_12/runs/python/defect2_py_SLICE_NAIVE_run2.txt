What happened (where and why)
- Failure occurs in GrayPaintScale.get_paint at line 73–77.
- You correctly clamp the input to v with:
  - line 70: v = max(value, self._lower_bound)
  - line 71: v = min(v, self._upper_bound)
  But then you never use v.
- Instead you compute g using the original value:
  - line 73: g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If the passed value is outside the bounds (e.g. value = -0.5, lower_bound = 0.0), that formula produces a negative g (here g < 0). The subsequent check at lines 76–77 raises ValueError: "Color parameter outside of expected range..."

Root cause
- A simple variable mix-up: you clamp into v but compute the gray level from value. Because the out-of-range value is used, g can be outside 0–255 and triggers the exception.

Exact location to change
- In get_paint, replace the expression that uses value with v (the clamped value). Specifically change line 73 from:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  to:
  g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

Steps to fix
1. Open GrayPaintScale.get_paint.
2. On line 73, use v instead of value when computing g (see code above).
3. Run the test:
   gps = GrayPaintScale(); color = gps.get_paint(-0.5); assert color == (0, 0, 0)
