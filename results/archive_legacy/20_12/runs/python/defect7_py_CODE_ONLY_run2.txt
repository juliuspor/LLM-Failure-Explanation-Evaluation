What happened
- The failure occurs in ClassUtils.to_class when you call ClassUtils.to_class(['Test', None, 99.0]).
- The traceback ends with AttributeError: 'NoneType' object has no attribute 'upper'.

Why it happened (root cause)
- In to_class (around these lines):
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))
- The code unconditionally calls array[i].upper() for every element. When array[i] is None, that expression becomes None.upper(), which raises AttributeError because NoneType has no method upper().
- The method incorrectly assumes every element is a string (or at least an object with upper()), but the input can contain None and non-strings (e.g., float 99.0). The test includes None, so the AttributeError is triggered immediately on the second element.

Where in the code it occurs
- Method: ClassUtils.to_class
- The offending line is the call to .upper() on the array element inside the loop. (Look for "_ = array[i].upper()" in to_class.)

What to do (concrete fixes, step by step)
1. Remove the unnecessary .upper() call altogether. It serves no purpose — the method only needs the element's type, which is obtained via type(array[i]).
   - Replace:
       _ = array[i].upper()
       classes.append(type(array[i]))
     with:
       classes.append(type(array[i]))
2. If the intent was to treat string element names specially (e.g., map class-name strings to actual classes), explicitly handle string and None cases, for example:
   - If element is None: append None
   - elif isinstance(element, str): try to resolve the string to a class (import/eval) and append that class
   - else: append type(element)
   That avoids calling string methods on None and makes behavior explicit.
3. Add tests for mixed inputs (None, real objects, class-name strings) to confirm behavior.

Why the proposed change fixes it
- Removing or guarding the .upper() call prevents calling a string method on None. Using explicit type checks ensures None and non-string values are handled safely and intentionally.

Short example patch (conceptual)
- Minimal fix (remove the stray .upper()):
    for i in range(len(array)):
        classes.append(type(array[i]))

- Safer handling:
    for val in array:
        if val is None:
            classes.append(None)
        elif isinstance(val, str):
            # resolve string to class or treat as str class
            try:
                classes.append(eval(val))
            except Exception:
                classes.append(str)
        else:
            classes.append(type(val))
