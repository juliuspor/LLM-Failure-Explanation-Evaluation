What happened (where it failed)
- The failure happens in GrayPaintScale.get_paint at line 73–77.
- The test calls gps.get_paint(-0.5). Inside get_paint you clamp the input to the bounds and store the clamped value in v (lines 69–71), but then you compute g using the original value variable instead of the clamped v (line 73).
- Because value is -0.5 and lower_bound is 0.0, the computed g becomes negative, triggering the ValueError at line 76.

Why it happened (root cause)
- You intended to clamp the input to the [lower_bound, upper_bound] range and then compute the grayscale level from that clamped value, but you never used the clamped variable v when computing g. Using the un-clamped value makes g fall outside 0–255 for inputs below lower_bound or above upper_bound, causing the explicit range check to raise ValueError.

Exactly where to change the code (fix steps)
1. In GrayPaintScale.get_paint, change the computation of g to use v (the clamped value) instead of value. Replace line 73:
   g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
   with:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

2. Keep the range check that follows to catch any unexpected rounding issues.

After this change, gps.get_paint(-0.5) will clamp to 0.0 and return (0, 0, 0) as the test expects.
