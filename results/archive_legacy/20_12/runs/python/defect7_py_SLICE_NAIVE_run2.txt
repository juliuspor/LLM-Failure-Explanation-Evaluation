What happened

Why it happens (root cause)
- to_class is iterating every element of the input array and immediately calls .upper() on that element without checking it for None.
- If any element of the input list is None (as in the test: ['Test', None, 99.0]), then array[i] is None and None.upper() raises AttributeError because None has no attribute upper.
- After that failed call the function never reaches the intended behavior of appending the element's type.

- Instead the code tries to call .upper() unconditionally, which only makes sense for strings and breaks for None and other non-string types (like floats).
