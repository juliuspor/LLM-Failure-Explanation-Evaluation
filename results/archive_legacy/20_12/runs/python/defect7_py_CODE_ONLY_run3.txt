What happened
- The failure occurs inside ClassUtils.to_class when it processes the input list ['Test', None, 99.0].
- The exception is AttributeError: 'NoneType' object has no attribute 'upper'.

Where it happened (exact location)
- In method ClassUtils.to_class, loop body:
  _ = array[i].upper()
  classes.append(type(array[i]))
- For the second element of the input list (None), array[i] is None, so array[i].upper() tries to call .upper() on None and raises AttributeError.
- This is at the two lines beginning the per-element loop in to_class (the lines with .upper() and type(...)).

Why it happened (root cause)
- The to_class implementation contains an unnecessary and unsafe call to array[i].upper(). That call assumes every element is a string (so .upper() exists), but the method contract explicitly allows None elements (and non-string objects such as floats).
- The method should not call .upper() on elements (and certainly not before checking for None). As written, when an element is None the call array[i].upper() fails immediately, so the method never reaches the logic that would append None or the correct type.

What the method intended to do
- According to its docstring and the unit test, to_class should return a list of classes/types corresponding to each element:
  - If element is None -> append None (or possibly type(None))
  - If element is an object -> append type(object)
- The stray .upper() call serves no purpose for that behavior.

Concrete, actionable fixes (step-by-step)
1. Remove the unnecessary .upper() call entirely.
   Replace the loop body:
     for i in range(len(array)):
         _ = array[i].upper()
         classes.append(type(array[i]))
   with:
     for i in range(len(array)):
         elem = array[i]
         if elem is None:
             classes.append(type(None))   # or classes.append(None) depending on desired behavior
         else:
             classes.append(type(elem))
2. Decide on the expected representation for None:
   - The test expects [str, type(None), float]. So for None you must append type(None), not None.
   - Implement accordingly:
         if elem is None:
             classes.append(type(None))
         else:
             classes.append(type(elem))
3. (Optional) Make the loop more Pythonic and robust:
     for elem in array:
         classes.append(type(elem) if elem is not None else type(None))
   or explicitly:
     for elem in array:
         if elem is None:
             classes.append(type(None))
         else:
             classes.append(type(elem))

Why this fixes it
- Removing the .upper() call avoids calling methods on None or non-string elements.
- Explicitly checking for None ensures the method follows its contract and the unit test expectation (None element maps to type(None)).
- The loop then always appends a valid Type object for each element, preventing the AttributeError.

Minimal code patch example
- Replace the current loop in to_class with:

    for elem in array:
        if elem is None:
            classes.append(type(None))
        else:
            classes.append(type(elem))

This will make the test pass: ClassUtils.to_class(['Test', None, 99.0]) -> [str, type(None), float].
