What happened (where it fails)
- The exception is raised in GrayPaintScale.get_paint at lines 73–77.
- You compute g using the original variable value (line 73), but you intended to use the clamped value v (lines 69–71).
- Because you ignore the clamped v, passing value = -0.5 produces g = int((-0.5 - 0.0) / (1.0 - 0.0) * 255.0) = -127, which makes the check at line 76 trigger and raises the ValueError.

Why it happened (root cause)
- The code clamps the input into v (lines 69–71) but then mistakenly uses value in the grayscale calculation (line 73). That means values outside the bounds are not actually clamped before computing g.

Precise location
- get_paint(), calculation at line 73:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  should use v instead of value.

How to fix it (concrete steps)
1. Change line 73 to use the clamped variable v:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
2. Keep the range check that follows (lines 75–77); it will now only trigger for real numeric errors (should not trigger for inputs clamped to bounds).
3. Optional: for clarity rename v to clamped_value or similar.

After this change, your test gps.get_paint(-0.5) will compute g = 0 and return (0, 0, 0) as expected.
