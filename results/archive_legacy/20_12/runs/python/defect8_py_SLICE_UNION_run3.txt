What happened (where it fails)
- The failure occurs in LocaleUtils.to_locale (starting at line 193).
- The code raised ValueError: "Invalid locale format: fr__POSIX".
- The check that throws is at lines 242–245 where it validates the two country characters (ch3 and ch4) as uppercase letters A–Z.

Why it fails (root cause)
- The input "fr__POSIX" is intended to mean language "fr", empty country "", variant "POSIX".
- Your code accepts three valid shapes: length 2 ("xx"), length 5 ("xx_YY"), or length >=7 ("xx_YY_xxx"). To accept the empty-country form "xx__variant" you must allow the underscore at positions 2 and 5 and an empty country (positions 3–4 blank).
- But the implementation assumes that when length != 2 you always have a non-empty two-letter country at positions 3–4. It unconditionally reads ch3 = locale_str[3] and ch4 = locale_str[4] and checks they are between 'A' and 'Z' (lines 242–245). For "fr__POSIX", locale_str[3] == '_' and locale_str[4] == 'P', so ch3 fails the A–Z test and triggers ValueError.
- In short: the code does not handle the valid "language__variant" form (empty country) and wrongly enforces a non-empty country when deciding between the country-only and country+variant forms.

Exactly where to change (concrete steps)
1. At to_locale, after verifying locale_str[2] == '_', do not unconditionally inspect characters 3 and 4 as uppercase letters.
2. Distinguish three cases:
   - length == 5: expect country letters at 3–4 and validate A–Z (current code handles this).
   - length >= 7 and locale_str[5] == '_':
     - Either country is two uppercase letters (e.g. "fr_FR_POSIX"), or country is empty (locale_str[3:5] == "__") for the form "fr__POSIX".
     - So check if locale_str[3] == '_' (empty country); if so, treat country as "" and take variant = locale_str[6:].
     - Otherwise validate that locale_str[3:5] are uppercase letters and proceed to parse country and variant.
3. Concretely: replace the unconditional ch3/ch4 checks (lines 242–245) with logic that first checks length and whether locale_str[3] == '_' to allow empty country, otherwise validate the two country letters only when they are present.
