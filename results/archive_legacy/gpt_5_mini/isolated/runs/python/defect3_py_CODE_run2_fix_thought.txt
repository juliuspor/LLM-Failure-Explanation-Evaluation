translate_codepoint currently returns the number of UTF-16 code units consumed (CSV translator returns 2 when it handles a surrogate pair). The original loop treated consumed as a number of codepoints and advanced pos per-codepoint using char_count and code_point_at, which double-advanced and skipped characters. The fix is to advance pos by the consumed code units directly (pos += consumed). Keep behavior for consumed==0 the same. Also keep input validation and conversion to UTF-16 as before.
