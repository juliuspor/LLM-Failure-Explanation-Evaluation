What happened (where, exactly)
- The crash happens in the loop at lines 765–767:
  - line 766 calls array[i].upper()
  - line 767 appends type(array[i])
- When array contains a None element (for example array[1] == None in your failing test), line 766 executes None.upper(), which raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The variable _ (result of .upper()) is never used, so that call serves no purpose other than causing the exception.

Why it happened (root cause)
- The code assumes every element is a string (or at least has an upper() method). That assumption is false because the input array can contain None or other non-string objects.
- The unnecessary call to .upper() is the direct cause of the AttributeError. The real goal of the method is to return the element types (classes), not to transform string contents, so calling .upper() is both irrelevant and unsafe.

Concrete fixes (step‑by‑step)
1. Remove the .upper() call entirely. It is unused and not needed:
   - Replace the loop body with just classes.append(type(array[i])) (or the clearer form below).

2. Ensure None elements map to type(None). Example corrected code:
   - Simple loop:
     def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
         if array is None:
             return None
         elif len(array) == 0:
             return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
         classes = []
         for item in array:
             if item is None:
                 classes.append(type(None))
             else:
                 classes.append(type(item))
         return classes
   - Or concise comprehension:
     classes = [type(item) if item is not None else type(None) for item in array]

3. (Optional) If you intended to normalize string elements (e.g., uppercase) do that separately and only for str instances:
   if isinstance(item, str):
       normalized = item.upper()
   but do not call .upper() without checking for None or non-str types.

Why this fixes the tests
- The corrected code does not call methods on None, so test_to_class_with_none_element and other tests with None or mixed types will produce the expected list of types, e.g. [str, type(None), float].

Notes
- The special-case for empty array (ArrayUtils.EMPTY_CLASS_ARRAY.copy()) is fine.
- Make sure the method returns type(None) for None elements (tests expect that), not the literal None.
