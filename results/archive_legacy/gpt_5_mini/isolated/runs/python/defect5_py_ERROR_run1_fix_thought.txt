The bug stems from inferring the list type via type(array) which yields list type, not element/component type, and causes incorrect type-checking/casting behavior similar to Java ClassCastException. Fix: infer the element type by scanning existing elements for first non-None element; if none use element's type; fallback to object. Preserve expected_type check behavior raising TypeError when element type unknown (object) but expected_type is specific. Use _copy_list_grow1 to build new list and append element.
