What happens
- At lines 102–108 the code has a boolean expression but never uses it. The fragment is these lines:
  - 102: no_more_matches_for_repl_index[i]
  - 103: or search_list[i] is None
  - 104: or len(search_list[i]) == 0
  - 105: or replacement_list[i] is None
  - 108:     temp_index = text.find(search_list[i], start)

- The intent is clearly to skip this replacement index when any of those conditions is true (i.e. do "if <conditions>: continue"). Because the leading "if" and "continue" are missing, the code always runs line 108 and calls text.find(search_list[i], start) even when search_list[i] is None or empty or replacement_list[i] is None.

Why that breaks (root cause)
- Calling text.find(search_list[i], start) with search_list[i] == None (or other invalid values) raises an error (TypeError: expected str or bytes-like object, not NoneType, or similar), so the loop crashes at line 108.
- The boolean expression on lines 102–105 evaluates but does nothing by itself; the code does not skip the index. So the logic that was supposed to prevent invalid values from being used never runs.

Where it occurs
- The error occurs inside the loop that selects the next match (the code block that computes temp_index and updates text_index / replace_index). The failing call is at line 108: temp_index = text.find(search_list[i], start). The condition that should prevent that call is on lines 102–105, but it is not applied.

How to fix it (concrete steps)
1. Replace the boolean-expression lines with an actual if that continues the loop. For example, immediately before line 108 add:

       or search_list[i] is None
       or len(search_list[i]) == 0
       or replacement_list[i] is None):

2. Keep the indent so temp_index = text.find(...) is only executed when the entry is valid.

Why this fix works
- The explicit if ...: continue enforces the original intent: skip indices that have no further matches or invalid search/replacement entries. That prevents calling text.find with None or empty values and avoids the TypeError crash at line 108.
