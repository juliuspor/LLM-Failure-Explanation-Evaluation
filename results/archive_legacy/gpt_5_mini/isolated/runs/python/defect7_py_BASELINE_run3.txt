What happened (root cause)
- At line 766 the code does: _ = array[i].upper()
- If array[i] is None, calling .upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The tests include None elements (e.g. test_array = ["Test", None, 99.0]), so the loop reaches a None and fails on that call.

Why that line is wrong
- .upper() is a string method. The code assumes every element is a str and calls .upper() unconditionally.
- The result "_" is never used anywhere — that call serves no purpose for constructing the classes list (line 767).
- Even when elements are strings, changing their case is irrelevant to returning their class/type.

Where it occurs
- In to_class, inside the for loop:
  - line 765: for i in range(len(array)):
  - line 766: _ = array[i].upper()  <-- raises for None
  - line 767: classes.append(type(array[i]))

What to do (concrete fixes and steps)
1. Remove the unnecessary .upper() call.
2. Handle None elements explicitly. The tests expect type(None) for None entries, so append type(None) for None values.

Suggested replacement code snippet (minimal change):
    for i in range(len(array)):
        elem = array[i]
        if elem is None:
            classes.append(type(None))
        else:
            classes.append(type(elem))

Or more compact:
    for elem in array:
        classes.append(type(elem) if elem is not None else type(None))

Notes
- Keep the existing early returns for None input and empty array (lines before the loop).
- After this change, the function will no longer raise AttributeError for None elements and will return the expected [str, type(None), float] for ["Test", None, 99.0].
