The original to_locale used rigid index-based checks that could raise IndexError or incorrectly validate. Replace with a safe parser that splits on '_' and validates parts: None returns None; language must be exactly 2 lowercase letters; optional country must be exactly 2 uppercase letters if present; optional variant must be non-empty; reject more than 3 parts or empty variant when third part present; construct and return Locale(language, country, variant). This matches the docstring and avoids index errors.
