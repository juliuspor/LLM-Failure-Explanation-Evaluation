What happened (root cause)
- At line 402 you set minutes_offset = hours_in_minutes - minutes_offset.
- At line 404 you then immediately set minutes_offset = hours_in_minutes + minutes_offset.
- Because both assignments execute, the second one overwrites the first. That leaves minutes_offset with the value hours_in_minutes + original_minutes_offset instead of the intended either (hours_in_minutes - original_minutes_offset) or (hours_in_minutes + original_minutes_offset) depending on a condition that is missing here.
- That incorrect minutes_offset is then multiplied into millis at line 405 by FieldUtils.safe_multiply and becomes larger than the allowed range. The code hits the check at line 407 and raises ValueError("Offset is too large"). The failure is therefore caused by the wrong minutes_offset value produced by the two back-to-back assignments.

Why that causes the error (concrete)
- The code intended to pick one formula for minutes_offset but instead executes both, leaving the second formula as the final value. That final value is larger than expected. FieldUtils.safe_multiply then computes an out-of-range millisecond offset, causing the later range check to fail and ValueError to be raised.

2. Replace the two unconditional assignments with an if/else so only one assignment runs.
