What happened (where, in code)

- At line 176 the code tests: if x == int(x) and not math.isinf(x) and not math.isnan(x): — this is true for -0.0 because -0.0 == 0 is True in Python.
- At line 177 value = int(x) converts -0.0 to 0 (int(-0.0) == 0).
- The integer branch eventually executes self.add(str(value)) at line 192, which writes "0".
- That loses the sign bit of negative zero, so the test expecting "-0.0" fails with "0".

Why this happens (root cause)

- The equality check x == int(x) does not distinguish -0.0 from +0.0. Both compare equal to 0 and int(-0.0) yields 0.
- There is no special-case handling for negative zero before converting to an int, so the sign is discarded when value is built and stringified.
