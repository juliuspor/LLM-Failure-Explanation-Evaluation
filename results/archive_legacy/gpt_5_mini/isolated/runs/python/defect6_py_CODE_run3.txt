What fails
- The code calls math.isinf(x) and math.isnan(x) on this line:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- If you have not imported the math module, Python raises:
  NameError: name 'math' is not defined
  at that line, so the method never completes.

Why it happens (root cause)
- The identifier math is used but never bound in the shown snippet. Python looks up names at runtime; because math was not defined (no import math nor from math import ...), name lookup fails and execution stops with NameError.

Exactly where it happens
- The failure occurs at the if-check that uses math (the line shown above). The NameError is raised when evaluating math.isinf(x) (or math.isnan(x)).

Concrete fixes (pick one)
1) Import the module at the top of the file
   - Add this line near your other imports:
     import math
   - No other change needed.

2) Import just the functions you use
   - At top:
     from math import isinf, isnan
   - And change the if to:
     if x == int(x) and not isinf(x) and not isnan(x):

3) Use math.isfinite instead (if you prefer one check)
   - At top: import math
   - Replace the if with:
     if x == int(x) and math.isfinite(x):
   - This avoids separate isinf/isnan calls.

Any of these will remove the NameError and allow the rest of the method (the mantissa/exp logic and self.add calls) to run.
