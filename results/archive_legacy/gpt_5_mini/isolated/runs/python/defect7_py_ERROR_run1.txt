Why this happens (concrete causes that produce None here)
- You called .upper() directly on a function return that can be None. Common examples:
  - dict.get(key) returns None if key missing: d.get("name").upper() fails if "name" not present.
  - regex search returned no match: re.search(..., s) is None, so re.search(...).group(1).upper() will fail (the .group call would already fail if search returns None, but code may call .upper() on a subsequent value).
  - a function explicitly returns None on error: myparser.parse(x) might return None for bad input, then you call .upper() on its result.
  - chaining methods where an intermediate step yields None.
- You unintentionally overwrote a string variable with None earlier:
  - value = fetch()  # returns None on failure
  - value = None
  - then value.upper()
