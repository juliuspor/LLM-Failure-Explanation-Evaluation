What happened (root cause)
- The test crashed because some variable that your code calls .upper() on is None. NoneType has no .upper attribute, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- In short: a value expected to be a string is actually None, and your code immediately calls .upper() on it.

How to find the exact place in your code
1. Open the full traceback the test produced (the test runner prints it above the AttributeError). The traceback shows a file name and line number where .upper() was called. Example lines in a traceback look like:
   File "my_module.py", line 42, in normalize_name
     name = name.upper()

2. Go to that file and the exact line number. That line is where the None value was used.

Three common causes and how to check which one applies to you
- Cause A: A dictionary lookup returned None.
  - Symptom: code like name = data.get('name'); name = name.upper()
  - Check: Print or log the result of data.get('name') before calling .upper(). If data.get('name') is None, the key is missing or mapped to None.

- Cause B: A function returned None instead of a string.
  - Symptom: code like name = build_name(user); name = name.upper()
  - Check: Inspect build_name and call it manually in a REPL or add a log to show its return value. If some code path in build_name has no return or returns None, fix that.

- Cause C: A variable was overwritten with None earlier.
  - Symptom: name is assigned multiple times; one assignment sets it to None.
  - Check: Search earlier lines in the same function for assignments to that variable and add a quick print/log before .upper().

Concrete fixes (pick one appropriate)
- Defensive check before calling .upper():
  - if name is None:
      # handle missing value explicitly
      name = ''
    name = name.upper()
  - Use a one-liner when empty string is acceptable:
    name = (name or '').upper()

- Ensure the provider returns a string:
  - If a function can return None, change it to always return a string (e.g., return '' instead of return None), or document that callers must handle None.

- Validate input earlier:
  - If input comes from external data (request, config, DB), validate and raise a clear error if a mandatory string is missing:
    if data.get('name') is None:
      raise ValueError("missing required 'name'")

Step-by-step debugging checklist you can follow now
1. Re-run the failing test and copy the full traceback.
2. Open the file and line number shown in the traceback.
3. Identify the exact expression that uses .upper() and note the variable name (e.g., name).
4. Insert a temporary log or breakpoint right before that line:
   print("DEBUG: name repr:", repr(name))
   or use a debugger to inspect the value.
5. Trace backwards to see where that variable gets its value. If it comes from:
   - dict.get(...) — check the key exists
   - function call — inspect that function's return paths
   - earlier assignment — look for branches that assign None
6. Apply one of the concrete fixes above, then run the test again.
