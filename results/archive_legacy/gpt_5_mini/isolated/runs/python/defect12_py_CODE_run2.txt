What happens
- If the caller passes lower greater than text.length(), the function throws an exception from substring_java because you end up calling substring_java(text, 0, upper) with upper > text.length().

Why it happens (root cause, with exact lines)
1. You clamp upper down to len(text) here:
   if upper == -1 or upper > len(text):
       upper = len(text)
   (lines 6–8 of the snippet)
2. Then you make upper follow lower if upper < lower:
   if upper < lower:
       upper = lower
   (lines 9–10)
   If lower > len(text), this sets upper to a value > len(text) again.
3. Later you call substring_java(text, 0, upper) (lines 15 and 22). Passing an end index > text.length() causes the substring implementation to fail (IndexOutOfBounds or equivalent).

Where it fails
- The failure occurs at either of the two places that call substring_java(text, 0, upper):
  - inside the block where index == -1 (the no-space case),
  - or where index > upper you return substring_java(text, 0, upper) + ...
- It is caused before or at the substring call; the problem originates at the upper = lower assignment (line 9–10) when lower > len(text).

How to fix (clear steps)
Option A — clamp lower to the string length before you set upper from lower:
1. After checking text and len(text), add:
   if lower > len(text):
       lower = len(text)
2. Keep the existing logic for upper, or re-run upper clamp if needed.

Option B — ensure upper is clamped after you set it from lower:
1. Change the order or add a final clamp:
   if upper < lower:
       upper = lower
   if upper > len(text):
       upper = len(text)
This guarantees upper is never greater than text.length() when passed to substring_java.

Minimal code change (Option B, exact lines)
Replace:
   if upper < lower:
       upper = lower
with:
   if upper < lower:
       upper = lower
   if upper > len(text):
       upper = len(text)

Why this fix works
- It prevents upper from exceeding text.length() in all paths, so substring_java will always get a valid end index (0..len(text)), avoiding the out-of-bounds exception.

Notes
- The problematic sequence is: clamp upper -> set upper = lower (where lower may be out-of-range) -> call substring. The fixes clamp the values in the correct order so substring indices are safe.
