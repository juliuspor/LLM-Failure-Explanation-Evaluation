What fails (where it happens)
- The infinite loop occurs inside add_number, in the block that tries to strip trailing zeros:
  - specifically the while line: while mantissa % 10 == 0:
  - and the body that does mantissa //= 10 and exp += 1.
- If mantissa ever becomes 0, mantissa % 10 == 0 will always be true, mantissa //= 10 leaves mantissa at 0, and the loop never breaks.

Why it happens (root cause)
- The code assumes mantissa will shrink to a non-multiple-of-10 so the loop will terminate. That assumption is false when mantissa == 0.
- When mantissa == 0 the loop condition is always true and nothing in the loop changes that fact (mantissa//=10 keeps mantissa at 0), so you get an infinite loop.
- The problematic expressions are:
  - mantissa = value  (mantissa can be 0)
  - while mantissa % 10 == 0:  (line that drives the loop)
  - mantissa //= 10            (does not change 0, so the loop never ends)

Concrete example that triggers it
- If value becomes 0 and abs(x) >= 100 (the code only enters the while when abs(x) >= 100), the loop will run forever:
  - mantissa starts at 0
  - check mantissa % 10 == 0 → True
  - check_val computed and equals value (both 0)
  - mantissa //= 10 → still 0
  - exp += 1 → exp grows but does not affect mantissa%10
  - repeat forever

How to fix it (clear, step-by-step)
- Make the while loop refuse to run when mantissa is 0. Two concrete fixes (pick one):

A) Add an explicit zero guard:
  - Change the while condition to: while mantissa != 0 and mantissa % 10 == 0:
  - Why: when mantissa == 0 the loop is skipped and you avoid the infinite loop.

B) Handle value == 0 before the loop:
  - After value = int(x) add:
      if value == 0:
          self.add("0")
          return
  - Why: zero is a special case that should be printed as "0" anyway; you never need to try to strip trailing zeros.

- Either fix addresses the infinite-loop root cause directly and is minimal and deterministic.

Other notes
- The surrounding guard if abs(x) >= 100 usually prevents value==0 from entering the loop, but the safe fix is still necessary if value can be 0 by other code changes or if you later change the abs(x) threshold.
- Use the exact names shown above (add_number, mantissa, value, while mantissa % 10 == 0) when you edit the code so you change the correct lines.
