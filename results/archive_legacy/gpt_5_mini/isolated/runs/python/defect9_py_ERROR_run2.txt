Why that variable is None (most common root causes)
- A function that was supposed to return a list returned None because it missed a return statement (common when the function only returns inside an if branch).
  Example: def parse_items(...): if ok: return parsed_items  # else falls through -> returns None
- You overwrote the variable with None earlier in the same scope (items = None).
- A call to an API or helper returned None on error (response.json() or get_items() returned None).
- You called a function that mutates in place and returns None (e.g., list.sort() returns None; using result = list.sort(); then len(result) fails).

Concrete examples of the mistake and exactly where it appears
- Missing return:

      if flag:

- Using a mutating method that returns None:
  items = [3,1]
  items = items.sort()   # .sort() returns None; items is now None
  count = len(items)     # raises TypeError here

Precise fixes (choose the one that matches your situation)
- If a function sometimes fails to return:

- If you accidentally used a mutating method that returns None:
