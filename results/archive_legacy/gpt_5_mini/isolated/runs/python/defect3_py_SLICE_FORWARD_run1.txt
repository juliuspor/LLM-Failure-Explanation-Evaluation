What happened (where it fails)
- At line 39 you detect a high surrogate in high = code_units[index].
- If the following code unit is not a matching low surrogate, the function falls through to line 44 and returns high.
- Returning high (the raw 16-bit surrogate value) is the bug: a high-surrogate value (0xD800–0xDBFF) is not a valid Unicode scalar value by itself. Later code that expects a real Unicode code point (0..0x10FFFF, excluding surrogate range) will see an invalid value and fail.

Why it fails (root cause)
- The code assumes that when high is a surrogate but the next unit is not a valid low surrogate it can safely return high. It cannot.
- A lone high-surrogate must be handled as an error (or replaced), not returned as if it were a valid code point. The same problem exists symmetrically if a lone low surrogate appears: the function will return that raw low-surrogate value (also invalid).
