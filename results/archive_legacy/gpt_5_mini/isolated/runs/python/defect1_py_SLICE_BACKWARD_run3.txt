What happened (where it fails)
- The code raises a ValueError on lines 396–397:
  396 |         if minutes_offset < 0 or minutes_offset > 59:
  397 |             raise ValueError(f"Minutes out of range: {minutes_offset}")
- That check runs when the function is converting a total offset into hours and minutes. If the total offset is negative (for example UTC−05:30), minutes_offset ends up negative (e.g. −30), so the condition minutes_offset < 0 is true and the code raises the ValueError.

Why it fails (root cause)
- The root cause is how minutes_offset is computed earlier: the code allows minutes_offset to carry the sign of the whole offset. The validation assumes minutes_offset is always non-negative and in 0..59, but for negative offsets minutes_offset can be negative (−1..−59). Because the code checks minutes_offset < 0, valid negative offsets like −30 are treated as invalid and trigger the exception.

Concrete example
- Suppose total_offset_minutes = −330 (that is −5 hours, −30 minutes).
- If your code computed hours_offset = total_offset_minutes // 60 => −6 (floor division) and minutes_offset = total_offset_minutes % 60 => 30 (depending on language), or minutes_offset might be computed as total_offset_minutes - hours_offset*60 => −330 - (−5)*60 => −30.
- In the latter case minutes_offset == −30, so the check on line 396 fails and a ValueError is raised even though −5:30 is a valid offset.

How to fix it (specific steps and code suggestions)
Pick one of these correct approaches, depending on how you want sign handled.

Option A — keep hours signed and make minutes non-negative (recommended)
1. Compute absolute minutes for the remainder:
   - total = total_offset_minutes
   - sign = 1 if total >= 0 else -1
   - abs_total = abs(total)
   - hours = (abs_total // 60) * sign
   - minutes = abs_total % 60
2. Validate minutes as 0..59 and don’t expect a negative value.
3. Replace your lines that set minutes_offset and the invalid check with the above.

Concretely:
  total = total_offset_minutes
  sign = 1 if total >= 0 else -1
  abs_total = abs(total)
  hours_offset = (abs_total // 60) * sign
  minutes_offset = abs_total % 60
  if not (0 <= minutes_offset <= 59):
      raise ValueError(f"Minutes out of range: {minutes_offset}")

Option B — allow minutes_offset to be signed, but normalize before validating
1. If your code uses signed minutes_offset, normalize it so the sign is carried only on hours:
  if minutes_offset < 0:
      hours_offset -= 1
      minutes_offset += 60
2. Then validate that minutes_offset is in 0..59.

Example normalization code to put before the check at line 396:
  if minutes_offset < 0:
      hours_offset -= 1
      minutes_offset += 60
  if not (0 <= minutes_offset <= 59):
      raise ValueError(f"Minutes out of range: {minutes_offset}")
