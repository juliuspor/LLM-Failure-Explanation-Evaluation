- Later, at lines 402–404, the code reuses the same variable minutes_offset and performs arithmetic with hours_in_minutes:

Why it fails (root cause)
- minutes_offset is being used for two different meanings:
  - initially it is the minute component (0..59),
  - later it is overwritten by an attempted "total minutes" calculation that mixes hours_in_minutes and minutes_offset.
- When hours_in_minutes is negative (for a negative hour offset), the code at line 402 does hours_in_minutes - minutes_offset. That produces a large negative number (for example, hours_in_minutes = -60, minutes_offset = 30 gives -90), but because the code never made a clear signed combination step, the arithmetic is wrong for many sign combinations and can exceed allowed range.
- Reusing minutes_offset hides the intent and causes incorrect total_minutes to be computed, which then either produces an incorrect offset value or triggers the "Offset is too large" ValueError at line 407.
- In short: the function validates the minute component, then mutates that same variable incorrectly when combining with hours, so the final offset calculation is wrong.
