What happened (where it fails)
- At line 176 you call int(x) inside the conditional:  
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- Python evaluates the left side of an and expression first, so int(x) is executed before the math.isinf and math.isnan checks.
- If x is not a normal finite scalar (for example x is float('nan'), float('inf'), or some non-scalar like a numpy array), calling int(x) raises an exception (ValueError for NaN, OverflowError for Inf, or TypeError for an array). That exception is why the code fails at line 176.

Why this is the root cause (no guessing)
- The code intended to avoid converting NaN/Inf to int by checking math.isinf(x) and math.isnan(x), but those checks come after int(x) in the same expression, so they never protect the conversion.
- The variable names/methods involved: x (the value), int(x) (the conversion that fails), math.isinf(x) and math.isnan(x) (the intended guards). The failure happens when int(x) is invoked before the guards run.

Concrete fixes (explicit step-by-step)
1. Check finiteness/NaN before calling int:
   - Replace the line with:
     if not math.isinf(x) and not math.isnan(x) and x == int(x):
   - Or more simply use math.isfinite:
     if math.isfinite(x) and x == int(x):
   - These ensure int(x) runs only when x is finite and not NaN.

2. If x can be non-scalar (e.g., numpy array), check type/shape first:
   - Example:
     import numbers
     if isinstance(x, numbers.Real) and math.isfinite(x) and x == int(x):
         ...
   - Or for numpy arrays use numpy.isfinite / vectorized checks instead of int().

3. If you want to be defensive and catch unexpected values:
   - Wrap the conversion in try/except and handle errors explicitly:
     try:
         xi = int(x)
     except (TypeError, ValueError, OverflowError):
         # handle non-convertible x
     else:
         if x == xi:
             ...

Pick one of the above fixes depending on whether x may be NaN/Inf or a non-scalar. The simplest correct change is to use math.isfinite(x) before int(x): that prevents the exception and preserves the intended logic.
