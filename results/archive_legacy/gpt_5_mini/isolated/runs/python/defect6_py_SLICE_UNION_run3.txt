What happens
- At line 178 you set mantissa = value (an int copy of x).
- At line 181 you enter while mantissa % 10 == 0: …
- Inside that while you never change mantissa (or exp). Because mantissa is never updated, if mantissa % 10 == 0 is true once it stays true forever and the loop never terminates. That makes the code hang / never return.

Where it occurs
- The infinite loop risk is in the block starting at line 180 and the while at line 181. The variables involved are:
  - value (set at line 177)
  - mantissa (set at line 178 and tested at line 181)
  - exp (read at line 182 and later at line 190)
  - check_val (computed at line 182)

Why that causes the failure
- check_val uses mantissa // 10 but does not assign that back to mantissa. So every iteration computes the same check_val and the while condition never changes; the loop never finishes when mantissa is divisible by 10.

Concrete fixes (step-by-step)
1. Decide the intended change to mantissa and exp inside the loop. Typical intent appears to be stripping trailing zeros from mantissa and increasing exp accordingly.
2. Update mantissa and exp inside the loop. For example:
   - Inside the while at line 181 do:
     mantissa = mantissa // 10
     exp += 1
   - Then recompute check_val (line 182) using the updated mantissa/exp.
3. Add a safety guard to avoid pathological infinite loops (optional but recommended), e.g. limit loop iterations to log10(value) or check mantissa > 0 before dividing.
4. Ensure exp is initialized to the correct starting value before the while (the code reads exp at line 182 but your snippet does not show its initialization).
