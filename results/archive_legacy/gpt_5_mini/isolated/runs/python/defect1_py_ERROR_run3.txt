Common root causes and specific fixes (step-by-step)
A. Negative minute value came from arithmetic (most common)
- Cause: You computed minutes by subtraction and didn’t normalize the result into 0–59.
- Example failing code:
  minute = base_minute - offset  # could be negative
  dt = datetime.time(hour, minute)
- Fix (normalize): adjust minutes and hours before constructing time.
  Steps:
  1. Compute a full-minute integer: total_minutes = hour * 60 + minute_value
  2. If using offsets, do total_minutes += offset_minutes (offset_minutes may be negative)
  3. Normalize: new_hour = (total_minutes // 60) % 24; new_minute = total_minutes % 60
  4. Use datetime.time(new_hour, new_minute)
- Example replacement code:
  total = hour * 60 + minute_value
  total += offset_minutes
  new_hour = (total // 60) % 24
  new_minute = total % 60
  dt = datetime.time(new_hour, new_minute)

- Cause: parsing "-00:15" or "-15" as a minute value directly instead of as a signed offset.
- Fix:

- Fix:

  2. Normalize the minute before passing it to the library, or use library methods that accept timedeltas or offsets instead of raw minute integers.
