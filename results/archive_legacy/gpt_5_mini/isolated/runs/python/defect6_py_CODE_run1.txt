What fails
- The code raises NameError: name 'math' is not defined at this line:
    if x == int(x) and not math.isinf(x) and not math.isnan(x):
  because you call math.isinf and math.isnan but never imported the math module.

Why that happens (root cause)
- Python looks up the name math at runtime and can't find it in the function or global scope, so it raises NameError immediately when that line runs.
- This is unrelated to the rest of the numeric logic (mantissa/exp loop) — the function never reaches those lines if math is undefined.

Where it occurs
- In add_number, on the condition that checks x == int(x) and then calls math.isinf and math.isnan. Exact failing expression: not math.isinf(x) and not math.isnan(x).
