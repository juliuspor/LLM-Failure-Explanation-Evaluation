What happened
- The tests call StringUtils.replace_each("aba", ["a"], [None]) expecting the original string to be returned unchanged.
- The implementation of replace_each uses the replacement value directly (for example by calling text.replace(search, replacement) or doing text concatenation with replacement) without checking whether replacement is None.
- When Python tries to use None as the replacement in a string operation you get a TypeError, for example:
  - str.replace(search, replacement) raises TypeError: replace() argument 2 must be str, not NoneType
  - or doing some_text + replacement raises TypeError: can only concatenate str (not "NoneType") to str

Why this is the root cause
- The test cases mirror the Java StringUtils behavior: a None replacement should be treated as “no replacement for that entry” (i.e. skip it), not as an empty string or cause an exception.
- The code fails because it assumes every replacement entry is a string and never checks for None before using it in a string API call. That is where the error originates inside replace_each.

Where exactly to look
- Open hit09_stringutils.py and find the replace_each function (the method name you call from the tests).
- Inside that function find the loop or code that iterates over the search list and replacement list, e.g. something like:
    for search, repl in zip(search_list, replacement_list):
        text = text.replace(search, repl)
  or similar code that uses repl unconditionally.
- The exception occurs on the line that attempts to call a string method or do string concatenation using repl when repl is None.

How to fix it (concrete steps)
1. Edit hit09_stringutils.py and locate def replace_each(...).
2. In the loop where each replacement is applied, add an explicit check for None and skip applying that replacement. For example, change code like:
    for search, repl in zip(search_list, replacement_list):
        text = text.replace(search, repl)
   to:
    for search, repl in zip(search_list, replacement_list):
        if repl is None:
            # skip this replacement; leave occurrences of 'search' unchanged
            continue
        text = text.replace(search, repl)
3. Save and run the tests. They should no longer raise TypeError and should implement the intended behavior (original string unchanged if the replacement is None for that search term).
4. Optional: add a unit-test asserting that passing None in replacements doesn't change other replacements and that mismatched lengths or None search items are handled as desired.

Notes and alternatives
- Don’t convert None to '' unless you intend None to mean “replace with empty string”. The tests expect skipping the replacement (no change).
- If your implementation performs multiple passes or handles overlapping searches specially, make sure your None check is applied in the correct place (before any use of the replacement value).
- If replace_each supports recursive/iterative replacements, ensure the None-skipping still prevents TypeError during all iterations.
