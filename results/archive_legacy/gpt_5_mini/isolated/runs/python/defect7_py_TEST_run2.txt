What happened (root cause)
- The test fails because ClassUtils.to_class tries to call a Java-style method on each element instead of using Python's type system. Specifically, the implementation is calling something like obj.getClass() on each element (or otherwise assumes the element provides a getClass() method).
- When the element is None (the Python None), that call raises AttributeError: 'NoneType' object has no attribute 'getClass' and the conversion aborts.
- You can see the problematic scenario in this test: test_array = ["Test", None, 99.0]. The failure occurs while processing the None element inside ClassUtils.to_class.

Where it happens
- In hit07_classutils.py, inside ClassUtils.to_class(...) you iterate the input array and access a per-element attribute/method such as obj.getClass(). The AttributeError is raised as soon as the loop reaches the None element.
- The test that demonstrates the failure is test_to_class_with_none_element in your test file: result = ClassUtils.to_class(test_array) triggers the AttributeError for the None element.

Why it happens (concrete explanation)
- In Java you call object.getClass() or use ClassUtils.toClass equivalents, and a Java null would be mapped to null Class. The original Java bug described null handling too.
- In Python:
  - None is an object of type NoneType and does not have a getClass() method — calling obj.getClass() on obj == None raises AttributeError.
  - The correct Python approach is to use type(obj) (or obj.__class__) to get the class. For None you can either return type(None) or keep None (depending on the API contract). Your tests expect type(None) for None elements.

Exact places and variables to change
- File: hit07_classutils.py
- Method: ClassUtils.to_class(array)
- Problem line (example): classes[i] = array[i].getClass()  # or similar call to getClass
- Replace with logic that checks for None and uses type(...) otherwise.

Concrete fix (copy-paste ready)
- Replace the per-element code with something like:

  if array is None:
      return None
  classes = [None] * len(array)
  for i, obj in enumerate(array):
      if obj is None:
          classes[i] = type(None)       # or None if API should preserve null
      else:
          classes[i] = type(obj)       # use type(obj) or obj.__class__

- If you prefer a one-liner comprehension:

  if array is None:
      return None
  return [type(x) if x is not None else type(None) for x in array]

Why this fixes it
- type(obj) works for all Python objects, including builtins and user types.
- The explicit None check prevents calling a method on None and avoids AttributeError.
- The return value then matches your test expectations: [str, type(None), float] for ["Test", None, 99.0].

Extra note
- Also ensure ClassUtils.to_class(None) returns None (your test expects that). The suggested code handles that by checking array is None first.
