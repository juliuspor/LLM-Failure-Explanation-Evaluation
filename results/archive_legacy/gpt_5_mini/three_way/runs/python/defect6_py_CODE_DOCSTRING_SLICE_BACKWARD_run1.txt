What happens
- At line 176 you call int(x) as part of the condition:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- Python evaluates the operands left-to-right, so int(x) runs before the isinf/isnan checks.
- If x is math.inf or math.nan (or other non-finite values), int(x) raises an exception:
  - int(float('inf')) raises OverflowError ("cannot convert float infinity to integer")
  - int(float('nan')) raises ValueError ("cannot convert float NaN to integer")
- That exception is the failure you see. The intent was to avoid int(x) when x is non-finite, but the check order defeats that intent.

Where it occurs
- The problem happens at the exact expression int(x) in the if at line 176 (the left-hand evaluation of the and chain).

Why it fails (short)
- int(x) is evaluated before math.isinf(x) and math.isnan(x), and int() can't handle NaN/Infinity, so an exception is raised.
