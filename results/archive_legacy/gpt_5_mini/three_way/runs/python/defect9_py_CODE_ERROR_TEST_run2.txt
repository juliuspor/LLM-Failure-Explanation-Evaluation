What happened
- The error TypeError: object of type 'NoneType' has no len() comes from the loop that computes increase:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      ...
- If either replacement_list[i] or search_list[i] is None, calling len(...) raises TypeError. The test that fails passes replacement_list = ["c", None], so replacement_list[1] is None and the code crashes in that loop.

Where it occurs in the code
- The problematic loop is the one immediately after the comment:
    # get a good guess on the size of the result buffer so it doesnt have to double if it goes over a bit
  (the loop that begins with "for i in range(search_length):" and is labeled "BUG" in the source).
- Variables involved: search_length, search_list[i], replacement_list[i], increase.

Why it was overlooked
- Earlier in the function there is null-safety logic when searching for matches that skips entries where search_list[i] is None or empty or replacement_list[i] is None. But this resize-estimation loop did not repeat those checks, so it tried to take len() on values that could be None.

How to fix it (concrete steps)
1. In the "increase" loop, skip entries where search_list[i] is None or empty, or where replacement_list[i] is None — exactly the same condition used elsewhere in the function. For example:
    for i in range(search_length):
        if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater
2. Alternatively, guard each len() call with a conditional or use a helper function to test validity (but the previous approach keeps behavior consistent).
3. Run the existing tests (including test_null_safety_inside_arrays_mixed) to verify the error is gone.
