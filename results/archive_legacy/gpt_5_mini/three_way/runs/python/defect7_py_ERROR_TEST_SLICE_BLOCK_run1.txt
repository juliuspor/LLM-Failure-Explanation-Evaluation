What happened (root cause)
- Where: in ClassUtils.to_class, inside the loop at lines 765–767.

  - line 766: _ = array[i].upper()
  - line 767: classes.append(type(array[i]))
- Why it fails: the code always calls array[i].upper() without checking the element type. When array[i] is None, None has no attribute upper, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'. That is exactly what your failing test (test_to_class_with_none_element) triggers because the input contains None.

Why this is a bug (concise)
- The method is supposed to accept arbitrary element types (including None). Calling .upper() assumes the element is a string. That assumption breaks on None (and would also break on other non-string objects that do not implement upper()).
