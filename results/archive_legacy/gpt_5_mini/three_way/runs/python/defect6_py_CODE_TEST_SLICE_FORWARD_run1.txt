What happened (root cause)
- At line 177 you do: value = int(x). That converts -0.0 to integer 0 and loses the negative sign.
- The integer branch is entered because the condition at line 176 (x == int(x) and not math.isinf(x) and not math.isnan(x)) is True for -0.0 (because -0.0 == 0).
- Later, when you output the integer form (line 192: self.add(str(value))), you therefore append "0" (positive zero) instead of "-0.0". That's why test_negative_zero fails: the sign of negative zero was discarded before any special handling could run.

Where it occurs in the code
- Decision to treat x as an integer happens at the if that starts the integer-handling block (the branch that includes lines 177–192 in your slice).
- The conversion that destroys the sign happens at line 177: value = int(x).
- The output that prints the wrong thing is at line 192: self.add(str(value)).
- The earlier code that checks the previous character and inserts a space (the prev = self.get_last_char() / if x < 0 and prev == '-': self.add(" ")) does not help here because the negative sign was already lost by int(x).

Why the test fails (simple summary)
- -0.0 compares equal to 0, so your "is integer" branch runs and you convert to int, losing the sign. The code then prints "0" instead of "-0.0".

Concrete fixes (step-by-step, exact changes)
You must detect negative zero before you coerce to int. Two options; both are explicit and simple.

Option A — use an is_negative_zero check early (recommended)
1. Add a check right after you compute prev (or immediately after entering the method), before the integer branch:
   if self.is_negative_zero(x):
       self.add("-0.0")
       return
2. Keep the rest of the function unchanged.

Option B — prevent integer branch for negative zero
1. Change the integer-branch condition so it excludes negative zero:
   if x == int(x) and not math.isinf(x) and not math.isnan(x) and not self.is_negative_zero(x):
       ...
2. Then later you must still ensure negative zero will be printed somewhere else (e.g., by falling through to the final else that does self.add(str(x)), but str(-0.0) yields "-0.0" so that will work).

Example change (Option A) — exact placement
- Insert at top of method (immediately after prev = self.get_last_char()):
    if self.is_negative_zero(x):
        self.add("-0.0")
        return

Why these fixes work
- They prevent converting -0.0 to int(0) and thus preserve the sign.
- They are minimal: they only gate the integer-special-case logic so negative zero is handled explicitly.

Notes
- Do not rely on x == int(x) to decide behavior for zeros with sign — signed zero is still equal to zero numerically.
- The helper CodeConsumer.is_negative_zero (tested in your suite) is exactly the right tool to detect negative zero cleanly.
