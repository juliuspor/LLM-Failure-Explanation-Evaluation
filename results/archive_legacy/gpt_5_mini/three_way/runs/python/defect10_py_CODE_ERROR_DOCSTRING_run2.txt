What happened
- Input like "l" or "L" reaches the branch where last_char is in ("l", "L").
- The code sets numeric = val[:-1]. For "l" or "L", numeric becomes the empty string "".
- The code then calls char_at(numeric, 0). That attempts to index the empty string and raises IndexError: "String index out of range: 0".
- So instead of raising the intended ValueError for an invalid number, the function throws an IndexError.

Where it occurs in the code
- The faulty lines are in this block:
  - last_char = val[-1]
  - numeric = val[:-1]
  - if last_char in ("l", "L"):
      if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
          return int(numeric)
- The call char_at(numeric, 0) triggers the IndexError when numeric == "".

Why it fails (root cause)
- There is no guard that numeric is non-empty before accessing numeric[0] via char_at or numeric[1:].
- For single-character inputs that are qualifiers only ("l" or "L"), numeric is empty and the code assumes it has at least one character. That assumption is false and causes the IndexError.

Concrete fixes (step-by-step)
1. Explicitly handle the single-character qualifier case before accessing numeric[0]:
   - If len(numeric) == 0: raise ValueError(f"{val} is not a valid number.")
2. Or change the conditional to check numeric is non-empty before indexing:
   - if (numeric and ((char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit())): ...
3. Either approach prevents char_at(numeric, 0) from being called on an empty string.

Example corrected code (minimal change)
Replace the current "l"/"L" branch with:

    if last_char in ("l", "L"):
        if len(numeric) == 0:
            raise ValueError(f"{val} is not a valid number.")
        if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
            return int(numeric)
        raise ValueError(f"{val} is not a valid number.")

This ensures inputs like "l" or "L" produce ValueError instead of IndexError.
