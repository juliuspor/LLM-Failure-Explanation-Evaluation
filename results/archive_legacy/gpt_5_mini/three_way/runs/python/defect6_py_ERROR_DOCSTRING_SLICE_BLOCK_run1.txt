What happened (where it failed)
- At line 176 you test if x == int(x). For x == -0.0 that comparison is True (because -0.0 == 0.0).
- At line 177 you do value = int(x). int(-0.0) is 0 — the negative zero sign is lost there.
- Later, because exp <= 2, you call self.add(str(value)) at line 192. That produces "0" (no negative sign), so the test expected "-0.0" but got "0".

Why that happens (root cause)
- The code treats -0.0 as an ordinary integer-valued float and converts it to the integer 0. Converting to int and then to str discards the negative-zero sign bit. The sign information for -0.0 is only present on the float; equality and int() lose it.

Exact location to fix
- The problematic logic starts at line 176 (the x == int(x) check) and the sign is lost by line 177 (value = int(x)), then shown at line 192 (self.add(str(value))).
