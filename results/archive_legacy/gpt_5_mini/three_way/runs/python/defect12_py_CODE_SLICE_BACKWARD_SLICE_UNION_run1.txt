What happened (root cause)
- The code can set end (the third argument passed to substring_java) to a value larger than the string length. substring_java checks its bounds and raises IndexError when end > len(s). That is exactly what happens at the check shown in slice_backward (lines 32–33): the end value is out of range.

Why that can occur in this function
- After clamping upper to the string length you do:
    if upper < lower:
        upper = lower
  (this is in your code right after the "upper == -1 or upper > len(text)" branch)
- If lower > len(text) then:
    - the earlier branch sets upper = len(text)
    - then upper < lower is true, so you set upper = lower — which is now greater than len(text)
- Later you call substring_java(text, 0, upper) (for example inside the if index == -1 branch). That call passes an end value > len(text), causing substring_java's bounds check to fail and raising IndexError (the error you see).

    - then substring_java(text, 0, upper) is called with end=5 > len(text)=3 → IndexError

Where it occurs
- The bad assignment occurs in these lines in your function:

        upper = len(text)
    if upper < lower:
        upper = lower
- The exception is raised inside substring_java when you later call substring_java(text, 0, upper) (see the slice_backward bounds check at lines 32–33).

How to fix it (step-by-step, explicit)
Option A — Clamp lower to len(text) before you compare to upper
1. After you ensure text is not None and get its length, add:
       if lower > len(text):
           lower = len(text)
2. Keep the existing clamping of upper to len(text). Then the "if upper < lower: upper = lower" will not produce upper > len(text).

Option B — Make the update to upper respect the string length
1. Replace the pair of statements with a single safe clamp, for example:
       if upper == -1 or upper > len(text):
           upper = len(text)
       if upper < lower:

   This ensures upper never exceeds len(text).

Option C — Reorder and clamp after both assignments
1. Set upper = max(upper, lower)
2. Then clamp: upper = min(upper, len(text))

           upper = len(text)

- Option A is simplest and most explicit: clamp lower to len(text) early. That makes the subsequent logic safe and keeps intent clear.
