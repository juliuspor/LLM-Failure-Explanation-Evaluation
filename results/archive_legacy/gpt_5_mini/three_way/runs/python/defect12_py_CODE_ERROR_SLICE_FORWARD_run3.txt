What happened
- The code lets upper become larger than text.length before calling substring_java(text, 0, upper).
- substring_java does s[start:end] which raises IndexError when end > len(s): "String index out of range: 15".

Where it happens in your code
- After this line: if upper == -1 or upper > len(text): upper = len(text)
- this line then runs: if upper < lower: upper = lower
- and later you call: result = substring_java(text, 0, upper)

Why that produces the error (concrete example)
- Let len(text) == 15, lower == 20, upper == -1.
  - The first normalization sets upper = len(text) → upper = 15.
  - The second normalization sees upper < lower (15 < 20) and sets upper = lower → upper = 20.
  - Then substring_java(text, 0, 20) tries to slice beyond the string end and raises IndexError.

Root cause
- The line "if upper < lower: upper = lower" can push upper past the string length when lower itself is > len(text). You clamp upper to len(text) first, but you then overwrite it with a value (lower) that may be out of range. There is no subsequent clamp before calling substring_java.

How to fix (concrete steps)
Option A — clamp lower before using it:
- Ensure lower is at most len(text) before you compare/assign to upper.
  - e.g. add: if lower > len(text): lower = len(text)
  - then the existing logic is safe because upper will never be set > len(text).

Option B — clamp upper after the upper<lower assignment:
- After "if upper < lower: upper = lower" add another clamp:
  - if upper > len(text): upper = len(text)

Option C — compute upper as min(max(lower, requested_upper), len(text)):
- Replace the two-step normalization with a single expression that keeps upper within [lower, len(text)] and never > len(text].

Exact minimal code change (Option B example):
    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower
    # Add this clamp:
    if upper > len(text):
        upper = len(text)

Why this resolves it
- After this change upper is never greater than len(text) when passed into substring_java(text, 0, upper), so s[start:end] cannot go out of range.

Summary
- The bug occurs when lower > len(text): the line "upper = lower" can set upper beyond the string length. Fix by clamping lower or re-clamping upper to len(text) before calling substring_java.
