The bug arises from advancing pos inside the for-loop and then calling code_point_at with out-of-range pos. Based on the diagnosis, consumed is number of codepoints consumed by translator and they say translators handle surrogate pairs, so consumed should represent codepoints. Safer fix is to iterate consumed times but check pos < length before each code_point_at call. Implement fix (2): for _ in range(consumed): if pos >= length: break; pos += char_count(code_point_at(code_units, pos))
