Exact root causes

2. There is no explicit check of value before the normalization on line 71, so the code only discovers the out-of-range condition after computing g.
3. The raised message ("Color parameter outside of expected range: Red Green Blue") is unhelpful because it does not show the offending value, or the lower/upper bounds, so it’s hard to debug what went wrong.
4. (Edge case) If self._upper_bound == self._lower_bound, the division on line 71 will divide by zero. You should guard against that too.
