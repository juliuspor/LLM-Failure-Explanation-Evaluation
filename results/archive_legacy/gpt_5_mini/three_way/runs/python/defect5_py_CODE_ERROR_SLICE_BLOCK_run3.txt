Why it happened (root cause)
- The code determines inferred_type incorrectly at the start (lines shown in your snippet):
  - Line: if array is not None: inferred_type = type(array)
    - type(array) returns the Python list class (e.g. <class 'list'>), not the element/component type.
    - If you intended to detect the list's element type, type(array) is the wrong thing to ask for.
  - The logic then falls through to _copy_list_grow1 using that inferred_type. Because inferred_type is not a real element/component type, _copy_list_grow1 produces a list whose component type ends up treated as a generic "object" array in later checks.
- A common failing scenario:
  - array is an empty list (array is not None but has no elements). type(array) is <class 'list'>, so you never learn the element type from the contents. The code ends up with inferred_type that does not reflect the intended element type and ultimately becomes treated as object.
  - Or element is None (so element can't supply a type), and expected_type is str — the code sets inferred_type = object in the final else, so the raise condition (object vs str) triggers.
- In short: you are inferring the element type from type(array) (the container type) and you treat None/empty cases as object. That leads to inferred_type == object while expected_type == str, so the explicit TypeError is triggered.
