What happened (root cause)
- In GrayPaintScale.get_paint the input value is transformed into a gray level by computing something like
  fraction = (value - lower) / (upper - lower)
  gray = int(fraction * 255)
  and then a color (r,g,b) is created from gray.
- That code never clamps value or the computed gray to the valid byte range 0–255.
- When value is < lower you get a negative fraction → gray < 0; when value > upper you get fraction > 1 → gray > 255.
- The attempted creation of the color object with an out-of-range component triggers the ValueError: "Color parameter outside of expected range: Red Green Blue".

Where it fails (what to look for in your file)
- Open hit02_grayscale.py and find the method GrayPaintScale.get_paint.
- Look for the lines that compute the normalized fraction from the input value and the line that turns that into an integer gray level (variable names will be similar to value, lower (or lower_bound), upper (or upper_bound), fraction, gray).
- The exception is raised at the point you form the color tuple or call the color constructor (the line that returns/creates (r, g, b) or calls some Color(...) API). That line receives r/g/b values outside 0–255 because you never clamped them.

Why the tests fail
- Tests pass negative input (-0.5) or >1.0 (1.5) expecting clamping to 0 or 255. Because get_paint does not clamp, a negative gray is produced and the color library rejects it with that ValueError.

Exact fixes (step-by-step)
1. Clamp the input value to the [lower, upper] interval before normalizing:
   - value_clamped = min(max(value, lower), upper)
2. Compute the fraction safely:
   - if upper == lower: decide a sensible default (e.g., fraction = 0.0 or 1.0 or 0.5); avoid division by zero.
   - else fraction = (value_clamped - lower) / (upper - lower)
3. Compute the integer gray and clamp it into 0–255:
   - gray = int(round(fraction * 255))   # or int(fraction * 255) if you want floor
   - gray = min(max(gray, 0), 255)
4. Return (gray, gray, gray) or create the Color object with these guaranteed-in-range integers.

Minimal example replacement for get_paint body
- (use your actual variable names and line numbers)
  value = min(max(value, self.lower), self.upper)
  if self.upper == self.lower:
      fraction = 0.0
  else:
      fraction = (value - self.lower) / (self.upper - self.lower)
  gray = int(round(fraction * 255))
  gray = min(max(gray, 0), 255)
  return (gray, gray, gray)

Additional notes
- Use round() if you want 0.5 -> 128; the test expects 0.5 -> 127 so you may prefer int(fraction * 255) to reproduce the current expected mapping.
- Also add an explicit earlier check to avoid division by zero (upper == lower).
- Add unit tests for lower==upper and for exact boundaries to prevent regressions.
