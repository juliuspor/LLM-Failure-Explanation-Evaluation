What happened (root cause)
- On line 177 you do: value = int(x). For x == -0.0, int(-0.0) returns 0 and loses the sign bit. That makes value == 0 (not -0.0).
- On line 192 you call self.add(str(value)). Because value is 0, that adds "0" into the output.
- The original float x still holds -0.0, so if you later do self.add(str(x)) (line 194) you would get "-0.0". But you add str(value) earlier, so the test sees "0" where it expected "-0.0".

Where it occurs (lines)
- Line 177: value = int(x) — converts -0.0 to 0 and drops the negative sign.
- Line 192: self.add(str(value)) — formats and emits the unsigned integer 0.
- Line 194: self.add(str(x)) — this would produce "-0.0" but is executed after the incorrect 0 was already added.

Why this matters
- IEEE 754 distinguishes -0.0 from +0.0; tests expect the negative sign to be preserved. Casting to int removes that distinction.

Concrete fixes (pick one)
1) Preserve sign for zero explicitly
   - Before casting, detect negative zero:
     import math
     if x == 0.0 and math.copysign(1.0, x) < 0:
         # treat as negative zero; avoid int() path
         self.add(str(x))        # will be "-0.0"
         continue/return as appropriate
   - This prevents using value = int(x) for -0.0.

2) Avoid using int(x) for zero values
   - Change the logic so you only call value = int(x) when |x| >= 1 (or some other range where an integer conversion makes sense). For zero, use str(x).

3) Use the float string when sign matters
   - Replace the problematic emission at line 192 with:
       self.add(str(x))
     if the intention is to keep the original float formatting/sign. (Be careful: this may change other expected outputs.)

4) Preserve sign when converting
   - If you must derive an integer mantissa but still need sign info, carry the sign separately:
       sign = '-' if math.copysign(1.0, x) < 0 else ''
       value = int(abs(x))
       ...
       self.add(sign + str(value))

Which to choose
- If tests expect -0.0 specifically, implement fix (1) or (4) so negative zero is detected and emitted correctly.
- If the integer conversion is only used for formatting large numbers, adjust conditions so zero never goes through the int() path.
