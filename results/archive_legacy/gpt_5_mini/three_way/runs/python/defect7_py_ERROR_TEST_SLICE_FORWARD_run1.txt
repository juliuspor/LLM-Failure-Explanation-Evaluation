What happened (root cause)
- The test calls ClassUtils.to_class(test_array) with test_array = ["Test", None, 99.0].
- Inside hit07_classutils.py the code iterates the input array and, without checking for None, calls a string method on the element (array[i].upper()). Because the second element is None, Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The failure happens while converting elements to their classes; the code assumes array elements are strings (or otherwise non-None) and never guards against None.

Where it happens (file / variables / line)
- File: hit07_classutils.py
- Variables: array (the input list), i (loop index), classes (the list being built)
- Relevant slice shows classes.append(type(array[i])) at around line 767. The AttributeError comes from a nearby line that calls array[i].upper() (or otherwise calls .upper() on the element) before the code handles None.
- In short: the code uses array[i].upper() on array[i] even when array[i] is None.

Why that causes the observed error
- None has no attribute upper(), so array[i].upper() raises AttributeError.
- The test expects to_class to return [str, type(None), float] for ["Test", None, 99.0], but the unguarded .upper() call prevents reaching the logic that would return type(None).

How to fix it (concrete steps and exact change)
1. Open hit07_classutils.py and find the loop that iterates the input array (around line ~760).
2. Before any use of array[i] that assumes it is not None (especially any array[i].upper() call), add a None-check. For example:

Replace the problematic code (pseudocode)
    # problematic: calls methods on array[i] without checking None
    if some_condition_using(array[i].upper()):
        ...
    classes.append(type(array[i]))

With this safe version:
    if array[i] is None:
        classes.append(type(None))
        continue
    # safe to call string methods or other attributes now
    if isinstance(array[i], str):
        name = array[i].upper()   # only called for strings
        ...
    classes.append(type(array[i]))

3. If the code builds classes by inspecting element.__class__ or element type names, ensure every path handles None explicitly. Prefer using type(None) when the element is None.

Minimal concrete patch example
- If current code is something like:
    for i in range(len(array)):
        # ...some code that uses array[i].upper()...
        classes.append(type(array[i]))

- Change to:
    for i in range(len(array)):
        if array[i] is None:
            classes.append(type(None))
            continue
        # now safe to use array[i].upper() if necessary
        classes.append(type(array[i]))

Why this fix satisfies the tests
- It prevents calling .upper() on None, so no AttributeError will be raised.
- It returns type(None) for None elements, matching the tests that expect [str, type(None), float] for ["Test", None, 99.0].

Notes
- If you have code that tries to detect classes by looking at element.__class__.__name__.upper(), replace that with safe guards or use type(element) directly (with a None check first).
- Review other callers of to_class to ensure they also expect type(None) and not None itself.
