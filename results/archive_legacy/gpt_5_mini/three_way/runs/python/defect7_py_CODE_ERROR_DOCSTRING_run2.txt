What happened (where it failed)
- The exception is raised on this line inside the loop:
  _ = array[i].upper()
- The error "'NoneType' object has no attribute 'upper'" means array[i] was None and you tried to call .upper() on it.
- That line is executed for every element, so any None element triggers the AttributeError.

Why it happened (root cause)
- The code unconditionally calls array[i].upper() even though the docstring says "If any of these objects is None, a None element will be inserted into the array." The unconditional call assumes elements are strings (or otherwise have .upper()), but an element can be None.
- The code then uses type(array[i]) to build the result. If you wanted to insert None for None input elements you should append None, not type(None).
- In short: the guard for None elements is missing inside the loop; the offending call is array[i].upper().

Exactly where to change (line references)
- Remove or guard the call on line: _ = array[i].upper()
- Change the append behavior on line: classes.append(type(array[i])) to append None when array[i] is None.

Concrete fixes (pick one depending on intended behavior)

A — If you want to follow the docstring ("If any of these objects is None, a None element will be inserted"):
Replace the loop with:
for i in range(len(array)):
    if array[i] is None:
        classes.append(None)
    else:
        classes.append(type(array[i]))

B — If you intended to raise AttributeError explicitly when an element is None (docstring currently contradicts itself):
for i in range(len(array)):
    if array[i] is None:
        raise AttributeError(f"Element at index {i} is None")
    classes.append(type(array[i]))

C — If the .upper() call was for a different purpose (e.g., validating string elements), do that only for non-None strings:
for i in range(len(array)):
    val = array[i]
    if val is None:
        classes.append(None)
        continue
    if isinstance(val, str):
        _ = val.upper()   # safe now
    classes.append(type(val))

Why these fixes work
- They avoid calling .upper() on None by checking for None first.
- They append None rather than type(None) when the docstring says to insert None.
- They make the method behavior consistent and avoid the AttributeError.

One more minor note
- The branch for empty input returns ArrayUtils.EMPTY_CLASS_ARRAY.copy() — ensure that constant and .copy() exist and are appropriate in your environment.
