What happened (where it fails)
- The crash occurs inside replace_each, in the loop that starts at line 81. The exact offending statement is line 82:
  greater = len(replacement_list[i]) - len(search_list[i])
- If either replacement_list[i] or search_list[i] is None, calling len(None) raises TypeError and the function crashes.
- The code never checks for None before calling len(), so although the docstring says None replacements should be ignored, the implementation still assumes each entry is a string.

Why it happens (root cause)
- The code assumes every element in search_list and replacement_list is a string and uses len(...) without guarding against None.
- There is no skip/guard logic before computing lengths (lines 59 and 81 iterate over indices blindly), so None entries lead to len(None) and a TypeError.
- This is a deterministic bug at line 82 (and anywhere else len(...) is used on those lists) — the code tries to measure lengths of values that might be None.

Concrete fixes (step-by-step)
1. Add a guard before any use of len(...) on list elements. For the loop starting at line 81, change it to skip None entries:

   - Before:
     for i in range(search_length):
         greater = len(replacement_list[i]) - len(search_list[i])
         if greater > 0:
             increase += 3 * greater  # assume 3 matches

   - After:
     for i in range(search_length):
         if search_list[i] is None or replacement_list[i] is None:
             continue
         greater = len(replacement_list[i]) - len(search_list[i])
         if greater > 0:
             increase += 3 * greater

2. Alternatively, iterate over pairs instead of indices and explicitly skip None:

   for s, r in zip(search_list, replacement_list):
       if s is None or r is None:
           continue
       greater = len(r) - len(s)
       ...

   This removes index-based errors and clearly encodes the "ignore None" behavior.

3. Add unit tests to cover:

   - cases where search or replacement is the empty string

4. (Optional) If the intended behavior is to treat None as an instruction to leave that entry out, document it in the docstring and make sure all other places that access these lists apply the same skip logic (not just this loop).
