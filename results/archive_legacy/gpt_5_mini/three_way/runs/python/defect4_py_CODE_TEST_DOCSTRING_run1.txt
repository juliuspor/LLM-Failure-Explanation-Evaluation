What happened (root cause)
- In the block that updates self._max_middle_index you fetch the wrong existing period for comparison:
  - Current code (inside the "if self._max_middle_index >= 0:" branch):
    max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  - That uses self._min_middle_index instead of self._max_middle_index.
- Because you compare the new period's middle to the middle of the "min" period, the comparison logic is wrong and can set _max_middle_index to the wrong index.
- Concrete example from your test:
  - Existing middles: index 0=50, 1=150 (true max), 2=25 (min), 3=75.
  - When checking index 3 (middle=75) the code compares 75 against the min_middle (25) because of the bug, sees 75 > 25, and sets _max_middle_index = 3. That produces the incorrect result (3 instead of 1).

Where it occurs (exact location)
- In the provided function _update_bounds, the bug is in the block:
  if self._max_middle_index >= 0:
      max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  ... 
- Specifically the wrong variable is used: self._min_middle_index should be self._max_middle_index.

Why this causes the observed failure
- The code should compare the new period's middle to the current maximum middle. Using the minimum-middle index instead corrupts the max computation: any middle larger than the minimum (which is common) will incorrectly overwrite the true maximum index.

How to fix it (concrete steps)
1. Replace the wrong index variable with the correct one in that block.
2. Run the test suite.

One-line patch (exact change)
- Replace:
    max_middle_period = self.get_data_item(self._min_middle_index).get_period()
- With:
    max_middle_period = self.get_data_item(self._max_middle_index).get_period()

Why this fix works
- After the change the function will correctly fetch the current maximum-middle period for comparison, and will only update self._max_middle_index when the new period's middle is strictly greater than the current maximum middle (as intended).

Suggested additional action
- Run your test file (you already have tests that reproduce the bug). After applying the one-line change, the tests you included (especially test_max_middle_index_simple) should pass.
