What happened
- The exception comes from GrayPaintScale.get_paint producing an R/G/B value outside 0–255 and then passing it to whatever color constructor raises: "Color parameter outside of expected range: Red Green Blue".
- That happens because get_paint does not clamp the input value into the configured bounds before converting it to a gray level, so fraction and therefore the computed gray can be negative or >255.

Where it occurs (method and variables)
- In GrayPaintScale.get_paint:
  - value (the input to get_paint) is used to compute a fraction like
    fraction = (value - lower) / (upper - lower)
    (variable names may differ slightly in your file, but this is the calculation you do).
  - Then gray = int(fraction * 255.0) (or similar).
  - Finally you build the color (gray, gray, gray) or call a Color constructor with those channels.
- If value < lower then fraction < 0 → gray < 0; if value > upper then fraction > 1 → gray > 255. Passing those integers to the color constructor triggers the ValueError you saw.

Why it failed (root cause)
- No clamping of value or the computed gray to the valid range [0.0, 1.0] (for fraction) or [0, 255] (for channel integers).
- Edge case also: if upper == lower you will divide by zero when computing fraction.

How to fix it (concrete steps)
1. Clamp the input value into [lower, upper] right away:
   value = max(lower, min(upper, value))
2. Compute fraction safely:
   if upper == lower:
       fraction = 0.0   # or decide a sensible default or raise a clear ValueError
   else:
       fraction = (value - lower) / (upper - lower)
3. Ensure fraction is exactly within [0.0, 1.0] (defensive):
   fraction = max(0.0, min(1.0, fraction))
4. Compute the gray channel and make it a valid integer 0..255 (use rounding or int as intended, but clamp afterward):
   gray = int(round(fraction * 255.0))
   gray = max(0, min(255, gray))
5. Return the color as a triple of ints (gray, gray, gray) or pass those ints to the color constructor.

Minimal replacement for get_paint
- Replace the body of get_paint with code like:

    def get_paint(self, value):
        # clamp input into bounds
        value = max(self.lower, min(self.upper, value))
        # avoid zero-division
        if self.upper == self.lower:
            fraction = 0.0
        else:
            fraction = (value - self.lower) / (self.upper - self.lower)
        # ensure fraction in [0,1]
        fraction = max(0.0, min(1.0, fraction))
        gray = int(round(fraction * 255.0))
        gray = max(0, min(255, gray))
        return (gray, gray, gray)

Notes and verification
- This change will make get_paint(-0.5) clamp to lower (-> black) and get_paint(1.5) clamp to upper (-> white), fixing the failing tests.
- It also prevents the division-by-zero crash when lower == upper and prevents any R/G/B channel from falling outside 0–255, which is the direct cause of the ValueError.
