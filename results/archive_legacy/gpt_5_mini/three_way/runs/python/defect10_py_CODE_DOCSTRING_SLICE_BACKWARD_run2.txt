What happened
- Input "l" or "L" reaches the branch for a trailing qualifier because last_char is non-digit and in ("l","L").
- The code sets numeric = val[:-1]. For "l" that makes numeric == "" (empty string).
- The code then calls char_at(numeric, 0) (inside the conditional on the next line). Because numeric is empty, char_at(numeric, 0) raises IndexError ("String index out of range: 0") instead of the intended ValueError.
- The IndexError originates in the char_at implementation (see slice showing it raises IndexError when index < 0 or index >= len(s)).

Where it occurs in your function
- In the branch:
  if last_char in ("l", "L"):
      # BUG: missing guard for len(val) == 1 and non-digit input.
      if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
          return int(numeric)
      raise ValueError(f"{val} is not a valid number.")
- The problem is the call to char_at(numeric, 0) when numeric == "".

Why this is a bug (short)
- The code assumes numeric has at least one character before reading numeric[0] (via char_at). For inputs that are only the qualifier (e.g. "l"), numeric is empty and accessing index 0 throws IndexError. The function should instead treat "l"/"L" as an invalid number and raise ValueError.

Concrete fixes (pick one)
1) Add an explicit empty check before calling char_at:
   if last_char in ("l", "L"):
       if len(numeric) == 0:
           raise ValueError(f"{val} is not a valid number.")
       if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
           return int(numeric)
       raise ValueError(f"{val} is not a valid number.")

2) Replace the combined condition with checks that do not index into numeric unless it has content:
   if last_char in ("l", "L"):
       if numeric.isdigit():
           return int(numeric)
       if numeric.startswith("-") and numeric[1:].isdigit():
           return int(numeric)
       raise ValueError(f"{val} is not a valid number.")

3) Shorter guard at top of function:
   if len(val) == 1 and val.lower() == "l":
       raise ValueError(f"{val} is not a valid number.")
   (then keep existing logic)

Why these fixes work
- All ensure you never call char_at(numeric, 0) or access numeric[1] when numeric is empty, preventing IndexError and allowing the function to raise the intended ValueError for inputs like "l"/"L".

Recommendation
- Use fix (2) — it's clear, avoids manual indexing, and matches the existing tests' intent (accept only pure digits or a leading "-" + digits before converting).
