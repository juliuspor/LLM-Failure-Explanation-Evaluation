What happened (root cause)
- The exception comes from this line inside the for-loop:
  _ = array[i].upper()
- That line assumes array[i] has an upper() method (e.g. it's a str). In your failing run some element of array is None, so array[i] is None and None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The failure occurred at the loop iteration that hit the None element. The stack slice shows the next executed line was classes.append(type(array[i])) (line 767 in your slice), but the AttributeError was raised earlier on the .upper() call.

Why this is the real problem (concise)
- You're calling .upper() unconditionally on every element. If any element is None (or any non-string without upper()), the call fails.
- The subsequent classes.append(type(array[i])) indicates you only need the element's type; the .upper() call doesn't affect that result and is probably accidental or leftover.

Concrete fixes (pick one based on intended behaviour)

1) If you only want the element types (remove the accidental call)
- Remove the .upper() line entirely.
  for i in range(len(array)):
      classes.append(type(array[i]))

2) If you expected only strings and want to ignore/skip None values
- Skip None elements before calling .upper() or getting type:
  for i in range(len(array)):
      if array[i] is None:
          continue
      _ = array[i].upper()   # safe now if you know remaining elements are strings
      classes.append(type(array[i]))

3) If you want to treat None as a string-like value (e.g. treat None as "")
- Coerce to safe value before calling .upper():
  for i in range(len(array)):
      value = array[i] or ""
      _ = value.upper()
      classes.append(type(array[i]))

4) If you want to record types but guard against elements that lack upper()
- Check attribute before calling:
  for i in range(len(array)):
      val = array[i]
      if hasattr(val, "upper"):
          _ = val.upper()
      classes.append(type(val))
