The to_class function currently calls .upper() on array elements without checking for None, raising AttributeError. The intended behavior per docstring: If any element is None, a None element will be inserted into the array. Implement that behavior: return None for None input, return a copy of EMPTY_CLASS_ARRAY for empty input, iterate elements and append None if element is None else append type(element). Remove unnecessary .upper() call. Keep signature and exceptions consistent.
