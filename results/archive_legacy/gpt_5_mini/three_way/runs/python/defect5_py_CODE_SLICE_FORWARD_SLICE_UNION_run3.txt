What happened (root cause)
- At line 335 you set inferred_type = type(array). That returns the Python class of the list object (e.g. builtins.list), not the type of the list elements. The rest of the function and ArrayUtils._copy_list_grow1 expect a component/element type, not the list container type.
- Because inferred_type ends up being the list class, the call at line 342 (new_list = ArrayUtils._copy_list_grow1(array, inferred_type)) hands the wrong type to _copy_list_grow1. That causes a runtime type mismatch later (the code raises a ClassCastException-style error in the string at 346, and/or the underlying array-copy logic fails because it was given [Ljava.lang.Object; vs the intended element array type).
- The conditional at lines 346–347 (if inferred_type == object and expected_type != object) also becomes meaningless because inferred_type will rarely equal object — it will be list — so you miss the intended detection when you really had an “unknown element type”.

Where exactly it occurs
- Line 335: inferred_type = type(array) — wrong value (list class instead of element class).
- Line 342: call to ArrayUtils._copy_list_grow1(array, inferred_type) — receives incorrect type and produces a list/array with the wrong element-component type.
- Line 346: the check inferred_type == object will not detect the “unknown element type” case because inferred_type is the container class.

Why that produces the failure
- The copy/grow helper expects a component (element) type to create a new array/list of the right runtime component type. Passing the container class causes the created structure to have a wrong runtime component type, which leads to a ClassCastException or similar when elements are stored or consumed as the expected element type.

How to fix it (clear steps)
1. Use the element’s runtime type when the array already exists:
   - If array is not None and has at least one element, use type(array[0]) as the inferred element type.
   - If array is not None but is empty, fall back to type(element) when element is provided, otherwise to object.
2. Keep the existing fallback behavior when array is None: use type(element) if provided, otherwise object.
3. Update the check at line 346 to reflect the corrected inferred_type logic (it should be object only when you truly have no element type).

Concrete corrected logic outline:
- if array is not None:
    - if len(array) > 0: inferred_type = type(array[0])
    - else: inferred_type = type(element) if element is not None else object
- elif element is not None:
    - inferred_type = type(element)
- else:
    - inferred_type = object

Apply that and the call new_list = ArrayUtils._copy_list_grow1(array, inferred_type) will receive the correct component type and the subsequent assignment new_list[len(new_list)-1] = element will not fail from a component-type mismatch.
