What happened
- The test inputs "l" and "L" hit the branch that handles trailing long-qualifiers:
  - last_char = val[-1] is "l"/"L", so numeric = val[:-1] becomes the empty string "".
- The code then calls char_at(numeric, 0) without checking that numeric is non-empty.
- char_at("", 0) raises IndexError("String index out of range: 0") (see the slice showing char_at raises that on bad indexes), so the test fails with IndexError instead of the expected ValueError.

Where it occurs (exact variables / location)
- Inside the branch: if last_char in ("l", "L"): (the qualifier-handling branch).
- Variables involved: val (e.g. "L"), numeric = val[:-1] (becomes ""), then char_at(numeric, 0) is called.
- The exception comes from char_at when index 0 is out of range for an empty string (see char_at code raising IndexError at the check index < 0 or index >= len(s)).

Why it's wrong (root cause)
- The code assumes numeric has at least one character before accessing numeric[0] via char_at.
- For single-character inputs "l" or "L", numeric is empty and that assumption is false, so char_at throws an IndexError instead of the function raising the intended ValueError for invalid numeric strings.

Concrete fixes (exact steps you can apply)
1. Guard the empty numeric case before calling char_at:
   - Either immediately reject a lone qualifier:
     if len(numeric) == 0:
         raise ValueError(f"{val} is not a valid number.")
   - Then keep the rest of the logic unchanged.

2. Or avoid char_at and use safer string methods that don't throw on empty strings:
   - Replace the condition
     if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
   - With
     if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():
   - startswith handles empty strings safely and numeric[1:].isdigit() will be False when numeric is "-" or empty.

Either change will ensure "l" and "L" produce ValueError as the tests expect, and will prevent the IndexError coming from char_at.
