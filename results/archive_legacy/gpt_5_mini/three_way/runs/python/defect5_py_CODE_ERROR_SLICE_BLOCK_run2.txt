Why inferred_type became object (root cause)

  - elif element is not None: inferred_type = type(element)

- type(array) is the Python type of the list object (e.g. list), not the element type. So using type(array) gives the wrong value whenever array is a list: you get list, not the element class.
- In the failing test inferred_type was object because both array and element were None (or element was None and array absent), so the function had no element-type information and fell back to object at line 344.
- After that fallback, the code unconditionally treats an object-typed list as incompatible with a non-object expected_type and raises the TypeError at line 346 — even if the caller provided expected_type=str and intended to add a None placeholder for a str list. In short:
  - wrong source of type info for arrays (type(array) is wrong)
  - losing element type when element is None
  - then raising when inferred_type == object and expected_type != object
