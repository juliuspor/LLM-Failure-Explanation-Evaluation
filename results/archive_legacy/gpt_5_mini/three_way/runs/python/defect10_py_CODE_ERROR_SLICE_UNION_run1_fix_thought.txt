Added a guard to avoid calling char_at on an empty string by replacing the char_at logic with safe string methods. For 'l'/'L' qualifiers, the code now checks numeric.isdigit() or numeric.startswith('-') and numeric[1:].isdigit() before converting to int. All other behavior preserved: None input returns None, empty string raises ValueError with the same message, '--' prefix returns None, non-digit qualifiers produce ValueError, and digit-only strings (with optional leading '-') return int. This prevents IndexError for inputs like 'l' or 'L'.
