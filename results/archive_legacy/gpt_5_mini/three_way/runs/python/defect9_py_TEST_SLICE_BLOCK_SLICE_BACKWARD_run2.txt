What failed
- The tests pass None as one of the replacements (replacement_list contains None).
- At line 82 the code does: greater = len(replacement_list[i]) - len(search_list[i])
- Calling len(None) raises TypeError: object of type 'NoneType' has no len(), so the function crashes instead of treating None as “no replacement”.

Where it happens
- Method: replace_each (see header at line ~28 in your slice).
- Variable: replacement_list and search_list (used at lines 81–83).
- Exact failing statement: line 82 in your slice: greater = len(replacement_list[i]) - len(search_list[i]).

Why this is incorrect
- The tests expect null-safety: a None replacement means “do not replace that search element” (Java StringUtils behavior).
- The code assumes all search/replacement elements are strings and unconditionally calls len() on them. When replacement_list[i] or search_list[i] is None, len(None) raises an exception instead of allowing the function to skip that entry.

How to fix (concrete steps)
1. Skip pairs where either search_list[i] or replacement_list[i] is None. For example, in both loops over search_length (the loop at line 59 and the loop at line 81), add a guard:
   - if search_list[i] is None or replacement_list[i] is None:
       continue
   This preserves the “no change” behavior when replacement is None.

2. If you prefer to keep the arithmetic, compute lengths guarded against None. Replace line 82 with something explicit:
   - search_len = len(search_list[i]) if search_list[i] is not None else 0
   - repl_len   = len(replacement_list[i]) if replacement_list[i] is not None else 0
   - greater = repl_len - search_len
   But remember: treating a None search entry as length 0 probably also means you should skip that pair entirely, so the skip approach in step 1 is clearer/correct.

3. Make the same None-checks anywhere else in replace_each that assumes string methods or len() will work on elements of search_list or replacement_list (the initial search loop at line ~59 and the construction/replace loop later).

Example minimal patch (pseudo-Python; apply at both loops before using lengths or string ops):
   for i in range(search_length):
       if search_list[i] is None or replacement_list[i] is None:
           continue
       greater = len(replacement_list[i]) - len(search_list[i])
       if greater > 0:
           increase += 3 * greater

Result
- With the guard, len() is never called on None, and entries with None replacements are treated as “do not replace,” matching the tests:
  - replace_each("aba", ["a"], [None]) -> "aba"
  - replace_each("aba", ["a","b"], ["c", None]) -> "cbc"
