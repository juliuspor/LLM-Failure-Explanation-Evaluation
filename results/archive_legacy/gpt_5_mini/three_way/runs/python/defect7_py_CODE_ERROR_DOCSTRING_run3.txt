What happened (root cause)
- The failure occurs inside the for-loop, specifically at the line:
  _ = array[i].upper()
- When array[i] is None, calling .upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The docstring claims "If any of these objects is None, a None element will be inserted into the array," but the code never checks for None before calling .upper(), so it instead raises the AttributeError you saw.

Where it happens (exact spot)
- In the method to_class, inside the for-loop:
  for i in range(len(array)):
      _ = array[i].upper()   <- this line raises when array[i] is None
      classes.append(type(array[i]))

Why that line is wrong
- .upper() is a string method; it assumes array[i] is a str (or another object that implements upper).
- The function should accept arbitrary objects including None (per the docstring). Calling a string-only method unconditionally violates that assumption.

How to fix it (concrete steps and code)
1. Remove the unnecessary .upper() call unless you specifically need to ensure the element is a string. The current code does not use the result stored in _.
2. Explicitly check for None and append None to the result in that case; otherwise append the element's type.

Corrected implementation (replace the loop with this):
    classes = []
    for i in range(len(array)):
        if array[i] is None:
            classes.append(None)
        else:
            classes.append(type(array[i]))
    return classes

Alternate, more Pythonic version:
    classes = [None if x is None else type(x) for x in array]
    return classes

If you actually intended to verify that elements are strings and call .upper() for that reason:
- Check the type before calling .upper(), e.g.:
    for x in array:
        if x is None:
            classes.append(None)
        elif isinstance(x, str):
            _ = x.upper()   # only safe because we've checked x is str
            classes.append(type(x))
        else:
            classes.append(type(x))

Summary
- The AttributeError came from calling .upper() on a None element (array[i]). Fix by removing that call or adding a None/type check before calling .upper(), and append None to the output when input elements are None as your docstring requires.
