What happened (where): At line 176 you test "if x == int(x) ..." and for x == -0.0 that condition is True because -0.0 == 0.0. At line 177 you do value = int(x), which becomes the integer 0 (losing the negative-zero sign). At line 192 you call self.add(str(value)), which converts that integer 0 to the string "0". So the function prints "0" instead of preserving "-0.0". The root cause is that converting negative zero to an integer (int(-0.0) -> 0) and then using str(value) destroys the signed-zero information.

Why it happens (why int() loses sign): Python numeric equality treats -0.0 == 0.0 as True and int(-0.0) is 0; integers have no negative-zero concept. The code never checks for the signed-zero case before converting to int, so the sign is lost.

Exactly where to change it (concrete fixes):
- Detect negative zero before converting to int and handle it specially. Use math.copysign to detect sign of zero because comparisons like x < 0 are False for -0.0.

Two concrete fixes you can apply:

1) Keep current integer-path but skip it for negative zero:
- Replace the condition on line 176 with:
  if x == int(x) and not math.isinf(x) and not math.isnan(x) and not (x == 0.0 and math.copysign(1.0, x) < 0):

This prevents entering the integer branch for -0.0 so it falls through to the else and self.add(str(x)) will produce "-0.0".

2) Or explicitly detect negative zero and output it before converting:
- Add before line 177:
  if x == 0.0 and math.copysign(1.0, x) < 0:
      self.add("-0.0")
      return
Then the rest of the integer handling can proceed unchanged.

Either fix uses math.copysign(1.0, x) < 0 to reliably detect a negative zero (int(x) and x < 0 do not work for -0.0).
