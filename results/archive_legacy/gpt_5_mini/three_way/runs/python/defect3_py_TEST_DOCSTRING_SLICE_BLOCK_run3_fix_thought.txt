The translate method currently may call code_point_at with index beyond end after translators consume codepoints and the loop increments pos by char_count(code_point_at(code_units, pos)) inside a for loop leading to out-of-range. I'll rewrite translate to carefully manage pos, verify bounds before calling code_point_at, and handle surrogate pairs and malformed surrogates. The method will iterate while pos < length, call translate_codepoint, if consumed==0 will get code_point_at safely by checking UTF-16 pair, append appropriate code units and advance pos by 1 or 2. If consumed>0, advance pos by the number of code units consumed by iterating consumed times but ensuring pos < length before accessing. This prevents IndexError.
