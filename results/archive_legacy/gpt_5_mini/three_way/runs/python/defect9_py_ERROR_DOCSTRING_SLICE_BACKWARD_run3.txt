What happened (root cause)
- At line 82 the code does: greater = len(replacement_list[i]) - len(search_list[i]).
- If either replacement_list[i] or search_list[i] is None, calling len(...) raises TypeError: "object of type 'NoneType' has no len()".
- The docstring says None entries in search_list/replacement_list should be ignored, but here the code assumes every entry is a real string and never checks for None before calling len. That is the direct cause of the crash.

Where it happens (exact places)
- search_length = len(search_list) at line 52 — that only measures the list length, it does not validate elements.
- The failing expression is at line 82 inside the loop starting at line 81:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])  # line 82 — crashes if either element is None

Why the check was missing
- The code uses indexing (replacement_list[i], search_list[i]) without any preceding guard to skip None entries. There is no check like "if search_list[i] is None: continue" before computing lengths, so None values reach len(...).

How to fix it (concrete steps)
1. Skip None pairs before calling len:
   - At the start of the loop that begins on line 81, add an explicit check:
     if search_list[i] is None or replacement_list[i] is None:
         continue
   - This implements the "ignore None" behavior in the docstring.

2. (Optional but recommended) Handle lists of different lengths safely:
   - If replacement_list may be shorter than search_list, indexing replacement_list[i] can raise IndexError. Use zip or itertools.zip_longest to pair entries explicitly and avoid out-of-bounds indexing:
     - Replace for i in range(search_length): with:
         for s, r in zip_longest(search_list, replacement_list, fillvalue=None):
             if s is None or r is None:
                 continue
             greater = len(r) - len(s)
     - This also makes intent clearer: operate on pairs rather than by numeric index.

3. Add unit tests:
   - Test with search_list containing None, replacement_list containing None, and both lists containing valid strings to confirm None entries are ignored and no TypeError occurs.

Minimal code edit example (insert at line 81)
- Before computing lengths, add:
    if search_list[i] is None or replacement_list[i] is None:
        continue

Why this fix works
- It prevents calling len(None). It enforces the docstring behavior (ignore None replacements) at the exact point the code relies on element lengths. Using zip or zip_longest avoids indexing errors and makes the loop robust if list sizes mismatch.
