What happened
- The code lets upper be pushed above the string length when lower > len(text). That leads to substring_java(text, 0, upper) being called with end > len(s), which triggers the IndexError checked in your slice implementation (the check at slice_block lines 32–33).

Why it happens (step‑by‑step, with exact lines/vars)
1. You clamp upper only in this branch:
   if upper == -1 or upper > len(text):
       upper = len(text)
   (so after these lines upper ≤ len(text))
2. Immediately after you do:
   if upper < lower:
       upper = lower
   If lower > len(text), that sets upper = lower > len(text).
3. Later you call substring_java(text, 0, upper) (in two places). substring_java enforces end ≤ len(s) and raises IndexError when end > len(s) (slice_block lines 32–33). Because upper was set > len(text), substring_java gets an invalid end and the code fails.

Where it fails
- The failure occurs when lower > len(text) and execution reaches substring_java(text, 0, upper). The slice function raises IndexError "String index out of range: {end}" (see the check at slice_block lines 32–33).

How to fix it (concrete steps)
Option A — Clamp lower before you use it (recommended)
- After your early returns, add:

  Do this before any comparison or calls that use lower (index_of and the upper < lower check). That guarantees lower never exceeds len(text), so upper = lower will never push upper above len(text).

Option B — Make the upper adjustment safe
- Replace the problematic lines:
    if upper < lower:
        upper = lower
  with:
    if upper < lower:
        upper = min(lower, len(text))
  This ensures upper is never set > len(text).

Also ensure calls that use lower as a start for index_of are safe:
- If StringUtils.index_of(text, " ", lower) requires start ≤ len(text), either clamp lower (Option A) or check lower first and skip index_of if lower > len(text) (e.g. treat as not found).

Summary
Root cause: the assignment if upper < lower: upper = lower can set upper > len(text) when lower > len(text), and substring_java enforces end ≤ len(s) and throws. Fix by clamping lower (or limiting upper to len(text) when assigning it).
