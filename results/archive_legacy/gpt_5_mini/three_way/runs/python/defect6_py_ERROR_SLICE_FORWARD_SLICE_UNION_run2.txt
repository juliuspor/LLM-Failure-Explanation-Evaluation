What happened (root cause)
- At line 176 you test "if x == int(x)" to detect whole-number floats. For x == -0.0 this is True because -0.0 == 0.0.
- At line 177 you do "value = int(x)". int(-0.0) is 0 and this conversion discards the negative-zero sign.
- Later you call self.add(str(value)) at line 192, which adds "0" (no sign) to whatever output the test checks.
- The test expected the preserved negative zero representation "-0.0", but your code produced "0", so the assertion failed.

Why this happens (brief technical reason)
- In IEEE floats, -0.0 compares equal to +0.0, and converting to int yields 0 with no sign. Your branch assumes converting to int is safe for all equalities, but that discards the distinct -0.0 sign.
