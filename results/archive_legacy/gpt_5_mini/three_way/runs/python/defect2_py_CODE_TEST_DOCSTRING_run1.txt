What happened (where / why)
- In get_paint you correctly clamp the input into v:
  - Line ~3: v = max(value, self._lower_bound)
  - Line ~4: v = min(v, self._upper_bound)
- But on line ~6 you compute g using the original value variable instead of the clamped v:
  - g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- So when value is outside the bounds (for example value == -0.5 and _lower_bound == 0.0), v becomes 0.0 but g is computed from -0.5. That yields a negative g (int(-127.5) -> -127) which then triggers the ValueError check on line ~9.
- In short: you clamped into v but never used v for the interpolation. The code fails because an out-of-range input produces a negative RGB component before the method returns the clamped color.

- v (lines 3–4): clamped value — should be used for interpolation
- value (line ~6): still the original, un-clamped input — currently used incorrectly
- g (line ~6): calculated from the wrong variable and thus can be < 0 or > 255
- ValueError check (line ~9): correctly detects the bad g but is triggered only because of the wrong input to the formula

Minimal corrected function
- Change only the calculation to use v and add the zero-range guard. Example:

    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)

        # Avoid division by zero if bounds are equal
        if self._upper_bound == self._lower_bound:
            g = 0
        else:
            g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

        if g < 0 or g > 255:
            raise ValueError(f"Computed color component {g} outside 0-255 for value={value}, bounds=({self._lower_bound},{self._upper_bound})")

        return (g, g, g)

Why this will make your tests pass
- Now values below the lower bound will be clamped to self._lower_bound (v) before interpolation, so -0.5 will produce g == 0 and return (0,0,0).
- Values above the upper bound will be clamped to self._upper_bound and yield 255.
- Mid-range values are interpolated from the clamped v, matching your tests.
