What happened (where)
- At line 396 the code rejects any negative minutes:
  if minutes_offset < 0 or minutes_offset > 59:
      raise ValueError(f"Minutes out of range: {minutes_offset}")
  This enforces minutes in [0, 59]. The test expects minutes to be allowed in [-59, 59] (e.g. (0, -15) -> -00:15). So test_zero_hours_negative_minutes fails immediately with a ValueError raised at this check.

- At lines 402–404 the code incorrectly adjusts minutes when hours are negative:
  hours_in_minutes = hours_offset * 60
  if hours_in_minutes < 0:
      minutes_offset = hours_in_minutes - minutes_offset
  else:
      minutes_offset = hours_in_minutes + minutes_offset
  offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)

  Problems:
  1. minutes_offset is reused to hold the total minutes, which is confusing but not itself fatal.
  2. For negative hours and negative minutes (e.g. hours_offset = -2, minutes_offset = -15) the code computes:
     hours_in_minutes = -120
     minutes_offset = hours_in_minutes - minutes_offset  -> -120 - (-15) = -105
     That yields -01:45 instead of the expected -02:15. The subtraction of a negative minutes value cancels part of the hours instead of adding to the negative magnitude.
  3. For negative hours and positive minutes (e.g. -2, 15) the formula gives -120 - 15 = -135 (correct), but because negative minutes are allowed by tests, the previous wrong result for (-2, -15) will also fail tests.

Why it fails (root cause)
- The minutes validation is too strict: it disallows negative minutes (should allow -59..59).
- The negative-hours branch uses the wrong sign logic: when hours are negative you must subtract the absolute minutes (i.e. make the total more negative). The current code subtracts minutes_offset directly, which incorrectly flips sign when minutes_offset itself is negative.

Concrete places to change
1. Line ~396: change the minutes range check to allow negative minutes:
   if minutes_offset < -59 or minutes_offset > 59:
       raise ValueError(f"Minutes out of range: {minutes_offset}")

2. Lines ~402–404: compute total minutes explicitly and correctly, don't reuse minutes_offset:
   hours_in_minutes = hours_offset * 60
   if hours_offset < 0:
       total_minutes = hours_in_minutes - abs(minutes_offset)
   else:
       total_minutes = hours_in_minutes + abs(minutes_offset)
   offset = FieldUtils.safe_multiply(total_minutes, DateTimeConstants.MILLIS_PER_MINUTE)

Alternate simpler computation (handles signs consistently):
   total_minutes = hours_offset * 60 + ( -abs(minutes_offset) if hours_offset < 0 else abs(minutes_offset) )

Notes
- This preserves the intended behavior from the docstring where (-2, 15) and (-2, -15) both become -02:15, and allows (0, -15) -> -00:15.
- Also keep the hours bounds check (lines ~392) as-is (hours in [-23, 23]).
- Make sure to pass total_minutes (not the reused minutes_offset) to safe_multiply and then to for_offset_millis.

With those two fixes the calculations and validations will match the tests and the documented behavior.
