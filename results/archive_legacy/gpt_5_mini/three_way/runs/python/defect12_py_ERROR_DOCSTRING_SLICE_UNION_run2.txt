What happened (what failed)
- The code in slice_union raised an IndexError at lines 32–34 because it refused to accept start or end values beyond the string length:
  - if start < 0 or end < 0 or start > end or start > len(s) or end > len(s):
  - raise IndexError(f"String index out of range: {end}")

Why this is a bug (root cause)
1. The function enforces exact index bounds instead of following Python slice semantics:
   - In Python slicing, an end index greater than len(s) is allowed and is treated as len(s); start == len(s) is allowed (it yields an empty slice). The code rejects those valid slice combinations by requiring start <= len(s) and end <= len(s).
   - The higher-level function abbreviate expects to be able to call slice_union with lower > len(text) and get a safe result (usually empty or clamped), but slice_union raises instead. That is the immediate cause of the reported failure.

2. The error message is misleading:
   - The raise uses f"String index out of range: {end}" (line 34) even when start is the offending index. That can hide which index caused the failure.
