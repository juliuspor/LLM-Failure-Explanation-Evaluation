What happened (what failed and where)
- The AttributeError "'NoneType' object has no attribute 'upper'" is raised on line 766:
  _ = array[i].upper()
- That line runs inside the loop at lines 765–767:
  765 | for i in range(len(array)):
  766 |     _ = array[i].upper()
  767 |     classes.append(type(array[i]))
- When array[i] is None (for example the test passes ["Test", None, 99.0]), calling .upper() on None raises the AttributeError. The code never checks for None before calling .upper().

Why that causes the failing test
- The tests expect None elements to be handled and turned into type(None) in the output (see test_to_class_with_none_element and test_to_class_all_none). Instead, the code attempts to call a string method .upper() on every element unconditionally, which crashes as soon as an element is None.

Root cause
- An unnecessary and unsafe call to array[i].upper() was added. That call is not needed to determine an element’s class (type(array[i]) is sufficient) and it fails whenever an element is None.

How to fix it (concrete steps)
1. Remove the .upper() call. It serves no purpose for computing the element type.
2. Handle None explicitly so the result contains type(None) for None elements.

Minimal corrected implementation:

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    
        classes = []
        for i in range(len(array)):
            elem = array[i]
            if elem is None:
                classes.append(type(None))
            else:
                classes.append(type(elem))
        return classes

Why this fixes the tests
- No call is made on array elements, so None elements no longer cause AttributeError.
- None is explicitly converted to type(None) so the output matches expected lists such as [str, type(None), float].

Additional note
- If the original intent was to normalize strings (e.g., call .upper()), do that only after checking the element is a str:
      if isinstance(elem, str):
          _ = elem.upper()
  but for converting to classes this normalization is unnecessary.
