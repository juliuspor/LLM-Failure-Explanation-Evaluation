What fails
- The function raises ValueError at line 52: "if gap <= 0: raise ValueError('bound must be positive')".

- The code never validates the relationship between the start and end parameters before computing gap. If a caller passes end <= start (for example start == end, or start > end), gap becomes 0 or negative and the function immediately raises at line 52.
- The error occurs before the loop that picks characters (lines 56–58).
