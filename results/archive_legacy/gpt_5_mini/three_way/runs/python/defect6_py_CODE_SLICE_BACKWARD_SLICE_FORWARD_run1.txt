What goes wrong
- Where: inside add_number, lines 180–186 (the while block that strips trailing zeros from mantissa).
- What happened: the code can enter an infinite loop when value is 0 (so mantissa starts at 0).
- Why: the loop condition is while mantissa % 10 == 0. If mantissa is 0, mantissa % 10 == 0 is true, and mantissa //= 10 keeps mantissa at 0. check_val is computed as (0 // 10) * (10 ** (exp + 1)) == 0, so the if check_val == value is always true and exp increments forever. The code never breaks out of the while, so the function hangs.

Exactly where to look
- Line 177 sets value = int(x) and mantissa = value.
- Line 180 checks if abs(x) >= 100 before entering the loop.
- Lines 181–186 (the while and its body) are the loop that can never terminate when mantissa == 0.

How to reproduce quickly
- Call add_number with x == 0.0 (or any float that converts to int 0) and a previous character not blocking entry to this branch. Because abs(0.0) < 100 the branch won't be entered in the shown code, but if you ever remove or change that guard the infinite loop will appear; more generally the same issue appears whenever mantissa becomes 0 inside the loop.

Definite fixes (clear steps)
1. Prevent the loop when value is 0:
   - Right after value = int(x) add:
     if value == 0:
         self.add("0")
         return
   This ensures you never enter the trailing-zero stripping logic with mantissa == 0.

2. Or make the loop condition robust:
   - Change the while condition to require mantissa != 0:
     while mantissa != 0 and mantissa % 10 == 0:
         ...
   This avoids the mantissa==0 infinite-case while preserving normal stripping behavior for nonzero integers.

3. (Optional) Simplify the check inside the loop:
   - You don't need to recompute check_val using powers of ten to compare to value. Instead track the reconstructed value directly or compare mantissa * (10 ** exp) to value, which is clearer and less error-prone.

Pick one of (1) or (2). Either will stop the infinite loop by ensuring mantissa == 0 cannot drive an endless increment of exp.
