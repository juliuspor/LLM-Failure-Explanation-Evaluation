Why it happened (root cause)
- The immediate cause is that the caller computed an invalid index and passed it into this function (let’s call it char_at(s, index) for clarity).
- Typical places to look in NumberUtils.create_number:
  - Any loop that advances an integer position pointer over the string (commonly named index, pos, or i) and then calls this function without checking bounds.
  - Code that uses 1-based indexing logic (from a Java port) but then passes the same value into Python where string indices are 0-based.
  - Code that allows index to reach len(s) (one past the last valid index) and still calls char_at(s, index). For example:
    - using `while index <= len(s):` then calling char_at inside the loop
    - incrementing index and calling char_at before verifying it is < len(s)
