What happened (what failed)
- The failure occurs inside ClassUtils.to_class, in the loop shown at lines 765–767:
  - Line 766 calls array[i].upper() unconditionally.
  - If array[i] is None (or any non-string object like an int/float), calling .upper() raises AttributeError: "'NoneType' object has no attribute 'upper'".
- As a result, the test test_to_class_with_none_element (and other tests with None or non-string elements) fail because the method raises an exception instead of returning the list of types.

Why it happened (root cause)
- The code assumes every element in array is a string and tries to call .upper() for each element. That assumption is wrong:
  - None has no .upper() method (NoneType), so array[i].upper() raises AttributeError.
  - Many non-string objects (int, float, list, bool) also have no .upper() and would raise the same error.
- The only purpose of to_class should be to produce type(array[i]) (and map None to type(None)), but the stray call to .upper() is unnecessary and unsafe.

Where it occurred (specific references)
- Method: ClassUtils.to_class
- File slice lines: 765–767
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()          <-- problematic call
  - 767:     classes.append(type(array[i]))

Concrete fixes (step-by-step)
1. Remove the call to .upper() entirely (it serves no purpose for computing types).
   - Replace the loop with:
     for i in range(len(array)):
         if array[i] is None:
             classes.append(type(None))
         else:
             classes.append(type(array[i]))
2. Alternative more concise versions:
   - Using comprehension:
     classes = [type(x) if x is not None else type(None) for x in array]
   - Or simply classes = [type(x) for x in array] — this already yields type(None) for None, so the explicit check is unnecessary unless you want to emphasize type(None).
3. If the original intention was to check string contents before computing types (unlikely for to_class), guard the .upper() call with isinstance(array[i], str) and only call .upper() on strings:
   - if isinstance(array[i], str):
         _ = array[i].upper()

Why these fixes work
- Removing or guarding the .upper() call prevents AttributeError for None and non-string types. The function then completes and returns the list of types as the tests expect (e.g., [str, type(None), float]).

Minimal suggested patch (apply in ClassUtils.to_class)
- Replace lines 765–767 with:
  for i in range(len(array)):
      classes.append(type(array[i]))

(or, if you prefer explicit None handling)
  for i in range(len(array)):
      if array[i] is None:
          classes.append(type(None))
      else:
          classes.append(type(array[i]))

That will make the tests pass and avoid the AttributeError.
