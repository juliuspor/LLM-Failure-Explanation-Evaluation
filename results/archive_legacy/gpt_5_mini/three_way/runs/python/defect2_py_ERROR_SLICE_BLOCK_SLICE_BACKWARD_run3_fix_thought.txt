The bug occurs because get_paint computes g from the original value even after clamping and may produce g outside 0..255 when value is out of bounds. To fix this, clamp the input value to the configured bounds and use the clamped value to compute the grayscale component. Keep a defensive check that g is within 0..255 and raise a clear ValueError if not. This prevents the original failure and provides predictable behavior for out-of-range inputs.
