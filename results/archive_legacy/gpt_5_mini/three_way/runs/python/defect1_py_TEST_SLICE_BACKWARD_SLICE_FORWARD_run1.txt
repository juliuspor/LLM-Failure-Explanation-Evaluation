What failed
- The tests that pass negative minute values (for example test_zero_hours_negative_minutes calling DateTimeZone.for_offset_hours_minutes(0, -15), and tests using (-2, -15)) fail because the method immediately rejects negative minutes.
- The rejection happens at the minutes range check on line 396:
  if minutes_offset < 0 or minutes_offset > 59:
      raise ValueError(f"Minutes out of range: {minutes_offset}")
- That condition treats any negative minutes as out of range, so inputs like -15 cause a ValueError before any later logic (lines 402–405) runs. In short: negative minutes never get past the early guard, so the method cannot implement the intended behavior for negative-minute cases.

Why this is wrong (root cause)
- The minutes range should be allowed to be negative in the range -59..+59 (the tests and the file docstring expect negative minutes to be supported).
- The current check enforces minutes ∈ [0, 59] instead of minutes ∈ [-59, 59].
- There is later code (lines 402 and 404) that appears to try to combine hour and minute signs, but that code is unreachable for negative minutes because of the early check. So the method’s control flow prevents correct handling of negative minutes.

Where it occurs (exact references)
- Line 396: the minutes range check that rejects negative minutes.
- Lines 402 and 404: subsequent logic that manipulates minutes_offset to form a signed total—this code is what should run for some cases, but it never does for negative minutes because of the earlier rejection.

   with this:

       raise ValueError(f"Minutes out of range: {minutes_offset}")
   This allows minutes from -59 to +59.

2. Ensure sign-combining logic handles the mixed-sign rules your tests expect:
   - If hours > 0 and minutes < 0 => treat as invalid (raise ValueError).
   - If hours < 0 and minutes >= 0 => treat as negative offset with minutes increasing the magnitude.
   - If hours == 0 => minutes sign determines sign of offset.
   Implement that before computing final millis, for example:

   offset = FieldUtils.safe_multiply(minutes_total, DateTimeConstants.MILLIS_PER_MINUTE)
