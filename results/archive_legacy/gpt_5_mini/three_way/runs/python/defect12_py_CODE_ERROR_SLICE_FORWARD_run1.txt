What happened (root cause)
- The IndexError comes from substring_java returning s[start:end] when end is 15 but the string length is less than 15. The call sites that pass `end` are the three substring_java calls in your function.
- You clamp `upper` to the string length on these lines:
  - if upper == -1 or upper > len(text): upper = len(text)
  but later you do:
  - if upper < lower: upper = lower
  That second assignment can make `upper` larger than len(text`), if `lower` is larger than len(text). There is no check that `lower` is within the string length before you use it to set `upper` or to pass into StringUtils.index_of.
- Concretely:
  - If caller passes lower > len(text), then after `if upper < lower: upper = lower` you have upper > len(text).
  - Then you call substring_java(text, 0, upper) (e.g. in the `if index == -1` branch or when index > upper). substring_java tries to slice s[0:upper] where upper > len(s), which raises IndexError ("String index out of range: 15").

Where it occurs in your code
- The problematic sequence is:
  1. Line that sets upper = len(text) when upper > len(text) (first clamp)
  2. Line that sets upper = lower when upper < lower
  3. Any subsequent call to substring_java(text, 0, upper)
- Also you call StringUtils.index_of(text, " ", lower) with a start `lower` that may be beyond the string length; that may return -1 and send execution down the path that uses the now-out-of-range `upper`.

How to fix it (clear concrete steps)
Option A — clamp lower, keep existing logic:
1. After you handle the None/empty checks, add:
   - if lower > len(text): lower = len(text)
2. Leave the rest as-is. This prevents lower from ever being larger than the string, so upper = lower cannot exceed len(text).

Option B — ensure upper is clamped after comparing with lower (safe, minimal change):
1. Move or add a clamp after you possibly set upper = lower:
   - if upper == -1 or upper > len(text): upper = len(text)
   - if upper < lower: upper = lower
   - // add this:
     if upper > len(text):
         upper = len(text)
This guarantees you never pass an end > len(text) to substring_java.

Option C — explicit defensive checks before substring calls:
1. Right before each substring_java(text, 0, upper) call, ensure upper = min(upper, len(text)).
2. And ensure the start passed to index_of is <= len(text) (or handle the case where lower > len(text) earlier).

A recommended minimal patch (replace lines in your function):
- After the two existing clamping blocks, add:
    if lower > len(text):
        lower = len(text)

        upper = len(text)

Why these fixes work
- substring slicing requires 0 <= end <= len(s). Ensuring both lower and upper are within [0, len(text)] prevents substring_java from being invoked with an end past the string end, which is what triggered the IndexError.
