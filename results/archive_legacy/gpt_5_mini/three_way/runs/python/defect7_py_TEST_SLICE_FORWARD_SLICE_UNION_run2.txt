What happened (what fails)
- In ClassUtils.to_class the code iterates the input array and calls array[i].upper() on every element (see slice lines 765–767).  
- When an element is None this raises AttributeError: "'NoneType' object has no attribute 'upper'". That is the exception your tests hit for test_to_class_with_none_element and other tests involving None.

Where it occurs (exact lines & variables)
- The problematic code is in to_class's loop:
  - Line 765: for i in range(len(array)):
  - Line 766: _ = array[i].upper()      <-- this assumes array[i] is a non-None string
  - Line 767: classes.append(type(array[i]))
- Variable names: array, i, array[i], classes.

Why it fails (root cause)
- Calling .upper() assumes the element is a string and not None. For None (or other non-string types like int) .upper() does not exist, so attribute lookup fails immediately. The .upper() call is unnecessary for determining the element's class/type; the code only needs type(array[i]) (line 767) to build the classes list.

Exact consequences
- Any None element causes AttributeError and aborts to_class early. Tests expecting to_class([..., None, ...]) to return [..., type(None), ...] fail with an exception instead of returning a list of types.

Concrete fixes (exact steps you can apply)
1. Remove the unnecessary .upper() call. Replace the loop with a None-safe type extraction. Example patch you can apply:
   - Replace lines 765–767 with:
     for i in range(len(array)):
         elem = array[i]
         classes.append(type(elem))
   - Or more Pythonic:
         for elem in array:
             classes.append(type(elem))

2. Also ensure the top-level None input check remains (your tests expect ClassUtils.to_class(None) -> None). If not present, add:
     if array is None:
         return None

3. If you want to explicitly use type(None) for None elements (identical to type(None) from type(None) above), no extra handling is required because type(None) is returned by type(None).

Why this fix is correct
- type(elem) works for all possible elem values (including None and builtins) and does not require any string-specific methods. The .upper() call is both unnecessary and dangerous for non-string values.

Summary
- Root cause: array[i].upper() (line 766) assumes string and causes AttributeError for None.  
- Fix: remove the .upper() call and use type(elem) (with early return if array is None). This will satisfy all your tests (None elements become type(None), mixed types handled correctly).
