What went wrong (root cause)
- At line 335 you do inferred_type = type(array). That returns the type of the container (for a Python list that is <class 'list'>), not the element/component type you actually need.
- ArrayUtils._copy_list_grow1 (called on line 339) expects the element/component type (the type used to create the new list/array), but you pass the container type. Passing the container class causes the copy/resize operation to build a list/array with the wrong component type, so subsequent assignments (new_list[len(new_list) - 1] = element) or later casting checks behave incorrectly and lead to the error you see.
- The wrong inferred_type also breaks the runtime check at lines 346–351. That check tests inferred_type == object to decide whether to raise a TypeError for an unsafe cast. Because inferred_type was set to <class 'list'> instead of the element type (or object), the logic doesn’t reflect the real situation and may either skip a needed error or raise the wrong one.

Where it occurs
- Line 335: inferred_type is set from the whole array instead of from an element.
- Line 339: new_list = ArrayUtils._copy_list_grow1(array, inferred_type) is called with the wrong inferred_type.
- Line 346: the later cast-check uses that wrong inferred_type.

Why that produces the failure
- The code must know the element/component type to allocate/resize a typed array or to validate an expected_type. type(array) gives the wrong type; using it leads to a mis-sized/mis-typed result from _copy_list_grow1 and incorrect cast checks. The symptom is a failure when assigning element into new_list or a misleading/incorrect TypeError.

Concrete fixes (step-by-step)
1. If expected_type is provided, prefer it as the component type:
   - if expected_type is not None: inferred_type = expected_type
2. Else, if array is not None and non-empty, use the element type of an existing element:
   - if array is not None and len(array) > 0: inferred_type = type(array[0])
3. Else, if element is not None use its type:
   - elif element is not None: inferred_type = type(element)
4. Else fall back to object:
   - else: inferred_type = object

Example patch (replace lines 335–340):
    if expected_type is not None:
        inferred_type = expected_type
    elif array is not None and len(array) > 0:
        inferred_type = type(array[0])
    elif element is not None:
        inferred_type = type(element)
    else:
        inferred_type = object

Notes and caveats
- If array can contain heterogeneous element types this inference may still be imperfect; consider validating that all elements are instances of inferred_type, or require expected_type to be passed by the caller.
- If array may be empty and expected_type is None, you must decide whether to default to object or require expected_type.
- Ensure ArrayUtils._copy_list_grow1 expects the element type (not the container type) — this change assumes that contract.
