Replace unsafe .upper() call with proper handling of None and non-string objects. According to original semantics, method converts an array of objects to array of class objects; original raised trying to call .upper() possibly to ensure stringness - but unnecessary. Implement: if array is None return None; if empty return copy of EMPTY_CLASS_ARRAY; iterate elements: if element is None append None, else append type(element). That preserves behavior and avoids AttributeError. Use ArrayUtils.EMPTY_CLASS_ARRAY.copy() for empty case.
