  ```

      greater = len(replacement_list[i]) - len(search_list[i])
  ```

- Earlier in the function you explicitly skip None or empty elements when searching for matches (see the condition used in the "Find the first match" loop and the later "Find the next earliest match" loop):

- But the size-estimate loop (lines 81–84) does not perform the same checks; it assumes every element is a non-None string and calls len(...) on them. If the caller passed None for some elements (which the rest of the function is written to tolerate), this loop crashes because len(None) is invalid.

- Crash occurs at line 82 (the len(...) calls). The exception message pinpoints the None argument.
- This happens even if the None element would never be used for replacement; the size-estimate loop inspects all list elements unguarded.
