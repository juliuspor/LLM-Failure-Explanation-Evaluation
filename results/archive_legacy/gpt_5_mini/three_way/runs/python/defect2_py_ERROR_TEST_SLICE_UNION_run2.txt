What happened (where it fails)
- The error comes from get_paint (lines shown in the slice): at line 71 you compute
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- Immediately after (lines 74–75) you check:
  if g < 0 or g > 255:
      raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- The unit tests deliberately pass values outside the configured bounds (e.g. value = -0.5 or 1.5). For those inputs the computed g is < 0 or > 255, so the code raises the ValueError instead of clamping to the expected black (0) or white (255). The test failure message exactly matches that exception.

Why it happens (root cause)
- The class is supposed to clamp out-of-range inputs to the nearest endpoint (black/white). Instead of clamping, the code raises an exception when the computed grayscale channel g falls outside [0,255].
- The problematic variables/methods are: value (input), self._lower_bound and self._upper_bound (used in the formula), and g (the computed channel). The check at lines 74–75 causes the exception when clamping was intended.

How to fix it (concrete steps)
1. Replace the raise with clamping logic. After computing g, limit it to [0,255] and return the triple (g, g, g).
2. Also guard against division by zero if upper and lower bounds are equal.

Minimal change (illustrative patch)
- Replace the check+raise at lines 74–75 with clamping and return:

    # compute g (line 71)
    g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

    # clamp g into [0, 255] instead of raising
    if g < 0:
        g = 0
    elif g > 255:
        g = 255

    return (g, g, g)

- And add a small guard before the computation to avoid division by zero:

    if self._upper_bound == self._lower_bound:
        return (0, 0, 0)  # or choose an appropriate constant color for equal bounds

Why this fixes the tests
- Negative inputs will produce g < 0, get clamped to 0, and return (0,0,0) as the tests expect.
- Inputs > upper bound will produce g > 255, get clamped to 255, and return (255,255,255).
- Values inside the range will be converted and truncated as intended (e.g. 127.5 -> int -> 127).
