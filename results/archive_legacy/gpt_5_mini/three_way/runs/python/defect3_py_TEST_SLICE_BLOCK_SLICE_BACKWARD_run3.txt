What happened (where it fails)
- The IndexError is raised from to_utf16_code_units at these lines:
  - Line 35-36: the guard `if index < 0 or index >= len(code_units): raise IndexError(...)`.
- The exception is triggered when CsvEscaper.translate_string calls code_point_at (which uses the list produced by to_utf16_code_units) while iterating a string that contains a UTF‑16 surrogate pair (an emoji) followed by another character (the test input "\U0001F630A").
- In that scenario the code advances the loop index incorrectly after handling a surrogate pair, so it ends up calling code_point_at with index == len(code_units) (or otherwise out of range), which fails the check at line 35 and raises IndexError.

Why it fails (root cause)
- The code is working with UTF‑16 code units (the list named code_units returned by to_utf16_code_units) and also treating the surrogate pair as two separate 16‑bit units.
- When a high surrogate is found, the translator is supposed to consume both the high and low surrogate (advance the index by 2 code units). Instead the translator advances by the wrong amount (usually only by 1) or otherwise mismanages the index that walks code_units.
- As a result, on a string like "\U0001F630A" (which is two UTF‑16 code units for the emoji plus one for 'A' → total 3 code units), the loop index can be set to a value equal to the length of code_units (or beyond) and then code_point_at is called with that out‑of‑range index.
- The immediate symptom is the check at lines 35–36 failing: the function defensively raises IndexError because index >= len(code_units).

Where exactly in the code to look
- to_utf16_code_units: builds the list code_units and contains the index bounds check at lines 35–36.
- code_point_at: reads code_units at a given index and needs to verify the presence of a following low surrogate when a high surrogate is seen.
- CsvEscaper.translate_string (the translator loop): is the place that advances the index through code_units; this logic is mishandling the increment when surrogate pairs are encountered.
