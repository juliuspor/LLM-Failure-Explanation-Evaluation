What happened (where it fails)
- The IndexError comes from substring_java(s, start, end) raising when end > len(s) (see the slice code at lines 32–34).
- Calls that pass end are:
  - substring_java(text, 0, upper) (several places)
  - substring_java(text, 0, index)
- The bug is in the ordering of the bounds logic around these lines of your function:

    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower

If the caller passes a lower value larger than len(text), then the second if (upper < lower) sets upper = lower, making upper > len(text). That new value is never clamped back down, so later substring_java(text, 0, upper) is called with end > len(text) and the slice code raises IndexError: "String index out of range: {end}".

Why it happens (root cause)
- You clamp upper to len(text) first, but then you overwrite upper with lower without re-checking whether lower is within the string length.
- The call site that triggers the exception is substring_java(text, 0, upper) where upper now can be > len(text). The slice function explicitly rejects end > len(s) (line 32), causing the error.

Concrete steps to fix (exact changes you can make)
1. Validate or clamp lower before you set upper from it. For example, immediately ensure lower is at most len(text):
   - if lower > len(text): lower = len(text)
   - (or return text / None depending on desired behaviour)

2. Or, after you set upper = lower, clamp upper again to len(text). Example options:

Option A (clamp lower):
- Insert after the early len/text checks:
    if lower > len(text):
        lower = len(text)

Option B (clamp upper after assignment):
- Replace your two lines with:
    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)

Either option prevents end being > len(text) when you call substring_java(..., 0, upper).

3. Additionally, you may want to validate negative lower (if callers can pass negatives): check if lower < 0 and handle or raise.

Minimal suggested patch (one safe change):
- Add a clamp for lower right after you have text length:

    if lower > len(text):
        lower = len(text)

This keeps the rest of your logic unchanged and prevents upper being set > len(text).

Where to change (line references)
- Insert the new lower clamp immediately after the block that returns for empty/null text and before you adjust upper. That prevents the path that sets upper = lower from producing an invalid end index.

This fixes the direct cause: substring_java(...) being called with end > len(text), which the slice function rejects.
