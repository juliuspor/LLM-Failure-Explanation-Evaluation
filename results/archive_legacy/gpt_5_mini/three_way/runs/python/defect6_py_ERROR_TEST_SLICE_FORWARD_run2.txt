What happened (root cause)
- At line 177 the code does value = int(x). For x == -0.0 this produces value == 0 and loses the negative sign.
- Later, the code writes str(value) (line 192) — which becomes "0" — and that is what gets emitted.
- In other words, the code converts -0.0 into an integer 0 and then prints that integer form, so the negative sign of the negative zero is dropped and you get "0" instead of "-0.0".

Where it occurs (specific lines)
- Conversion to int happens on line 177: value = int(x)
- That integer value is used to produce output on line 192: self.add(str(value))
- There is also a fallback write on line 194: self.add(str(x)), but that may not be reached in the path that chose the integer branch.
- The incorrect printing therefore originates in the integer-branch logic around those lines.

Why that loses the sign
- In IEEE floats, -0.0 compares equal to 0.0; int(-0.0) == 0, so using int(x) to test/format numbers removes the sign information.
- Detecting "is integer" by int(x) and then printing str(value) throws away the sign bit for negative zero.

Concrete fixes (step-by-step, choose one)
1) Preferred: detect negative zero before the integer/formatting branch
   - Add an explicit test immediately before line 177 (before computing value = int(x)):
     - if CodeConsumer.is_negative_zero(x):
           self.add("-0.0")
           return
   - Why: you keep the existing integer-path logic untouched but ensure -0.0 is handled specially so its sign is preserved.

2) Alternative: detect negative zero using math.copysign
   - Import math and add before line 177:
       if x == 0.0 and math.copysign(1.0, x) < 0.0:
           self.add("-0.0")
           return

3) Alternative (less preferred): avoid converting to int for formatting or use float-aware formatting that preserves sign of zero
   - Instead of value = int(x) and self.add(str(value)) for integers, format using a float formatter that will preserve sign for -0.0 (but this is trickier and may change other outputs).
   - Example: if you want to print integer-looking floats but preserve sign, you could do:
       if x.is_integer():
           if CodeConsumer.is_negative_zero(x):
               self.add("-0.0"); return
           else:
               self.add(str(int(x))); return

Exactly where to change (replace or add)

- Reference lines:

Why this will fix the failing test
- The test expects "-0.0". By checking is_negative_zero(x) and emitting "-0.0" early, you avoid the int conversion that strips the sign. Subsequent tests for 0.0, integers, and floats remain unchanged.
