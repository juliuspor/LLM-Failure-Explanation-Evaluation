What happened (root cause)
- At line 335 you set inferred_type = type(array). That takes the Python type of the container (e.g. list), not the element type. So inferred_type becomes list (the class) whenever array is non-None, not the element class you actually care about.
- When both array and element are None (the test case that fails), your code falls through to inferred_type = object at line 340. You then call _copy_list_grow1(..., inferred_type) and later check at line 346:
  if inferred_type == object and expected_type != object:
    raise TypeError(...)
  Because inferred_type is object and expected_type is str, that condition is true and you raise TypeError. The test expects ArrayUtils.add(None, None, expected_type=str) to succeed (return [None]) — not to throw.

Why that produces the observed failure
- The logic never uses the caller-provided expected_type when both array and element are None. That means you treat a null/unknown element list as object and then refuse to "cast" it to the caller's expected type (line 346), causing an unnecessary TypeError.
- Separately, using type(array) (line 335) is wrong for inferring element type. If array is non-empty, you should inspect the array's elements (e.g. array[0] or better: determine a common element type), not the container's type (list).

Exactly where to change (concrete steps)
1. Prefer expected_type first when it is provided and meaningful:
   - If expected_type is not None, use it as the inferred element type when you have no better information (especially when array and element are both None).
   - This fixes the failing test where expected_type=str and both array and element are None: inferred_type should be str (or at least accept str as the intended element type), not object.

2. When array is not None, infer element type from the array contents, not from type(array):
   - If array has at least one element: inferred_type = type(array[0])
   - If array is empty, fall back to expected_type (if provided) or to element (if non-None) or to object only as last resort.

3. When element is not None and no array info is available, use type(element).
