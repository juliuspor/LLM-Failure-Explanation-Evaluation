What happened (where): In to_class, the loop at lines 765–767 causes the test failures:

- Line 766: _ = array[i].upper()
- Line 767: classes.append(type(array[i]))

Why it failed (root cause):

- The call to array[i].upper() assumes every element is a string and has an upper() method.
- When array contains None (or any non-string, e.g. an int, float, list), array[i].upper() raises AttributeError (NoneType has no attribute 'upper') or similar, so the method crashes before appending the element's type.
- The test that fails is test_to_class_with_none_element which passes ["Test", None, 99.0]; the None at index 1 triggers the AttributeError on line 766.

Extra note about behavior: The .upper() call is unnecessary for determining the element's class. The method's goal is to return a list of types; calling upper() has no effect on that and only introduces crashes for non-string elements.

What to change (concrete steps):

1. Remove the unnecessary call to .upper() on line 766.
2. Iterate over elements directly (for element in array) or keep the index but avoid calling string methods.
3. Append type(element) for each element. For None, type(None) is returned automatically, which matches the test expectations.

Minimal corrected implementation:

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for element in array:
            classes.append(type(element))
        return classes

Other remarks:
- This change preserves the behavior that None maps to type(None).
- If ArrayUtils.EMPTY_CLASS_ARRAY.copy() does not return a plain list in your environment, ensure it returns [] for the empty-array case (the tests expect []).
