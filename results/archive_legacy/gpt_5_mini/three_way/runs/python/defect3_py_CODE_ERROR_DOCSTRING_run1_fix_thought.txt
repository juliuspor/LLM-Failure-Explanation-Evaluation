translate_codepoint implementations in this module return the number of UTF-16 code units consumed (e.g., CsvEscaper returns 2 for a surrogate pair, 1 for a single unit). The original loop advanced pos by repeatedly calling code_point_at(...) which could read past the end when pos reached length during iteration. The fix is to advance pos by the consumed number of code units directly. Also keep existing input/out validation and conversion to UTF-16 code units and the fallback when consumed==0. This avoids out-of-range calls to code_point_at.
