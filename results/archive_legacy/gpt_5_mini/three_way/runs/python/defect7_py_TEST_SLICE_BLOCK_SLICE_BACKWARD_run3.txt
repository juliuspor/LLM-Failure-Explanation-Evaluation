What happened (where / what line)
- In ClassUtils.to_class the code iterates the input variable named array and calls array[i].upper() on each element at line 766:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))
- When an element is None, array[i] is None and None.upper() raises AttributeError: "'NoneType' object has no attribute 'upper'".
- The failing test uses test_array = ["Test", None, 99.0], so the loop hits the None element and the AttributeError occurs before type(array[i]) is appended.

Why it fails (root cause)
- The call to .upper() is unnecessary for producing the class/type. It assumes every element is a string (has .upper), which is incorrect:
  - None has no .upper -> AttributeError.
  - Non-string objects (e.g. numbers) may not have .upper either.
- The code should only inspect the element’s type; it must not call a string method on every element.

Concrete fixes (step-by-step)
1. Remove the .upper() call and rely on type() directly:
   - Replace the loop body (lines 765–767) with:
     for i in range(len(array)):
         classes.append(type(array[i]))
2. Handle None input up front:
   - At start of to_class, if array is None: return None (this is already tested).
3. Optionally make it more pythonic and robust:
   - Use a list comprehension and explicit None handling:
       if array is None:
           return None
       return [type(x) for x in array]
   - This returns type(None) for None elements (matches expected tests).
4. If there was an intended special-case behavior for strings (e.g. normalizing by uppercase), move that logic so it only runs when isinstance(element, str), for example:
       if isinstance(x, str):
           normalized = x.upper()  # only if needed
       classes.append(type(x))

What to check after change
- Run the provided unit tests:
  - test_to_class_with_none_element should now succeed (returns [str, type(None), float]).
  - test_to_class_null_input should still return None.
  - All other tests (empty list, mixed types, all None, nested lists) should pass.

Summary
- The immediate bug is the unconditional call to array[i].upper() (line 766). It causes AttributeError on None or non-string elements. Remove the .upper() call (or guard it with isinstance(..., str)) and use type(array[i]) to fix the failure.
