What went wrong (what fails)
- The code assumes that when length > 2 and locale_str[2] == '_' the two characters at positions 3 and 4 (ch3 = locale_str[3], ch4 = locale_str[4]) are always an uppercase country code and validates them as such.
- For an input like "fr__POSIX" the string has a double underscore meaning "empty country, variant follows". In that case locale_str[3] is '_' (not an uppercase letter), so the code raises ValueError at the country-case check:
  - The problematic check is these lines:
    ch3 = locale_str[3]
    ch4 = locale_str[4]
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(f"Invalid locale format: {locale_str}")
- That is exactly why test_locale_with_empty_country_variant fails: the code treats '_' at position 3 as an invalid country letter instead of recognizing the "language__variant" form where country == "".

Where it happens (line/variable references)
- After verifying locale_str[2] == '_' (the separator) the code immediately reads ch3 and ch4 and enforces uppercase:
  - variable names: ch3 = locale_str[3], ch4 = locale_str[4]
  - these lines cause the ValueError for inputs like "fr__POSIX".
- The root cause is missing handling for the double-underscore case (empty country) before validating ch3/ch4.

Clear steps to fix (exact changes to make)
1. After confirming locale_str[2] == '_' (the separator), check whether the next character is also an underscore. If it is, treat the country as empty and set the variant to the remainder after the second underscore.
2. Only when the second character after the separator is not '_' should you validate ch3 and ch4 as uppercase country letters.

Concrete code change (replace the current country/variant block with this logic)
- Pseudocode you can paste in place of the current block that reads ch3/ch4 and validates them:

    # after checking locale_str[2] == '_'
    if locale_str[3] == '_':
        # empty country, variant starts at index 4
        if length == 3:
            # string "xx_" with nothing after second underscore -> invalid length
            raise ValueError(...)
        return Locale(locale_str[0:2], "", locale_str[4:])
    else:
        # validate country letters at positions 3 and 4
        ch3 = locale_str[3]
        ch4 = locale_str[4]
        if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
            raise ValueError(...)
        if length == 5:
            return Locale(locale_str[0:2], locale_str[3:5])
        else:
            if locale_str[5] != '_':
                raise ValueError(...)
            return Locale(locale_str[0:2], locale_str[3:5], locale_str[6:])

Notes and edge cases
- This preserves the original validations (language must be lowercase, separator must be '_', country uppercase) but adds the missing branch for language__variant.
- Be careful with indexing/length checks when variant is empty or when the string ends too early; keep the earlier length sanity checks or add explicit checks (e.g. ensure length >= 4 when you look at locale_str[3], and >=5 when accessing locale_str[4], etc.).
- After this change LocaleUtils.to_locale("fr__POSIX") will return Locale("fr", "", "POSIX") as expected.
