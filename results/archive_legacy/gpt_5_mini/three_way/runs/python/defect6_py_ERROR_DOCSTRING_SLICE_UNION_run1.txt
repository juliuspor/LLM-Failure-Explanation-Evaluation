What happened (where)
- The code at line 176 checks if x == int(x), then at line 177 does value = int(x). This is true for -0.0 because -0.0 == 0.0 and int(-0.0) == 0.
- Converting -0.0 to int loses the negative sign. Later the code uses value (and adds str(value) at line 192) so the negative-zero sign is lost and you output "0" instead of "-0.0".
- The failing test shows exactly that: it expected -0.0 but got 0.

Why it fails (root cause)
- Python’s int() discards the sign of a signed zero: int(-0.0) == 0. The code’s branch assumes integer conversion is safe to represent the original value, but that assumption fails for signed zeros. The sign information for zero is only in the floating value x, not in value after int().
- The condition if x == int(x) (line 176) therefore incorrectly treats -0.0 as an integer value and proceeds to a path that generates "0".
