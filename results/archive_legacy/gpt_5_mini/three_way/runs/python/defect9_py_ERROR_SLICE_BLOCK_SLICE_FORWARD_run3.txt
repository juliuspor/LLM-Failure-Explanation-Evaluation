What happened (root cause)
- At line 82 you call len(replacement_list[i]) (and len(search_list[i])) before you ever check whether those items are None.
- If replacement_list[i] (or search_list[i]) is None, Python raises TypeError: object of type 'NoneType' has no len() at line 82.
- The later guard that tests replacement_list[i] is None (lines 102–108) runs only in the matching loop, so it’s too late — the code has already tried to take len() of a None and crashed earlier.

Where it occurs in the snippet
- Crash point: line 82: greater = len(replacement_list[i]) - len(search_list[i])
- The check for None exists later at lines 102–108 (the condition that includes "or replacement_list[i] is None"), but that check does not protect the earlier len() call at line 82.

Why this is the bug (clear sequence)
1. You iterate i in range(search_length) and immediately use len(replacement_list[i]) (line 82).
2. If replacement_list[i] is None, len(None) throws TypeError.
3. The code intended to skip None entries later, but performing len() first guarantees the crash for None elements.

Concrete fixes (explicit change steps)
Option A — Check for None before using len (recommended)
- Change the first loop (lines 81–84) to skip None entries explicitly:
  - Before computing greater, add:
    if replacement_list[i] is None or search_list[i] is None:
        continue
  - Then compute greater = len(replacement_list[i]) - len(search_list[i])

Option B — Normalize inputs up front
- Before any loops, replace any None entries with empty strings (or raise if None is invalid):
  for idx in range(len(replacement_list)):
      if replacement_list[idx] is None:
          replacement_list[idx] = ""
  for idx in range(len(search_list)):
      if search_list[idx] is None:
          search_list[idx] = ""

Option C — Safe length call
- Use a helper to get length safely:
  def safe_len(x): return 0 if x is None else len(x)
  greater = safe_len(replacement_list[i]) - safe_len(search_list[i])

Which to pick
- If None means “no replacement” / should be skipped, use Option A.
- If None should be treated as empty string, use Option B or C.
- If None is an input error, raise a clear ValueError early instead of letting len() fail.
