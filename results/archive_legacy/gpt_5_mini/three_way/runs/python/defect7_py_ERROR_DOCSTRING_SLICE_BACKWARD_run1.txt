What happened (where it fails)
- At line 766 the code calls array[i].upper(). The runtime error is AttributeError: 'NoneType' object has no attribute 'upper'. That means array[i] was None for some index i, so Python tried to call .upper() on None and failed.

Why it happened (root cause)
- The loop (lines 765–766) assumes every element in array is a string-like object that provides .upper(). It never checks for None before calling .upper(), so a None element causes the AttributeError.
- The docstring is inconsistent and misleading: it says "If any of these objects is None, a None element will be inserted into the array" but also claims the method "Raises: AttributeError: If any element is None." The code behaviour (calling .upper() unguarded) matches neither of those intended behaviors clearly.

Exactly where to change (what to edit)
- At the start of the function: confirm the None-array behavior. If you intend to return None for a None input, keep the early return:
  if array is None:
      return None
  (Do this before the loop.)
- In the loop that iterates array (lines 765–766), replace the unguarded call to .upper() with an explicit check for None. For example:
  for i in range(len(array)):
      if array[i] is None:
          result[i] = None            # or continue, or raise a controlled error
      else:
          result[i] = array[i].upper()

What to choose (concrete fix options)
- Insert None elements into the output (matches the first sentence of your docstring):
  - Implement the loop check shown above and return an array where output[i] is None when input[i] is None.
- Or raise a clear error (if you want to forbid None elements):
  - At the top of the loop:
      if array[i] is None:
          raise ValueError(f"array[{i}] is None; expected string-like object")
    This is better than letting a raw AttributeError bubble up because it gives a clear message and an index.
- Also fix the docstring so it matches the chosen behavior: either state “None inputs produce None outputs” or state “None elements raise ValueError”, but don’t claim both.

Minimal patch to avoid the current crash (replace lines 765–766):
  if array is None:
      return None
  result = [None] * len(array)
  for i in range(len(array)):
      if array[i] is None:
          result[i] = None
      else:
          result[i] = array[i].upper()

This change prevents calling .upper() on None, removes the AttributeError, and implements the "insert None element" behavior described earlier in the docstring. If you prefer to forbid None elements, replace the inner if with a raise as shown.
