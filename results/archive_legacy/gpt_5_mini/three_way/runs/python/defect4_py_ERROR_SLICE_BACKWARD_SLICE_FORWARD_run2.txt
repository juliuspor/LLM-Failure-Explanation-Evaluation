What happened (root cause)
- At lines 340–349 the code tries to update self._min_start_index when it finds a data item with an earlier start time than the current minimum.
- The comparison block is correct up to line 347: it sets self._min_start_index = index only when the current item's start (variable start) is less than the recorded min_start.
- But line 349 then unconditionally assigns self._min_start_index = index, so every time this code runs it overwrites the min index with the current index regardless of the comparison result.
- Because of that unconditional assignment (line 349), _min_start_index always ends up equal to the last index seen by this code path instead of the index of the earliest start. That explains why your test expected max_middle_index = 1 but got 3: the recorded minimum-start index was overwritten and became 3.

Where this is in the code
- Problem region: lines 340–349.
  - Line 340: if self._min_start_index >= 0:
  - Lines 341–347: compute min_start and do if start < min_start: self._min_start_index = index
  - Line 349: self._min_start_index = index  <-- unconditional overwrite (the bug)

What to change (concrete fixes)
Option A — keep the conditional behavior (most likely intended):
- Remove line 349 entirely, or change it to be the else branch of the earlier if:
  - Replace
    if self._min_start_index >= 0:
        ...
        if start < min_start:
            self._min_start_index = index
    self._min_start_index = index
  - With
    if self._min_start_index >= 0:
        ...
        if start < min_start:
            self._min_start_index = index
    else:
        # first time setting min start
        self._min_start_index = index

Option B — if the intent was always to set it on first run only:
- Use the else branch as shown above so the unconditional assignment runs only when _min_start_index < 0 (first item).

Tests and verification steps

Summary
Line 349 unconditionally overwrites self._min_start_index, wiping out the conditional update done when a new minimum start is detected (lines 345–347). Remove that unconditional assignment or make it an else branch so you only set the min index on first assignment — that will fix the reported failure.
