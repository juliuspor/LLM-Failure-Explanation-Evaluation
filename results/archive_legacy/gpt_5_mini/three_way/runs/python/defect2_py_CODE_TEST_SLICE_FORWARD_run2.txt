What happened (root cause)
- In get_paint you clamp the input into v:
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
- But when you compute g you mistakenly use the original value instead of the clamped v:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- Because you used value (not v), values outside the bounds (e.g. -0.5 or 1.5) produce g < 0 or g > 255 and trigger the ValueError. That is exactly what the failing tests exercise.

Where it occurs
- Inside get_paint, immediately after clamping lines. The incorrect line is the one that computes g from value rather than v.

Why it fails for your tests
- test_lower_bound_clamping passes -0.5. v becomes self._lower_bound (0.0), but g is computed from -0.5 -> negative g -> the code raises ValueError instead of returning (0,0,0).
- test_upper_bound_clamping similarly produces g > 255 and raises.

Other minor issues / edge cases to be aware of
- If self._upper_bound == self._lower_bound you will get a ZeroDivisionError when computing the ratio; you should guard against that if that state is possible.
- The ValueError message is unhelpful: f"Color parameter outside of expected range: Red Green Blue" — it does not show the actual bad values. You may want to include the numeric g or the inputs.

Concrete fixes (step-by-step)
1. Use the clamped variable v when computing the gray component:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
2. (Optional but recommended) Guard against division by zero:
   denom = (self._upper_bound - self._lower_bound)
   if denom == 0:
       g = 0  # or 255, or raise a clear error depending on intended semantics
   else:
       ratio = (v - self._lower_bound) / denom
       g = int(ratio * 255.0)
3. (Optional) Improve the error message or remove the check because clamping ensures g will be in range:
   if g < 0 or g > 255:
       raise ValueError(f"Computed gray value out of range: {g}")

Minimal corrected code snippet
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)

        denom = self._upper_bound - self._lower_bound
        if denom == 0:
            g = 0  # or handle as appropriate
        else:
            g = int((v - self._lower_bound) / denom * 255.0)

        if g < 0 or g > 255:
            raise ValueError(f"Computed gray value out of range: {g}")

        return (g, g, g)

That change fixes the immediate failing tests: the clamped value v will be used for interpolation, so -0.5 and 1.5 map to 0 and 255 respectively.
