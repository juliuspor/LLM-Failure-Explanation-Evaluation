What happened
- The failing call is StringUtils.replace_each("aba", ["a"], [None]).
- Inside replace_each the code assumes every element of search_list and replacement_list is a string and calls len(...) on them.
- When the code reaches replacement_list[0] which is None, it does len(None) and Python raises TypeError: object of type 'NoneType' has no len().

Where it occurs (likely locations in your code)
- Any line that does something like:
  - search_len = len(search_list[i])
  - rep_len = len(replacement_list[i])
  - or any expression that uses len(item) without checking item is not None
- Also any code that indexes into search_list[i] or replacement_list[i] and then uses that value assuming it’s a string (e.g. slicing, concatenation, or building buffer sizes) will hit the same problem.

Why it happens (root cause)
- The implementation does not treat None in the arrays as “ignore this replacement”. According to the tests/docstring, None entries should be skipped (no replacement), but the code treats them like strings and immediately calls string operations (len(), slicing, etc.) on them.
- So the failure is not in the test: the test intentionally passes None to verify null-safety. The failure is the implementation calling len(None).

How to fix it (concrete steps)
1. In StringUtils.replace_each, before you call len(...) or otherwise operate on search_list[i] or replacement_list[i], check for None and skip that pair. For example:
   - if search_list[i] is None or replacement_list[i] is None:
       continue
   This ensures pairs containing None are ignored.
2. If you use precomputed lengths (search_len, rep_len) for buffer sizing or loops, compute them only after the None check:
   - if search is None: continue
   - search_len = len(search)
   - rep_len = len(replacement)  # only if replacement is not None
3. If code builds a new string based on rep_len (e.g. to estimate expansion), treat replacement None as empty or skipped; but the intended behavior from tests is to skip replacing when replacement is None, not to substitute empty string.
4. Run the three unit tests you provided to verify:
   - test_null_safety_inside_arrays_single
   - test_null_safety_inside_arrays_mixed
   - test_replace_each_basic

Minimal patch sketch (illustrative)
Replace code like this:
- search_len = len(search_list[i])
- rep_len = len(replacement_list[i])
with:
- if search_list[i] is None or replacement_list[i] is None:
    continue
  search = search_list[i]
  replacement = replacement_list[i]
  search_len = len(search)
  rep_len = len(replacement)

Why this solves it
- It prevents calling len(None) and avoids string operations on None, matching the specified behavior that None replacements should be ignored. Running the tests after this change should remove the TypeError and pass the null-safety tests.
