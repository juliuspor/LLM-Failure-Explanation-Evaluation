What happened (where / what / why)
- Where: inside ClassUtils.to_class, lines shown in your slice:
  - Line 765: for i in range(len(array)):
  - Line 766:     _ = array[i].upper()
  - Line 767:     classes.append(type(array[i]))
- What: the code calls array[i].upper() on every element before appending its type.
- Why it fails: None does not have an upper() method, so when array[i] is None Python raises:
    AttributeError: 'NoneType' object has no attribute 'upper'
  This is exactly the case in your failing test test_to_class_with_none_element (and test_to_class_all_none / test_to_class_single_none). Because the call to .upper() happens before type(array[i]) is appended, the function never reaches the point where it would record type(None).

Concrete fixes (steps and exact code to change)
Option A — simplest, correct behavior: remove the needless .upper() call and handle None explicitly
- Replace the loop body (lines 765–767) with:

  for i in range(len(array)):
      val = array[i]
      if val is None:
          classes.append(type(None))
      else:
          classes.append(type(val))

- This fixes the AttributeError and yields the expected [str, type(None), float] for ["Test", None, 99.0].

Option B — if .upper() was intended only for string normalization: check type before calling .upper()
- Replace with:

  for i in range(len(array)):
      val = array[i]
      if isinstance(val, str):
          # if you intended to do something with uppercase, do it here
          _ = val.upper()
      if val is None:
          classes.append(type(None))
      else:
          classes.append(type(val))

- This avoids calling .upper() on None or non-string types.

Why these fixes work
- They stop calling instance methods (.upper()) on elements that may be None (or non-strings).
- They append type(None) when the element is None, matching your tests' expected behavior.

Tests that fail before fixing
- test_to_class_with_none_element (AttributeError on None)
- test_to_class_all_none
- test_to_class_single_none
- Any other test with None in array

Summary
Remove or guard the call to array[i].upper() (line 766). Call type(array[i]) (or append type(None)) without invoking .upper() first. The smallest, safest change is the Option A snippet above.
