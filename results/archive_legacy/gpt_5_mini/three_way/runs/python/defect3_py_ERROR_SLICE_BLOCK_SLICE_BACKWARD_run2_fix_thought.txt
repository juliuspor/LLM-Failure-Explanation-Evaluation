The bug arises because some translators (like CsvEscaper) return 2 when handling a surrogate pair (they append two UTF-16 code units) whereas the contract says they should return codepoints consumed. To be robust, adjust translate to accept both behaviors: if a translator returns 0 do default; if it returns 1 treat as one codepoint consumed and advance by char_count(code_point_at(...)); if it returns >1 treat the returned value as number of code units consumed and advance pos by that many units.
