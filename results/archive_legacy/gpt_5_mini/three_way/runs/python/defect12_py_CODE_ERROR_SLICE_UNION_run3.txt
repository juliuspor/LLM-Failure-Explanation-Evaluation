What happened (root cause)
- The code lets upper become larger than the string length when lower > len(text).
- That makes substring_java(text, 0, upper) call slice_union with end > len(s); slice_union raises IndexError ("String index out of range: 15").

Where it occurs in your code
- You clamp upper here:
  if upper == -1 or upper > len(text):
      upper = len(text)
- But later you do:
  if upper < lower:
      upper = lower
  (this can set upper to a value > len(text) if lower > len(text))
- After those lines you call substring_java(text, 0, upper) (either in the branch where index == -1 or where index > upper). That call directly triggers slice_union's bounds check and the IndexError.

Why this happens (concrete example)
- Suppose len(text) == 10, lower == 15, upper == 5.
- First clamp sets upper to len(text) (10).
- Then because upper < lower, you set upper = lower (15).
- You then call substring_java(text, 0, 15) → slice_union checks end (15) against len(s) (10) → IndexError.

What to change (explicit fixes, pick one)
1) Re-clamp upper after you compare it to lower (preferred minimal change)
   - After this line:
       if upper < lower:
           upper = lower
     add:
       if upper > len(text):
           upper = len(text)
   Result: upper will never exceed len(text) before you call substring_java.

2) Alternatively, clamp lower first to the string length
   - At the start (after text checks) do:
       if lower > len(text):
           lower = len(text)
     Then keep your existing logic. This prevents lower from forcing upper > len(text).

3) Or be defensive on each substring call
   - Replace substring_java(text, 0, upper) with substring_java(text, 0, min(upper, len(text)))

Which to pick
- Option 1 is the smallest change and preserves the intended semantics of treating upper == -1 specially.
- Option 2 is reasonable if you want lower never to point past the end (may change behavior of index_of).
- Option 3 is a safety net but duplicates bounds logic in many places.

Exact code patch (option 1)
Replace:
    if upper < lower:
        upper = lower
with:
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)

That prevents substring_java from being called with end > len(text) and fixes the IndexError.
