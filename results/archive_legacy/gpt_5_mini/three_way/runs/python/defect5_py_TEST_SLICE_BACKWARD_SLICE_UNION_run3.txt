- At lines 335–347 the code tries to choose a component type for the new array using a single variable inferred_type:

- The bug is that type(array) on line 336 returns the Python list type (i.e. type(array) == list), not the type of the elements stored in the list. That makes inferred_type wrong as soon as array is non-None. Later code compares inferred_type to expected_type (a user-supplied element type) and may raise a TypeError because list != str (or other element types). In short: you inferred the wrong thing (the container's type instead of the element type).

- A second related failure is the handling when both array and element are None: inferred_type is set to object (line 340), and then the code raises TypeError at line 346 when expected_type is something else (e.g. str). That produces the observed failure when the call is ArrayUtils.add(None, None, expected_type=str): the method refuses to create a [None] of expected element type and raises instead.
