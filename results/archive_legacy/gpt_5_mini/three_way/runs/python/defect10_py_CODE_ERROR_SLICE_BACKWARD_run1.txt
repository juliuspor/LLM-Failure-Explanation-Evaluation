What happened
- The IndexError comes from this call: char_at(numeric, 0) inside the branch for trailing "l"/"L".
- When val is a single character "l" or "L", numeric = val[:-1] becomes the empty string "". Passing index 0 to char_at on an empty string raises IndexError ("String index out of range: 0").
- The code comment even notes a bug: there is no guard for the case len(val) == 1 (i.e. numeric == "").

Why it fails (root cause)
- The code assumes numeric has at least one character before calling char_at(numeric, 0). That assumption is false for inputs like "l" or "L" (or any non-digit single-character input whose last char is 'l'/'L').
- The char_at function checks index bounds and raises IndexError when index >= len(s). For numeric == "" the check triggers.

Where it occurs (exact spot)
- In create_number(), inside:
  if last_char in ("l", "L"):
      # BUG: missing guard for len(val) == 1 and non-digit input.
      if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
          return int(numeric)
- The call char_at(numeric, 0) is the precise line that causes the IndexError when numeric == "".

How to fix it (concrete steps)
Choose one of these explicit changes:

Option A — fast guard for single-character input
1. Immediately after computing numeric = val[:-1], add:
     if len(numeric) == 0:
         raise ValueError(f"{val} is not a valid number.")
2. Keep the rest of the logic unchanged.

Option B — remove char_at and use safe Python string methods
Replace the if expression with one that never indexes into numeric directly:
    if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():
        return int(numeric)
This uses startswith() and .isdigit(), which are safe on empty strings (startswith returns False on empty, numeric[1:].isdigit() is only evaluated when startswith("-") is True).

Option C — check before indexing
If you must keep char_at, check length before calling it:
    if len(numeric) > 0 and ((char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit()):
        return int(numeric)

Why these fixes work
- They prevent calling char_at with index 0 on an empty string, eliminating the IndexError and making the function properly treat "l" / "L" (and other single-char non-digit inputs) as invalid numbers.

Recommended: Option B (use startswith and isdigit). It’s concise, idiomatic Python, and avoids manual index checks.
