What happened (root cause)
- The error "TypeError: object of type 'NoneType' has no len()" means some variable that the code expects to be a sequence (so len(...) is valid) is actually None.
- In this function the very first len(...) call is on line 52: 
  - search_length = len(search_list)
  If search_list is None that single call will raise the TypeError immediately.
- Even if search_list is not None, there are other len(...) uses later that would raise if one of those values is None:
  - line 82: len(replacement_list[i]) or len(search_list[i]) — would fail if replacement_list or search_list is None, or if an element replacement_list[i] or search_list[i] is None.
  - line 85: len(text) // 5 — would fail if text is None.
  - line 94 and 115: uses of len(search_list[replace_index]) and text[start:] assume those variables/elements are not None.
  - line 104 explicitly checks len(search_list[i]) == 0 — that check itself will raise if search_list[i] is None.

Where it occurs
- Most likely the immediate cause is that search_list (the input passed to replace_each) is None, causing the failure at line 52.
- Other plausible causes depending on inputs: replacement_list is None, text is None, or an individual element search_list[i] or replacement_list[i] is None (those will fail later at lines 82 or 104).

Why it happened (typical scenarios)
- Caller passed None for one of the arguments (text, search_list, replacement_list) instead of a string/list.
- Caller passed a list that contains None as an element.
- The function assumes all inputs are non-None and elements are strings; that assumption is not enforced.
