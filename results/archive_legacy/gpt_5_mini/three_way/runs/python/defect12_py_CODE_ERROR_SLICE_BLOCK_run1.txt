What happened (root cause)
- On line 12 you clamp upper down to len(text):
  if upper == -1 or upper > len(text):
      upper = len(text)
- Then on line 16 you may increase upper to equal lower:
  if upper < lower:
      upper = lower

If lower is greater than len(text), that second step can set upper to a value > len(text). Later you call substring_java(text, 0, upper) (lines 20 and 26). substring_java checks bounds and raises IndexError when end > len(s) — which is exactly what the test shows: "String index out of range: 15".

Why this occurs in plain terms
- lower can be passed in with a value larger than the string length.
- You first clamp upper to the string length, but then you raise upper to lower without re-checking against len(text).
- The substring call uses the (now too-large) upper, causing the IndexError.

Exactly where it happened
- The problematic sequence is:
  - Line 12: clamp upper down to len(text)
  - Line 16: upper = lower (when upper < lower)
  - Line 20 or 26: call substring_java(text, 0, upper) with upper > len(text)
- The exception is thrown inside substring_java’s bounds check (slice block), because end > len(s).

How to fix it (concrete steps)
Option A — clamp upper after adjusting it to respect lower:
- Move the "clamp to len(text)" logic to after you enforce upper >= lower, or re-run the clamp after that change:

    if upper == -1:
        upper = len(text)
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)

This guarantees upper is never > len(text) when used.

Option B — clamp lower to the string length before using it to adjust upper:
- If you prefer lower should never exceed the string:

    if lower > len(text):
        lower = len(text)
    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower

Either option prevents setting upper > len(text) and avoids the IndexError.

Recommended small change (minimal patch)
Replace the current order:

    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower

with:

    if upper == -1:
        upper = len(text)
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)

This keeps behavior the same in normal cases but prevents upper being set above len(text) when lower is large.
