What failed
- On line 336 the code calls type(array). That returns the Python type of the list object (e.g. <class 'list'>), not the type of the elements inside the list. Because of that the variable inferred_type becomes list instead of the element type.
- Later (lines 346–347) the code checks inferred_type == object and expected_type != object and raises TypeError. Because inferred_type is wrong (list), that check behaves incorrectly and can raise a TypeError when it should not.
- There is also no clear handling when array is an empty list: line 336 will still set inferred_type = type(array) (a list), even though there are no elements to inspect.

Where it happens (exact lines)
- Line 336: inferred_type = type(array) — this is the core mistake.
- Lines 346–347: the TypeError is raised based on the wrong inferred_type value.
- Line 337–338 and 340 are related branches (element-based inference and fallback to object), and the logic here can make the wrong decision when array is empty or when both array and element are None.

Why that produces the failure
- The function intends to infer the element type from the existing list. Using type(array) infers the list’s type, not its element type. Later type-compatibility code compares inferred_type against expected_type (or object) and ends up concluding there is a type mismatch because it is comparing list vs element type.
- Example: if array = [1, 2], expected_type = int, the code sets inferred_type = list, then compares list == int (false) or list == object (false), and may raise a TypeError wrongly.
- If array is empty, inferred_type becomes list as well, but there are no elements to determine element type from; the code needs to treat that case separately rather than assuming the list type is the element type.

Concrete fixes (step-by-step)
1. Infer the element type from the array contents, not from the list object:
   - Replace line 336 with logic like:
     - if array is not None and len(array) > 0:
         inferred_type = type(array[0])
     - elif array is not None and len(array) == 0:
         inferred_type = expected_type if expected_type is not None else object
     - elif element is not None:
         inferred_type = type(element)
     - else:
         inferred_type = object
2. Decide how expected_type None should be treated:
   - If expected_type is None, treat it as "no constraint" or treat as object consistently. Use that choice in the branch above.
3. When comparing types, be explicit: either compare classes (inferred_type is expected_type) or use issubclass checks when you expect subclasses to be acceptable. Avoid accidental equality against the list class.
4. Only raise TypeError when you have a real type conflict between a known inferred element type and a provided expected_type. For example:
   - if inferred_type is not object and expected_type not in (None, object) and inferred_type is not expected_type:
         raise TypeError(...)
5. Add tests for these edge cases:
   - array = [1,2], element = 3, expected_type=int (should succeed)
   - array = [], element = 3, expected_type=int (should succeed)
   - array = None, element = None, expected_type=None (should return [None] with element type object)
   - array = [1], expected_type=str (should raise TypeError)
