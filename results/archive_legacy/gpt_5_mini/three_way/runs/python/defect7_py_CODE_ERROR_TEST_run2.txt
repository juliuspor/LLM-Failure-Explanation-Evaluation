What happened (root cause)
- The for loop in to_class calls array[i].upper() unconditionally:
  _ = array[i].upper()
- When array[i] is None (the test's second element), Python raises AttributeError: 'NoneType' object has no attribute 'upper' because None does not have an upper() method.
- The same line would also fail for non-string elements (e.g. a float) because float objects have no upper() either. So that line is the direct cause of the crash.

Where it occurs
- Inside the to_class method, in the for loop that iterates over array:
  for i in range(len(array)):
      _ = array[i].upper()   <-- this line causes the AttributeError
      classes.append(type(array[i]))

Why it was probably added
- That call looks like a stray debug/remnant line that assumes elements are strings. It is not needed for computing the element types (classes).

How to fix (clear steps)
1. Remove the unnecessary call to .upper() — it serves no purpose for determining types.
2. Use the element itself and append type(element). type(None) correctly yields type(None), so None elements are handled automatically.
3. Return the list of types.

Corrected implementation (drop-in replacement)
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    
        classes = []
        for i in range(len(array)):
            elem = array[i]
            classes.append(type(elem))
        return classes

Why this fixes the tests
- For ["Test", None, 99.0] this returns [str, type(None), float].
- For None input returns None.
- For empty list returns an empty list copy.
- For other mixed or all-None arrays, type(...) correctly produces the expected types.

Notes
- No special-case needed for None: type(None) is the proper Python representation and matches the test expectation.
- Make sure ArrayUtils.EMPTY_CLASS_ARRAY.copy() really returns an empty list (as the original code assumed).
