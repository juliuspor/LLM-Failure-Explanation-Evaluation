The bug arises because get_paint computes the grayscale value from the original input 'value' instead of the clamped 'v', then raises an exception if the computed channel is outside 0..255. Tests expect inputs outside bounds to be clamped and return (0,0,0) or (255,255,255). Fix: clamp the input first, compute the grayscale channel from the clamped value, ensure the channel is clamped to integer 0..255, and return the tuple. Remove the exception raising for out-of-range channel.
