What happened (root cause)
- At line 10 you define to_utf16_code_units(s: str) -> List[int] (it should build and return a list named code_units).
- At line 35 the code does this check: if index < 0 or index >= len(code_units): ...
- That check uses the name index, but to_utf16_code_units has no parameter or local named index. Because index is not defined in that function scope, Python raises a NameError at runtime (or if index exists elsewhere as a different variable, the logic is still wrong because the check and the following surrogate logic belong to a different responsibility).

Why this is wrong (concise)
- The bounds check and the surrogate-pair decoding (lines 38–44) are not part of building a UTF-16 code unit list — they belong in a separate function that takes an index into code_units and returns the code point at that index. Placing that code inside to_utf16_code_units makes it reference variables it doesn’t own (index), so the program fails where the name index is used.

Where it fails (exact lines)
- The immediate failure is at line 35: if index < 0 or index >= len(code_units): — index is undefined in to_utf16_code_units.
- The following lines (38–44) also assume index and code_units are available to decode a surrogate pair; they’re in the wrong function/scope.

How to fix (clear steps)
Option A — move the decoding logic into its own function (recommended)
1. Keep to_utf16_code_units(s: str) -> List[int] to only build and return code_units.
2. Create a new function, e.g.:
   def code_point_at(code_units: List[int], index: int) -> int:
       if index < 0 or index >= len(code_units):
           raise IndexError(f"String index out of range: {index}")
       high = code_units[index]
       if 0xD800 <= high <= 0xDBFF and index + 1 < len(code_units):
           low = code_units[index + 1]
           if 0xDC00 <= low <= 0xDFFF:
               return 0x10000 + ((high - 0xD800) << 10) + (low - 0xDC00)
       return high
3. Call to_utf16_code_units(s) to get code_units, then call code_point_at(code_units, index).

Option B — if the intent was to accept an index in the same function
1. Change the signature to accept index: def to_utf16_code_units(s: str, index: int) -> List[int]: (but this mixes responsibilities; not recommended).
2. Ensure code_units is created before you use it, and then perform the bounds check and decoding.

Pick Option A unless you intentionally want to combine both tasks. The essential fix is: put the index check and surrogate decoding into a function that actually receives index (and receives code_units), so index is defined where it’s used.
