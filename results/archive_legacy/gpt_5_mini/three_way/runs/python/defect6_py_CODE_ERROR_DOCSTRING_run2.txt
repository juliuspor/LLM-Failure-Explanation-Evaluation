What happened (where it fails)
- The failing behavior happens inside the integer branch beginning at:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- On the very next line you do:
  value = int(x)
  This converts -0.0 to the integer 0 and thereby discards the negative sign.
- Later you call self.add(str(value)) for the non-scientific case. str(0) is "0", so you emit "0" instead of the expected "-0.0".
- The test that expected "-0.0" fails because the code turned -0.0 → int(-0.0) → 0 and lost the sign.

Why that happens (root cause)
- In IEEE-754 floats, -0.0 compares equal to +0.0 (x == int(x) is True) but has a sign bit. int(-0.0) returns the integer 0 and there is no way to represent the negative zero sign in an int. Converting to int therefore loses the sign information for negative zero.

Exactly where to look in the code
- The loss happens at value = int(x) and then when you call self.add(str(value)) inside the if block for integer-valued floats.

Concrete fixes (step-by-step, pick one)

1) Preserve the negative-zero case explicitly
- Add a check for negative zero before converting to int:
    if x == 0.0 and math.copysign(1.0, x) < 0:
        self.add("-0.0")
        return
  Then keep the rest unchanged. This ensures -0.0 is printed as "-0.0".

2) Keep sign information and use it when formatting integers
- Record whether x is negative before casting:
    neg = math.copysign(1.0, x) < 0
    value = int(x)
    if value == 0 and neg:
        self.add("-0.0")
        return
  This is useful if you want to preserve the rest of your integer formatting logic.

3) Avoid converting to int for zeros
- Change the integer branch condition to skip zero:
    if x != 0.0 and x == int(x) and not math.isinf(x) and not math.isnan(x):
  and let the final else (self.add(str(x))) handle both +0.0 and -0.0 (str(-0.0) -> "-0.0").

Which to choose
- If you need to preserve the exact "-0.0" output, option 1 or 2 are simplest and explicit.
- If you prefer formatting consistency and str(x) handles your other cases, option 3 is the least intrusive.
