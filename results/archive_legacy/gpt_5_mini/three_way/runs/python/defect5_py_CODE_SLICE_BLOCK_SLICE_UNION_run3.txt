What went wrong (short answer)
- At line 335–337 you set inferred_type = type(array). That makes inferred_type the type of the list object itself (e.g. <class 'list'>), not the type of the elements inside the list. Later code (ArrayUtils._copy_list_grow1 and the check at line 346) expects a component/element type (or object when unknown). Because inferred_type holds the wrong value, the logic that detects unknown element type and raises the cast error never runs correctly and the copy/assignment behaves incorrectly — that is the root cause.
