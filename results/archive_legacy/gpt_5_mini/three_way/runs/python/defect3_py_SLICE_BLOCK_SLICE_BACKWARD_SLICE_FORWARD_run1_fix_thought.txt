The bug arises because unmatched UTF-16 surrogate code units can be returned as-is from code_point_at and then propagated. To fix without changing other helpers, update translate to detect unmatched high/low surrogates and replace them with U+FFFD (0xFFFD) instead of returning invalid surrogate values. Also properly handle valid surrogate pairs and regular code units, and keep contract with translate_codepoint. The function will append replacement for malformed surrogates and advance by one code unit in that case.
