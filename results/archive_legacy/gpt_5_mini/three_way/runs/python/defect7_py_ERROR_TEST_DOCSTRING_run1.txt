What happened (root cause)
- The failing test calls ClassUtils.to_class(test_array) with test_array = ["Test", None, 99.0].
- Inside ClassUtils.to_class the code tries to call .upper() on a variable that can be None. That raises AttributeError: 'NoneType' object has no attribute 'upper'.
- In other words, the method assumes the array elements are non-None and calls a string method (upper) on an element variable (commonly named element or obj) without first checking whether that element is None.

Where it occurs (what to look for)
- Open hit07_classutils.py and find the to_class method.
- Look for a loop like:
  for i, element in enumerate(array):
      name = element.upper()   # or something equivalent that calls .upper() on element
  or any line that does element.upper() (or calls a string method on a variable named element or obj).
- The AttributeError comes from that line when element is None.

Why that causes the test failure
- The test intentionally passes an array containing None to make sure to_class tolerates None elements.
- Because the implementation calls element.upper() (or similar) without guarding for None, Python tries to call .upper on the None object and raises AttributeError.
- The test expects either type(None) or None to be returned for that position; instead the method errors out before producing any result.

How to fix it (exact steps and specific code changes)
1) Handle a None input array up front (the tests expect None input to return None). At the top of to_class:
   if array is None:
       return None

2) In the loop over elements, check for None before calling any methods on the element. Replace code that does element.upper() (or any attribute access that assumes a non-None) with a guarded branch. Example replacement:

   # before: (buggy)
   classes = []
   for element in array:
       # element.upper() will fail when element is None
       classes.append(element.__class__)   # or code that depends on element.upper()

   # after: (correct)
   classes = []
   for element in array:
       if element is None:
           classes.append(type(None))    # tests expect type(None) for None elements
       else:
           classes.append(type(element))

3) Ensure the returned list/array uses the same semantics as tests:
   - Tests expect a list of class objects: [str, type(None), float] etc.
   - So append type(element) for non-None elements and type(None) for None elements.

Minimal complete fix (example implementation)
- Replace the body of to_class with:

def to_class(array):
    if array is None:
        return None
    classes = []
    for element in array:
        if element is None:
            classes.append(type(None))
        else:
            classes.append(type(element))
    return classes

Notes on the docstring mismatch
- The docstring says "If any of these objects is None, a None element will be inserted into the array." The tests expect type(None) at that position. Decide which you want:

  - Otherwise (recommended) return type(None) for None elements, and update the docstring to say "type(None)" instead of "None element".

Why this fix resolves the error
- It avoids calling any attribute or method on element when element is None, so no AttributeError is raised.
- It returns the correct class objects for each array element, matching the unit tests.
