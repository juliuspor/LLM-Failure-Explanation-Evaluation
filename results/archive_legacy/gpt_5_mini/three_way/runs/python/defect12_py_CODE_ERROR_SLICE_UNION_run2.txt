What happened (root cause)
- The IndexError comes from calling substring_java(text, 0, upper) with upper greater than len(text). The slice checker (slice_union) raises because end > len(s).
- That happens when lower is larger than len(text). Walkthrough:
  1. You first clamp upper to len(text):
     if upper == -1 or upper > len(text):
         upper = len(text)
  2. Then you do:
     if upper < lower:
         upper = lower
     If lower > len(text), this sets upper = lower (which is still > len(text)).
  3. Later you call substring_java(text, 0, upper) (either in the index == -1 branch or the index > upper branch). Because upper was set to lower and not re-clamped, the end argument to substring_java is > len(text) and the slice checker raises IndexError.

Where it occurred in your code
- The problematic assignment is the sequence:
  - "if upper == -1 or upper > len(text): upper = len(text)" (clamp upper)
  - then "if upper < lower: upper = lower" (this can re-expand upper beyond len(text))
- The failing call is substring_java(text, 0, upper) (the call that runs after those two lines).

How to fix it (clear steps)
Option A — Clamp upper after comparing with lower (minimal change)
- After you possibly set upper = lower, clamp again to len(text):
  if upper == -1 or upper > len(text):
      upper = len(text)
  if upper < lower:
      upper = lower
  if upper > len(text):
      upper = len(text)
This guarantees upper <= len(text) before any substring_java call.

Option B — Clamp lower to the string length (clearer intent)
- Ensure lower never exceeds len(text) up front:
  if lower > len(text):
      lower = len(text)
  if upper == -1 or upper > len(text):
      upper = len(text)
  if upper < lower:
      upper = lower
Now upper will never be > len(text).

Option C — Use a single safe clamp expression
- Replace the adjustments with a single calculation that guarantees limits:
  n = len(text)
  lower = min(lower, n)
  upper = n if upper == -1 else min(upper, n)
  if upper < lower:
      upper = lower
This keeps all values in [0, n] before substring_java calls.

Extra note about index calculation
- If you clamp lower down to n, calling StringUtils.index_of(text, " ", lower) will start at n (end of string) and return -1 — which is handled by your code. That is the correct and safe behavior.

Which change I recommend
- Use Option C (single safe clamp) — it's concise, explicit, and prevents upper being reset to > len(text) later.

Example fix (pseudo-replacement)
  n = len(text)
  if lower > n:
      lower = n
  if upper == -1:
      upper = n
  else:
      upper = min(upper, n)
  if upper < lower:
      upper = lower

This will prevent the substring_java(text, 0, upper) calls from ever using an end index greater than len(text).
