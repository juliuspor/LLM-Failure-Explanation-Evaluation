What goes wrong
- If mantissa ever becomes 0 the while loop at line 181 never ends.  
  - On entry the loop condition is `while mantissa % 10 == 0:`. For mantissa == 0 that is True (0 % 10 == 0).
  - Inside the loop you compute
    - `check_val = (mantissa // 10) * (10 ** (exp + 1))` (line 182). For mantissa == 0 this is 0.
    - You then compare `if check_val == value:` (line 183). If value is also 0 the condition is True and you do `mantissa //= 10` (line 184) which leaves mantissa == 0, and `exp += 1` (line 185). Nothing in the loop changes mantissa away from 0, so the loop repeats forever.
- Where it occurs: the infinite loop happens inside the while at lines 181–187 when mantissa == 0 and value == 0 (or whenever division would produce a 0 mantissa that still satisfies the check_val == value test).
- Effect: the function hangs (infinite loop) instead of returning / adding a value.

How to fix it (concrete steps)
1. Prevent entering the loop for a zero mantissa (or break if mantissa becomes 0). For example replace line 181 with:
   - `while mantissa != 0 and mantissa % 10 == 0:`
   This avoids the infinite loop because the loop will not run when mantissa == 0.
2. Or add an explicit break when mantissa becomes 0 inside the loop:
   - After `mantissa //= 10` (line 184) add:
     - `if mantissa == 0: break`
3. Extra defensive option: if you only want to strip trailing zeros from positive integers safely, check value != 0 before the loop:
   - `if abs(x) >= 100 and value != 0:`

Pick one of these fixes and apply it at the while condition or immediately after the division. Any of them will remove the infinite loop while keeping your intended behavior of compressing trailing zeros into a mantissa + exponent form.
