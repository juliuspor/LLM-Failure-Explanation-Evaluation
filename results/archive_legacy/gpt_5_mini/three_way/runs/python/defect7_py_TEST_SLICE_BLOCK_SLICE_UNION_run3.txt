What happened (where)
- Method: ClassUtils.to_class
- Lines: 765–767 (the loop)
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))

Why it fails (root cause)
- The code calls array[i].upper() on every element before recording its type.
- If array[i] is None (or any non-string object like an int or float), it has no upper() method. That raises an AttributeError at line 766.
- Because the code never checks for None or for the element being a str, the loop immediately crashes on the first non-string/None element. In your tests that include None (test_to_class_with_none_element, test_to_class_all_none, test_to_class_single_none) this is exactly what triggers the failure.

Why this is incorrect behavior
- The intended job of to_class is to return the classes/types of the elements, not to call string methods on them. Calling upper() is unnecessary and unsafe for non-strings.
- For None elements the correct returned type is type(None), not an exception.

How to fix it (concrete steps, exact places to change)
Option A — simplest, remove the unnecessary call:
- Change lines 765–767 to:
  for i in range(len(array)):
      classes.append(type(array[i]))
- Explanation: do not call array[i].upper() at all; just append type(array[i]). This works for strings, numbers, None, lists, etc.

Option B — if the intent was to normalize strings first, only call upper() for strings and handle None:
- Replace the loop with:
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          classes.append(type(None))
      elif isinstance(elem, str):
          _ = elem.upper()   # safe: elem is a str
          classes.append(str)
      else:
          classes.append(type(elem))
- Explanation: check for None and str explicitly so you only call upper() when it's valid.

Option C — defensive using getattr (if you still wanted to try calling upper when present):
- Replace call with:
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          classes.append(type(None))
          continue
      # call upper only if present and callable (but not recommended unless needed)
      upper = getattr(elem, "upper", None)
      if callable(upper):
          _ = upper()
      classes.append(type(elem))

Which fix to choose
- If the goal is simply to produce the types array, use Option A. It's the minimal, correct approach.
- If you need string normalization as a side effect, use Option B and explicitly check types.

What to expect after the fix
- None elements will no longer raise AttributeError.
- Tests expecting [str, type(None), float] and other None cases will pass because type(None) is appended for None elements.
- Mixed-type arrays will be handled correctly without invoking methods that may not exist on elements.
