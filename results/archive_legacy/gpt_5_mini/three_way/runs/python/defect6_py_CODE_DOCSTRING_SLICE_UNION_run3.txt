What failed
- At runtime you get UnboundLocalError: local variable 'exp' referenced before assignment (or the code behaves as if exp is not set) when execution reaches line 182:
  check_val = (mantissa // 10) * (10 ** (exp + 1))

Why it happens (root cause)
- The variable exp is used on line 182 inside the inner loop but it is not guaranteed to be defined there. In the pasted slice the assignment exp = 0 is missing or not in scope before the while starts, so exp is uninitialized when the code runs that branch.
- Practically this happens when exp = 0 is mis‑indented or removed (so it never runs before the if abs(x) >= 100 block). The while block then references exp, triggering the UnboundLocalError.

Exactly where
- Use of exp occurs at line 182 in the slice:
    182 | check_val = (mantissa // 10) * (10 ** (exp + 1))
- The intended initialization (exp = 0) is supposed to be before that (original code shows it at line 179) but in the failing version it isn’t actually executed (missing or mis‑indented).

How to fix (concrete steps)
1. Ensure exp is initialized before you ever use it. Put the initialization immediately after mantissa is set:
    mantissa = value
    exp = 0

   So the code becomes (lines approximate):
    value = int(x)
    mantissa = value
    exp = 0
    if abs(x) >= 100:
        while mantissa % 10 == 0:
            check_val = (mantissa // 10) * (10 ** (exp + 1))
            ...

2. Alternatively, restructure the loop so exp is a local loop variable created before use, for example:
    exp = 0
    while abs(mantissa) >= 10 and mantissa % 10 == 0:
        mantissa //= 10
        exp += 1

   and then use mantissa and exp after the loop.

3. Add tests to confirm:
   - x = 123000 -> should produce "123E3" (or whatever your formatting expects)
   - x = 1000 -> not crash, produce expected output
   - x = 0 and x < 100 -> ensure not entering the abs(x) >= 100 branch

Why this explanation rules out other suspects
- The rest of the logic (mantissa //= 10 and the check comparing recomposed value to value) is valid for trimming trailing zeros; the immediate error is caused by using exp before assigning it. Once exp is initialized before the while, the code will not raise that UnboundLocalError.
