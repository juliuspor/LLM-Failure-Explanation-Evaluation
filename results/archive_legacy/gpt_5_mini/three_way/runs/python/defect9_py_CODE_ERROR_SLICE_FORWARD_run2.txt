- The TypeError ("object of type 'NoneType' has no len()") is raised on these lines (shown in your slice):
  - line 83: greater = len(replacement_list[i]) - len(search_list[i])
  - line 84: if greater > 0:
- In that loop you assume every element of search_list and replacement_list is a str and call len(...) on each element.
- But earlier you deliberately allowed None entries in the arrays and guarded against them in the match-finding loops (you check search_list[i] is None or replacement_list[i] is None). The "BUG" comment is correct: the size-estimate loop is not null-safe. If either search_list[i] or replacement_list[i] is None, calling len(...) raises the TypeError.

Where it occurs in code
- The failing code is the estimation loop right after you set increase = 0 (around lines 77–85 in your slice). You iterate for i in range(search_length): and immediately call len(replacement_list[i]) and len(search_list[i]) without any null checks. That leads to the exception when any list element is None.

Why the rest of the code didn't prevent this
- You do check for None/empty replacements and searches in the match-finding loops, so those loops work around None entries. But the size-guess loop runs earlier and independently, so it must also account for None values. The later usages of len(search_list[replace_index]) are safe because replace_index comes from the guarded selection, but the estimation loop is unguarded.
