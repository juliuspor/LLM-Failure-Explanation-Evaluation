What happened (where and why)
- Location: inside ClassUtils.to_class, in the loop at lines 765–767:
  - Line 766 calls array[i].upper()
  - Line 767 appends type(array[i]) to classes
- Root cause: calling array[i].upper() assumes every element is a string and non-None. If an element is None (or any non-string type, e.g. int, float), Python raises AttributeError because NoneType (or int, float) has no attribute 'upper'. That stops the method before it can append the element's type, so tests that pass None or non-string values fail.

Why this produces the observed failure
- The test case passes an array containing None and other non-string objects. On encountering None at array[i], the statement array[i].upper() raises AttributeError ("'NoneType' object has no attribute 'upper'"), so the function never returns the intended list of types.
- This mirrors the Java NullPointerException described in the comment — the code was performing an operation on the element before checking/using its type.

Concrete fixes (exact steps and code)
1. Remove the unsafe .upper() call. You do not need to call .upper() to get an object's class/type.
2. Handle None explicitly so its type becomes type(None) (or keep type(None) from type(None) directly).
3. Keep behavior simple: for each element val in array, append type(val).

Suggested minimal patch for the loop (replace lines 765–767):

    for i in range(len(array)):
        val = array[i]
        classes.append(type(val))

Or, more Pythonic:

    for val in array:
        classes.append(type(val))

If you want to keep a special case (explicitly store type(None) for None), the above already does that because type(None) is returned by type(None). No extra code required.

Additional safe-guards
- At start of to_class, ensure None input returns None (your tests expect this). Example:

    if array is None:
        return None

- If you need to enforce only sequences, consider type checks, but for your tests the simple approach above is correct.

Summary
- The immediate cause is the call array[i].upper() (line 766) which assumes string/non-None elements and raises AttributeError for None or other non-string types.
- Fix: remove that call and simply use type(array[i]) (or iterate val in array and append type(val)). This will allow None and mixed types to be handled as expected.
