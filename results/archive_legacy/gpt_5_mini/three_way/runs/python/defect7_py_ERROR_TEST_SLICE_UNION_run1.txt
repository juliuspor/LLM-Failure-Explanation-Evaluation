What happened (root cause)
- In ClassUtils.to_class, the loop at lines 765–767 accesses array[i].upper() without checking whether array[i] is None:
  - line 765: for i in range(len(array)):
  - line 766: _ = array[i].upper()
  - line 767: classes.append(type(array[i]))
- When array contains a None element (e.g. the second element in ["Test", None, 99.0]), array[i] is None and None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The test expects None elements to be translated to type(None) in the result. The call to .upper() is unnecessary for determining the element's class and is the immediate cause of the failure.

Why it failed here and not elsewhere
- The code assumes every element is a string (or at least has .upper()), so it tries to call .upper() before getting the type. That assumption is false for None and other non-string types (though in this test only None triggers the exception).
- The exception occurs exactly at line 766 when encountering the None element.

How to fix it (concrete steps and example patches)
Pick one of these explicit fixes and apply it where the loop is implemented:

1) Remove the .upper() call entirely (best if .upper() is not needed):
Replace lines 765–767 with:
  for i in range(len(array)):
      classes.append(type(array[i]))

2) Guard against None and non-strings before calling .upper():
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          classes.append(type(None))
      else:
          # only call .upper() if elem is a str (if that call is needed elsewhere)
          if isinstance(elem, str):
              _ = elem.upper()
          classes.append(type(elem))

3) Use try/except to avoid crashing (less preferred because it hides unexpected errors):
  for i in range(len(array)):
      try:
          _ = array[i].upper()
      except AttributeError:
          pass
      classes.append(type(array[i]))

Which to choose
- If .upper() was only used as a type-check or leftover debug line, remove it (option 1).
- If .upper() has a real side-effect or is needed for logic when elements are strings, use the explicit isinstance guard (option 2).
- Avoid broad try/except for general logic (option 3) unless you have a good reason.

What to verify after the change
- Run the provided unit tests, especially test_to_class_with_none_element and test_to_class_all_none, to confirm None elements produce type(None).
- Confirm other tests (mixed types, strings, nested lists) still pass.

Summary
The code calls array[i].upper() on every element (line 766) without checking for None, so encountering None raises AttributeError. Remove or guard that .upper() call and append type(array[i]) directly (or check for None first) to fix the bug.
