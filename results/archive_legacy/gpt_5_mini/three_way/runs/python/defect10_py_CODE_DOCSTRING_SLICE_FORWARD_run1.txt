What happened
- Input: val = "l" (or "L").
- At line that sets last_char = val[-1], last_char == 'l', so execution enters the branch for last_char in ("l", "L").
- numeric = val[:-1] produces the empty string "".
- The code then evaluates this condition:
  (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit()
  That calls char_at(numeric, 0).
- char_at is implemented as return s[index] (see slice_forward: line 32 -> return s[index]).
- Because numeric == "", s[0] is out of range and char_at raises IndexError ("String index out of range: 0").
- Instead of raising the intended ValueError for a non-numeric input, you get an IndexError coming from char_at on an empty string.

Why this is the root cause
- The code assumes numeric has at least one character before calling char_at(numeric, 0) but does not check len(numeric). For a one-character input whose qualifier is stripped (like "l"), numeric becomes empty and the direct index access fails.
- The failure occurs inside the "l"/"L" qualifier branch, specifically where char_at(numeric, 0) is evaluated.

Where exactly it fails
- The immediate exception originates in char_at at line 32 (return s[index]) when index == 0 and s == "".
- In the provided function, the call to char_at(numeric, 0) is the offending expression inside the if that tries to validate negative numbers for the "l"/"L" case.

How to fix it (concrete steps)
Option A — reject empty numeric immediately:
1. After numeric = val[:-1], add:
   if len(numeric) == 0:
       raise ValueError(f"{val} is not a valid number.")
   This stops before char_at is ever called.

Option B — check numeric safely in the conditional:
1. Replace the condition
   if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
with a safe check that ensures numeric is non-empty, e.g.
   if (len(numeric) > 0 and numeric[0] == "-" and numeric[1:].isdigit()) or numeric.isdigit():

Option C — avoid char_at and use Python string methods:
1. Replace char_at usage with native checks:
   if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():

Any of these will ensure you raise the intended ValueError for "l"/"L" instead of letting char_at trigger an IndexError.
