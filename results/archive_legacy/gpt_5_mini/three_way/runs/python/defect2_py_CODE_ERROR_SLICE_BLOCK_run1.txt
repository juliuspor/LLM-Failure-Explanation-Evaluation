What happened (root cause)
- You clamp the input into v (lines that set v), but when you compute g you still use the original value variable:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If value is outside the [self._lower_bound, self._upper_bound] range, g can be < 0 or > 255, so the check on lines 74–75 raises ValueError. In short: you clamped v but never used it.

Where it occurred
- The clamp lines create v:
    v = max(value, self._lower_bound)
    v = min(v, self._upper_bound)
- The bug is on the next computation line that still references value instead of v. The ValueError is raised at the guard on lines 74–75 when g is out of [0,255].

How to fix it (concrete steps)
1. Use the clamped v in the g computation (replace value with v):
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

2. Also guard against a zero denominator (upper_bound == lower_bound). For example:
   if self._upper_bound == self._lower_bound:
       g = 0  # or 127 or 255 depending on desired behavior
   else:
       g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

3. After computing g, clamp it to [0,255] to avoid any floating rounding issues:
   g = max(0, min(255, g))

Minimal corrected version
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)

        # Avoid division by zero
        if self._upper_bound == self._lower_bound:
            g = 0
        else:
            g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

        # Clamp rounding artifacts
        g = max(0, min(255, g))

        return (g, g, g)

Extra note
- The current ValueError message "Color parameter outside of expected range: Red Green Blue" doesn't show which value triggered it. After the fixes you probably won't need that exception; if you keep it, consider including the offending g and the input value for easier debugging.
