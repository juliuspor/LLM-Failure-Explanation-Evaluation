What happened (root cause)
- The code calls len(...) on elements of search_list and replacement_list at multiple places before checking whether those elements are None.
- Python evaluates Boolean expressions left-to-right, so when there is a condition like:
  or search_list[i] is None
  or len(search_list[i]) == 0
the len(...) is evaluated before the is None check if it appears earlier in the expression. That allows len(None) to run and raises TypeError: object of type 'NoneType' has no len().
- Specific occurrences in your slice:
  - Line 82: greater = len(replacement_list[i]) - len(search_list[i]) — this assumes both replacement_list[i] and search_list[i] are strings; if either is None you get the error here.
  - Lines 102–107: the code tests multiple conditions including len(search_list[i]) == 0 and replacement_list[i] is None. If search_list[i] is None, len(search_list[i]) is evaluated and raises the TypeError before the is None check can short-circuit it.
  - Line 108: text.find(search_list[i], start) will also misbehave (TypeError) if search_list[i] is None, but your reported error is the len(...) one earlier.

Why that leads to the error
- Because len() cannot be called on None. The code both:
  1) computes lengths (line 82) without guarding for None, and
  2) uses boolean chains that check len(...) and None in the wrong order, so the None check does not protect the len(...) call.

1. Ensure you never call len(...) on a possibly-None element. Check for None first. For example change the block at line 82 to:

   r = replacement_list[i]
   if s is None or r is None:
       continue   # or handle appropriately
   greater = len(r) - len(s)

2. Reorder the boolean checks at lines ~102–107 so None checks come before any len(...) calls. Example:

       or search_list[i] is None
       or replacement_list[i] is None
       or len(search_list[i]) == 0):

   This guarantees you never evaluate len(search_list[i]) when search_list[i] is None.

3. Before calling text.find(search_list[i], start) (line 108), ensure search_list[i] is not None:

4. Alternatively (often better): validate input up-front. If search_list and replacement_list should only contain strings, sanitize them once at function start:

Summary
- The error is caused by calling len(...) on None because the code checks length before checking for None (and Python evaluates expressions left-to-right). Fix by checking for None first (or sanitizing inputs) before any len(...) or text.find(...) calls.
