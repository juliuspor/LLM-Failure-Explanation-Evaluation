What failed
- In the "max middle" section you fetch the wrong data item. The code checks self._max_middle_index but then calls:
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  i.e. it uses self._min_middle_index instead of self._max_middle_index.

Why that causes the failure (what happened, where)
- Location: the block that starts with
    if self._max_middle_index >= 0:
  (see the "max middle" block in your posted code).
- Root cause: you are comparing the new period's middle value with the period at the wrong index. That makes the conditional middle > max_middle compare against a min-index period, so self._max_middle_index will never be correctly maintained.
- Immediate bad effects:
  1. Logic bug — the max-middle tracking becomes wrong: you may never update _max_middle_index properly or update it incorrectly.
  2. Potential runtime error — if _min_middle_index is -1 (not set) while _max_middle_index >= 0, get_data_item(self._min_middle_index) will be called with -1 and will likely raise an IndexError or similar. That mismatch (checking _max_middle_index then using _min_middle_index) is what can produce an exception at runtime.

Exact line to change
- Replace:
    max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  with:
    max_middle_period = self.get_data_item(self._max_middle_index).get_period()

   - Change the line above so the max-middle block uses self._max_middle_index.
