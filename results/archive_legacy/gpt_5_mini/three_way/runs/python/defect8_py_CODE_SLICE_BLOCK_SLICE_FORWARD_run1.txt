What goes wrong
- Root cause: the length check at lines 234–237 uses a compound boolean expression that’s hard to read and led to the wrong intent. The code currently does:
  if length != 2 and length != 5 and length < 7:
This is intended to raise when the string length is not one of the allowed sizes (2, 5, or >=7). The correct test is: raise when NOT (length == 2 or length == 5 or length >= 7). The current expression is error-prone and can be misread; writing it explicitly avoids the logic mistake.

- Where it happens: the failure is triggered in the early validation block (the if on length). If that expression is wrong, some valid lengths can be incorrectly accepted or rejected and the function continues to the character-index checks below (lines 239–247). That then causes IndexError or incorrect ValueError at:
