What happened (root cause)
- The failing code is ClassUtils.to_class. When it loops over the input array it calls .upper() on an element (or on a variable derived directly from an element) without first checking whether that element is None.
- When the input array contains a None (e.g. test_array = ["Test", None, 99.0]) the code reaches the line that does element.upper() and Python raises AttributeError: 'NoneType' object has no attribute 'upper' because None has no upper() method.

Why that error shows up in your tests
- The test test_to_class_with_none_element passes an array containing None. The to_class implementation assumes every array element is a string (or otherwise has upper()) and therefore calls .upper() unconditionally.
- The docstring already says "If any of these objects is None, a None element will be inserted into the array" and "This method returns None for a None input array", so the implementation should explicitly handle None values instead of calling string-only methods on them.

Where it occurs (what to look for)
- In ClassUtils.to_class: find the loop over the input array and the line that calls .upper() (or any other attribute/method) on the element (common names used in implementations: element, obj, value, clsname).
  Example problematic pattern:
    for element in array:
        name = element.upper()   # <-- fails when element is None
- If the code computes a class name string from the element and then uppercases it (e.g. to test primitive names or do lookups), that upper() call is the exact failing operation.

How to fix it (concrete steps)
1. Short-circuit None input:
   - If array is None, return None immediately.
     if array is None: return None

2. Handle None elements before calling any methods on them:
   - In the loop, check element is None and append type(None) (or None if you want a None in the class array per your docstring):
     if element is None:
         result.append(type(None))
         continue

3. For non-None elements, use safe, explicit ways to get the element's class:
   - Prefer type(element) or element.__class__ instead of string operations on the value.
     result.append(type(element))

4. If you need special handling for strings or primitives (e.g. mapping numeric types, boxed primitives), do that only after confirming element is not None.

Minimal fixed implementation you can paste into hit07_classutils.py

def to_class(array):
    if array is None:
        return None
    result = []
    for element in array:
        if element is None:
            result.append(type(None))
        else:
            result.append(type(element))
    return result

Why this fix works
- It never calls methods on None because we test for None first.
- It uses type(element) which always works for any non-None Python object and returns the class object your tests expect (e.g. str, float, int, bool, list).

Extra notes
- If your original code attempts to detect primitive-type equivalents by comparing uppercased class-name strings, move that logic after the None check and base comparisons on type(element).__name__ (a string) or isinstance checks, not on calling .upper() on the element itself.
- Make sure tests expecting type(None) vs None in the returned list are consistent with your docstring. The tests you provided expect type(None), so the implementation above returns type(None) for None elements.
