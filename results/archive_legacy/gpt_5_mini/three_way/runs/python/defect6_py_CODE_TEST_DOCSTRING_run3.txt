What happened (root cause)
- The branch at the start of add_number:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
  treats -0.0 as an integer. In Python -0.0 == 0 is True, so value = int(x) becomes 0 (no sign).
- Later you call self.add(str(value)) which appends "0". That loses the negative sign of -0.0, so test_negative_zero fails: the code produced "0" but the test expects "-0.0".

Where it occurs (lines / variables)
- The failing logic is in add_number, the integer-handling branch:
  - The test that makes this branch true is x == int(x).
  - value = int(x) converts -0.0 to 0 (variable name value).
  - self.add(str(value)) outputs "0" and drops the sign.
- Also note the earlier check:
  prev = self.get_last_char()
  if x < 0 and prev == '-':
      self.add(" ")
  This check doesn't help for -0.0 because x < 0 is False for -0.0, so you won't add a space even though you're going to output a leading '-' for negative zero — which would create "--0.0" if the code were changed to output "-0.0" without adjusting this.

Why it happens (concise)
- Equality and int conversion lose the sign of negative zero. The code branches to integer printing before considering signed zero semantics.
- The "x < 0" guard for inserting a space also fails for -0.0 because -0.0 is not less than 0 in Python.

Concrete, actionable fix (exact steps and code)
1) Provide a precise negative-zero test (use math.copysign or a helper). For example add a helper on CodeConsumer:

    @staticmethod
    def is_negative_zero(x: float) -> bool:
        return x == 0.0 and math.copysign(1.0, x) < 0.0

2) In add_number, handle negative zero before the integer branch and before the "x < 0" space-insertion check. Also make sure to insert a space if previous char is '-' to avoid "--":

    def add_number(self, x: float):
        prev = self.get_last_char()

        # Handle negative zero first (preserve sign)
        if CodeConsumer.is_negative_zero(x):
            if prev == '-':
                self.add(" ")
            self.add("-0.0")
            return

        # Existing space-insertion for normal negative numbers
        if x < 0 and prev == '-':
            self.add(" ")

Notes about implementation choices
- math.copysign(1.0, x) < 0.0 detects negative zero reliably and is simple.
- You must return after handling negative zero so you don't fall into the integer branch that would convert it to 0.
- Place the negative-zero handling before the prev/space logic (or at least ensure you add the space when prev == '-') so you don't accidentally produce "--0.0".

Why this fix fixes the failing test
- For x == -0.0 the new early branch detects negative zero and emits "-0.0" (and inserts a space if previous char is '-'), so the test that expected "-0.0" will pass.
- Positive zero (0.0) still goes through the integer branch and prints "0" as before.
