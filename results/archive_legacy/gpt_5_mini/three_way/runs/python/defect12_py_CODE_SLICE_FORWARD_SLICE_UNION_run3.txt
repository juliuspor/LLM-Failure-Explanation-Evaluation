What happened (runtime symptom)
- The program raises IndexError("String index out of range: ...") from substring_java (the slice function) at slice_union line 32–34 when it calls return s[start:end]. That means substring_java was called with end > len(s) or start > len(s).

Where it happened in your code
- substring_java is called from abbreviate at these sites:
  - result = substring_java(text, 0, upper)
  - return substring_java(text, 0, upper) + ...
  - return substring_java(text, 0, index) + ...
- The slice function checks for invalid bounds and raises if start > len(s) or end > len(s) (slice_union lines 32–34).

Why it happened (root cause)
- The variable upper can end up greater than len(text) because of the order of the checks in abbreviate.
  1. You first do:
     if upper == -1 or upper > len(text):
         upper = len(text)
     (this caps upper to len(text) when upper is too large).
  2. Directly after you do:
     if upper < lower:
         upper = lower
     If lower > len(text), this assignment sets upper to lower, which can make upper > len(text) again.
- Later you call substring_java(text, 0, upper). If upper > len(text) at that moment, substring_java raises IndexError because it enforces end <= len(s).

Concrete example that triggers the failure
- text = "hello" (len(text) == 5)
- lower = 10
- upper = 3
  - First block: upper is not > len(text) and not -1, so upper remains 3.
  - Then if upper < lower: upper = lower → upper becomes 10.
  - substring_java(text, 0, upper) is called with end == 10 > 5 → IndexError.

Where to change (explicit steps)
1. Ensure lower cannot force upper above len(text) after you clamp upper. You can:
   - Clamp lower to len(text) before comparing to upper:
       lower = min(lower, len(text))
   - Or clamp upper again after assigning upper = lower:
       if upper > len(text): upper = len(text)
2. Or rework the bounds computation in one place to avoid the back-and-forth:
   - Example replacement (explicit, minimal change):
       if lower > len(text):
           lower = len(text)
       if upper == -1 or upper > len(text):
           upper = len(text)
       if upper < lower:
           upper = lower
   - Or compute in one expression:
       lower = min(lower, len(text))
       if upper == -1:
           upper = len(text)
       else:
           upper = min(upper, len(text))
       if upper < lower:
           upper = lower

3. Also validate lower before using it as a start index for index_of or for slices (if StringUtils.index_of doesn't handle a start > len safely).

Summary
- Root cause: the line if upper < lower: upper = lower can reintroduce upper > len(text) when lower > len(text), leading substring_java(text, 0, upper) to be called with end > len(s), which the slice function rejects.
- Fix by clamping lower and/or upper against len(text) in the correct order (examples given above).
