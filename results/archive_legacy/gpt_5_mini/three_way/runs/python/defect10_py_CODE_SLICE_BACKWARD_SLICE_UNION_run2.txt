What happened (short): create_number crashes with IndexError when called with a one-character input that ends in "l" or "L" (for example "l" or "L"). The code tries to read numeric[0] and numeric[1] without checking that numeric is long enough, and char_at(numeric, 0) raises the IndexError.

Where it happens (precise):
- In create_number, when last_char in ("l", "L") the code sets numeric = val[:-1] and then immediately evaluates
  if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
- If val is "l" or "L", then numeric == "" (empty string). char_at(numeric, 0) calls the helper char_at which checks index bounds and raises IndexError at line 31:
  if index < 0 or index >= len(s):
      raise IndexError(f"String index out of range: {index}")
- So char_at(numeric, 0) raises IndexError("String index out of range: 0") instead of create_number returning or raising a ValueError.

Why it fails (root cause):
- The code assumes numeric has at least one character before doing char_at(numeric, 0) and numeric[1:], but it does not check len(numeric).
- For a single-character val like "l", numeric is empty and the access is out-of-bounds, producing an unexpected IndexError rather than the intended ValueError.

Reproducible example:
- create_number("l") -> IndexError from char_at at the bound-check line.
- create_number("L") behaves the same.

How to fix it (concrete steps you can apply):
1. Add a guard that checks numeric is non-empty before using char_at or numeric[1:].
   Example replacement for the l/L branch:
   if last_char in ("l", "L"):
       if len(numeric) == 0:
           raise ValueError(f"{val} is not a valid number.")
       if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
           return int(numeric)
       raise ValueError(f"{val} is not a valid number.")

2. Alternatively, check val length earlier (before stripping qualifier):
   if last_char in ("l","L") and len(val) == 1:
       raise ValueError(f"{val} is not a valid number.")

3. Also avoid indexing numeric[1] directly; use numeric[1:] as you already do but only after confirming len(numeric) >= 2 when you depend on numeric[1].

Any of these changes prevents the call to char_at on an empty string and makes the function raise a ValueError for inputs like "l" instead of crashing with IndexError.
