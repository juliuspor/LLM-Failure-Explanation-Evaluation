- The problematic lines are 402 and 404:

  - line 404: minutes_offset = hours_in_minutes + minutes_offset
- Those two assignments are reversed relative to the intended sign logic. As written they cause the wrong sign to be applied to minutes when hours is positive vs negative, producing an invalid minute value (e.g. -15) and later a ValueError.

Why that causes the specific error
- The docstring requires that when hours is positive you cannot have a negative minutes value (e.g. (2, -15) is invalid). The code is supposed to build a total offset in minutes from hours_in_minutes (hours * 60) and minutes_offset (0..59).
- Because lines 402/404 are swapped, for a call like (hours_offset = 2, minutes_offset = 15) the code may execute the wrong branch and compute minutes_offset = hours_in_minutes - minutes_offset (or the equivalent wrong sign), producing a negative minute component (for some paths) or a final minutes value that violates the allowed range, which triggers the ValueError you observed.
- The code later multiplies minutes_offset by MILLIS_PER_MINUTE and checks size (line 405 and 407). The earlier incorrect sign leads to an invalid intermediate minutes value and thus the ValueError.
