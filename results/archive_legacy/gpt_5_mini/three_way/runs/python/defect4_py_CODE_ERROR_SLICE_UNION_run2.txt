What happened (root cause)
- In the "max middle" block you fetch the wrong data item. Instead of using self._max_middle_index you use self._min_middle_index:
  - Code: max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  - Location: the max-middle block (the fourth block, just after the min-middle block). In your slice this is the block starting at "if self._max_middle_index >= 0:".
- Because you read the min-middle period when you meant to read the max-middle period, you compute max_middle from the wrong period. That lets the wrong index win and causes self._max_middle_index to be set incorrectly (which explains the failing assertion Expected max_middle_index=1, got 3).

Why this produces the observed failure
- If _min_middle_index and _max_middle_index point to different items, comparing the new period's middle against min_middle (instead of the true max_middle) can wrongly decide the new period is the new maximum and update _max_middle_index to index 3 rather than the expected 1.

- Replace the incorrect reference to self._min_middle_index with self._max_middle_index in the max-middle block.
- Change this line:

  max_middle_period = self.get_data_item(self._max_middle_index).get_period()
