What happened (what failed)
- The test calling WordUtils.abbreviate("0123456789", 15, 20, None) causes substring_java(text, 0, upper) to be called with upper == 15 while len(text) == 10.
- substring_java then checks the slice bounds and raises IndexError("String index out of range: 15"). The slice check is in slice_block lines 32–33:
  if start < 0 or end < 0 or start > end or start > len(s) or end > len(s):
      raise IndexError(f"String index out of range: {end}")

Why that happened (root cause)
- In abbreviate() you try to clamp upper to the string length, then afterwards you possibly overwrite that clamp with lower:
  1. if upper == -1 or upper > len(text): upper = len(text)   <-- clamps upper to  string length
  2. if upper < lower: upper = lower                         <-- may set upper to lower
- For the failing test: len(text) == 10, lower == 15, upper == 20
  - Step 1 sets upper = 10
  - Step 2 sees upper < lower and sets upper = 15
- That leaves upper > len(text), so later substring_java(text, 0, upper) is called with end = 15 and raises the IndexError shown in the slice_block.

Where it occurred (specific lines / calls)
- The problematic assignment ordering is in your abbreviate() function:
  - line with if upper == -1 or upper > len(text): upper = len(text)
  - immediately after the line if upper < lower: upper = lower
- The exception is raised inside substring_java(...) (the slice bounds check in slice_block lines 32–33).

How to fix it (concrete steps)
Choose one of the following precise fixes:

Option A — Clamp lower before comparing to upper (recommended)
- If lower can be beyond text length, clamp it to len(text) first. Then apply the existing logic.
  Example change (pseudocode):
    if lower > len(text):
        lower = len(text)
    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower

Option B — Reorder the upper clamp so upper is clamped after you adjust it to lower
- Move the upper-clamping line to after the upper < lower check:
    if upper < lower:
        upper = lower
    if upper == -1 or upper > len(text):
        upper = len(text)

Either option ensures you never call substring_java with end > len(text).
