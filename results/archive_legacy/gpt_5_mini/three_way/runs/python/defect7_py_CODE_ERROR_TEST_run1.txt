What happened (the direct cause)
- In to_class, the line `_ = array[i].upper()` (inside the for loop) tries to call .upper() on each array element.
- When array[i] is None, Python raises AttributeError: 'NoneType' object has no attribute 'upper' because None has no .upper() method.
- That exception occurs before you append type(array[i]) to classes, so the function never reaches the intended behavior for None elements.

Where it occurs (exact variables/lines)
- Method: to_class
- Loop: for i in range(len(array)):
- Faulty line: `_ = array[i].upper()` — this is the line that throws when array[i] is None.
- Next line (the intended work): `classes.append(type(array[i]))` — never reached for None elements because of the previous call.

Why that line is there is irrelevant to the function’s goal
- The function’s job is to return the types of the elements. Calling .upper() is unnecessary and unsafe because not all elements are strings (and some may be None). It looks like a leftover or mistaken attempt to normalize strings, but it breaks handling of None and non-string types.

What the tests expect
- When an element is None, the function should append type(None) (i.e., type(None) or NoneType) to the result array. The current code raises before that happens.

How to fix it (concrete steps)
1. Remove the `.upper()` call (or guard it behind a type check) because it’s not required for determining an element’s type.
2. Iterate directly over elements (for element in array) for clarity, then append type(element) — this handles None correctly because type(None) works.
3. If you do need to normalize string elements, check isinstance(element, str) before calling .upper().

Corrected implementation (drop-in replacement)
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()

        classes = []
        for element in array:
            # do not call element.upper() unguarded — it raises on None or non-strings
            classes.append(type(element))
        return classes

Alternative if you want to uppercase strings before computing type (not necessary for type detection):
    for element in array:
        if isinstance(element, str):
            _ = element.upper()   # safe because element is a str
        classes.append(type(element))

Summary
- Root cause: unguarded call to .upper() on each array element causes AttributeError when an element is None (or any non-string).
- Fix: remove or guard that call and use type(element) to build the result list (handles None as type(None)).
