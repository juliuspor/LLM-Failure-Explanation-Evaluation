What happened (where it fails)
- The test passed x = -0.0 into add_number.
- At the top you check x < 0 (line: if x < 0 and prev == '-':). For -0.0, x < 0 is False in Python, so that branch does not run.
- Next you do if x == int(x) ... and then value = int(x). For x == -0.0 both x == int(x) is True and int(x) becomes 0 (losing the sign bit).
- You then call self.add(str(value)) (the last branch for integer-valued floats), so you add "0" — the negative sign is lost and the test expected "-0.0".

Why it happens (root cause)
- Negative zero is equal to positive zero by IEEE and Python float comparisons, so x < 0 and x == int(x) behave as if x were +0.0.
- Converting to int discards the sign bit (int(-0.0) -> 0), so later str(value) can't produce "-0.0".
- The code never checks "is negative zero" before converting to integer/string, so it prints "0" instead of "-0.0".

Exactly which lines:
- Line with if x < 0 and prev == '-': — this test does not detect -0.0.
- Line value = int(x) — converting -0.0 to int loses the sign.
- Line self.add(str(value)) — prints "0" because value is 0.

How to fix it (clear steps)
1. Detect negative zero before converting to int. Use math.copysign to test the sign bit:
   - is_neg_zero = (x == 0.0 and math.copysign(1.0, x) < 0.0)
2. If is_neg_zero, add the desired representation directly (e.g. "-0.0") and return, or ensure you preserve the sign through later branches.
3. Use that same sign test for the earlier spacing logic instead of x < 0 so -0.0 is handled the same as other negative numbers.

Concrete replacement sketch
- Add near the top, before the x<int and int conversion branches:
    is_neg_zero = (x == 0.0 and math.copysign(1.0, x) < 0.0)
    if is_neg_zero:
        # if you want to print as "-0.0" exactly:
        self.add("-0.0")
        return
    # Replace `if x < 0` with `if x < 0 or is_neg_zero` for the spacing logic.

Alternative: instead of an early return, incorporate is_neg_zero into the integer branch so you don't convert to int for -0.0 and instead format "-0.0".

This will preserve the negative zero sign and make the test pass.
