What happened
- The code crashes when search_list or replacement_list contains None because the function calls len(...) on those list entries before it ever checks for None.
- Specifically, in the growth-estimation loop at lines 81–85 the code does:

  That raises TypeError when either replacement_list[i] or search_list[i] is None, so the function crashes there.
- Later in the code there are checks that try to ignore None entries (lines 102–105), but they run after the length-based calculation. So the safety checks come too late to prevent the earlier crash.

Why this is the root cause
- The code assumes entries are strings and calls len(...) on them. None has no length, so len(None) raises a TypeError. The later guard that would skip None entries exists but is executed after the offending len(...) call, therefore it does not prevent the crash.
