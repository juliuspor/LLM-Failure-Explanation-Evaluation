What happened (root cause)
- The failure occurs on this line inside to_class:
      _ = array[i].upper()
  When the input array contains a None element (test_array = ["Test", None, 99.0]), array[1] is None. Calling .upper() on None raises AttributeError: 'NoneType' object has no attribute 'upper'.
- That line is unnecessary for the function’s purpose. The method should return the element classes (type(array[i])), but the stray call to .upper() attempts to treat every element as a string before computing its type.

Where it occurs
- In the for loop:
      for i in range(len(array)):
          _ = array[i].upper()        <-- causes AttributeError when array[i] is None
          classes.append(type(array[i]))
- Specifically, the error happens when i == 1 for the test input ["Test", None, 99.0].

Why this was reached
- The test includes None elements intentionally. The code does not check for None before calling .upper(), so execution reaches the .upper() call with a None value.

How to fix (concrete steps)
1. Remove the unnecessary call to .upper().
2. Compute the type directly and handle None explicitly if you want the literal type(None) for None elements.
3. You can also simplify the loop with a direct iteration.

Corrected implementation (one clear option):

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    
        classes = []
        for element in array:
            # handle None -> type(None)
            if element is None:
                classes.append(type(None))
            else:
                classes.append(type(element))
        return classes

Alternate compact version:

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        if not array:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        return [type(x) if x is not None else type(None) for x in array]

Why these fixes work
- They avoid calling string methods on non-string values and explicitly handle None by returning type(None), matching the test expectations.

Summary
- The AttributeError comes from the stray .upper() call when array contains None. Remove or guard that call and return types directly (with a None check) to fix the failing tests.
