What happened (where it failed)
- At line 176 the code tests if x == int(x). For x == -0.0 this is true because int(-0.0) == 0 and -0.0 == 0.0.
- At line 177 value = int(x) sets value to 0 (losing the negative zero sign).
- Later, at line 192 self.add(str(value)) adds the string "0".
- The test expected "-0.0", so the code produced the wrong sign for zero.

Why that happens (root cause)
- Python treats -0.0 as equal to 0.0 for numeric equality and int(-0.0) → 0. Converting to int removes the sign bit of negative zero.
- The equality check x == int(x) at line 176 therefore collapses -0.0 into the integer branch and loses the sign before any string formatting, so the final output becomes "0" instead of "-0.0".

Exactly where to fix it
- You must detect negative zero before you convert x to int or otherwise branch on integerness. A reliable, simple check is:
  - x == 0.0 and math.copysign(1.0, x) < 0.0
  - (Alternatively 1.0/x == float("-inf"), but math.copysign is clearer.)
