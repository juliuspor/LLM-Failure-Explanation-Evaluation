What happened (root cause)
- The TypeError comes from this loop (lines ~81–85):
  > for i in range(search_length):  
  >     greater = len(replacement_list[i]) - len(search_list[i])  
  >     if greater > 0:  
  >         increase += 3 * greater

- That loop calls len(...) on search_list[i] and replacement_list[i] without checking whether either element is None. If any element in search_list or replacement_list is None, len(None) raises TypeError: "object of type 'NoneType' has no len()".

Why it happens here and not elsewhere
- Earlier and later in the function you do check for None and empty strings (for example in the match-finding loops at lines ~28–49 and ~102–113 you have conditions that skip elements when search_list[i] is None or replacement_list[i] is None).
- But the sizing loop runs before those checks. The comment even notes the bug: "# BUG: this loop is not null-safe for elements inside the arrays". So the code assumes every element is a str for that loop, but the inputs may contain None, causing the crash.

Exactly where to look
- Faulty loop: lines 81–85 (the place that computes "increase").
- Error happens when either replacement_list[i] or search_list[i] is None for any i in 0..search_length-1.

Concrete fixes (step-by-step)
1. Skip None or empty search entries in that loop, same as elsewhere. Replace the loop with a null-safe version:
   - Example replacement:
     for i in range(search_length):
         if search_list[i] is None or replacement_list[i] is None or len(search_list[i]) == 0:
             continue
         greater = len(replacement_list[i]) - len(search_list[i])
         if greater > 0:
             increase += 3 * greater

2. Alternatively, compute safe lengths with conditional expressions:
   - Example:
     for i in range(search_length):
         s_len = len(search_list[i]) if isinstance(search_list[i], str) else 0
         r_len = len(replacement_list[i]) if isinstance(replacement_list[i], str) else 0
         greater = r_len - s_len
         if greater > 0:
             increase += 3 * greater

3. (Optional) Move the sizing loop to after the initial scan that already filters valid entries. That way you can build a filtered list of valid (search, replacement) pairs and size only those.

Any of the above ensures you never call len(None), preventing the TypeError and making the size estimation null-safe.
