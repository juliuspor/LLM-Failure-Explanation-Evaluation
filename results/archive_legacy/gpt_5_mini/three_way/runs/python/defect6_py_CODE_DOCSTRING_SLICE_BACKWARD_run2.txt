What happened (where it fails)
- The failure occurs at line 176: this expression is evaluated left-to-right:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- Python evaluates x == int(x) first, so int(x) is called before the checks for infinity/NaN.
- If x is infinity or NaN (or in some cases an out-of-range float), int(x) raises an exception (OverflowError for inf, ValueError for NaN). That is the runtime error you see coming from this line.

Why it fails (root cause)
- The code tries to convert x to an int before verifying that x is a finite, normal float.
- The intended safety checks (not math.isinf(x) and not math.isnan(x)) come after the int() call, so they don’t prevent the conversion when x is invalid for int().

Exact location and variables

- The problematic call is int(x). The checks math.isinf(x) and math.isnan(x) are placed after that call and therefore don’t protect it.

How to fix it (concrete steps)
1. Check finiteness before calling int(x). Replace the condition with one that tests math.isfinite(x) (or checks isinf/isnan first). For example:

   - math.isfinite(x) returns False for inf and NaN, so int(x) will only be executed for finite numbers.

2. Alternatively, reorder the original checks:

   - This also prevents int(x) from being called for inf/NaN because the earlier checks short-circuit.

3. If you want to be defensive against unexpected conversion errors, you can cast inside a try/except:
