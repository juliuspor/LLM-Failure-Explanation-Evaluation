What happened (root cause)
- The code calls len(...) on items inside search_list and replacement_list before it ever checks whether those items are None.
- The tests pass None as a replacement element (replacement_list = [None] or ["c", None]). When the code reaches a len(...) call for that None element it raises TypeError: object of type 'NoneType' has no len().

Where it occurs (exact places)
- At line 82: the expression len(replacement_list[i]) - len(search_list[i]) will raise if replacement_list[i] or search_list[i] is None.
- At line 104: the condition includes len(search_list[i]) == 0 but the code only checks search_list[i] is None later in the same boolean expression (so if search_list[i] is None the len(...) already crashed).
- Any other len(...) calls on search_list[i] or replacement_list[i] will fail the same way unless those entries are checked for None first.

Why it fails (step-by-step)
1. The test passes replacement_list containing None.
2. The code attempts to compute size differences to estimate buffer growth (lines ~81–85), using len(replacement_list[i]) and len(search_list[i]) for every i.
3. Because replacement_list[i] is None, len(replacement_list[i]) raises TypeError immediately.
4. The later logic that intends to treat None as “no replacement” (checks like replacement_list[i] is None) never gets a chance to run because the code already crashed on len(None).

Concrete fixes (clear actionable steps)
1. Never call len(...) on search_list[i] or replacement_list[i] without first checking for None. For example:
   - Change the growth-estimate loop (around lines 81–85) to skip entries where search_list[i] is None or replacement_list[i] is None.
   - Example logic:
     - if search_list[i] is None or replacement_list[i] is None: continue
     - else compute greater = len(replacement_list[i]) - len(search_list[i])
2. In the match loop (around lines 102–113) evaluate None checks before any len(...) calls. Replace expressions like:
     or len(search_list[i]) == 0
   with:
     or search_list[i] is None
     or len(search_list[i]) == 0
   (so the len(...) is only executed when search_list[i] is not None).
   Or safer: compute slen = 0 if search_list[i] is None else len(search_list[i]) and use slen.
3. Consider normalizing inputs at the start of replace_each:
   - If search_list or replacement_list is None, treat as empty lists.
   - Ensure search_list and replacement_list are same length (or handle mismatched lengths).
   - Optionally convert None elements to a sentinel and document that None means “no replacement”.

Why this fix is correct

Summary
The TypeError arises because the code calls len(...) on list elements that can be None before it checks for None. Move/insert None checks before any len(...) calls (or guard lengths with conditional expressions) in the growth-estimate loop and the match loop (lines ~81–85 and ~102–104) to fix the failure.
