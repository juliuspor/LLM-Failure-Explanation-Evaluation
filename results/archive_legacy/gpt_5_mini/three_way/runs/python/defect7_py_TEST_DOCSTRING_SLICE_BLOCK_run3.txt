What happened (where, exactly)
- The failing code is in hit07_classutils.py at lines shown in your slice:
  - Line 765: start of the loop for i in range(len(array)):
  - Line 766: it does _ = array[i].upper()
  - Line 767: classes.append(type(array[i]))
- When an element of array is None, line 766 attempts to call None.upper(), which raises:
  AttributeError: 'NoneType' object has no attribute 'upper'
- That exception causes tests like test_to_class_with_none_element and test_to_class_all_none to fail.

Why it happens (root cause)
- The code calls array[i].upper() unconditionally. That assumes every element is a string (or at least has .upper()). None has no .upper() method, so accessing array[i].upper() on a None element raises AttributeError.
- The docstring and your tests indicate the method should handle None elements (either by inserting a None entry or type(None) into the result). Instead, the code treats None as if it were a string and thus fails before ever appending anything.

Concrete fixes (steps you can apply)
1. Decide intended behavior for None elements:
   - If the method should return type(None) for None inputs (your tests expect type(None)), use type(None).
   - If the method should return None elements in the classes array, append None instead.

2. Replace the loop body around lines 765–767 with a safe check. Example that matches your tests (append type(None) for None elements):
   - for i in range(len(array)):
       if array[i] is None:
           classes.append(type(None))
       else:
           classes.append(type(array[i]))

   Or more Pythonic:
   - for obj in array:
       classes.append(type(obj) if obj is not None else type(None))

3. Remove the unnecessary call to .upper(). That call appears to be a leftover assumption that every element is a string; it serves no purpose for producing class/type objects and must be deleted.

4. Ensure the method still returns None if input array is None:
   - if array is None:
       return None

5. Run the test suite to confirm the changes fix:
   - python -m unittest path.to.your.tests

Why this fix solves it
- Removing the unconditional .upper() call prevents AttributeError for None elements.
- Explicitly checking for None ensures the method returns the expected type(None) (or None) entry for each None input element.

Example corrected implementation (concise)
- def to_class(array):
    if array is None:
        return None
    classes = []
    for obj in array:
        classes.append(type(obj) if obj is not None else type(None))
    return classes
