What happens
- At line 181 you enter this loop:
  while mantissa % 10 == 0:
but nothing inside the loop changes mantissa (or exp). Because mantissa is set at line 178 to value and never reduced inside the loop, if mantissa ends in a zero the condition mantissa % 10 == 0 stays true forever -> an infinite loop at lines 181–183.

- There is a second related problem: the variable exp is referenced on line 182:
  check_val = (mantissa // 10) * (10 ** (exp + 1))
but exp is never initialized anywhere in the shown snippet. That will raise a NameError if the code execution reaches that line.

Where it occurs
- Infinite loop occurs starting at line 181 because mantissa remains unchanged.
- NameError (undefined exp) would occur at line 182 when trying to compute check_val, if execution reaches that point.

Why it fails (root cause)
- The loop's body does not modify the loop condition (mantissa) and uses an uninitialized variable exp. The intended algorithm clearly needs to divide the mantissa by 10 and track how many tens were removed (exp), but those steps are missing. Leaving out the mantissa update causes the loop never to terminate; leaving out exp initialization causes a runtime NameError when check_val is computed.

How to fix it (concrete steps)
1. Initialize exp before the loop, e.g. at line 179:
   exp = 0

2. In the loop (lines 181–183) update mantissa and exp so the condition can change. For example:
   while mantissa % 10 == 0:
       mantissa //= 10          # reduce mantissa by one power of 10
       exp += 1                 # record how many tens were removed
       check_val = mantissa * (10 ** exp)   # recompute check_val with current exp
       if check_val == value:
           break

3. Make sure check_val is computed consistently with how you update mantissa and exp. In the original code you used (mantissa // 10) * (10 ** (exp + 1)) — that is equivalent to recomputing after reducing mantissa and incrementing exp, but writing it as above is clearer and avoids off-by-one mistakes.

4. Add a safety guard to avoid accidental infinite loops (optional): limit the number of iterations, or break when mantissa becomes 0 or very small.

Example replacement for the problematic block (replace lines 178–183):
   mantissa = value
   exp = 0
   if abs(x) >= 100:
       while mantissa % 10 == 0 and mantissa != 0:
           mantissa //= 10
           exp += 1

           if check_val == value:
               break

This addresses both the infinite-loop and the undefined-variable errors by explicitly changing mantissa and initializing exp.
