What happens
- The line "_ = array[i].upper()" calls the string method upper() on every element in array.
- If any element is None (for example test_array = ["Test", None, 99.0]) then array[i] is None for that index and None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- That exception occurs before you ever reach the classes.append(...) line, so the function crashes on arrays that contain None.

Where it occurs (exact code/variable)
- In method to_class(cls, array: Optional[List[Any]]).
- The problematic statement is: _ = array[i].upper() (inside the for loop).
- The loop then does classes.append(type(array[i])) — that append is correct, but it never runs for None elements because of the .upper() call.

Why it was failing (root cause)
- The code assumes every element has an upper() method (i.e. is a string). None does not; calling .upper() on None raises an AttributeError.
- The call to .upper() is unnecessary for determining a value's class/type; only type(array[i]) is needed.

How to fix (concrete steps)
1. Remove the unnecessary call to .upper().
2. Iterate directly over elements instead of indexing (cleaner).
3. Handle None naturally by using type(element) (type(None) is the correct result).

Fixed implementation example:
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for elem in array:
            classes.append(type(elem))
        return classes

Notes
- This returns type(None) for None elements, matching your tests.
- The original .upper() looks like an accidental leftover (debug or string-specific logic) and should be removed.
