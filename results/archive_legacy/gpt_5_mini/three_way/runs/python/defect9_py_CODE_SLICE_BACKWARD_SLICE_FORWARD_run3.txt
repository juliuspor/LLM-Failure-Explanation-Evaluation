What happens
- The function crashes in the "guess result buffer size" loop at line 82:
  greater = len(replacement_list[i]) - len(search_list[i])
- If any search_list[i] or replacement_list[i] is None (or search_list[i] is an empty string? — empty string is handled elsewhere), calling len(...) raises
  TypeError: object of type 'NoneType' has no len()

Why it happens (root cause)
- Earlier you intentionally skip null/empty search strings and null replacements when scanning for matches (lines 59–73 and again later at lines 102–107).
- But the pre-sizing loop (lines 81–85) does not apply those same guards. It unconditionally calls len() on both search_list[i] and replacement_list[i].
- If either element is None, len(None) raises the TypeError and the function fails before any replacements happen.

Exactly where (file/lines/variables)
- Faulty line: line 82 (in the code slice), variable uses:
  - replacement_list[i] (may be None)
  - search_list[i] (may be None)
- Surrounding code: the loop beginning at line 81:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater

How to reproduce quickly
- Call replace_each("abc", ["a", None], ["x", "y"]) or replace_each("abc", ["a", None], ["x", None]) — function will raise TypeError at the sizing loop.

Exact fixes (step-by-step, code-level)
Option A — skip None/empty entries (recommended, consistent with other loops)
- Replace the sizing loop with the same guard used elsewhere:

    for i in range(search_length):
        if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater

- This mirrors the match-finding guards (lines 59–73 and 102–107) and prevents len() on None.

Option B — treat None as zero-length safely (alternative)
- If you prefer to tolerate None and treat it like empty (not recommended because the rest of the code treats None as "skip"), use safe lengths:

    def safe_len(s):
        return len(s) if s is not None else 0

    for i in range(search_length):
        greater = safe_len(replacement_list[i]) - safe_len(search_list[i])
        if greater > 0:
            increase += 3 * greater

Tests to add
- Unit tests where search_list or replacement_list contain None elements (and at least one valid pair), e.g.:
  - search_list = ["a", None], replacement_list = ["b", None]
  - search_list = [None], replacement_list = [None]
  - ensure no TypeError is raised and behavior matches intent (None elements are skipped)

Summary
- The failure is a simple null-safety bug at the sizing loop (line 82): len(...) is called on elements that can be None. Fix by applying the same null/empty/replacement None guard used elsewhere (or use safe_len) so the loop never calls len() on None.
