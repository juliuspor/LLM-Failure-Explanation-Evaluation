What happens
- In abbreviate() you first do:
  - if upper == -1 or upper > len(text): upper = len(text)
  - if upper < lower: upper = lower
- For the failing test: text length = 10, lower = 15, upper = 20.
  - The first if sets upper = len(text) => upper = 10.
  - The next if sees upper < lower (10 < 15) and sets upper = lower => upper = 15.
- Later you call substring_java(text, 0, upper). That calls the slice validation in slice_union (lines 32–34), which raises IndexError because end (15) > len(s) (10).

Root cause
- You moved upper toward lower without ensuring upper remains <= len(text). As a result upper can become greater than the string length and you pass an out-of-range end to substring_java, which raises IndexError (see slice_union lines 32–34).

Where it occurs (exact lines/variables)
- Problem lines in your code:
  - "if upper == -1 or upper > len(text): upper = len(text)"
  - "if upper < lower: upper = lower"
  - then "result = substring_java(text, 0, upper)" (or other substring_java calls using upper)
- The exception originates in substring_java’s validation (slice_union lines 32–34), which rejects end > len(s).

How to fix (clear steps)
Pick one of these fixes depending on the intended behavior:

1) If lower beyond the string should return the original string (matches the test):
   - Add an early check after the Null/empty checks:
     if lower >= len(text):
         return text
   - This keeps the method returning the full string when lower is beyond the length.

2) If you want to clamp upper into [0, len(text)] after any adjustment:
   - After the existing adjustments, ensure:
     if upper > len(text):
         upper = len(text)
   - That prevents passing an out-of-range end to substring_java.

3) Alternatively combine safely:
   - Set upper = len(text) if upper == -1
   - Set lower = min(lower, len(text))
   - Set upper = min(upper, max(lower, len(text))) — but simpler is option 1 or 2 above.

Concrete minimal change to pass the shown test (apply right after the two existing ifs):
    if upper < lower:
        upper = lower
    if lower >= len(text):
        return text
    if upper > len(text):
        upper = len(text)

This ensures you never call substring_java with end > len(text) and satisfies the test case.
