Why it happens (root cause)
- The method does not clamp the input value (or the computed g) to the allowed range. The computation at line 71:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  produces values outside 0..255 for value < lower_bound or value > upper_bound.
- Immediately after, lines 74–75 raise an exception for out-of-range g instead of clamping and returning a valid RGB tuple. Tests expect clamping behavior but code raises an error.
- Minor secondary issue: the ValueError message ("Color parameter outside of expected range: Red Green Blue") does not include the offending value g, so it's unhelpful for debugging.
