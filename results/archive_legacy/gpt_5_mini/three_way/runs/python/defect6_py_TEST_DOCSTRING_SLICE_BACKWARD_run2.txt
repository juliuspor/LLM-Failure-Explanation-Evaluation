What happened (where and what exactly failed)
- The failure happens in CodeConsumer.add_number when it decides how to format a float-looking value stored in variable x.
- At the decision on line 176:

  the code treats any value equal to an integer as an integer and emits "0", "42", "-42", etc.
- For x == -0.0 this condition is true (because -0.0 == 0 and int(-0.0) == 0). The code therefore takes the integer branch and outputs "0", dropping the negative sign.
- Your test test_negative_zero expects "-0.0" but got "0". That is why the test fails.

Why this happens (root cause)
- The integer-format branch runs before checking for negative zero. Negative zero is numerically equal to zero, so x == int(x) is true and the code never uses the negative-zero preservation logic (the is_negative_zero utility or any sign-aware handling).
- In short: the code's ordering/condition lets -0.0 be handled as a plain integer zero, losing the sign.
