What happened (where it fails)
- The failure occurs inside LocaleUtils.to_locale when parsing strings like "fr__POSIX".
- In the code at lines 239–240 you have:
  if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
      raise ValueError(f"Invalid locale format: {locale_str}")
- For input "fr__POSIX", the character at the position corresponding to ch3 is '_' (an underscore). The condition therefore rejects the input and raises ValueError with the message "Invalid locale format: fr__POSIX".

Why it fails (root cause)
- The parser is enforcing that the two characters for the country component must be uppercase letters unconditionally.
- It never allows the special valid format language__variant (language + empty country + variant), which is represented by a double underscore. In that format the first of those two country-character positions is '_' and the country is empty.
- Because the code checks ch3 and ch4 for being 'A'..'Z' even when the country is intentionally empty (identified by the underscore separator), a valid input like "fr__POSIX" is incorrectly classified as invalid.

Which variables and lines are involved
- locale_str: the input string ("fr__POSIX")
- ch3 and ch4: the code variables used to validate the two country-character positions
- Lines 239–240: the exact check that unconditionally rejects non-uppercase characters in those positions

How to fix it (explicit steps)
1. Detect the empty-country case (double underscore) before validating country characters:
   - If locale_str[2] == '_' (i.e., the separator at the country start is an underscore), treat country as empty and parse the variant from locale_str[3:].
2. Only validate ch3/ch4 as uppercase letters when you know a country code is present (i.e., when locale_str[2] is not an underscore).
3. Update the logic so you do not perform the unconditional check at lines 239–240.

Suggested minimal patch (showing the exact change)
Replace the unconditional uppercase check at lines 239–240 with a conditional branch:

- Current (problematic):
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(f"Invalid locale format: {locale_str}")

- Replace with:
    # If there's a double underscore at position 2, country is empty and variant starts at 3
    if locale_str[2] == '_':
        # valid language__variant form: no uppercase country check
        country = ""
        variant = locale_str[3:]
    else:
        # country present: ensure two uppercase letters
        if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
            raise ValueError(f"Invalid locale format: {locale_str}")
        country = locale_str[2:4]
        variant = locale_str[5:] if len(locale_str) > 4 and locale_str[4] == '_' else ""
