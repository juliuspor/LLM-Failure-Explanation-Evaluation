What happened (where it failed)
- The test calls ClassUtils.to_class(["Test", None, 99.0]).
- In the implementation, the loop at lines 765–767 does this:
  - line 766: _ = array[i].upper()
  - line 767: classes.append(type(array[i]))
- When i == 1, array[i] is None. Calling None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- So the failure occurs at line 766 inside ClassUtils.to_class when the code unconditionally calls .upper() on every element, including None.

Why it happened (root cause)
- The code assumes every element in the input array is a string (or at least has an .upper() method).
- The test includes None (and the suite also includes mixed-type inputs). None has no .upper(), so the unconditional call fails.
- In short: there is no null/None check before calling .upper(), causing the AttributeError.

How to fix it (concrete steps and code)
1. Don’t call .upper() on elements that can be None or non-strings.
2. If the intent was only to normalize string elements, call .upper() only when the element is a string.
3. For None elements, append type(None) (or otherwise treat them as None) without calling any methods on them.

Example replacement for the loop (replace lines 765–767):

    for i in range(len(array)):
        item = array[i]
        if item is None:
            classes.append(type(None))
        else:
            # If you need to normalize strings, only call .upper() for str
            if isinstance(item, str):
                _ = item.upper()   # safe, item is str
            classes.append(type(item))

Alternative minimal change:

    for i in range(len(array)):
        item = array[i]
        if hasattr(item, 'upper'):
            _ = item.upper()
        classes.append(type(item))

(But prefer isinstance(item, str) if the intent is specifically string normalization.)
