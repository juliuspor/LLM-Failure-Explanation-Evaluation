  - greater = len(replacement_list[i]) - len(search_list[i])
  This assumes both replacement_list[i] and search_list[i] are non-None and non-empty. If either is None this raises TypeError because len(None) is invalid.
- That early calculation runs before the later guard at lines 102–105 that is intended to ignore entries with None:
  - lines 102–105 check "or search_list[i] is None ... or replacement_list[i] is None" when searching for matches, but it's too late — the code already crashed at the length calculation.
- There are other related places that also assume non-None:

  - line 92: parts.append(replacement_list[replace_index]) will append None (or could index -1) if replace_index isn't validated.
