What happened (where it failed)
- The exception is raised on line 766: `_ = array[i].upper()`.
- The error message "'NoneType' object has no attribute 'upper'" means array[i] is None at the moment you call .upper(), so Python tries to call upper() on None and fails.

Why it happened (root cause)
- array contains at least one None element. On that iteration i, array[i] is None instead of a string, so calling array[i].upper() raises the AttributeError.
- Because line 767 then uses type(array[i]) and appends it to classes, you will also end up appending type(None) (NoneType) for that index if the code continued — but it doesn't get that far because line 766 already raised.

Concrete ways to fix it (exact steps you can apply)
1. Defend against None before calling .upper():
   - Replace line 766 with a guard:
     - if array[i] is None:
         raise TypeError(f"array[{i}] is None; expected str")
       else:
         _ = array[i].upper()
   - This makes the failure explicit and tells you which index contains None.

2. Or skip None values if they are expected:
   - if array[i] is None:
         continue
     _ = array[i].upper()

3. Or convert values to strings before calling upper() if None should be treated as empty string:
   - _ = (array[i] or "").upper()

4. Or ensure earlier in your code that array only contains strings:
   - Validate/populate array when you build it (filter out None or convert to str).
   - Example earlier-validation: array = [x for x in array if x is not None]

Which option to choose
- If a None in array is an unexpected bug, use option 1 so you get a clear error pointing to the bad index.
- If None is allowed and should be ignored, use option 2.
- If None should be treated as an empty string, use option 3.
- If the source of array can be fixed, prefer filtering/validation at the point you build array (prevents this class of errors later).

Summary
- The immediate cause: array[i] is None at line 766. Fix by checking for None, converting to string, or preventing None from entering array earlier.
