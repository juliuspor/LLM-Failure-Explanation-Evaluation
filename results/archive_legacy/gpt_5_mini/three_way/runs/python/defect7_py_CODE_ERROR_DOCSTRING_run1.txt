What happened (where and why)
- Failure occurs inside to_class(...) on the line:
    _ = array[i].upper()
  The test error shows AttributeError: 'NoneType' object has no attribute 'upper'. That means array[i] was None and the code attempted to call .upper() on it.
- The code checks only whether the whole input array is None (if array is None: return None) but does not check elements before calling .upper().
- After that line the code does:
    classes.append(type(array[i]))
  so the intent appears to be to produce a list of classes (types) corresponding to each element. The call to .upper() is unrelated to producing the type and causes the crash when an element is None.

Why this is the root cause
- Calling .upper() requires array[i] to be a str (or an object implementing upper). When array[i] is None, Python raises AttributeError because None has no .upper() method. The exception originates at the .upper() call, so the rest of the function never runs for that element.

Also note a docstring/behavior mismatch
- The docstring says: "If any of these objects is None, a None element will be inserted into the array." But the Raises section says "AttributeError: If any element is None". Those statements contradict each other. The code as written raises an AttributeError instead of inserting None, so either the implementation or the docstring is wrong.

Where to fix it (exact lines to change)
- Remove or guard the offending line:
    _ = array[i].upper()
  (this is the line that raised AttributeError).
- Modify the loop to handle None elements before using element methods. The important lines to change are:
    for i in range(len(array)):
        _ = array[i].upper()
        classes.append(type(array[i]))

Concrete fixes (pick one depending on intent)

1) If intent is to return types and allow None elements (match the docstring "insert None"):
- Remove the .upper() call and append None when element is None.
Example replacement for the loop:
    for i in range(len(array)):
        elem = array[i]
        if elem is None:
            classes.append(None)
        else:
            classes.append(type(elem))
- This will avoid calling any methods on None and will return a list where None elements remain None.

2) If intent was to normalize strings to upper-case before using their type (unlikely useful, but if needed):
- Only call .upper() for strings and ignore it for None:
    for i in range(len(array)):
        elem = array[i]
        if elem is None:
            classes.append(None)   # or raise if you actually want to fail
        else:
            if isinstance(elem, str):
                _ = elem.upper()
            classes.append(type(elem))

3) If None should cause an exception (match the Raises section):
- Explicitly raise a clear error when an element is None, instead of letting .upper() trigger an AttributeError:
    for i in range(len(array)):
        elem = array[i]
        if elem is None:
            raise AttributeError(f"Element at index {i} is None")
        classes.append(type(elem))

Other minor notes
- ArrayUtils.EMPTY_CLASS_ARRAY.copy() may or may not be appropriate depending on what type you expect to return; confirm it returns a List[Type] (not an array-like object) and that .copy() gives the desired empty list.
- Consider iterating directly over array (for i, elem in enumerate(array):) to simplify code and give better error messages with the index.

Summary
- Root cause: the code calls array[i].upper() without checking for None, causing AttributeError when an element is None.
- Fix: remove or guard the .upper() call and decide the function contract (insert None, or raise). Update implementation and docstring to be consistent.
