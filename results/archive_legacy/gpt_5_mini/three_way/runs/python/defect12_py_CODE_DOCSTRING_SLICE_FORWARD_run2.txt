What happened (what failed)
- With input text = "0123456789" (length 10), lower = 15 and upper = 20, the function runs into substring_java(text, 0, upper) with end = 15 (or greater than len(text)). substring_java slices with s[start:end] (see SLICE_FORWARD line 34), and Python raises IndexError when end > len(s). That is the crash you saw.

Why it happened (root cause)
- After these lines:
  - if upper == -1 or upper > len(text): upper = len(text)   # sets upper -> 10
  - if upper < lower: upper = lower                         # sets upper -> 15
  The second line blindly sets upper to lower even when lower > len(text). That puts upper > len(text) again, undoing the earlier clamping and causing the later substring_java(text, 0, upper) slice to request an out-of-range end index. The problematic assignment is the "if upper < lower: upper = lower" line.

Where it occurs (exact variables/lines)
- The problem occurs after upper is clamped to len(text) and then possibly overwritten:
  - variable: upper (set to len(text) then re-set to lower)
  - variable: lower (can be > len(text))
  - failing call: substring_java(text, 0, upper)
  - failing location: SLICE_FORWARD return s[start:end] (line 34) inside substring_java

How to fix it (concrete steps you can apply)
Option A — clamp upper to text length when you make it equal to lower:
- Replace
    if upper < lower:
        upper = lower
  with
    if upper < lower:
        upper = lower if lower <= len(text) else len(text)

Option B — clamp lower up-front so it never exceeds len(text):
- At top, after handling None/empty, add
    if lower > len(text):
        lower = len(text)
  This ensures later comparisons never push upper beyond len(text).

Option C — explicitly bound any slice parameters right before slicing:
- Just before any substring_java(text, 0, end) call ensure end = min(end, len(text)).

Any of these will prevent substring_java from receiving an end > len(text) and eliminate the IndexError. Option A is the smallest change that preserves the intended "upper should be at least lower" semantics while keeping upper within the string length.
