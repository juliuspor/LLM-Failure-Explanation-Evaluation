What happened (where it fails)
- At line 49 you set gap = end - start.
- At line 52 you immediately check if gap <= 0 and raise ValueError("bound must be positive").
- Later at line 56 you call _RANDOM.randrange(gap).

If the caller passes start == end, then gap == 0 and the code raises the ValueError at line 52. That is the exact failure you're seeing.

Why it happens (root cause)
- The code treats end as an exclusive bound (gap = end - start), and then uses randrange(gap) which picks values 0..gap-1 and adds start (start + randrange(gap)). That only works when end > start.
- If the caller expects end to be inclusive and passes start == end to request a single-character range, gap becomes 0, causing the error.
- If the caller passed end < start, gap is negative and you also get the same ValueError.
