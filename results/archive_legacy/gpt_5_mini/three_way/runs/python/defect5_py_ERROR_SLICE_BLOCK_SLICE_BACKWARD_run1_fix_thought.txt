The original add() incorrectly used type(array) (the list type) instead of inferring the element type, leading to inferred_type == object and unwanted TypeError when expected_type was provided. Fix: infer element type from the first non-None element of the array when array is given; if none found fall back to object. If element is provided and array is None, use its type. When expected_type is provided, only raise TypeError if there is a concrete inferred_type that conflicts with expected_type. If inferred_type is object and expected_type is given, accept it by setting inferred_type to expected_type so the cast check doesn't raise. Keep behavior of returning a new list with element appended.
