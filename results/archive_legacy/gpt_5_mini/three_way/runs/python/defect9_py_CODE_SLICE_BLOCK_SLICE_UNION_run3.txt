What happens
- The code crashes with a TypeError in the "guess result buffer size" loop at lines 81–84:
  > greater = len(replacement_list[i]) - len(search_list[i])

- The root cause: that loop calls len(...) on elements of search_list and replacement_list without checking whether those elements are None. Earlier in the function you explicitly allowed None entries (you check for None in the later match-finding loops), so len(None) raises:
  TypeError: object of type 'NoneType' has no len()

Where it occurs (specific variables and lines)
- In the increase-calculation loop:
  - Line 81: for i in range(search_length):
  - Line 82: greater = len(replacement_list[i]) - len(search_list[i])
  - If search_list[i] is None or replacement_list[i] is None, line 82 will throw the TypeError.

Why it happens (step-by-step)
1. The function accepts search_list and replacement_list that may contain None elements.
2. Later loops (lines ~59 and ~102) are defensive and skip entries when either search_list[i] is None or replacement_list[i] is None.
3. But the early buffer-size estimation loop (lines 81–84) does not check for None/empty entries and immediately calls len(...) on both elements.
4. If any element is None, len(None) throws the TypeError and the function fails before it ever does the safe matching.

Minimal reproducible example
- Call replace_each("abc", ["a", None], ["x", None]) — the function crashes at line 82 with TypeError.

Concrete fixes (clear steps)
Option A — skip null/empty entries the same way as the matching loops:
- Replace the loop at lines 81–84 with:

    for i in range(search_length):
        if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater

Option B — treat None as zero-length safely:
- Compute lengths with fallbacks to 0:

    for i in range(search_length):
        s_len = 0 if search_list[i] is None else len(search_list[i])
        r_len = 0 if replacement_list[i] is None else len(replacement_list[i])
        greater = r_len - s_len
        if greater > 0:
            increase += 3 * greater

Which to choose
- Option A matches the function’s existing behavior (it already ignores None/empty search strings and None replacements later), so use Option A to be consistent.

Summary
- The bug is the missing None checks when computing len(...) in the increase-estimation loop (lines 81–84). Add the same guards used elsewhere (skip when search_list[i] is None or empty, or replacement_list[i] is None) and the TypeError will be fixed.
