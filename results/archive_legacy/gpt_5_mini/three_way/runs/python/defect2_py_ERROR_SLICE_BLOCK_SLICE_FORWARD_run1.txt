What happened (root cause)

Why that's a problem (exact issues)
- The exception is raised because g is out of range. The check specifically tests g; it does not test r or b. So the root cause is an out-of-range value in g.

- There is no diagnostic information in the exception (no actual value of g, no calling context), so a developer cannot quickly see what input produced the bad g.
