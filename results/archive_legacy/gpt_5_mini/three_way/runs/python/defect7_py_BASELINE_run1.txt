What happened (root cause)
- In to_class, the loop at lines 765–767 does:
  - _ = array[i].upper()    # line 766
  - classes.append(type(array[i]))  # line 767
- When an element array[i] is None, calling array[i].upper() raises AttributeError: 'NoneType' object has no attribute 'upper'. The test that fails passes ["Test", None, 99.0], so the error occurs on the second element (i where array[i] is None).

Why that line is wrong
- The .upper() call assumes array[i] is a string (or at least has an upper method). The method's purpose (as implemented on line 767) is only to record the element's type with type(array[i]). There is no need to call .upper() at all.
- That unnecessary call causes the crash when any element is None (or any non-string without upper).

Where it occurs
- In to_class, inside the for loop at lines 765–767 (see the supplied slice). The offending statement is line 766: _ = array[i].upper().

Concrete fixes (step-by-step)
1. Remove the unnecessary .upper() call. Change the loop to only append the element type:
   for i in range(len(array)):
       classes.append(type(array[i]))

   Or more Pythonic:
   for elem in array:
       classes.append(type(elem))

2. (Optional) If you want to preserve the Java behavior of inserting None rather than type(None), decide between appending type(None) (current tests expect type(None)) or a literal None. The tests you provided expect type(None), so keep type(array[i]) for None elements.

3. Ensure the empty-array branch returns an empty list. Returning ArrayUtils.EMPTY_CLASS_ARRAY.copy() is fine if that constant is an empty list; otherwise returning [] is safe.

Minimal corrected code snippet
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        
        classes = []
        for elem in array:
            classes.append(type(elem))
        return classes

This will prevent AttributeError for None elements and satisfy the tests (e.g., [str, type(None), float]).
