What went wrong (root cause)
- At line 336 you set inferred_type = type(array). That sets inferred_type to the Python class of the list object itself (i.e. list), not the class of the elements stored in the list.
- Later you pass inferred_type into ArrayUtils._copy_list_grow1 and you compare it to object at line 346. _copy_list_grow1 and the subsequent logic expect a component/element type (e.g. int, str, or object), not the container type list. Because inferred_type is list, the code follows the wrong path and the type checks and casts fail.
- This mistake occurs whenever array is not None (even when array contains the desired element type). If array is None but element is given, you do the right thing (inferred_type = type(element), line 337). If both array and element are None you set inferred_type = object (line 340), which is fine.
