What failed
- The failure happens inside to_class() at these lines:
  - line 765: for i in range(len(array)):
  - line 766: _ = array[i].upper()
  - line 767: classes.append(type(array[i]))
- The immediate exception is an AttributeError raised on line 766: calling .upper() on array[i] when array[i] is not a str (for example an int, None, list, etc.). Non-string objects don’t have an upper() method, so Python raises "'X' object has no attribute 'upper'" and execution never reaches the append on line 767.

Why it happens (root cause)
- The code assumes every element of array is a string and blindly calls array[i].upper(). That assumption is false for inputs that contain non-strings (or None). The variable that actually triggers the error is array[i] at line 766.

Exact places to look
- Method: to_class(cls, array: Optional[List[Any]])
- Variables: array, array[i]
- Lines: 765–767 (the for loop and the .upper() call)
