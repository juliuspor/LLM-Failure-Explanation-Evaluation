  - Lines 177–178 set value = int(x) and mantissa = value.

- For x == -0.0:
  - Python compares -0.0 == int(-0.0) as True because int(-0.0) == 0 and -0.0 == 0.0.
  - value becomes 0 (losing the sign).
  - The code then takes the integer branch and does self.add(str(value)), producing "0".
- In short: the integer-branch test (x == int(x)) treats negative zero as an ordinary zero and the code emits str(value) == "0", so the negative sign is lost. This is exactly why the test expecting "-0.0" fails.
