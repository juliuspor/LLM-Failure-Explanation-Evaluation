What happened (root cause)
- In the block that updates _max_middle_index you compare the new period's middle against the wrong stored period. Instead of loading the period at self._max_middle_index you load the period at self._min_middle_index:
  - Wrong line: max_middle_period = self.get_data_item(self._min_middle_index).get_period()
  - It should be: max_middle_period = self.get_data_item(self._max_middle_index).get_period()
- Because you compare against the min-middle period rather than the current max-middle period, the comparison middle > max_middle is using the wrong baseline. That lets a later item overwrite the max index incorrectly (for your test it sets max_middle_index to 3 instead of the correct 1).

Where it occurs
- In _update_bounds, inside the "if self._max_middle_index >= 0:" branch (the block that computes max_middle), you fetch the stored period using self._min_middle_index instead of self._max_middle_index. This is the direct cause of the failing test test_max_middle_index_simple.

Why that produced the observed failure
- Example from your test:
  - Indices and middle values: 0->50, 1->150, 2->25, 3->75
  - After index 1 is added, _max_middle_index should be 1 (middle 150).
  - When index 3 (middle 75) is processed the code compares 75 to the middle of the period at self._min_middle_index (likely 25) instead of to the middle of the period at self._max_middle_index (150). Since 75 > 25 it sets _max_middle_index = 3 erroneously.

How to fix it (explicit steps)
1. In _update_bounds, find the max-middle block:
   if self._max_middle_index >= 0:
       max_middle_period = self.get_data_item(self._min_middle_index).get_period()
       ...
2. Replace self._min_middle_index with self._max_middle_index:
   max_middle_period = self.get_data_item(self._max_middle_index).get_period()
3. Run your test suite. The failing test should now pass.

Suggested additional hardening
- Add an assertion or unit test that exercises both min and max middle updates (you already have tests — keep them).
- Consider a small refactor to avoid copy-paste errors: extract a helper to compute start/middle/end for a stored index, e.g.
  - def _period_middle_for_index(self, idx): ...
  - then use it for min/max blocks so you only write the indexing logic once.
- Optionally add a regression test explicitly for the bug (you already included such a test: test_max_middle_index_simple).

Summary
- Bug: a wrong index variable (self._min_middle_index) is used when fetching the period for the max-middle comparison.
- Fix: use self._max_middle_index in that line. This will make _max_middle_index updates correct and the tests pass.
