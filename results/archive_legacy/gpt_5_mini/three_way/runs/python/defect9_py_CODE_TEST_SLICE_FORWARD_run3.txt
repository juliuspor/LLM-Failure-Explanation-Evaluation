What happened (what fails)
- The function raises an exception while computing the "increase" estimate at lines 80–85 (the loop that calculates increase).
- With the failing test inputs (e.g. search_list = ["a"], replacement_list = [None] or search_list = ["a", "b"], replacement_list = ["c", None]) the code executes:
  greater = len(replacement_list[i]) - len(search_list[i])
  When replacement_list[i] is None (or search_list[i] is None) this calls len(None) and raises TypeError ("object of type 'NoneType' has no len()" or unsupported operand types for subtraction).

Why this happens (root cause)
- Earlier in the code you correctly skip entries that are None or empty when searching/replacing (the for-loops that call text.find use checks like search_list[i] is None or replacement_list[i] is None).
- But the preliminary increase-calculation loop (commented "get a good guess on the size...") does not perform the same null/empty checks before calling len(...) on the array elements.
- So if any element in search_list or replacement_list is None (or possibly not a string), len(...) will fail there before any of the later null-safety logic runs.

Where it occurs (exact lines / variables)
- The problematic loop is the one starting just before line 83 in your slice:
    for i in range(search_length):
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater
- The offending expressions are len(replacement_list[i]) and len(search_list[i]) when replacement_list[i] or search_list[i] are None.
- This is why tests test_null_safety_inside_arrays_* fail.

How to fix (precise, actionable steps)
1. Skip entries that are None or empty in the increase loop exactly as you do later. Replace the loop with a guarded version:
    for i in range(search_length):
        if (
            search_list[i] is None
            or len(search_list[i]) == 0
            or replacement_list[i] is None
        ):
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater

2. Alternatively, compute safe lengths using conditional expressions (but the explicit guard above is clearer and matches the rest of the function's logic).

3. Keep the rest of the function the same — after adding this guard, tests that rely on skipping None replacements will proceed without the early TypeError.

Final note
- The comment in your code already states the bug: "# BUG: this loop is not null-safe for elements inside the arrays". The fix is to make that loop null-safe by checking search_list[i] and replacement_list[i] before calling len(...) on them.
