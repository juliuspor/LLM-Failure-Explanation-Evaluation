Why this fails (root cause)
- None is not a string and does not implement .upper(). The code unconditionally calls .upper() on every element of the input array, so any None (or any non-string that lacks upper) will cause an AttributeError.
- The intent of to_class is to return the element types (e.g., str, type(None), float). Calling .upper() is unnecessary for that purpose and is unsafe for non-string values.

Where in your tests it shows up
- test_to_class_with_none_element triggers the exception because it includes a None element.
- Other tests that include None elements (test_to_class_all_none, test_to_class_single_none) would also fail until this is fixed.
