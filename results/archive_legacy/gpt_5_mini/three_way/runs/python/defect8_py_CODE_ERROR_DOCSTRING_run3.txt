What happened (where it failed)
- The input "fr__POSIX" caused the ValueError at the check of ch3/ch4 (the block that starts with ch3 = locale_str[3]; ch4 = locale_str[4]; if ch3 < 'A' ...).
- In "fr__POSIX" the characters are: 0='f', 1='r', 2='_' (first separator), 3='_' (second separator), 4='P', ...
- Your code assumes that when a variant is present there is always a two-letter country code at positions 3 and 4. It therefore insists that locale_str[3] and locale_str[4] are uppercase A–Z.
- For "fr__POSIX" the country is intentionally empty (two consecutive underscores), so locale_str[3] is '_' (not an uppercase letter). The ch3/ch4 check rejects that and raises ValueError.

Why it failed (root cause)
- The method enforces a non-empty country when a variant is present. It does not allow the valid form where the country is empty and represented by two underscores (language + "_" + "" + "_" + variant).
- Because of that, ch3 is '_' and the uppercase check (ch3 < 'A' or ch3 > 'Z') becomes true and throws.
- There is a related indexing/substring issue: when the country is empty the variant does not start at the same index as when a country is present, so the code that slices the variant (locale_str[6:]) would also be wrong for the empty-country case.

Exactly where to change (concrete steps)
1. After checking locale_str[2] == '_' and before validating ch3/ch4, detect the empty-country case by checking if locale_str[3] == '_'.
2. If locale_str[3] == '_' then:
   - Treat country as empty: country = ""
   - The variant starts at index 4, so variant = locale_str[4:] (if any)
   - Return Locale(locale_str[0:2], "", locale_str[4:]) (or Locale(lang, "") if there is no variant)
3. Else (country not empty) continue the existing uppercase checks for ch3/ch4 and proceed with the existing logic (country = locale_str[3:5], variant = locale_str[6:] if a second '_' is present).

Replace the block:

Notes
- This preserves strict validation for the normal country-present case while accepting the valid empty-country + variant form.
