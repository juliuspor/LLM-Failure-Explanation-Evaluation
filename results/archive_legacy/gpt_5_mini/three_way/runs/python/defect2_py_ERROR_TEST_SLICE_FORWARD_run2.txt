What happened
- The test calls GrayPaintScale.get_paint(-0.5). Your implementation does not clamp the input value into the allowed range, so the function calculates a gray level outside 0–255 and then hands those out-of-range values to whatever builds the color. That triggers the ValueError: "Color parameter outside of expected range: Red Green Blue".

Where it goes wrong (which variables/methods)
- get_paint(value): the input argument value is negative in the failing test (value == -0.5).
- lower (or lower_bound) and upper (or upper_bound): these are the scale endpoints used to normalize value.
- fraction (or t) = (value - lower) / (upper - lower): if value < lower this becomes < 0; if value > upper it becomes > 1.
- gray (or g) = fraction * 255 (and then converted to int): when fraction is outside [0,1] this produces numbers < 0 or > 255.
- The code that constructs the color (the RGB tuple or a Color object) receives these invalid numbers (negative or >255, or possibly floats). The color constructor raises ValueError.

Why the exception message matches
- The color creation code enforces RGB components in 0–255 (and typically integer). Passing negative integers or >255 triggers the exact error you saw: "Color parameter outside of expected range: Red Green Blue".

Reproduce locally
- Call gps.get_paint(-0.5) and inspect intermediate values:
  - fraction = (value - lower) / (upper - lower)
  - gray = fraction * 255
  - rgb = (gray, gray, gray)
- You will see fraction < 0 and gray < 0 (e.g. -127.5) before the error is raised when the color is built.

Concrete fixes (step‑by‑step)
1. Clamp the input value to the configured range before normalizing:
   - value_clamped = max(lower, min(value, upper))
2. Compute fraction with the clamped value:
   - fraction = (value_clamped - lower) / (upper - lower)   (handle upper == lower defensively if necessary)
3. Compute gray and make sure it is an integer inside 0–255:
   - gray = int(round(fraction * 255))
   - gray = max(0, min(255, gray))
4. Return an RGB tuple of integers (0..255): (gray, gray, gray)
