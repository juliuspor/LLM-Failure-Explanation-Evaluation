The bug is that after a translator reports it consumed N codepoints, the loop advances pos by repeatedly calling code_point_at(code_units, pos). If advancing earlier iterations reaches the end, subsequent calls raise IndexError. Fix by advancing pos by inspecting UTF-16 code units directly, using surrogate detection to increment by 2 for surrogate pairs and by 1 otherwise, and guard against pos >= length. Preserve existing behavior for null checks and writing untranslatable codepoints.
