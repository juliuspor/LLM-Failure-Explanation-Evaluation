What went wrong
- The guard at lines 396–397 rejects any negative minute value:

  - That condition treats minutes_offset < 0 as invalid, so calls like DateTimeZone.for_offset_hours_minutes(0, -15) or (-2, -15) raise ValueError immediately.
- Several tests expect negative minutes to be accepted (e.g. test_zero_hours_negative_minutes, test_negative_hours_negative_minutes) or expect mixed-sign behavior described later in the file (lines ~402–405 adjust minutes_offset). Because negative minutes are rejected early, the logic that normalizes sign and computes the total offset never runs and the tests fail.
