What happened (where it fails)
- The code receives x == -0.0.
- At line 176 you test x == int(x) and then at line 177 you do value = int(x). Converting -0.0 to int yields 0 (the negative-zero sign is dropped).
- Later, at line 192 you call self.add(str(value)) and that produces "0". The test expected "-0.0", so the negative sign was lost by int(x).

Why this happens (root cause)
- Python floats can represent negative zero (-0.0), but int(-0.0) == 0 loses the sign. The equality x == int(x) returns True because -0.0 == 0, so the code enters the integer-handling branch and then destroys the sign by converting to int.
- The earlier check about prev/space (to avoid "x- -4" misparsing) does not prevent this: the sign is lost by the numeric conversion before any string-building or prefix-space logic can restore it.

Exactly where to change (concrete fix steps)
1. Detect negative zero before converting to int. Insert a check immediately before line 176 (the x == int(x) branch) such as:
   - if x == 0.0 and math.copysign(1.0, x) < 0:
         self.add("-0.0")   # or "-0" depending on desired output
         return

2. Or, if you prefer to keep most of the integer branch, preserve the sign after the int conversion:
   - after value = int(x) (line 177) add:
         if value == 0 and math.copysign(1.0, x) < 0:
             self.add("-0")
             return
     (or add "-0.0" if you want a decimal-style zero)

3. Make sure math.copysign is imported (your code already uses math elsewhere, so that should be available).

Why these fixes work
- math.copysign(1.0, x) < 0 is a reliable way to detect the negative sign on a zero-valued float. Checking for x == 0.0 alone is not enough because it doesn't tell you the sign bit.
- Handling -0.0 before converting to int prevents losing the sign when you turn the float into an integer value.

Suggested minimal patch (showing where to insert)
- Right before the if on line 176 add:

    if x == 0.0 and math.copysign(1.0, x) < 0:
        self.add("-0.0")
        return

This preserves the negative zero output exactly and prevents the code from entering the integer branch that discards the sign.
