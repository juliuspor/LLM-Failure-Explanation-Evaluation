What happened (root cause)
- The code lets upper become larger than len(text) when lower > len(text).
- That leads to a call substring_java(text, 0, upper) where upper > len(text). substring_java (your slice helper) checks bounds and raises IndexError: String index out of range: 15.

Where it occurs in your code
- These lines in your function are the problem (order matters):
  if upper == -1 or upper > len(text):
      upper = len(text)
  if upper < lower:
      upper = lower
- If lower > len(text) and upper was initially <= len(text), the second line (upper = lower) sets upper > len(text) after you already capped upper to len(text). That makes the later calls such as substring_java(text, 0, upper) pass an end value past the string length and trigger the IndexError in the slice checker (the error shows end == 15).

Why this exact ordering fails
- You cap upper to len(text) first, then possibly raise it again to match lower. Raising it above len(text) is what causes the out-of-range slice.
- Example scenario:
  - len(text) = 10
  - lower = 15
  - upper = 9
  - After first if: upper becomes 10
  - After second if: upper becomes 15
  - substring_java(text, 0, 15) -> IndexError

How to fix it (clear concrete steps)
Option A — simplest correct ordering (minimum change):
- Ensure you clamp upper after you compare with lower:
  if upper == -1:
      upper = len(text)
  if upper < lower:
      upper = lower
  if upper > len(text):
      upper = len(text)

Option B — use min/max to make intent explicit:
  if upper == -1:
      upper = len(text)
  upper = max(upper, lower)
  upper = min(upper, len(text))

Option C — also guard lower (if lower should never exceed len):
  lower = min(lower, len(text))
  if upper == -1:
      upper = len(text)
  upper = max(upper, lower)

Pick the option that matches the intended semantics (whether lower should be clamped to len(text) or you want upper to be clamped after matching lower). After applying one of these changes, substring_java will never be called with end > len(text) and the IndexError will stop occurring.
