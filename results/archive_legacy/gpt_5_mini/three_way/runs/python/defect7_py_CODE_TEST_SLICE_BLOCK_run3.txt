What went wrong (exact place and error)
- The failure happens in the loop at lines 765–767:
  - line 766 executes: _ = array[i].upper()
- If array[i] is None the call array[i].upper() raises:
  - AttributeError: 'NoneType' object has no attribute 'upper'
- This is exactly what your failing tests demonstrate (e.g. test_to_class_with_none_element and test_to_class_all_none). The code never checks for None before calling .upper(), so any None element triggers the exception.

Why that causes the wrong behavior
- The method's job is to return the element classes (type(element)) — not to manipulate string contents.
- The .upper() call is unnecessary for the function's purpose and was left in accidentally. It assumes each element is a string and has an .upper() method. That assumption is false for None and for many non-string elements in your tests (and will fail similarly for any object without an upper() method).
- Because the exception occurs before classes.append(type(array[i])) runs, the function never reaches the correct behavior for that element (or the rest of the array).

Concrete places to change (exact lines)
- Remove or guard the call at line 766:
  - Remove: _ = array[i].upper()
  - Or check before calling:
    - if array[i] is not None and hasattr(array[i], "upper"): ... but preferable to remove entirely

Simple, correct replacement (step-by-step)
1. Replace the loop (lines 765–767) with:
   for el in array:
       classes.append(type(el))
2. That yields:
   - If el is None, type(el) is type(None) — which matches your expected test values.
   - No AttributeError occurs because nothing is being called on the element.

Alternative safe minimal change (if you prefer keeping index access):
- Before calling .upper(), test for None:
  for i in range(len(array)):
      if array[i] is not None:
          _ = array[i].upper()   # still unnecessary — consider removing
      classes.append(type(array[i]))

Why the recommended fix is best
- It removes an extraneous operation that caused the bug.
- It implements the method's intended behavior (return the types of the elements) for all elements, including None and non-string objects.
- It keeps the code simple and robust against different element types.

Summary
- Root cause: unguarded call to array[i].upper() at line 766 causes AttributeError for None (and other non-string) elements.
- Fix: remove that .upper() call and simply append type(element) for each element (see replacement loop above). This resolves the failing tests and matches the intended behavior.
