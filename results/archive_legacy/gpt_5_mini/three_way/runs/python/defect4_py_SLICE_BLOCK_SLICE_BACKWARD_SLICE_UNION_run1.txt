Why it happens (root cause)

- The code still reaches line 346 and tries to compare start < min_start regardless of whether min_start was set. Python raises UnboundLocalError because min_start is referenced before assignment.
- There is also a second logic bug: after the conditional at line 346 that sets self._min_start_index = index if start < min_start (line 347), there is another unconditional assignment at line 349: self._min_start_index = index. That means even when start is not less, you still overwrite _min_start_index, so the comparison result is ignored — this is likely not intended.
