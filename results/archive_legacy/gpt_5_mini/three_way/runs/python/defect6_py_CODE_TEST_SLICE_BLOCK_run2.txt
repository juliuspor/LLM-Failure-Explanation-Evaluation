What happened (where, what variable/method caused it)
- At line 176 you decide whether x is an "integer value" using:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- For x == -0.0 this condition is True because -0.0 == 0 and int(-0.0) == 0.
- You then run the integer branch (lines 177–192). value = int(x) becomes 0 (line 177), and later you call self.add(str(value)) (line 192). That converts the numeric 0 to the string "0", losing the negative sign.
- The test test_negative_zero expects "-0.0", but the code produced "0" — hence the failure.

Why it happens (root cause)
- The code tests numeric equality (x == int(x)) before checking whether x is negative zero. In IEEE-754 floats, -0.0 compares equal to +0.0, so that equality hides the sign.
- Converting to int (value = int(x)) and then to str(value) discards the negative-zero sign because integers have no signed zero. So the sign is lost inside the integer branch.

Where to change it (exact place and changes)
- Before the integer-branch test at line 176 you must detect negative zero and handle it specially. There is already a utility referenced in the tests: CodeConsumer.is_negative_zero. Use it early.
- Example exact change (pseudocode placing lines before current line 176):
    if self.is_negative_zero(x):
        # preserve the sign and fractional form required by tests
        self.add("-0.0")
        return
    if x == int(x) and not math.isinf(x) and not math.isnan(x):
        ...

- Alternatively detect negative zero with math.copysign or checking x == 0.0 and math.copysign(1.0, x) < 0.

Suggested minimal patch (concrete lines)
Insert immediately before line 176:
    # Preserve negative zero sign: -0.0 must not be converted to "0"
    if CodeConsumer.is_negative_zero(x):
        self.add("-0.0")
        return

Why this fix works
- It prevents negative zero from entering the integer branch where value=int(x) => 0 and the sign is lost.
- It outputs the exact string the tests expect ("-0.0").
- It doesn't change behavior for other numbers (positive zero, integers, floats, inf, NaN).

Other notes
- Do not try to preserve negative zero by comparing x == int(x) and then re-adding a sign; once you convert to int and to string you lost the needed float sign information. Detect negative zero while x is still a float.
- Keep this early check before any conversion to int or use of int(x).
