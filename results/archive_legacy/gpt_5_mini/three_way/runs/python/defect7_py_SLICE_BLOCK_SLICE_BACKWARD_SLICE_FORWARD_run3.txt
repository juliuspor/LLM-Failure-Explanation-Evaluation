What happens (where it fails)
- Failure occurs at line 766: _ = array[i].upper()
- That line calls the string method .upper() on whatever is stored in array[i].
- If array contains a value that is not a string (for example None, an int, a list, etc.), Python raises an AttributeError like "'int' object has no attribute 'upper'". That stops the loop before you ever reach line 767.

Why it happens (root cause)
- array is assumed to contain only str objects, but at least one element is not a str.
- .upper() is a method defined only on str objects. Calling it on a non-str value triggers the exception at line 766.
- Because the exception stops execution inside the loop, classes.append(type(array[i])) at line 767 is never reached for that iteration (and the program fails).

How to confirm (quick checks)
1. Add a debug print right before line 766:

   This shows which index and which value caused the failure.
2. Or run a short type-check loop:
