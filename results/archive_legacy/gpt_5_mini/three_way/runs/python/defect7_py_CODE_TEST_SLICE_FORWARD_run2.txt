What happened
- The failing line is the call to array[i].upper(), which appears just before the append. That call raises AttributeError when array[i] is None because NoneType has no .upper() method.
- In other words: your code touches array[i].upper() unconditionally, so any None element in the input array triggers an AttributeError before you ever get to classes.append(type(array[i])).

Where it occurs (exact names / lines)
- Method: to_class
- Variable: array, array[i]
- Problem statement line (from your snippet): the failing behavior originates at the statement
    _ = array[i].upper()
  that appears immediately before
    classes.append(type(array[i]))  (the slice shows this append on line 767).
- Root cause: calling .upper() on a None value (NoneType has no attribute 'upper').

Why the .upper() is wrong here
- The purpose of to_class is to return the element types. Calling .upper() is unrelated to that purpose and also unsafe: it assumes every element is a string-like object that implements .upper().
- That assumption is false for inputs that contain None or non-string objects (ints, lists, etc.), which your tests exercise.

Exact fixes (step-by-step)
1. Remove the needless .upper() call.
2. Handle None elements explicitly: map None to type(None) (so the result contains type(None) for None elements).
3. For non-None elements, append type(array[i]).

Concrete corrected code (minimal change)
Replace the loop body with:

    for i in range(len(array)):
        if array[i] is None:
            classes.append(type(None))
        else:
            classes.append(type(array[i]))

Or more Pythonic:

    for item in array:
        classes.append(type(item))

Why this fix works
- It never calls .upper() so it cannot raise AttributeError on None.
- It produces the expected types, including type(None) for None elements, matching your tests.

Optional extra note
- The original special-case for empty arrays returning ArrayUtils.EMPTY_CLASS_ARRAY.copy() is fine if ArrayUtils is defined as intended. If you prefer simplicity, you can return [] for empty input.
