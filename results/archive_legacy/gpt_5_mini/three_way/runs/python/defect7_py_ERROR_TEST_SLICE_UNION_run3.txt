What happened (root cause)
- In ClassUtils.to_class the code iterates the input array and calls array[i].upper() on each element (line 766 in your slice).
- When array contains None (e.g. the second element in ["Test", None, 99.0]), array[i] is None and None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The failing test uses test_array = ["Test", None, 99.0], so the loop reaches i == 1 and hits the exception.

Where it happened
- File/method: hit07_classutils.py, ClassUtils.to_class
- Lines shown in slice:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))
- Offending line: 766 (array[i].upper()).

Why that line is wrong
- The code assumes every element is a string (or has .upper()), but callers may pass None or other non-string objects (tests include None, ints, floats, lists, booleans).
- The purpose of to_class is to return the classes/types of the elements, so calling .upper() is unnecessary and unsafe for non-string values.

Concrete fixes (exact changes you can make)
Option A — Guard against None and avoid .upper():
Replace the loop body (lines 765–767) with:

for i in range(len(array)):
    if array[i] is None:
        classes.append(type(None))
    else:
        classes.append(type(array[i]))

Option B — Simpler and Pythonic (use comprehension):
classes = [type(x) if x is not None else type(None) for x in array]

Option C — If the .upper() call was intended for a specific reason (e.g. normalizing string class detection), make it conditional:
for i in range(len(array)):
    v = array[i]
    if v is None:
        classes.append(type(None))
    elif isinstance(v, str):
        # keep any intended string-only behavior here, but don't call .upper() just to get the type
        classes.append(str)
    else:
        classes.append(type(v))
