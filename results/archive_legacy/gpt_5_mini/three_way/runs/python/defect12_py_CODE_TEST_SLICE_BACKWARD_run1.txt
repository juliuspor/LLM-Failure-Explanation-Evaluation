- Calling WordUtils.abbreviate("0123456789", 15, 20, None) raises an IndexError from substring_java because substring is invoked with end > len(text).

1. You clamp upper to the string length here:

2. Immediately after you do:

Why that causes the failure (root cause)
- The code first reduces upper to len(text) then overwrites that with lower when lower > upper. That lets upper become greater than len(text). substring_java is not tolerant of end > len(text) and raises an IndexError. Also index_of was called with start=lower (15), which is out of range; even if index_of returned -1, the subsequent substring call still uses an out-of-range end.

How to fix it (exact, actionable steps)
You need to ensure lower and upper are clamped to valid ranges before using them. Two straightforward, correct options:

Option A — if lower is beyond the string, return the original string (matches the Java behavior referenced in the test):
- Add at top after computing len(text):
    if lower >= len(text):
        return text
- Keep the rest of the code, but ensure lower is validated before any index_of or substring_java calls.

Option B — clamp lower to len(text) before using it and then recompute/clip upper:
- Replace the current upper/lower adjustments with:
    if upper == -1 or upper > len(text):
        upper = len(text)
    if lower > len(text):
        lower = len(text)
    if upper < lower:
        upper = lower
- Then index_of and substring_java will be called with start/end <= len(text), preventing the IndexError.

Also ensure you clamp lower before calling StringUtils.index_of(text, " ", lower), because giving index_of an out-of-range start can be problematic.
