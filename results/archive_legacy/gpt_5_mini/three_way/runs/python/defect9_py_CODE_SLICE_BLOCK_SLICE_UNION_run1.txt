What happens (where and what error)
- The function crashes inside the sizing loop at lines 81–84:
  > for i in range(search_length):
  >     greater = len(replacement_list[i]) - len(search_list[i])
- If either search_list[i] or replacement_list[i] is None this raises:
  TypeError: object of type 'NoneType' has no len()
- So the failure occurs before any of the later checks that skip None/empty entries (the later matching loops at lines ~102–113).

Why it happens (root cause)
- The code assumes every element in search_list and replacement_list is a non-None string when it computes lengths to guess the output size.
- But earlier in the method the code allows None elements in those lists and only skips them later when searching for matches (see the conditional in the match-finding loops: no_more_matches_for_repl_index[i] or search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None).
- Because the size-guess loop (lines 81–84) does not apply the same null/empty checks, it calls len(...) on None and crashes.

Where to change it (exact lines and condition)
- Modify the sizing loop at lines 81–84 to skip elements that the matching logic would skip. Concretely, add the same guard used later:

Replace
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater

With e.g.
  for i in range(search_length):
      if (
          no_more_matches_for_repl_index[i]
          or search_list[i] is None
          or len(search_list[i]) == 0
          or replacement_list[i] is None
      ):
          continue
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater

Why that fix works
- It prevents len(...) from being called on None and makes the sizing loop consistent with the match-finding loops, so only valid (non-None, non-empty-search, non-None-replacement) entries are used to estimate buffer increase.

Alternative fixes (short)
- Use safe-length helpers like:
    s_len = len(search_list[i]) if isinstance(search_list[i], str) else 0
    r_len = len(replacement_list[i]) if isinstance(replacement_list[i], str) else 0
  then skip when s_len == 0 or replacement is missing.
- Move the sizing calculation to after you confirm which entries are usable (after the initial match-finding), but the simplest is to add the same guard above.
