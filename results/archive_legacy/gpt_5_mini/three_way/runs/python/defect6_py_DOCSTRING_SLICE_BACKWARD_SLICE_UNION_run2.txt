What happens (where it fails)
- The code hangs in an infinite loop when x is an integer with trailing zeros and abs(x) >= 100.
- The loop that never terminates is the while on line 181:
  181 |                 while mantissa % 10 == 0:
  Inside that loop the code reads mantissa (line 182 uses mantissa // 10) but never changes mantissa itself. Because mantissa keeps the same value that is divisible by 10, mantissa % 10 == 0 remains true forever and the loop never exits. This makes the program spin / use 100% CPU and never reach the later lines (190, 192, 194).

- Root cause: a loop condition depends on mantissa having its trailing zeros removed, but the loop body never updates mantissa. Concretely:
