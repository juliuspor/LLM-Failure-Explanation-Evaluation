What failed
- The failure happens at line 766 in the loop:
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))
- The call to array[i].upper() assumes the element is a string. If array[i] is None (or any non-string like int or float), calling .upper() raises an AttributeError because NoneType (or int/float) has no attribute 'upper'.

Why it happens (root cause)
- Variable names: array is the input list, array[i] is the current element.
- Method: .upper() is a string method. There is no prior check that array[i] is a str before calling it.
- Consequence: When a test passes test_array = ["Test", None, 99.0], the second element is None. At line 766 the code executes array[1].upper(), which raises AttributeError. The code never reaches classes.append(type(array[i])) for that element, so the function fails instead of returning the expected [str, type(None), float].

Where it occurred
- In to_class(cls, array) inside the for loop (lines 765–767 shown in the slice).
- Specifically line 766 (the .upper() call) is the offending statement.

How to fix it (concrete steps)
1. Remove the unnecessary .upper() call. It is not used for computing the result (the result is based on type(array[i])), so simply delete the line at 766.
2. Add explicit handling for None elements so they become type(None) in the result (or rely on type(None) when you call type(None) directly).
3. Use a simpler, clearer loop or list comprehension.

Minimal corrected implementation examples you can use:

- Fixed loop (explicit):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for i in range(len(array)):
            elem = array[i]
            if elem is None:
                classes.append(type(None))
            else:
                classes.append(type(elem))
        return classes

- Or idiomatic list comprehension:
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        return [type(x) if x is not None else type(None) for x in array]

Why these fixes work
- They remove the erroneous .upper() call that caused AttributeError.
- They explicitly handle None, so the function returns type(None) for None elements as the tests expect.
- They preserve the existing behaviors for None input and empty input.

Notes
- The empty-array branch using ArrayUtils.EMPTY_CLASS_ARRAY.copy() is fine as long as ArrayUtils.EMPTY_CLASS_ARRAY is a list; keep that if you need to return a separate list instance.
