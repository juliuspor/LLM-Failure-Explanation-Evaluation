- But code_point_at (the function that uses the code_units list) expects an index into the UTF-16 code_units list (a code-unit index). The caller passes the code-point index i directly to code_point_at.
- Because the emoji occupies 2 UTF-16 code units but only 1 Python code point, the code-point index and the code-unit index diverge. That is why line 36 raises IndexError: the index given to code_units[index] is out of range.
- In short: the code mixes two incompatible indexing systems:
  - Python str indexes by Unicode code points
  - code_point_at / to_utf16_code_units expect UTF-16 code-unit indexes
  Mixing them causes the out-of-range access when a surrogate pair is present.
