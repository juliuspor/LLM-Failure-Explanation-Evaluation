- The failure happens in GrayPaintScale.get_paint. Relevant lines:

  - lines 74–75:

- For inputs outside the configured range (e.g. value = -0.5 or 1.5 with defaults), the computed g is < 0 or > 255. Instead of clamping these to 0 or 255, the code raises a ValueError. The unit tests expect clamping (return (0,0,0) for below lower_bound and (255,255,255) for above upper_bound), so the raise causes the tests to fail.

Why it happens (root cause)
- The code never clamps the input value or the computed gray component g. After computing g (line 71) it explicitly treats out-of-range g as an error and raises (lines 74–75). The intended behavior in the tests (and in the original Java) is to clamp to bounds, not to raise an exception.
