Why it fails (root cause)
- The code unconditionally calls array[i].upper() on every element.
- When array[i] is None, Python tries to call None.upper(), which does not exist, producing AttributeError: 'NoneType' object has no attribute 'upper'.
- The tests expect None elements to be handled (returned as type(None)) rather than causing an exception.

Where exactly it happens
- In method ClassUtils.to_class, inside the for loop at line 766. The variable array[i] is None for the failing test case, so calling .upper() on it triggers the exception before the code reaches the append at line 767.

How to fix it (concrete steps)
1. Remove or guard the .upper() call so it is not invoked on None values.
2. Ensure that for None elements the function appends type(None) (or None, depending on the intended API) instead of raising.

Minimal example fixes (pick one):

Option A — remove the unnecessary .upper() call
- Replace the loop body with just:

  This only calls .upper() when the element is not None.
