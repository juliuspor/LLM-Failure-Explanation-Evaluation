What happened

- At the moment that check runs, code_units is still empty (code_units = [] on line 12) so len(code_units) == 0. Any non‑negative index (for example 0) will be >= 0 and trigger the IndexError even when the index is valid for the input string s.

Why it happened (root cause)
- The code is checking the index against the wrong sequence. The variable index is intended to refer to a position in the input string s (or in the eventual code_units after it has been filled), but the condition compares index to len(code_units) instead of the correct length. Because code_units is empty at that point, the check is too strict and always fails for valid indices.
