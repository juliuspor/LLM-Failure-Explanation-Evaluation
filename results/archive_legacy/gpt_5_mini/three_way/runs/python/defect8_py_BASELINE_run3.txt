What happened (where & why)
- At lines 231–247 the code assumes that after the first underscore (locale_str[2]) there is always a two-letter country code at indices 3–4 (uppercase), and only if length == 5 it returns language+country; otherwise it expects another underscore at index 5 and the variant after index 6.
- For the input "fr__POSIX" (length 9) that assumption is wrong: there is an empty country (double underscore). Concretely:
  - locale_str[2] == '_' (checked at line 236) — OK.
  - ch3 = locale_str[3] is '_' (line 237), not an uppercase letter.
  - The test at lines 239–240 (if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' ...) fails because ch3 == '_' is not between 'A' and 'Z', so the code raises ValueError("Invalid locale format: fr__POSIX").

Why it occurs (root cause)
- The validator at lines 237–240 checks positions 3 and 4 unconditionally. It never checks for the double-underscore case (locale_str[3] == '_'), so an empty country makes ch3 '_' and triggers the uppercase-letter check to fail.
