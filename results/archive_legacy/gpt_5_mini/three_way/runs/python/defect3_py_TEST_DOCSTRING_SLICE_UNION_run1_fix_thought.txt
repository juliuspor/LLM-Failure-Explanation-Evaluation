The translate loop can call code_point_at with an index equal to length when a translator returns a count that overstates consumed codepoints (e.g., CsvEscaper returning 2 for a surrogate pair). To avoid IndexError, guard calls to code_point_at by checking pos < length before invoking. When consumed>0, advance pos by the UTF-16 code unit length of each consumed codepoint but stop if pos reaches length. This makes the method robust against translators returning counts larger than remaining codepoints. Also keep original behavior for normal cases.
