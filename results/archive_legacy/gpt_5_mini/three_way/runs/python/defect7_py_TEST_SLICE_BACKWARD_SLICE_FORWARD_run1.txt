What happened (where and what failed)
- The failure occurs inside ClassUtils.to_class while iterating the input array.
- At line 765 the code does: _ = array[i].upper()
- When array[i] is None (for example in your test test_to_class_with_none_element), None has no .upper() method. That raises AttributeError and causes the test to fail.
- Immediately after that (line 767) the code appends classes.append(type(array[i])), but execution never reaches that because of the exception raised at line 765.

Why it fails (root cause)
- The code unconditionally calls array[i].upper() without checking whether array[i] is None (or even whether it’s a string). NoneType has no upper attribute, so the call raises AttributeError.
- The intent of to_class is to record the element types (type(array[i])). Calling .upper() is unnecessary to determine type and is unsafe for non-string or None values.

Exact places to change
- Location: ClassUtils.to_class, lines around 765–767. Remove or guard the call to array[i].upper().

Concrete fixes (pick one)

1) Simple guard for None
- Replace the two lines:
    _ = array[i].upper()
    classes.append(type(array[i]))
  with:
    if array[i] is None:
        classes.append(type(None))
    else:
        classes.append(type(array[i]))

2) Avoid .upper() entirely (preferred if .upper() served no essential purpose)
- Replace those lines with:
    classes.append(type(array[i]))

3) If .upper() was meant to normalize strings beforehand, handle types explicitly:
    if array[i] is None:
        classes.append(type(None))
    elif isinstance(array[i], str):
        # if you need to use the upper() result, use it on a string
        _ = array[i].upper()
        classes.append(str)
    else:
        classes.append(type(array[i]))

Why these fixes work
- They prevent calling .upper() on None (or other non-string types), avoiding the AttributeError.
- They ensure None maps to type(None), which matches your tests expecting [str, type(None), float] for ["Test", None, 99.0].

Extra note
- Also confirm ClassUtils.to_class already handles input array being None (your test test_to_class_null_input checks that). If not, add an early return if array is None:
    if array is None:
        return None

Summary
- Root cause: unconditional call to array[i].upper() at line 765 when array[i] can be None.
- Fix: remove/guard that call and append type(array[i]) (or type(None) for None) instead. This yields the expected behavior in your tests.
