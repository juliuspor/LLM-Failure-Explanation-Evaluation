What failed
- The code raises an AttributeError at line 766:
  classes.append(type(array[i])) is never reached because the preceding line 766 does:
    _ = array[i].upper()
  That call assumes array[i] is a string. If array contains a non-string (for example a class object, a type object, or None) then array[i].upper() fails with something like:
    AttributeError: 'type' object has no attribute 'upper'
  or
    AttributeError: 'NoneType' object has no attribute 'upper'

Why it fails (root cause)
- The code treats every element in array as if it were a str and calls .upper() unconditionally.
- In your loop the variable array[i] is not guaranteed to be a string. The .upper() call therefore fails when array[i] is a type, a class object, None, or any object that does not implement upper().
- The intent is unclear from the code: you then call type(array[i]) and append that. Calling .upper() here has no effect on the returned value (you discard the result into _), so the .upper() call is both unsafe and unnecessary if your goal is to return the element types.

Where it happens
- Exact failing line: line 766 in the slice block:
    _ = array[i].upper()
- The loop is at lines 765–767.

How to fix it (concrete steps)
Choose one of these fixes depending on what you want to accomplish:

1) If array is supposed to contain strings (class names) and you want their corresponding classes:
- Verify elements are strings, use .upper() only after that check, then resolve the class name to a class object (e.g., with importlib or globals()).

Example fix:
    from typing import Any, List, Optional, Type
    import importlib

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()

        classes: List[Type] = []
        for i, elem in enumerate(array):
            if not isinstance(elem, str):
                raise TypeError(f"expected string at index {i}, got {type(elem).__name__}")
            name = elem.upper()  # if you actually need uppercase for lookup
            # resolve name to class (example using importlib for "module.ClassName")
            module_name, _, class_name = name.rpartition('.')
            if not module_name:
                raise ValueError(f"fully-qualified class name required at index {i}: {elem}")
            module = importlib.import_module(module_name)
            classes.append(getattr(module, class_name))
        return classes

2) If array already contains class/type objects and you just want their types (or the objects themselves):
- Remove the .upper() call entirely; it is unnecessary and causes the failure.

Example fix:
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()

        classes = []
        for i, elem in enumerate(array):
            if elem is None:
                raise TypeError(f"None found at index {i}")
            classes.append(type(elem))  # or append(elem) if you want the class itself
        return classes

3) If elements may be mixed (some strings, some classes), handle both cases explicitly:
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()

        classes = []

                resolved = resolve_class_from_string(elem)

                classes.append(elem)
            else:
                raise TypeError(f"unsupported element at index {i}: {type(elem).__name__}")
        return classes

Summary
- The immediate cause is the unconditional call to array[i].upper() at line 766 on a non-string element.
- Fix by removing or guarding that call and by clearly handling the element types you expect (strings vs types), with explicit checks and clear error messages.
