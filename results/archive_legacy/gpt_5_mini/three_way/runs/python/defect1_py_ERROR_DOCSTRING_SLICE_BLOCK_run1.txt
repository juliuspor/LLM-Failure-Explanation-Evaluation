What happened
- The factory method that builds a fixed-offset time zone raised ValueError("Minutes out of range: -15") at lines 396–397.
- The check at line 396 is:
  if minutes_offset < 0 or minutes_offset > 59:
      raise ValueError(f"Minutes out of range: {minutes_offset}")
- That check rejects any negative minutes (minutes_offset < 0), so passing minutes_offset = -15 triggers the exception.

Why this is wrong (root cause)
- The docstring and intended behavior allow negative minute offsets in some cases:
  - minutes can be negative when hours_offset is zero or negative (examples: (0, -15) -> -00:15, (-2, -15) -> -02:15).
  - minutes should only be forbidden when negative and the hours offset is positive (example in docstring: (2, -15) is invalid).
- The code at lines 396–397 enforces "minutes must be 0..59" for all cases, ignoring the permitted negative-minute cases. That mismatch between the code and the documented rules is why a valid input like (hours_offset = 0 or negative, minutes_offset = -15) fails.

Where to change it (concrete steps)
1. Replace the unconditional negative-minute check with checks that match the docstring:
   - Allow minutes in the range -59..+59
   - Additionally forbid negative minutes when hours_offset > 0
   - Also still check any combined-range rule (the total offset must not exceed ±23:59:59)

2. Suggested code (replace lines 396–397 and add the combined-range check afterwards). Example:

  # validate minute range first
  if minutes_offset < -59 or minutes_offset > 59:
      raise ValueError(f"Minutes out of range: {minutes_offset}")

  # forbid negative minutes with a positive hour value
  if hours_offset > 0 and minutes_offset < 0:
      raise ValueError("Positive hours with negative minutes is not allowed")

  # verify resulting total offset doesn't exceed +/- 23:59:59
  total_seconds = hours_offset * 3600 + minutes_offset * 60
  if total_seconds > 23*3600 + 59*60 + 59 or total_seconds < -(23*3600 + 59*60 + 59):
      raise ValueError("Resulting offset exceeds permitted range")

Why this fix works
- It accepts minutes_offset = -15 when hours_offset is 0 or negative (allowed by the docstring).
- It still rejects invalid minute values outside [-59, +59].
- It rejects the documented invalid combination (e.g., hours_offset = 2, minutes_offset = -15).
- It enforces the overall offset limit of ±23:59:59.
