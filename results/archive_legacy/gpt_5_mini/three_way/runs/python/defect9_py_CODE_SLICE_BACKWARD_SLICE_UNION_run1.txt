What happened (what failed)
- The code raises a TypeError at the sizing loop around line 81–85:
  > TypeError: object of type 'NoneType' has no len()
- That happens when either search_list[i] or replacement_list[i] is None. The loop on lines 81–85 calls len(replacement_list[i]) and len(search_list[i]) without guarding against None values.

Where it occurs (specific lines & variables)
- Method: replace_each (line 28)
- Problem loop: lines ~81–85
  - Code: greater = len(replacement_list[i]) - len(search_list[i])
  - Variables: replacement_list[i], search_list[i]
- Earlier loops (lines ~59–76 and ~102–113) already skip entries where search_list[i] is None, search_list[i] == "" or replacement_list[i] is None. The sizing loop does not, so it tries len(None) and crashes.

Why it happens (root cause)
- The sizing loop attempts to estimate the output buffer growth by computing length differences for every pair in the lists. It assumes every element is a non-None string. But the code allows None entries in search_list/replacement_list and explicitly skips those entries in the matching loops; the sizing loop was not made "null-safe" (as the inline BUG comment states). When an element is None, calling len(None) raises the TypeError.

How to fix it (exact steps)
1. Guard the sizing loop in the same way you guard the matching loops. Skip an index if:
   - no_more_matches_for_repl_index[i] is True (optional)
   - search_list[i] is None
   - search_list[i] == ""
   - replacement_list[i] is None
2. Only compute lengths when both search_list[i] and replacement_list[i] are non-None and search_list[i] is non-empty.
