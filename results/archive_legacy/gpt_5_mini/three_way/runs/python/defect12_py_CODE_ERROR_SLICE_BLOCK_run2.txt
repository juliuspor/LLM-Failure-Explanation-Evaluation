What happened (root cause)
- substring_java(text, 0, end) raises IndexError when end > len(text). The failing call passed end == 15 while len(text) < 15.
- That too-large end value comes from the variable upper. Look at these lines in your function (roughly):
  - You clamp upper to len(text) here:
    if upper == -1 or upper > len(text):
        upper = len(text)
  - But then you allow upper to be increased later:
    if upper < lower:
        upper = lower
  - If lower > len(text), the assignment upper = lower makes upper > len(text) again. You never clamp upper back down after that, so substring_java is called with end > len(text), triggering the IndexError in the slice check (the code shown at lines 32–33).

Where it occurred
- The exception comes from substring_java(..., end) (the slice block that checks end > len(s)). The caller passed end == upper which had been set to lower (because upper < lower), and lower was larger than len(text).

How to fix (concrete steps)
Option A — clamp upper after adjusting for lower (smallest change):
- After this block:
    if upper < lower:
        upper = lower
  add:
    if upper > len(text):
        upper = len(text)

Option B — compute a bounded upper using min/max in one place (preferred for clarity):
- Replace the two adjustments with:
    if upper == -1:
        upper = len(text)
    upper = max(upper, lower)
    upper = min(upper, len(text))
  or in one expression:
    upper = min(len(text), (len(text) if upper == -1 else max(upper, lower)))

Option C — also guard lower value explicitly (if you want lower never to exceed len(text)):
- Before index_of, cap lower:
    if lower > len(text):
        lower = len(text)

Why these fixes work
- They ensure upper is never greater than len(text) when calling substring_java(text, 0, upper) or substring_java(text, 0, index). That prevents the slice-end check (start < 0 or end > len(s) ...) from failing.

Recommended minimal patch (example)
- After your existing if upper < lower: upper = lower add:
    if upper > len(text):
        upper = len(text)

That directly prevents the end parameter being larger than the string and resolves the IndexError.
