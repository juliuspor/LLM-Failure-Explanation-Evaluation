What happened (where: lines 335–351)
- At line 336 you set inferred_type = type(array) when array is not None.
- At line 337 you set inferred_type = type(element) when element is not None.
- If neither array nor element is present you set inferred_type = object (line 340).
- Later (lines 346–351) you check:
  if inferred_type == object and expected_type != object:
      raise TypeError(...)

Why it fails (root cause)
- type(array) returns the Python type of the list object itself (e.g. <class 'list'>), not the type of the items inside the list.
- When the caller passes an empty list (or a list variable whose elements haven’t been inspected here), array is not None but contains no items. Your code takes type(array) — which is list — rather than the element type, so inferred_type ends up being the generic container type (or falls through to object if array is None and element is None).
- That makes the condition inferred_type == object (or the equivalent generic-case you treat as “unknown element type”) true while expected_type is some specific class. The code then raises the TypeError, because it thinks you are trying to cast an “object list” to a more specific list type.
- In short: you are inferring the wrong thing (the container’s type rather than the element type), so empty or uninspected arrays cause inferred_type to be the generic type and trigger the exception erroneously.

1. If you want to infer the element type from a non-empty array, inspect the first element, not the array object:

     if array is not None and len(array) > 0:
         inferred_type = type(array[0])
2. Keep the existing element branch:

3. Only fall back to object when you truly have no element information:
   - else:

4. Optionally, if you need to handle heterogeneous lists, you may want to:
   - check every item’s type and decide whether to use a common base class or raise earlier, or
   - treat an empty list as “unknown element type” and allow casting only when expected_type is object or when an explicit override is provided.
5. Update the error message if necessary to reflect “unknown element type” rather than treating type(array) (a list) as the element type.
