What happened (root cause)
- The error comes from this line (around line 82 in your snippet):
  greater = len(replacement_list[i]) - len(search_list[i])
- That line assumes both replacement_list[i] and search_list[i] are strings. If either element is None, calling len(None) raises TypeError: "object of type 'NoneType' has no len()".
- You already handle None/empty entries when searching for matches (the for-loops that call text.find(...) skip entries where search_list[i] is None or replacement_list[i] is None). But the earlier "guess buffer size" loop (the one with the BUG comment) does not perform the same null checks, so it tries to take len(...) on None.

Where it occurs
- The failing loop starts at your comment:
  # BUG: this loop is not null-safe for elements inside the arrays
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      ...
- Because some search_list[i] or replacement_list[i] can be None, len(...) fails there.

Why it was missed
- You validated/filtered the elements in the matching loops (you skip None/empty search strings and None replacements), but forgot to apply the same checks in the buffer-size estimation loop.

How to fix it (concrete steps and code)
Replace the "guess buffer size" loop with a null-safe version that mirrors the checks you already use. For example:

    # get a good guess on the size of the result buffer so it doesnt have to double if it goes over a bit
    increase = 0
    for i in range(search_length):
        # skip null/empty search strings and null replacements
        if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches
    increase = min(increase, len(text) // 5)

Alternative minimal change (if you prefer not to duplicate checks):
- Compute safe lengths without calling len on None:

    for i in range(search_length):
        s_len = len(search_list[i]) if search_list[i] is not None else 0

Why this fix is correct
