What happened (where)
- Failure occurs inside ClassUtils.to_class when iterating the input array.
- The relevant lines (from your slice) are lines 765–766:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
- The test passed an array containing None (e.g. ["Test", None, 99.0]). On the second iteration array[1] is None, so array[1].upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.

Why it happened (root cause)
- The code assumes every array element is a string and calls .upper() on each element. That assumption is wrong:
  - None does not have an .upper() method → AttributeError.
  - Non-string elements (ints, floats, lists, etc.) also won’t necessarily have .upper().
- The purpose of ClassUtils.to_class (per tests) is to return the class/type of each element (e.g. str, type(None), float), not to uppercase strings. Calling .upper() is either a leftover copy/paste from a string-specific routine or a logic bug.

Exactly where to change (concrete fix)
Replace the loop body that currently calls array[i].upper() with logic that records the element’s type and handles None explicitly.

Minimal concrete code to use:
- If you want the element’s type: result[i] = type(array[i])
- If the method currently builds a list "result" of same length, change the loop to:

for i in range(len(array)):
    if array[i] is None:
        result[i] = type(None)
    else:
        result[i] = type(array[i])

Or equivalently:

for i, el in enumerate(array):
    result[i] = type(el)

Also ensure the method already returns None if the input array is None and returns [] for an empty array (the tests expect that).

Why this fix works
- type(None) is type(None) (the tests expect type(None) for None elements).
- type(el) works for any element: str, int, float, list, bool, etc., and will not call any method on the element itself, so None won’t raise.

Extra notes
- Don’t use el.__class__ vs type(el)—both work; type(el) is idiomatic.
- If you originally intended to treat strings specially (e.g., normalize case) do that after checking for None and only when isinstance(el, str).
- Make sure ClassUtils.to_class handles input None (should return None) and empty lists (should return []).
