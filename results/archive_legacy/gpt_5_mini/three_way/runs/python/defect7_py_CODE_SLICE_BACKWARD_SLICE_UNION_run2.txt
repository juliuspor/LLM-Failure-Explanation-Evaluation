What happened (where it fails)
- The crash happens at line 766: `_ = array[i].upper()`.
- That line calls the string method upper() on whatever is stored in array[i].
- If array contains a class object (a type) or another non-str value, calling upper() raises AttributeError: 'type' object has no attribute 'upper' (or a similar AttributeError for the actual non-str type).
- In other words, the code assumes each array[i] is a str, but the caller passed non-strings (probably class objects). The failure happens immediately in the loop, before classes.append(...).

Why it fails (root cause)
- The function name to_class and the later line classes.append(type(array[i])) indicate the array elements are not strings but already objects or class-like values.
- upper() only exists on str. Calling .upper() on a type or on None/other types will raise an AttributeError.
- The code does not check the element type before calling .upper(), so a wrong input type causes the exception.

Exactly where to look
- Line 765: for i in range(len(array)):
- Line 766: _ = array[i].upper()  ← this is the failing statement
- Line 767: classes.append(type(array[i]))  ← this runs only if line 766 succeeds (it does not)

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    
        classes = []
        for i in range(len(array)):
            elem = array[i]
            if elem is None:
                raise TypeError(f"to_class: element at index {i} is None")
            if isinstance(elem, str):
                name = elem.upper()
                classes.append(resolve_class_from_name(name))
            elif isinstance(elem, type):
                classes.append(elem)
            else:
                raise TypeError(f"to_class: unsupported element type {type(elem).__name__} at index {i}")
        return classes

Summary
