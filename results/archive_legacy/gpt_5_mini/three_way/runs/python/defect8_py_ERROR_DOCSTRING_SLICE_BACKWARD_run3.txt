- Method: LocaleUtils.to_locale (the code shown around lines 188–240)

Why it failed (root cause)
- The code assumes that when a country part is present, it occupies locale_str[3] and locale_str[4] and therefore checks:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z': raise ValueError(...)
- For "fr__POSIX" the characters are:
  - index 0 = 'f', 1 = 'r', 2 = '_', 3 = '_', 4 = 'P', ...
- ch3 therefore is '_' (at index 3). The code treats '_' as invalid because it is not between 'A' and 'Z', so it raises the ValueError.
- The real intent for "fr__POSIX" is a valid locale with language = "fr", empty country = "" and variant = "POSIX" (a double-underscore denotes an empty country and a variant). The code does not recognize the double-underscore form and incorrectly enforces a two-letter country at indexes 3–4.
