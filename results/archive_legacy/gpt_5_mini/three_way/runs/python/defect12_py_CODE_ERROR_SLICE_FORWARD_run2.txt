What happened (where it failed)
- The IndexError comes from substring_java returning s[start:end] with end (the second argument) larger than len(text). The error trace shows the slice attempt at substring_java line 34.
- In your function the only places that pass an "end" to substring_java are:
  - result = substring_java(text, 0, upper) (two places)
  - return substring_java(text, 0, index) + ...
- So either upper or index ended up greater than len(text). The code that makes that possible is the block:
  if upper == -1 or upper > len(text):
      upper = len(text)
  if upper < lower:
      upper = lower

Why it happens (root cause)
- If lower is greater than len(text), the first if sets upper = len(text), then the second if notices upper < lower and sets upper = lower. That leaves upper > len(text). Later you call substring_java(text, 0, upper), which tries to slice past the string end and raises IndexError.
- Example concrete values that trigger it:
  - len(text) == 10, lower == 15, upper == -1 (or upper ≤ 10). After the first if upper becomes 10, then because 10 < 15 the code sets upper = 15. substring_java(..., 0, 15) triggers the IndexError.

Exactly where in the code
- Line: if upper < lower: upper = lower
- Followed later by: result = substring_java(text, 0, upper) (or return substring_java(text, 0, upper) ...)
- substring_java then executes s[start:end] with end == upper > len(s) → IndexError.

How to fix it (clear step-by-step options)
You must ensure the end argument passed to substring_java never exceeds len(text). Two safe approaches:

Option A — Clamp lower first (recommended)
1. Immediately clamp lower to the text length:
   if lower > len(text):
       lower = len(text)
2. Then keep your existing upper normalization:
   if upper == -1 or upper > len(text):
       upper = len(text)
   if upper < lower:
       upper = lower
This guarantees upper ≤ len(text).

Option B — Ensure upper is clamped after comparing with lower
1. Keep your current comparisons but add one more clamp before any substring call:
   if upper == -1 or upper > len(text):
       upper = len(text)
   if upper < lower:
       upper = lower
   if upper > len(text):
       upper = len(text)   # extra clamp to protect substring_java
That final clamp prevents upper from exceeding len(text).

Option C — Reject or handle invalid lower explicitly
1. If lower > len(text) you may want to return text (or None) or append default_ending, depending on intended behavior:
   if lower >= len(text):
       return text + StringUtils.default_string(append_to_end)  # or text alone
This documents the boundary behavior instead of silently clamping.
