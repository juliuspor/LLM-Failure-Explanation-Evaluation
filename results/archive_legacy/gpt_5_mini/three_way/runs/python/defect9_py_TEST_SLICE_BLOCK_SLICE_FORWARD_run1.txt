What happens
- The code crashes while computing how much to grow the output buffer (lines 81–84). Specifically line 82 does:
  greater = len(replacement_list[i]) - len(search_list[i])
- If replacement_list[i] (or search_list[i]) is None this calls len(None) and raises a TypeError. That is exactly what your tests exercise (they pass None in replacement_list).

- The failure occurs in the loop starting at line 81:
  - 81: for i in range(search_length):
  - 82:     greater = len(replacement_list[i]) - len(search_list[i])    <-- TypeError when replacement_list[i] is None
  - 83:     if greater > 0:
  - 84:         increase += 3 * greater
- Later in the code the function correctly checks for None at decision time (lines 102–107), for example:
  - it checks "replacement_list[i] is None" before attempting to find occurrences (lines ~102–108), but that check comes after the length calculation above.

Why this causes the test failure
- Your tests call StringUtils.replace_each("aba", ["a"], [None]) and StringUtils.replace_each("aba", ["a","b"], ["c", None]). The presence of None in replacement_list triggers len(None) at line 82 and the function raises TypeError before it reaches the later None checks or performs replacements.

How to fix it (concrete steps)
1. Guard the length computation against None (and empty) values before using len():
   - Either skip entries where search_list[i] is None or replacement_list[i] is None, or treat None as length 0 for the purpose of estimating growth.
2. Example minimal change at the top of that loop:
   - Replace:
       greater = len(replacement_list[i]) - len(search_list[i])
     With:
       if search_list[i] is None or replacement_list[i] is None:
           continue
       greater = len(replacement_list[i]) - len(search_list[i])
   - This matches the later logic that ignores None entries and avoids calling len(None).
3. Alternatively, compute lengths with safe helpers:
       repl_len = 0 if replacement_list[i] is None else len(replacement_list[i])
       search_len = 0 if search_list[i] is None else len(search_list[i])
       greater = repl_len - search_len
   - Then proceed to use greater only when search_len > 0 and replacement_list[i] is not None.

Why these fixes work
- They prevent any call to len(None) by ensuring the code only measures lengths for non-None strings, keeping the initial growth estimation consistent with the later checks that skip None replacements.
