The get_paint method currently computes the gray level from the raw input value and raises ValueError if the computed channel is outside 0-255. Tests expect values outside the bounds to be clamped to black or white. I will clamp the input value to [lower_bound, upper_bound] before computing the gray channel. I will also guard against zero-length range to avoid division by zero by returning black (0) in that case. The function will return an (r,g,b) tuple with the same gray channel.
