What happened (root cause)
- The IndexError comes from substring_java(...) being called with end > len(text). That triggers the slice check that raises "String index out of range" (the error you saw at lines 32–33 in your slice function).
- How that happened in your code: you first clamp upper down to len(text) with
  if upper == -1 or upper > len(text):
      upper = len(text)
  but then you have
  if upper < lower:
      upper = lower
  If lower is greater than len(text), that second assignment sets upper back to a value > len(text). Later calls substring_java(text, 0, upper) therefore pass an end index larger than the string length and cause the IndexError.

Where exactly it occurs
- The problematic lines are the two checks that manipulate upper:
  1) the "clamp to len(text)" if-block
  2) the "if upper < lower: upper = lower" block
- The error surface is when substring_java(text, 0, upper) is executed (the slice function raises the IndexError because end > len(s)).

Why this ordering is wrong
- You first clamp upper down, but then you step on that clamp by copying lower into upper without ensuring lower is within bounds. That lets upper exceed len(text) again.

Concrete fixes (pick one)
1) Keep upper always <= len(text) by changing the second check to adjust lower instead of increasing upper:
   if upper == -1 or upper > len(text):
       upper = len(text)
   if upper < lower:
       lower = upper
   Rationale: if lower is too large, bring lower down to the valid upper bound rather than pushing upper out of range.

2) Always clamp upper after adjusting it relative to lower:
   if upper < lower:
       upper = lower
   if upper == -1 or upper > len(text):
       upper = len(text)
   Rationale: ensure final upper is never > len(text).

3) Use min/max in one expression to make intent clear:
   if upper == -1:
       upper = len(text)
   upper = min(upper, len(text))
   upper = max(upper, lower)  # but be careful: this can still exceed len(text) if lower > len -> then min/max order must be chosen to avoid that; prefer option 1 or 2.

Recommended (safe and simple): option 1 — adjust lower to not exceed upper/len. That prevents any subsequent substring_java(..., upper) from receiving an end index > len(text).
