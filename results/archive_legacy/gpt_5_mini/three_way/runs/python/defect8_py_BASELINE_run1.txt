What happened (where it failed)
- Failing input: "fr__POSIX".
- The failure occurs at the country-letter validation at lines 237–240:

  ch4 = locale_str[4]
  if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':

- For "fr__POSIX", locale_str[3] is '_' (the second underscore for an empty country) and locale_str[4] is 'P'. Because ch3 == '_' < 'A' the condition is true and the code raises ValueError("Invalid locale format: fr__POSIX").

Why it happened (root cause)
- The code assumes that when length > 2 the two characters at positions 3 and 4 (indexes 3 and 4) are always an uppercase country code. That assumption fails for the language__variant format where the country is empty (two underscores, e.g. "fr__POSIX"). In that case index 3 is '_' not an uppercase letter, so the strict uppercase check rejects the valid empty-country format.

3. Adjust the variant slice for the empty-country case: variant starts at index 4 (not 6), so use locale_str[4:].
