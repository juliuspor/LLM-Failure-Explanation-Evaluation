What goes wrong
- Where: lines 765–767 (the loop in to_class).
- What: you call array[i].upper() on every element, then append type(array[i]).
- Why it fails: not every element in array is a string. If an element is None or an int (or any non-string object), calling .upper() raises AttributeError: "'int' object has no attribute 'upper'" or "'NoneType' object has no attribute 'upper'". The code never checks the element type before calling .upper(), so the loop crashes as soon as it hits a non-string value.

Exactly what will appear at runtime
- If an element is an int: AttributeError: 'int' object has no attribute 'upper' at line 766.
- If an element is None: AttributeError: 'NoneType' object has no attribute 'upper' at line 766.
- If all elements are strings, no exception is raised and the function returns a list of types (all <class 'str'>).

Why the code likely does not do what you intended
- You call .upper() and assign it to "_" but never use "_" afterward. That suggests either a leftover debug step or the intent was to normalize string items before using them. Meanwhile the function’s return uses type(array[i]) so the .upper() call is pointless for the current output and only risks causing errors.

Concrete fixes (pick one depending on intent)
1) If you only want the element types and do not need .upper(), remove the call:
   - Change lines 765–767 to:
     for i in range(len(array)):
         classes.append(type(array[i]))

2) If you intended to normalize string elements to uppercase before taking type (unnecessary for type), but want to avoid errors for non-strings:
   - Check the type before calling .upper():
     for i in range(len(array)):
         item = array[i]
         if isinstance(item, str):
             _ = item.upper()
         classes.append(type(item))

3) If you intended to operate on string values and skip non-strings:
   - Only process string items and optionally append their types:
     for i in range(len(array)):
         item = array[i]
         if not isinstance(item, str):
             continue
         upper = item.upper()
         classes.append(type(item))

4) If you intended to convert everything to string then uppercase:

     for i in range(len(array)):
         upper = str(array[i]).upper()
         classes.append(type(array[i]))  # still returns original types

Steps to apply a fix
- Decide whether .upper() is needed. If not, remove it (fix 1).
- If .upper() is intended, make the call only when item is a string (fix 2 or 3) or wrap it with try/except to handle non-strings:
     try:
         _ = array[i].upper()
     except AttributeError:
         pass
- Run tests or reproduce the failing input that caused the exception to confirm the change fixes the crash.

Summary
The crash happens because line 766 calls .upper() on array elements without ensuring they are strings. Fix by removing the unnecessary .upper(), or guard the call with an isinstance check (or try/except) so non-strings and None do not cause AttributeError.
