What happened (where it failed)
- The tests call GrayPaintScale.get_paint(value) with values outside the range, e.g. value = -0.5 in test_lower_bound_clamping.
- Inside get_paint the code computes a grayscale component named g and then checks it with:

  75 |             raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- Because the method never clamps the input value to the configured bounds before computing g, a negative input produces a negative g (e.g. -0.5 → fraction < 0 → g < 0). That triggers the check on line 74 and raises ValueError. This is exactly what your tests expect to NOT happen (they expect clamping to 0 and 255 instead of an exception).

Why it failed (root cause)
- Missing clamping of the input value to the GrayPaintScale bounds (lower/upper) before converting to the 0–255 grayscale component. The code assumes g will always fall in 0..255 and enforces that with a raise, rather than handling out-of-range input by clamping.
- The offending check is at lines 74–75, acting on variable g. Because get_paint can produce g < 0 or g > 255 for inputs outside the bounds, the method raises instead of clamping.
