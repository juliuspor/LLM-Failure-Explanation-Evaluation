What happened (what failed)
- The code raises an exception at the pre-sizing loop starting on line 81:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
- That line calls len(...) on replacement_list[i] and search_list[i] without checking whether those elements are None.
- If any element in search_list or replacement_list is None, calling len(None) raises TypeError: object of type 'NoneType' has no len(), so the function crashes there.

Why it happened (root cause)
- Earlier you check/get rid of None or empty entries when scanning the text for matches (the loops that call text.find(...) use guards like "or search_list[i] is None ... or replacement_list[i] is None").
- But the sizing loop (lines 81–84) does not replicate those guards. It assumes every entry is a non-None string and therefore calls len(...) unconditionally.
- Because the arrays may legally contain None elements (the method parameters are Optional[List[Optional[str]]]), the assumption is false and len(...) can be invoked on None — causing the failure exactly at line 82 (len(replacement_list[i]) or len(search_list[i])).
