What happens
- The failure occurs in the loop that computes the "increase" buffer estimate (lines 81–85 in your slice). That loop does:
  greater = len(replacement_list[i]) - len(search_list[i])

- If any element in search_list or replacement_list is None (allowed by your function signature and by the tests), calling len(...) on None raises:
  TypeError: object of type 'NoneType' has no len()

- In your test test_null_safety_inside_arrays_single you call:
  replace_each("aba", ["a"], [None])
  That triggers len(replacement_list[0]) where replacement_list[0] is None, so the function raises a TypeError at that loop before any of the later null-safe checks run.

Why this happens (root cause)
- You intended to skip null/empty search strings and null replacements elsewhere (see the filtering logic in the two "Find match" loops at lines ~59 and ~102–105). But the buffer-size estimation loop (lines 81–85) runs earlier and does not perform the same null/empty checks.
- In short: the buffer-estimate loop is not null-safe; it assumes every element is a valid string and calls len(...) without checking for None.

Exactly where it occurs
- The problematic code is the for i in range(search_length): loop that starts at line 81 (per your slice). It accesses replacement_list[i] and search_list[i] with len() without any None checks.
