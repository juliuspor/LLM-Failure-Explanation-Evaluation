What happened
- In test_abbreviate_lower_beyond_length you call WordUtils.abbreviate("0123456789", 15, 20, None). The input text length is 10, lower = 15, upper = 20.
- The code first clamps upper to the string length:
  if upper == -1 or upper > len(text):
      upper = len(text)   # upper becomes 10
- Next it does:
  if upper < lower:
      upper = lower       # upper becomes 15 (because 10 < 15)
- Later the code calls substring_java(text, 0, upper) with upper == 15. substring_java enforces end <= len(s) (see the slice_backward excerpt) and raises IndexError: "String index out of range: 15". That is the runtime failure you see.

Why this is wrong (root cause)
- When lower is beyond the length of text, the code sets upper = lower (line: if upper < lower: upper = lower). That can produce an upper > len(text). The code then uses upper as the substring end index without clamping it back to len(text). substring_java therefore gets an end index greater than the string length and throws IndexError.
- The code should either detect lower >= len(text) and return the full string immediately, or clamp both lower and upper to the string length before calling index_of and substring_java.

Exactly where it occurs
- The problematic sequence is the two lines:
    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower
  followed by:
    index = StringUtils.index_of(text, " ", lower)
  and the substring_java calls (substring_java(text, 0, upper)) that follow. The IndexError originates in substring_java when end (upper) > len(text).

Concrete fixes (pick one)

Option A — Return the whole string when lower is beyond text length
- After the empty-string check, add:
    if lower >= len(text):
        return text
  This matches the Java behavior and fixes the reported test.

Option B — Clamp lower before using it and before calling substring_java
- Replace the upper/lower adjustment with clamping:
    if upper == -1 or upper > len(text):
        upper = len(text)
    if lower > len(text):
        lower = len(text)
    if upper < lower:
        upper = lower
  This prevents upper from ending up > len(text) and avoids the IndexError.

Option C — When fixing the existing conditional, set upper to len(text) instead of lower
- Change:
    if upper < lower:
        upper = lower
  to:
    if upper < lower:
        upper = len(text)
  (But be careful: this is less explicit than A or B and may not match intended behavior in all edge cases.)

Recommended: use Option A (if lower >= len(text): return text). It directly implements the intended behavior for the test case and avoids any out-of-range substring calls.

Code example for Option A (exact insertion point)
- Immediately after:
    if len(text) == 0:
        return StringUtils.EMPTY
  add:
    if lower >= len(text):
        return text

This will stop the function before any index_of or substring_java calls when lower is beyond the string length.
