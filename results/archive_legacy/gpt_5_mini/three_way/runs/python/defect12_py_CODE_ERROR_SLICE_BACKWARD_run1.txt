What happened (root cause)
- The code allows upper to be set to lower (line: if upper < lower: upper = lower) before ensuring upper is no larger than the string length. If lower > len(text), that makes upper > len(text).
- Later the code calls substring_java(text, 0, upper) (the first call is at the branch starting with "if index == -1:" and there are two other calls returning substring_java(text, 0, upper) and substring_java(text, 0, index)). substring_java checks that end <= len(s) and raises IndexError when end > len(s). That is exactly the exception you saw: "String index out of range: 15".

Where it happens (exact lines)
- The problematic assignment is:
  if upper < lower:
      upper = lower
- The failing call is any of:
  result = substring_java(text, 0, upper)
  return substring_java(text, 0, upper) + ...
- Those calls occur after the if upper < lower clause; if lower > len(text) the subsequent substring_java call will get an end argument > len(text) and throw IndexError.

Why it happens in simple terms
- The code clamps upper down when it’s too big (if upper == -1 or upper > len(text): upper = len(text)), but later may overwrite upper with lower without re-checking the string length. If lower itself is out of bounds (greater than len(text)), you end up passing an out-of-range end index into substring_java.

Concrete fixes (exact steps to apply)
1) Clamp lower to the valid range [0, len(text)] before using it:
   - Add near the top (after handling text is None/empty):
       if lower < 0:
           lower = 0
       if lower > len(text):
           lower = len(text)
   This prevents lower from exceeding len(text) and avoids upper being set to an out-of-range value.

2) Or, if you prefer minimal change, after you do the "if upper < lower: upper = lower" line, re-clamp upper:
       if upper < lower:
           upper = lower
       if upper > len(text):
           upper = len(text)

3) Also ensure index_of is not called with a start beyond len(text). Either clamp lower as in (1), or guard the index_of call:
       if lower >= len(text):
           index = -1
       else:
           index = StringUtils.index_of(text, " ", lower)

Example minimal patch (apply to your code):
    # after handling empty text
    if lower < 0:
        lower = 0
    if lower > len(text):
        lower = len(text)

    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower
    # now upper is guaranteed <= len(text)

Why these fixes work
- They guarantee both lower and upper are in the valid [0, len(text)] range before any substring_java calls or index_of calls that expect a valid start/end. That prevents the substring_java validation from seeing an end index greater than len(s) and raising IndexError.

Summary
- Root cause: upper can be set to lower when lower > len(text), producing an end index > len(text) fed to substring_java.
- Fix by clamping lower (or re-clamping upper) so neither index passed to substring_java is greater than len(text).
