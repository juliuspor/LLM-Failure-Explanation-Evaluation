What happened (where it fails)
- The exception happens at the if on line 176:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- Python evaluates the operands left-to-right. int(x) is evaluated before math.isinf(x) and math.isnan(x).
- If x is float('inf') or float('-inf'), int(x) raises OverflowError.
- If x is float('nan'), int(x) raises ValueError ("cannot convert float NaN to integer").
- Because int(x) runs first, the code crashes on inf/NaN values instead of short-circuiting or being filtered out.

Why it fails (root cause)
- The code tests int(x) before checking that x is finite. Converting special floating-point values to int is invalid and raises exceptions, so the guard conditions must be evaluated before int(x).
