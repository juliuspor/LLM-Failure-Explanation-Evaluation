What happened (where it fails)
- The failure happens on the condition at line 176:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- When x is NaN or infinite, int(x) is evaluated before the math.isinf / math.isnan checks because Python evaluates the left-hand side of and first.
- Calling int(float('nan')) raises ValueError ("cannot convert float NaN to integer") and int(float('inf')) raises OverflowError, so the comparison x == int(x) raises an exception before your isinf/isnan guards run.

Why it fails (root cause)
- The code calls int(x) in the boolean expression before confirming x is a finite number. The checks for math.isinf(x) and math.isnan(x) are written after the int(x) call and therefore are not reached when int(x) itself throws.
- In short: the order of operations in the if condition is wrong — you attempt a conversion that is invalid for NaN/inf before ruling those cases out.

Where in the code to change (specific lines/variables)
- The problematic expression is at line 176 (the if statement shown above). The variables involved are x and the int(x) conversion.
- The safe check must verify finiteness (or non-NaN/non-inf) before calling int(x).

Exact fixes (concrete steps)
1. Check for NaN/inf first, then test integerness. For example:

   This ensures math.isinf/math.isnan are evaluated before int(x).

   if math.isfinite(x):
       int_x = int(x)

           value = int_x

   That way you explicitly convert only after confirming x is finite and you avoid repeated conversions.

Minimal corrected snippet (one direct replacement)
- Replace:
    if x == int(x) and not math.isinf(x) and not math.isnan(x):
  with:
    if math.isfinite(x) and x == int(x):

Why this fix works
- math.isfinite(x) returns False for NaN and infinities, so int(x) is never called on those values and you avoid ValueError/OverflowError.
- The integer test x == int(x) now only runs when it is safe to convert x to an int.

No other parts of the code need to change to address the exception; the rest of the algorithm can run unchanged once x is confirmed finite.
