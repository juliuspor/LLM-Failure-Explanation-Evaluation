What happened (where, exactly)
- The failure occurs at lines 396–397:
  if minutes_offset < 0 or minutes_offset > 59:
      raise ValueError(f"Minutes out of range: {minutes_offset}")

  That check rejects any negative minutes (e.g. -15). The test that called for_offset_hours_minutes(0, -15) gets ValueError: Minutes out of range: -15 because the code demands minutes >= 0, even though the docstring and tests allow minutes in the range -59..+59.

Why it happened (root cause)
- The code uses the wrong range check for minutes. The docstring and tests expect minutes to be allowed between -59 and +59, but the code enforces 0..59.
- The code later reuses the minutes_offset variable and applies different arithmetic depending on the sign of hours:
    hours_in_minutes = hours_offset * 60
    if hours_in_minutes < 0:
        minutes_offset = hours_in_minutes - minutes_offset
    else:
        minutes_offset = hours_in_minutes + minutes_offset
  This logic assumes minutes_offset has a sign and is combined differently for negative hours, but the initial range check prevents negative minute inputs from ever reaching that logic (causing the failing test). The combination logic is also confusing because minutes_offset is overwritten with a total-minutes value; it's clearer to compute a separate total_minutes.
