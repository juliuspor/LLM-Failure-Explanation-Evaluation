What went wrong (what happened and where)
- The crash occurs at line 766: _ = array[i].upper()
- That line assumes array[i] is a string and calls the string method upper(). If array[i] is not a string (for example it is already a class/type object or some other non-string), Python raises AttributeError: 'type' object (or whatever type it actually is) has no attribute 'upper'.
- Because the code never checks the element type before calling .upper(), the error can occur as soon as any element in array is not a str.

Why it happened (root cause)
- The code mixes two different assumptions:
  - It calls .upper() treating the element as a string.
  - It appends type(array[i]) which takes the Python type of the element (not a class lookup by name).
- Likely intention was to accept either string class names or actual class objects, but the code does not branch on which case it has. The unconditional call to .upper() is the direct cause of the crash when a non-string is present.

Exactly where in the code
- Line 765: for i in range(len(array)):
- Line 766: _ = array[i].upper()   ← fails if array[i] is not a str
- Line 767: classes.append(type(array[i]))  ← probably not doing what you expect either

Concrete fixes (step-by-step)
1. Decide the intended input types:

   - If input elements may be either strings or class objects, handle both cases explicitly.
2. Replace the unconditional .upper() with type checks and clear behavior for each case.
3. Return actual class objects instead of type(element) when element is already an instance.
