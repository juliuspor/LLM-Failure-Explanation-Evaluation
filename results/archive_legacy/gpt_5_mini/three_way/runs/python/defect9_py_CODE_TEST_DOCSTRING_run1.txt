What happened
- The function crashes before it ever does any replacements because it calls len(...) on a None element.
- That happens in the "increase" sizing loop:

  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater  # assume 3 matches

- If replacement_list[i] or search_list[i] is None, len(...) raises TypeError ("object of type 'NoneType' has no len()").
- The test test_null_safety_inside_arrays_single passes a None replacement (replacement_list = [None]) so the crash reproduces immediately.

Why it happens (root cause)
- Earlier in the function you do null-safety checks when searching for matches (the for loops that check search_list[i] is None, len(search_list[i]) == 0, or replacement_list[i] is None).
- The "increase" loop is supposed only to estimate output buffer growth, but it does not perform the same null/empty checks before using len(...) on elements of search_list and replacement_list.
- Because that loop is not null-safe, it attempts len(None) and crashes. The function therefore fails for inputs containing None even though the remainder of the algorithm is written to ignore those entries.

Where it occurs (exact location)
- In the function replace_each, directly after the comment:
    # get a good guess on the size of the result buffer so it doesnt have to double if it goes over a bit
  The buggy loop follows that comment (the line with the "BUG:" comment in your code notes the exact problem).

Concrete fix (step-by-step)
1. Skip entries where search_list[i] is None, search_list[i] is empty (len == 0), or replacement_list[i] is None — same conditions used elsewhere in the function.
2. Only call len(...) for entries that pass those checks.

Replace the buggy loop:

    for i in range(search_length):
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches

with a null-safe version, for example:

    for i in range(search_length):
        if (
            search_list[i] is None
            or len(search_list[i]) == 0
            or replacement_list[i] is None
        ):
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches

Result
- After this change the function will no longer call len(None) and will behave consistently with the later match-finding logic: entries with None (or empty search strings) are ignored instead of causing a crash.
