What failed
- The test calls GrayPaintScale.get_paint(-0.5) and the code raises ValueError from these lines in get_paint:
  - Line 74: if g < 0 or g > 255:
  - Line 75:     raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- That happens because the implementation computes color components (r, g, b) and then checks only g for being out-of-range. When the input value is below the lower bound, g becomes < 0 and the method raises instead of clamping to black.

Why it fails (root cause)
1. The method raises when any component is out of range instead of clamping inputs to the [0.0, 1.0] domain and mapping to [0,255]. The tests expect values outside the domain (like -0.5, 1.5) to be clamped to the bounds and return (0,0,0) or (255,255,255).
2. The range check uses only the variable g (line 74). It does not check r and b, so it is incomplete and inconsistent.
3. The error message is misleading: it always says "Red Green Blue" while checking only g.
4. Likely the code also does not clamp the input value before scaling, so intermediate computed components can be negative or >255 and trigger the raise.

Where it occurs
- In GrayPaintScale.get_paint, around lines 74–75 (the posted slice block). The variables involved are the computed color components r, g, b and the incoming parameter value.

What to change (clear, specific fixes)
1. Clamp the input value to the configured lower and upper bounds immediately at the start of get_paint:
   - If value < lower_bound: treat value = lower_bound
   - If value > upper_bound: treat value = upper_bound
   - This will stop negative or >1 intermediate gray values from producing out-of-range RGB values.
2. After computing the gray level, convert to integer and clamp the final RGB components to [0, 255] to be safe.
3. If you still want to raise errors for truly invalid internal state, check all three components (r, g, b) and include them in the error message — but prefer clamping here so tests pass.
4. Replace the single-variable check at lines 74–75 with either no exception (use clamping) or a three-variable check with a clear message.
