What happened (root cause)
- In replace_each the code precomputes a buffer-size "increase" by iterating the pairs and doing:
  greater = len(replacement_list[i]) - len(search_list[i])  (line 82)
- That line runs before the later loop that checks for None (line 105). If replacement_list[i] is None (one of your tests passes None), calling len(replacement_list[i]) raises TypeError.
- So the failure occurs in replace_each at the initial size-estimate loop (around lines 81–85), because replacement_list elements are used without checking for None.

Why that causes the test to fail
- Your tests pass replacement_list containing None (tests test_null_safety_inside_arrays_single and _mixed). The method intends to treat None replacements as "no replacement" (the later code checks replacement_list[i] is None at line 105), but the pre-check tries to measure len(None) first and crashes before the None check can be applied.

Where exactly in the code
- Method: replace_each (top of slice, line ~28).
- Problematic lines:
  - Line 82: greater = len(replacement_list[i]) - len(search_list[i])
  - Line 83–85: use greater to update increase
- The safe None-check exists later at line 105:
  or replacement_list[i] is None
  but it's too late — we've already accessed replacement_list[i].

How to fix it (concrete steps)
1. In the initial loop that calculates increase (lines 81–85), skip entries where replacement_list[i] is None, search_list[i] is None, or search_list[i] is empty. Example logic:
   - if replacement_list[i] is None or search_list[i] is None or len(search_list[i]) == 0:
       continue
   - else compute greater = len(replacement_list[i]) - len(search_list[i])
2. Alternatively set replacement_len = 0 when replacement_list[i] is None, but skipping is clearer because a None replacement means "do nothing" and should not contribute to estimated increase.
3. Make the same None/empty checks consistently wherever replacement_list[i] or search_list[i] are read.

Minimal patch (pseudo-Python change)
Replace the block around line 81–85 with:

for i in range(search_length):
    if search_list[i] is None or replacement_list[i] is None or len(search_list[i]) == 0:
        continue
    greater = len(replacement_list[i]) - len(search_list[i])
    if greater > 0:
        increase += 3 * greater  # assume 3 matches

Result
- After this change the precomputation no longer calls len(None) and the method will handle None replacements the same way the later matching loop does. The tests that pass None in replacement_list should then pass.
