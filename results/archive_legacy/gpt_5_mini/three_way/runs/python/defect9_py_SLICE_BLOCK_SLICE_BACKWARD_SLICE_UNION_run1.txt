What happened (root cause)
- The code assumes every index i in range(search_length) is valid for replacement_list and that replacement_list[i] is a string. That assumption can be false.
- At line 52 you set search_length = len(search_list) and then at lines 59 and 81 you loop for i in range(search_length). At line 82 you do len(replacement_list[i]) without checking replacement_list’s length or contents. If replacement_list is shorter than search_list you get IndexError; if replacement_list[i] is None you get TypeError when calling len(None).

Where it fails (exact places)
- The failure occurs inside replace_each:
  - search_length is set at line 52
  - the loop at line 81 iterates using that length
  - the problematic access is at line 82: greater = len(replacement_list[i]) - len(search_list[i])
  - the conditional at line 83 and arithmetic at 84 happen only after that unchecked access
- Later code (lines 102–108) does validate replacement_list[i] (checks for None etc.), but the earlier loop does not, so the code crashes before those checks run.

Why this leads to an exception
- If len(replacement_list) < len(search_list) → replacement_list[i] raises IndexError at line 82.
- If replacement_list[i] is None (or another non-sequence) → len(replacement_list[i]) raises TypeError at line 82.
- The code never verifies replacement_list length or elements before using len(replacement_list[i]).

How to fix it (concrete steps)
1. Validate inputs up front (recommended). At the start of replace_each (before using search_length):
   - If you require the lists to be the same length, raise a clear ValueError:
     - if len(replacement_list) != len(search_list): raise ValueError("replacement_list and search_list must have the same length")
   - Or, if different lengths are allowed, set loop_length = min(len(search_list), len(replacement_list)) and use that for loops that index both lists.
