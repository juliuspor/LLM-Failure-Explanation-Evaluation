- Because inferred_type becomes list (the container class), the later condition

never matches the expected cases. In the specific failing test (add(None, None, expected_type=str)):
  - array is None, element is None → inferred_type = object (line with else inferred_type = object).
  - Then new_list = _copy_list_grow1(array, inferred_type) makes an object-typed list (intended).
  - Immediately after you run the check that raises a TypeError whenever inferred_type == object and expected_type != object. That raises when the caller expects str, although the correct behavior is to produce a list usable as List[str] (i.e. [None]) — so the code raises a TypeError incorrectly.
- In other scenarios (non-empty array) the wrong use of type(array) means you infer list instead of the element type, so downstream logic that expects an element/component type behaves incorrectly or inconsistently.
