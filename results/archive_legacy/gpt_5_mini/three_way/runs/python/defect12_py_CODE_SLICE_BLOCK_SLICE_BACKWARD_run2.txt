What happened (where it fails)
- The code raises IndexError inside substring_java(text, 0, upper).
- The slice check shown in your trace (lines 32–33) rejects end > len(s) and raises: "String index out of range: {end}".
- That happens when upper > len(text) at the time substring_java is called.

Why it happens (root cause)
- You clamp upper to len(text) only once at the top:
    if upper == -1 or upper > len(text):
        upper = len(text)
- But later you do:
    if upper < lower:
        upper = lower
  This can increase upper beyond len(text) (when lower > len(text)).
- If lower > len(text), that second assignment makes upper > len(text), and you never re-clamp upper before calling substring_java(text, 0, upper). substring_java then receives an end index bigger than the string length and raises the IndexError shown.

Exactly where (which variables and lines)
- index = StringUtils.index_of(text, " ", lower) — this uses lower as a start but is not the crash point.
- The crash occurs at calls like substring_java(text, 0, upper) later in:
    result = substring_java(text, 0, upper)
  or
    return substring_java(text, 0, upper) + ...
  after upper was set to lower (when upper < lower).
- The problematic assignment is the if upper < lower: upper = lower line, which can make upper exceed len(text) if lower > len(text).

How to fix it (concrete steps)
Option A (recommended, simple):
- After you change upper to lower, clamp it again:
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)
This guarantees upper is never > len(text) before calling substring_java.

Option B (handle lower first):
- Clamp lower to the string length up front:
    if lower > len(text):
        lower = len(text)
- Then keep your existing logic (so upper < lower will not produce upper > len(text)).

Option C (defensive checks before substring call):
- Before every substring_java(text, 0, upper) call, ensure upper <= len(text) (e.g. upper = min(upper, len(text))).

Any of these will prevent passing an out-of-range end to substring_java and stop the IndexError.
