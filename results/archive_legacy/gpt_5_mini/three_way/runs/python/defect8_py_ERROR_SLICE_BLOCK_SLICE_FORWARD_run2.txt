Why that is the root cause
- The code assumes that after the first underscore there will always be a two-letter country code (positions 3 and 4). It does not allow the common form where the country is omitted and the variant follows immediately after a second underscore (language__variant), which is exactly what "fr__POSIX" is.
- The failing check (lines 239–240) therefore rejects valid locale strings of the form language__variant because ch3 is '_' instead of an uppercase letter.
