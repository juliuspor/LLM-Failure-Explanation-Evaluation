What happened (root cause)
- The test input is ["Test", None, 99.0]. ClassUtils.to_class loops over the input array and treats each element as if it supports string operations.
- When the loop reaches the None element, the code calls a string method on it (e.g. array[i].upper()). None has no attribute upper, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The failure happens during the to_class conversion loop. The slice you provided shows one intended action on each element is classes.append(type(array[i])) at line 767, but elsewhere in the same method the code calls .upper() (or another attribute) on array[i] before appending the type. That call is what triggers the AttributeError for None elements.

Where it occurred
- Method: ClassUtils.to_class
- Variable names: array (the input list), i (loop index), classes (the result list)
- Line: the error arises in the element-processing code inside the loop (the snippet you showed shows classes.append(type(array[i])) at line 767, but there is an earlier call that does array[i].upper() on the same element).

Why this is wrong
- None is a valid list element; to_class should treat None as "no class" and map it to type(None) (or to None, depending on intended semantics). It must not call string methods on elements without checking for None.
- Calling array[i].upper() assumes every element is a str. That assumption fails for None and for non-string types (and is unnecessary for discovering an object's class).

Concrete fixes (step-by-step)
1. Guard against a None input array:
   - If array is None, return None (your tests expect that).
2. In the per-element loop, check for None before calling any attribute or method on the element.
3. Use Python's type() to get the element's class rather than calling string methods on the element. For None, return type(None).

Minimal corrected implementation sketch:

def to_class(array):
    if array is None:
        return None
    classes = []
    for v in array:
        # handle None explicitly
        if v is None:
            classes.append(type(None))
        else:
            classes.append(type(v))
    return classes

If you want to preserve an API that returns None for elements (instead of type(None)), change the None branch to classes.append(None).
