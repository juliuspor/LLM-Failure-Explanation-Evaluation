What happened (root cause)
- At line 2 (the first if-block) you do:
  inferred_type = type(array)
  But type(array) gives you the Python class of the list object itself (i.e. list), not the element/component type you intended.
- That wrong value is passed into ArrayUtils._copy_list_grow1 as the component type. That helper then creates a new list with the wrong component/type metadata (effectively an "object" component type in your environment).
- Later the code checks:
  if inferred_type == object and expected_type != object:
      raise TypeError(...)
  Because the copy ended up as an object-typed list, this condition becomes true and you raise the TypeError complaining you cannot cast an object list to a str list.

Where it occurs (line references)
- Line where you set inferred_type when array is present:
    if array is not None:
        inferred_type = type(array)        # <-- wrong: this is the list's type, not element type
- Line where _copy_list_grow1 is called:
    new_list = ArrayUtils._copy_list_grow1(array, inferred_type)
- Line where the TypeError is raised:
    if inferred_type == object and expected_type != object:
        raise TypeError(...)

Why that causes the specific error shown
- Because you passed the wrong inferred_type into _copy_list_grow1, the copied list ended up with "object" as its component type. Then with expected_type == str, the equality check detects the object->str mismatch and raises:
  TypeError: Cannot cast object list to str list (...)

How to fix it (clear concrete steps)
1. When array is not None, set inferred_type from the element type, not from the list object. For example:
   - If array is non-empty: inferred_type = type(array[0])
   - If array is empty: you cannot infer from elements; either use expected_type if provided, or default to object.
2. Prefer this logic order:
   - If array is not None:
       - If array has elements: inferred_type = type(array[0])
       - Else if expected_type is not None: inferred_type = expected_type
       - Else inferred_type = object
   - Else if element is not None: inferred_type = type(element)
   - Else: inferred_type = object
3. Then call ArrayUtils._copy_list_grow1(array, inferred_type) with that inferred_type.

Example corrected snippet (conceptual)
    if array is not None:
        if len(array) > 0:
            inferred_type = type(array[0])
        elif expected_type is not None:
            inferred_type = expected_type
        else:
            inferred_type = object
    elif element is not None:
        inferred_type = type(element)
    else:
        inferred_type = object

Why this will stop the error
- You will supply the actual element/component type (e.g. str) into _copy_list_grow1 instead of list, so the new list won't be treated as an "object list" and the later type-check won't raise when expected_type is str.

One more note
- If the existing array contains elements of mixed types, picking array[0] may not reflect all elements. Choose the behavior you want in that case (raise, coerce to object, or validate all elements), but the current code's use of type(array) is definitely incorrect.
