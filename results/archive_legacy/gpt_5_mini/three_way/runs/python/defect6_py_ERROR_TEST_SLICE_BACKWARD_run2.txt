What happened (what failed)
- The test called CodeConsumer.add_number(-0.0).
- On line 176 the code does: if x == int(x) and not math.isinf(x) and not math.isnan(x):
- For x == -0.0 that equality is True (Python treats -0.0 == 0.0), so the code takes the "integer" branch and prints int(x), producing "0".
- That loses the sign bit of negative zero, so the test expected "-0.0" but got "0" (AssertionError).

Why it happened (root cause)
- The branch that formats integer-looking numbers runs before any special-case handling of negative zero.
- Numeric equality x == int(x) is true for -0.0, so negative zero is mis-classified as an ordinary integer zero.
- The code therefore formats by converting to int and printing "0", dropping the negative sign encoded in the float's sign bit.
- The failing spot is where x (the input value) is tested at line 176; the negative-zero detection must be applied before that test.
