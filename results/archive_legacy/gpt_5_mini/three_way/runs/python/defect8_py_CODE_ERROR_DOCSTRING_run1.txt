What happened
- The input "fr__POSIX" raised ValueError at the lines that validate ch3 and ch4:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
    raise ValueError(...)
- For "fr__POSIX", locale_str[3] is '_' (because the country part is empty and you get two underscores in a row). The code expects an uppercase country letter at index 3, so that check fails and the code rejects a perfectly valid locale string that uses an empty country and a variant.

Why it fails (root cause)
- The method assumes that whenever length != 2 you must have a two-letter country at indices 3 and 4. It does not handle the case where the country is empty and the form is language + "_" + "_" + variant (for example "fr__POSIX").
- In other words, the validation around ch3/ch4 is too strict: it never allows the country to be empty when a variant is present.

Where it occurs in the code
- Immediately after checking locale_str[2] == '_' you assign:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - then validate ch3/ch4 as uppercase letters.
- That validation is the failing point for inputs like "fr__POSIX".

Exact fix (step-by-step)
1. After confirming locale_str[2] == '_', test whether the country is empty by checking locale_str[3] == '_':
   - If locale_str[3] == '_' then the country is empty; the variant starts at index 4 (variant = locale_str[4:]).
     - Validate that the variant is non-empty (raise ValueError if empty).
     - Return Locale(locale_str[0:2], "", variant).
2. Otherwise (country is present), keep the existing checks for ch3/ch4 to be uppercase letters, then:
   - If length == 5 return Locale(lang, country)
   - If length > 5 require locale_str[5] == '_' and return Locale(lang, country, variant) with variant = locale_str[6:].

Concrete code change (replace the current ch3/ch4 block)
- Replace:
    ch3 = locale_str[3]
    ch4 = locale_str[4]
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(...)
    if length == 5:
        return Locale(locale_str[0:2], locale_str[3:5])
    else:
        if locale_str[5] != '_':
            raise ValueError(...)
        return Locale(locale_str[0:2], locale_str[3:5], locale_str[6:])
- With:
    # handle empty country (double underscore) -> variant-only
    if locale_str[3] == '_':
        variant = locale_str[4:]
        if not variant:
            raise ValueError(f"Invalid locale format: {locale_str}")
        return Locale(locale_str[0:2], "", variant)
    # country present: validate two uppercase letters
    ch3 = locale_str[3]
    ch4 = locale_str[4]
    if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
        raise ValueError(f"Invalid locale format: {locale_str}")
    if length == 5:
        return Locale(locale_str[0:2], locale_str[3:5])
    if locale_str[5] != '_':
        raise ValueError(f"Invalid locale format: {locale_str}")
    variant = locale_str[6:]
    if not variant:
        raise ValueError(f"Invalid locale format: {locale_str}")
    return Locale(locale_str[0:2], locale_str[3:5], variant)

Notes
- This change keeps the strict checks you described (language lowercase, country uppercase, separators are underscores) while allowing the valid "lang__variant" form like "fr__POSIX".
- Also validate that the variant is non-empty when expected (avoid accepting "fr__").
