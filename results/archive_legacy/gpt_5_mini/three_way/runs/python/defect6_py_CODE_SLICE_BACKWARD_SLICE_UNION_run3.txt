What happened (where it failed)
- The failure occurs at line 176 in the condition:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- Python evaluates the leftmost subexpression first, so it tries to call int(x) before checking math.isinf(x) or math.isnan(x).
- If x is math.nan or math.inf, int(x) raises an exception (ValueError for nan, OverflowError for inf in some cases), so you get a crash before the guard checks run.

Why it failed (root cause)
- The code intended to avoid converting NaN/Inf to int, but the ordering of the checks calls int(x) first. The guard comes too late. int(x) is not safe for NaN/Inf, so the code raises instead of skipping the integer branch.

Exact place: line 176 (the if condition). Exact variables/methods involved: x, int(x), math.isinf(x), math.isnan(x).
