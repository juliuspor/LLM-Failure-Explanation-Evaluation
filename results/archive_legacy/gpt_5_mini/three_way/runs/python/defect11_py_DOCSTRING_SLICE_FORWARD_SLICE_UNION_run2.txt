- This can also happen for other ranges (e.g., non-letter Unicode blocks), so the failure is deterministic whenever the requested character type(s) and the start/end range do not overlap.

Where it occurs
- The root mismatch occurs between:
  - gap calculation and random selection: gap = end - start (line 49 and 56)
  - the acceptance test: the if at line 57
- The observable failure is that line 58 is never executed because the condition at line 57 never becomes true for the chosen range.

1. Validate the input range before entering the loop:
   - Build or test whether there exists at least one code point between start (inclusive) and end (exclusive) that satisfies the chosen masks (letters/numbers).
   - If none exists, raise a ValueError describing the mismatch (for example: "no characters of requested type in given range").

2. Or change selection strategy:
