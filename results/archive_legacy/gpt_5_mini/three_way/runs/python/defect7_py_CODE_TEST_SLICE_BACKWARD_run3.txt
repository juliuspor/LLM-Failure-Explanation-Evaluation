What happened (where): in the loop at lines 765–766 your code calls array[i].upper():

 765 |         for i in range(len(array)):
 766 |             _ = array[i].upper()

Why it fails (root cause): calling .upper() assumes every element of array is a string. If array[i] is None (or any non-string object such as an int or float), Python raises AttributeError because that object has no .upper() method. The test that fails uses an input like ["Test", None, 99.0], so when i == 1 the code tries None.upper() and crashes.

Why the .upper() is wrong here: the purpose of to_class is to return the element types (you already do classes.append(type(array[i]))). The temporary call to .upper() is unnecessary and unsafe — it both causes the exception and serves no purpose for the function’s output.

Concrete fixes (step-by-step)
1. Remove the .upper() call entirely. Replace the loop with a simple type extraction that handles None:
   for i in range(len(array)):
       classes.append(type(array[i]))

2. Alternatively, if you prefer explicit handling, do:
   for i in range(len(array)):
       if array[i] is None:
           classes.append(type(None))
       else:
           classes.append(type(array[i]))

3. (Optional cleanup) Iterate directly over elements instead of indexing:
   for elem in array:
       classes.append(type(elem))

4. Ensure ArrayUtils.EMPTY_CLASS_ARRAY.copy() returns a plain list for the empty-array case (that line is fine if EMPTY_CLASS_ARRAY is a list; otherwise return []).

Minimal corrected function example:

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for elem in array:
            classes.append(type(elem))
        return classes

This removes the .upper() call that caused the AttributeError and correctly returns type(None) for None elements.
