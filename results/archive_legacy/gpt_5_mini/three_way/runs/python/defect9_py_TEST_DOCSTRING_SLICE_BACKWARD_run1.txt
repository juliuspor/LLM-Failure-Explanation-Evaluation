What happened
- The function replace_each crashes when either search_list[i] or replacement_list[i] is None because the code calls len(...) on those entries without checking for None.
- From the slice you gave, the crash occurs at lines 81–82:

  That line assumes both replacement_list[i] and search_list[i] are strings, but if either is None Python raises TypeError: object of type 'NoneType' has no len().

Why it happens (root cause)
- Earlier in the function there is some handling that may skip None entries (the snippet shows another loop around line 59), so the author intended to be null-safe. But that safety is incomplete: one loop checks for None, another later uses the same arrays without re-checking. In short, the code is inconsistent about null-safety: it sometimes treats None as “ignore this pair,” but later it still tries to compute lengths of those same entries.
- Specifically:
  - search_length = len(search_list) is computed at line 52 and used to drive loops.
  - The loop at line 59 may skip or ignore None entries.
  - The loop at lines 81–82 still indexes every i in range(search_length) and does len(replacement_list[i]) and len(search_list[i]) without guarding for None, so a None entry causes the crash.

- Add guards before any use of len(...) on list entries. For example, at the top of the loop that contains line 81, do:
  if search_list[i] is None or replacement_list[i] is None:
      continue
  Then it is safe to call len() on both entries.
