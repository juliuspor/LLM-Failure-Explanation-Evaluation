What happened (where it fails)
- The failure happens inside the for loop:
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))

- If any array element is None, the expression array[i].upper() tries to call .upper() on None and raises AttributeError (NoneType has no attribute 'upper'). That is the root cause of the test failure (test_to_class_with_none_element and other None tests).

Why that line is wrong
- The method's purpose is to convert each element to its class/type. There is no need to call .upper() on the element at all.
- Calling .upper() is:
  - unnecessary for determining type,
  - assumes each element is a string-like object with an upper method,
  - causes an exception for None and for non-string types that don't have upper().
- The docstring says "If any of these objects is None, a None element will be inserted into the array." The current code violates that by raising instead of inserting type(None) or None into the output.

Exact places to change
- Remove the line: _ = array[i].upper()
- Use a None check or simply append type(element) (type(None) for None elements). For example:
  elem = array[i]
  classes.append(type(elem))

Suggested corrected implementation
- Replace the function body with:

    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for elem in array:
            # If elem is None, type(None) will be appended
            classes.append(type(elem))
        return classes

Notes and small considerations
- This fixes the AttributeError and makes tests expecting type(None) pass.
- If ArrayUtils.EMPTY_CLASS_ARRAY.copy() returns something other than an empty list (tests expect []), verify that it is an empty list; otherwise return [].
- Alternatively, if the intended behavior in Java was to insert null rather than a class object representing null, adjust to: classes.append(None) for None elements. Your tests expect type(None), so append type(elem) is correct here.

Summary
The code fails because of an unneeded call to array[i].upper() that raises AttributeError when array[i] is None. Remove that call and directly append type(array[i]) (or check for None and append type(None) explicitly) to fix the bug.
