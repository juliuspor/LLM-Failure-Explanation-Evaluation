What failed
- The code raises an IndexError coming from the small helper shown at lines 30–32:
  30 |     if index < 0 or index >= len(s):
  31 |         raise IndexError(f"String index out of range: {index}")
  32 |     return s[index]

Why it fails (root cause)
- The caller computes an index that equals len(s) and then calls this helper. Python string indices run from 0 to len(s)−1, so an index of len(s) is out of range and triggers the check on line 30, producing the IndexError.
- In other words, there is an off-by-one mistake: the code is trying to read s[len(s)] instead of s[len(s)-1] (or use s[-1] for the last character). The check on line 30 is correct for guarding against out-of-range access, so the visible failure is caused by the caller computing the wrong index value.

Where it happened
- The exception is raised inside the helper at the check on line 30 when index == len(s).
- The immediate symptom is the IndexError with the message "String index out of range: {index}".

How to fix it (concrete steps)
1. Find where the index is computed in the caller of this helper (the code that passes index to this function). That code is producing index == len(s). Change that computation so you use:
   - len(s) - 1 if you intended the last character, or
   - an index in the range 0 .. len(s)-1 for the desired character.
   Example for last char: last_char = s[len(s) - 1] or last_char = s[-1]
2. Add defensive checks before calling the helper:
   - Reject None (if s can be None) and empty strings explicitly with a clear ValueError so tests fail with a meaningful error instead of IndexError.
   - Example:
       if s is None or s == "":
           raise ValueError("input string must be non-empty")
3. (Optional) Keep the helper’s check as-is (lines 30–31). It is correct and should remain to prevent invalid accesses.

Why this explanation rules out other possibilities
- The helper’s check is explicitly detecting an out-of-range index. That means the failure point is an incorrect index value, not e.g. a broken indexing operation or a malformed string. The fix must therefore correct the index calculation or validate inputs earlier.
