What happens
- The function crashes when it tries to compute len(replacement_list[i]) for an entry that is None.
- The crash is a TypeError: "object of type 'NoneType' has no len()".
- It happens in the initial size-estimation loop at lines 81–85:

  81 |         for i in range(search_length):
  82 |             greater = len(replacement_list[i]) - len(search_list[i])
  83 |             if greater > 0:
  84 |                 increase += 3 * greater  # assume 3 matches
  85 |         increase = min(increase, len(text) // 5)

Why this is wrong (root cause)
- The code later (lines 102–105) explicitly treats None entries as "no more matches / skip":

  102 |                     no_more_matches_for_repl_index[i]
  103 |                     or search_list[i] is None
  104 |                     or len(search_list[i]) == 0
  105 |                     or replacement_list[i] is None

  That shows the intended behavior: if replacement_list[i] is None (or search_list[i] is None/empty) it should be skipped.
- But the size-estimation loop (lines 81–85) runs earlier and unconditionally calls len(replacement_list[i]) and len(search_list[i]). If replacement_list[i] is None (as in your failing tests), calling len(None) raises the TypeError before the later guard can prevent anything.

Where you see the failure in your tests
- test_null_safety_inside_arrays_single and test_null_safety_inside_arrays_mixed pass a None in replacement_list. The code crashes while executing the size-estimation loop at line 82.

Clear fix (concrete steps)
1. In the size-estimation loop (around lines 81–85) skip entries that are None or empty exactly the same way the later matching loop does. Replace:

    for i in range(search_length):
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches

with a guarded version, e.g.:

    for i in range(search_length):
        if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches

2. Optionally add the same guard before any other place that calls len() on search_list[i] or replacement_list[i] to ensure no other early access causes the same crash.

Why this fix works
- It prevents len(None) from being called. It makes the size-estimation loop consistent with the later matching logic (which already skips None/empty entries). After this change the function will treat a None replacement as "no replacement" and your tests will behave as expected.
