What went wrong (root cause)
- On line 3 you do:

  but type(array) returns the type of the container (normally <class 'list'>), not the component/element type you intend to carry forward. The rest of the function treats inferred_type as the element/component type, so you end up passing the wrong type information into ArrayUtils._copy_list_grow1 and into later checks.
- Consequence: ArrayUtils._copy_list_grow1 is given the list class instead of the element class. That produces a list with the wrong "component" type (or triggers whatever type logic ArrayUtils uses), so when you later assign element into the new list you get a type mismatch / ClassCast-like error at runtime.
- A secondary issue is the check:
  if inferred_type == object and expected_type != object:
  This assumes inferred_type will be the element class or the literal object class. But because inferred_type can be <class 'list'>, this comparison won't express the intended semantics (and using equality on types here is brittle).
