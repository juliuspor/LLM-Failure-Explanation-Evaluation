What happened (what failed)
- In ClassUtils.to_class, the loop at lines 765–767 calls array[i].upper() for every element:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))
- When an element is None (or any non-string like int/float), None has no method upper(), so the call raises AttributeError: "'NoneType' object has no attribute 'upper'".
- That is exactly why your test test_to_class_with_none_element fails (and any test with a None element). The code attempts to use .upper() unconditionally instead of only when the element is a string.

Why this is the root cause (clear and concrete)
- Variable names and lines: array[i] is referenced at lines 766 and 767. The offending call is array[i].upper() on line 766.
- The purpose of to_class is to return the element types. Calling .upper() is unnecessary for determining type and is unsafe for non-strings or None.
- For a None element, Python immediately raises AttributeError at line 766 before it can append the type at line 767. So the method never returns the expected [type(None), ...].

How to fix it (specific steps and code)
Option A — simplest correct change: remove the .upper() call
- Replace the loop with:
  for i in range(len(array)):
      if array[i] is None:
          classes.append(type(None))
      else:
          classes.append(type(array[i]))
- This avoids calling any string-only method and records type(None) for None elements.

Option B — only call .upper() when you actually need to operate on strings
- If .upper() was there for a reason (e.g. case-insensitive logic), guard it:
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          classes.append(type(None))
      elif isinstance(elem, str):
          _ = elem.upper()   # safe: elem is a string
          classes.append(str)
      else:
          classes.append(type(elem))

Option C — compact / Pythonic:
  if array is None:
      return None
  return [type(x) for x in array]

Other notes to apply

- Make sure to return type(None) for None elements (type(None) is the canonical representation).

Summary
- The failure is caused by calling array[i].upper() on every element (line 766). None (and non-strings) have no .upper(), so that raises AttributeError before the type is appended at line 767.
- Fix by not calling .upper() unconditionally — either remove it entirely or guard it with isinstance(elem, str) and handle None explicitly.
