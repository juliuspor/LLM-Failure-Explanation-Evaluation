What happened (what failed)
- The test fed replacement_list entries that are None (e.g. ["c", None]).
- At line 82 the code does:
  greater = len(replacement_list[i]) - len(search_list[i])
- When replacement_list[i] is None, len(replacement_list[i]) raises TypeError: object of type 'NoneType' has no len().

Where it occurs
- In replace_each, at/around:
  - line 52: search_length = len(search_list) (this is fine)
  - loop starting line 59 (iterating i)
  - line 81–82: a calculation uses len(replacement_list[i]) and len(search_list[i])
- The failing expression is len(replacement_list[i]) when replacement_list[i] is None.

Why it happens (root cause)
- The code assumes every element of replacement_list and search_list is a string and calls len(...) on each element without checking for None.
- The tests expect null-safety: a None replacement should be treated as "no replacement" (or as an empty string), but the code instead blindly calls len() on None and crashes.

1. Before calling len() on list elements, check for None and handle it. Decide how you want to treat None:
   - Treat None as an empty string ('') so replacements calculate length 0 and use '' when replacing, or
   - Skip that index entirely (no replacement) if replacement_list[i] is None.
2. Update the code at the len() usage and anywhere the code actually uses replacement_list[i] to substitute, for example:

Replace this kind of line (around 81–82)

With a safe version:
  repl_i = replacement_list[i] if replacement_list[i] is not None else ''
  search_i = search_list[i] if search_list[i] is not None else ''
  greater = len(repl_i) - len(search_i)

And later, when applying the replacement, use repl_i ('' if None) instead of replacement_list[i].

3. Also check any other places where you call len(...) or use the element directly (insertion, concatenation, comparisons): use the same None-to-'' mapping or skip logic consistently.
