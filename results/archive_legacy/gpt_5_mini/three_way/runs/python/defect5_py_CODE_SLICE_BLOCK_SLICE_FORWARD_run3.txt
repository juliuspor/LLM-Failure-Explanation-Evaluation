What went wrong (root cause)
- At line where you set inferred_type you do:
  - if array is not None:
      inferred_type = type(array)
  That takes the Python type of the list object (i.e. <class 'list'>), not the type of the list elements. ArrayUtils._copy_list_grow1 and the later logic expect the element (component) type, not the list container type. Because inferred_type is the list class, later code builds an array/list with the wrong component type and your cast check (lines 346–351) and assignment fail in ways that produce the ClassCastException message you see.

Exactly where it happens
- The problematic assignment is at the first branch of inferred_type (the "if array is not None" branch). That branch runs whenever array is not None — even when array is an empty list — and sets inferred_type = type(array) (the list type) instead of the element type.
- This wrong value is passed to ArrayUtils._copy_list_grow1 (call site shown), which creates a new list/array using that type. When you later assign element into new_list[len(new_list) - 1], the container’s component type may be incompatible with element and you get the cast-like error the code is trying to detect (the error message in lines 346–351).
