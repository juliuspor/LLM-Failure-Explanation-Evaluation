What fails
- The failure happens in the size-guess loop that starts at the comment "BUG: this loop is not null-safe for elements inside the arrays" (the loop beginning at about line 81).
- Inside that loop the code does:
  greater = len(replacement_list[i]) - len(search_list[i])
  That assumes replacement_list[i] and search_list[i] are strings. If either is None, calling len(None) raises TypeError and the code crashes.
- This exact situation is hit by your tests:
  - test_null_safety_inside_arrays_single calls replace_each("aba", ["a"], [None]) — replacement_list[0] is None, so len(replacement_list[0]) raises a TypeError in that loop.
  - test_null_safety_inside_arrays_mixed also includes a None replacement and would hit the same problem.

Why it happened (root cause)
- Earlier in the function, when searching for matches, the code explicitly guards against None or empty elements:
    if no_more_matches_for_repl_index[i] or search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
        continue
  So the matching logic is null-safe.
- The size-guess loop, however, does not repeat those null/empty checks and directly computes lengths, so it is not null-safe. The mismatch between the two places (safe checks vs unsafe assumptions) is the root cause.

Where exactly (variable & line references)
- Variables: replacement_list, search_list, search_length, increase.
- Problematic lines: the loop labelled with the BUG comment (around line 81) and specifically the line computing greater = len(replacement_list[i]) - len(search_list[i]) (around line 82).

How to fix it (concrete steps)
1. Make that loop skip elements that are None or empty exactly the same way the matching loops do. Replace the loop body with a guard identical to the search loops.
   Example change:
   for i in range(search_length):
       if (
           no_more_matches_for_repl_index[i]
           or search_list[i] is None
           or len(search_list[i]) == 0
           or replacement_list[i] is None
       ):
           continue
       greater = len(replacement_list[i]) - len(search_list[i])
       if greater > 0:
           increase += 3 * greater

2. Alternatively, compute lengths safely with conditional expressions:
   for i in range(search_length):
       s = search_list[i] or ""
       r = replacement_list[i] or ""
       if s == "" or r == "":
           continue
       greater = len(r) - len(s)
       ...

Either approach prevents calling len() on None and keeps behavior consistent with the rest of the function.

What to expect after the fix
- The TypeError goes away.
- The null-safety tests (replace with None inside arrays) will pass because the size-guess loop no longer tries to measure None entries and behaves consistently with the matching logic.
