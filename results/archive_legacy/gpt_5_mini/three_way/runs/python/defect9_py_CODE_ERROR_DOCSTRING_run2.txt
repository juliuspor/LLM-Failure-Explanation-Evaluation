What happened
- The code raises TypeError: object of type 'NoneType' has no len() because the loop that computes increase calls len(...) on list elements that can be None.
- Specifically, this line (the "BUG" comment is right above it) runs before any null checks:
    greater = len(replacement_list[i]) - len(search_list[i])
  If search_list[i] or replacement_list[i] is None this becomes len(None) and crashes.

Where it occurs
- In replace_each, after the first match-finding block and before constructing parts:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater
- That loop is at the lines labeled by the "get a good guess ..." comment and the "BUG: this loop is not null-safe ..." comment.

Why it happens (root cause)
- Earlier loops explicitly skip entries that are None or have empty search strings:
    if no_more_matches_for_repl_index[i] or search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
        continue
  But the increase-calculation loop does not perform the same checks. So when either search_list[i] or replacement_list[i] is None, calling len(...) raises TypeError.
- The code intends to ignore None replacements/searches, but this one loop forgot to be null-safe.

How to fix it (explicit steps)
1. Add the same null/empty checks to the increase loop so it skips unsafe elements. Replace that loop with:

    for i in range(search_length):
        if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater

(You can also include no_more_matches_for_repl_index[i] if you want to skip entries already known to have no matches — not required here but consistent.)

2. Alternative: compute safe lengths without raising, e.g.:

    s_len = len(search_list[i]) if isinstance(search_list[i], str) else 0
    r_len = len(replacement_list[i]) if isinstance(replacement_list[i], str) else 0
    greater = r_len - s_len
    if greater > 0:
        increase += 3 * greater

Either approach prevents calling len(None) and matches the intended behavior of ignoring None elements.

Why these fixes work
- They prevent len(...) from being called on None by skipping or treating None as length 0, matching the rest of the function's logic that ignores None/empty entries. This removes the TypeError and keeps the function behavior consistent with the documented intent.
