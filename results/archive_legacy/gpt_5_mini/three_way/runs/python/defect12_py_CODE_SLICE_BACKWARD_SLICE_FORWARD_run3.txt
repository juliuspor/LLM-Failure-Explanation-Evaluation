What happened (what error you saw)
- The code raises an IndexError from substring_java (the slice check at lines 32–34), because substring_java is called with end > len(s). The slice helper rejects an end index larger than the string length and throws: "String index out of range: {end}".

Why it happens (root cause, exact variable and line locations)
- On these lines you clamp upper to the string length:
  if upper == -1 or upper > len(text):
      upper = len(text)
- Immediately after you allow upper to be pushed back up to lower:
  if upper < lower:
      upper = lower
- If lower is greater than len(text), the second assignment sets upper to a value larger than len(text). That reintroduces an illegal end index. Later calls such as:
  return substring_java(text, 0, upper)  # triggers slice_forward check
  will pass end = upper > len(text), causing the IndexError in slice_forward (line 34).

A concrete example you can run to reproduce
- text = "abc", lower = 5, upper = 5
  Step through the code:
  - first clamp: upper > len(text) → upper = 3
  - next: upper < lower → upper = 5
  - substring_java(text, 0, 5) → end = 5 > len(text)=3 → IndexError

How to fix it (clear, actionable steps)
Option A — clamp upper after respecting lower (minimal change)
- Swap the two checks so you ensure upper >= lower first and then clamp to len(text):
  if upper < lower:
      upper = lower
  if upper == -1 or upper > len(text):
      upper = len(text)
This prevents setting upper back above len(text).

Option B — compute upper with min/max in one expression (safer & clearer)
- Replace the two lines with:
  if upper == -1:
      upper = len(text)
  upper = max(lower, min(upper, len(text)))
This ensures upper is at least lower and never greater than len(text).

Option C — guard lower separately if that’s invalid in your semantics
- If lower should never exceed len(text), clamp lower early:
  lower = min(lower, len(text))
  then clamp upper as needed (either Option A or B).
Choose this only if lowering 'lower' matches your intended behavior.
