What happened
- Input like "l" or "L" reaches the branch that handles trailing 'l'/'L'.
- You compute numeric = val[:-1], which for "l" becomes the empty string "".
- On the next line you call char_at(numeric, 0). That tries to index into an empty string, which raises IndexError: "String index out of range: 0".

Why it fails (root cause)
- The code assumes numeric has at least one character before calling char_at(numeric, 0). It does not check len(numeric) == 0.
- The defect is pointed out by your inline comment: "# BUG: missing guard for len(val) == 1 and non-digit input." When val has length 1 and its single character is a non-digit qualifier ('l'/'L'), numeric becomes empty and the char_at call is out-of-range.
- The error originates at the char_at function (your slice-backward shows the IndexError raised at the index check) and is triggered from the conditional in the L/ l branch.

Exactly where in your code
- numeric = val[:-1] produces "" for val == "l" or "L".
- The condition line that follows:
    if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
  calls char_at(numeric, 0) without verifying numeric is non-empty, causing the IndexError.

How to fix it (concrete steps)
1. Add a guard that handles empty numeric before accessing numeric[0] or numeric[1:].
2. Replace the fragile condition with one that checks lengths first. For example:

    if numeric.isdigit() or (len(numeric) > 1 and numeric[0] == "-" and numeric[1:].isdigit()):
        return int(numeric)

This ensures you only access numeric[0] and numeric[1:] when numeric has the required length.

Alternative (shorter) options
- Use numeric.lstrip("-").isdigit() to allow a single leading '-' correctly but still reject the empty case:
    if numeric.lstrip("-").isdigit() and not (numeric == "-"):
        return int(numeric)
  (Careful: numeric == "-" should be rejected.)

What will change after the fix
- Inputs like "l" or "L" will now be treated as invalid numbers and raise the intended ValueError instead of causing an IndexError.
- The code will no longer attempt to access characters out of bounds.
