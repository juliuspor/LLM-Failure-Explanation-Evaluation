What happened (where and why)
- Problem occurs inside ClassUtils.to_class when iterating the input variable named array (see loop at lines 765–767).
- At line 766 the code calls array[i].upper(). That assumes every element in array is a string (has .upper()).
- If array[i] is None, array[i].upper() raises AttributeError because NoneType has no attribute 'upper'. That is the direct cause of the test failure.
- The docstring explains the method should insert a None element (or return type(None)) for None inputs, and should return None for a None array input. The .upper() call contradicts that behavior: it causes an exception instead of handling None.

Exact lines and variables
- array: the input list/array.
- i: loop index (used in for i in range(len(array))).
- line 766: array[i].upper() — this is the offending call.
- line 767: classes.append(type(array[i])) — intended to append the element class/type.

Why this is wrong
- Calling array[i].upper() is unnecessary for determining an element's type and unconditionally fails for non-string values (None) or any object without .upper().
- The correct behavior (per the docstring/tests) is:
  - If array is None => return None.
  - For each element:
    - If element is None => append type(None) (or None per your documented choice).
    - Otherwise => append type(element).

Concrete fixes (steps to apply)
1. Ensure a top-level None check:
   - If array is None: return None.
2. Replace the loop so it does not call .upper() and handles None explicitly. For example:
   - classes = []
   - for i in range(len(array)):
       elem = array[i]
       if elem is None:
           classes.append(type(None))   # or append None if you want literal None in output
       else:
           classes.append(type(elem))
   - return classes
3. Remove the array[i].upper() call entirely.

Minimal patch (replace lines 765–767)
- Original:
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))
- Replace with:
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          classes.append(type(None))
      else:
          classes.append(type(elem))

Notes about tests
- With the above change:
  - test_to_class_with_none_element will no longer raise AttributeError and will produce [str, type(None), float].
  - test_to_class_null_input must still pass: make sure you return None when array is None.
  - Other tests (empty array, mixed types, all None) will work because no string-specific call is made.
