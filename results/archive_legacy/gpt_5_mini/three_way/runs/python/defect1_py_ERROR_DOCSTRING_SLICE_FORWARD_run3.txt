What happened
- You called the factory with hours_offset = 2 and minutes_offset = -15 (or a similar combination).
- The method checks the sign rules in the docstring and rejects a positive hours value with a negative minutes value. The code validates minutes early and raises ValueError: "Minutes out of range: -15".

Why it fails (root cause, with variables and lines)
- The method expects minutes_offset to be in -59..+59 (docstring) but also forbids the combination hours_offset > 0 with minutes_offset < 0. That rule is enforced before the final offset calculation.
- The relevant variable is minutes_offset. The code later converts an (hours,minutes) pair into a single minutes count using lines around the shown slice:
  - At line 402 the code handles one sign case: minutes_offset = hours_in_minutes - minutes_offset
  - At line 404 it handles the other: minutes_offset = hours_in_minutes + minutes_offset
  - At line 405 it converts minutes to milliseconds: offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)
- Because the incoming minutes_offset was negative while hours_offset was positive, the validation fails earlier and the method raises ValueError("Minutes out of range: -15") instead of computing an offset. The error originates from the minutes validation step (before the arithmetic at lines 402–405).

How to fix it (explicit steps)
1. Fix the call site (preferred):
   - If you intended +02:15, call with hours_offset = 2, minutes_offset = 15.
   - If you intended -02:15, call with hours_offset = -2, minutes_offset = 15.
   - If you intended +01:45 (which is equivalent to +02:-15), normalize the values before calling: hours_offset = 1, minutes_offset = 45.
   - In general, do not pass a positive hours_offset with a negative minutes_offset.

2. Or change the factory to accept mixed signs and normalize (if you control it):
   - Before validation, normalize a (hours_offset, minutes_offset) pair so minutes_offset is non-negative:
     - if hours_offset > 0 and minutes_offset < 0:
         hours_offset -= 1
         minutes_offset += 60
     - if hours_offset < 0 and minutes_offset < 0:
         minutes_offset = -abs(minutes_offset)  (keep semantics as intended)
   - Then re-run the existing range checks and proceed to the arithmetic at lines 402–405.
   - This requires updating the validation code so it does not raise on the combinations you now accept.

Summary
The error is raised because the function explicitly disallows a positive hours_offset together with a negative minutes_offset. Either call the factory with matching signs (or normalize the pair to non-negative minutes) or change the factory to normalize mixed-sign inputs before validating and computing the offset.
