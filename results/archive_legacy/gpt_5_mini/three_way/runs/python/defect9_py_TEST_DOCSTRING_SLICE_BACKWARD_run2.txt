What happened
- The crash happens inside replace_each when replacement_list contains None.
- The code assumes every replacement_list[i] is a string and does len(replacement_list[i]) without checking for None.
- Calling len(None) raises TypeError: "'NoneType' object has no len()", so the function crashes.

Where it happens (exact places)
- In replace_each you compute search_length = len(search_list) (line 52).
- You iterate for i in range(search_length) (line 59 and again at line 81).
- At line 82 you do:
  greater = len(replacement_list[i]) - len(search_list[i])
  — this is the immediate crash point when replacement_list[i] is None.

Why it happens (root cause)
- The code expects search_list and replacement_list entries to be strings and does not treat None as a special case.
- The tests (and original Java behavior) expect None in replacement_list to be ignored (no replacement), but the code still tries to measure the length of the replacement string and therefore breaks on None.

Concrete fixes (step-by-step)
1. Before using replacement_list[i] or search_list[i], check for None and skip that index:
   - At the start of each loop over i (both loops, lines 59 and 81), add:
     if search_list[i] is None or replacement_list[i] is None:
         continue
   This makes the function ignore pairs where either side is None.

2. Ensure any code that assumes replacement_list[i] is a str is reached only after that check:
   - For example, replace the line at 82 with code that runs only when replacement_list[i] is not None:
     if replacement_list[i] is not None and search_list[i] is not None:
         greater = len(replacement_list[i]) - len(search_list[i])
     else:
         continue

3. (Optional) If desired behavior is to treat None as an empty replacement rather than "ignore", explicitly convert:
   rep = replacement_list[i] if replacement_list[i] is not None else ""
   and then use len(rep). But the tests expect the pair to be ignored, so prefer skipping.

Why these steps fix it
- They prevent calling len() on None by ensuring replacement_list[i] is a string before measuring it.
- They match the expected semantics (ignore None replacements) and prevent the TypeError observed at line 82.

Summary
- Root cause: len(replacement_list[i]) called when replacement_list[i] is None (line 82).
- Fix: skip indices where search_list[i] or replacement_list[i] is None (add checks at the start of the loops that use those values).
