What happened (root cause)
- At line 52 the code does: search_length = len(search_list).
- The TypeError means search_list is None, so len(search_list) raised "object of type 'NoneType' has no len()".
- Later code (lines 102–105) does check whether search_list[i] or replacement_list[i] is None, but that check runs after the code already assumed search_list itself was a list and used len(search_list) and later len(search_list[i]) / len(replacement_list[i]) (line 82). Those earlier len(...) calls are where the failure occurs.

Where it occurred (specific lines)
- Primary failure: line 52: search_length = len(search_list) — search_list is None.
- There are other risky len(...) calls later (line 82: len(replacement_list[i]) - len(search_list[i])) that would fail if either replacement_list or search_list or their elements are None.

Why it happened (how the bug arose)
- The function expects search_list and replacement_list to be real sequence objects (lists/tuples) and also expects their items to be strings. The caller passed None (or an element is None). There is no early validation, so the code tries to get lengths before any None checks and crashes.

Exact fixes (clear steps you can apply)
1. Validate inputs at the start of replace_each (before line 52)
   - Check the lists themselves:
     - if search_list is None or replacement_list is None:
         raise ValueError("search_list and replacement_list must be non-None sequences")
   - Check types:
     - if not hasattr(search_list, '__len__') or not hasattr(replacement_list, '__len__'):
         raise TypeError("search_list and replacement_list must be sequences (lists/tuples)")
   - Check matching lengths:
     - if len(search_list) != len(replacement_list):
         raise ValueError("search_list and replacement_list must have the same length")

2. Validate elements before taking len(...) of elements
   - Before using len(search_list[i]) or len(replacement_list[i]) (line 82), ensure elements are not None and are strings:
     - for i, (s, r) in enumerate(zip(search_list, replacement_list)):
         if s is None or r is None:
             raise ValueError(f"search_list[{i}] and replacement_list[{i}] must be non-None")
         if not isinstance(s, str) or not isinstance(r, str):
             raise TypeError(f"search_list[{i}] and replacement_list[{i}] must be strings")

3. Alternative defensive fallback (if you prefer to treat None as empty):
   - Convert None to empty lists or skip None entries explicitly before computing lengths and searching:
     - if search_list is None: search_list = []
     - if replacement_list is None: replacement_list = []
     - Or filter out pairs where search_list[i] is None or replacement_list[i] is None.
