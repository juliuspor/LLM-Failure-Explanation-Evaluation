What happened (where it failed)
- At line 765–766, inside the loop:
  765 |         for i in range(len(array)):
  766 |             _ = array[i].upper()
- That call to array[i].upper() assumes the element is a string. If an element is None (or any non-string that has no upper method), Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The failing test test_to_class_with_none_element passes ["Test", None, 99.0], so when the loop reaches the None element line 766 raises AttributeError and the function never returns the expected list of types.

Why this is the root cause (clear, simple)
- The function’s job is to return the classes (types) of each element. It only needs type(array[i]).
- The unnecessary call to .upper() has no role in computing types and will crash on None or on non-string elements.
- The crash occurs exactly at the call to array[i].upper() (line 766) when array[i] is None.

Concrete fixes (steps you can apply)
1. Remove the extraneous call to .upper() entirely.
2. Iterate directly over elements and append their types. For clarity and correctness when an element is None, append type(None).

Minimal replacement code for the loop:
    classes = []
    for elem in array:
        classes.append(type(elem))
    return classes

Or equivalently keeping the index form:
    classes = []
    for i in range(len(array)):
        classes.append(type(array[i]))
    return classes

Why these fixes work
- type(None) is type(None), so None elements are handled correctly.
- No AttributeError will be raised because we no longer call methods on the elements.
- The function will return [str, type(None), float] for ["Test", None, 99.0], matching the tests.

Optional extra note
- If ArrayUtils.EMPTY_CLASS_ARRAY.copy() is intended to return an empty list, that branch is fine; tests expect [] for empty input. The real runtime failure is the .upper() call on line 766, so removing it is the necessary and sufficient fix.
