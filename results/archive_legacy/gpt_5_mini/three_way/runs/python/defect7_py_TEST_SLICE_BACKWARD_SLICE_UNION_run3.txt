What happened (what failed)
- The failure occurs inside ClassUtils.to_class when it loops over the input array and calls array[i].upper() (see lines 765–766).
- When an element is None, calling None.upper() raises an AttributeError because NoneType has no attribute upper.
- Your test test_to_class_with_none_element passes an array containing None, so the call at line 766 crashes before the code appends the element's class at line 767.

Where it happened (exact locations)
- Method: ClassUtils.to_class
- Variables: array (the input list), classes (the output list)
- Lines: 765–767 in hit07_classutils.py:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))

Why it happened (root cause)
- The code assumes every array element is a string and calls .upper() unconditionally. That assumption is false for None (and for non-string objects), so the method throws when it encounters a None element.
- The intent of to_class is to record each element's class (type), but the unnecessary .upper() call both causes the exception and is unrelated to determining types.

Concrete fixes (steps you can apply)
1. Remove the unconditional .upper() call. It isn’t needed to determine the element’s class.
2. Add a None check so type(None) is used for None elements.
3. Use type(value) (or value.__class__) for non-None elements.

Minimal replacement code (replace lines 765–767):
- Old:
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))
- New:
  for i in range(len(array)):
      value = array[i]
      if value is None:
          classes.append(type(None))
      else:
          classes.append(type(value))

Additional notes
- Also confirm the method already handles array is None (your test expects None) and empty arrays (should return []) — the fix above preserves that behavior.
- After applying the change, rerun the tests; the AttributeError should be gone and tests that expect type(None) for None elements will pass.
