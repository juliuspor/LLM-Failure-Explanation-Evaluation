What fails
- The code reuses the input parameter minutes_offset as a running variable for the "total minutes" value and that causes the math to be done incorrectly/unclear and is the root cause of the failure you see.

Where it happens (lines refer to your slice)
- Line 402: hours_in_minutes = hours_offset * 60
- Line 403–405: the code mutates minutes_offset:
  - if hours_in_minutes < 0: minutes_offset = hours_in_minutes - minutes_offset
  - else: minutes_offset = hours_in_minutes + minutes_offset
- Line 405: offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)

Why this causes the bug
- minutes_offset is the name of the input parameter that represents the minute portion of the zone offset (0..59). That value is validated at lines 396–397.
- Later the code overwrites that same variable with the computed total minutes (hours_in_minutes ± minutes_offset). Reusing the same variable name makes the code fragile and hard to reason about, and it also caused the code to apply the wrong arithmetic in the path you hit.
  - The arithmetic intent is: total_minutes = hours_offset*60 + sign(hours_offset)*minutes_input
  - But because minutes_offset is reused, the subtraction/addition is performed with the mutated variable in one of the branches (this is easy to get wrong and is exactly what happened here).
- As a result the value passed to FieldUtils.safe_multiply is not the intended total minutes for some inputs, so the produced offset is wrong (and triggers the error you observed).

Exact place to look: the mutation lines 402–405. The check lines 396–397 validate the original minutes, but that validated value is lost immediately afterward.

How to fix (step-by-step, concrete)
1. Stop reusing the parameter minutes_offset. Introduce a new local variable for the computed total minutes:
   - int hours_in_minutes = hours_offset * 60;
   - int total_minutes;
2. Compute total_minutes explicitly and clearly:
   - if (hours_in_minutes < 0) total_minutes = hours_in_minutes - minutes_offset;
   - else total_minutes = hours_in_minutes + minutes_offset;
3. Multiply the new total_minutes:
   - offset = FieldUtils.safe_multiply(total_minutes, DateTimeConstants.MILLIS_PER_MINUTE);
4. Pass offset to cls.for_offset_millis(offset) as you already do.
5. (Optional) Rename variables to make intent clear (minutes_input, total_minutes).

Why these steps fix it
- They keep the validated input (minutes_input) intact, avoid accidental use of a mutated variable, and make the arithmetic explicit so you can't accidentally subtract/add the wrong value. The result passed to safe_multiply will be the intended total number of minutes and the offset calculation will be correct and predictable.
