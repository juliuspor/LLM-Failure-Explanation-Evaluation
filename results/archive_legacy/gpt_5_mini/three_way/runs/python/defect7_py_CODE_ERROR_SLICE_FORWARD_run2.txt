What happened (root cause)
- The code calls array[i].upper() on every element (line with "_ = array[i].upper()").
- At least one element in array is None, so array[i] is None and None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The exception occurs at the line that calls .upper(), not at classes.append; the traceback shows classes.append because that is nearby, but the actual failing call is the prior .upper() call.

Where it happened (exact variable and line)
- Variable: array
- Element: array[i] (one element is None)
- Failing statement: _ = array[i].upper()
- In the supplied trace slice the failing behaviour originates at the .upper() call immediately before the classes.append line.

Why it fails
- .upper() is a string method. If array can contain non-string values (including None), calling .upper() unguarded will fail when an element is None or any non-string that lacks upper().
- The code does not check the element type or presence before calling .upper().

Concrete fixes (choose one depending on intended behavior)
1) If you intended to ignore None elements:
   - Skip elements that are None before calling .upper() and before appending their type.
   Example:
     for i in range(len(array)):
         if array[i] is None:
             continue
         _ = array[i].upper()
         classes.append(type(array[i]))

2) If you intended to treat None as the string "None" (i.e., call .upper() safely by converting to string):
   - Convert to str before upper():

         _ = str(array[i]).upper()
         classes.append(type(array[i]))

3) If you only want to call .upper() when the element is a string:
   - Check instance before calling:
     for i in range(len(array)):
         if isinstance(array[i], str):
             _ = array[i].upper()
         classes.append(type(array[i]))

4) If you don't need the .upper() call at all (it isn't used):
   - Remove that line. It looks like "_" is unused, so the simplest fix may be to delete the .upper() call entirely:

         classes.append(type(array[i]))

Additional notes
- If array can be empty you already handle that. But you must also handle elements that are None or non-strings.
- Prefer iterating directly over elements (for elt in array:) instead of indexing, for clarity.
- Example minimal corrected function if you only want types and to safely ignore None elements:
     def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:

             return None
