What happened
- The failure happens inside LocaleUtils.to_locale when parsing "fr__POSIX".
- At lines 237–240 the code does:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]
  - then checks whether ch3 and ch4 are uppercase letters; if not it raises ValueError.
- For "fr__POSIX" the characters are:
  - index 0 = 'f', 1 = 'r', 2 = '_', 3 = '_', 4 = 'P'
- ch3 therefore is '_' (index 3). The uppercase check at line 239 fails because '_' is not between 'A' and 'Z', so the method raises ValueError ("Invalid locale format").

Why this is the root cause
- The code assumes that when a separator '_' is present after the language, a two-letter country code must follow immediately (positions 3 and 4). It never handles the valid Java-style form language__variant (empty country, double underscore) where position 3 is '_' and the variant begins at position 4.
- Thus the function misclassifies language__variant as invalid by checking uppercase letters at positions 3 and 4 (lines 237–240) instead of recognising the empty-country case.

Where to change it (concrete steps)
1. Add an explicit check for the empty-country case before the uppercase-country check:
   - At or before line 237, test if locale_str[2] == '_' and locale_str[3] == '_'. For example:
     - if locale_str[2] == '_' and locale_str[3] == '_':
         return Locale(locale_str[0:2], "", locale_str[4:])
   - That handles "fr__POSIX" (language = locale_str[0:2], country = "", variant = locale_str[4:]).
2. Only if the empty-country pattern is not present, perform the existing country-uppercase validation (lines 237–240) and subsequent variant parsing (lines 243–247).
3. Ensure you also guard index access (locale_str[3], locale_str[4], locale_str[5]) with length checks so you don't get IndexError for short inputs.

Alternative robust approach
- Replace the manual index checks with splitting by underscore (maxsplit=2):
  - parts = locale_str.split('_', 2)
  - then interpret:
    - parts == [lang] → language-only
    - parts == [lang, country] where country == '' → empty country (variant may be in parts[2] if present)
    - parts == [lang, '', variant] → empty country, variant = parts[2]
    - parts == [lang, country, variant] → normal case
  - Validate lengths and cases for parts[0] (lowercase) and parts[1] (uppercase if non-empty).
