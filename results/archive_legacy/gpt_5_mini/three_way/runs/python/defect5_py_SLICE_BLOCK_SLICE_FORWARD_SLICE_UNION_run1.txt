What happened (where, in the code)

Why that is wrong (root cause)
- type(array) is the type of the container (e.g., list), not the type of the elements inside the container.
- That means inferred_type becomes something like <class 'list'> instead of the element type (e.g., <class 'int'>).
- Because inferred_type is a container type, it will never equal the element sentinel you intended to represent (object), or you may end up in the wrong branch and raise the TypeError incorrectly.
- Concretely: if array is present, inferred_type should reflect the element type (the type of items inside array), but the code uses type(array), so the later comparison at line 346 is comparing the wrong thing — which is why the TypeError is raised inappropriately.

Exactly where it occurs

- Lines 337–340 and 340: there is a fallback that sets inferred_type = object when the element/array content isn’t available.
