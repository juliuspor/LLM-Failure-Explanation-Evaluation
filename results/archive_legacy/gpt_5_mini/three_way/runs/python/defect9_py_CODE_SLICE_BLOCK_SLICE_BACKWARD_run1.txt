What happened (where it fails)
- The function replace_each crashes at lines 81–84 in this loop:
  ```
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater  # assume 3 matches
  ```
- The crash occurs when either search_list[i] or replacement_list[i] is None. Calling len(None) raises a TypeError: "'NoneType' object has no len()".
- This is inconsistent with earlier code: when scanning for matches (lines ~59 and ~97) you explicitly skip entries where search_list[i] is None, search_list[i] is empty, or replacement_list[i] is None. But the size-estimation loop (lines 81–84) does not perform the same checks and so is not null-safe.

Why it happens (root cause)
- The increase estimation loop assumes every element in search_list and replacement_list is a string and directly calls len(...) on them.
- If any element is None (allowed by the function's Optional[List[Optional[str]]]), the len(...) call throws a TypeError before the function gets to the later safe loops that would have skipped that element.

Exactly when it will happen
- If either input list contains None at any index i (or if the lists contain mismatched None/strings), the code fails at the increase loop before building the result. The payload lists passed into replace_each determine this.

How to fix it (concrete steps)
Option A — Make the estimation loop null-safe (small change)
- Replace the loop at lines 81–84 with the same guard used later:
  ```
  for i in range(search_length):
      if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
          continue
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater
  ```
- This keeps the same behavior as the matching loops and prevents len(None) calls.

Option B — Normalize inputs up-front (recommended if you want clearer invariants)
- At the start (after length checks), build filtered parallel lists that exclude entries where search is None or empty or replacement is None. Then compute search_length from the filtered list and run the rest of the algorithm assuming all entries are non-None strings. Example:
  ```
  pairs = [(s, r) for s, r in zip(search_list, replacement_list)
           if s is not None and s != "" and r is not None]
  if not pairs:
      return text
  search_list, replacement_list = zip(*pairs)
  search_length = len(search_list)
  ```
- Benefits: you avoid repeated checks in multiple loops and make the rest of the function simpler and safer.

Which to choose
- If you want the minimal change: use Option A (add the same if-checks inside the increase loop).
- If you prefer cleaner logic and fewer per-loop guards: use Option B (filter invalid entries once up-front).
