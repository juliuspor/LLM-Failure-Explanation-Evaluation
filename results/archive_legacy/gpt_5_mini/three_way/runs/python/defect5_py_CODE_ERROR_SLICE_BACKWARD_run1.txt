Root causes, spelled out
1. type(array) is the list class, not the element component type. Using type(array) at line 336 makes inferred_type incorrect whenever array is non-None (even if it contains elements).
2. If the element passed in is None, you cannot determine element type from element, so you fall back to object (line 340). If expected_type is provided (e.g. str) you then raise the TypeError at lines 346–347 because you treated the list as an "object list" that cannot be cast to a str list.
3. The code never inspects array contents to get the element type (for example, array[0]) and does not prefer expected_type when array is empty but expected_type is known.
