What happened
- The method is documented to "return None for a None input array." That means when you call it with array = None it returns None.
- Somewhere after the call the code immediately does .upper() on the returned value. Because the method returned None, Python raises AttributeError: 'NoneType' object has no attribute 'upper'.

Where it happened
- The conversion method appends classes with classes.append(type(array[i])) at line 767 when array is not None.
- But the failing call is when the entire input array itself was None: the method returned None and the caller later did result.upper(), causing the AttributeError. (Search your call sites for code that does ... = convert(...); ... .upper() on that result.)

Why it failed (root cause)
- The method and the caller disagree about the return value for a None input:
  - The method returns None for a None input (docstring and current implementation).
  - The caller assumes it will get an object that supports .upper() and does not check for None.
- Because the caller didn't check for the None return, .upper was invoked on None and Python raised AttributeError.
