What happened (where it fails)
- The failure happens inside the for-loop of to_class, at this line:
      _ = array[i].upper()
  When array[i] is None this attempts to call .upper() on None, raising AttributeError: 'NoneType' object has no attribute 'upper'.
- That line is at the top of the loop and is unnecessary: its result (_) is never used. Because of it, any input list that contains None (e.g. ["Test", None, 99.0]) triggers an exception before the code reaches the next line that actually builds the class list:
      classes.append(type(array[i]))

Why it fails (root cause)
- The code assumes every element has an .upper() method (strings do) and calls it unconditionally. None does not have .upper(), so the unguarded call throws.
- The intended purpose of the method (convert elements to their classes and insert None for None) is implemented by the type(...) call; the .upper() call is irrelevant to that purpose and introduces the bug.

Exact places to change
- Remove the line "_ = array[i].upper()" from the loop.
- Ensure the loop uses the actual element variable and handles None explicitly (or rely on type(None) produced by type(None) for None values).

Concrete minimal fixes (choose one)

1) Remove the stray .upper() call and use a simple loop:
    classes = []
    for i in range(len(array)):
        elem = array[i]
        classes.append(type(elem))
    return classes

2) More Pythonic / concise:
    return [type(elem) for elem in array]

Either fix will make tests pass (including arrays containing None), because type(None) is type(None) and will be appended.

Other small notes
- ArrayUtils.EMPTY_CLASS_ARRAY.copy() is fine for the empty-array case (assuming that constant is a list).
- The stray "_" assignment is the sole cause of the AttributeError and should be removed.
