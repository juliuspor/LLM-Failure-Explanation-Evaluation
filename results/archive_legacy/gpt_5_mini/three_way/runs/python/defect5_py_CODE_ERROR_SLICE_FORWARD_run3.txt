What happened (where it fails)
- At the top of the function you set inferred_type with:
  - line "if array is not None: inferred_type = type(array)" (this gives you the list class, not the element type),
  - then "elif element is not None: inferred_type = type(element)",
  - else "inferred_type = object".
- Later you call ArrayUtils._copy_list_grow1(array, inferred_type) and then you raise a TypeError when inferred_type == object and expected_type != object (the raise is in the "if expected_type is not None:" block).

The failing test produced inferred_type == object and expected_type == str, so the function raised:
"Cannot cast object list to str list ..." — that is exactly the check that triggers when inferred_type is object.

Why this happens (root cause)
1. You use type(array) when array is not None. type(array) returns the list class (e.g. <class 'list'>) — not the element type. That is wrong if the code expects the element/component type.
2. More importantly for this failing test: both array and element were effectively unknown as far as a component type:
   - If array is None or empty, and
   - if element is None,
   then your code falls through to inferred_type = object. You then check expected_type and, because expected_type is str, you immediately raise. In short: adding a None element to an empty/None array with expected_type provided makes inferred_type object and triggers the TypeError.
3. The logic never uses expected_type to resolve the component type before deciding to raise. You only use expected_type to decide whether to throw — but expected_type could have told you what array component type to create.

Which lines to change
- The line "inferred_type = type(array)" is wrong for getting element/component type.
- The final check:
  if inferred_type == object and expected_type != object:
should instead allow using expected_type as the component type.

Concrete fixes (step-by-step)
1. Determine the element/component type correctly:
   - If array is not None and len(array) > 0: inferred_type = type(array[0])
   - elif element is not None: inferred_type = type(element)
   - elif expected_type is not None: inferred_type = expected_type
   - else: inferred_type = object
2. Use expected_type as a fallback BEFORE raising. For example:
   - After computing inferred_type as above, if inferred_type is object but expected_type is provided, set inferred_type = expected_type (do not raise).
3. Only raise if inferred_type is object AND expected_type is None (i.e. you truly have no information).
4. Replace "type(array)" with an element-type lookup (array[0]) as noted.

Minimal corrected pseudocode:
    if array is not None and len(array) > 0:
        inferred_type = type(array[0])
    elif element is not None:
        inferred_type = type(element)
    elif expected_type is not None:
        inferred_type = expected_type
    else:
        inferred_type = object

    # now create new list with inferred_type
    new_list = ArrayUtils._copy_list_grow1(array, inferred_type)
    new_list[len(new_list) - 1] = element

    # only raise if we still have no element type and no expected_type
    if expected_type is None and inferred_type == object:
        raise TypeError(...)

Why this will fix the failing test
- When element is None but expected_type is str, the corrected flow will set inferred_type to expected_type (str) and create a str list instead of treating it as an object list and raising immediately. That prevents the TypeError seen in the test.
