What goes wrong
- The while loop at lines 181–183 can never terminate when mantissa becomes 0. Specifically:
  - At line 177 you set value = int(x) and mantissa = value.
  - The loop condition at line 181 is while mantissa % 10 == 0.
  - Inside the loop you compute check_val = (mantissa // 10) * (10 ** (exp + 1)) (line 182) and then compare check_val == value (line 183).
  - If mantissa ever becomes 0, then mantissa % 10 == 0 stays true, check_val becomes 0, check_val == value will be true if value is also 0, and then mantissa //= 10 keeps mantissa at 0. The loop continues forever.

Where it can happen
- That situation occurs when value is 0 (i.e. x == 0.0 and the code enters the integer branch). If value == 0 you will reach the block guarded by if abs(x) >= 100 only when x satisfies that (so 0 normally won't reach that exact branch), but even if other inputs cause mantissa to reach 0 via successive //= 10 steps (for example due to some other logic change or unexpected values), the same infinite-loop condition will occur.

Why it fails (short)
- When mantissa == 0 the while condition keeps being true, mantissa remains 0 after //= 10, check_val remains 0, check_val == value is true, so the loop never breaks -> infinite loop / hang.
