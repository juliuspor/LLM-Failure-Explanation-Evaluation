What happened (root cause)
- In ClassUtils.to_class you loop over the input variable named array and at line 766 you call array[i].upper() unconditionally.
- When array contains a None element (e.g. the test passes ["Test", None, 99.0]) the expression array[i].upper() tries to call upper on None, which raises AttributeError: 'NoneType' object has no attribute 'upper'.
- That exception occurs before you append the element's type at line 767, so the method never returns the expected list of types. The failing test is test_to_class_with_none_element (and any other test with None elements).

Exactly where it fails
- File: hit07_classutils.py
- Function: ClassUtils.to_class
- Lines: 765–767 in your slice:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))
- The call on line 766 is the cause; calling .upper() on None triggers AttributeError.

Why the .upper() call is wrong here
- The purpose of to_class is to produce the class/type of each element (type(array[i])). There is no need to call .upper() on the element here. That call appears unrelated to computing the element's class and is unsafe for non-string elements (None, numbers, etc.).
- If the code intended to normalize string values, that logic must be guarded and should not run for None or non-string values.

Concrete, actionable fixes (pick one)

1) Simple and correct: remove the stray .upper() call and handle None explicitly
Replace the three lines with:
- if array[i] is None:
-     classes.append(type(None))
- else:
-     classes.append(type(array[i]))

Or as a compact loop:
for elem in array:
    classes.append(type(elem))

This will make:
- None -> type(None)
- numbers/strings/lists -> their correct type
- work for empty array and for array is None (assuming you keep the current None input check that returns None).

2) If you need to treat strings specially (e.g. uppercase them for some other reason), guard the call:
for elem in array:
    if elem is None:
        classes.append(type(None))
    elif isinstance(elem, str):
        _ = elem.upper()      # only for strings
        classes.append(str)
    else:
        classes.append(type(elem))

3) Unit-test considerations
- Keep the test test_to_class_null_input which asserts ClassUtils.to_class(None) returns None — ensure that outer guard remains.
- The tests expecting type(None) should pass once you handle None as shown.
