What happened
- The code that examines a character in the string calls a helper that checks the index, and that helper (lines 30–32) raises IndexError when index >= len(s):
  - 30: if index < 0 or index >= len(s):
  - 31:     raise IndexError(f"String index out of range: {index}")
  - 32: return s[index]
- create_number is passing an index equal to len(s) (one past the last valid index) when it tries to inspect a character (for example, when testing for suffix letters like "L", "F", etc.). For single-character inputs such as "l" or "L" the code ends up calling the helper with index == len(s) == 1, which triggers the IndexError instead of producing the ValueError the tests expect.

Why it fails (root cause)
- Off-by-one error: create_number uses len(s) where it should use len(s) - 1 to access the last character. Because Python string indices go from 0 to len(s)-1, requesting s[len(s)] is invalid and triggers the IndexError raised on line 31.
- The exception type is wrong for the tests: tests expect NumberUtils.create_number to raise ValueError for those inputs, but the index-out-of-range check raises IndexError earlier, so the wrong exception propagates.

Where it happens
- In the helper that returns s[index] (lines 30–32). The helper correctly guards against bad indices, but create_number supplies an invalid index (len(s)).

How to fix (concrete steps)
1. When you want the final character of a non-empty string s, use index = len(s) - 1 (or s[-1]) instead of index = len(s).
   - Example change: replace any call like char_at(s, len(s)) with char_at(s, len(s) - 1) or simply s[-1] after verifying s is non-empty.
2. Ensure you check for empty strings before indexing:
   - if not s: raise ValueError("empty string") or handle according to your API.
3. Prefer raising ValueError for invalid numeric input so tests get the expected exception:
   - Catch IndexError (if any) inside create_number and re-raise ValueError, or better, avoid the IndexError by fixing indices as above.
4. Add unit tests for empty string and single-character inputs to prevent regressions.
