What happened (root cause)
- The exception occurs on this line in your function:
  _ = array[i].upper()
- That call tries to run the string method upper() on array[i]. The error AttributeError: 'NoneType' object has no attribute 'upper' means array[i] is actually None at runtime, so None.upper() is invalid.
- Where it happened: the failing statement is the line with _ = array[i].upper() (the line immediately before classes.append(type(array[i])) in your snippet / the trace).
- Why it happened: your function signature allows array to be Optional[List[Any]] and the list elements are typed as Any, so some elements in the list can be None. The code assumes every element is a string and calls .upper() without checking; when an element is None you get the AttributeError.

Why the later line classes.append(type(array[i])) also matters
- After the failing .upper() call, execution never reaches classes.append(...). If you intended to collect element classes (type(...)), a None element would result in type(None) == NoneType — but the current crash prevents even reaching that step because of the unguarded .upper() call.

Concrete fixes (pick one depending on desired behavior)
1) If you only expect strings and want a clearer error for None elements
- Check for None and raise a specific error:
  if array[i] is None:
      raise ValueError(f"array[{i}] is None; expected str")
  _ = array[i].upper()
  classes.append(type(array[i]))

2) If you want to skip None elements and continue
- Skip None before calling upper and before appending:
  if array[i] is None:
      continue
  _ = array[i].upper()
  classes.append(type(array[i]))

3) If you want to treat non-strings by coercing to str
- Convert non-None values to string before calling upper:
  if array[i] is None:
      classes.append(type(None))
  else:
      s = str(array[i])
      _ = s.upper()
      classes.append(type(array[i]))

4) If you only care about element types (no upper call needed)
- Remove the unnecessary .upper() entirely:
  for i in range(len(array)):
      classes.append(type(array[i]))

Exact places to change
- Edit or remove the line _ = array[i].upper() (the line directly before classes.append(...)). Add a guard like array[i] is None or isinstance(array[i], str) depending on which fix you choose.

Quick recommendation
- If array may contain None, add an explicit None check at the top of the loop or before calling .upper(). This prevents the AttributeError and makes your intent clear (skip, raise, or coerce).
