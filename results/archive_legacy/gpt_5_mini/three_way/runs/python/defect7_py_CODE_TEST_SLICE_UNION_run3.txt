What happened (root cause)
- The method to_class (lines shown) calls array[i].upper() inside the loop (line 766).
- When array[i] is None, None has no attribute upper — Python raises AttributeError. That is exactly what breaks the test test_to_class_with_none_element.
- More generally, many non-string items (int, float, list, bool, etc.) also don’t have .upper(). So the call is unsafe for any element that is not a str.
- The .upper() call does nothing with its result (assigned to "_" and never used), so it appears to be an unnecessary leftover that introduces the failure.

Where it happens
- In to_class(cls, array) inside the for loop:
  - line 765: for i in range(len(array)):
  - line 766: _ = array[i].upper()  <-- causes AttributeError for None or non-str
  - line 767: classes.append(type(array[i]))

Why this fails the tests
- test_to_class_with_none_element passes an array ["Test", None, 99.0].
  - When i == 1, array[1] is None; array[1].upper() raises AttributeError and the method crashes before appending types.
- Even tests with ints, floats, lists, booleans would fail if the code reached their iteration because those types also lack .upper().

How to fix it (concrete steps)
Option A — simplest: remove the unnecessary .upper() call entirely.
- Change the loop to:
    for i in range(len(array)):
        classes.append(type(array[i]))

Option B — if you truly intended to call .upper() only for strings: guard the call.
- Example:
    for i in range(len(array)):
        if isinstance(array[i], str):
            _ = array[i].upper()
        classes.append(type(array[i]))

Either fix will make the tests pass. The recommended fix is Option A because the .upper() result is unused and likely an accidental leftover.
