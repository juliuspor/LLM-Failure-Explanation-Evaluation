Why it happened (root causes)
1. The computation that produces r, g, b produced values outside 0–255. Common causes:
   - Wrong scaling factor (e.g. using 256 or using value in the wrong units).
   - Not converting floats to integers where required.

2. The ValueError message was built incorrectly: the code used a literal string containing the words "Red Green Blue" rather than interpolating the variables r, g, b. Example incorrect code:
   raise ValueError("Color parameter outside of expected range: Red Green Blue")
   or
   raise ValueError("Color parameter outside of expected range: %s %s %s" % ("Red","Green","Blue"))
   Instead of showing the actual numbers it reports the literal names.
