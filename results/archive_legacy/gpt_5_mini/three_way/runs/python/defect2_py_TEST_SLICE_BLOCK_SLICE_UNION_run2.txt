What happened (where and why)
- Failure occurs in GrayPaintScale.get_paint at these lines:
  - Line 71 computes g:
    g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  - Lines 74–75 immediately raise an exception when g is outside [0,255]:
    if g < 0 or g > 255:
        raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- Why the tests fail:
  - The tests intentionally pass values outside the configured range (e.g. value = -0.5, value = 1.5, value = 5.0 when bounds are 10.0..20.0) and expect those inputs to be clamped to the nearest bound (black or white).
  - Instead of clamping, the code computes g for the out-of-range input and raises ValueError when g < 0 or g > 255. For example:
    - With default bounds (0.0..1.0), value = -0.5 gives a negative g → triggers the raise on line 74.
    - With custom bounds (10.0..20.0), value = 5.0 gives a negative g → triggers the same raise.
  - Because of that explicit exception, the tests fail instead of receiving clamped colors.

What to change (exact, actionable steps)
You should clamp either the input value to the lower/upper bounds before computing g, or clamp g itself to [0,255] after computing it. Two safe, concrete fixes:

Option A — clamp the input value (preferred, clearer intent)
1. After method start and before computing g, add:
   value = min(max(value, self._lower_bound), self._upper_bound)
2. Then keep the existing computation of g as-is.
3. Remove the ValueError raise (or keep it only for programming errors like lower_bound == upper_bound).

Option B — clamp g after computing it
1. Keep line 71, then replace the raise with:
   g = max(0, min(255, g))
2. Do not raise for out-of-range g.
