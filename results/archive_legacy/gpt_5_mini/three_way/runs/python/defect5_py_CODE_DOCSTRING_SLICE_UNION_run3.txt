What goes wrong
- On line 335 you set inferred_type = type(array). That gets the Python class of the list object (i.e. <class 'list'>), not the element type stored inside the list.
- _copy_list_grow1(array, inferred_type) (line 342) expects the component/element type, not the list class. Passing <class 'list'> as the element type causes the new list to be constructed with the wrong component type and later assignments or internal type checks fail (you see the error where the code tries to put element into the newly copied list).
- The code only looks at the element's type when array is None (line 337). If array is not None you never inspect its contents, so even a non-empty list of ints will produce inferred_type == list, not int.
- The type-comparison at line 346 (inferred_type == object) also relies on inferred_type being the element class. Because inferred_type is the list class, that logic is wrong and may hide or misreport type mismatches.
