- At line 335 the code sets inferred_type = type(array). That takes the type of the list object itself (e.g. <class 'list'>), not the element type stored in the list. So inferred_type becomes list, not the type of the contained elements.
- At lines 337–340 the code falls back to inferred_type = object when the element is None (both array and element None path uses object).
- At line 346 the code checks if inferred_type == object and expected_type != object and then raises TypeError.
