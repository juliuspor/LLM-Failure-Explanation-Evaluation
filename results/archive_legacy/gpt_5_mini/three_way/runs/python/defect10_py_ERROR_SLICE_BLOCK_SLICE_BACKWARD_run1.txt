- That check fails (raises) when index >= len(s). For index == 0 to be out of range, len(s) must be 0 — i.e., s is an empty string.

- The guard at lines 30–31 correctly treats index 0 as invalid for an empty string because there are no characters to index.
- In short: the caller passed an empty string but still tried to access position 0; the function rejects that and raises the IndexError.
