What happened
- The test inputs "l" and "L" hit the branch handling trailing long-qualifiers:
  - last_char = val[-1] is "l" or "L"
  - numeric = val[:-1] becomes the empty string "" when val is "l" or "L"
- The code then calls char_at(numeric, 0) without checking that numeric is non-empty.
- char_at("", 0) raises an IndexError ("String index out of range: 0") from the slice_union helper (see lines 30–32), so the test sees an IndexError instead of the expected ValueError.

Why it fails (root cause)
- Missing guard for the case len(val) == 1. The comment in the code even notes the bug:
  # BUG: missing guard for len(val) == 1 and non-digit input.
- The condition that checks the digits uses char_at(numeric, 0) unconditionally:
  if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
  When numeric == "", char_at(...) throws IndexError before the function can raise the intended ValueError.

Where it occurs
- In create_number, inside the branch for last_char in ("l", "L") (the block with the BUG comment). The IndexError originates from slice_union lines 30–32 when char_at is called with index 0 on an empty string.

How to fix (concrete steps)
1. Detect the single-character input before calling char_at. For example, add an explicit check for empty numeric (len(numeric) == 0) and raise ValueError:
   - Right after numeric = val[:-1]:
       if len(numeric) == 0:
           raise ValueError(f"{val} is not a valid number.")
2. Or rewrite the digit-check to avoid char_at on empty strings:
   - Replace the condition with a safe equivalent:
       if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():
3. Either change will ensure inputs like "l" and "L" raise ValueError as expected by the tests.

Minimal patch example (replace the problematic if with this):
    if last_char in ("l", "L"):
        numeric = val[:-1]
        if len(numeric) == 0:
            raise ValueError(f"{val} is not a valid number.")
        if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():
            return int(numeric)
        raise ValueError(f"{val} is not a valid number.")

Effect of the fix
- For "l" and "L" the function will now raise ValueError (test passes).
- For valid long-qualifier inputs like "123L" or "-5l" it will still parse to int.
- The IndexError from char_at is eliminated.
