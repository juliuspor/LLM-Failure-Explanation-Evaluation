What happened (root cause)
- At line 52 the code sets search_length = len(search_list) — that is fine because search_list itself is a list.
- Later, at lines 81–82 the code does:
  greater = len(replacement_list[i]) - len(search_list[i])
  This calls len() on the individual entries replacement_list[i] and search_list[i].
- If either replacement_list[i] or search_list[i] is None, calling len(None) raises TypeError: object of type 'NoneType' has no len().
- The docstring says None entries should be ignored, but the code never checks for None before calling len() on list elements. That is precisely where the crash occurs.

Where it occurs
- Line 52: search_length = len(search_list) — no problem here.
- Loop at line 59 uses that length to iterate over indices 0..search_length-1.
- The crash happens in the second loop at line 81–82 when the code computes lengths of replacement_list[i] or search_list[i] without checking for None.

Why it was overlooked
- The code assumes every element in search_list and replacement_list is a string (or other object with len()), so it does no per-element validation or skipping. The presence of None violates that assumption.

How to fix it (concrete steps)
1. Ensure replacement_list and search_list have matching lengths up front (if that’s required), or handle mismatched lengths explicitly.
2. Before you call len() on individual elements, check for None and skip those entries. For example, inside the loops replace direct len() calls with guarded code:

   - At the start of the per-index handling (inside the for i in range(search_length) loops):
     if search_list[i] is None:
         continue
     if replacement_list[i] is None:
         continue

   - Then you can safely do:
     search = search_list[i]
     repl = replacement_list[i]
     greater = len(repl) - len(search)

3. Alternatively, sanitize the lists once at the start by filtering out pairs where search_list[i] is None or replacement_list[i] is None:
   - Build new lists:
     filtered_pairs = [(s, r) for s, r in zip(search_list, replacement_list) if s is not None and r is not None]
     Then iterate over filtered_pairs instead of indexing the original lists.

4. If None is invalid input rather than something to ignore, raise a clear ValueError early with a descriptive message instead of letting a TypeError bubble up.

Concrete minimal change to avoid the crash (example placed where the len(...) is used):
- Before computing greater:
    if search_list[i] is None or replacement_list[i] is None:
        continue
    greater = len(replacement_list[i]) - len(search_list[i])

This change directly fixes the TypeError by ensuring you never call len() on None, and it implements the docstring behavior of ignoring None replacements.
