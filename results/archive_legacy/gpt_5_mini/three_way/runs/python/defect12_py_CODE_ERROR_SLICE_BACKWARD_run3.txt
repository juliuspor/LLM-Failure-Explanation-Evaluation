What happened (root cause)
- The code lets lower be larger than the length of text. If lower > len(text) the line
  if upper < lower:
      upper = lower
  will set upper to a value greater than len(text).
- Later you call substring_java(text, 0, upper) (or substring_java(text, 0, index) when index >= 0). substring_java enforces 0 <= start <= end <= len(s) and raises IndexError when end > len(s). That is exactly what happened: the failing error shows "String index out of range: 15", which is the end argument passed in (upper or index) and is larger than len(text).

Where it occurs in your code
- The problematic sequence is:
  1. index = StringUtils.index_of(text, " ", lower)
  2. if index == -1:
         result = substring_java(text, 0, upper)     <-- here end can be > len(text)
- But upper may have been set to lower by the earlier conditional if upper < lower: upper = lower. If lower > len(text) this makes upper > len(text) and causes the IndexError inside substring_java.

Why index_of(lower) doesn’t save you
- Calling StringUtils.index_of with a start index >= len(text) will often return -1 (no space found past that point). That leads you into the index == -1 branch, where you call substring_java with upper still > len(text). So the index_of call does not prevent the out-of-range slice.

How to fix it (concrete fixes you can apply)
Choose one of these exact changes:

Option A — clamp lower and upper to text length (recommended):
- After you check for text None/empty, add:
    lower = max(0, min(lower, len(text)))
    if upper == -1:
        upper = len(text)
    else:
        upper = max(0, min(upper, len(text)))
  This ensures both lower and upper are inside [0, len(text)] before any substring_java calls.

Option B — clamp upper after you modify it by lower:
- Replace
    if upper < lower:
        upper = lower
  with
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)
  This preserves your semantics of increasing upper to lower but then clamps it so it doesn't exceed len(text).

Option C — validate and raise a clear error:
- If you prefer to treat invalid lower as caller error, add:
    if lower > len(text):
        raise ValueError(f"lower ({lower}) must be <= text length ({len(text)})")
  and keep the rest unchanged.

What to change where (line references)
- The bug originates around these lines:
    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower
  Modify here to clamp lower or clamp upper after this sequence (see Options A or B).

Summary
- Root cause: lower can be greater than len(text), you assign upper = lower and later call substring_java with end > len(text), which throws IndexError.
- Fix: clamp lower and/or upper to len(text) (or validate lower) before any substring_java calls.
