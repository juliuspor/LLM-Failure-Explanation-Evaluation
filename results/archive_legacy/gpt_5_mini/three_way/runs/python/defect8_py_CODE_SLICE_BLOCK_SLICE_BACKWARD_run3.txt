What failed
- The code raises ValueError at the check on lines 237–240:
  ch3 = locale_str[3]
  ch4 = locale_str[4]
  if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
      raise ValueError(f"Invalid locale format: {locale_str}")

Why it fails (root cause)
- The code requires the country part (locale_str[3] and locale_str[4]) to be uppercase A–Z. If the input uses lowercase country letters (for example "en_us"), those characters are outside the 'A'..'Z' range and the condition becomes true, so the function raises ValueError.
- This is a strict check of case, not a check that the characters are letters. The code treats lowercase country codes as invalid even though "en_us" is a common representation of the same locale as "en_US". The failure therefore happens at the lines shown (237–240) whenever the country letters are lowercase.

Where it happens
- At lines 237–240 (the ch3/ch4 comparison):
  - ch3 is locale_str[3]
  - ch4 is locale_str[4]
  - The comparison ch3 < 'A' or ch3 > 'Z' rejects lowercase letters

How to fix it (clear steps)
1. Decide whether you want to accept lowercase country codes or require uppercase:
   - If you want to accept mixed case input, normalize the case before testing.
   - If you want to enforce uppercase input, document that and keep the current checks.

2a. To accept either case and still validate letters, normalize country to upper-case before checks and use that when constructing the Locale:
   - Replace the ch3/ch4 checks with either:
     - Convert country = locale_str[3:5].upper() and validate that country.isalpha() and len(country) == 2
     - Or check ch3.isalpha() and ch4.isalpha() (then use .upper() when creating the Locale)
   - Example change:
     country = locale_str[3:5]
     if not (country.isalpha() and len(country) == 2):
         raise ValueError(...)
     country = country.upper()
     return Locale(locale_str[0:2], country, ...)

2b. Alternatively, if you want to keep checking characters individually, compare with lowercase range too:
   - Replace the check with:
     if not ('A' <= ch3 <= 'Z' or 'a' <= ch3 <= 'z') or not ('A' <= ch4 <= 'Z' or 'a' <= ch4 <= 'z'):
         raise ValueError(...)

3. Always normalize when constructing the Locale (so the Locale object uses a canonical case):

   - variant = locale_str[6:] (leave as-is or normalize if you want)
   - return Locale(language, country, variant) or return Locale(language, country) for the 5-character case
