What happened (where it fails)
- At line 766 you call array[i].upper(). If array[i] is None, Python raises AttributeError: 'NoneType' object has no attribute 'upper'. The exception occurs on line 766, so the loop never reaches line 767 to append a None (or any type) for that element.

Why it happens (root cause)
- The code uses array[i].upper() as if every element is a string. That call is not a harmless "check" — it actually invokes a string method and will fail for non-string objects, in particular for None. The docstring promises that None elements will be preserved in the output, but the code does the opposite: it raises before it can append a None.

Exactly where to change the code
- Replace the call to array[i].upper() at line 766 with an explicit None check before calling any methods on array[i]. Keep the append at line 767 but make it conditional.
