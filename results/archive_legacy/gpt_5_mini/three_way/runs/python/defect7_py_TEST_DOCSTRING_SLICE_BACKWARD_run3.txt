- That AttributeError is why the tests fail; the code never returns the expected list of types.

- Tests and the docstring expect None elements to be handled (inserted as None in the result, or at least not crash). The current code contradicts that: it raises AttributeError when an element is None.
- The docstring also claims "If any of these objects is None, a None element will be inserted" but the implementation does the opposite (it crashes).

1. Guard against None before calling any attribute on array[i].

2. Return types using type(...) (or array[i].__class__) for non-None entries, and return type(None) or None consistently for None entries depending on desired API.
