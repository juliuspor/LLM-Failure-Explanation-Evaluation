What happened (where)
- The exception occurs on line 766: the code calls array[i].upper().
- Python reports AttributeError: 'NoneType' object has no attribute 'upper', which means array[i] is None for at least one i.
- Because the error happened on line 766, the loop never reaches line 767 (classes.append(type(array[i]))) for that iteration.

Why it happened (root cause)
- upper() is a method on string objects. Calling .upper() on None raises AttributeError because NoneType has no .upper().
- The root cause is that the list variable array contains at least one None element. That None likely came from earlier code that appended or returned None instead of a string (for example, a failed lookup, a function that sometimes returns None, or a missing value).

How to confirm where the None came from (quick debug steps)
1. Print or log the offending index and value before calling .upper(), e.g.:
   for i in range(len(array)):
       print("i =", i, "value =", array[i], "type =", type(array[i]))
       _ = array[i].upper()
   This will show which index has None.
2. Or quickly find indices containing None:
   bad_indices = [i for i, v in enumerate(array) if v is None]
   print("None at indices:", bad_indices)

Concrete fixes (choose one appropriate to your intent)

A. If None indicates missing data and you want to skip those entries
   for i in range(len(array)):
       if array[i] is None:
           continue
       _ = array[i].upper()
       classes.append(type(array[i]))

B. If None should be treated as an empty string
   for i in range(len(array)):
       s = array[i] if array[i] is not None else ""
       _ = s.upper()
       classes.append(type(s))

C. If array should never contain None and you want an explicit error earlier
   for i in range(len(array)):
       if array[i] is None:
           raise ValueError(f"array[{i}] is None; expected a string")
       _ = array[i].upper()
       classes.append(type(array[i]))

D. If you want to filter None out beforehand
   array = [v for v in array if v is not None]
   for i in range(len(array)):
       _ = array[i].upper()
       classes.append(type(array[i]))

E. If values may not be strings at all, coerce to str first
   for i in range(len(array)):
       s = str(array[i])
       _ = s.upper()
       classes.append(type(array[i]))  # note: type still reflects original type

Which one to use depends on whether None is expected:
- If None is a sign of a bug earlier, use option C and then fix the code that produced the None.
- If None is acceptable missing data, use A, B or D depending on how you want to treat it.

Summary
- The crash is caused by calling .upper() at line 766 on a None value in array.
- Find the index(es) containing None (debug step above) and either skip/replace/filter them or add a clear check/validation before calling .upper().
