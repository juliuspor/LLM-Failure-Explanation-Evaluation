Why it fails (root cause)
- Inside that while loop mantissa is never changed, so the loop condition mantissa % 10 == 0 never becomes false. The loop body computes check_val and tests it against value but does not update mantissa (or exp), so the loop can never make progress and will spin forever when mantissa has a trailing zero.
- Specifically:
  - On entry to the loop mantissa is set to value (line 177).
  - If mantissa ends in 0, mantissa % 10 == 0 is true and the loop body runs.
  - The body computes check_val but does not assign to mantissa (no mantissa = mantissa // 10 or mantissa //= 10), and does not change exp.
  - If check_val != value the loop continues to the next iteration with the same mantissa, causing an infinite loop.

Concrete fix (exact change to make)
- You must update mantissa inside the loop so the loop can terminate. A minimal fix is to divide mantissa by 10 each iteration. If the algorithm also intends to increase exp when stripping a trailing zero, update exp too.

Example patch (replace lines 181–184):
  while mantissa % 10 == 0:
      check_val = (mantissa // 10) * (10 ** (exp + 1))
      if check_val == value:
          mantissa //= 10        # progress: remove one trailing zero
          exp += 1              # if intended: account for the removed zero in exponent
      else:
          break                 # stop if removing a zero no longer matches value

Notes on choosing the right change
- If the intention was only to detect a removable trailing zero and then break, use mantissa //= 10 and possibly break when appropriate.
- If you only do mantissa //= 10 without changing exp, the rest of the code that uses exp (line 190) may be wrong; the example above shows the common pattern: strip a zero from mantissa and increment exp.

Summary
- The code fails because the while loop at line 181 never updates mantissa (or exp), so it never exits when mantissa ends with a zero. Fix by updating mantissa (and exp if required) inside the loop or breaking out when appropriate.
