What happened (where and what error)
- At line 766 you call array[i].upper(). If array[i] is None this raises AttributeError: "'NoneType' object has no attribute 'upper'".
- The traceback will point to the slice shown (lines 765–767). The exception originates before you ever append anything to classes (line 767).
- The code therefore fails whenever any element is None (or any element that does not have an upper() method, e.g. an int).

Why it fails (root cause)
- The docstring says "If any of these objects is None, a None element will be inserted into the array", but the code does the opposite: it tries to call .upper() on each element unconditionally. Calling .upper() assumes the element is a string (and non-None). For None (NoneType) there is no .upper method, so Python raises AttributeError immediately.
- The check/side-effect using .upper() is unnecessary and unsafe: it neither checks for None correctly nor preserves non-string types.

- Use type(array[i]) (or array[i].__class__) only after confirming array[i] is not None.
