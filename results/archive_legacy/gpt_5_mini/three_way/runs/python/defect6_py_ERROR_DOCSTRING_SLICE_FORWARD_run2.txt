What happened (root cause)
- At line 177 you do value = int(x). For x == -0.0, int(-0.0) yields 0 (a signed-zero is lost by converting to int).
- Later, at line 192 you do self.add(str(value)). Because value is 0, str(value) is "0". The test expected "-0.0" but the code emitted "0" (the negative sign was discarded by int()).
- The loss of the negative-zero sign happens right where you coerce x to an integer (line 177). That is why the test failed with "Expected -0.0, got 0".

Where to change (exact spot)
- Before line 177 (before value = int(x)), detect whether x is negative zero:
  - Use math.copysign or check x == 0.0 and math.copysign(1.0, x) < 0.
- Use that sign information when you later produce the string for zero instead of blindly using str(value).

   - sign_is_negative_zero = (x == 0.0 and math.copysign(1.0, x) < 0)

       self.add("-0" or "-0.0" depending on the expected format)

Why this fix works
- math.copysign lets you detect the negative sign on a zero-valued float. Using that flag when building strings preserves the negative-zero representation instead of losing it via int().

Summary
- The negative sign on -0.0 was lost by int(x) at line 177. Detect negative zero before that conversion and use that flag when creating the string at line 192 (or avoid adding the integer form when x is negative zero). This restores the expected "-0.0".
