What happened (where it fails)
- In replace_each (shown at the top of your slice), the loop starting at line 81 does:
  greater = len(replacement_list[i]) - len(search_list[i])
- If search_list[i] or replacement_list[i] is None, calling len(...) raises TypeError: object of type 'NoneType' has no len().
- That is why the function crashes when either list contains None even though those entries are intended to be ignored.

Why it happens (root cause)
- The code assumes every entry in search_list and replacement_list is a string and calls len(...) on them unconditionally (lines 81–83).
- There is no guard that skips None entries before taking their length.
- Also search_length is set from len(search_list) (line 52) so the loop may try to read replacement_list[i] even if replacement_list is shorter (potential IndexError), but the immediate crash reported (Lang-39) is the TypeError from len(None).

1. At or before line 81, skip entries that are None:
   - Replace the body of the for-loop starting at line 81 with a guard that checks for None before calling len().
   - Example replacement (conceptual, exact placement below):

     for i in range(search_length):
         if search_list[i] is None or replacement_list[i] is None:
             continue
         greater = len(replacement_list[i]) - len(search_list[i])
         if greater > 0:
             increase += 3 * greater

2. Ensure search_length is safe:
   - Compute search_length as the minimum of the two lists' lengths so you cannot index past the end of replacement_list:
     search_length = min(len(search_list), len(replacement_list))
   - This prevents IndexError when replacement_list is shorter than search_list.

3. (Optional, more robust) Iterate over pairs and skip None:
   - Replace indexed loop with a zip-based loop to make intent clearer:

     for s, r in zip(search_list, replacement_list):

Why these changes fix it
- The guard prevents calling len() on None, eliminating the TypeError.
- Using min(len(...)) or zip avoids indexing past the end of the shorter list and makes the function correctly ignore None entries as intended.
