What happened (root cause)
- The code treats any float that compares equal to its int value as an integer (line 176: if x == int(x) ...).
- For the input -0.0, x == int(x) is true because -0.0 == 0.
- The code then converts x to an int (value = int(x) at line 177). int(-0.0) yields 0, losing the negative-zero sign.
- Later the code calls self.add(str(value)) (line 192) and so emits "0" instead of the string representation of the original float "-0.0".
- The test expected "-0.0" but got "0" because negative zero was converted to int and its sign was lost.

Where it occurred (lines)
- The decision to treat the float as an integer happens at line 176.
- The sign loss happens at line 177 (value = int(x)) and then at line 192 (self.add(str(value))).
- You can preserve the negative-zero behavior by avoiding the int conversion for negative zero, or by detecting negative zero explicitly before converting.

How to fix it (concrete steps and patch)
Option A — simplest, preserve -0.0 by early check:
- Insert an explicit check for negative zero immediately after line 176 and before converting to int.

Example patch (insert after line 176):
  if x == 0.0 and math.copysign(1.0, x) < 0:
      # preserve negative zero string form
      self.add(str(x))
      return

This keeps "-0.0" because str(-0.0) yields "-0.0" and avoids int(x).

Option B — skip integer branch for any zero:
- Change the integer test so it does not trigger for ±0.0:

Replace line 176:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):

With:
  if x == int(x) and not math.isinf(x) and not math.isnan(x) and not (x == 0.0):

This causes any zero (including -0.0) to go to the else branch (line 194), which calls self.add(str(x)) and preserves the sign.

Why these fixes work
- math.copysign(1.0, x) < 0 detects negative zero reliably.
- Avoiding int(x) for negative zero prevents losing its sign, and using str(x) keeps Python's "-0.0" formatting.

Note on other edge cases
- The rest of your integer-to-mantissa/exponent logic is unchanged; this fix only prevents the integer-branch conversion from stealing the sign on -0.0.
