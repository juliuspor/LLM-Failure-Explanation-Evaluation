What happens
- The code crashes with a TypeError at lines 81–84 when it evaluates len(replacement_list[i]) or len(search_list[i]) for some i. The crash happens if either replacement_list[i] or search_list[i] is None (or a non-string without a len), because len(None) is invalid.

Why it happens (root cause)
- Earlier in the function you correctly skip null/empty search strings and null replacements when searching (see the checks at lines 28–49 and again at lines 101–106). But the pre-sizing loop at lines 81–85 does not perform the same guards. That loop assumes every element is a non-None string and does:
    greater = len(replacement_list[i]) - len(search_list[i])
  If replacement_list[i] or search_list[i] is None, calling len(...) raises TypeError. So the function fails before it ever completes its first match scan when the arrays contain None elements.

Where it occurs (specific lines and variables)
- Line 81: start of the pre-size loop for i in range(search_length)
- Line 82: greater = len(replacement_list[i]) - len(search_list[i])  <-- this is the failing expression when either element is None
- Related variables: replacement_list[i], search_list[i], increase

How to fix it (clear, actionable steps)
1. Apply the same guard used elsewhere before computing lengths. Replace lines 81–85 with a guarded version that skips null/empty search strings and null replacements, for example:

           or len(search_list[i]) == 0
           or replacement_list[i] is None
       ):
           continue
       greater = len(replacement_list[i]) - len(search_list[i])

3. Run tests covering cases where search_list or replacement_list contain None or empty strings to ensure no other unguarded uses remain.
