What happened (root cause)
- The test that failed calls ClassUtils.to_class with an array that contains a None element: test_array = ["Test", None, 99.0].
- The failure message "'NoneType' object has no attribute 'upper'" means somewhere inside ClassUtils.to_class the code calls .upper() on the array element without first checking for None.
- Because the second element is None, calling None.upper() raises AttributeError and the method crashes.

Where it happened (likely location)
- Inside hit07_classutils.py, in the ClassUtils.to_class method, in the loop that processes each element of the input array.
- There will be a line similar to:
    elem.upper()
  or
    if elem.upper() == ...
  that runs for every array element. When elem is None, elem.upper() triggers the AttributeError.

Why that causes the tests to fail
- The tests expect None elements to be handled gracefully: to_class([ "Test", None, 99.0 ]) should return [str, type(None), float].
- Because the code assumes every element is a string (or otherwise non-None) and calls .upper(), it never reaches the logic that maps an element to its class for None values. The AttributeError aborts the method before it can return the expected result.

Exact things to look for in the source
- In hit07_classutils.py, open the ClassUtils.to_class method.
- Find the loop that iterates the input array (variable names may be arr, array, objects, items, etc.).
- Look for any direct call to .upper() on the element (elem.upper(), item.upper(), value.upper()) without a prior None check.
- Also check for other string-method calls (.strip(), .lower(), etc.) used similarly.

Concrete fixes (step-by-step, minimal changes)
1. Add a None check before calling string methods on the element. Example change:

    # Before (problematic)
    for i, elem in enumerate(array):
        if elem.upper() == 'SOMEFLAG':
            ...
        result[i] = elem.__class__

    # After (fix)
    for i, elem in enumerate(array):
        if elem is None:
            result[i] = type(None)
            continue
        # safe to call string methods now
        if isinstance(elem, str) and elem.upper() == 'SOMEFLAG':
            ...
        result[i] = elem.__class__

2. Or keep mapping concise by using Python's type() safely:

    for i, elem in enumerate(array):
        result[i] = type(elem)   # type(None) for None automatically

   (This avoids any .upper() call entirely unless your logic needs string-specific checks.)

3. If the code must treat string-like tokens specially, check for strings explicitly:

    if isinstance(elem, str):
        tok = elem.upper()
        ...

Why these fixes work
- They prevent calling .upper() on None by testing elem is None or by checking isinstance(elem, str) first.
- Using type(elem) returns type(None) for None, which matches expectations in your tests.

Example minimal replacement implementing the intended behavior:

    def to_class(array):
        if array is None:
            return None
        result = [None] * len(array)
        for i, elem in enumerate(array):
            if elem is None:
                result[i] = type(None)
            else:
                result[i] = type(elem)
        return result

Summary
- The crash comes from calling a string method (.upper()) on a None element inside ClassUtils.to_class.
- Fix by checking for None (elem is None) or by using type(elem) and checking types with isinstance(elem, str) before calling .upper().
