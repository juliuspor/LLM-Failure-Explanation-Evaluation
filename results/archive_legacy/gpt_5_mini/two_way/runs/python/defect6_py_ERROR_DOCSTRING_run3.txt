- Common ways that happens:

Concrete examples of causes and fixes
- Cause: you did result = a - a or some operation that returns 0.0 (positive) even if inputs contained -0.0.
  Fix: avoid canceling ops if sign matters. If you need to preserve -0.0, compute sign explicitly and reapply:
  - Python:
    - import math
    - if math.copysign(1.0, a) < 0: result = -0.0 else: result = +0.0
  - Or when you need to keep sign across normalization: result = math.copysign(0.0, a)
- Cause: JSON or serialization normalized -0.0 to 0.
  Fix: JSON and many text formats do not preserve sign bit. If you must preserve sign, store an explicit flag or use a binary format that preserves IEEE sign bit (or store string "-0.0").
  - Example: instead of json.dump(value), dump {"value": value, "is_negative_zero": math.copysign(1.0, value) < 0}
- Cause: you converted float to integer (int(-0.0) → 0) or to string that omitted the minus.
  Fix: avoid the conversion or keep an explicit signed-zero flag, or reconstruct with math.copysign.
- Cause: the test used literal -0.0 but the function returned 0 (an integer) rather than float 0.0.
  Fix: ensure the function returns float 0.0. In Python return float(0) or 0.0, or in JS return -0 explicitly (Object.is(-0, value) can test).
- Cause: library function you relied on intentionally returns canonical +0.0.
  Fix: consult the library docs; if it canonicalizes zero you must post-process result with math.copysign to reapply sign where appropriate.
