What happened (where and what failed)
- The test that passes replacements = ["c", None] triggers an exception inside StringUtils.replace_each when the code reaches the loop at lines 81–84:
  - Line 82 executes len(replacement_list[i]) but replacement_list[i] is None for the second element.
  - That raises TypeError: object of type 'NoneType' has no len() (or generally fails because you can't call len() on None).
- So the failure happens at line 82 when the code assumes every item in replacement_list is a string and calls len() on it without checking for None.

Why it failed (root cause)
- The code intends to allow None in replacement_list to mean "no replacement" (the unit tests expect that behavior).
- But it unconditionally uses len(replacement_list[i]) to compute how much a replacement could increase output length:
  - greater = len(replacement_list[i]) - len(search_list[i])
- If replacement_list[i] is None, len(...) raises an error. There is no null/None check before using replacement_list[i], so the function crashes instead of skipping or treating None as "no replacement".

Where exactly (line numbers)
- The problem is in the loop starting at line 81; specifically line 82:
  - 81 |         for i in range(search_length):
  - 82 |             greater = len(replacement_list[i]) - len(search_list[i])
  - 83 |             if greater > 0:
  - 84 |                 increase += 3 * greater  # assume 3 matches

Concrete fixes (step-by-step)
1. Check for None before calling len():
   - Replace line 82 with guarded logic that handles None replacements. For example:
     - replacement = replacement_list[i]
     - if replacement is None:
         greater = 0
       else:
         greater = len(replacement) - len(search_list[i])
   - This prevents len(None) and makes a None replacement contribute zero to the preallocation estimate.

2. Ensure the rest of the code treats None as "no replacement":
   - Later when actually performing replacements, skip replacing when replacement is None (leave the original substring intact).
   - Example: when matching search_list[i], only append replacement_list[i] if replacement_list[i] is not None; otherwise append the matched text unchanged.

3. (Optional) Defensive checks for search_list entries:
   - If search_list may contain None, add similar guards (but based on your expected API, search_list entries should probably be strings).

Why this fix works
- It removes the invalid len(None) call and aligns behavior with the tests: None in replacement_list means "do not replace". The size-estimation code no longer assumes every replacement is a string and will not crash.

Minimal code change example (pseudo-patch to lines 81–84)
- Before:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater
- After:
  for i in range(search_length):
      replacement = replacement_list[i]
      if replacement is None:
          greater = 0
      else:
          greater = len(replacement) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater

This addresses the TypeError and makes the function behavior match the unit tests.
