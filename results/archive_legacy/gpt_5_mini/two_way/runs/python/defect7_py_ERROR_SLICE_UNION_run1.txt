What happened (where it failed)
- The exception came from line 766 in your slice: the code calls array[i].upper().
- The error says AttributeError: 'NoneType' object has no attribute 'upper' — that means array[i] was None at that index, so Python tried to call .upper() on None and crashed.

Why it happened (root cause)
- array is expected to contain string objects, but at least one element is None.
- At line 766 you do no check before calling .upper(), so when array[i] is None the call fails immediately.
- Because you still call classes.append(type(array[i])) on line 767, that would append NoneType for that element if the code reached it, but the exception prevents reaching line 767.

How to fix it (concrete options and exact steps)
Choose one of these depending on the intended behavior:

1) If None values should be ignored
- Replace the loop body with a check that skips None before calling .upper().

Example change (lines 765–767):
- Replace:
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))
- With:
  for i in range(len(array)):
      if array[i] is None:
          continue
      _ = array[i].upper()
      classes.append(type(array[i]))

2) If None should be treated as the empty string
- Convert None to "" before calling .upper():

Example:
  for i in range(len(array)):
      value = array[i] if array[i] is not None else ""
      _ = value.upper()
      classes.append(type(array[i]))

3) If every element must be a string and None indicates a bug earlier
- Fail early with a clear error message so the source of bad data is obvious:

Example:
  for i in range(len(array)):
      if array[i] is None:
          raise ValueError(f"array[{i}] is None but a string was expected")
      _ = array[i].upper()
      classes.append(type(array[i]))

4) Defensive approach with try/except that records the problem
- If you want to continue but log bad elements:

Example:
  for i in range(len(array)):
      try:
          _ = array[i].upper()
          classes.append(type(array[i]))
      except AttributeError:
          # handle or log the unexpected None or non-string value
          logger.warning("array[%d] is not a string: %r", i, array[i])
          classes.append(type(array[i]))

What to check upstream
- Inspect where array is created or modified to find why None appears. A None might come from a function that returns nothing, from missing dictionary keys, or from an optional field that wasn't set.
- Fixing the data source is usually best (so you don't need runtime checks everywhere).

Summary
- Root cause: array[i] is None at line 766, and None has no .upper().
- Fixes: either skip None, convert it to a string, raise a clear error, or log and continue — pick one and apply the exact replacement shown above.
