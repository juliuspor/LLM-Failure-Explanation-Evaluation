What happened (where it crashed)
- The crash happens at line 82 inside replace_each():
  82 |             greater = len(replacement_list[i]) - len(search_list[i])
- The exception you will see is IndexError: list index out of range. It means replacement_list[i] does not exist for some i that the loop tries to read.

Why it happened (root cause)
- At line 52 you set search_length = len(search_list).
- You then loop for i in range(search_length) (lines 59 and 81), so i ranges over indices valid for search_list.
- You use that same index i to access replacement_list[i] at line 82.
- If replacement_list has fewer elements than search_list, using indices up to search_length-1 will try to read past the end of replacement_list and raise IndexError. In short: the code assumes replacement_list and search_list are the same length but does not enforce that.

1) Validate lengths early and fail with a clear message (preferred if they must match):
   - Add after line 52 (after computing search_length):

         raise ValueError(
             f"replace_each: replacement_list length ({len(replacement_list)}) "
             f"does not match search_list length ({search_length})"
         )
   - This makes the failure explicit and points to the mismatched variables and lengths.

2) Or iterate pairs safely using zip (if it's acceptable to only process pairs that exist):
   - Replace the for loop (lines 59/81) with:
     for i, (s, r) in enumerate(zip(search_list, replacement_list)):
         greater = len(r) - len(s)
         ...
   - This prevents IndexError because zip stops at the shorter list.

3) Or iterate up to the minimum length and handle leftovers explicitly:
   - After line 52:
     loop_len = min(len(search_list), len(replacement_list))
   - Then use for i in range(loop_len): ...
   - And afterward handle any remaining unmatched items in whichever list is longer.

Why these fixes work
- Validation gives a clear, immediate error when the assumption (equal lengths) is broken.
- zip or min(...) removes the unsafe assumption by only accessing indices that exist in both lists.
- All fixes reference replacement_list and search_list explicitly and prevent the out-of-range index access at line 82.

Pick the approach that matches your intended behavior:
- If the algorithm requires a 1:1 mapping, add the explicit length check (option 1).
- If missing pairs should be skipped, use zip (option 2).
- If you need to process leftovers differently, use min(...) and then handle the remainder (option 3).
