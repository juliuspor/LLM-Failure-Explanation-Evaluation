What happened
- The tests that include None elements fail because ClassUtils.to_class tries to call .upper() on each array element.
- The failing code is at the slice you provided (lines 765–766): 
  - for i in range(len(array)):
  -     _ = array[i].upper()
- When array[i] is None, None has no .upper() method, so Python raises AttributeError (equivalent to the Java NullPointerException described in the comment). That stops execution and makes the test fail.

Why it fails (root cause)
- The method assumes every element is a string (or at least has an .upper() method). It does not:
  - check whether array is None,
  - check whether each element is None, or
  - determine the element's type safely.
- The test test_to_class_with_none_element (and others with None) expects None to be handled by returning type(None) for that slot, but the implementation tries to use a string operation instead.

Where exactly it occurred
- In ClassUtils.to_class, while iterating the input array, at line 766 the code calls array[i].upper(). That single call causes an AttributeError when array[i] is None. So the failure is inside the loop over array elements in ClassUtils.to_class.

Concrete fixes (step-by-step)
1. Handle a None input array:
   - If the passed array is None, return None immediately.
     - Example: if array is None: return None

2. For the per-element processing, do not call .upper() on the element. Determine the element's class/type safely:
   - If element is None, the element's class should be type(None).
   - Otherwise use type(element) (or element.__class__) to get its class.

3. Replace the .upper() line and any dependent logic with safe type extraction. For example:
   - result = []
   - for elem in array:
   -     if elem is None:
   -         result.append(type(None))
   -     else:
   -         result.append(type(elem))
   - return result

4. Add tests already present (they cover the cases) and run the suite.
