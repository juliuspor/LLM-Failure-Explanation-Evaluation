What happened (where and why)
- At line 49 the code computes gap = end - start.
- At line 52 it checks if gap <= 0 and raises ValueError("bound must be positive").
- The test expected the exception message to mention the variable start, but the raised message is the generic string "bound must be positive".
- Root cause: when end <= start the function raises a ValueError whose message does not reference start (or end), so the test that looks for the substring "start" in the exception message fails.

Exactly where to change it
- Modify the ValueError raised at line 52 so the message includes start (and preferably end) or describes their relationship. For example:
  - Replace line 52 with:
    raise ValueError(f"start ({start}) must be less than end ({end})")
  - Or, if you want to keep “positive” wording:
    raise ValueError(f"bound must be positive: end ({end}) - start ({start}) = {gap}")
