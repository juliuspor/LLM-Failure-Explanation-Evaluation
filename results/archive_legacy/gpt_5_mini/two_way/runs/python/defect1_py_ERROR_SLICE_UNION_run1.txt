- The exception message shows minutes_offset was -15, so the code rejected -15 as out of range.

Why it happened (root cause)
- The code validates minutes_offset before it applies the sign-adjustment logic at lines 402–405:
  - lines 402–405 adjust minutes_offset using hours_in_minutes and the sign logic:
    - minutes_offset = hours_in_minutes - minutes_offset
    - or minutes_offset = hours_in_minutes + minutes_offset
- That means minutes_offset can be negative immediately after parsing (for example when parsing "-4:15" the minutes part was parsed as -15), but the validation at 396–397 runs first and rejects negative values.
- In short: the minutes value was parsed as a negative number, the code checks range too early, and so valid inputs that need sign normalization (negative hour with positive minutes) are rejected.
