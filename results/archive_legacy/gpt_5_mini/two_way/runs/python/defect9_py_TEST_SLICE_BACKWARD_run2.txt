What happened
- The code crashes with a TypeError when a replacement entry is None.
- The failure occurs inside replace_each, in the second for-loop starting at line 81. The immediate bad expression is on line 82:
  greater = len(replacement_list[i]) - len(search_list[i])
- If replacement_list[i] is None, len(replacement_list[i]) raises TypeError. The test calls replace_each(..., ["a"], [None]) and replace_each(..., ["a","b"], ["c", None]), so replacement_list contains None and the code hits that line and fails.

Why that is the root cause
- The code assumes every element of replacement_list and search_list is a string and calls len(...) on them without checking for None.
- The tests expect null-safety: a None in replacement_list means "no replacement" (i.e. skip that pair) rather than treating None as an empty string. So the code should not attempt len(None).

- At the start of the loop that begins at line 81 (the loop that uses replacement_list[i] and search_list[i]), add guards for None:
  - If replacement_list[i] is None: skip this iteration (do not compute len(...) or attempt replacement).
  - Also skip if search_list[i] is None.
- This prevents the len(...) calls on None that cause the TypeError.

1. Open the method and find the for-loop starting at line 81 (the one that contains line 82).
