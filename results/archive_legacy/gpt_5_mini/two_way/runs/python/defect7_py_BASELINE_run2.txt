What happened (where, exact line)
- The failure occurs at line 766: _ = array[i].upper()
- When array[i] is None, calling .upper() raises AttributeError: 'NoneType' object has no attribute 'upper'. The test that triggers this is test_to_class_with_none_element which passes ["Test", None, 99.0].

Why it fails (root cause)
- The loop assumes every element is a string and calls .upper() unconditionally. None is not a string and has no .upper method, so Python raises an AttributeError.
- The method's purpose (and the tests) is to produce a list of types, inserting type(None) for None elements. The .upper() call is unnecessary for that purpose and is the immediate cause of the crash.

1) Simplest correct fix (recommended): remove the .upper() call entirely and append the element's type:

       elem = array[i]

   This returns [str, type(None), float] for ["Test", None, 99.0] as the tests expect.

2) If the .upper() was intended to normalize strings before some other operation, guard it:

Why these fixes work
- Removing or guarding the .upper() call prevents calling methods on None. The function only needs to compute types; calling .upper() provides no benefit and causes the crash for non-string elements.
