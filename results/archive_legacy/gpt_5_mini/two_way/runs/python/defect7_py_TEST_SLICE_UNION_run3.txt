What happened (what failed)
- The test fails when ClassUtils.to_class is given an array that contains None (e.g. ["Test", None, 99.0]).
- The failure occurs because the code calls array[i].upper() on every element without checking for None first. When array[i] is None, calling .upper() raises an AttributeError: 'NoneType' object has no attribute 'upper'.

Where it happened (exact place)
- In the provided slice, the problematic code is in the loop at lines 765–767:
  - Line 766: _ = array[i].upper()
  - Line 767: classes.append(type(array[i]))
- The AttributeError is raised at line 766 when array[i] is None.

Why it happened (root cause)
- The code assumes every element is a string (or at least has an .upper() method). The test suite passes non-string elements and None. None has no .upper() method, so array[i].upper() fails.
- The .upper() call is unnecessary for determining the element's class/type. The code already calls type(array[i]) on the next line, which is sufficient to return the runtime class/type of the element (including type(None) for None).

Concrete fixes (step-by-step)
1. Remove the unnecessary .upper() call, or guard it behind a None check. The simplest correct implementation of to_class is to return the types directly.
2. Ensure the method handles a None input array by returning None (your tests expect that).
3. Handle empty arrays and arrays with None elements by returning type(None) for None elements.

Replace the lines 765–767 with either:
