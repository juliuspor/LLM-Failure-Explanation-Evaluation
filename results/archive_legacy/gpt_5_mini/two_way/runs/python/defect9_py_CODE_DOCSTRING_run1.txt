What happened (where it fails)
- The failure occurs in the "get a good guess on the size of the result buffer" loop, at this line:
  greater = len(replacement_list[i]) - len(search_list[i])
- If either replacement_list[i] or search_list[i] is None, len(...) raises a TypeError. That crash happens before the later checks that would ignore None entries.

Why it happens (root cause)
- The code does correctly skip None or empty entries when searching and replacing (see the checks inside the first for-loop and inside the while/inner for-loop).
- But the size-estimation loop above those search/replace loops does not perform the same null/empty checks. It unconditionally calls len(...) on elements that may be None. So when one array element is None the function crashes even though the intention is to ignore such entries.

Exactly where to change it
- File: replace_each function
- Problematic lines (the "BUG" comment is already above it):
    for i in range(search_length):
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches
- These lines must skip entries where search_list[i] is None or empty, or replacement_list[i] is None.

Concrete fixes (pick one)
1) Skip None/empty elements (preferred):
   Replace that loop with:
       for i in range(search_length):
           if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
               continue
           greater = len(replacement_list[i]) - len(search_list[i])
           if greater > 0:
               increase += 3 * greater

2) Alternatively, treat None as length 0 (if that matches your semantics):
       for i in range(search_length):
           rep_len = len(replacement_list[i]) if replacement_list[i] is not None else 0
           search_len = len(search_list[i]) if search_list[i] is not None else 0
           greater = rep_len - search_len
           if greater > 0:
               increase += 3 * greater

Test example that triggers the crash
- Call replace_each("abc", ["a", None], ["X", None]) — the function will raise TypeError at the size-estimation loop.

Summary
- Root cause: a pre-check sizing loop calls len(...) on possibly None entries.
- Fix: add the same null/empty checks there (or treat None as length 0). After that change, entries with None will be ignored as intended and the crash will be resolved.
