What fails
- At runtime you get a NameError because the variable exp is used before it’s defined.
- The first use is on line 182: check_val = (mantissa // 10) * (10 ** (exp + 1)). Python raises NameError there. exp is used again on line 190 when building the "E" string.

Why it happens (exact lines and variables)
- The code reaches the while on line 181 because mantissa % 10 == 0. Entering the loop executes line 182, which evaluates (exp + 1). But exp never exists in the surrounding scope, so evaluation of check_val raises NameError at line 182.
- Because exp is not initialized before that loop, the later use at line 190 (self.add(str(mantissa) + "E" + str(exp))) would also fail if the NameError didn’t stop execution earlier.

Additional logic problem to be aware of
- Even if you simply add exp = 0 before the loop, the loop body as shown never changes mantissa (no division) or exp, so if mantissa % 10 == 0 the loop could either become infinite or be pointless. The intended behavior appears to be to strip trailing zeros from mantissa while counting how many tens you removed (increment exp as you divide by 10).

How to fix (concrete steps and a patch)
1. Initialize exp before the loop:
   - Add exp = 0 on the line before the while (before line 181).
2. Inside the while, divide mantissa by 10 and increment exp (or otherwise update them) so the loop progresses and check_val becomes meaningful. Use the intended test to decide whether to accept that division or break.

A concrete corrected snippet (replace lines 178–183 with this logic)
- Before the while: exp = 0
- Loop body example:

    mantissa = value
    exp = 0
    if abs(x) >= 100:
        while mantissa % 10 == 0:
            next_mantissa = mantissa // 10
            check_val = next_mantissa * (10 ** (exp + 1))
            if check_val == value:
                mantissa = next_mantissa
                exp += 1
            else:
                break

- Then lines 190 and 192 that use mantissa and exp will work because exp is defined and mantissa has been reduced correctly.

What you should see after the fix
