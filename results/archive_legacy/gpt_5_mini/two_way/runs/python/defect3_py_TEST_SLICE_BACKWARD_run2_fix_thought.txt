The bug is that translate advances pos incorrectly after translate_codepoint returns >0. Need to advance pos by the number of UTF-16 code units consumed, not by summing char_count of code_point_at repeatedly. Simpler: when consumed>0, determine how many code units were consumed by reading code_point_at at pos to get codepoint then increment pos by char_count(codepoint) for each consumed translator codepoint processed. But consumed is number of codepoints (not code units) consumed; if translator consumed n codepoints, we should advance over n codepoints, each might be 1 or 2 code units. Implement a loop for _ in range(consumed): cp = code_point_at(code_units,pos); pos += char_count(cp). Also ensure index bounds guard. This fixes off-by-one. Replace function accordingly and keep signature same.
