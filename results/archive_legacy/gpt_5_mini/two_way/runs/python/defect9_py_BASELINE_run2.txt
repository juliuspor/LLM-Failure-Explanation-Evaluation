What happened (root cause)
- The code at lines 81–84 computes lengths without checking for None:
  > greater = len(replacement_list[i]) - len(search_list[i])
- If either replacement_list[i] or search_list[i] is None, calling len(None) raises TypeError: "object of type 'NoneType' has no len()".
- That is exactly what the failing test does: replace_each("aba", ["a"], [None]) passes replacement_list[0] == None, so len(replacement_list[0]) raises the TypeError. The crash happens at the loop starting on line 81.

Why this is inconsistent with the rest of the function
- Earlier loops (lines ~59 and ~102) explicitly skip entries where search_list[i] is None or empty or replacement_list[i] is None. The size-estimate loop (lines 81–84) was not made null-safe and so contradicts the intended "ignore None entries" behavior described in the docstring and used by the rest of the function.

Exactly where to change it
- Modify the loop at lines 81–84 to skip entries that the rest of the function also skips:
  if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None: continue
- Only then compute len(replacement_list[i]) and len(search_list[i]).

Suggested fix (precise code to replace the buggy loop)
Replace:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater  # assume 3 matches
with:
  for i in range(search_length):
      if (
          no_more_matches_for_repl_index[i]
          or search_list[i] is None
          or len(search_list[i]) == 0
          or replacement_list[i] is None
      ):
          continue
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater  # assume 3 matches

Why this fixes the tests
- It prevents calling len(...) on None values, matching the earlier null/empty checks so entries that should be ignored are ignored consistently. After this change the tests that use None in the arrays will no longer trigger the TypeError.
