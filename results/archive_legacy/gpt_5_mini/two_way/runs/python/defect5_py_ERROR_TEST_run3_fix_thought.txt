In Python context, lists are untyped, but tests expect TypeError when inferred_type==object and expected_type!=object. The bug arises because inferred_type is set to object when both are None. Implement logic: if array is not None: inferred_type=type(array) elif element is not None: inferred_type=type(element) elif expected_type is not None: inferred_type=expected_type else: inferred_type=object.
