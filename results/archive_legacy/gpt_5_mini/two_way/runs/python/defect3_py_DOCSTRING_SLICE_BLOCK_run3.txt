What happened (symptom)

- That IndexError is observed instead of the ValueError promised by the docstring when out is None (or instead of a controlled runtime behavior).

Why it happened (root cause)
1. The code never checks whether out is None before it computes or uses index and code_units. The docstring says the implementation should raise ValueError if out is None, but the actual code reaches the index-bounds check (lines 35–36) first. If code_units is built from input_seq or an abstract method result that is shorter than expected, index becomes out of bounds and the method raises IndexError.

In plain terms: the program uses the variables index and code_units without ensuring the preconditions the docstring promises (specifically that out is not None and that index values are valid for code_units). Because the bounds check happens later, you see IndexError at the point of indexing rather than the intended ValueError at the start.

- Actual failing check: lines 35–36 (the index bounds test that raises IndexError).
