What happened (root cause)
- You intended to clamp the input value to the allowed range using v:
  ```
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
  ```
  but the grayscale computation still uses the original value variable:
  ```
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  ```
  Because you used value instead of v there, the clamping has no effect. If value is outside the bounds you set with _lower_bound/_upper_bound, g can be < 0 or > 255 and you hit the ValueError check. That is the direct reason the code "fails" when you pass an out-of-range input.

Exactly where it occurs
- Lines that set v (the clamp) are correct but unused for the conversion.
- The failing line is the g calculation (the one that starts with g = int(...)). It should use the clamped v.
- The ValueError check that follows then raises because g was computed from the unclamped value.

Other issues you should also fix
1. Division by zero: if self._upper_bound == self._lower_bound the denominator (self._upper_bound - self._lower_bound) is zero and the calculation will raise ZeroDivisionError.
2. The ValueError message is unhelpful: it literally says "Color parameter outside of expected range: Red Green Blue" instead of showing which value or range caused the error.
3. You might want round() instead of int() if you want nearest integer instead of truncation.

Concrete fixes (step-by-step)
1. Use the clamped v in the grayscale calculation (replace value with v).
2. Guard against zero range before dividing.
3. Improve the error message to show g and the allowed range, or clamp g to [0,255] if that behavior is acceptable.
4. Optionally use round() for better rounding.

Suggested corrected code (drop-in replacement)
```
def get_paint(self, value: float) -> Tuple[int, int, int]:
    # Clamp value to bounds
    v = max(value, self._lower_bound)
    v = min(v, self._upper_bound)

    span = self._upper_bound - self._lower_bound
    if span == 0.0:
        # avoid division by zero; decide desired behavior (use lower bound here)
        normalized = 0.0
    else:
        normalized = (v - self._lower_bound) / span

    g = int(round(normalized * 255.0))

    # Validate the range
    if g < 0 or g > 255:
        raise ValueError(f"Computed grayscale {g} outside 0-255 for input value {value} (clamped to {v})")

    return (g, g, g)
```

How to verify the fix
- Test with value < _lower_bound and value > _upper_bound. The returned tuple should reflect the clamped color (0 or 255) rather than raising.
- Test with value exactly equal to bounds and mid-range values.
- Test with _upper_bound == _lower_bound to ensure no crash and that the behavior matches your intent.

That pinpoints the bug: the clamp is applied to v but the conversion used value, so the clamp was ignored. Use v in the conversion and add the zero-span check and a clearer error message.
