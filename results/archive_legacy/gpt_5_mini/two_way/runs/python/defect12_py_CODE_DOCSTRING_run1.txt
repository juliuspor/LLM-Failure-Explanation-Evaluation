What happened (where it failed)
- The code raises IndexError in the call to substring_java(text, 0, upper).
- This occurs when lower > len(text) (for example abbreviate("0123456789", 15, 20, None)).
- Why: earlier you clamp upper to len(text), but then you do: if upper < lower: upper = lower. If lower is greater than len(text), this sets upper to a value > len(text). Later substring_java(text, 0, upper) is called with end > len(text) and that causes the IndexError.

Exactly which lines cause it
- Line: if upper == -1 or upper > len(text): upper = len(text)
  -> upper is set to at most len(text).
- Line: if upper < lower: upper = lower
  -> when lower > len(text), this reassigns upper to a value > len(text).
- Line: result = substring_java(text, 0, upper)
  -> substring_java is called with end > len(text), raising IndexError.

Why this is the root cause (brief)
- You clamp upper down, but then you may raise it back above the string length by setting upper = lower. There is no protection against lower itself being out of range. substring_java expects end <= len(text), so calling it with end > len(text) triggers the exception.
