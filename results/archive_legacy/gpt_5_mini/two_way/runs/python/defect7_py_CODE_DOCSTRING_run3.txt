What fails
- The line "_ = array[i].upper()" causes an AttributeError in many normal cases.
  - If array[i] is None, calling .upper() raises AttributeError: 'NoneType' object has no attribute 'upper' (this matches your "Raises" doc but contradicts the docstring text that says a None element should be inserted).
  - If array[i] is any non-string object (int, dict, custom object, etc.), .upper() does not exist and you get AttributeError: '<type>' object has no attribute 'upper'.
- Because that .upper() call runs before you append type(array[i]), the whole function aborts on the first non-string / None element instead of returning a list of types or inserting a None entry.

Where it happens (exact code reference)
- The failing statement is the third line inside the loop:
    _ = array[i].upper()
  This line is executed for every element before classes.append(type(array[i])).

Why it fails (root cause)
- You appear to be trying to normalize strings by calling .upper(), but the function's purpose (per the docstring) is to return the runtime Class/type of each element, and to insert None into the output where the input element is None. Calling .upper() on every element assumes every element is a string and will throw AttributeError for None or any non-string object. That causes the method to fail at that line rather than doing what the docstring promises.

How to fix it (clear steps)
1. Remove the unconditional .upper() call.
2. Check each element for None first; if None, append None to classes.
3. Otherwise append the element's type (type(array[i])).
4. (Optional) If you need to treat strings specially (e.g., normalize case before deciding type) do that only when isinstance(array[i], str).
5. Ensure ArrayUtils.EMPTY_CLASS_ARRAY is a list (so .copy() works) or replace it with an empty list [].

Suggested corrected implementation (explicit steps you can paste)
- Replace the loop with:

    classes = []
    for i in range(len(array)):
        if array[i] is None:
            classes.append(None)
        else:
            classes.append(type(array[i]))
    return classes

- Or equivalently, a concise comprehension:

    return [None if x is None else type(x) for x in array]

Notes about other minor issues
- The type hint Optional[List[Type]] is okay conceptually, but in Python you usually use typing.Type or simply List[type]. That is not what causes the runtime failure here.
- If ArrayUtils.EMPTY_CLASS_ARRAY is not a list (e.g., a tuple or undefined), calling .copy() may also fail. If you expect an empty list, returning [] is simpler.

Summary
- Root cause: the unconditional call to array[i].upper() (line "_ = array[i].upper()") tries to call .upper() on values that are None or not strings, causing AttributeError and preventing the function from returning the expected list of types. Fix by removing that call and checking for None / string only where appropriate.
