What happens
- Python raises an IndentationError (or "unindent does not match any outer indentation level") at line 184.
- The immediate cause is that line 184 ("mantissa //= 10") is indented differently than the other lines in the same block.

Where it occurs
- Inside add_number, in the while-loop block that starts at line 181.
- Specifically: lines 181–186 form the while and its if; line 184 has one extra leading space compared to lines 183 and 185 (see slice lines 181–186).

Why it fails (root cause)
- Python uses indentation to define blocks. Every statement in the same block must have identical indentation (and mixing tabs and spaces or different counts of spaces breaks that).
- Line 184 is misindented (extra space), so Python cannot match it to the surrounding block and raises an indentation error.

How to fix it (concrete steps)
1. Open the file in an editor that shows invisible characters (or run a whitespace diff).
2. Make the indentation of line 184 exactly match the other lines in the if-block (use 4 spaces per indentation level). For example, replace:
       if check_val == value:
            mantissa //= 10
            exp += 1
   with consistent 4-space indents:
       if check_val == value:
           mantissa //= 10
           exp += 1
3. Ensure you use spaces consistently (PEP 8 recommends 4 spaces). If your file mixes tabs and spaces, convert all tabs to spaces.
   - Many editors have "convert tabs to spaces" or "reindent" commands.
   - Or run: python -m tabnanny <file> to find inconsistent indentation.
4. Optionally re-run python -m py_compile <file> to confirm there are no syntax/indentation errors.

What to expect after fixing
- The IndentationError will go away and the code will run into the intended logic that strips trailing zeros from integer-valued floats >= 100.
