What happens
- Root cause: the loop that computes increase (lines 81–82 in your slice) calls len(replacement_list[i]) and len(search_list[i]) without checking whether those elements are None (or empty). If either element is None this raises TypeError: object of type 'NoneType' has no len(), so the function fails there.
- Where it occurs: the problematic loop starts at "for i in range(search_length):" just before the comment about assuming 3 matches (your slice lines ~81–82).
- Why it happens: earlier you explicitly skip None/empty search strings and None replacements when finding matches (the for-loops that look up temp_index). But the increase-calculation loop does not perform the same null/empty checks, so it is not null-safe for elements inside the arrays (you even note this in the BUG comment).

Concrete symptom
- If search_list or replacement_list contains None at index i, the line
    greater = len(replacement_list[i]) - len(search_list[i])
  raises TypeError because len(None) is invalid.

Exact fix (step-by-step)
1. Apply the same null/empty checks you use elsewhere before calling len on elements. For example, replace that loop with:

    for i in range(search_length):
        if (
            search_list[i] is None
            or len(search_list[i]) == 0
            or replacement_list[i] is None
        ):
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches

2. Keep the rest of the code the same (including the cap: increase = min(increase, len(text) // 5)).

Test cases to verify
- search_list = ["a", None], replacement_list = ["b", "c"] → must not crash (skip None).
- search_list = ["a", ""], replacement_list = ["b", "c"] → skip empty search entries.
- search_list = ["a"], replacement_list = [None] → skip None replacement and not crash.

Why this is the right fix
- It makes the increase-calculation loop consistent with the match-finding loops that already guard against None/empty values.
- It prevents calling len() on None and avoids raising TypeError while preserving the intended buffer-sizing heuristic.
