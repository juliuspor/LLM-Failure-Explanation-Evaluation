Why it fails (root cause)
- The code assumes `index` is always an integer. Python will pass a slice object to `__getitem__` when the caller uses slice syntax (`a:b`).
- A slice object cannot be compared with integers, so the boolean expression `index < 0` is invalid for slices.
- In short: the function does not handle the slice case before doing integer-only checks.
