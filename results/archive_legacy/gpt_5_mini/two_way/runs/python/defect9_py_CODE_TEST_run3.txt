What happens
- The code crashes while computing the "increase" heuristic:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater
- If either replacement_list[i] or search_list[i] is None (or search_list[i] is ""), calling len(...) raises a TypeError. In your tests replacement_list contains None (e.g. ["a"], [None]) so len(None) fails.

Where it occurs (specific variables/lines)
- The buggy loop is the one immediately after the comment "# BUG: this loop is not null-safe for elements inside the arrays".
- It uses replacement_list[i] and search_list[i] without the null/empty checks that the rest of the function uses (the later matching loops check for search_list[i] is None, len(search_list[i]) == 0, or replacement_list[i] is None before using them).
- The failing test is test_null_safety_inside_arrays_single which calls replace_each("aba", ["a"], [None]); the len(replacement_list[0]) call raises TypeError: object of type 'NoneType' has no len().

Why it happens (root cause)
- The function intends to skip null/empty search strings and null replacements in matching loops, but the "increase" pre-sizing loop forgot to perform the same checks. So a None value reaches len(), causing the exception. The "increase" value is just a heuristic for buffer sizing; it is not required for correctness, so this bug is purely from missing checks in that loop.

How to fix it (concrete steps)
1. Add the same null/empty checks inside the "increase" loop as used elsewhere:
   - Skip the entry if search_list[i] is None
   - Skip if len(search_list[i]) == 0
   - Skip if replacement_list[i] is None
2. Then compute greater only for entries that pass the checks.

Example corrected loop (replace the buggy loop with this):
    for i in range(search_length):
        if (
            search_list[i] is None
            or len(search_list[i]) == 0
            or replacement_list[i] is None
        ):
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater

Alternative: remove the heuristic entirely (safe but less efficient). Either way, ensure you don't call len(...) on None.

What will happen after the fix
- The function will no longer raise a TypeError for inputs where some search or replacement entries are None or empty; tests test_null_safety_inside_arrays_single and test_null_safety_inside_arrays_mixed will pass.
