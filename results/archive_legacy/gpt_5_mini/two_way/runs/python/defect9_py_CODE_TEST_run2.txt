What fails
- The code raises a TypeError at runtime when one of the entries in search_list or replacement_list is None.
- This happens inside the "increase" sizing loop (the loop immediately after the comment "# BUG: this loop is not null-safe for elements inside the arrays"), where the code does:
  greater = len(replacement_list[i]) - len(search_list[i])
  If replacement_list[i] or search_list[i] is None, calling len(...) raises TypeError: object of type 'NoneType' has no len().

Why it happens (root cause)
- Earlier in the function you deliberately treat None and empty search/replacement entries as "skip" entries in the matching loops:
  if no_more_matches_for_repl_index[i] or search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
      continue
- But the sizing loop that calculates "increase" does not perform the same null/empty checks. It assumes every search_list[i] and replacement_list[i] is a non-empty string, so len(...) is called unguarded.
- For example, test_null_safety_inside_arrays_single calls replace_each("aba", ["a"], [None]). In that case replacement_list[0] is None, so len(replacement_list[0]) raises the TypeError in the sizing loop before any matching logic runs.

Exactly where this happens (variable and location)
- Function: replace_each
- Faulty code block (right after "increase = 0"):
    for i in range(search_length):
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater
- Problem variables: replacement_list[i], search_list[i]
- Failing operation: calling len(...) on a None value
- Failing tests: test_null_safety_inside_arrays_single and test_null_safety_inside_arrays_mixed (both include None in arrays)

How to fix it (clear, actionable steps)
1. Skip entries where search_list[i] is None or empty, or replacement_list[i] is None, in the sizing loop—use the same guards as the matching loops.
2. Replace the sizing loop with a null-safe version, for example:

    increase = 0
    for i in range(search_length):
        if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater

3. Alternatively you can check types or wrap len(...) in try/except, but the explicit guard above keeps behavior consistent with the rest of the method.

What to expect after the fix
- The tests that supply None entries in the arrays will no longer cause a TypeError.
- The sizing estimate will be computed only for valid (non-None, non-empty search) pairs, matching the rest of the function's null-safety policy.
