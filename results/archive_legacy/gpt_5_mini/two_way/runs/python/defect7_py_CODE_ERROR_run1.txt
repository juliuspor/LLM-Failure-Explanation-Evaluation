What happened (root cause)
- The exception is raised on this line inside the loop:
  _ = array[i].upper()
- That AttributeError means array[i] is None at runtime, and None has no method upper().
- So the code fails because it unconditionally calls .upper() on every element of array, but some elements are None.

Where it occurs
- Inside the for loop that begins at:
  for i in range(len(array)):
- Precisely on the first element for which array[i] is None.

Why this happened (clear, concrete)
- Your function accepts Optional[List[Any]] (so elements can be any type, including None).
- You call .upper() without checking element type or None, assuming every element is a string.
- When the list contains a None (or any non-string object that lacks upper()), Python raises AttributeError.

How to fix it (step-by-step options)
Pick the behavior you actually want and apply one of these explicit fixes:

1) If you want to skip None elements:
- Before calling .upper(), check for None.
- Example change inside the loop:
    if array[i] is None:
        continue
    _ = array[i].upper()
  This keeps other elements but ignores None.

2) If you want to treat None as an empty string (or some default):
- Replace None with a default before calling .upper():
    value = array[i] if array[i] is not None else ""
    _ = value.upper()

3) If you only want to call .upper() for strings and otherwise leave elements alone:
- Check the type first:
    if isinstance(array[i], str):
        _ = array[i].upper()
    else:
        # handle non-strings how you want (skip, convert, raise)
        pass

4) If you expect the list to contain only strings and want to fail fast with a clearer error:
- Validate up-front and raise a helpful TypeError:
    for idx, v in enumerate(array):
        if not isinstance(v, str):
            raise TypeError(f"array[{idx}] must be str, got {type(v).__name__}")
