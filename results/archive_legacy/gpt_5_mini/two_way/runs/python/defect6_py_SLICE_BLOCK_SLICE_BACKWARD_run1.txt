What happens
- The test on line 176 calls int(x) before math.isinf(x) / math.isnan(x). Python evaluates the whole expression left-to-right, so int(x) is executed even when x is NaN or ±Inf.
- Converting NaN or ±Inf to int raises an exception:
  - int(float('nan')) raises ValueError: cannot convert float NaN to integer
  - int(float('inf')) raises OverflowError: cannot convert float infinity to integer
- Because those conversions happen inside the if condition, the code throws at line 176 and never reaches the subsequent checks or the rest of the block.

Where it occurs
- Line 176: the conditional expression `if x == int(x) and not math.isinf(x) and not math.isnan(x):` — int(x) is evaluated first and can fail for NaN/Inf.

Why that order matters
- The boolean operators do not prevent evaluation of the left operand first; Python must evaluate int(x) to compare it to x before it can decide the whole condition. The guards intended to protect the int conversion are placed after it, so they are ineffective.

How to fix (clear steps)
1. Test for finiteness before converting:
   - Replace line 176 with a finiteness check first:
     if math.isfinite(x) and x == int(x):
   - Or equivalently:
     if not math.isinf(x) and not math.isnan(x) and x == int(x):
   This ensures you only call int(x) when x is a finite number.

2. Avoid duplicated conversions: compute int(x) once after the finiteness check, e.g.
   if math.isfinite(x) and x == int(x):
       value = int(x)
       ...

3. Prefer float helper methods (cleaner):
   if math.isfinite(x) and x.is_integer():
       value = int(x)
       ...

Minimal corrected version of the if line (replace line 176):
- Option A: if math.isfinite(x) and x == int(x):
- Option B (more Pythonic): if math.isfinite(x) and x.is_integer():

That will stop the ValueError/OverflowError on NaN/Inf and keep the rest of your logic intact.
