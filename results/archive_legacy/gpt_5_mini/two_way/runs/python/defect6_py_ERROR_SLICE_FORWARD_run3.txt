What happened (root cause)
- At line 177 you do value = int(x). Converting -0.0 to an int produces 0, and the negative sign is lost. Python’s int(-0.0) -> 0, so value no longer carries the negative-zero sign.
- Later you call self.add(str(value)) (line 192). That adds "0" (no sign), so the test sees 0 instead of -0.0 and the assertion fails: Expected -0.0, got 0.

Where it occurs
- The sign loss occurs exactly at line 177 (value = int(x)).
- The wrong output is produced at line 192 (self.add(str(value))). The code path that checks/normalizes mantissa and exp (lines 180–183 and 190) also uses value, so the sign loss can affect those outputs as well.

Why int() is the problem
- int() only preserves numeric magnitude; it does not preserve the sign of a signed zero. Negative zero is a distinct floating-point value (x == 0.0 is True but sign is negative), and int() maps both -0.0 and 0.0 to the same integer 0.
- Because you rely on value for later string output, you lose the negative-zero information.
