What happened (where)
- Crash occurs at the code shown in the slice: lines 81–84:
  ```
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater  # assume 3 matches
  ```
- The code calls len(replacement_list[i]) and len(search_list[i]) without checking their types. If either list element is None, len(None) raises TypeError and the function crashes.

Why it happened (root cause)
- The docstring says None entries in search_list or replacement_list should be ignored, but the loop assumes every element is a string.
- When replacement_list[i] or search_list[i] is None, the expression len(None) is invalid. That is the immediate cause of the crash.
- The logic also assumes both lists have usable entries at index i; even if lists are different lengths this loop may mis-handle indices, but the reported bug is the None-case.

Exactly where to change it (concrete fix)
Replace the body of the loop with an explicit check for None before calling len():

1) Minimal change (skip None entries):
  ```
  for i in range(search_length):
      if search_list[i] is None or replacement_list[i] is None:
          continue
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater
  ```

2) Safer alternative using zip (avoids index mismatches and is clearer):
  ```
  for s, r in zip(search_list, replacement_list):
      if s is None or r is None:
          continue
      greater = len(r) - len(s)
      if greater > 0:
          increase += 3 * greater
  ```

3) If you must preserve search_length (and some lists may be shorter), use itertools.zip_longest and explicit defaults:
  ```
  from itertools import zip_longest
  for s, r in zip_longest(search_list, replacement_list, fillvalue=None):
      if s is None or r is None:
          continue
      greater = len(r) - len(s)
      if greater > 0:
          increase += 3 * greater
  ```

Notes / rationale for the steps
- The key is to check for None before calling len(). That satisfies the requirement "None replacements should be ignored".
- Using zip or zip_longest avoids off-by-one or IndexError problems if the two lists have different lengths.
- The examples show exactly which variables and lines to change: add the if-check at the start of the loop before len(...) is used.
