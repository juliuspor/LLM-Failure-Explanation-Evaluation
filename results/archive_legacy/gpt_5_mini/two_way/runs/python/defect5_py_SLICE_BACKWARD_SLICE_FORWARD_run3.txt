What happened (where it fails)
- At line 336 the code sets inferred_type = type(array) when array is not None.
- At line 338 the code sets inferred_type = type(element) when element is not None.
- Then at line 340 inferred_type = object appears with no conditional guarding it — it always runs and overwrites whatever was set at lines 336 or 338.
- At line 346 the code checks if inferred_type == object and expected_type != object and raises a TypeError. Because line 340 unconditionally set inferred_type to object, that check becomes true and the function raises the TypeError even when array or element actually provided a concrete type.

Why this is the root cause (simple)
- inferred_type is overwritten unconditionally at line 340. The earlier assignments at lines 336 and 338 become useless because inferred_type is always reset to object before the later type-check at line 346. The code therefore treats every call as if no type could be inferred.
