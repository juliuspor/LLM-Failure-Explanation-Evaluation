What happened

  396 |         if minutes_offset < 0 or minutes_offset > 59:
  397 |             raise ValueError(f"Minutes out of range: {minutes_offset}")
- That negative value comes from applying the sign to the parsed minutes component too early. A negative timezone like "-00:30" produces minutes_offset = -30 before the range check, so the check fails even though the minutes component (30) is valid.

Why it fails (root cause)
- The code validates minutes_offset while minutes_offset still carries the sign. The minutes component should be validated as a non‑negative value 0..59, then the sign should be applied when you compute the total offset in minutes.
- Because the sign is applied before validation, valid inputs with negative offsets (e.g. "-00:30", "-05:15") trigger the "Minutes out of range" error incorrectly.

- Later code (lines 402, 404) expects minutes_offset to be the unsigned minute component or to be recombined with hours_in_minutes, so the sign/validation mismatch is the root problem.
