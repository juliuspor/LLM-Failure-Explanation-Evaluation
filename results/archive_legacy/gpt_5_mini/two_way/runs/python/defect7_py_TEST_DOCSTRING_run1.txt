What happened (what failed)
- The test fails with an AttributeError when ClassUtils.to_class is given an array that contains None (example: ["Test", None, 99.0]).
- The failure occurs inside ClassUtils.to_class where the code calls an attribute or method on the array element without first checking for None. Concretely, the code is calling something like obj.getClass() (a Java-style call) or otherwise accessing an attribute on obj when obj is None, producing "'NoneType' object has no attribute 'getClass'" (or similar) at the line that processes the element at index 1.

Why it happened (root cause)
- The method assumes every element is a real object and uses an instance method/attribute to obtain its class. In Python, None does not have that method/attribute, so attempting to call it raises AttributeError.
- The docstring even says "If any of these objects is None, a None element will be inserted", but the implementation does not implement that behavior — it directly accesses a member of the element instead of checking for None first.
- The bug is isolated to the loop that converts elements to classes. Relevant variable names: array (input), obj (each element), classes (output). The error occurs when processing obj == None.

Where it occurs
- In hit07_classutils.py inside ClassUtils.to_class, in the loop over the input array where the code does something like:
    classes[i] = obj.getClass()   # or obj.__class__ without guarding None
  This is the line that raises AttributeError when obj is None. The failing test demonstrates it at index 1 of the test_array in test_to_class_with_none_element.

How to fix it (explicit steps)
1. Handle None array input:
   - At the start of to_class, if array is None: return None.

2. Check each element for None before trying to access its class:
   - Replace the direct attribute call on the element with a guarded version. Example implementation:

    def to_class(array):
        if array is None:
            return None
        classes = [None] * len(array)
        for i, obj in enumerate(array):
            if obj is None:
                classes[i] = type(None)
            else:
                classes[i] = type(obj)   # or obj.__class__
        return classes

   - This ensures None elements produce type(None) (or whatever sentinel you prefer) instead of raising an exception.

3. Remove any Java-style getClass() calls:
   - If the code currently uses obj.getClass() (a Java idiom), replace with Python's type(obj) or obj.__class__ as shown above.

Why this fix is correct
- It matches the documented behavior: None input returns None; None elements result in a corresponding None-class entry rather than an exception.
- It prevents AttributeError by refusing to call methods or access attributes on None.
