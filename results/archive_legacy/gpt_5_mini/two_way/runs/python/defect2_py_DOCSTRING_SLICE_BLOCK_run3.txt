Why it fails (root cause)
1. Only g is tested at lines 74–75. There is no check shown for r or b at or around these lines, so out-of-range r or b would not be caught here (or may be caught elsewhere, but not at these lines).
2. The error message is misleading:
   - It always prints "Red Green Blue" instead of naming the offending variable or showing its value.
   - The message implies all three components are the problem, but the check is only for g. That leaves the developer guessing which component failed.
3. The docstring is inconsistent with the implementation:
   - Docstring says "A tuple (r, g, b) representing the grayscale color" — that’s contradictory (grayscale uses a single intensity, not RGB components). This makes it unclear what the function should return and what values to validate.

         raise ValueError(f"Color parameter out of range: r={r} (expected 0–255)")
     - if not (0 <= g <= 255):
         raise ValueError(f"Color parameter out of range: g={g} (expected 0–255)")
     - if not (0 <= b <= 255):
         raise ValueError(f"Color parameter out of range: b={b} (expected 0–255)")
   - This gives the exact variable, its value, and the expected range.
