What happened (where it failed)
- The call to substring_java(text, 0, upper) raises IndexError in substring_java (the slice check at lines 32–33: "if ... end > len(s): raise IndexError(...)").
- That IndexError happens because upper can end up greater than len(text) at the time substring_java is called.

Why it happened (root cause)
- The code first clamps upper down to len(text) only if upper == -1 or upper > len(text).
  (your code: if upper == -1 or upper > len(text): upper = len(text))
- Then it does: if upper < lower: upper = lower.
- If lower is larger than len(text), the second step sets upper = lower (so upper > len(text)). Because you already did the clamp earlier, you never re-clamp upper after raising it to lower. As a result upper can be > len(text) by the time substring_java(text, 0, upper) is called, causing substring_java to raise IndexError ("String index out of range: end") at the slice check (lines 32–33).
