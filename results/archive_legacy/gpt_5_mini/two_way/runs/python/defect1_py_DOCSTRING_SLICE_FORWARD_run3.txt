What happened (where)
- The bug is in the code that combines hours_in_minutes and minutes_offset into a single minutes value, around lines 402–405:
  - line 402: minutes_offset = hours_in_minutes - minutes_offset
  - line 404: minutes_offset = hours_in_minutes + minutes_offset
  - line 405: offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)
- For some sign combinations the arithmetic produces the wrong total minutes (and can then trigger the "Offset is too large" ValueError at line 407).

Why it fails (root cause)
- The code mixes signs incorrectly. It uses minutes_offset (the input minutes, which may be negative) directly in subtraction at line 402. When both hours_offset and minutes_offset are negative (for example hours_offset = -2, minutes_offset = -15), the code computes:
  - hours_in_minutes = -2 * 60 = -120
  - minutes_offset = hours_in_minutes - minutes_offset = -120 - (-15) = -105
  That produces -105 minutes (−1:45) instead of the expected −135 minutes (−2:15).
- In short: the code subtracts a negative minutes value, which effectively adds it, producing the wrong sign/magnitude for the combined offset. The function therefore constructs an incorrect offset (or one outside allowed bounds) and raises ValueError.

How to fix it (concrete steps, with exact replacements)
Replace the current sign-handling arithmetic with explicit, correct combination of hours and minutes. Two equivalent, clear fixes:

1) Use a sign based on the hours and apply it to the absolute minutes:
- Replace the block at/around line 402 with:
  - sign = 1 if hours_offset >= 0 else -1
  - total_minutes = hours_in_minutes + sign * abs(minutes_offset)
  - offset = FieldUtils.safe_multiply(total_minutes, DateTimeConstants.MILLIS_PER_MINUTE)

2) Or compute total_minutes explicitly using hours and minutes with consistent sign semantics:
- Replace the block with:
  - total_minutes = hours_offset * 60
  - if hours_offset >= 0:
  -     total_minutes += minutes_offset
  - else:
  -     total_minutes -= abs(minutes_offset)
  - offset = FieldUtils.safe_multiply(total_minutes, DateTimeConstants.MILLIS_PER_MINUTE)

Why this fix fixes the root cause
- Both fixes treat the minutes component consistently: when hours are negative the minutes should push the total further negative, not cancel or reduce it. Using sign * abs(minutes) or explicit branch ensures you don’t subtract a negative minutes value and thus avoids the incorrect addition seen in the current code.
