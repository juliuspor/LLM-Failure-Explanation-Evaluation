What happened (root cause)
- The code can call substring_java(text, 0, upper) with upper > len(text). substring_java raises an IndexError when end > len(s) (see SLICE_UNION lines 32–34).
- That happens because you set upper = lower when upper < lower, but you never re-check that upper is within the string length afterward. If lower > len(text), upper becomes > len(text) and the subsequent substring_java call fails.

Where it occurs in the posted code
- At this line: index = StringUtils.index_of(text, " ", lower)
  - If lower > len(text), index_of probably returns -1 (no space found after that position).
- Then this block runs:
    result = substring_java(text, 0, upper)
  - Because earlier you did:
    if upper < lower:
        upper = lower
  - If lower > len(text) that makes upper > len(text), so substring_java receives end > len(s) and triggers the IndexError raised in SLICE_UNION (lines 32–34).

Why that exact sequence produces the error
1. Input: lower > len(text) (for example text length 10, lower = 20).
2. First adjustments: if upper == -1 or upper > len(text): upper = len(text) — (this only runs if upper was > len(text) or -1).
3. Next: if upper < lower: upper = lower — this sets upper = lower (20), even though 20 > len(text).
4. Later you call substring_java(text, 0, upper) with end == 20 -> substring_java checks end > len(s) and raises IndexError (String index out of range).

Concrete places to change (step-by-step fixes)
Option A — clamp lower before use (recommended):
- After you validate text and length, add:
    if lower > len(text):
        lower = len(text)
  Then keep your existing logic. This prevents lower (and thus upper when you assign upper = lower) from exceeding text length.

Option B — re-clamp upper after you adjust it:
- Replace the block
    if upper < lower:
        upper = lower
  with:
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)
  This ensures substring_java is always called with end <= len(text).

Option C — check before calling substring_java:
- Instead of relying on upstream variables, ensure every call does safe bounds. For example:
    safe_end = min(upper, len(text))
    result = substring_java(text, 0, safe_end)

Which to use
- Option A is simplest and aligns with the existing intent (lower should not be beyond the end).
- Option B is minimal change if you only want to ensure upper is valid after the lower adjustment.
- Option C is defensive and guarantees safety at each substring call.

One-line patch example (apply right after you set upper = lower):
    if upper < lower:
        upper = lower
    if upper > len(text):
        upper = len(text)

This directly prevents the IndexError coming from substring_java.
