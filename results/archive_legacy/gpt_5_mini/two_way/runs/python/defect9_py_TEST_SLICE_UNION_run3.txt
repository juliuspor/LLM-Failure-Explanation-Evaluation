What happens
- The test passes replacement_list entries that are None (e.g. ["c", None]).
- In replace_each there is a precomputation loop at lines 81–85 that calls len(replacement_list[i]) (and len(search_list[i])) unconditionally:
  - 81–85:
    greater = len(replacement_list[i]) - len(search_list[i])
    if greater > 0:
        increase += 3 * greater
- If replacement_list[i] is None this raises a TypeError: object of type 'NoneType' has no len(), so the whole function crashes before it gets to the later logic that treats None as “no replacement”.

Why this is the root cause
- The later matching loop (lines 102–111) explicitly checks for None values and for empty search strings:
  - 102–107:
    no_more_matches_for_repl_index[i]
    or search_list[i] is None
    or len(search_list[i]) == 0
    or replacement_list[i] is None
  That shows the function intends to treat None entries as “ignore this pair”.
- But the earlier size-estimate loop (lines 81–85) runs before those None-checks, and it assumes replacement_list[i] and search_list[i] are strings and uses len() on them. This disconnect is why None slips through and causes an immediate exception.

Where it occurs
- The error occurs in the precomputation block around lines 81–85 (len(replacement_list[i]) is called on a None). The later logic that would skip None entries is at lines 102–107 but runs too late to prevent the TypeError.

Concrete fixes (step-by-step)
1. Guard the precomputation loop against None and empty search entries before calling len():
   - Replace the body of the loop at 81–85 with something like:
     if search_list[i] is None or replacement_list[i] is None or len(search_list[i]) == 0:
         continue
     greater = len(replacement_list[i]) - len(search_list[i])
     if greater > 0:
         increase += 3 * greater
2. Also ensure you only iterate to the minimum length of both lists (or validate lists are the same length) to avoid index errors:
   - Replace search_length = len(search_list) (line 52) with:
     search_length = min(len(search_list), len(replacement_list))
   or raise a clear ValueError if unequal lists are not allowed.
3. Run the existing tests to confirm:
   - After the guard, tests with replacement_list containing None will no longer call len(None) and will follow the intended “ignore this replacement” path in the later matching loop.

Summary
The function crashes because it calls len() on replacement_list[i] before checking whether replacement_list[i] is None. Move or duplicate the None/empty checks so they run before any len() calls, and consider using the min length of both lists (or validating list lengths) to avoid index and type errors.
