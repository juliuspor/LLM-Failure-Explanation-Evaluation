Most likely root cause (very common, and fits the docstring behavior)
- Using list.append and then returning its result. list.append mutates the list and returns None. If your code does something like:
  new_list = array.append(element)
  return new_list
  that will return None instead of a list. This produces immediate failures later (TypeError or AttributeError) when callers expect a list.

Why this fails (precise):
- list.append modifies array in place and returns None. The variable new_list ends up set to None. Any following code that treats new_list as a list (indexing, iterating, calling len()) will error.
- If you intended to return a new list (docstring says “Copies the given list and adds the given element at the end of the new list”), calling array.append modifies the original array in place and does not produce a copy.
