What fails
- The method is declared with cls as the first parameter (def to_locale(cls, locale_str: str) ...) but there is no @classmethod decorator. Because of that, Python treats it as a normal instance method:
  - Calling LocaleUtils.to_locale("en") passes the string "en" into the cls parameter, leaving locale_str missing and causes a TypeError: to_locale() missing 1 required positional argument: 'locale_str'.
  - Calling instance.to_locale("en") passes the instance into cls and "en" into locale_str, which will run but is not the intended class-level API.

Why it fails (root cause)
- In Python, methods defined inside a class without a decorator expect to be instance methods; their first parameter should be self. You used cls as if this were a classmethod, but you never added @classmethod, so Python does not bind the class automatically. That mismatch between the signature (cls) and the actual binding is the direct cause of the runtime error or confused calls.

Where it happens in your code
- The offending line is the function header:
      def to_locale(cls, locale_str: str) -> Optional[Locale):
  (the missing @classmethod decorator should be placed just above this line inside the class definition).

How to fix (concrete steps)
Option A — make it a true class method (recommended if you want to call it on the class):
1. Add the @classmethod decorator immediately above the def line:
      @classmethod
      def to_locale(cls, locale_str: str) -> Optional[Locale]:
2. Call it as LocaleUtils.to_locale("en") (or MyClass.to_locale(...)).

Option B — make it a static/utility function (if it does not use cls):
1. Remove the cls parameter and adjust the signature:
      def to_locale(locale_str: str) -> Optional[Locale]:
2. Optionally add @staticmethod if you keep it inside the class:
      @staticmethod
      def to_locale(locale_str: str) -> Optional[Locale]:
3. Call it as LocaleUtils.to_locale("en") (staticmethod is bound without passing an instance).

Option C — use an instance method (unlikely here):
1. Rename cls to self and call via an instance:
      def to_locale(self, locale_str: str) -> Optional[Locale]:
2. Call it via instance.to_locale("en").

Which to choose
- Use @classmethod (Option A) if the method is intended as a class-level utility (you named the first parameter cls, which implies this intent).
- Use @staticmethod or remove cls if the method does not use any class or instance state.

Example fix (recommended):
    class LocaleUtils:
        @classmethod
        def to_locale(cls, locale_str: str) -> Optional[Locale]:
            ...

This will stop the TypeError and let the rest of your validation code run as written.
