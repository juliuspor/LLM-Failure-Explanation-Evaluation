What went wrong (root cause)
- On line 336 you set inferred_type = type(array). That takes the Python type of the list object itself (e.g. <class 'list'>), not the type of the elements stored in the list.
- ArrayUtils._copy_list_grow1 and the later check expect inferred_type to represent the element/component type, not the container type. Because inferred_type becomes list instead of the element class (or object), the logic that detects "object list vs typed list" and the name used in the error message are wrong.
- Concretely, when array is a list (even a list of plain objects), type(array) == list, so:
  - inferred_type == object is never true, so you won't raise when you should, or you might raise with the wrong class names.
  - _copy_list_grow1 receives the wrong type information and produces a list with the wrong assumed component type, which leads to the casting/class-name mismatch and the TypeError you see at runtime.
- The bug occurs at line 336 (inferred_type = type(array)) and then manifests at lines 346–347 when the code performs the type check / raises the error.

- The code is trying to reason about the element type, but uses the container's type.
