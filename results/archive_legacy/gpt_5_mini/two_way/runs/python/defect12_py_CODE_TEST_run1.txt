What happens (root cause)
- In your test test_abbreviate_lower_beyond_length you call WordUtils.abbreviate("0123456789", 15, 20, None).
- At runtime len(text) == 10. The code first runs:
  - if upper == -1 or upper > len(text): upper = len(text)  -> upper becomes 10.
  - if upper < lower: upper = lower  -> upper (10) < lower (15) so upper is set to 15.
- Now upper is 15 which is greater than len(text). Later the code does:
  - index = StringUtils.index_of(text, " ", lower)  (start search at lower==15)
  - because lower > len(text) index_of returns -1, so the code takes the branch that builds result = substring_java(text, 0, upper)
- substring_java(text, 0, upper) is then called with end index 15 while text length is 10. That leads to an out-of-range substring call (or otherwise incorrect slicing), causing the test to fail.

Where it occurs in the code
- The problematic sequence is the pair of lines that clamp upper then force it up to lower:
  - if upper == -1 or upper > len(text): upper = len(text)
  - if upper < lower: upper = lower
- For the input lower > len(text), those two lines make upper exceed the text length, producing an invalid end index passed to substring_java.

Why that logic is wrong
- The intent of clamping upper to len(text) is correct, but then forcing upper up to lower ignores the fact that lower may itself be out of range. You must not produce an end index greater than the string length. Java's original behavior returns the original string when lower is beyond the text length; this code instead tries to substring past the end.

How to fix it (clear steps)
1. If lower is greater than or equal to len(text), return text unchanged. That matches the expected behavior in the test.
   - Example change (insert early, after handling empty text):
     if lower >= len(text):
         return text
2. Alternatively, clamp lower before comparing to upper so upper never gets set > len(text):
   - lower = min(lower, len(text))
   - if upper == -1 or upper > len(text): upper = len(text)
   - if upper < lower: upper = lower
3. Make sure substring_java is only ever called with end <= len(text).

Minimal patch example (preferred):
    if text is None:
        return None
    if len(text) == 0:
        return StringUtils.EMPTY

    # fix: if lower beyond length, return original text
    if lower >= len(text):
        return text

    if upper == -1 or upper > len(text):
        upper = len(text)
    if upper < lower:
        upper = lower

This ensures substring_java(text, 0, upper) never receives an end index > len(text), and satisfies the test case.
