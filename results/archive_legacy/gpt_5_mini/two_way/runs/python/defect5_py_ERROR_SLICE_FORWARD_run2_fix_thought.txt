The bug stems from using type(array) to infer element type which gives list type, not element component type, leading to incorrect expected_type checks and potential incorrect behavior when adding None. I will infer element type from existing array elements if present; if array is None, infer from element. I will enforce expected_type by validating all non-None elements in the resulting list are instances of expected_type, and if all elements are None while expected_type is not object, raise TypeError to mimic Java ClassCastException behavior.
