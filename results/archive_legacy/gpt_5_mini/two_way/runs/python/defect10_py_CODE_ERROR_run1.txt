What happened (exact place and exception)
- The IndexError occurs inside the branch for trailing 'l' or 'L':
  if last_char in ("l", "L"):
      if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
          return int(numeric)
  Here numeric is val[:-1].
- When val has length 1 (for example val = "L" or val = "l"), numeric becomes the empty string "".
- The code then evaluates char_at(numeric, 0) (or numeric[0] if char_at is implemented that way). Accessing index 0 of an empty string raises IndexError: String index out of range: 0.

Why it fails (root cause, in plain terms)
- There is no guard that numeric is non-empty before reading its first character.
- The compound boolean expression evaluates the left side (checking numeric[0]) even when numeric == "", so the code attempts numeric[0] and crashes.
- The comment already points to the missing guard: "# BUG: missing guard for len(val) == 1 and non-digit input."

Exactly where to change the code (line/condition)
- In the if last_char in ("l", "L") branch, before accessing numeric[0] or numeric[1:], ensure numeric is not empty.
- Either:
  1) Test numeric.isdigit() first, then check the negative-number form only if numeric is long enough; or
  2) Explicitly check len(numeric) > 0 (or len(numeric) > 1 for the negative-case) before indexing.

Concrete fixes (step-by-step)
Option A — safer boolean order (preferred):
Replace the branch with:
    if last_char in ("l", "L"):
        # First check the simple all-digit case (numeric could be empty, .isdigit() is safe)
        if numeric.isdigit():
            return int(numeric)
        # Then check the negative form, but only if numeric has at least 2 chars ("-<digits>")
        if len(numeric) >= 2 and numeric[0] == "-" and numeric[1:].isdigit():
            return int(numeric)
        raise ValueError(f"{val} is not a valid number.")

Option B — explicit empty check:
    if last_char in ("l", "L"):
        if not numeric:
            raise ValueError(f"{val} is not a valid number.")
        if (numeric[0] == "-" and numeric[1:].isdigit()) or numeric.isdigit():
            return int(numeric)
        raise ValueError(f"{val} is not a valid number.")

Notes
- numeric.isdigit() is safe on the empty string (it returns False), so checking it first avoids indexing an empty string.
- Make sure you do not evaluate numeric[0] unless len(numeric) >= 1, and do not evaluate numeric[1:] checks unless len(numeric) >= 2.

This will prevent the IndexError when val is a single-character non-digit like "L" or "l", and will produce the intended ValueError instead.
