What happened (where it failed)
- The failure occurs in the "guess buffer size" loop at lines 81–84 (the SLICE_BLOCK you showed).
- On line 82 the code calls len(replacement_list[i]) and len(search_list[i]) without checking whether replacement_list[i] or search_list[i] is None.
- If either array contains a None element, len(None) raises TypeError: object of type 'NoneType' has no len(), so the function crashes there.

Why it failed (root cause)
- The function accepts Optional[List[Optional[str]]] for search_list and replacement_list, and elsewhere in the code it explicitly treats individual elements as possibly None (see the checks in the matching loops that skip entries when search_list[i] is None or replacement_list[i] is None).
- But the buffer-size estimation loop does not perform those per-element None checks; it assumes every element is a str and calls len(...) unconditionally. That mismatch causes a TypeError if any element is None.

Exact location and variables
- Failure happens in replace_each, in this loop:
    for i in range(search_length):
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater
- Variables: replacement_list[i], search_list[i], and the built-in len. The problematic call is len(replacement_list[i]) or len(search_list[i]) when either is None.

How to fix it (concrete steps)
1. Skip elements that are None or empty, the same way the other loops do. Add the same conditions used later:
   - if no_more_matches_for_repl_index[i] or search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None: continue
2. Then compute greater only for valid strings.

Minimal corrected code for that loop:
    for i in range(search_length):
        if (
            no_more_matches_for_repl_index[i]
            or search_list[i] is None
            or len(search_list[i]) == 0
            or replacement_list[i] is None
        ):
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater  # assume 3 matches

Why this fix works
- It prevents calling len(...) on None, matching the element‑validity logic used elsewhere in replace_each. It preserves the original intent (estimate extra buffer size only for valid, non-empty search strings and present replacements).

Alternative (if you prefer)
- If you want a more defensive one-liner:
    repl = replacement_list[i] or ""
    search = search_list[i] or ""
    greater = len(repl) - len(search)
  But be aware this treats None the same as "" (which may or may not be desired). The explicit skip is closer to the existing function logic.
