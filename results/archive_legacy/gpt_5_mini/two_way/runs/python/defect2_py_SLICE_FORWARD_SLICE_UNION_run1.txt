What happened (where it fails)
- The code crashes at line 71, inside get_paint:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If self._upper_bound equals self._lower_bound, the expression divides by zero and Python raises a ZeroDivisionError. That is the immediate root cause.

Why it fails (root cause)
- __init__ (lines 30, 43–44) allows creating an object where lower_bound and upper_bound are equal (or upper_bound < lower_bound) because there is no validation.
- get_paint assumes (self._upper_bound - self._lower_bound) is non-zero. When it is zero, the division fails.
- Separately: if the division works but the computed g falls outside 0..255, the code currently raises a ValueError at lines 74–75. The raised message is unhelpful because the f-string is literal "Color parameter outside of expected range: Red Green Blue" and does not include the offending value(s), so debugging is harder.
