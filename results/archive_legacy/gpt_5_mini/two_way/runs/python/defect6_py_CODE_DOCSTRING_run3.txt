What goes wrong (where it happens)
- The failure happens in the while loop that starts at the line
  if abs(x) >= 100:
      while mantissa % 10 == 0:
  (the loop body is the block that computes check_val and possibly does mantissa //= 10).
- If mantissa ever becomes 0 the condition mantissa % 10 == 0 stays true forever and mantissa //= 10 leaves mantissa at 0, so the loop never breaks → infinite loop / hang.

How that can occur (root cause)
- You initialize mantissa = value where value = int(x).
- For most positive or negative integer values that are powers of 10 (1000, 10000, ...) the loop reduces mantissa down to 1 and stops. But if value == 0 (or you ever reduce to 0 for some reason) the loop condition mantissa % 10 == 0 is true and mantissa //= 10 remains 0 on every iteration.
- Because the loop body only ever reduces mantissa (mantissa //= 10) and increments exp, and because check_val is (mantissa // 10) * (10 ** (exp + 1)) which will equal value==0 for every iteration once mantissa is 0, the inner if check (check_val == value) never fails to keep the loop going. That is the infinite-loop case.

Exactly where to change the code (concrete fixes, step-by-step)
1) Guard the loop against a zero mantissa. Replace the while condition with a test that ensures mantissa is nonzero:
   - Change
       while mantissa % 10 == 0:
     to
       while mantissa != 0 and mantissa % 10 == 0:
   This prevents the infinite loop if mantissa reaches 0.

2) (Optional, safer) You can simplify and make the intention clearer by avoiding the reconstruct-and-compare trick. If you want to strip trailing zeros from value and count them:
   - Use:
       while mantissa != 0 and mantissa % 10 == 0:
           mantissa //= 10
           exp += 1
     This removes the need for check_val entirely and is easier to reason about.

3) If you intended to skip the trailing-zero reduction for small numbers anyway, keep the existing abs(x) >= 100 check; the mantissa != 0 guard is still necessary for safety.

Why this fix works
- Adding mantissa != 0 stops the loop when mantissa is zero, so the loop cannot loop forever.
- The simplified loop (option 2) directly performs the intended operation — dividing away trailing zeros and counting them — without reconstructing value and comparing, which removes the subtlety that allowed a constant-true condition when mantissa == 0.
