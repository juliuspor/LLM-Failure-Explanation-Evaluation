What happened
- Root cause: char_at(numeric, 0) is called when numeric == "" (empty string). That raises IndexError ("String index out of range: 0") from the char_at implementation (see slice_backward lines 30–31).
- Where it happens: inside the block guarded by if last_char in ("l", "L"): — specifically the line
    if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
  This line assumes numeric has at least one character but the code never checks that.
- When it triggers: when val is a single-character string whose last (and only) character is 'l' or 'L' (for example val == "l"). Then numeric = val[:-1] == "" and char_at(numeric, 0) throws IndexError.

Why the check that looks for a leading '-' fails
- The code tries to detect a signed integer by calling char_at(numeric, 0) == "-" and numeric[1:].isdigit(), but that uses an index into numeric without first confirming numeric has length >= 1. For val == "l" the numeric part is empty, so numeric[1:] and numeric[0] are invalid operations.

Concrete repro
- Call create_number("l") or create_number("L") — result: IndexError from char_at, not a ValueError saying "l is not a valid number."

Fixes (clear, step-by-step)
Option A — simplest explicit guard
1. In the last_char in ("l","L") branch, check if numeric is empty before using char_at:
   if len(numeric) == 0:
       raise ValueError(f"{val} is not a valid number.")
   if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
       return int(numeric)
   raise ValueError(f"{val} is not a valid number.")
This prevents the IndexError and returns the intended ValueError.

Option B — avoid char_at and use Python string methods
1. Replace the condition with safe string checks that do not index into numeric:
   if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():
       return int(numeric)
   raise ValueError(f"{val} is not a valid number.")
This both fixes the bug and is clearer / safer.

Why these fixes work
- Both ensure you never attempt to read numeric[0] when numeric is empty. Option B uses startswith and slicing which tolerate empty strings, so it can’t cause the IndexError.

Pick Option B unless char_at is required elsewhere — it is simpler and eliminates the out-of-range access.
