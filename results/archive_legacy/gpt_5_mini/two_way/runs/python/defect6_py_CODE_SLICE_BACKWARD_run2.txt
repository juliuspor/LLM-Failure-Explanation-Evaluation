What went wrong
- The code compares x == int(x) before checking for NaN or infinity (line shown in the slice: "if x == int(x) and not math.isinf(x) and not math.isnan(x):").
- Calling int(x) when x is NaN or infinite raises an exception (ValueError for NaN, OverflowError for ±inf / extremely large floats).
- That exception happens at the int(x) call (so the later math.isinf / math.isnan checks never run), causing the function to fail.

Where it happens
- At line 176 (the shown if statement), specifically when executing int(x).

Why it happens (root cause)
- The boolean expression evaluates left-to-right. int(x) is evaluated before math.isinf(x) and math.isnan(x). Converting NaN or infinity to int is invalid and raises, so the program crashes before the NaN/inf guards can protect it.

Concrete fixes (step-by-step)
1. Reorder the checks so you guard against NaN/Infinity first:
   - Replace the if with:
     if not math.isinf(x) and not math.isnan(x) and x == int(x):
   - This ensures math.isinf/math.isnan are evaluated first and avoid calling int(x) on invalid values.

2. Alternatively (recommended, clearer and more robust), use the float method is_integer(), which does not attempt a conversion to int:
   - Replace the condition with:
     if not math.isinf(x) and not math.isnan(x) and x.is_integer():
   - Or simply:
     if x.is_integer() and not math.isinf(x) and not math.isnan(x):

3. If you prefer to avoid multiple checks, you can use try/except around int(x) to catch ValueError/OverflowError, but reordering or using x.is_integer() is cleaner.

  To:
