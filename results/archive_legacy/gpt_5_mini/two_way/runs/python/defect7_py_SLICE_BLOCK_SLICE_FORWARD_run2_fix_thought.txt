The bug arises from calling .upper() on non-string objects. The to_class function intends to convert array of objects to their class types. The .upper() call is unnecessary; probably originally intended to ensure string elements non-empty. Fix by removing the upper() call and handle None elements by raising AttributeError per docstring? The docstring says Raises: AttributeError: If any element is None. So if element is None, should raise AttributeError. Therefore implement: if array is None: return None; if empty: return copy; iterate: if array[i] is None: raise AttributeError; classes.append(type(array[i])); Return classes.
