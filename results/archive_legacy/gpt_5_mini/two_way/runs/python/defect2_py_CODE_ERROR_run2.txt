What happened (where it failed)
- The exception is raised on this line:
  if g < 0 or g > 255:
      raise ValueError(...)
- That check is failing because g was computed out of range.

Why it happened (root cause)
- You intended to clamp the input first to v, but then you never use v when computing g. Instead you compute g from the original value:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If value is outside the bounds, using value (not v) makes the normalized result < 0 or > 255, so the range check fails and the ValueError is raised.
- A second possible cause: if self._upper_bound == self._lower_bound you will divide by zero when computing g. That will raise a different error (ZeroDivisionError), but it’s worth handling.

Exactly where to change it (step-by-step fix)
1. Use the clamped variable v in the normalized calculation instead of value:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
2. Guard against zero range (upper == lower). Decide desired behavior (e.g., return 0, 255, mid-gray, or raise a clear error). For example, return 0 (black) if bounds are equal.
3. Improve the ValueError message to include the actual g and the parameters so debugging is easier.

Corrected function (exact replacement)
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)
        
        # Handle degenerate range
        if self._upper_bound == self._lower_bound:
            # choose behavior; here we return black
            g = 0
        else:
            # Use clamped v (not original value)
            g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
        
        # Validate the range
        if g < 0 or g > 255:
            raise ValueError(f"Color parameter outside of expected range: g={g} "
                             f"(lower={self._lower_bound}, upper={self._upper_bound}, value={value}, clamped={v})")
        
        return (g, g, g)

How to reproduce quickly
- Set self._lower_bound = 0, self._upper_bound = 1 and call get_paint(2.0). With the original code, g is computed from value=2.0 → out of range → ValueError.
- With the corrected code, value=2.0 is clamped to v=1.0 and returns (255,255,255).

That is the direct root cause: you clamped to v but then used value in the calculation. Fixing the one line to use v (and handling equal bounds) resolves the failure.
