What happened (root cause)
- At line 177 you do: value = int(x).
- When x is -0.0, int(x) returns 0 and loses the negative sign. That 0 is assigned to value and copied to mantissa.
- Later the code takes the integer branch and calls self.add(str(value)) (line 192), which produces "0".
- The test expected to see "-0.0" but the code emitted "0" because the negative zero sign was discarded by int(x).

Where it occurs
- The sign is lost at line 177 (value = int(x)).
- The lost-sign value is used later when you call self.add(str(value)) at line 192 (and possibly elsewhere in the integer-handling branch between 180–192).
- The final self.add(str(x)) at line 194 would have kept "-0.0", but the integer branch runs first and publishes the unsigned "0".

Why that matters
- Floating-point -0.0 compares equal to 0.0, so int(-0.0) -> 0. There is no arithmetic difference, but string/format output and some specs distinguish "-0.0". Your tests expect that preserved sign.

Concrete fixes (step-by-step)
1. Detect and preserve negative zero before converting to int:
   - Add at or just before line 177:
     import math
     if x == 0.0 and math.copysign(1.0, x) < 0:
         # preserve negative zero, avoid int conversion that loses sign
         self.add(str(x))   # or self.add("-0.0") if fixed formatting is required
         return

   This ensures -0.0 is emitted immediately and the integer branch is skipped.

2. Alternatively, avoid using int(x) to decide what to print. Use is_integer() to test integrality while keeping sign:
   - Replace value = int(x) with:
     if x.is_integer():
         value = int(x)   # only used for numeric checks; but preserve sign when x == 0.0 as above
     else:
         # non-integer path

   And combine with the negative-zero check above.

3. If you need the integer branch but want to preserve sign for zero, convert back to a signed string using copysign:
   - After computing value, before self.add(str(value)), do:
     if value == 0 and x == 0.0 and math.copysign(1.0, x) < 0:
         self.add("-0")
     else:
         self.add(str(value))

Pick the approach that fits your formatting rules (do you want "-0" or "-0.0" or other). The simplest fix for this failing test: detect negative zero with math.copysign and emit str(x) (which preserves "-0.0") instead of int(x)-derived output.
