What happened (where and why)
- The failure occurs at the line
    _ = array[i].upper()
  inside to_class (the for loop).
- When array contains None (test_to_class_with_none_element) that line tries to call .upper() on None and raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The same line also causes errors for any non-string element (int, float, list, etc.) because most objects don’t have an .upper() method.
- That line is unnecessary: the method’s purpose is to return the element classes (classes.append(type(array[i]))), so touching .upper() serves no purpose and introduces the bug.

Exact location
- In the provided snippet: inside def to_class(...), inside the for i in range(len(array)) loop, the problem is the line that calls array[i].upper().

How to fix it (explicit steps)
1. Remove the offending line:
   - Delete: _ = array[i].upper()
2. Use the element directly and append its type (this handles None because type(None) is the NoneType your tests expect):
   - Replace the loop with:
       for elem in array:
           classes.append(type(elem))
3. (Optional but recommended) Make the method signature and decoration match how you call it:
   - Tests call ClassUtils.to_class(test_array). If this method is meant to be callable on the class, declare it as a @staticmethod:
       @staticmethod
       def to_class(array: Optional[List[Any]]) -> Optional[List[Type]]:
           ...
   - Or if you intend a classmethod, keep cls and use @classmethod and call as ClassUtils.to_class(test_array) will then pass the class automatically; but a staticmethod is simplest here because cls is unused.

Minimal corrected implementation
- Inside the class, use:

    @staticmethod
    def to_class(array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for elem in array:
            classes.append(type(elem))
        return classes

Why this fixes the tests
- Removing .upper() avoids AttributeError on None or non-string elements.
- Using type(elem) returns type(None) for None and the correct type for other elements, matching the expected lists in the tests.

Notes
- The empty-array branch returning ArrayUtils.EMPTY_CLASS_ARRAY.copy() is fine if that constant is an empty list; otherwise ensure it returns an empty list as tests expect [].
- Keep the parameter list and decorator consistent with how the method is called (classmethod vs staticmethod).
