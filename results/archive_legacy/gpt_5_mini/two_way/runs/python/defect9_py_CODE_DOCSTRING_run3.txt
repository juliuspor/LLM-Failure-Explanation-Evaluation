What happened
- The code crashes when search_list or replacement_list contains None because of this block (the "BUG" comment):

  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
      if greater > 0:
          increase += 3 * greater  # assume 3 matches

- In that loop the code calls len(replacement_list[i]) and len(search_list[i]) without checking for None. If either element is None, Python raises:
  TypeError: object of type 'NoneType' has no len()

Why it happens (root cause)
- Earlier in replace_each the code correctly guards against None values when searching for matches (both the initial match-finding loop and the per-iteration match-finding loop check "search_list[i] is None" and "replacement_list[i] is None" and skip those entries).
- The size-estimation loop (above) is the only place that inspects elements with len(...) without those same None checks. So when a list element is None, that len(...) call throws the TypeError and the function crashes before doing any replacements.

Where it occurs (exact variables/locations)
- The failing expressions are len(replacement_list[i]) and len(search_list[i]) inside the "get a good guess on the size of the result buffer" loop (the loop right after the comment "get a good guess on the size of the result buffer so it doesnt have to double if it goes over a bit").
- The variables involved are replacement_list[i], search_list[i], and the local variable greater which is computed from their lengths.

How to fix it (clear steps)
1. Make the length-estimation loop null-safe by skipping elements that are None (same checks used elsewhere). Replace that loop with, for example:

   for i in range(search_length):
       if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
           continue
       greater = len(replacement_list[i]) - len(search_list[i])
       if greater > 0:
           increase += 3 * greater

   This mirrors the checks used when finding matches and prevents calling len(None).

2. Alternatively, treat None as length 0 (less recommended because semantics differ). Example:

   def safe_len(s):
       return 0 if s is None else len(s)
   ...
   greater = safe_len(replacement_list[i]) - safe_len(search_list[i])

3. Add unit tests that include None in search_list and replacement_list (and empty strings) to ensure no exceptions and that ignored entries are indeed skipped.

Why these fixes are appropriate
- They align the length-estimation logic with the existing match-skipping rules, avoiding the TypeError while preserving the intended behavior: None or empty search strings and None replacements are ignored.
