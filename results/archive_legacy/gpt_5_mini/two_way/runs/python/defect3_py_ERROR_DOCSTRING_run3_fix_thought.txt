The bug arises when a translator returns a consumed count that causes translate to call code_point_at when pos is already at or past length, raising IndexError. To make translate robust, validate out and input_seq, convert to UTF-16 code units, and when a translator returns consumed>0, advance pos by char_count of each consumed codepoint but guard against reading past the end (check pos < length before calling code_point_at). This prevents IndexError even if a translator returns an incorrect consumed value. Return behavior is preserved.
