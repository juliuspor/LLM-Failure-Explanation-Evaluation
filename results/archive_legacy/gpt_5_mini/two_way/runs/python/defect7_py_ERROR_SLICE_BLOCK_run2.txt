What happened (where it failed)
- The failure happened on line 766 in the slice you posted:
  765 |         for i in range(len(array)):
  766 |             _ = array[i].upper()
  767 |             classes.append(type(array[i]))
- The AttributeError "'NoneType' object has no attribute 'upper'" means array[i] was None for at least one index i. Python tried to call .upper() on None, but None has no .upper() method.

Why it happened (root cause)
- Your code assumes every element of array is a string (or at least an object with an upper() method).
- At runtime at least one element is None, so array[i].upper() fails.
- Because the exception occurs on line 766, line 767 (classes.append(...)) is never reached for that iteration.

How to confirm where the None is
- Add a quick diagnostic before the call to see the index and value that caused the error:
  for i in range(len(array)):
      if array[i] is None:
          raise ValueError(f"array[{i}] is None")
      _ = array[i].upper()
      classes.append(type(array[i]))
- Or print array[i] with the index before calling .upper() to inspect the bad element.

Clear, actionable fixes (choose one depending on intent)
1) If None is unexpected (it indicates a bug earlier):
   - Fail early with an informative error:
     for i in range(len(array)):
         if array[i] is None:
             raise TypeError(f"Unexpected None at array[{i}]")
         _ = array[i].upper()
         classes.append(type(array[i]))
   - Then fix the code that builds array so elements are not None.

2) If None is allowed and should be skipped:
   - Skip None elements before calling .upper():
     for i in range(len(array)):
         if array[i] is None:
             classes.append(type(array[i]))   # optional: record type(None)
             continue
         _ = array[i].upper()
         classes.append(type(array[i]))

3) If None should be treated as an empty string or default value:
   - Convert None to a string before calling .upper():
     for i in range(len(array)):
         val = array[i] if array[i] is not None else ""
         _ = val.upper()
         classes.append(type(val))

4) If you only want to call .upper() on strings:
   - Check type before calling:
     for i in range(len(array)):
         val = array[i]
         if isinstance(val, str):
             _ = val.upper()
         else:
             # handle non-string values appropriately
         classes.append(type(val))

Notes
- Always check for None (or unexpected types) before calling methods you assume exist.
- Prefer using enumerate(array) instead of range(len(array)) if you want cleaner code and direct access to both index and value. Example:
    for i, val in enumerate(array):
        if val is None:
            raise TypeError(f"Unexpected None at array[{i}]")
        _ = val.upper()
        classes.append(type(val))

This pinpoints the exact cause (array[i] == None at line 766) and gives concrete, testable changes you can apply depending on how you want None handled.
