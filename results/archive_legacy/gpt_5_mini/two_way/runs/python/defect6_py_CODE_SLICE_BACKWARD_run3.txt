What happens
- The failure occurs at the line shown (your slice):  
  176 |         if x == int(x) and not math.isinf(x) and not math.isnan(x):

- The expression is evaluated left-to-right. Python tries to evaluate x == int(x) before it evaluates not math.isinf(x) and not math.isnan(x).

- If x is NaN or +/-Inf, calling int(x) raises an exception (ValueError for NaN, OverflowError for Inf). That exception happens before the checks that would have excluded those values, so the function crashes.

Why that is the root cause
- You intended to avoid converting NaN or Inf to int by checking math.isinf(x) and math.isnan(x), but because x == int(x) is evaluated first the conversion happens anyway.
- The variables and methods involved: x, int(x), math.isinf(x), math.isnan(x). The exception occurs inside int(x) when x is not a finite real number.

How to fix it (concrete steps)
1. Check finiteness before calling int(x). Replace the if condition with a test that calls math.isfinite (or tests isinf/isnan) first. For example:
   - if math.isfinite(x) and x == int(x):
     (math.isfinite is a single call equivalent to not isinf and not isnan)
2. Or reorder the existing checks so they run before int(x):
   - if not math.isinf(x) and not math.isnan(x) and x == int(x):
     (put the safe checks first so short-circuiting prevents int(x) from being called for NaN/Inf)
3. Alternatively, wrap the int conversion in try/except if you expect other problematic values:
   - try:
         is_integer = (x == int(x))
     except (ValueError, OverflowError):
         is_integer = False
     if is_integer:
         ...

Which fix I recommend
- Use math.isfinite and then compare to int: if math.isfinite(x) and x == int(x):. This is clear, concise, and avoids exceptions.

Where to change
- Modify the condition at line 176 (the if on x == int(x) ...) so that the finiteness check happens before calling int(x), as shown above.
