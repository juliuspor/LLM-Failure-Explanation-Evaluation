What happens
- When x is an exact integer and abs(x) >= 100 you enter the loop at line 181:
  while mantissa % 10 == 0:
- If value (and therefore mantissa) is 0, mantissa % 10 == 0 is true, and mantissa //= 10 leaves mantissa at 0. The loop condition stays true forever and exp keeps increasing. That is an infinite loop (hangs your program). The loop body that computes check_val at line 182 also produces 0, so the equality test at line 183 keeps succeeding and nothing ever breaks you out.

Where it occurs (exact lines)
- value = int(x) (line 177)
- mantissa = value (line 178)
- if abs(x) >= 100: (line 180)
- while mantissa % 10 == 0: (line 181)
- check_val = (mantissa // 10) * (10 ** (exp + 1)) (line 182)
- if check_val == value: (line 183)
  then mantissa //= 10 and exp += 1 keep mantissa == 0, causing the loop to never terminate.

Why it fails (root cause)
- You never handle the special case value == 0. For 0 the loop condition and the body never change mantissa to a value that breaks the loop, so the loop never exits.

Concrete fixes (exact steps and code)
Option A — simplest fix: skip the trailing-zero trimming for value == 0
1. Add a guard before the while loop:
    if mantissa == 0:
        exp = 0
    else:
        while mantissa % 10 == 0:
            ...
2. Example patch around lines 179–186:
    if abs(x) >= 100:
        if mantissa == 0:
            # don't try to compress 0 into mantissa/exponent
            exp = 0
        else:
            while mantissa % 10 == 0:
                check_val = (mantissa // 10) * (10 ** (exp + 1))
                if check_val == value:
                    mantissa //= 10
                    exp += 1
                else:
                    break

Option B — avoid zero-loop by requiring mantissa != 0 in loop condition
1. Change line 181 to:
    while mantissa != 0 and mantissa % 10 == 0:
2. That prevents the infinite loop when mantissa becomes 0.

Either fix prevents the infinite loop and preserves the intended behavior for nonzero integers with trailing zeros.

Notes
- This is not a floating-point precision bug or an import error — it's simply an infinite loop triggered by value == 0.
