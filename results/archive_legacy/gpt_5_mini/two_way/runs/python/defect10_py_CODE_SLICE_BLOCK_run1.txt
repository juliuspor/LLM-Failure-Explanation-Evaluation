What happened (what you saw)
- Calling create_number("l") (or any one-character non-digit that ends with 'l'/'L') causes an IndexError raised from char_at(...), not a ValueError. The stack trace points at the slice block lines 30–31:
  - line 30–31: if index < 0 or index >= len(s): raise IndexError(...)

Why it happens (root cause)
- In create_number, when last_char is not a digit you compute numeric = val[:-1].
  - If val is "l" then numeric == "" (empty string).
- The code then enters the last_char in ("l","L") branch and immediately calls char_at(numeric, 0).
  - char_at("", 0) checks index >= len(s) and raises IndexError because len(numeric) == 0.
- In other words: the code assumes numeric has at least one character before accessing numeric[0] / char_at(...), but that assumption is false for single-character inputs like "l". There is no guard for len(val) == 1 (comment even calls this a BUG).

Where it occurs (exact lines / variables)
- create_number: when last_char in ("l","L") branch.
  - numeric = val[:-1]
  - char_at(numeric, 0) is called while numeric may be empty.
- The exception is raised in the char_at function at the slice block lines 30–31 (IndexError: String index out of range).

How to fix (concrete steps)
1. Add an explicit check that numeric is non-empty before accessing numeric[0] or calling char_at:
   - e.g. if len(numeric) == 0: raise ValueError(f"{val} is not a valid number.")
2. Or change the condition to test numeric safely:
   - if numeric and ((char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit()):
       return int(numeric)
   - This ensures you never call char_at or numeric[1:] when numeric == "".
3. Alternatively move the existing len(val) == 0 check so it also covers this case, but the safest is to check len(numeric) before using numeric[0] or numeric[1:].

Minimal corrected snippet (replace the buggy block)
    if last_char in ("l", "L"):
        if len(numeric) == 0:
            raise ValueError(f"{val} is not a valid number.")
        if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
            return int(numeric)
        raise ValueError(f"{val} is not a valid number.")

That prevents char_at from being called on an empty string and makes single-character inputs like "l" produce the intended ValueError instead of an IndexError.
