What happened (where it fails)
- In ClassUtils.to_class, the loop at lines 765–767 unconditionally calls array[i].upper() (line 766).
- If array[i] is None, None has no attribute upper, so array[i].upper() raises an AttributeError (or in Java terms this mirrors a NullPointerException).
- The failure occurs inside the loop when processing the None element in your test array ["Test", None, 99.0].

Root cause (why it fails)
- The code assumes every element is a string or otherwise supports .upper(), but the method is supposed to work with arbitrary objects including None, numbers, booleans, lists, etc.
- Because there is no check for None (or for the element type) before calling .upper(), the call fails on None and aborts the method before appending the intended type information.

Exact place in code
- Lines referenced in your slice:
  - 765: for i in range(len(array)):
  - 766:     _ = array[i].upper()
  - 767:     classes.append(type(array[i]))

What to change (concrete fixes, step-by-step)
1. Guard against None before calling .upper(). You should not call .upper() unconditionally.
2. Use the actual element type for the result. For None, append type(None). For other elements, append type(element). If you need special handling for strings, check isinstance(element, str).

Minimal patch (replace lines 765–767):
- Current:
  for i in range(len(array)):
      _ = array[i].upper()
      classes.append(type(array[i]))

- Replace with:
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          classes.append(type(None))
      else:
          # If you intended to normalize string values, handle it explicitly:
          # if isinstance(elem, str):
          #     _ = elem.upper()
          classes.append(type(elem))

Why this fixes it
- The code no longer tries to call .upper() on None, so the loop completes and classes for all elements (including None) are produced.
- The result for None elements becomes type(None), which matches your tests' expected [str, type(None), float] etc.

Extra notes
- Make sure the method also returns None if the input array is None (your test test_to_class_null_input expects that).
- Ensure an empty input list returns an empty list (your test_to_class_empty_array).
- If the original purpose of calling .upper() was to detect string-like class names, that logic needs to be implemented only after checking the element is a str, not on every element.
