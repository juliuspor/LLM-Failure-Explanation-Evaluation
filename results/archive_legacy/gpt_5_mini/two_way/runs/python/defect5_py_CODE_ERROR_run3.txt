What went wrong (root cause)

- On these lines you set inferred_type from the container object itself:
  - if array is not None:
      inferred_type = type(array)
  type(array) returns the class of the list object (e.g. builtins.list), not the element type held inside the list. So inferred_type is the list class, not the element class you need.

- Later you call:
  new_list = ArrayUtils._copy_list_grow1(array, inferred_type)
  and then you perform the type check:
  if inferred_type == object and expected_type != object:
      raise TypeError(...)

  Because inferred_type was calculated incorrectly, the code flow and the value passed into _copy_list_grow1 are wrong. That makes ArrayUtils produce (or treat) the result as an object-typed list while expected_type is str, which triggers the TypeError you saw:
  "Cannot cast object list to str list..."

  - array is an empty list, so type(array) is list → inferred_type = list (wrong; you wanted the element type)
  - you never inspect the elements (there are none) and you don't use expected_type to infer element type
  - downstream logic misinterprets the element type and raises the TypeError about casting object[] to str[]

Why this is the root cause (no guessing)
- The purpose of inferred_type is to identify the element type for the new list. type(array) gives you the container type (list), not the element type.
- If the array is empty you must fall back to a different source of truth (expected_type or element), otherwise you cannot know the intended element type.
- Because the wrong type value is passed into _copy_list_grow1 and the later check, the code detects a mismatch and throws the TypeError you saw.
