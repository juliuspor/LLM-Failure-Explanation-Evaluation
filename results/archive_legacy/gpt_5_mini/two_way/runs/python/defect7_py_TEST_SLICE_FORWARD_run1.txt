What happened (where it fails)
- The failure occurs at line 767: classes.append(type(array[i])).
- The exception is an AttributeError "'NoneType' object has no attribute 'append'".
- That means classes is None at the moment the code reaches line 767, so the code tries to call append on None instead of on a list.

Why it happens (root cause)
- The method assumes classes is a list but, in the current implementation, classes can be left as None before the loop runs. The code does not guarantee classes is initialized to an empty list before calling classes.append(...).
- The presence of None elements in array is not directly the cause of append failing — type(None) is valid — the real cause is a prior code path that sets or leaves classes as None (for example an early assignment like classes = None or returning None for some inputs and then continuing) so when handling arrays with None elements the loop still executes but classes is not a list.
- In short: classes is not initialized correctly (or is overwritten to None) before the append loop. The observed symptom (failing when array contains None elements) happens because the code path that initializes classes is skipped or replaced when the input contains None items.
