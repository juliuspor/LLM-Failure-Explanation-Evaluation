translate_codepoint implementations in this codebase return the number of UTF-16 code units consumed (1 or 2). The previous implementation iterated consumed times and called code_point_at each time, which could call code_point_at with pos equal to length and raise IndexError. The simplest correct fix is to advance pos by consumed code units directly. I'll replace the loop that advanced pos per codepoint with pos += consumed. I will preserve existing checks and behavior otherwise.
