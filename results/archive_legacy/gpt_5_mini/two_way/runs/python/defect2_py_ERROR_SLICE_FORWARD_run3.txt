What the error means
- Error: ValueError: Color parameter outside of expected range: Red Green Blue
- This means some function received a color argument whose value is the string "Red Green Blue" (or a value that prints like that), and that function validates color values and rejected it because it’s not in the allowed format or numeric range.

Why this happens (most common concrete reasons)
1. You passed a single string containing three words ("Red Green Blue") instead of a recognized color name, RGB tuple, or numeric channels. Many libraries accept:
   - A named color string like "red" (lowercase, one word),
   - An RGB tuple (255, 0, 0) or normalized floats (1.0, 0.0, 0.0),

2. You passed a string representation of three values joined by spaces instead of a sequence/tuple. Example: color_param = "255 0 0" or "Red Green Blue" when the function expects (255, 0, 0) or ["Red","Green","Blue"] depending on API.

3. The color variable was built incorrectly earlier (concatenation or join produced a single string), so by the time it reaches the drawing/plotting function it’s invalid.
