What happened (where it fails)
- The failure happens on lines 30–31 in both blocks:
  ```
  30 |     if index < 0 or index >= len(s):
  31 |         raise IndexError(f"String index out of range: {index}")
  ```
- The code raises IndexError whenever index < 0. That means any negative index (for example index == -1) is rejected here.

Why that is the root cause
- In Python, negative indexes are valid for string indexing and slicing: s[-1] is the last character, s[-2] the one before last, etc.
- Your bounds check treats any negative index as out of range. If the caller passes a negative index intentionally (common for backward slicing or to refer to the tail of the string), your code wrongly flags it as an error and raises IndexError on lines 30–31.

Concrete example showing the problem
- Suppose s = "hello" and index = -1.
- At line 30 the condition index < 0 is true, so line 31 raises IndexError("String index out of range: -1").
- That is incorrect because s[-1] is a valid reference to "o".

Summary
