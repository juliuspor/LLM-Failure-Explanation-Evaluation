What happened (where, with which variables)
- At line 177 you do value = int(x). If x is -0.0 (a negative zero float), int(-0.0) produces 0. That removes the negative sign.
- Later you use value (and mantissa = value at line 178) to build output at line 192 (self.add(str(value))). Because value is 0 (not -0), the code adds "0" instead of keeping "-0.0".
- The test expected the negative-zero representation "-0.0", but the code emitted "0". That is why the assertion says Expected -0.0, got 0.

Why this happens (root cause)
- In IEEE-754 floats Python respects the sign of zero (there is a distinct -0.0), but converting to int or formatting an integer loses that sign information. The cast at line 177 is the exact point where the sign bit is discarded, so any later use of value/ mantissa cannot restore the negative zero information.
