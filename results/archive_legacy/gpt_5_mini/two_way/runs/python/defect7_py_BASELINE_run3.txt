What happened (root cause, where it occurred)
- In the loop at lines 765–767 you call array[i].upper() unconditionally:
  - Line 766: _ = array[i].upper()
- If array[i] is None, calling .upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- That matches the failing test test_to_class_with_none_element which passes ["Test", None, 99.0]; the second element is None and triggers the exception.

Why this is wrong
- The method should accept arrays containing None and produce the class/type for each element (the tests expect type(None) for None entries).
- The .upper() call is unnecessary for computing the element type and has no relation to producing a Class/type. It appears to be a leftover check that assumes every element is a string, which is not true and causes the error.

Exactly where to change (lines and variables)
- Remove or guard the call to array[i].upper() at line 766.
- Replace the loop with logic that handles None explicitly, then appends type(array[i]) for non-None elements.

Suggested fix (concrete steps and code)
1. Replace the for loop (lines 765–767) with a guarded version:
   classes = []
   for i in range(len(array)):
       if array[i] is None:
           classes.append(type(None))
       else:
           classes.append(type(array[i]))
   return classes

2. Alternatively, more Pythonic:
   classes = [type(x) if x is not None else type(None) for x in array]
   return classes

3. Also update the docstring so the method does not claim it raises AttributeError on None elements.

Why this will fix the tests
- The new code never calls .upper(), so None elements no longer trigger AttributeError.
- It returns type(None) for None elements, matching the test expectations (e.g. [str, type(None), float]).

Notes
- The empty-array handling (ArrayUtils.EMPTY_CLASS_ARRAY.copy()) is fine.
- If your intent was to return None (not type(None)) for None elements, change classes.append(type(None)) to classes.append(None) — but the tests expect type(None).
