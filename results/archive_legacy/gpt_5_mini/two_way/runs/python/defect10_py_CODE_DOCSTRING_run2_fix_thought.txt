The bug arises when val is a single-character qualifier like 'l'/'L' producing numeric == '' and subsequent indexing causes IndexError. Fix by checking numeric length before any indexing, using a length-aware condition numeric.isdigit() or (numeric.startswith('-') and len(numeric) > 1 and numeric[1:].isdigit()). This preserves intended behavior for valid long-qualifier numbers and ensures invalid single-qualifier inputs raise ValueError.
