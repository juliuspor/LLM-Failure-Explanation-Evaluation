What happened
- Input: "l" (or "L")
- At line 10 the code sets last_char = val[-1] -> 'l'.
- At line 12 the code sets numeric = val[:-1] -> "" (empty string).
- At line 14 the code enters the branch for last_char in ("l", "L").
- At line 16 the code calls char_at(numeric, 0) (or otherwise accesses numeric[0]) to test whether numeric starts with "-" and then checks numeric[1:].isdigit().
- Because numeric is empty, char_at(numeric, 0) (or numeric[0]) raises an index error ("String index out of range: 0") instead of the intended ValueError from lines 18/21.

Root cause
- The code assumes numeric has at least one character before it reads numeric[0] (line 16). It does not guard for len(numeric) == 0, so for inputs that are a single qualifier character ("l" or "L") numeric is empty and the direct index access fails.

Exactly where to change it
- The problematic code is lines 14–17:
  - line 12: numeric = val[:-1]
  - line 14: if last_char in ("l", "L"):
  - line 16: if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():

How to fix it (clear step-by-step)
1. Add an explicit empty-string guard for numeric before accessing numeric[0]. For example, insert immediately after line 12:
   - if len(numeric) == 0:
       raise ValueError(f"{val} is not a valid number.")
   This prevents the index access and produces the correct ValueError for "l"/"L".

2. Alternatively, make the condition at line 16 safe without direct indexing by using startswith and length-safe slices:
   - replace line 16 with:
       if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():
   startswith("-") returns False for the empty string, so numeric[1:] is only evaluated in the context where numeric.startswith("-") is True, avoiding the index error.

Either change ensures inputs like "l" / "L" produce the intended ValueError instead of an IndexError.
