What happened (where it fails)
- At line 335–336 the code does:
  if array is not None:
      inferred_type = type(array)
- That sets inferred_type to the Python type of the list object itself (e.g. <class 'list'>), not the type of elements inside the list.
- Later code expects inferred_type to represent the element type (it compares inferred_type to object and to expected_type). Because inferred_type holds the list type instead of an element type, those comparisons and downstream logic are wrong and can raise TypeError (see the raise at line 346).

Why this is the root cause (why it fails)
- The docstring and the later checks make clear inferred_type is meant to represent an element type. Using type(array) gives you the container type, not the element type.
- Two concrete failure modes that result from this:
  1. If you pass a non-empty list, inferred_type becomes <class 'list'> instead of the list element class (e.g. int). Later comparisons to expected_type will fail because <class 'list'> != <class 'int'>, possibly raising TypeError.
  2. If you pass an empty list, you cannot tell the element type from the list contents; the code should treat that case specially (use expected_type or fall back to object). Using type(array) still produces <class 'list'> and confuses the type checks.

Minimal reproduction example
- Call add([1, 2], 3, expected_type=int) — inferred_type becomes list, not int, so the type check will not behave as intended and may raise TypeError.
- Call add([], "x", expected_type=str) — inferred_type becomes list; code does not correctly use expected_type as the element type.

What to change (concrete steps)
1. Replace inferred_type = type(array) with logic that gets the element type:
   - If array is not None and len(array) > 0: inferred_type = type(array[0])
   - Else if array is not None and len(array) == 0: inferred_type = expected_type or object
2. Keep the existing branch for element is not None (line 337) that does inferred_type = type(element).
3. Ensure the fallback to object (line 340) happens only when both array is None or empty and element is None and expected_type is not supplied.
4. After inferring the element type, perform the type-check comparisons against that element type.

Why this fixes it

- This aligns the inferred_type used in the check at line 346 with what the rest of the function expects, preventing incorrect TypeError raises.
