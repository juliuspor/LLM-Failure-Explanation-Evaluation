- The code calls StringUtils.index_of(text, " ", lower) without checking that lower is within the string bounds.
- If lower is greater than text's length, the index_of implementation being used throws an IndexError (String index out of range: 15 in your failing test) before any of the later safeguards run.
- In short: lower can be out of range, and StringUtils.index_of (or whatever underlying string.index implementation it delegates to) does not accept a start index greater than len(text), so the function raises instead of returning -1.

Where it fails (specific lines / variables)
- The failure occurs at the line:

  because lower is larger than len(text) in the failing test and index_of raises IndexError.
- Later code assumes index_of will return -1 for “not found” and only then uses substring_java(...), but the exception happens first.

- The checks for upper < lower and upper == -1 happen after you already call index_of, so they do not protect this call.
