What happened (root cause)
- At line 340 the code unconditionally sets inferred_type = object.
- That assignment is not inside the if/elif that begin at line 335; it overwrites the inferred_type that you set at lines 336 or 338.
- Later, at line 346 you test inferred_type == object and raise a TypeError when expected_type != object. Because inferred_type was always reset to object at line 340, that check triggers incorrectly even when array or element had a concrete type.

Where it occurred
- Lines shown:
  - 335: if array is not None:
  - 336:     inferred_type = type(array)
  - 337: elif element is not None:
  - 338:     inferred_type = type(element)
  - 340: inferred_type = object     ← this is the problematic unconditional overwrite
  - 346: if inferred_type == object and expected_type != object:
  - 347:     raise TypeError(...)

Why this causes the failure
- When you pass a non-None array or element, you intended inferred_type to be set to that object’s type (lines 336/338). But line 340 resets inferred_type to object regardless, so the later type check at line 346 always sees inferred_type == object and raises the TypeError when expected_type is not object. In short: your detected type is thrown away by an unintended overwrite.
