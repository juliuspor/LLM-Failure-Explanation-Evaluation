What happened (root cause)
- At line 52 your code does: search_length = len(search_list)
- That call raised TypeError: object of type 'NoneType' has no len()
- That means search_list is None when replace_each runs. The code then also uses replacement_list later (lines 81–82), so either of those being None would cause the same error.

Where it failed (exact places)
- Function: replace_each (definition starts at line 28)
- First failing expression: line 52 — len(search_list)
- Later lines that would also fail if either list/element is None: line 81 (loop over search_length) and line 82 — len(replacement_list[i]) - len(search_list[i]).

Why it became None (three very common, concrete causes to check)
1. The caller passed None for search_list (or replacement_list). Check every call site of replace_each; verify the actual arguments are lists, not None.
2. You assigned a variable using list.sort() or dict.update() and passed that variable in. Example mistake:
   - wrong: search_list = some_list.sort()  # .sort() returns None
   - right:  some_list.sort(); search_list = some_list
   - or:    search_list = sorted(some_list)
   Passing search_list after calling .sort() produces None.
3. A previous function that is supposed to return a list returned None (forgotten return). Example:
   - def build_search_list(...):
         x = [...]
         # missing "return x"
     Then search_list = build_search_list(...) becomes None.

How to confirm quickly
- Add a debug assertion at start of replace_each:
  - if search_list is None or replacement_list is None: raise ValueError("search_list and replacement_list must be lists, got %r and %r" % (search_list, replacement_list))
- Or print/debug the caller right before calling replace_each:
  - print("search_list repr:", repr(search_list))
  - print("replacement_list repr:", repr(replacement_list))

Concrete fixes (step-by-step)
1. Inspect the call sites of replace_each and ensure you pass actual lists:
   - Replace any use of variable = some_list.sort() with either sorted(some_list) or do some_list.sort(); variable = some_list.

Summary
The TypeError occurs because search_list (or replacement_list) is None when the code calls len(...) on it at line 52. This usually comes from passing the result of an in-place method that returns None (like list.sort()) or from a function that forgot to return its list.
