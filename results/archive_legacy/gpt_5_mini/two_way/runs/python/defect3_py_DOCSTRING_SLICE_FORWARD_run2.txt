What happens
- At line 38 the code does: high = code_units[index].
- If index equals len(code_units) (i.e. index is past the last element), that line raises IndexError: list index out of range before the function ever reaches the later bounds check at line 39.

Why it happens (root cause)
- The code only checks index + 1 < len(code_units) (line 39) to decide whether it can read a low surrogate at index + 1. It never checks that index itself is valid before reading code_units[index]. The read on line 38 therefore can fail when index is out of range.
- In short: the code reads code_units[index] without first verifying index < len(code_units).

Where it occurs
- In this function, at line 38: high = code_units[index]. That is the exact statement that causes the IndexError when index is equal to len(code_units).

How to fix (concrete steps)
1. Validate index before reading code_units[index]. For example, add an explicit check at the top:
   - if index >= len(code_units): raise IndexError("index out of range") or return an appropriate sentinel.
2. Or change the existing conditional to avoid the out-of-range read, e.g.:
   - if index < len(code_units):
         high = code_units[index]
         if 0xD800 <= high <= 0xDBFF and index + 1 < len(code_units):
             low = code_units[index + 1]
             if 0xDC00 <= low <= 0xDFFF:
                 return 0x10000 + ((high - 0xD800) << 10) + (low - 0xDC00)
         return high
     else:
         raise IndexError("index out of range")
3. Make a design decision about how the function should behave when index is invalid:
   - If index being out of range is caller error, raise IndexError with a clear message.
   - If the function should silently handle it, return None or a sentinel value (document this choice).

Why these fixes solve the problem
- They prevent reading code_units[index] when index is invalid, so the IndexError no longer occurs. The check at line 39 is only meaningful after confirming index is within the list, so moving/adding the index check ensures correct control flow and prevents the crash.
