What happened
- The code crashes with an IndexError coming from the char_at helper (slice block lines 30–31) when you pass inputs like "L".
- Root cause: in the branch that handles trailing 'l' or 'L' (the if last_char in ("l", "L") block), the code sets numeric = val[:-1]. For val == "L" this makes numeric == "" (empty string).
- Then the code calls char_at(numeric, 0) without checking numeric's length. char_at("", 0) raises IndexError("String index out of range: 0") (see slice block lines 30–31). That's the crash.

Exactly where in the code
- Function: create_number
- Branch: the block starting with if last_char in ("l", "L"):
- Problematic line: the condition expression (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) — this calls char_at(numeric, 0) even when numeric is empty. That causes the IndexError shown in the slice block (lines 30–31).

Why this occurs (simple explanation)
- When val is a single-character string ending with L/l (e.g. "L"), numeric becomes empty.
- The code assumes numeric has at least one character and accesses index 0 via char_at, which is invalid for an empty string.

How to reproduce quickly
- Call create_number("L") — this raises IndexError: "String index out of range: 0".
- Any single-character input that is just "l" or "L" will reproduce the failure.

Concrete fixes (pick one)
1) Add an explicit length guard before calling char_at:
   - Right after numeric = val[:-1], add:
     if len(numeric) == 0:
         raise ValueError(f"{val} is not a valid number.")
   - Then the existing condition using char_at is safe.

2) Avoid char_at and use safe checks on numeric:
   - Replace the condition with:
     if (numeric.startswith("-") and numeric[1:].isdigit()) or numeric.isdigit():
         return int(numeric)
   - This does not index numeric[0] when numeric is empty and covers the same logic.

3) Combine guard and simpler condition:
   - if not numeric:
         raise ValueError(f"{val} is not a valid number.")
     if (numeric[0] == "-" and numeric[1:].isdigit()) or numeric.isdigit():
         return int(numeric)

Which I recommend
- Use option 2 (replace with startswith and isdigit). It's clearer, avoids manual index checks, and prevents the IndexError.

Summary
- Crash caused by calling char_at(numeric, 0) when numeric == "" (single-character "L"/"l" input).
- Add a length check or use startswith/is digit checks to prevent accessing index 0 of an empty string.
