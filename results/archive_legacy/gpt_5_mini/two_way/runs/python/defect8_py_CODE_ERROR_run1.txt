- The code fails at the check:

Why it fails (root cause)
- "fr__POSIX" is a valid form meaning language = "fr", empty country, variant = "POSIX".
- After confirming the third character (locale_str[2]) is '_', the code assumes characters at indexes 3 and 4 are both uppercase letters (a two-letter country). It does not allow the country to be empty.
- For "fr__POSIX", locale_str[3] is '_' (the second underscore), not an uppercase letter, so the check ch3 < 'A' ... raises ValueError.
- In short: the code does not handle the language + empty-country + variant pattern (double underscore) and therefore rejects "fr__POSIX".

Where it occurs in the code
- After the branch that handles length != 2 and after verifying locale_str[2] == '_', at the lines that set ch3/ch4 and validate them as uppercase letters.

How to fix it (concrete steps)
1. After confirming locale_str[2] == '_', check whether the country is empty by testing locale_str[3] == '_'. If true, parse variant from the remainder:

   - country = "" (empty)
   - variant = locale_str[4:] (variant starts after the second underscore)
   - return Locale(language, country, variant)
2. Otherwise, validate ch3 and ch4 as uppercase letters (existing checks), and continue the existing logic for either 2-part (language_country) or 3-part (language_country_variant) locales.
