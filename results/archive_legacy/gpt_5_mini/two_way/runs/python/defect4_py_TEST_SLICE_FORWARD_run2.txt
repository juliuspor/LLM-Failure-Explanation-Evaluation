What happened
- The failing tests show that TimePeriodValues._update_bounds sets the wrong index for the "max middle" case. The method computes the midpoint for each period but then stores the wrong index variable when it finds a new maximum midpoint. That is why get_max_middle_index() sometimes returns 3 (or some other index) instead of the index that actually has the largest midpoint (1 in your failing case).

Where it happens
- File: hit04_timeperiod.py
- Method: TimePeriodValues._update_bounds
- Faulty code location: the branch that updates the maximum midpoint value and index. The code assigns the index variable that was used for a different comparison (for example the start-index check) instead of the index corresponding to the midpoint comparison.

Why it fails (root cause)
- _update_bounds computes a midpoint value for a period and also has an index variable that corresponds to the midpoint (let's call it middle_index in the explanation). But when it updates the maximum midpoint it uses the wrong variable for the index assignment (for example it does self.max_middle_index = index or self.max_middle_index = start_index) instead of self.max_middle_index = middle_index.
- Because the wrong index variable is written into self.max_middle_index, get_max_middle_index() returns an index that belongs to whichever other check last set that wrong variable, not the index of the actual maximum midpoint.

How to confirm quickly
- Open hit04_timeperiod.py, find _update_bounds.
- Look for a block like:
  if middle > self.max_middle:
      self.max_middle = middle
      self.max_middle_index = index   # <-- wrong variable
- Replace the right-hand side with the variable that tracks the midpoint's index (often named middle_index, mid_index, i, or the loop index used for that midpoint check).

Precise fix (step-by-step)
1. Open hit04_timeperiod.py and find the definition of TimePeriodValues._update_bounds.
2. Locate the branch that updates the "max middle" tracking variables. It will look similar to:
   if middle > self.max_middle:
       self.max_middle = middle
       self.max_middle_index = <wrong-variable>
   where <wrong-variable> is something like index, start_index or min_middle_index.
3. Change that assignment so it uses the index variable that corresponds to the midpoint comparison. For example, if the code computed the midpoint while iterating with loop variable i and used middle_index = i, assign:
   self.max_middle_index = middle_index
   or simply:
   self.max_middle_index = i
   (pick whichever name is the loop/index variable used for the midpoint).
4. Run your unit tests (the tests you pasted). The tests that asserted get_max_middle_index() should now pass.
