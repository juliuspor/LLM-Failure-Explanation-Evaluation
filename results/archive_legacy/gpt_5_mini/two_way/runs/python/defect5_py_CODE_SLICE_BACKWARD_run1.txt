What went wrong (root cause)
- At line 335 you set inferred_type = type(array). That takes the Python type of the list object itself (which is the list class), not the element/component type you need.
- ArrayUtils._copy_list_grow1 (called at line 343) expects a component/element class (the type of items the list contains), not the list type. Passing the list class produces the wrong kind of new list/array and leads to the later failure when you assign element into new_list.
- Because inferred_type becomes list when array is not None, the fallback check at line 346 (inferred_type == object and expected_type != object) never fires, so you don't detect the unresolved element-type situation when the real intent is to use object/expected_type. That lets an invalid conversion/assignment proceed and causes the runtime error (ClassCast-like failure reflected in your message).

- Line 335: inferred_type = type(array) — wrong value (list type, not element type).
