What happened (where and what error)
- Failure occurs inside ClassUtils.to_class at the loop shown in the slice: lines 765–767.
- The code does:
  - for i in range(len(array)):
  -     _ = array[i].upper()
  -     classes.append(type(array[i]))
- When array[i] is None, the expression array[i].upper() tries to call upper on None. None has no attribute upper, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- In your tests the second element is None (test_array = ["Test", None, 99.0]), so the AttributeError happens immediately at line 766 and the method never returns the expected list of types.

Why it happened (root cause)
- The code unconditionally calls .upper() on every array element regardless of its type.
- .upper() only exists on str (and some other objects) — not on NoneType, numbers, lists, etc.
- The attempt to call .upper() on None is the direct cause of the exception. This is analogous to the Java NullPointerException mentioned in your docstring, but in Python it appears as AttributeError.

How to fix it (concrete steps and exact code suggestions)
- Remove or guard the .upper() call. You have three clear options depending on the intended behavior:

1) If you do not need .upper() at all (simplest, preferred if you're only collecting types):
Replace the three lines with:
  for i in range(len(array)):
      classes.append(type(array[i]))

2) If .upper() was intended only for strings (normalize string values) and shouldn’t be called on other types:
Replace the three lines with:
  for i in range(len(array)):
      value = array[i]
      if isinstance(value, str):
          _ = value.upper()   # safe: only called for strings
      classes.append(type(value))

3) If you want to defensively attempt .upper() but ignore elements that don’t support it:
  for i in range(len(array)):
      value = array[i]
      try:
          _ = value.upper()
      except AttributeError:
          pass
      classes.append(type(value))

- Also ensure earlier in the function you already handle the None input case (your tests expect ClassUtils.to_class(None) to return None). If that check is missing, add:
  if array is None:
      return None

Why these fixes work
- All three fixes avoid calling .upper() on None (or other non-string types), so no AttributeError is raised.
- They preserve the intended behavior of returning the types of each element (including type(None) for None).

Summary
- Root cause: unconditional call to array[i].upper() at line 766 when array[i] can be None.
- Fix: remove the .upper() call or only call it for strings (see code snippets above).
