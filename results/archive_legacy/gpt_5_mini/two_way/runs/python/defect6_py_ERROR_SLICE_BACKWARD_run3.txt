What happened (root cause)
- At line 176 your code uses the test:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- That equality test x == int(x) treats -0.0 and 0.0 as equal. In Python -0.0 == 0 evaluates True. Because the code thinks x is an “integer-valued” float it later converts or normalizes it to an integer/zero value.
- Converting -0.0 to an integer or to a plain zero loses the negative-zero sign bit. The test expected -0.0, but the code produced 0 (positive zero), so the assertion failed: Expected -0.0, got 0.

Why this happens (specifics)
- Floating-point -0.0 carries a sign bit different from +0.0, but equality compares them as equal. So x == int(x) is not a safe way to detect that x is mathematically an integer without also treating signed zero specially.
- The problem happens at line 176 (the equality branch). Whatever code after that branch returns or prints a normalized integer/zero value, producing 0 instead of preserving -0.0.
