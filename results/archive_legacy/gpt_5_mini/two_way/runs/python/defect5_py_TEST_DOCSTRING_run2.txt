Why it fails (root cause)
1. The add method signature in hit05_arrayutils.py does not include expected_type (or does not accept **kwargs). That means when the test calls ArrayUtils.add(..., expected_type=str) Python immediately raises TypeError before any logic runs. This is a mismatch between the test caller and the method API.
2. The docstring for add documents expected_type and describes behavior when both array and element are None. The implementation does not follow that contract:
   - Either expected_type is completely missing from the signature, or
   - expected_type exists but is ignored so the method can't satisfy the caller's need to "declare" the expected element type when both array and element are None.
3. The specific failing place: the call in test_add_null_to_null_with_expected_type (line where result = ArrayUtils.add(None, None, expected_type=str)) triggers the error. The variables involved are:
   - array (passed as None),
   - element (passed as None),
   - expected_type (passed as keyword str) — this is the one not handled by the implementation.
