What happened (what fails)
- In replace_each, the code sets search_length = len(search_list) at line 52 and later iterates i from 0..search_length-1 at line 81.
- On line 82 the code computes greater = len(replacement_list[i]) - len(search_list[i]).
- If either search_list[i] or replacement_list[i] is None, calling len(...) raises TypeError: "object of type 'NoneType' has no len()", and the function crashes.

Why it fails (root cause)
- The code assumes every entry in search_list and replacement_list is a string. It does not guard against None before using len(...) at line 82.
- The docstring/bug report says None entries should be ignored, so the code should skip those indices instead of calling len on them. The skip logic that may exist earlier (around the first loop at line 59) is not present (or is incomplete) in the second loop that computes lengths — so None slips through and causes the crash.
- An additional related hazard: search_length was set to len(search_list) (line 52). If replacement_list is shorter than search_list, replacement_list[i] can be out of range (IndexError). That makes the code fragile when lists differ in length.

- Variables and lines: search_length at line 52; first loop beginning at line 59 (likely does some work); second loop beginning at line 81; failure point at line 82 where len(replacement_list[i]) and len(search_list[i]) are used.
