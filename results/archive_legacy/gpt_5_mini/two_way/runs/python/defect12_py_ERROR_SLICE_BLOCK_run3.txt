  if start < 0 or end < 0 or start > end or start > len(s) or end > len(s):
    raise IndexError(f"String index out of range: {end}")
- The test shows IndexError: String index out of range: 15, so end is 15.
- The guard uses strict > len(s). If len(s) == 15 then end == 15 does NOT trigger the raise at lines 32–33.

Why the check is wrong
- You used start > len(s) and end > len(s). Those allow start == len(s) and end == len(s).
- If subsequent code does direct indexing (s[start] or s[end]) those indices must be < len(s). Allowing equality lets an out-of-range index through and causes the runtime IndexError.

  - replace end > len(s) with end >= len(s)
- If instead your function is meant to behave like Python slicing (where s[a:b] allows b == len(s)), then you must not later do s[end]. Either:
  - stop indexing s[end] and use slicing semantics only (use s[start:end]); or
