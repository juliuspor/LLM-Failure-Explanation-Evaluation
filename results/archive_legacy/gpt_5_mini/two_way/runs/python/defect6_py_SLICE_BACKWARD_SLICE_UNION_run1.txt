What happens
- The code enters an infinite loop at lines 180–183. The loop condition is:
  while mantissa % 10 == 0:
- Inside that loop you never change mantissa. Because mantissa stays divisible by 10 the condition never becomes false and the loop never ends.

Where it occurs (lines you gave)
- At line 176 you set value = int(x) and mantissa = value (lines 177–178).
- At line 180 you check if abs(x) >= 100, and at line 181 start the loop while mantissa % 10 == 0.
- Inside the loop you compute check_val = (mantissa // 10) * (10 ** (exp + 1)) at line 182, then compare check_val == value at line 183.
- You never assign to mantissa inside that while block, so once the loop starts it cannot terminate (unless an exception occurs).

Why that causes failure
- If value (and therefore mantissa) ends in one or more zeros, mantissa % 10 == 0 is true and stays true forever because mantissa is not changed. The code will hang at that point instead of progressing to later lines (for example the adds at lines 190,192,194).

Exact root cause
- Missing update to mantissa in the while loop. The loop intends to strip trailing zeros from mantissa (or otherwise reduce it) but does not perform that reduction.

Two concrete fixes (pick one)

1) Strip trailing zeros by updating mantissa inside the loop
Replace the body of the while with an update, for example:
  while mantissa % 10 == 0:
      check_val = (mantissa // 10) * (10 ** (exp + 1))
      if check_val == value:
          mantissa = mantissa // 10   # <<-- reduce mantissa so loop can end
      else:
          break
Rationale: this reduces mantissa each iteration. The break prevents an infinite loop when check_val never equals value.

2) Do the test without a while when you only need one reduction, or restructure the loop condition
If the intention was to test successive reductions until you match value, use:
  while mantissa % 10 == 0:
      mantissa = mantissa // 10
      check_val = mantissa * (10 ** (exp + 1))
      if check_val == value:
          continue
      else:
          break
Or explicitly test and break when appropriate. The important part is to ensure mantissa is changed or the loop condition will become false.

How to verify the fix
- Run with an x that is an integer ending in zeros and abs(x) >= 100 (e.g. x = 1200.0). Before the fix the code hangs; after the fix it should progress to lines 190/192/194.
