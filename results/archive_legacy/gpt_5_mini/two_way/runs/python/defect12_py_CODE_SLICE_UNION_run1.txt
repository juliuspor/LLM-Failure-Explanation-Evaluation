What happened (root cause)
- The code can call substring_java(text, 0, upper) with upper > len(text). That triggers the IndexError raised in slice_union at lines 32–34:
  - slice_union checks if end > len(s) and raises IndexError(f"String index out of range: {end}").

Why that can occur in this function
- At the start you try to ensure upper is in range with:
  - if upper == -1 or upper > len(text): upper = len(text)
- But later you do:
  - if upper < lower: upper = lower
- If lower > len(text), that assignment makes upper > len(text) again. There is no subsequent cap to len(text), so substring_java(text, 0, upper) gets called with an end index > len(text) and slice_union raises the IndexError.
- The problem can occur when the caller gives a lower value greater than text length. For example:
  - text = "short" (len 5), lower = 10, upper = 7 (or upper = -1). The code sets upper to len(text) (5) then upper < lower so upper = lower (10). Calling substring_java(text, 0, 10) causes the error shown.

Where it happens in your code
- The error originates when substring_java(text, 0, upper) is called (lines shown in your code: the two calls inside the index == -1 branch and the index > upper branch). The slice_union check at lines 32–34 raises because end (upper) > len(s).

How to fix it (concrete steps)
Option A — clamp lower and upper before using them (recommended):
1. Immediately after checking text is not None/empty, add:
   - lower = max(0, min(lower, len(text)))
   - if upper == -1: upper = len(text)
   - else: upper = max(0, min(upper, len(text)))
2. Then keep the rest of your logic (or adjust the upper < lower check accordingly). This guarantees substring_java is always called with end <= len(text).

Option B — handle the case lower > len(text) explicitly:
1. After determining upper initially, add:
   - if lower > len(text):
       return substring_java(text, 0, len(text)) + StringUtils.default_string(append_to_end) if you want a clipped result,
       or return text (or None) depending on desired semantics.
2. Keep the upper < lower logic but ensure upper is clamped to len(text) before any substring_java calls.

Minimal patch example (Option A):
- Replace the current block:
    if upper == -1 or upper > len(text):
        upper = len(text)

        upper = lower
- With:

Summary
- The function fails because upper can become larger than len(text) when you set upper = lower and lower > len(text). Clamp lower and upper to the valid [0, len(text)] range (or handle lower > len(text) explicitly) before calling substring_java to avoid the IndexError from slice_union.
