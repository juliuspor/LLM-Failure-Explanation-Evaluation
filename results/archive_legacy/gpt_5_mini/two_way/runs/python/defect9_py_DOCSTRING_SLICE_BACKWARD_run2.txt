What happened (what fails)
- The method replace_each (starts at line 28) crashes when either search_list or replacement_list contains None.
- The crash occurs at line 82 where the code does:
  greater = len(replacement_list[i]) - len(search_list[i])
  If replacement_list[i] or search_list[i] is None, calling len(...) raises TypeError: object of type 'NoneType' has no len().

Why it happens (root cause)
- At line 52 the code sets search_length = len(search_list). That length counts entries that may be None.
- The loops at line 59 and again at line 81 iterate i from 0..search_length-1 and index into search_list[i] and replacement_list[i] without first confirming those entries are non-None.
- The code apparently intended to "ignore" None entries, but it never filters them out or skips them before calling len(...) or using them. So the attempt to measure lengths on a None value triggers the crash.

Where exactly in the code
- replace_each method: start line 28 (method header).
- search_length computed: line 52.
- first loop index: line 59 (for i in range(search_length):).
- second loop index and failing expression: lines 81–82, specifically line 82: greater = len(replacement_list[i]) - len(search_list[i]).

Concrete fixes (step-by-step, specific)
Option A — Filter pairs before looping (recommended)
1. Replace the single search_length = len(search_list) at line 52 with:
   pairs = [(s, r) for s, r in zip(search_list, replacement_list) if s is not None and r is not None]
   search_length = len(pairs)
2. Change all loops that use range(search_length) and index into search_list[i]/replacement_list[i] to instead use the pairs list:
   for i in range(search_length):
       s, r = pairs[i]
       ... use s and r instead of search_list[i], replacement_list[i] ...
3. At line 82 use len(r) and len(s) (or len(pairs[i][1]) - len(pairs[i][0])) so None is never passed to len().

Option B — Skip None entries inside the loops
1. Keep search_length as originally set, but at the top of each loop (lines 59 and 81) add:
   if search_list[i] is None or replacement_list[i] is None:
       continue
2. This ensures you never call len(...) on a None value.
3. Also check elsewhere in the method for any other uses of search_list[i] or replacement_list[i] and guard them the same way.

Additional checks to add
- Ensure search_list and replacement_list are the same length before zipping or indexing. If lengths can differ, either raise a clear error or use zip_longest and handle missing entries explicitly.
- Decide whether an entry with search_list[i] is None but replacement_list[i] is not (or vice versa) should be ignored or treated as an error and document that choice.

Why these fixes work
- Both options prevent calling len(...) or other string operations on None by either removing None pairs before use (Option A) or skipping None entries at the moment of use (Option B). That prevents the TypeError at line 82 and matches the intended behavior of ignoring None replacements.
