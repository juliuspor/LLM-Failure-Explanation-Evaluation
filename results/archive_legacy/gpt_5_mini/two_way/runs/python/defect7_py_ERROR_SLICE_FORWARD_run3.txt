What happened (root cause)
- On line 767 you do: classes.append(type(array[i])).
- Some entries in array are None. When array[i] is None you either end up later trying to call .upper() on that same array element (array[i].upper()) or on the value you stored in classes (something like classes[j].upper()).
- None has no .upper attribute, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.

Why this happened at that place
- Line 767 stores a value derived from array[i]. If array contains None, you will get either None (if you append the element) or type(None) (if you append the result of type(...)).
- Later code (after line 767) assumes the stored value is a string and calls .upper() without checking for None. That assumption breaks when array[i] is None.

Exactly where to look
- array — inspect its contents at the index i used on line 767.
- line 767: classes.append(type(array[i])) — this is where the problematic value gets put into classes.
- the next location in your code where you call .upper() on items from classes or on array elements. That is the expression raising the AttributeError.

Concrete ways to fix it (pick one, with exact steps)
1) Skip None entries
   - Replace line 767 with:
     if array[i] is not None:
         classes.append(type(array[i]))
   - This prevents storing None/type(None) and avoids later .upper() on None.

2) Store a safe string instead of a raw value/type
   - If you need class names as strings (so .upper() is valid), change line 767 to:
     classes.append(type(array[i]).__name__ if array[i] is not None else '')
   - Or explicitly:
     if array[i] is None:
         classes.append('')
     else:
         classes.append(type(array[i]).__name__)

3) Defend the .upper() call
   - Instead of assuming the value is string, check before calling .upper():
     val = classes[j]   # or val = array[i] where .upper() is used
     if val is None:
         # handle None case (skip, set default, raise meaningful error)
     else:
         result = val.upper()

How to verify the fix
- Print or log the offending element immediately before line 767:
    print(f"i={i} array[i]={array[i]!r}")
- Run your tests: the AttributeError should disappear once None is handled or converted to a string.

Summary
- The error means you tried to call .upper() on None.
- That None originates from array[i] being None and getting propagated at line 767 into classes (or used later directly).
- Fix by either skipping None, converting it to a string, or checking for None before calling .upper().
