What happened (root cause)
- At line 176 the code decides whether x is an integer by testing x == int(x). For -0.0 that test returns True because -0.0 == 0.0 and int(-0.0) == 0.
- That sends -0.0 down the integer-handling branch (lines 176–192). At line 177 value = int(x) converts -0.0 → 0 (losing the negative sign).
- Later, because exp <= 2, the code runs line 192 self.add(str(value)), which adds "0".
- The result is the negative sign for negative zero is stripped before any string conversion, so the test that expects "-0.0" fails.

Where it occurs
- The problem is caused by the condition and code starting at line 176 (the integer branch) and specifically the conversion at line 177 and the write at line 192.
- The fallback branch (line 194) would have done self.add(str(x)) which would have preserved the sign because str(-0.0) yields "-0.0" in Python. But the code never reaches that branch for -0.0 because of the integer test.

Why this is wrong
- Negative zero is a distinct IEEE floating-point value with a sign bit. For the intended output, negative zero must be printed with a minus sign as "-0.0". Treating it the same as integer zero loses that sign.
- The integer test x == int(x) is too coarse: it groups -0.0 with 0 and causes the sign to be lost by int().

Concrete fixes (exact steps)
1. Detect negative zero before the integer branch and handle it explicitly. Add a check just before line 176:

   if CodeConsumer.is_negative_zero(x):
       self.add("-0.0")
       return

   This ensures -0.0 bypasses the integer path and is printed with its sign preserved.

2. Alternatively, prevent the integer branch from capturing negative zero by changing the condition on line 176 to exclude negative zero:

   if x == int(x) and not CodeConsumer.is_negative_zero(x) and not math.isinf(x) and not math.isnan(x):

   That keeps the rest of the integer logic unchanged but leaves negative zero to be handled by the non-integer branch (which will use str(x) and produce "-0.0").

Why these fixes work
- They stop -0.0 from being converted by int(), therefore preserving its sign. If you handle it explicitly, you print "-0.0" directly. If you exclude it from the integer branch, the code falls through to self.add(str(x)) (line 194), which yields "-0.0".

Which lines to change
- Add the explicit negative-zero check immediately before line 176 (the integer branch).
- Or modify line 176 to add and not CodeConsumer.is_negative_zero(x) to the condition.
