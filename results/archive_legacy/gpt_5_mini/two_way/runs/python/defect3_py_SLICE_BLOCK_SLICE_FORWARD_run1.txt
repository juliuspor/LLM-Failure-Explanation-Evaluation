Why that fails (root cause)
- If high is a high surrogate (0xD800..0xDBFF) and there is no valid low surrogate following it (either because index == len(code_units)-1 or because the next unit is not a low surrogate), the function returns the raw surrogate value (high) instead of signalling an error or otherwise handling the unpaired surrogate.
- The caller expects a valid Unicode scalar value (a code point). A raw high-surrogate value is not a valid Unicode code point on its own. Passing this invalid value downstream causes failures later (exceptions, corrupt text, or invalid encoding output).
- Concretely, the failure occurs when:

  - index + 1 >= len(code_units) (no next unit), or the next unit is not in 0xDC00..0xDFFF (line 41 false),
