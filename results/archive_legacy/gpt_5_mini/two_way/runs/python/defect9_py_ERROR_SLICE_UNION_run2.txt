What happened (root cause)
- At line 52 the code calls len(search_list). That fails because search_list is None, so Python raises TypeError: object of type 'NoneType' has no len().
- In other words, the function received search_list = None (or replacement_list = None in a similar place) but the code assumes those variables are sequences and uses len(...) immediately.

Where it occurred (exact lines)
- The failing call is at line 52: search_length = len(search_list).
- There are other places that would also fail if the outer list is None:
  - line 82: greater = len(replacement_list[i]) - len(search_list[i]) (if replacement_list or search_list is None)
  - line 94 / 104 checks array elements for None but that does not protect against search_list itself being None.

Why it happened (how the None got there)
- The function's API (replace_each) expects a list-like object but the caller passed None, or the function's default argument for search_list/replacement_list is None and the function does not convert/validate it before using len(...).
- The inner guard at line 103 (search_list[i] is None or len(search_list[i]) == 0) only protects against elements inside the list being None, not the list variable itself.

Clear fixes (concrete steps)
1) Validate inputs at start of replace_each (preferred if None should be illegal):
   - Add right after function entry:
     if search_list is None or replacement_list is None:
         raise TypeError("replace_each: search_list and replacement_list must be lists, not None")
   - Or be stricter:
     if not isinstance(search_list, (list, tuple)) or not isinstance(replacement_list, (list, tuple)):
         raise TypeError("replace_each: search_list and replacement_list must be list/tuple")

2) Or normalize None to empty list (if the intended behavior is to treat None as no replacements):
   - At top:
     if search_list is None:
         search_list = []
     if replacement_list is None:
         replacement_list = []

3) Also consider adding a length check to match lists:
   - if len(search_list) != len(replacement_list):
         raise ValueError("search_list and replacement_list must have the same length")

Why these fixes work
- They prevent calling len(...) on None by either refusing invalid input with a clear error (so caller can fix the call) or converting None into a safe value before any len() calls.
- They keep the existing element-level checks (line 103) meaningful, because those checks assume the container exists.

Minimal example patch to add at top of replace_each:
   if search_list is None or replacement_list is None:
       raise TypeError("replace_each requires non-None search_list and replacement_list")
   if not isinstance(search_list, (list, tuple)) or not isinstance(replacement_list, (list, tuple)):
       raise TypeError("replace_each requires list/tuple for search_list and replacement_list")
   if len(search_list) != len(replacement_list):
       raise ValueError("search_list and replacement_list must be same length")

This will stop the TypeError at line 52 and make the error clearer to the caller.
