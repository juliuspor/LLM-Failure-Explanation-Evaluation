What happens
- The conditional on line 176 evaluates int(x) before it tests math.isinf(x) or math.isnan(x).
- If x is float('inf') or float('nan'), calling int(x) raises (depending on the value) OverflowError / ValueError: you cannot convert infinity/NaN to integer.
- That exception occurs at line 176, so the code fails before the isinf/isnan checks can guard against it.

Why this happens (root cause)
- Python evaluates function calls and expressions left-to-right inside the if condition.
- The expression x == int(x) forces int(x) to run even when x is infinite or NaN.
- The code intended to avoid calling int(x) for non-finite values, but the order of operands defeats that intention.

Where it occurs
- At line 176: if x == int(x) and not math.isinf(x) and not math.isnan(x):

Symptoms / exact errors you would see
- ValueError: cannot convert float NaN to integer (for NaN)
- OverflowError: cannot convert float infinity to integer (for inf)

How to fix it (concrete steps)
1. Check finiteness before converting to int. Change the condition order so math.isinf/math.isnan or math.isfinite runs first. For example:
   - if not math.isinf(x) and not math.isnan(x) and x == int(x):
   - or better: if math.isfinite(x) and x == int(x):  (math.isfinite checks both not inf and not NaN)
2. Optionally, avoid calling int(x) twice by storing it:
   - if math.isfinite(x):
         xi = int(x)
         if x == xi:
             value = xi
             ...
3. Run tests with x = float('inf'), float('nan') and some normal floats to confirm the error is gone.

Why this fix works
- By checking math.isfinite(x) first, you prevent calling int(x) on NaN/Infinity, so no exception is raised and the rest of the logic runs only for finite numeric values.
