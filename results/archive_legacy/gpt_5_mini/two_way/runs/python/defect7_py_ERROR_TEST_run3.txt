What happened (root cause)
- The failure happens inside ClassUtils.to_class when it iterates the input array and calls element.upper() (or otherwise calls .upper on an element) without checking for None.
- When the test passes the list ["Test", None, 99.0], the second element is None. Calling None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'.
- The exception therefore occurs inside ClassUtils.to_class on the loop iteration that handles the None element — the code assumes every element is a string (or has an .upper method) and does not handle None specially.

Where it occurred (method and variable names)
- Method: ClassUtils.to_class
- Variable in the method that triggers the error: element (the loop variable that holds each array item)
- Specific operation: element.upper() (or any direct .upper() call on element)
- Triggering test input: test_array = ["Test", None, 99.0] in test_to_class_with_none_element

Why that logic is wrong for these tests
- The purpose of to_class is to return the runtime classes/types of the input elements (e.g., str, type(None), float). That requires checking the element’s actual type, not treating every element as a string.
- None does not have string methods. Calling .upper() on None is invalid and throws AttributeError; that’s the exact exception seen by the test.

Concrete fixes (step-by-step, specific)
1. Open hit07_classutils.py and find the to_class method.
2. Inside the loop that iterates the input array (where you currently reference the per-item variable element), replace the code that calls element.upper() (or any .upper use on element) with a None-safe type extraction.
3. Use type(element) to get the element’s class, and handle None explicitly.

Minimal safe replacement (example)
- Current (buggy) code (approximate):
  for i, element in enumerate(array):
      name = element.upper()   # <-- raises when element is None
      classes[i] = lookup_class_by_name(name)

- Replace with (correct logic):
  for i, element in enumerate(array):
      if element is None:
          classes[i] = type(None)
      else:
          classes[i] = type(element)

Explanation of the replacement
- If element is None, we set classes[i] = type(None) to match the tests’ expectation that a None maps to type(None).
- Otherwise, type(element) returns the proper Python type (e.g., str, float, int, list).
- This avoids calling any string methods on the element, so no AttributeError for None.

Optional improvement (if to_class is expected to accept class-name strings)
- If the original implementation intended to accept strings that name classes (and so used .upper to normalize names), separate the two behaviors:
  - If element is a str: interpret it as a class name (call element.upper() only after confirming isinstance(element, str)).
  - Else: treat element as an object and use type(element).
- Example:
  if element is None:
      classes[i] = type(None)
  elif isinstance(element, str):
      # map string class name to actual class, using element.upper() if needed
      classes[i] = lookup_class_by_name(element)
  else:
      classes[i] = type(element)

Summary
- Root cause: calling element.upper() while element can be None.
- Fix: check for None (and for str) first, then use type(element) for normal objects. This matches the tests (which expect type(None) for None elements) and eliminates the AttributeError.
