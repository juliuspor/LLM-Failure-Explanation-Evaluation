What happened (where it failed)
- The test fails with an AttributeError at the code fragment shown (lines 765–767).
- Specifically, at line 766 the code executes: array[i].upper()
- When array[i] is None, array[i].upper() raises: 'NoneType' object has no attribute 'upper' (or in Java terms, NPE). That is the direct cause of the failing test for input like ["Test", None, 99.0].

Why it failed (root cause)
- The method to_class iterates the input array and calls array[i].upper() before it inspects the element's type.
- Calling .upper() assumes the element is a string-like object; it does not guard against None (or other non-string objects).
- Because the test includes None, calling .upper() on None raises an exception before type(array[i]) can be appended. The code never checks for None first.
- The bug is therefore the wrong operation order and a missing None check: the code mutates/inspects the element as if it were string-like instead of simply determining its class safely.

Exact location in code
- Failure occurs at the shown lines:
  - Line 765: for i in range(len(array)):
  - Line 766:     _ = array[i].upper()    <-- causes AttributeError when array[i] is None
  - Line 767:     classes.append(type(array[i]))  <-- never reached for None element

What to change (clear steps to fix)
1. Stop calling .upper() (or any method) on array elements when the goal is only to determine their class.
2. Check for None first and append type(None) when appropriate.
3. For non-None elements, append type(obj) directly.

Concrete patch (replace lines 765–767)
- Replace the three lines with:

for i in range(len(array)):
    obj = array[i]
    if obj is None:
        classes.append(type(None))
    else:
        classes.append(type(obj))

Or, more idiomatically:

for obj in array:
    classes.append(type(None) if obj is None else type(obj))

Why this fix works
- It never calls any attribute or method on the element before checking for None.
- It directly records the element's class via type(obj), which works for strings, numbers, lists, booleans and None.
- It matches the tests’ expectations (e.g., None → type(None), "Test" → str, 99.0 → float).

Other notes
- Ensure to_class also handles None input (the tests call ClassUtils.to_class(None) and expect None) and empty lists (should return []).
- If the original code intended special string handling (e.g., using .upper()), do that only after the None check and after confirming the object is a string: if isinstance(obj, str): ... .upper() ... . But for getting the class, calling type(obj) is sufficient.
