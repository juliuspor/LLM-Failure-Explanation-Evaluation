What happened (root cause)

- The code first clamps upper down to len(text):
  if upper == -1 or upper > len(text):
      upper = len(text)

- Then it *possibly increases* upper to match lower:
  if upper < lower:
      upper = lower

If lower > len(text), that second step can set upper to a value greater than len(text) again. Later you call substring_java(text, 0, upper) (in the index == -1 branch and the index > upper branch), and substring_java throws IndexError because its end index (upper) is now past the string end (the error message shows index out of range: 15).

Where it fails
- The failing call is the substring_java(text, 0, upper) call in the index == -1 branch (and the other branches that use upper). The problem originates at the two lines that adjust upper:
  - first clamping upper to len(text)
  - then replacing upper with lower if upper < lower

Why that ordering is wrong
- Clamping upper first and then forcing it up to lower allows upper to exceed len(text) when lower is already past the string end. That produces an invalid end index for substring_java.

Concrete example
- Suppose len(text) == 10, lower == 15, upper == 5:
  - first clamp sets upper = 5 (since upper <= len)
  - then upper < lower so upper = 15
  - substring_java(text, 0, 15) -> IndexError

How to fix it (clear, actionable steps)

Option A — clamp upper after you adjust it to respect lower:
- Change the order so you ensure upper is at least lower, then clamp to len(text):
  if upper == -1:
      upper = len(text)
  if upper < lower:
      upper = lower
  if upper > len(text):
      upper = len(text)

Option B — compute upper in one step using min/max:
- Replace the three adjustments with:
  if upper == -1:
      upper = len(text)
  upper = min(max(upper, lower), len(text))

Option C — clamp lower itself before using it:
- Ensure lower <= len(text) (or treat lower > len(text) as an empty/no-op case), for example:
  if lower > len(text):
      lower = len(text)
  ... then keep your existing upper logic.

Pick one of these and apply it before any substring_java() calls. Any of the options prevents upper from exceeding len(text) and therefore prevents the IndexError.
