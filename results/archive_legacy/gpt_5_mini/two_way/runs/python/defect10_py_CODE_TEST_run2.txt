What happened
- The test inputs like "l" and "L" hit this branch:
  if not last_char.isdigit():
      numeric = val[:-1]
      if last_char in ("l", "L"):
          if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():
              return int(numeric)
- For val == "l" (or "L"), numeric becomes the empty string "".
- The code then evaluates char_at(numeric, 0) and numeric[1:], both of which are invalid for an empty string. Also there is no definition of char_at in this module, so attempting to call char_at will raise NameError (or if char_at were defined differently, indexing numeric[0] would raise IndexError). Either way, an exception other than ValueError is raised.
- The unit test expects a ValueError for inputs such as "l" and "L", but the code raises a different exception (NameError/IndexError), causing the tests to fail.

Why it fails (root cause)
- The code assumes numeric has at least one character before doing char_at(numeric, 0) and numeric[1:]. There is no guard for len(val) == 1 (numeric == ""), so the invalid access and/or undefined function call occurs for single-character inputs ending in L/l.

Where it occurs (exact spot)
- Inside create_number, in the branch handling last_char in ("l", "L"):
  if (char_at(numeric, 0) == "-" and numeric[1:].isdigit()) or numeric.isdigit():

Concrete fixes (step-by-step)
1. Replace the undefined/unsafe char_at usage and guard against empty numeric. For example:
   - Either check len(numeric) first:
       if numeric and ((numeric[0] == "-" and len(numeric) > 1 and numeric[1:].isdigit()) or numeric.isdigit()):
           return int(numeric)
   - Or add an early check for single-character inputs:
       if len(val) == 1:
           raise ValueError(f"{val} is not a valid number.")
2. Make sure numeric[1:] is only accessed when len(numeric) > 1.
3. Remove or replace char_at with direct indexing numeric[0].

Minimal corrected fragment:
    if last_char in ("l", "L"):
        if not numeric:
            raise ValueError(f"{val} is not a valid number.")
        if (numeric[0] == "-" and len(numeric) > 1 and numeric[1:].isdigit()) or numeric.isdigit():
            return int(numeric)
        raise ValueError(f"{val} is not a valid number.")

That change ensures inputs like "l" and "L" raise ValueError (as the tests expect) instead of raising NameError/IndexError.
