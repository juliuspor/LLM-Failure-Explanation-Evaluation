- In replace_each(), the code computes len(search_list[i]) and len(replacement_list[i]) in the length/growth estimation loop (lines 59–85). That happens before the later checks that exclude invalid entries (lines 102–105).
- If any search_list[i] or replacement_list[i] is None (or search_list[i] is ""), calling len(...) raises an exception (TypeError for None, or bad logic for empty strings).
- Concretely: the failure occurs at line 82 when this executes:

  If search_list[i] is None, len(search_list[i]) raises TypeError; if replacement_list[i] is None, len(replacement_list[i]) raises TypeError.

Why this is a bug (plain terms)
