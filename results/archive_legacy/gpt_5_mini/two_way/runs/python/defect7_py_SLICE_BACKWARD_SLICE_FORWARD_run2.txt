What happens
- At line 766 you call array[i].upper().
- If array[i] is not a string (for example None, an int, a list, etc.), Python raises:
  AttributeError: '<TYPE>' object has no attribute 'upper'
- That exception stops the loop before you reach line 767, so classes.append(type(array[i])) never runs for that iteration.

Why it fails (root cause)
- The code assumes every element in array is a string and therefore has an .upper() method. That assumption is false: at least one element in array is a non-string. The call to .upper() on that element throws the AttributeError at line 766.

Where it occurs
- Line 766: _ = array[i].upper()
- The exception prevents execution of line 767: classes.append(type(array[i])) for the same i.

How to fix it (clear step-by-step options)
Pick one of these depending on intended behavior:

1) Only call .upper() for string elements (recommended if you expect mixed types)
- Replace the loop body with an explicit check:
  for i in range(len(array)):
      value = array[i]
      if isinstance(value, str):
          upper_value = value.upper()
      else:
          upper_value = None   # or whatever fallback you need
      classes.append(type(value))

2) Convert non-strings to string before calling .upper() (if coercion is OK)
- This will turn numbers/None into strings:
  for i in range(len(array)):
      value = str(array[i])
      upper_value = value.upper()
      classes.append(type(array[i]))

3) Use try/except to handle unexpected types (if you want to continue on error)
  for i in range(len(array)):
      try:
          upper_value = array[i].upper()
      except AttributeError:
          upper_value = None
      classes.append(type(array[i]))

4) Iterate directly (style improvement) and avoid index usage:

      if isinstance(value, str):
          upper_value = value.upper()
      else:
          upper_value = None
      classes.append(type(value))

Summary
The code fails because array contains at least one non-string element, and calling .upper() on a non-string at line 766 raises an AttributeError and stops execution before line 767. Fix by checking type before calling .upper(), converting to string, or catching the exception.
