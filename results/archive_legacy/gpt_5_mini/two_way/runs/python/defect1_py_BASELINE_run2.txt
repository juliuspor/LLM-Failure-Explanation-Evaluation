      raise ValueError(f"Minutes out of range: {minutes_offset}")
  That rejects any negative minutes (e.g. -15). The docstring and tests allow minutes in the range -59 to +59, so -15 is valid. The test calling DateTimeZone.for_offset_hours_minutes(0, -15) triggered this check and produced the "Minutes out of range: -15" error.

1) Wrong minutes range check (line 396)
   - The condition uses minutes_offset < 0 which forbids negative minutes entirely.
   - The docstring and tests expect minutes_offset to be allowed down to -59.
   - This is the immediate cause of the failing test.

2) (Additional bug you should fix while you're here) Incorrect combination logic (lines ~401–404)
   - The code later converts hours to minutes and then reuses minutes_offset to compute the combined minutes:
       hours_in_minutes = hours_offset * 60
       if hours_in_minutes < 0:
           minutes_offset = hours_in_minutes - minutes_offset
       else:
           minutes_offset = hours_in_minutes + minutes_offset
   - This logic is wrong for negative hours + negative minutes. Example: hours_offset = -2, minutes_offset = -15:
       hours_in_minutes = -120
       code computes minutes_offset = -120 - (-15) = -105 minutes
     but the correct total is -135 minutes (-(2*60 + 15) = -135). So the sign handling here produces incorrect totals in some combinations.
   - Also reusing the input variable minutes_offset for the total minutes is confusing and error-prone.
