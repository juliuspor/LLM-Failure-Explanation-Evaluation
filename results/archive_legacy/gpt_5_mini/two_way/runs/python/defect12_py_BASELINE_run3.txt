What happened (summary)
- In the failing call WordUtils.abbreviate("0123456789", 15, 20, None) you pass lower=15 but text length is 10.
- The function sets upper to len(text) (10), then the next line (if upper < lower: upper = lower) makes upper = 15.
- Later the function calls substring_java(text, 0, upper) with end = 15 > len(text). substring_java checks bounds and raises IndexError: "String index out of range: 15".
- So the root cause is that the code allows upper to be increased above len(text) (by copying lower into upper) after upper was clamped to len(text), and then tries to take a slice with an end index beyond the string length.

Where this occurs (specific lines / variables)
- After: if upper == -1 or upper > len(text): upper = len(text)
- Then: if upper < lower: upper = lower
  - For text length = 10, lower = 15, this sets upper = 15.
- Later calls that fail:
  - result = substring_java(text, 0, upper)
  - return substring_java(text, 0, upper) + ...
- substring_java raises IndexError because end (upper) > len(text). The Slice bounds check (in your helper) is:
  if ... end > len(s): raise IndexError(f"String index out of range: {end}")

Why it’s wrong (brief)
- The code assumes lower is a valid index (<= len(text)) but does not handle the case lower > len(text). Updating upper to match lower can produce an illegal end index for slicing.

Concrete fixes (pick one)
1) Return original text when lower is beyond the string (matches test expectation and original Java behavior).

     if lower >= len(text):
         return text
   - This ensures you never set upper to a value > len(text) or call substring_java with an invalid end.

2) Or clamp lower to len(text) before using it:
   - After you clamp upper to len(text), add:

         lower = len(text)
   - This keeps upper <= len(text) and avoids invalid slice indices.

3) Alternatively, reorder checks so lower is validated first:
   - If lower >= len(text): return text
   - Then clamp upper: if upper == -1 or upper > len(text): upper = len(text)
   - Then existing logic
