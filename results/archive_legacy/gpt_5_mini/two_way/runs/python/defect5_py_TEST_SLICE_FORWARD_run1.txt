What fails
- The first test calls ArrayUtils.add(None, None, expected_type=str) (line 19 in the test file). The failure you see is a TypeError:
  TypeError: ArrayUtils.add() got an unexpected keyword argument 'expected_type'
- That happens because the hit05_arrayutils.ArrayUtils.add function does not define an expected_type argument in its signature (or otherwise accept keyword args). The test was written to reproduce a Java bug (LANG-35) by adding an optional expected_type hint; the implementation you have doesn't accept that hint.

Why it fails (root cause)
- The test calls ArrayUtils.add with three parameters (array, element, expected_type=...), but the ArrayUtils.add function only accepts two parameters (array, element). Python raises a TypeError when you pass an unknown keyword argument.
- This is purely a signature/mismatch issue. Nothing in the test body (the assertion logic) is reached because the call itself fails immediately.
- The failure happens at the call site in the test (test_add_null_to_null_with_expected_type), and ultimately within the function dispatch machinery for ArrayUtils.add — the interpreter compares the call arguments to the function signature and rejects the unexpected keyword.

How to fix it (clear steps)
1. Add an optional expected_type parameter to ArrayUtils.add's signature:
   - Change the function header to accept expected_type, for example:
       def add(array, element, expected_type=None):
   - This accepts the keyword used in the test and keeps backward compatibility.

2. Implement the expected_type behavior the tests expect:
   - If array is None and element is None and expected_type is provided, return a list containing a single None: [None].
   - If array is None and element is not None, return [element].
   - If array is a list, return a new list with element appended, leaving original list unmodified.

   Minimal example logic to insert into add (pseudo-Python):
       def add(array, element, expected_type=None):
           if array is None:
               # expected_type only matters for the test case where element is None,
               # but Python lists are dynamically typed so we just return a list.

           # array is a list: make a shallow copy, append, return it

           new.append(element)
           return new

   Note: The Java original required creating an array of the caller-expected component type; in Python that mapping is simpler — returning a list is sufficient. The key for the tests is to accept the expected_type parameter and not raise TypeError.
