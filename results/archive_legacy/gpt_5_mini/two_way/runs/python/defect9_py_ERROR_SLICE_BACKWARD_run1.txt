What happened (root cause)
- The TypeError "object of type 'NoneType' has no len()" means the code called len(...) on a value that is None.
- In your stack slice the first place that happens is line 52 inside replace_each:
  - 52 |         search_length = len(search_list)
  - That fails because search_list is None, not a list.
- Even if search_list were non-None, the loop on lines 81–82 would also fail if any element is None:
  - 82 |             greater = len(replacement_list[i]) - len(search_list[i])
  - If replacement_list[i] or search_list[i] is None, len(...) will raise the same error.

Why it happened (how None got there)
- The method replace_each was called with either:
  - search_list == None, or
  - replacement_list == None, or
  - one of the elements inside search_list or replacement_list == None.
- The code assumes inputs are non-None lists and their elements are strings (or other objects with length), but there is no check enforcing that. So when a caller passed None (or a list containing None), len(...) fails.

- replace_each method:
  - Line 52: len(search_list) — check the variable passed in as search_list.
  - Lines 81–82: len(replacement_list[i]) and len(search_list[i]) — check the lists’ elements.
- The caller of replace_each: find the code that invokes replace_each and inspect what it passes for search_list and replacement_list. That caller is the source of the None value.
