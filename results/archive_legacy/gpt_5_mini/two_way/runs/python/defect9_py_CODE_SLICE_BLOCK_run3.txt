What happened (what error you'll see)
- The code raises a TypeError: 'object of type 'NoneType' has no len()' at the loop starting on line 81 (slice block lines 81–84).
- That happens because len(replacement_list[i]) or len(search_list[i]) is being called when one of those list elements is None.

Where it occurs
- The problematic lines are 81–84:
  - line 82: greater = len(replacement_list[i]) - len(search_list[i])
  - these calls assume replacement_list[i] and search_list[i] are strings, but the function accepts Optional[List[Optional[str]]] so individual elements may be None.

Why it happens (root cause)
- Earlier in the function you explicitly allow None elements in the input lists (search_list and replacement_list are lists of Optional[str]).
- You do check for None elements in the two loops that search for matches (the loops that call text.find(...)). Those loops skip entries when search_list[i] is None or replacement_list[i] is None.
- However, the "guess result buffer size" loop (lines 81–84) runs before the first match-search loop and does not check for None. If any search_list[i] or replacement_list[i] is None, len(...) on that element raises the TypeError.
- The code comment even notes "BUG: this loop is not null-safe for elements inside the arrays" — that is exactly the problem.

Summary

- Fix by skipping None/empty entries in that loop or by filtering invalid entries up front.
