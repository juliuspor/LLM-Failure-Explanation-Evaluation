The bug stems from mixing UTF-16 code-unit indices with Unicode codepoint (character) indices. Rewriting translate to iterate over characters (codepoints) and mapping each character index to the corresponding UTF-16 code-unit index before calling translate_codepoint prevents IndexError. Build code_units and an offsets list mapping char index -> code_unit index, then use char_index as the unit for consumed counts while passing the correct code_unit index to translators.
