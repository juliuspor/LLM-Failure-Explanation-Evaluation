What happened (root cause)
- The test expected the exception message to contain the word "start", but the code actually threw an exception whose message was exactly "(bound must be positive)". The mismatch caused the assertion failure:
  AssertionError: Message (bound must be positive) must contain 'start'

Where this typically occurs in code
- This exact message is produced by java.util.Random.nextInt(int) when its argument (the bound) is <= 0. That means somewhere in the method under test you called Random.nextInt(bound) with bound <= 0.
- Common pattern that causes this:
  int bound = end - start;
  int r = random.nextInt(bound); // throws when end <= start

Why the test failed (clear, step-by-step)
1. The test invokes your method (for example, randomInRange(start, end) or similar).
2. Your method computed a bound as end - start and passed it to Random.nextInt(bound) without validating start and end.
3. When the test provided inputs where end <= start, Random.nextInt threw an IllegalArgumentException with message "(bound must be positive)".
4. The test caught the exception but asserted that the exception message must include the substring "start". Because "(bound must be positive)" does not contain "start", the test failed with the shown AssertionError.
