What went wrong (exact places in your code)
- Line: if array is not None: inferred_type = type(array)
  - Problem: type(array) returns the Python list class (e.g. <class 'list'>), not the element/component type you need. You later treat inferred_type as the element/component class, so this is incorrect.
- Lines: elif element is not None: inferred_type = type(element) and else: inferred_type = object
  - These are fine for the element case, but they create a second problem: when both array and element are None you set inferred_type = object.
- Lines: if expected_type is not None: if inferred_type == object and expected_type != object: raise TypeError(...)
  - Here you detect an unknown (object) inferred_type and raise when expected_type is provided. In the failing test you call ArrayUtils.add(None, None, expected_type=str). Because inferred_type was set to object (see previous point), the code raises a TypeError. The test expects [None] and no error.

Root cause summary
- You infer the wrong "type" for an existing array by calling type(array) (this returns list) instead of the element/component type.
- Because of that, when array is None and element is None you set inferred_type = object and then unconditionally raise if expected_type is provided, even though the caller explicitly gave expected_type and intended that to be used. The raise therefore happens in the call ArrayUtils.add(None, None, expected_type=str).

Where it occurs (specific variables and method)
- Variable inferred_type is computed from type(array) (wrong), type(element), or object.
- Method ArrayUtils._copy_list_grow1(array, inferred_type) is then called with the wrong inferred_type.
- The conditional that raises (checking inferred_type == object and expected_type != object) runs right after setting new_list and triggers the TypeError.

Concrete, step-by-step fixes
1. Prefer the caller-supplied expected_type first. If expected_type is supplied, use it as the component/element type to grow the list.
2. If expected_type is not supplied:
   - If array is not None and has at least one element, infer the element type from array[0] (type(array[0])).
   - If array is not None but empty, fall back to element type if element is provided, otherwise fall back to object (or leave as None) — but do not raise just because expected_type is missing.
   - If array is None and element is not None, infer type(element).
   - If everything is None, set inferred_type = object.
3. Do not raise when expected_type is provided and inferred_type is object; instead, use expected_type as the component type when calling _copy_list_grow1 so the caller's expectation is honored.
