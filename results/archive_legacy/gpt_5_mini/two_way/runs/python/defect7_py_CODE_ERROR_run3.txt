What happened (where it failed)
- The failure happens inside your function to_class, on this line in the loop:
    _ = array[i].upper()
- Python raised AttributeError: 'NoneType' object has no attribute 'upper'. That means array[i] was None for at least one element of array, so calling .upper() failed because None has no .upper method.

Why it happened (root cause)
- Your code assumes every element of array is a string (or an object with an upper() method). That assumption is false: one or more elements in array are None.
- When the loop reaches an element that equals None, array[i].upper() tries to call upper on None and fails immediately. The failure occurs before you append anything to classes.

Where to look in the code
- Method: to_class
- Problem line: the call to array[i].upper() inside the for loop
- A related suspicious line: classes.append(type(array[i])) — this appends the element’s type, not the element converted to a class or the string after .upper(), so the code intent is unclear and likely wrong.

Clear fix options (pick one that matches your intent)
1) If you want to ignore None elements
- Check for None before calling upper and skip them.
  Example:
    for i in range(len(array)):
        if array[i] is None:
            continue
        _ = array[i].upper()
        classes.append(type(array[i]))

2) If you want to treat None as an empty string (or default string)
- Replace None with '' (or some default) before calling upper:
    for i in range(len(array)):
        val = array[i] or ''
        _ = val.upper()
        classes.append(type(val))

3) If you expect every element to be a string and want a clear error when not
- Validate and raise a descriptive exception early:
    for i, val in enumerate(array):
        if val is None or not isinstance(val, str):
            raise TypeError(f"to_class: array[{i}] must be str, got {type(val).__name__}")
        _ = val.upper()
        classes.append(type(val))

4) If you intended to convert string names to classes (likely intent)
- Replace classes.append(type(array[i])) (which appends the element’s Python type, e.g. str) with code that resolves a class name from the string. Example using globals() or importing:
    for i, val in enumerate(array):
        if not isinstance(val, str):
            raise TypeError(f"to_class: expected str at index {i}")
        class_name = val.upper()   # probably you want val, not uppercased name; adjust as needed
        cls_obj = globals().get(class_name)
        if cls_obj is None:
            raise LookupError(f"class {class_name} not found")
        classes.append(cls_obj)

Specific short fix to avoid current error but keep behavior
- If you only want to stop the AttributeError and preserve current append logic:
    for i in range(len(array)):
        if array[i] is None:
            classes.append(None)
            continue
        _ = array[i].upper()
        classes.append(type(array[i]))

Why the suggested checks are necessary
- They make the code explicit about how to treat None values and non-string elements, preventing the unexpected AttributeError and making the function’s behavior predictable.
