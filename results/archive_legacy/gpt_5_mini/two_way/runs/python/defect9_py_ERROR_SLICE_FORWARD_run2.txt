What happened (root cause)
- The TypeError came from calling len(...) on a None value. The two places that call len in the slice are:
  - line 85: len(text) // 5
  - line 94: len(search_list[replace_index])
- The real failure is at line 94: search_list[replace_index] is None, so len(search_list[replace_index]) raises TypeError: 'NoneType' has no len().

Why that happened (how the code reached that state)
- Earlier you compute replace_index inside the search loop (lines 108–114). That loop uses temp_index = text.find(search_list[i], start).
- The code intends to skip entries where search_list[i] is None (there is a check including "or search_list[i] is None" on lines 102–105), but that check is only consulted in a different branch and does not prevent using search_list[i] when choosing replace_index.
- Concretely: the loop assigns replace_index = i based on temp_index without first guaranteeing search_list[i] is non-None. As a result, replace_index can point to an entry where search_list[replace_index] is None; later the code calls len(search_list[replace_index]) at line 94 and crashes.

Where to look (exact lines)
- Decision to skip/mark unusable replacements: lines 102–105 (the condition that contains "or search_list[i] is None").
- The search that sets temp_index and replace_index: lines 108–114 (temp_index = text.find(search_list[i], start) … replace_index = i).
- The failing call to len(): line 94 (start = text_index + len(search_list[replace_index])).

How to fix it (concrete steps)
1. Prevent search_list entries that are None from being used in the find/selection loop:
   - At the top of the loop that computes temp_index (line 108), add an explicit skip if search_list[i] is None:
     if no_more_matches_for_repl_index[i] or search_list[i] is None or replacement_list[i] is None:
         continue
   - That ensures you never call text.find(search_list[i], ...) with search_list[i] == None and never assign replace_index = i for a None search string.

2. Add defensive checks right before using replace_index:
   - Before line 92–94, assert/guard:
     if replace_index is None or search_list[replace_index] is None:
         break_or_raise_an_error()
   - Or raise a clear ValueError with a message like "replace_index refers to None search_list entry".

3. Validate inputs earlier:
   - At function entry, check that text is not None (line 85 uses len(text)) and that search_list and replacement_list are lists of same length without None entries unless you explicitly support None:
     if text is None: raise TypeError("text must be a string, not None")
     if any(s is None for s in search_list): raise ValueError("search_list contains None at index ...")

How to verify the fix
- Add a unit test that includes None in search_list and confirms the function either raises a clear error or skips that entry, depending on intended behavior.
- Add asserts/logging to confirm replace_index always refers to a non-None search_list entry before the len(...) call.

Summary
- The crash is caused by calling len(search_list[replace_index]) where search_list[replace_index] is None (line 94). The selection logic (lines 108–114) can assign replace_index to an entry that should have been skipped. Fix by skipping None search_list entries in the search loop (or validating inputs up front) and adding a guard before using replace_index.
