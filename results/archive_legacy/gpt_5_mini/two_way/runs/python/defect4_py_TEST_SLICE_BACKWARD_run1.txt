What happened (root cause)
- The updating logic in TimePeriodValues._update_bounds is overwriting the computed min/max index with the current index unconditionally. In the slice you provided the min_start update shows this clearly:
  - Line 346: if start < min_start: self._min_start_index = index
  - Line 349: self._min_start_index = index
- That unconditional assignment on line 349 makes _min_start_index always become the most recently added index, ignoring the earlier comparison. The same pattern (a wrong variable used or an unconditional assignment) exists for the "middle" bounds code path, causing get_max_middle_index() to return the wrong index (the last added index or some other wrong index) instead of the index selected by the comparison.
