What happened (where it failed)
- Failure occurs in to_class at the line marked in your slice: line 766:
    _ = array[i].upper()
- The Python exception is an AttributeError like:
    'NoneType' object has no attribute 'upper'
  or
    'int' object has no attribute 'upper'
  depending on the actual element type in array.
- That exception stops execution before you return classes.

Why it failed (root cause)
- The code calls .upper() on array[i] without ensuring array[i] is a string.
- If an element is None, an int, an object, or any non-string value, it does not implement .upper(), so calling .upper() raises AttributeError.
- The code does not use the result stored in _, so the .upper() call appears unnecessary as well.

Exactly where to look
- Method: to_class
- Line: the loop line with _.upper() (line 766 in your slice)
- Variable: array[i] — this element is the one that lacks an upper() method

Fixes (clear steps you can follow)
1. Remove the unnecessary .upper() call if you don't need it:
   - Replace the loop with:
     for item in array:
         classes.append(type(item))
   - This avoids touching item methods and directly produces the types.

2. If you intended to normalize string values before using them, only call .upper() when the element is a string:
   - Example:
     for item in array:
         if isinstance(item, str):
             name = item.upper()   # use name for whatever reason
         elif item is None:
             # handle None explicitly, or raise a clear error
             raise TypeError("to_class: array contains None at index X")
         classes.append(type(item))

3. If array should only contain strings, validate and raise an informative error early:
   - Example:
     for idx, item in enumerate(array):
         if not isinstance(item, str):
             raise TypeError(f"to_class: expected str at index {idx}, got {type(item).__name__}")
         classes.append(type(item))

Minimal fixed version (direct replacement)
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for item in array:
            classes.append(type(item))
        return classes

Why that minimal fix works
- It avoids calling .upper(), so no AttributeError is raised for non-string elements.
