What happened (where, exactly)
- The failure happens at line 766 in the loop:
  766 |             _ = array[i].upper()
- The traceback you saw will point to line 766 and raise an AttributeError like:
  AttributeError: 'int' object (or 'NoneType', etc.) has no attribute 'upper'

Why it failed (root cause)
- The code assumes every element in array is a string and therefore has an .upper() method.
- At least one element in array is not a str (for example an int, None, list, etc.). When Python evaluates array[i].upper() it fails because that object type does not implement upper().
- The failure occurs before line 767 runs, so classes.append(type(array[i])) is never reached for that iteration.

How to confirm the problem quickly
- Print or inspect the offending element and its type right before the call:
  print(i, array[i], type(array[i]))  # put this before line 766
  Running that will show the index and the non-string value causing the error.

Concrete fixes (specific code changes and steps)
1) If you only want to call upper on strings, check the type first:
   for i in range(len(array)):
       if isinstance(array[i], str):
           _ = array[i].upper()
       else:
           # handle non-strings explicitly
           _ = None
       classes.append(type(array[i]))

2) If you want to coerce everything to strings before upper:
   for i in range(len(array)):
       s = str(array[i])
       _ = s.upper()
       classes.append(type(array[i]))

3) If you want to skip non-strings and still record types:
   for i in range(len(array)):
       if not isinstance(array[i], str):
           classes.append(type(array[i]))
           continue
       _ = array[i].upper()
       classes.append(type(array[i]))

4) If you prefer to handle unexpected values and keep running, use try/except and log the index:
   for i in range(len(array)):
       try:
           _ = array[i].upper()
       except AttributeError:
           # log and continue
           print(f"Skipping index {i}: {array[i]!r} is {type(array[i])}")
       classes.append(type(array[i]))

Which to choose
- Use option 1 if non-strings are expected but should not be uppercased.
- Use option 2 if you want a string version of every element.
- Use option 4 if you want robustness and diagnostic output while preserving processing.

Summary
- The code fails at line 766 because array[i] is not always a string and therefore does not have an upper() method. Add a type check, convert to str, or catch the AttributeError so the loop can handle non-string elements and reach line 767.
