What failed
- Root cause: at line 335 you call type(array). That returns the Python type of the list object itself (e.g. <class 'list'>), not the type of the elements the list is supposed to hold. The rest of the function (ArrayUtils._copy_list_grow1 and the later type check at line 346) expects an element type (like int, str, or a specific class), not the list class. Because inferred_type is wrong, the wrong kind of list/array is created and the type checks later are meaningless. This is where the bug originates.

Where it occurs (specific lines)
- Line 335: inferred_type = type(array) — uses the list object’s class instead of the element class.
- Line 336–338: the fallback sequence is wrong for the array case; it never inspects array contents.
- Line 346: if inferred_type == object and expected_type != object: ... — this check assumes inferred_type will be the element type or object. Because inferred_type is <class 'list'> when an array was supplied, this check never works as intended.

Why that causes a failure (what happens)
- If array is non-None, type(array) yields <class 'list'> (or whatever container class), so ArrayUtils._copy_list_grow1 receives the wrong type and will create a container incompatible with the element's type expectation. If the array is empty, you also have no element type information from the array, so using type(array) gives no useful element type. The final type-check and potential TypeError are therefore wrong or ineffective, and assignments like new_list[len(new_list) - 1] = element can fail or produce incorrect behavior because the container was created for the wrong element type.

Concrete fixes (step‑by‑step)
1. When array is provided and not empty, infer the element type from an actual element: use type(array[0]) (or scan to find a non-None element if elements can be None).
   - Replace line 335 with something like:
     if array is not None and len(array) > 0:
         inferred_type = type(array[0])
2. If the array is provided but empty, fall back to element or expected_type:
   - elif element is not None:
         inferred_type = type(element)
   - elif expected_type is not None:
         inferred_type = expected_type
   - else:
         inferred_type = object
3. Adjust the check at line 346 to compare types appropriately. For example:
   - if inferred_type is object and expected_type not in (None, object): raise TypeError(...)
   - Or use issubclass checks if expected_type is a class.
4. If list elements may be instances of subclasses, consider using isinstance/issubclass instead of equality on types.
