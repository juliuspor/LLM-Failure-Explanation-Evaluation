What happened
- In get_paint the code clamps the input into v:
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
  but then computes g using value instead of v:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- Because value may be outside the bounds, that expression can produce a negative number or a number >255, so the subsequent check raises ValueError even though you already clamped into v.

Why it fails (root cause)
- The clamp is performed on v, but the clamped variable is never used. The code uses the original (possibly out-of-range) value when computing g. That mismatch between clamping and calculation is the direct cause of the failure.

Exactly where it happens
- In get_paint:
  1. The two lines that set v clamp the input:
     v = max(value, self._lower_bound)
     v = min(v, self._upper_bound)
  2. Immediately after, this line uses the wrong variable:
     g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  The error (ValueError) is raised because g is out of [0,255] when value is out of bounds.

Concrete fixes (step-by-step)
1. Use the clamped variable v when computing g. Replace the g line with:
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

2. Guard against zero range (division by zero) if upper and lower bounds can be equal:
   if self._upper_bound == self._lower_bound:
       g = 0  # or 255 or some defined default
   else:
       g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

3. Improve the error message or remove the manual range check if you guarantee v is in range. If you keep it, include actual values for easier debugging:
   if g < 0 or g > 255:
       raise ValueError(f"Color parameter outside of expected range: g={g}, value={value}, v={v}")

Minimal corrected version
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)
        
        # Avoid division by zero
        if self._upper_bound == self._lower_bound:
            g = 0
        else:
            g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
        
        if g < 0 or g > 255:
            raise ValueError(f"Color parameter outside of expected range: g={g}, value={value}, v={v}")
        
        return (g, g, g)

That change uses the clamped variable v for the calculation and prevents the spurious ValueError.
