What went wrong (short): the code calls array[i].upper() inside the loop. That assumes each element is a string and will raise AttributeError for None (and for other non-string elements). The AttributeError is the failure your tests exercise.

Where it happens (exact spot):
- Inside the function to_class, in the for loop:
  - Problem line: `_ = array[i].upper()`
  - This is the first statement inside the loop that runs for each element (the loop starts `for i in range(len(array)):`).

Why it fails (what happens and why):
- When array contains None (e.g. the test passes ["Test", None, 99.0]) the evaluation of `array[i].upper()` on the second element attempts to call .upper on None.
- None has type NoneType and does not have an attribute upper, so Python raises:
  AttributeError: 'NoneType' object has no attribute 'upper'
- The line `_ = array[i].upper()` is unnecessary for computing the element types. It both causes the crash and wrongly assumes all elements are strings. Any non-string (None, int, float, list, etc.) will break.

What you should change (concrete steps):
1. Remove the call to .upper() entirely.
2. Use type(...) (or elem.__class__) to get the element's class, and handle None normally — type(None) gives the NoneType you expect in the tests.
3. Simplify the loop to iterate elements directly instead of indexing.

Fixed implementation (replace the body of the function with this):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
    
        classes = []
        for elem in array:
            classes.append(type(elem))
        return classes

Why this fix works:
- It no longer calls any method on array elements, so None and other non-string elements do not cause AttributeError.
- type(elem) returns the correct class for each element including None (type(None)), matching your tests.
- Iterating over elements directly is clearer and avoids indexing mistakes.
