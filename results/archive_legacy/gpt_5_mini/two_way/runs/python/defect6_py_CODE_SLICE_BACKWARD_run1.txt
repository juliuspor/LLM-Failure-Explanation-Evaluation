What happened (where and what error)
- The failure happens at line 176:
  if x == int(x) and not math.isinf(x) and not math.isnan(x):
- When x is float('nan') or float('inf') the code still tries to evaluate int(x) before the isnan/isinf checks because Python evaluates the left side of the and first.
- int(float('nan')) raises ValueError ("cannot convert float NaN to integer") and int(float('inf')) raises OverflowError. That exception aborts the function — the later checks (math.isinf, math.isnan) never run because the int() call already failed.

Why that causes the crash (root cause)
- The boolean expression order is the root cause. The code asks for int(x) before ensuring x is a finite, non-NaN float, so conversion can raise an exception.

Exact places to change (concrete fixes)
1) Short-circuit the checks first (preferred minimal change)
   - Move the isnan/isinf checks before int(x) so int() is never called for NaN/Inf:
     if not math.isinf(x) and not math.isnan(x) and x == int(x):
   - Python evaluates left-to-right and will skip x == int(x) when x is NaN/Inf.

2) Or use try/except around int conversion
   - Wrap int(x) in try/except to catch ValueError/OverflowError and treat them as "not integer":
     try:
         int_x = int(x)
     except (ValueError, OverflowError):
         int_x = None
     if int_x is not None and x == int_x:
         ...

3) Or explicitly test finite-ness with math.isfinite (shortest)
   - Use math.isfinite to combine the two checks:
     if math.isfinite(x) and x == int(x):
     (math.isfinite returns False for NaN and +/-inf)

Recommended change (exact code)
Replace the current line 176 with either:
- if math.isfinite(x) and x == int(x):
or
- if not math.isinf(x) and not math.isnan(x) and x == int(x):

Either fix prevents int(x) from being evaluated for NaN/Inf and stops the exception.
