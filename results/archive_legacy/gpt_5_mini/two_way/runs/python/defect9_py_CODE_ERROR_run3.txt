What happened
- The TypeError comes from calling len(...) on a None element inside the sizing loop:
  for i in range(search_length):
      greater = len(replacement_list[i]) - len(search_list[i])
  If either replacement_list[i] or search_list[i] is None, len(None) raises TypeError: object of type 'NoneType' has no len().

Where it occurs
- The failing line is the unguarded sizing loop (the line marked with the BUG comment). That loop runs before the later checks that skip None/empty values, so it can hit None elements even when the rest of the code would otherwise skip those entries.

Why it happens (root cause)
- The code allows None values inside search_list and replacement_list, and elsewhere you explicitly check for None before using an element (for example inside the "Find the first match" and subsequent search loops). But the sizing loop does not check elements for None before calling len(...). So if any element is None, calling len(search_list[i]) or len(replacement_list[i]) raises the TypeError.

Minimal reproduction
- Example that triggers the error:
  text = "abc"
  search_list = ["a", None]
  replacement_list = ["x", "y"]
  The sizing loop will call len(search_list[1]) which is len(None) -> TypeError.

How to fix it (concrete steps)
1. Only compute lengths when both entries are non-None and search_list[i] is not empty:
   - Add a guard in that loop identical to the guard used later:
     if search_list[i] is None or len(search_list[i]) == 0 or replacement_list[i] is None:
         continue
2. Then compute greater safely:
     greater = len(replacement_list[i]) - len(search_list[i])
     if greater > 0:
         increase += 3 * greater
3. (Optional) Keep the existing cap:
     increase = min(increase, len(text) // 5)

Suggested replacement for the buggy loop
Replace the sizing loop with:
    for i in range(search_length):
        if (
            search_list[i] is None
            or len(search_list[i]) == 0
            or replacement_list[i] is None
        ):
            continue
        greater = len(replacement_list[i]) - len(search_list[i])
        if greater > 0:
            increase += 3 * greater

Why this fix works
- It matches the same null/empty checks you already use elsewhere in the function, preventing len(...) from being called on None and making the sizing step null-safe.

That resolves the TypeError and keeps the heuristic sizing behavior intact.
