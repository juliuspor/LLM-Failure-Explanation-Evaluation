The original implementation would raise a ValueError when `x` is NaN or Infinity because it tries to evaluate `int(x)` before checking for those special values. The fix reorders the condition to use `math.isfinite(x)` before the `x == int(x)` check. Additionally, using `self.add(' ')` to insert a space can inadvertently trigger `maybe_end_statement`, possibly adding an unwanted semicolon; replacing it with `self.append(' ')` directly emits the needed whitespace without side effects.
