I identified two bugs in the `add_number` method: (1) The condition `x == int(x)` fails for negative zero (-0.0) because `int(-0.0)` returns `0`, making the comparison `False` when it should be `True` for integer-like values. (2) For negative floats like -4.5, the same condition incorrectly evaluates to `True` (since -4.5 == int(-4.5) is -4, which is -4.0 != -4.5), but when x < 0 and x == int(x), the code converts using `int(x)` which loses the sign. The fix adds an explicit check for negative zero using `math.copysign` and uses `int(x) == x` which correctly handles both negative zero and avoids the precision issue with negative floats.
