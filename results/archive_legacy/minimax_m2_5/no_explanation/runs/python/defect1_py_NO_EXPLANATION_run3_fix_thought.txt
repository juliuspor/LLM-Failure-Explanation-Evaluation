The main bug is in the minutes calculation. When hours_offset is negative (e.g., -5 for UTC-05), the code incorrectly subtracts minutes_offset from hours_in_minutes instead of adding them. For UTC-05:30, this gives -300 - 30 = -330 (correct), but for negative minutes it would be wrong. Also, when hours_offset is negative and minutes_offset is positive, the logic fails: for UTC+05:30, hours_in_minutes = 300, then 300 + 30 = 330 (correct), but for UTC-05:30: hours_in_minutes = -300, then -300 - 30 = -330 (accidentally correct but wrong logic). The fix: calculate total_offset_minutes as hours_offset * 60 + minutes_offset consistently, then validate the range properly.
