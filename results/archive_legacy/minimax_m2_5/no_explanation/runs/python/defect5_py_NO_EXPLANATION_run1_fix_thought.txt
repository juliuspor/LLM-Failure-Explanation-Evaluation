The bug is in the type inference logic. When `array` is None but `element` is not None, the code correctly infers the type from the element. However, when `array` is an empty list (not None), the code incorrectly uses `type(array)` which returns `list`, not the element type. This causes incorrect behavior because the check `inferred_type == object` fails for empty lists even when it should pass (since an empty list should be treated similarly to None). The fix is to check if the array is empty (len == 0) and treat it like None for type inference purposes.
