The main bug in replace_each is that after processing a match and doing a replacement, the code fails to mark that specific search string as potentially having more matches. When we find a match for search_string[replace_index] and do the replacement, we set no_more_matches_for_repl_index[replace_index] = False so that in subsequent iterations we can still find more matches of that search string. Also, the buffer size calculation incorrectly limits the increase using min(increase, len(text) // 5), which can cause issues. The buffer estimation should use the actual occurrence count rather than a fixed multiplier.
