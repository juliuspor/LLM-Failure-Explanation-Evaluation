The bug is in the final for loop of the translate method. The loop incorrectly uses `pos` both as the index to read the codepoint AND increments it within the same iteration, causing incorrect advancement. After calling translate_codepoint which returns the number of codepoints consumed, the code should iterate through each consumed codepoint, read it at the current position, and then advance by that codepoint's char count. The current code calls code_point_at(code_units, pos) but pos keeps changing within the loop, leading to reading the wrong position or going out of bounds.
