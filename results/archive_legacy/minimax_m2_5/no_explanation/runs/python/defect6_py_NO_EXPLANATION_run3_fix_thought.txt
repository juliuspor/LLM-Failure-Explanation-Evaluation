I found two bugs in the add_number function: (1) The check `x < 0` for detecting negative numbers doesn't handle negative zero (-0.0) correctly since -0.0 is not less than 0 in Python. The class already has an `is_negative_zero` method that should be used. (2) The condition `x == int(x)` can have floating-point precision issues. A more robust approach is to check if the float is equal to its integer value after truncating towards zero.
