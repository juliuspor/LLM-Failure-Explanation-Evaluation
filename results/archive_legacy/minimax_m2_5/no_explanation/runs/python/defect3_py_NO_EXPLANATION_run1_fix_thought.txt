The bug is in the loop that advances the position after a translator consumes codepoints. The current code incorrectly reuses `pos` as the index when calculating char_count, causing it to read from the wrong position after the first iteration. After `pos` is incremented in the first iteration of the loop, subsequent iterations read from the new position instead of continuing to read from the original consumed codepoints. The fix is to track the advancement separately and use `pos + advancement` as the index when reading codepoints.
