The original implementation did not cap the `lower` bound to the length of the input string. When `lower` was larger than `len(text)`, the subsequent adjustment `if upper < lower: upper = lower` could set `upper` to a value greater than `len(text)`, causing `substring_java` to raise an IndexError. The fix adds a check to limit `lower` to `len(text)` (and optionally also ensures it is not negative). The rest of the logic remains unchanged, preserving the existing behavior for all other cases.
