**What Happened:**

The test code created a `GrayPaintScale` instance with default bounds of 0.0 (lower) and 1.0 (upper). It then called `get_paint(-0.5)`, expecting the output to be `(0, 0, 0)`.

**Why It Happened:**

The method `get_paint` does not correctly clamp the value to the lower bound before calculating the grayscale color. Instead, it directly computes `g` using the `value - self._lower_bound` calculation, which results in a negative value for `g` when the input `value` is less than `lower_bound`. Consequently, `g` is calculated as `(-0.5 - 0.0) / (1.0 - 0.0) * 255.0 = -127`, leading to a `ValueError` since `g` is outside the valid range 0-255.

**Where It Occurred:**

The issue is in the `get_paint` method within the line calculating `g`:
```python
g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
```

**Fix:**

1. Change the calculation of `g` to use the already clamped value `v` instead of `value`:

   ```python
   v = max(value, self._lower_bound)
   v = min(v, self._upper_bound)
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
   ```

By using `v` instead of `value`, the calculation will ensure `g` is within its valid range when values below `lower_bound` or above `upper_bound` are given. The corrected code will allow the test to pass, returning `(0, 0, 0)` when `value` is below `lower_bound`.
