This code problem arises from a bug in the `get_paint` method of the `GrayPaintScale` class. Here's a simple explanation:

### What Happened:
- The method `get_paint` is supposed to return a color (a shade of gray) based on an input value. This shade is represented as an RGB tuple where the red, green, and blue values are all equal.

### Why It Happened:
- The method intends to first "clamp" the input value, ensuring it's within the range defined by `lower_bound` and `upper_bound`.
- Clamping should adjust values that are out of range to the nearest bound value.
- The bug is on **line 73**. Instead of using the clamped value `v`, the code incorrectly uses the original `value` that hasn't been adjusted.
- This means that when `value` is less than `lower_bound`, the calculation produces a negative number for `g`, which is not valid for a color component.

### Where It Occurred:
- **Line 73** in the `get_paint` method is the specific line where the wrong variable (`value` instead of `v`) is used for calculating `g`.

### How to Fix It:
1. Change line 73 to use the variable `v` instead of `value`:
   ```python
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
   ```

By making this change, you ensure that calculations are done with the clamped value, which will prevent negative RGB outputs, resolving the "Color parameter outside of expected range" error.
