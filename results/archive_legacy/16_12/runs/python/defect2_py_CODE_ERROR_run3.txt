This code is supposed to create a scale of gray shades based on a numeric value. However, it raises a `ValueError` with the message, "Color parameter outside of expected range: Red Green Blue." This error occurs in the `get_paint` method due to incorrect calculation of the gray color value based on the input.

**What happened:**
- The method `get_paint` is supposed to map a numeric value to a shade of gray. It does this by computing an integer `g` related to the intensity of gray.

**Why it happened:**
- The variable `g` is calculated using the formula:
  ```python
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
  ```
  `value`, however, is not clamped correctly before this calculation; the initial clamping is done on a separate variable `v`, but `g` is calculated with the original `value` variable. If `value` is outside the defined bounds, this calculation can lead to `g` being calculated as a number below 0 or above 255.

**Where it occurred:**
- Line 55 in the `get_paint` method when calculating `g`.

**Steps to fix:**
1. Replace `value` with `v` for the calculation of `g` in the `get_paint` method so that it uses the clamped value:
   ```python
   g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
   ```
2. This will ensure `v` is always within the range of `self._lower_bound` and `self._upper_bound` before calculating `g`, so `g` will always be between 0 and 255.
