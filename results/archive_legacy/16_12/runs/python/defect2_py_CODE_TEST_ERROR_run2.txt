The code fails in the `get_paint` method of the `GrayPaintScale` class. When you call `get_paint` with the value `-0.5`, the method is intended to clamp this value to the lower bound, which is `0.0`. However, the variable `g` is calculated using the original `value` (`-0.5`), not the clamped value:

```python
g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
```

The problem is that the clamping logic is applied to `v`, but `v` is not used in calculating `g`. Consequently, when `value` is `-0.5`, the calculation results in a negative number, specifically `-127`, which falls outside the valid RGB range of `0-255`. This triggers the `ValueError`.

### What happened?
The program attempted to convert a negative value to a color, which led to a number outside the valid RGB range.

### Why did it happen?
The method `get_paint` used the original `value` instead of the clamped `v` for its calculation.

### Where did it occur?
This error is located in the `get_paint` method, at the line where `g` is calculated.

### How to fix it
Replace `value` with `v` in the calculation of `g` to ensure that the clamped value is used:

```python
def get_paint(self, value: float) -> Tuple[int, int, int]:
    # Clamp value to bounds
    v = max(value, self._lower_bound)
    v = min(v, self._upper_bound)
    
    # Use the clamped value for calculation
    g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
    
    if g < 0 or g > 255:
        raise ValueError("Color parameter outside of expected range: Red Green Blue")
    
    return (g, g, g)
```

This change ensures that all calculations are based on values within the allowed range, preventing invalid RGB values.
