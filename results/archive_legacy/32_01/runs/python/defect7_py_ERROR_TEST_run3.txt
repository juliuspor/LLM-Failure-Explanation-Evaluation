What happened (root cause)
- The failure comes from ClassUtils.to_class trying to call .upper() on a value that is None. None does not have an upper() method, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'.
- This occurs inside ClassUtils.to_class on the line that does value.upper() (the code path that maps a string name like "Test" to a class). Because the method does not check the input type before calling .upper(), a None value in the input list triggers the exception.

Why it happens (step-by-step)
1. Your test calls ClassUtils.to_class(['Test', None, 99.0]).
2. to_class iterates the list and, for each element, assumes it can call .upper() to normalize string names (e.g., value.upper()).
3. For the second element (None), the code attempts None.upper(), which raises AttributeError immediately.
4. Because there is no branch that checks for None (or non-string values) before calling .upper(), the method never reaches the logic that would return type(None) for None or float for 99.0.

Where exactly (method + line)
- In the ClassUtils.to_class method — specifically on the line that calls value.upper() to normalize string inputs. That is the line causing the AttributeError when value is None.

How to fix it (concrete steps)
Option A — explicit type checks (recommended)
1. In ClassUtils.to_class, before calling value.upper(), check the value’s type.
2. If value is None, append type(None) (or NoneType) to the result.
3. If value is a str, call value.upper() and map it to the class.
4. Otherwise handle other types (e.g., numbers) by using type(value).

Example patch (pseudo-code of the changes to make inside to_class):
- Replace:
    normalized = value.upper()
    ...map normalized to class...
  With:
    if value is None:
        result.append(type(None))
        continue
    if isinstance(value, str):
        normalized = value.upper()
        ...map normalized to class...
        continue
    # non-string objects
    result.append(type(value))

Option B — permissive try/except
1. Wrap the value.upper() call in try/except AttributeError and fall back to type(value) if it fails.
2. That will avoid the crash but is less explicit than type checks.

Example patch (pseudo-code):
    try:
        normalized = value.upper()
    except AttributeError:
        result.append(type(value))

    ...map normalized to class...
