What happened
- The test calls DateTimeZone.for_offset_hours_minutes(-2, -15) expecting "-02:15".
- The factory computes the total offset by combining hours and minutes, but the implementation first takes the absolute value of the minutes (or otherwise forces minutes to be positive) before combining them.
- As a result the code treats (-2, -15) like (-2, +15), producing -01:45 (or some other wrong offset) instead of -02:15, so the assertion fails.

Where it occurs
- In DateTimeZone.for_offset_hours_minutes (the factory method you're calling).
- Look for a line that normalizes the minutes with an absolute-value or sign-forcing operation, e.g.:
  minutes = Math.abs(minutes);
  or
  minutes = minutes < 0 ? -minutes : minutes;
  or any place that discards the sign of minutes before adding them to hours.

Why that causes the failure
- The correct total offset is (hours * 60) + minutes minutes. For (-2, -15) that is -120 + (-15) = -135 minutes = -02:15.
- If minutes are made positive before combining, the code computes -120 + 15 = -105 minutes = -01:45 (wrong).
- The bug is an incorrect handling of the minutes sign; the code throws away the intended negative part of minutes when hours are negative.

How to fix it (concrete steps)
1. In DateTimeZone.for_offset_hours_minutes, stop forcing minutes to be positive. Remove any Math.abs(minutes) or similar normalization that discards the sign.
2. Compute the total offset in minutes (or seconds/milliseconds) as a single operation:
     totalMinutes = hours * 60 + minutes
   (do this with integers).
3. Validate ranges against requirements using totalMinutes:
   - hours in [-23, 23] and minutes in [-59, 59] (keep the existing per-argument checks if you want),
   - and ensure absolute total offset does not exceed 23:59:59.000 (or <= 23*60 + 59 minutes).
4. Build the zone id from totalMinutes:
   sign = totalMinutes < 0 ? '-' : '+'
   absMinutes = Math.abs(totalMinutes)
   idHours = absMinutes / 60
   idMinutes = absMinutes % 60
   id = String.format("%s%02d:%02d", sign, idHours, idMinutes)
5. Add unit tests:
   - (-2, -15) -> "-02:15"
   - (-2, 15)  -> "-02:15"
   - (2, -15)  -> should raise ValueError (per doc) — make sure validation enforces that
   - (0, -15)  -> "-00:15"

Minimal code example (Java-like pseudocode)

    totalMinutes = hours * 60 + minutes;
- Good:
