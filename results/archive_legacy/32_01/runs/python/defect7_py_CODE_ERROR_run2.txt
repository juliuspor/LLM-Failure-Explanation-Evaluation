What happened (where it failed)
- The exception occurred inside the to_class method, on the line:
    _ = array[i].upper()
- Python raised AttributeError: 'NoneType' object has no attribute 'upper' because array[i] was None at runtime and you attempted to call .upper() on it.

Why it happened (root cause)
- The code assumes every element of array is a string (or at least has an upper() method). That assumption is false: the list contains at least one None.
- Calling None.upper() triggers the AttributeError. The offending call is executed before you get the element’s type (you then call type(array[i]) afterwards), so the failure happens on that first access.

Where exactly in the code
- In to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
  - After handling array is None and empty, inside the for loop:
    for i in range(len(array)):
        _ = array[i].upper()     <- raises AttributeError when array[i] is None
        classes.append(type(array[i]))

How to fix it (concrete steps, multiple explicit options)
Pick the behavior you want and apply one of these fixes:

1) If the list should never contain None — validate input and raise a clear error
- Add an explicit check and raise ValueError with a clear message:
    for i in range(len(array)):
        if array[i] is None:
            raise ValueError(f"to_class: array[{i}] is None; expected a string or object")
        _ = array[i].upper()
        classes.append(type(array[i]))

2) If None is allowed and should be skipped
- Skip None elements:
    for item in array:
        if item is None:
            continue
        _ = item.upper()
        classes.append(type(item))

3) If you only need the element’s class and don’t need upper()
- Remove the unnecessary .upper() call entirely:
    for item in array:
        classes.append(type(item))
  (This is the simplest if you never need the uppercase string.)

4) If you intended to operate on strings but elements may not be strings
- Check type before calling upper(), and handle non-strings:
    for i, item in enumerate(array):
        if item is None:
            classes.append(type(item))
            continue
        if isinstance(item, str):
            _ = item.upper()
        else:
            # handle or log non-string; here we just note its class
            pass
        classes.append(type(item))

Notes about other code points
- classes = [] then returning classes is fine; consider typing: Optional[List[Type]] is OK.
- ArrayUtils.EMPTY_CLASS_ARRAY.copy() is returned for empty input — that’s OK if it’s defined elsewhere.
- Using type(array[i]) is correct to get element class; if you wanted class name use type(item).__name__ or item.__class__.

Summary
- The immediate cause is calling .upper() on a None value at the line "_ = array[i].upper()".
- Fix by either removing the .upper() call, adding a check for None, enforcing the list contains only strings, or otherwise handling non-string elements as appropriate for your function’s intended behavior.
