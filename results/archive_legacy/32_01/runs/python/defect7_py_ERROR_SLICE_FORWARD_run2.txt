What happened (root cause)
- The code attempted to call .upper() on a value that was actually None. None has no .upper method, so Python raised AttributeError: 'NoneType' object has no attribute 'upper'.
- That None came from your array: at line 770 you do classes.append(type(array[i])). If array[i] is None, type(array[i]) is type(None). Later code expects that array element to be a string and calls array[i].upper(), causing the exception.

Where it occurred
- Location shown by your slice: line 770, in the loop that inspects array elements:
  770 |             classes.append(type(array[i]))
  That line documents that array[i] can be None. The AttributeError occurs later at whatever line calls array[i].upper() (search for ".upper(" to find the exact line). The problem is that an element in array is None when the code assumes it is a string.

Why it happened (likely causes)
- Upstream code produced a missing value (returned None) for at least one slot in array. Common causes:
  - A function that is supposed to return a string returned None in some branch.
  - Data parsing or lookup failed and returned None instead of '' or a default string.
  - An optional field was left empty and not converted to a string before use.
