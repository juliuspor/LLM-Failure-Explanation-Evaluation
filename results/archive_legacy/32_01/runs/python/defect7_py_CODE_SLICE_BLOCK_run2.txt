Why that happens (root cause)
- The code assumes every element in array has an .upper() method. That is true for str but not for most other types (including type/class objects). The next line, classes.append(type(array[i])), also suggests a mismatch between intent and implementation:
  - If array elements are strings representing class names, calling .upper() makes some sense but appending type(array[i]) is wrong (type("Foo") is str).
  - If array elements are already class objects, .upper() is wrong because classes don’t have upper(), and you should append the class itself, not type(class) (which would be type).
- In short: the code mixes up string names and class objects and calls a string method on a non-string value at line 769.
