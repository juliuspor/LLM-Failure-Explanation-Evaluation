What happened (what failed)
- The test called ClassUtils.to_class(['Test', None, 99.0]).
- Inside that method the loop at SLICE_UNION lines 768–770 runs:
  - line 769 calls array[i].upper()
  - line 770 then appends type(array[i])
- The code crashes on the second element (None) when array[i].upper() is attempted, because None has no upper() method. The exception is raised at line 769 and the loop never reaches line 770 for the remaining items.

Why it failed (root cause)
- The code assumes every element in array is a string and unconditionally calls .upper().
- For non-string values (None, float, int, etc.) calling .upper() raises AttributeError, so the function aborts before it can append the element’s type.
- In short: calling array[i].upper() on a non-string value caused the error; that call is the single point of failure.

     - if isinstance(array[i], str):
         _ = array[i].upper()
     - classes.append(type(array[i]))
   - Effect: strings are uppercased, other types are left alone and their types are appended.

2) Use try/except around the .upper() call
   - Wrap the call so non-strings don’t crash the loop:
     - try:

   - Effect: any object without upper() is skipped but its type is still appended.

3) Avoid calling .upper() entirely if you only need the type
   - If the only goal is to collect types, remove the .upper() call:
     - classes.append(type(array[i]))
   - Effect: simpler, no risk of AttributeError.
