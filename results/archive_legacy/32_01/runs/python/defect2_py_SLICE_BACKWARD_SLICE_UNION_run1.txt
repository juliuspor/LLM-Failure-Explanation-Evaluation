What happened (where it fails)
- The code fails inside get_paint at line 73:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If self._upper_bound == self._lower_bound that expression divides by zero and Python raises ZeroDivisionError. That is the immediate crash you will see at runtime.

Why it happens (root cause)
- The expression assumes the denominator (self._upper_bound - self._lower_bound) is nonzero. If the bounds are equal (either because they were set that way or not initialized properly), the denominator becomes 0 and the division is invalid. The code has no pre-check for this case, so the ZeroDivisionError is unhandled and the function fails.

Other problems to fix while you're here

It also raises a vague ValueError with the literal text "Color parameter outside of expected range: Red Green Blue" — that message doesn't say which component or what values caused the problem.
- Converting to int after the computation can hide whether the raw normalized value was out of range; you may want to clamp or validate before int().
