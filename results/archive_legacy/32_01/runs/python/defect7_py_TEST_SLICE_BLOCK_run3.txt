What happened (where it failed)
- The test calls ClassUtils.to_class(['Test', None, 99.0]).
- The failure happens in the loop at line 769: array[i].upper() is executed for every element.
- When i points to None or 99.0, calling .upper() raises an AttributeError because NoneType and float do not have an upper() method. This exception occurs before the method reaches the classes.append(...) at line 770, so the list of types is never built and the assertion fails.

Why it fails (root cause)
- The code assumes every element in array is a string and blindly calls .upper() on each element. That assumption is false for None and float.
- The root cause is trying to call a string method on non-string objects without checking their type or handling the error.

Exact location
- ClassUtils.to_class: line 769 calls array[i].upper() and causes the exception. Line 770 (classes.append(type(array[i]))) is never reached for the non-string elements because of the exception raised at 769.

Concrete fixes (steps you can apply)
1. Make the code check the element type before calling .upper():
   - Replace line 769 with a type check:
     if isinstance(array[i], str):
         _ = array[i].upper()
   - Then line 770 can remain to append the type for any element.

2. Or use duck-typing safely by checking for the attribute before calling:
   - Replace line 769 with:
     if hasattr(array[i], 'upper'):
         _ = array[i].upper()

3. Or catch the specific exception around the call:
   - Surround the call with try/except:
     try:
         _ = array[i].upper()
     except AttributeError:
         pass

4. Ensure None becomes type(None) in the result and float becomes float:
   - After applying one of the above, the loop will reach line 770 and append type(array[i]) producing [str, type(None), float] as expected.

Recommended minimal change (explicit, safe)
Replace lines 768–770 with:
for i in range(len(array)):
    if isinstance(array[i], str):
        _ = array[i].upper()
    classes.append(type(array[i]))

This fixes the root cause by only calling .upper() on strings and still appending the correct types for None and numeric values.
