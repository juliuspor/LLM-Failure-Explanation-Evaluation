Why it failed (root cause, plain language)
- The code always calls array[i].upper() for every element regardless of that element’s type.
- .upper() is a string method. If array[i] is None (NoneType) or a number (float), that attribute does not exist and Python raises AttributeError.
- In your test, the first element ('Test') works, but the second element None causes array[1].upper() to raise the error before the function can append any more types. The call to .upper() is unnecessary for the stated goal (returning the types of the elements), and it makes the routine crash on non-string elements.

1. Remove the unnecessary call to .upper() unless you actually need to normalize strings first. For simply returning types, you do not need .upper().
   - Change lines 8–10 to:

         classes.append(type(array[i]))
2. If you do need to call .upper() only for strings, guard it with a type check:

             _ = array[i].upper()
3. If you want to treat None as type(None) and accept any object, prefer a simple, clearer iteration:
   - Example replacement for the whole loop:

4. If you want to get the runtime type name regardless and avoid exceptions, you can use a try/except:

             classes.append(type(item))

Why that fix works
- It never calls .upper() on non-strings, so no AttributeError for None or floats.
- It directly returns the types you expect: [str, type(None), float] for the test input.

Summary
