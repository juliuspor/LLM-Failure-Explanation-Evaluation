What goes wrong (root cause)
- The method that builds a fixed-offset zone (commonly named something like forOffsetHoursMinutes, ofHoursMinutes, createOffset, etc.) converts the hour and minute inputs into a single signed offset (usually minutes or seconds).
- The bug is an incorrect sign/validation rule for minutes: the code rejects any negative minutes when the hours value is positive by doing a straight sign comparison between hours and minutes (for example, if (hours > 0 && minutes < 0) throw; — or worse, if (sign(hours) != sign(minutes)) throw).
- That simple sign-comparison also incorrectly rejects the valid case where both hours and minutes are negative (e.g. (-2, -15) should produce -02:15). It also mishandles the zero-hour cases if the implemented check assumes hours’s sign alone determines the offset sign.
- In short: the code is using the wrong rule to determine the final sign of the offset and to validate minute sign, so valid combinations get thrown as invalid.

Where to look in your code
- Open the method that creates the offset zone (look for names like forOffsetHoursAndMinutes, forOffsetHoursMinutes, create, ofHoursMinutes, or similar).
- Inspect the block that validates hours and minutes and the block that converts them to a total offset (minutes or seconds). Typical culprits:
  - A check like if (hours > 0 && minutes < 0) throw ValueError
  - A check like if (sign(hours) != sign(minutes)) throw ValueError
  - A conversion like totalMinutes = hours * 60 + minutes without handling negative hours correctly

Concrete fix (exact steps)
1. Validate ranges independently:
   - require -23 <= hours <= +23
   - require -59 <= minutes <= +59
   - (optional) require that the absolute total offset does not exceed 23:59
2. Compute the final signed offset using the sign implied by the inputs, not by naive sign-matching:
   - Let absHours = abs(hours), absMinutes = abs(minutes)
   - Determine sign:
     - If hours < 0 OR (hours == 0 AND minutes < 0) then sign = -1
     - Else sign = +1
   - Compute totalMinutes = sign * (absHours * 60 + absMinutes)
3. Validate total magnitude:
   - If abs(totalMinutes) > (23 * 60 + 59) then throw ValueError
4. Use totalMinutes (or totalSeconds = totalMinutes * 60) to construct the DateTimeZone.
