What happened (where it fails)
- The exception comes from the to_class method, on this line inside the for loop:
  _ = array[i].upper()
- Python raised AttributeError: 'NoneType' object has no attribute 'upper' because array[i] was None for at least one element. None does not have an upper() method.

Why it happened (root cause)
- The code assumes every element in array is a string (so calling .upper() is valid). That assumption is false: the input list can contain None (or other non-string values), so calling .upper() on that element fails.
- The .upper() call appears unnecessary for computing classes anyway — the method is supposed to return types, and the code uses type(array[i]) immediately after. So the .upper() call serves no purpose other than causing the crash when a non-string or None is present.

Where exactly in the shown code
- Method: to_class
- Problem line: the statement "_ = array[i].upper()" inside the for loop (the third-to-last line in your snippet).
- The next line "classes.append(type(array[i]))" is safe for None (type(None) is a valid type), but it is never reached when .upper() raises.
