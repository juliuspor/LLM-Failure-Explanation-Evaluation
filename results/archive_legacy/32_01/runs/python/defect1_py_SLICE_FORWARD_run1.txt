Why it happens (root cause)
- There is a missing conditional branch between the two arithmetic lines. The code at line 406:

  is meant to run only for one sign/case; the code at line 408:
    minutes_offset = hours_in_minutes + minutes_offset
  is meant to run for the opposite sign/case. Because there is no else/elif, both lines run in sequence, so the second assignment overwrites the first and produces an incorrect total. That incorrect total then flows into:
    offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)
  and eventually into cls.for_offset_millis(offset). The ValueError at line 411 is the visible failure.
