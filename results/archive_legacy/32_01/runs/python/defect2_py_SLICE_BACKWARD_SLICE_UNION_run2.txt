What happened (where it fails)
- The code crashes at line 73 inside get_paint:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If self._upper_bound equals self._lower_bound, that expression performs a division by zero and raises a ZeroDivisionError. The traceback will point to line 73.

Why it happens (root cause)
- The denominator (self._upper_bound - self._lower_bound) can be zero. There is no check before performing the division, so a ZeroDivisionError occurs when bounds are equal.
- Secondary problem visible in the slice: only g is computed and validated (lines 73 and 76–77), but the ValueError message says "Color parameter outside of expected range: Red Green Blue". That message implies you intended to validate Red, Green and Blue channels, but you only check g. This makes the error message misleading and hides the real failure mode (division by zero) when it occurs.

How to reproduce quickly
- Call get_paint(value) when self._upper_bound == self._lower_bound (for example when both are 0). Example: if lower_bound = 0 and upper_bound = 0, any call to get_paint triggers ZeroDivisionError at line 73.
