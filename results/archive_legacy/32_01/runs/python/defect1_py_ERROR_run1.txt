What happened (in plain language)
- The runtime raised ValueError: Minutes out of range: -15 because some code called a date/time constructor or setter and passed a minute value of -15. Python’s date/time constructors (for example datetime.time(...), datetime.datetime(...), or pandas.Timestamp(...)) and methods like datetime.replace(minute=...) require minutes in 0–59. Passing -15 is invalid, so the constructor raises that error immediately.

Why it happened (root cause)
- A calculation produced a negative minute value and that raw value was passed directly into a date/time API that doesn’t accept negatives.
- Typical ways this happens:
  - You subtracted minutes by arithmetic (minute = current_minute - 15) and then used datetime.replace(minute=minute) without adjusting the hour/day.
  - You tried to normalize time components yourself instead of using datetime arithmetic and ended up with negative components.
  - You parsed or transformed a string or numeric input incorrectly so minutes became negative.

Concrete examples of offending code and exact fixes
- Offending pattern A (using replace with negative minute)
  - Code (line numbers hypothetical):
    - app.py:82: minute = now.minute - 15
    - app.py:83: new = now.replace(minute=minute)
  - Why it fails: now.replace(minute=-15) triggers ValueError.
  - Fix (use timedelta or normalize with divmod):
    - Replace those two lines with:
      - new = now - datetime.timedelta(minutes=15)
    - Or explicitly normalize:
      - total_minutes = now.hour * 60 + now.minute - 15
      - hour, minute = divmod(total_minutes, 60)
      - new = now.replace(hour=hour % 24, minute=minute)
    - Both avoid passing negative minute values to replace().

- Offending pattern B (constructing time directly)
  - Code (hypothetical):
    - t = datetime.time(hour, minute)  # minute computed earlier and is -15
  - Fix:
    - Use datetime.datetime and timedelta, or normalize minutes before constructing:
      - dt = datetime.datetime.combine(date.today(), datetime.time(hour, 0)) + datetime.timedelta(minutes=minute)
      - t = dt.time()

- Offending pattern C (manual arithmetic on multiple components)
  - Code (hypothetical):
    - minute = some_value - 15
    - result = datetime.datetime(year, month, day, hour, minute)
  - Fix:
    - Do datetime arithmetic instead: base_dt + timedelta(minutes=offset)
    - Or compute minutes and hours with divmod as shown above.
