What fails
- The failure happens inside to_class, on the loop line:
      _ = array[i].upper()
  This raises AttributeError whenever array[i] is not a str (for example if it is a class object, an int, None, etc.). The code assumes every element has an upper() method, which is not true.

Why it fails (root cause)
- The code is mixing two different kinds of input without checking them:
  - It calls array[i].upper() unconditionally (line in the for loop), which only exists on str objects.
  - Immediately after it appends type(array[i]) to classes, so it appears the intent was to convert input items to class objects — but calling upper() on the item is unrelated to producing a type and will break when the item is not a string.
- In plain terms: the code expects strings sometimes, but treats every element like a string; when an element is e.g. a class object or number, .upper() does not exist and the function crashes in the for loop.

Where it occurs
- Method: to_class
- Exact failing line: the call to array[i].upper() inside the for i in range(len(array)) loop.
- Likely exception message you would see: AttributeError: 'type' object has no attribute 'upper' (or "'int' object has no attribute 'upper'", depending on the element type).

- Replace the loop with:
      for item in array:

              # decide how to handle non-type items (raise or convert)

Notes about other lines

Summary
- Root cause: unconditional call to array[i].upper() on every element; that only works for strings and causes AttributeError for non-strings.
- Fix: only call .upper() when the element is a str, or remove it entirely and explicitly handle the expected input types (strings vs class objects), and resolve string names to classes using a mapping or importlib if needed.
