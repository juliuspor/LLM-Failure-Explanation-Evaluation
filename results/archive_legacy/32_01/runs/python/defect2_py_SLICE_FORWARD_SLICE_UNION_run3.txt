What happened (where it fails)
- The failure happens inside get_paint at line 73:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If self._upper_bound == self._lower_bound, that expression performs division by zero and Python raises a ZeroDivisionError. That is the direct cause of the crash.

Why it fails (root cause)
- The code assumes the denominator (self._upper_bound - self._lower_bound) is non‑zero. When lower and upper bounds are equal (e.g., both 0.0 or both 1.0), the denominator becomes 0.0 and the division is invalid.
- The later check on lines 76–77 (if g < 0 or g > 255: raise ValueError(...)) never prevents the division from happening — it only examines the result after the division. So it cannot stop the ZeroDivisionError.
- The provided error message ("Color parameter outside of expected range: Red Green Blue") is also unhelpful: it does not say which value or which bounds caused the problem.

2) Clamp computed channel values rather than raising after conversion

3) Provide a clear error message if you must raise
- If raising is required, include the parameter name, the value, and the bounds in the message so the caller knows exactly what failed.
