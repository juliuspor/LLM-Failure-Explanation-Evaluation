- That happens inside ClassUtils.to_class when the method assumes every input is a string and calls value.upper() without first checking for None (or for non-string types). So the method hit the second list element None and tried to call None.upper(), which raises the exception.

Where it occurred
- In the ClassUtils.to_class method, at the line that invokes value.upper() (the call that transforms / inspects the input string). That is the exact place the failure happens: the method does not check value is a str or non-None before calling .upper().
