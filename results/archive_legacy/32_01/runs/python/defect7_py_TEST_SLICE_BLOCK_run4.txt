What happened (where and what line caused the failure)
- The test calls ClassUtils.to_class(['Test', None, 99.0]).
- Inside ClassUtils.to_class the loop at lines 768–770 runs:
  - line 769 calls array[i].upper()
  - line 770 appends type(array[i])
- When i == 1 array[1] is None, so line 769 raises:
  AttributeError: 'NoneType' object has no attribute 'upper'
- The method never reaches line 770 for the None element (and would also raise for the 99.0 float), so the test fails.

Why it fails (root cause)
- The code assumes every element in array is a string and calls .upper() unconditionally.
- None and float do not have an upper() method, so calling array[i].upper() fails as soon as the loop sees a non-string element.
- In short: the code performs a string operation without checking the element type first.

Concrete fixes (exact changes to make in ClassUtils.to_class around lines 768–770)
1) Only call upper() for strings
- Replace the unconditional call with a type check:

  if isinstance(array[i], str):
      _ = array[i].upper()
  classes.append(type(array[i]))

2) Or use a safe attribute check if you need to accept any object with upper()
- Replace line 769 with:

  upper = getattr(array[i], 'upper', None)
  if callable(upper):
      _ = upper()
  classes.append(type(array[i]))

3) Or remove the upper() call entirely if it has no purpose for computing types
- If the method only needs the type, simply do:

  classes.append(type(array[i]))

Why these fixes work
