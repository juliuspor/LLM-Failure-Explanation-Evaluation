What happened
- The test calls ClassUtils.to_class(['Test', None, 99.0]) and expects [str, type(None), float].
- The call fails with an AttributeError raised from inside ClassUtils.to_class when the code tries to use a member on the None element.

Why it fails (root cause)
- Inside ClassUtils.to_class the implementation calls a method or attribute on each element, e.g. element.getClass() (Java style) or element.__class__.__name__ without first handling None.
- When the loop reaches the second item (None), that call tries to access an attribute on the None object and raises AttributeError. That is the immediate exception you see.
- The docstring is self-contradictory: one sentence says "If any of these objects is None, a None element will be inserted into the array." but the Raises section says "AttributeError: If any element is None". The test expects the None element to map to type(None), so the code is implemented incorrectly relative to the intended behavior.

Where it happens
- In the method ClassUtils.to_class, at the line that reads something like:
    elem.getClass()
  or
    elem.__class__
  (the exact expression used to obtain the class of each element). That expression is executed for every element in the input list; when elem is None, it triggers AttributeError.

Exactly what to change (step-by-step fixes)
1. Handle a None input array first:
   - At the top of ClassUtils.to_class, if array is None: return None
   - This implements the docstring line "This method returns None for a None input array."

2. In the loop over elements, avoid calling methods/attributes on None; explicitly detect None:
   - Pseudocode:
     if elem is None:
         classes[i] = type(None)   # or None if you want to insert None per docstring — test expects type(None)
     else:
         classes[i] = type(elem)

3. Replace any Java-style getClass() call with Python's type(...) if this is Python code:
   - Change elem.getClass() to type(elem) (after the None check).

4. Make the docstring and Raises clause consistent:
   - If desired behavior is to insert type(None) for None elements (as the test expects), update the docstring to say that and remove the "Raises: AttributeError: If any element is None" line.
   - If instead you intended to preserve None in the output array, make the loop set classes[i] = None and update the test accordingly.

Minimal code example (Python)
- Correct approach inside ClassUtils.to_class:
  def to_class(array):
      if array is None:
          return None
      classes = [None] * len(array)
      for i, elem in enumerate(array):
          if elem is None:
              classes[i] = type(None)
          else:
              classes[i] = type(elem)
      return classes

Why this fixes it
- It never calls an attribute on None, so no AttributeError occurs.
- It returns the expected [str, type(None), float] for the test input.
- It honors the "None input returns None" requirement and lets you make the docstring consistent with the implementation.
