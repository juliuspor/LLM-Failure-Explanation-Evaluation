What happened (where it failed)
- At lines 400–401 the code compares minutes_offset to integers:
  - 400: if minutes_offset < 0 or minutes_offset > 59:
  - 401:     raise ValueError(f"Minutes out of range: {minutes_offset}")
- The code crashes at that comparison with a TypeError when minutes_offset is not a number (for example None or a string like "30"). Python cannot evaluate expressions like "None < 0" or '"30" > 59', so the comparison itself raises TypeError before the ValueError on line 401 can be raised.

Why it happened (root cause)
- The root cause is that minutes_offset is sometimes not an integer (or another comparable numeric type). The code assumes minutes_offset is already a numeric value, so the inequality operators (<, >) are applied without checking that assumption.
- Because the type assumption is violated, the interpreter raises TypeError at the comparison expression on line 400, not the ValueError intended on line 401.
