What happened (where it failed)
- The code fails at to_class, inside the loop shown at lines 768–770:
  - Line 769: `_ = array[i].upper()` calls .upper() on the element.
  - If array contains anything that is not a string (for example a class, an int, or None), that element does not have an upper() method and Python raises AttributeError: 'X' object has no attribute 'upper'.
- The next line (770) then does `classes.append(type(array[i]))`, which suggests the method wants the element's type — so the call to .upper() is unnecessary and not used. Because .upper() is executed unconditionally, a non-string element will cause the whole function to crash before you get the type.

Why it happened (root cause)
- Calling .upper() on array elements assumes every element is a str. That assumption is false for the method’s apparent purpose (it returns types), so when an element is not a str, .upper() raises AttributeError. The problematic line is line 769 inside to_class.

How to fix it (concrete steps)
1. Remove the unnecessary call to .upper() if you do not need it:
   - Delete line 769 (`_ = array[i].upper()`).
2. If you intended to uppercase string elements for some reason, guard the call with a type check:
   - Use `if isinstance(array[i], str): _ = array[i].upper()` so non-strings are not touched.
3. If you meant to convert elements to strings and uppercase them, explicitly convert:
   - `_ = str(array[i]).upper()` (but only do this if that behavior is desired).
4. Optional cleaner loop (same behavior but clearer):
   - for item in array:
       classes.append(type(item))

Example corrected version (removing the problematic call):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for item in array:
            classes.append(type(item))
        return classes

This fixes the AttributeError by eliminating the unguarded .upper() call.
