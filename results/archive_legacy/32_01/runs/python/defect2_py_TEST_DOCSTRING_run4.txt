What happened
- The test calls GrayPaintScale().get_paint(-0.5) and expects (0, 0, 0).
- Instead get_paint raised a ValueError (or otherwise failed) because the implementation treats a negative input as producing RGB outside 0–255 and reports that as an error.

Where it happens
- In the GrayPaintScale class, in the get_paint method. The failure occurs at the validation/raise statement inside get_paint that checks computed RGB values and raises ValueError when any channel is outside 0–255.

Why it happens (root cause)
- The method currently enforces that inputs producing out-of-range RGB must cause an exception. A negative input like -0.5 produces a negative channel value, so the method hits the raise ValueError path instead of returning a clamped grayscale color.
- The test expects get_paint to produce a clamped result (0,0,0) for values below the allowed range, but the implementation enforces strict range checking and signals an error. This is a mismatch between test expectation and implementation behavior.

   - (B) clamp computed RGB into the 0–255 range and return (0,0,0) for -0.5 (preferred by the test).

If you choose option B (make the method return (0,0,0) for -0.5), do these changes in get_paint:

2. At the start of GrayPaintScale.get_paint, clamp the input value into the valid domain (example assumes value maps to 0..255):
   - value = max(0.0, min(1.0, value))  # if gray scale input is 0..1
   OR
   - if your method computes an integer channel from value, clamp the computed channel:
     r = max(0, min(255, computed_r))
     g = max(0, min(255, computed_g))
     b = max(0, min(255, computed_b))

3. Remove or change the raise ValueError branch that triggers for out-of-range RGBs; replace it with clamping if you want no exception. If you must preserve validation elsewhere, raise only for inputs that are clearly invalid (e.g., wrong type).

4. Update the docstring to match the behavior:
   - If clamping: “Returns a tuple (r, g, b) representing the grayscale color. Values outside the range are clamped to 0–255.”

5. Add/modify tests:

   - For exception behavior: change test to assert that get_paint(-0.5) raises ValueError.
