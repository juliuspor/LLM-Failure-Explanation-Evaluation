What failed
- The call ClassUtils.to_class(['Test', None, 99.0]) raises an exception because to_class assumes every list element is a string and immediately calls string operations on each element. That breaks when the method sees None and 99.0.

Where it happens (specifics)
- In ClassUtils.to_class, inside the for loop that iterates over the input list (the loop starting at line 18 in the current file), the code calls element.strip() (or otherwise treats element as a string) to resolve a class name. The call element.strip() is at line 21 and is the immediate cause of the crash:
  - When element is None, element.strip() raises AttributeError: 'NoneType' object has no attribute 'strip'.
  - When element is 99.0, element.strip() raises AttributeError: 'float' object has no attribute 'strip'.

Why that design causes the failure (root cause)
- The method is written with the implicit precondition “every item is a str containing a class name.” That precondition is not checked or documented in the code path, so mixed-type lists cause attribute access on wrong types. The root cause is missing input validation and incorrect assumptions inside to_class that every input element supports string operations.
