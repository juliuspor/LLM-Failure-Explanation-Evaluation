Why the code was written incorrectly
- The .upper() call is unnecessary for determining the type. The method's purpose (based on name and the append at line 10) is to return types, but converting to uppercase is unrelated and unsafe for non-strings.
- There is no guard/check that array[i] is a string before calling .upper(), so any non-string (None, float, int, custom objects) will cause a runtime exception.

Consequence in your test
- result = ClassUtils.to_class(['Test', None, 99.0]) fails before it can build the classes list because the second element causes an AttributeError at line 9. The test never reaches the assertion.
