What happened (root cause)
- In the to_class method you call array[i].upper() on every element (the statement "_" = array[i].upper()) before you append type(array[i]).upper() throws an AttributeError when array[i] is not a string (for example None, an int, or any object that doesn't implement .upper()).
- That exception stops the function before it can complete; the traceback lands at the nearby line classes.append(type(array[i])) (line 770 in your slice) because execution failed while handling the same iteration, but the real failing call is the .upper() call immediately above it in to_class.

Where it occurs
- Method: to_class
- Failing location: the line that reads "_ = array[i].upper()" (the line immediately before the classes.append call shown at line 770). The append line shows in the slice, but the AttributeError originates at the .upper() call.

Why this is happening
- .upper() is a string method. If array contains non-string values (None, ints, objects, class objects, etc.) calling .upper() raises AttributeError: "'X' object has no attribute 'upper'".
- The code does not check the element type or handle the exception, so the function crashes on the first non-string element.

Fixes (concrete, step-by-step options)
1) If you only want to work with strings and skip non-strings:
   - Replace the .upper() call with a type check and skip or convert.
   Example:
     def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
         if array is None:
             return None
         elif len(array) == 0:
             return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
         classes = []
         for i in range(len(array)):
             if isinstance(array[i], str):
                 _ = array[i].upper()
             else:
                 _ = None  # or continue to skip non-strings
             classes.append(type(array[i]))
         return classes

2) If you intended to turn everything into strings before inspecting type, explicitly convert:
   - Use str(array[i]) to avoid AttributeError.
   Example:
     s = str(array[i]).upper()
     classes.append(type(array[i]))

3) If the .upper() call is accidental and unnecessary:
   - Remove the line entirely. If your goal is only to get classes, you can simplify:
     def to_class(cls, array):
         if array is None:
             return None
         if not array:
             return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
         return [type(x) for x in array]

4) If you expect elements might be None and you must treat that specially:
   - Add explicit None check before calling .upper():
     if array[i] is None:
         # handle None case
     else:
         _ = array[i].upper()

Which one to pick
- If you only need element types, option 3 is the simplest and safest.
- If you really need to call .upper(), choose option 1 or 2 depending on whether non-strings should be skipped or converted.

Summary
- The crash is caused by calling .upper() on non-string elements in to_class. Fix by removing the unnecessary .upper(), checking the element type before calling .upper(), or converting the element to string first.
