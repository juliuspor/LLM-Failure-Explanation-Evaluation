- You intended to include variable values in the exception message but wrote a plain string inside an f-string instead of using braces like `{g}` (or `{r}`, `{g}`, `{b}`).
- You also only check g; there is no check for r or b here, so the raised message claiming "Red Green Blue" is inaccurate.
- The code can therefore either produce a misleading ValueError or fail earlier with NameError if the variable g does not exist in scope.
