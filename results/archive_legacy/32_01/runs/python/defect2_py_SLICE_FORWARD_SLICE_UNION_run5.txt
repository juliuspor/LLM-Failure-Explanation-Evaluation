1. value can map to a g outside 0–255 when value is outside the configured bounds:

   - If value > self._upper_bound then numerator > denominator → g > 255.

2. The error message is useless and misleading (line 77):

3. The method only checks g (the green channel) — it does not validate whatever you compute for the red or blue channels (if any) in the same way. That can hide other problems.
4. There is a possible division-by-zero if self._upper_bound == self._lower_bound (not shown as the current exception, but a real risk at line 73).
