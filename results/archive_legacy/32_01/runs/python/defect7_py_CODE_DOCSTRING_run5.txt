What happens
- The method to_class fails at the statement "_ = array[i].upper()" because it calls .upper() on every element of array. Non-string elements (including None) do not have an upper() method, so Python raises AttributeError.
- That exception occurs inside the for loop in to_class, at the line "_ = array[i].upper()". The code never reaches classes.append(type(array[i])) for the offending element.

Why it fails (root cause)
- The call array[i].upper() assumes every element is a string. If array[i] is None or any non-string object, .upper() does not exist and AttributeError is raised immediately.
- The docstring says "If any of these objects is None, a None element will be inserted into the array" but the current code does the opposite: it will raise AttributeError for None (and for non-strings), contradicting the stated behavior.
- The variable "_" is unused; its only effect is to force the AttributeError when an element isn't a string.

- Method: to_class
- Line: the line "_ = array[i].upper()" (inside the for loop). This is the exact point of failure.
