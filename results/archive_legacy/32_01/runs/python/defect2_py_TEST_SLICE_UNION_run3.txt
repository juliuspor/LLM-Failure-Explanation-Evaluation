What happened (where it failed)
- The test calls GrayPaintScale.get_paint(-0.5) and expects (0, 0, 0).
- At line 73 the code computes g:

- For value = -0.5 this produces a negative g.
- At line 76 the code checks `if g < 0 or g > 255:` and raises ValueError instead of returning a clamped black color. So the method raises an exception rather than producing (0,0,0).

Why it happens (root cause)
- get_paint assumes values outside the [lower_bound, upper_bound] range are an error and raises a ValueError when the computed channel falls outside 0..255.
- The test expects out-of-range inputs to be clamped to the nearest valid color (e.g. below range -> black (0,0,0)), not to raise.
- The error is triggered at line 76 because the computed g is < 0 for value -0.5.
