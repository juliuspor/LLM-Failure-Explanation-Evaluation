What happened (where it failed)
- The ValueError was raised on line 251: raise ValueError(f"Invalid locale format: {locale_str}").
- The input that triggered it was "fr__POSIX".
- The code around lines 248–252 is trying to parse a locale string by using fixed-index slices:
  - line 248 returns Locale(locale_str[0:2], locale_str[3:5]) in one branch,
  - line 251 checks locale_str[5] != '_' and raises if that check fails,
  - line 252 returns Locale(locale_str[0:2], locale_str[3:5], locale_str[6:]).

Why it failed (root cause, simple)
- The parser assumes the locale is in one of these fixed forms:
  - "ll_CC" (2-letter language, underscore, 2-letter country),
  - "ll_CC_variant" (country + underscore + variant).
- It indexes fixed positions (0:2, 3:5, 6:) instead of splitting on underscores or checking for an empty country.
- The input "fr__POSIX" has an empty country (two underscores: language '_' '_' variant). At index positions:
  - locale_str[0:2] → "fr" (language) — correct,
  - locale_str[3:5] → locale_str[3] = '_' and locale_str[4] = 'P' → gives "_P", not a two-letter country.
- Because locale_str[5] is 'O' (not '_'), the check on line 251 fails and the code raises ValueError. In short: the parser misinterprets the double underscore as a malformed country because it relies on fixed index positions rather than splitting/validating parts.

Concrete fixes (specific steps and code example)
Option A — Replace the fixed-index logic with splitting (recommended)
1. On or before line 248, split into parts with a max of 2 splits:
   parts = locale_str.split('_', 2)
2. Then handle the 1–3 parts safely:
   - If len(parts) == 1 and parts[0]: language only → Locale(parts[0])
   - If len(parts) == 2: language + country (country may be empty) → Locale(parts[0], parts[1] or None)
   - If len(parts) == 3: language + country + variant → Locale(parts[0], parts[1] or None, parts[2])
3. Validate lengths/characters as needed (e.g. language length == 2).

Example replacement for lines 248–252:
  parts = locale_str.split('_', 2)
  if not parts[0]:
      raise ValueError(f"Invalid locale format: {locale_str}")
  if len(parts) == 1:
      return Locale(parts[0])
  if len(parts) == 2:
      return Locale(parts[0], parts[1] or None)
  # len(parts) == 3
  return Locale(parts[0], parts[1] or None, parts[2])

Option B — Minimal change to support empty country
- Before checking locale_str[5] on line 251, detect the double-underscore case and handle as language+variant:
  if locale_str[2] == '_' and locale_str[3] == '_':
      return Locale(locale_str[0:2], None, locale_str[4:])
- Then keep the existing country+variant handling for the normal "ll_CC_variant" case.
