What happened (where it failed)
- The exception comes from the line inside to_class:
  _ = array[i].upper()
- At runtime array[i] was None, so Python tried to call upper() on a None object and raised:
  AttributeError: 'NoneType' object has no attribute 'upper'

Why it failed (root cause)
- The code assumes every element of array is a string (or has an upper() method). It does not check for None or for non-string items before calling upper().
- When an element is None (or any non-string without upper()), array[i].upper() fails immediately. That is the direct cause of the AttributeError.

Where (which method/line)
- Method: to_class
- Problematic statement: _ = array[i].upper() (inside the for loop)
- Following line also inspects the same element: classes.append(type(array[i])), but the exception prevents that line from running for the offending element.

Concrete fixes (step-by-step, specific code options)
1) If you only want to handle strings and preserve None types:
   - Check for None before calling upper():
     for i in range(len(array)):
         val = array[i]
         if val is None:
             classes.append(type(None))   # or classes.append(None) depending on intent
             continue
         _ = val.upper()                  # safe now
         classes.append(type(val))

2) If you want to ignore non-strings:
   for val in array:
       if isinstance(val, str):
           _ = val.upper()
           classes.append(str)
       else:
           # skip or handle as needed
           continue

3) If you want to coerce values to strings before calling upper():
   for val in array:
       s = "" if val is None else str(val)
       _ = s.upper()
       classes.append(type(val))   # or classes.append(str) if you want string class

4) Clean, Pythonic iteration and None-check:
   classes = []
   for idx, val in enumerate(array):
       if val is None:
           classes.append(type(None))
           continue
       if not hasattr(val, "upper"):
           raise TypeError(f"array[{idx}] of type {type(val).__name__} has no upper()")
       _ = val.upper()
       classes.append(type(val))

Choose the fix that matches your intent:
- If None is valid data, add an explicit None check before calling upper().
- If array should never contain None, validate input at the start and raise a clear error (ValueError) explaining that None is not allowed.
- If you only need the element types and the upper() call was unnecessary, remove the upper() call entirely.

Summary
- Root cause: calling upper() without checking the element leads to AttributeError when an element is None.
- Fix: check for None (or for string type) before calling upper(), or remove/replace the upper() call depending on what you actually need.
