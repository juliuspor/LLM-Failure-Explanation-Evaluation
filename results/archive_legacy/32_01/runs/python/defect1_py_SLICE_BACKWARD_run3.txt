What happened (where it failed)
- The failure happens at the check on lines 400–401:
  - 400: if minutes_offset < 0 or minutes_offset > 59:
  - 401:     raise ValueError(f"Minutes out of range: {minutes_offset}")
- This check raised ValueError because minutes_offset was negative. That means the code computed a negative minutes component and then rejected it as out of range.

Why it happened (root cause)
- The code is splitting a total offset in minutes into an hours part and a minutes part, but it preserved the sign on the minutes component. For example, when the total offset is -330 minutes (UTC−05:30):
  - A naive split can produce hours = -5 and minutes = -30.
  - The check on line 400 treats -30 as invalid and raises the ValueError.
- The real rule: minutes must be in 0..59, and the sign (positive/negative) belongs to the whole offset (usually represented on the hours), not to the minutes field. The code used a splitting method that yields negative minutes for negative total offsets instead of taking the absolute minutes component.

Exactly where to change (concrete fixes)
- Replace the way minutes_offset is computed so minutes_offset is always in 0..59 and the sign is applied only to hours (or to the total offset).
- Two concrete, safe fixes:

  1) Use absolute division with divmod:
     - Let total = total_offset_in_minutes (the original signed minute count).
     - sign = 1 if total >= 0 else -1
     - abs_total = abs(total)
     - hours, minutes = divmod(abs_total, 60)
     - hours = hours * sign
     - set minutes_offset = minutes
     - This ensures minutes_offset is 0..59 and hours carries the sign.

  2) If you already computed hours by truncating toward zero, recompute minutes by:
     - abs_minutes = abs(total_offset_in_minutes) % 60
     - minutes_offset = abs_minutes
     - (do not keep a negative minutes value)

- Example change (pseudocode replacing the current split code before the check):
  - total = total_offset_in_minutes
  - sign = 1 if total >= 0 else -1
  - abs_total = abs(total)
  - hours_offset, minutes_offset = divmod(abs_total, 60)

  - # now minutes_offset is guaranteed 0..59; remove the negative check or keep it only for >59

Why this fix works
- It removes negative minute components by taking absolute minutes before splitting. This matches the conventional representation where minutes are 0..59 and the sign applies to the whole offset, so the check on lines 400–401 will no longer trigger for valid negative offsets.
