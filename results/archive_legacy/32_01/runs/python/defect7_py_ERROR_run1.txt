What happened (brief)
- Python raised AttributeError: 'NoneType' object has no attribute 'upper' because your code tried to call the string method .upper() on a value that is actually None.

Why that happens (root cause)
- .upper() is a method on str objects. If the variable is None, Python complains because NoneType has no .upper().
- That None got there because some earlier operation returned None or you didn’t set a fallback value. Common causes:
  - A function returned None (for example a function that only returns something in one branch).
  - dict.get(key) returned None because the key was missing and you didn’t supply a default.
  - A value assignment used something like my_var = maybe_value_or_none() and maybe_value_or_none() returned None.
  - Chained calls where an intermediate result was None (e.g., obj.get_sub().name where get_sub() returned None).

Where it occurred (how to locate the exact line)
1. Open the traceback you saw. It tells you the file, function, and line number. Example traceback line:
   File "app.py", line 42, in process_input
       name_upper = name.upper()
   Here the failing line is "name_upper = name.upper()" on line 42 of app.py.
2. Go to that exact file and line. That line is the code that attempted .upper() on a None value. Find the variable used there (in the example above, name).

Concrete steps to confirm the root cause
1. Inspect the variable right before the failing call. Add a quick check or logging on the failing line:
   - print(repr(name))
   - or use logging: logger.debug("name=%r", name)
2. Trace back where that variable was assigned. Find the function or expression that produced it and confirm whether it can return None.

Examples of specific fixes (choose the one that fits your situation)

A. If the variable can be missing from a dict:
- Before (problem):
  name = data.get('name')
  name_upper = name.upper()  # fails if 'name' not present
- Fix:
  name = data.get('name', '')          # give a default string
  name_upper = name.upper()

B. If a function sometimes returns None:
- Before:
  name = parse_name(payload)   # parse_name may return None
  name_upper = name.upper()
- Fix options:
  1) Make parse_name always return a string (preferred if that makes sense):
     def parse_name(payload):
         if something_missing:
             return ''
         return found_name
  2) Check the result before calling .upper():
     name = parse_name(payload)
     if name is None:
         # handle missing value explicitly
         name_upper = ''
     else:
         name_upper = name.upper()

C. If a method chain may produce None for an intermediate:
- Before:
  name_upper = obj.get_user().name.upper()
- Fix:
  user = obj.get_user()
  if user is None:
      # handle missing user
      name_upper = ''
  else:
      name_upper = user.name.upper()

D. Short defensive one-liner (use only when an empty string is acceptable):
  name_upper = (name or '').upper()

Checklist of safe steps to fix and verify

5. Add unit test(s) for the None/missing case so the failure won’t reappear.
