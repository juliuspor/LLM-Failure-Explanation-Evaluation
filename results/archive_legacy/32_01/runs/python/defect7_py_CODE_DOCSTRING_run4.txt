What happened (root cause)
- The failure happens inside the for loop of to_class where you call array[i].upper().
- If array[i] is None, that call raises AttributeError: 'NoneType' object has no attribute 'upper'.
- If array[i] is a non-string (int, dict, custom object, etc.) that also has no upper() method, you'll get a similar AttributeError.
- The call to .upper() is unnecessary for the intended operation (convert each element to its class) and appears to be a stray leftover. Because of that stray call the method fails for any None or non-string element, violating the docstring promise.

Where it occurs (precise location)
- In method to_class, in the loop:
    for i in range(len(array)):
        _ = array[i].upper()
        classes.append(type(array[i]))
  The line "_ = array[i].upper()" is the exact failing statement.

Why this breaks the behavior described in the docstring
- Docstring says "If any of these objects is None, a None element will be inserted into the array." Instead, the .upper() call causes an AttributeError before you can insert None into the result.
- The method should not assume elements are strings. The only required operation is to append the element's class (or None) to the output list.

Concrete fixes (step-by-step)
1. Remove the stray .upper() call.
2. Preserve None elements in the output (append None rather than type(None)), to match the docstring.
3. Optionally tighten the return type hint to reflect that elements can be None: Optional[List[Optional[type]]].

Corrected implementation (drop-in replacement)
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Optional[type]]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        
        classes: List[Optional[type]] = []
        for i in range(len(array)):
            elem = array[i]
            if elem is None:
                classes.append(None)            # preserve None per docstring
            else:
                classes.append(type(elem))     # append element's class
        return classes

Notes
- If you instead want to convert None elements to type(None) rather than keep None, change classes.append(None) to classes.append(type(None)). Make that choice deliberately and update the docstring accordingly.
- Keep the EMPTY_CLASS_ARRAY.copy() behavior if you need to return a new list for empty input.
