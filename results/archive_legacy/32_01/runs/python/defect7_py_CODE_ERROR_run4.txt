What happened (where and why)
- The exception comes from the to_class method, on the line:
  _ = array[i].upper()
  (this is the sixth line in the snippet inside the for loop).
- At runtime array[i] was None, so calling .upper() raised AttributeError: 'NoneType' object has no attribute 'upper'.
- Root cause: the code always calls .upper() on every element of array without checking whether that element is None (or even whether it's a string). When any element is None (or some other non-string without upper), .upper() fails.

Why .upper() is suspicious here
- The call to .upper() is unused — its result is assigned to the throwaway variable _ and never used later. That makes it likely either:
  - a leftover debug line that accidentally causes the crash, or
  - an attempt to coerce elements to strings that was applied incorrectly.
- The method then appends type(array[i]) to classes, so .upper() is not needed to compute the type.

Concrete fixes (pick one that matches your intent)
1) If you want to keep None elements and return their types (None -> None)
- Remove the .upper() call entirely:
  def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
      if array is None:
          return None
      elif len(array) == 0:
          return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
      classes = []
      for item in array:
          classes.append(type(item))
      return classes

2) If None elements are invalid and you want to raise an explicit error
- Validate and raise a clear error:
  for i, item in enumerate(array):
      if item is None:
          raise ValueError(f"to_class: array[{i}] is None")
      classes.append(type(item))

3) If you intended to operate only on strings (calling .upper()) before getting their types
- Check type before calling .upper():
  for i, item in enumerate(array):
      if item is None:
          classes.append(None)            # or raise
          continue
      if isinstance(item, str):
          item = item.upper()             # only call .upper() on strings
      classes.append(type(item))

4) If .upper() was left by mistake, remove the line entirely to avoid unchanged behavior and crashes.

Summary
- The immediate exception was caused by calling .upper() on array[i] when array[i] was None in to_class (loop line). Fix by removing the unnecessary .upper() or by checking for None/type before calling it, or by making callers guarantee non-None string elements.
