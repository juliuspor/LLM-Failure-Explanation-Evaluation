What happened (where it fails)
- Method: to_class
- Failing line: the for-loop line "_ = array[i].upper()" (line 6 in your snippet)
- Failure seen when running the test: ClassUtils.to_class(['Test', None, 99.0])

Why it fails (root cause)
- The code calls array[i].upper() unconditionally for every element.
- None and float (and many other types) do not have an upper() method. When the loop hits the second item (None) the interpreter raises:
  AttributeError: 'NoneType' object has no attribute 'upper'
- That exception stops execution before the function can append the element types and return [str, type(None), float], so the test fails.

Exact place it occurs
- Inside to_class, in the for i in range(len(array)) loop, at the statement "_ = array[i].upper()". The call to .upper() must only be used on strings; here it’s being applied to values that aren’t strings.

How to fix it (concrete steps)
1. Remove the unconditional .upper() call, or only call .upper() when the element is a string.
2. Make the loop append types directly (type(value)) for all values, handling None safely.
3. Optionally simplify the loop to iterate values rather than indexing.

Suggested corrected implementation (replace the body of to_class with one of these):

A — Minimal fix (only call upper on strings):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for v in array:
            if isinstance(v, str):
                _ = v.upper()    # only run for strings, if you truly need it
            classes.append(type(v))
        return classes

B — Cleaner and safe version (no unnecessary .upper()):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        return [type(v) for v in array]

After either fix, the test:

will return [str, type(None), float] as expected.
