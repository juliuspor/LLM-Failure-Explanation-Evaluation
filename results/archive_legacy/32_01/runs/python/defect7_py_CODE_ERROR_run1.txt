What happened (exact place and cause)
- The error occurs inside the to_class method, at the line:
  _ = array[i].upper()
- The exception AttributeError: 'NoneType' object has no attribute 'upper' means array[i] is None for at least one index i. None has no upper() method, so calling .upper() fails.

Why it happened (root cause)
- The code assumes every element in array is a str (or otherwise has upper()), but the input list can contain None. There is no check for element being None (or for element being a string) before calling .upper(), so the code crashes as soon as it encounters a None element.

Where it occurred in the method
- Method name: to_class
- Problematic line: the loop body
  for i in range(len(array)):
      _ = array[i].upper()        <-- this line raises when array[i] is None
      classes.append(type(array[i]))

Concrete fixes (pick one depending on intended behavior)

1) If None elements are allowed and should be preserved in the result:
- Check for None before calling .upper(), append None to classes for None entries.
- Patch:
  for i in range(len(array)):
      elem = array[i]
      if elem is None:
          classes.append(None)
          continue
      _ = elem.upper()
      classes.append(type(elem))

2) If you only meant to call .upper() for strings (and otherwise record the type):
- Test for str first:
  for i in range(len(array)):
      elem = array[i]
      if isinstance(elem, str):
          _ = elem.upper()
      classes.append(type(elem))

3) If None is invalid input and you want a clear error:
- Raise a ValueError with index information instead of letting AttributeError occur:
  for i in range(len(array)):
      if array[i] is None:
          raise ValueError(f"to_class: array[{i}] is None; expected non-None element")
      _ = array[i].upper()
      classes.append(type(array[i]))

4) Minor cleanups (recommended)
- Iterate directly over elements instead of index-based loop:
  for i, elem in enumerate(array):
      if elem is None: ...
- Use typing for return type: Optional[List[type]] rather than List[Type] (optional).
