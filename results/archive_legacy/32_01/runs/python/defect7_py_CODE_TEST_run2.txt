What happened (where it fails)
- In to_class, the loop calls array[i].upper() on every item (line 9).  
- With your test input ['Test', None, 99.0] this fails on the second item because None has no attribute upper. Python raises:
  AttributeError: 'NoneType' object has no attribute 'upper'
  This occurs at the call on line 9 inside def to_class (the for loop).

Why it fails (root cause)
- The code unconditionally calls .upper() on every element before determining its type (line 9). .upper() only exists on strings. For None and numeric values (like 99.0) .upper() does not exist, so the method raises an AttributeError and the function never reaches classes.append(type(...)) for those items.
- The intent appears to be to get each element's type, but the stray .upper() call is irrelevant and unsafe for non-string elements.

Exactly where to change (line references)
- Remove or guard the call on line 9:
    _ = array[i].upper()  # line 9 — this is the problematic call

Concrete fixes (step-by-step)
1. If you do not need to transform strings, simply remove line 9 and return the types:
   - Replace the loop with:
       for i in range(len(array)):
           classes.append(type(array[i]))
2. If you intended to normalize string values but still want types:
   - Only call .upper() when the item is a string:
       for i in range(len(array)):
           if isinstance(array[i], str):
               _ = array[i].upper()
           classes.append(type(array[i]))
3. If you want to treat None specially (to ensure type(None) is returned) the code above already does that once .upper() is removed or guarded.

Corrected example (minimal change)
- Remove the unsafe call:
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[Type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for i in range(len(array)):
            classes.append(type(array[i]))
        return classes

Why that corrected version passes your test
- For ['Test', None, 99.0] it returns [str, type(None), float] because type() is called directly on each element and no .upper() is invoked on None or float.

Summary
- The failure is caused by the unconditional call to .upper() on line 9. Remove it or only call it after confirming the element is a string.
