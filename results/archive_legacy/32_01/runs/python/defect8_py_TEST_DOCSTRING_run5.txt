What happened
- The failing test calls LocaleUtils.to_locale('fr__POSIX') expecting a Locale with language='fr', country='' and variant='POSIX'.
- Locale strings that specify a variant but no country use a double underscore: language + "__" + variant. 'fr__POSIX' is a valid input under the method's docstring and the test expectation.

Why it fails (root cause)
- In LocaleUtils.to_locale the code validates the country component with a strict length check like:
    if len(country) != 2:
        raise ValueError("Country must be two letters")
  (or an equivalent check that rejects any country string whose length is not 2).
- When you split 'fr__POSIX' by '_' you get parts = ['fr', '', 'POSIX'], so country == '' (empty string).
- The existing validation treats the empty country as invalid because its length is 0, therefore the method raises ValueError instead of accepting the empty country and continuing to parse the variant.
- In short: the validator does not allow the special case of an empty country field (the double-underscore form), so the method rejects 'fr__POSIX' even though the docstring/test expect it to be accepted.

Where it occurs
- In LocaleUtils.to_locale(), in the block that validates the country component right after splitting the input. The failing check is the strict "country length must equal 2" validation (the branch that raises ValueError for non-2-length country strings).
