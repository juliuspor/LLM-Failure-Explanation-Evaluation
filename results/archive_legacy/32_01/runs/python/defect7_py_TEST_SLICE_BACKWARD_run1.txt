What happened (root cause)
- In ClassUtils.to_class the code assumes every element in the input array is a string and calls .upper() on it.
- The failing input contains non-strings: ['Test', None, 99.0].
- On the second iteration array[1] is None, so None.upper() does not exist. That raises an AttributeError at the lines you showed:
  - ClassUtils.to_class, lines 768–769:
    768 |         for i in range(len(array)):
    769 |             _ = array[i].upper()
- Because the code tries to call .upper() on a NoneType (and would also fail on numbers), the method crashes before it can produce the expected output [str, type(None), float].

Why this is the problem (straightforward)
- .upper() is a string method. None and float objects don’t have that method.
- The code did not check the element type before calling .upper(), so it fails as soon as a non-string element is encountered.

Where it occurs
- Inside ClassUtils.to_class(), specifically at the loop starting at line 768 and the .upper() call on line 769.

Concrete fixes (step-by-step)
1. Decide desired behavior for non-string inputs (convert to their type, convert to string, raise a clearer error).
2. Replace the unguarded .upper() call with a type check or safe conversion. Example alternatives:

- If you want to support None and numeric types and return their types:
  - Inside the loop, check the element type:
    - if array[i] is None -> append type(None)
    - elif isinstance(array[i], str) -> use array[i].upper() and map to a class as original code intended
    - elif isinstance(array[i], (int, float, bool)) -> append type(array[i])
    - else -> handle/raise

- If you want to coerce everything to str first (less recommended if you want real types):

  - else: _ = array[i].upper()
