What happened (what failed)
- The test calls GrayPaintScale().get_paint(-0.5) expecting the method to return (0, 0, 0).
- Instead, execution hits the check at lines 76–77 and raises ValueError.
- Root cause: the code treats the single input value (g) as an out‑of‑range color channel because it checks "if g < 0 or g > 255" and raises for any value outside 0..255. Passing -0.5 therefore triggers that raise.

Why it happened (root cause, clearly)
- get_paint is using the raw argument g and enforcing that it must be in 0..255 (line 76). The test is passing a normalized grayscale value (-0.5) that is outside that integer channel range, so the conditional at line 76 evaluates true and the ValueError on line 77 is raised.
- The error message string ("Color parameter outside of expected range: Red Green Blue") also does not show which parameter or value caused the failure, so debugging is harder.

Where in the code it happened
- Method: get_paint
- Lines: 76–77 (the guard that raises the ValueError)
- Exact statement: if g < 0 or g > 255: raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
