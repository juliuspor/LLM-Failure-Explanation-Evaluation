What happened (where it fails)
- The code raises an AttributeError at line 769:
  Exception example: AttributeError: 'int' object has no attribute 'upper'
- The call that triggers the error is array[i].upper() on line 769. That method exists only on string-like objects (str, bytes, some custom types). If array contains a value that is not a string (e.g. int, None, dict), calling .upper() fails.

Why it fails (root cause)
- The loop assumes every element of array has an upper() method. It does not check the element’s type before calling upper(), so the code fails as soon as a non-string element is encountered.
- Also note: the result of .upper() is assigned to _ and not saved back into array, so the call cannot change array[i]; classes.append(type(array[i])) on line 770 will register the original type, not the uppercased string. Those are separate issues: the crash is due to the missing method, and a logic bug is that the uppercase result is discarded.
