What happened (where it fails)
- In the code that computes the total offset minutes you overwrite minutes_offset twice in succession so the first computation is lost and you end up with the wrong total minutes. That wrong total makes the computed offset too large and triggers the ValueError at line 411.

Exact lines involved
- Line 406: minutes_offset = hours_in_minutes - minutes_offset
- Line 408: minutes_offset = hours_in_minutes + minutes_offset
- Line 409: offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)
- Line 411: raise ValueError("Offset is too large")

Why this is wrong (root cause)
- The two assignments on lines 406 and 408 are mutually exclusive operations (one subtracts the minutes part, one adds it). As written both run in sequence, so the subtraction on line 406 is immediately discarded by the addition on line 408. That produces an incorrect minutes_offset value going into the millisecond conversion on line 409. Because that value is incorrect (too large or wrong sign), the code later hits the "Offset is too large" error at line 411.
- In short: a logic branch or variable was lost; the code does both subtract and add where it should do exactly one.

How to fix it (concrete steps)
1. Use a new variable for the total minutes to avoid overwriting the input minutes part, for clarity:
   - Introduce total_minutes and compute it either by addition or subtraction, not both:
     - total_minutes = hours_in_minutes - minutes_offset  # when hours are negative
     - OR total_minutes = hours_in_minutes + minutes_offset  # when hours are positive
2. Ensure the code chooses the correct branch (if/else) based on the sign of hours (or a flag you already have). For example:
   - if hours_negative:
         total_minutes = hours_in_minutes - minutes_offset
     else:
         total_minutes = hours_in_minutes + minutes_offset
3. Then compute offset from total_minutes:
   - offset = FieldUtils.safe_multiply(total_minutes, DateTimeConstants.MILLIS_PER_MINUTE)
4. Replace uses of minutes_offset in the rest of the method with total_minutes (or rename variables to make intent clear).

Why this will fix it
- Making the addition/subtraction mutually exclusive restores the intended math (minutes are either added or subtracted once), producing the correct total minutes passed to safe_multiply at line 409, and avoiding the spurious "Offset is too large" error at line 411.
