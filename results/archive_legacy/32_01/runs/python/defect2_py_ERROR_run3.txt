What the error means (root cause)
- "Color parameter outside of expected range: Red Green Blue" is raised when a function that expects a numeric RGB color tuple (e.g. (R,G,B) with each value in 0–255 or 0.0–1.0), or a recognized color string, instead received an invalid value: the literal string "Red Green Blue" (three words) or some other non-numeric, non-parseable value.
- In other words: the color argument passed is not in the format the library expects, so the color-parsing routine rejects it and raises ValueError.

Where this commonly happens (exact methods to inspect)
Check every call in your code that sets a color. Typical points to inspect (search your file for these method names):

- PIL / Pillow
  - Image.new(mode, size, color=...) — check the color argument passed here.
  - ImageDraw.ImageDraw.rectangle(..., fill=...) or .ellipse(..., fill=...) — check fill and outline values.
  - ImageColor.getrgb(color) — if you call this directly, it will raise when given an unparseable string.

- Matplotlib
  - plt.plot(..., color=...) or ax.plot(..., color=...) — the color argument must be a valid Matplotlib color.
  - matplotlib.colors.to_rgb(color) / matplotlib.colors.to_rgba(color) — these functions raise ValueError on invalid inputs.

- Other GUI/toolkits
  - Any function named set_background, set_color, drawText(..., color=...), CSS color parsers, etc. — the same principle applies.

What likely happened in your code (concrete example)
- You probably passed a single string containing three words, e.g. color="Red Green Blue", instead of:
  - a recognized color name like "red", or
  - a hex string like "#FF0000", or
  - an RGB tuple like (255, 0, 0) or normalized (1.0, 0.0, 0.0).
- Or you constructed the color string incorrectly by joining variables (e.g. f"{r} {g} {b}" instead of a tuple or "#rrggbb").

3. If you see a string like "Red Green Blue", convert it to a valid form:

     - Use an RGB tuple: color = (255, 0, 0)
     - or a hex string: color = "#FF0000"
     - or a single valid name: color = "red"
   - If using Matplotlib:
     - Use a valid color string ("red", "#ff0000") or tuple of floats in [0,1]: (1.0, 0.0, 0.0)

4. Replace the bad value construction. Concrete fixes:

     fill_color = (float(r)/255.0, float(g)/255.0, float(b)/255.0)

     - Ensure range: 0 <= int(r) <= 255
