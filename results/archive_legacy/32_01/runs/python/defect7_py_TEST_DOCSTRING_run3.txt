What happened
- The test called ClassUtils.to_class(['Test', None, 99.0]) and expected [str, type(None), float].
- Instead the method raised AttributeError when it reached the None element.

Why it happened (root cause)
- Inside ClassUtils.to_class the code unconditionally attempts to read a per-element attribute/method to get the element’s class. For example (pseudocode of the failing line in to_class):
  element.getClass()
  or
  element.__class__.__name__   # or similar attribute access that assumes element is not None
- When element is None, trying to call element.getClass() or to access a member on None raises AttributeError. The method does not handle the None element case before accessing that attribute, so the AttributeError is thrown and the method fails.

Where it occurred
- In ClassUtils.to_class, on the line that asks the element for its class (the line that calls element.getClass() or otherwise accesses element members). That is the first point where the code assumes element is non-None.

Why this is inconsistent with the docstring/test
- The docstring says: "If any of these objects is None, a None element will be inserted into the array" (and the test expects type(None) for None). But the Raises section contradicts that by saying "AttributeError: If any element is None".
- The implementation follows the Raises behavior (raising AttributeError) instead of the intended behavior stated in the docstring and required by the test.

Exact, actionable fixes (steps)
1. Decide the intended behavior and make it consistent:
   - If intended to return a placeholder for None values, either insert None (so result contains None) or insert type(None) (so result contains the NoneType). The test expects type(None), so choose that unless you prefer otherwise and change the test/docstring.
2. Implement explicit None handling before accessing element members. For example (Python-style):
   if array is None:
       return None
   result = []
   for element in array:
       if element is None:
           result.append(type(None))   # or result.append(None) if you prefer
       else:
           result.append(element.__class__)
   return result
3. Update the docstring and raises section to match the implementation:
   - If you insert type(None) for None elements, change "If any of these objects is None, a None element will be inserted into the array" to "If any element is None, type(None) will be inserted" (or "None will be inserted" if you choose that).
   - Remove the "Raises: AttributeError: If any element is None" line, since None is handled explicitly.
4. Add a unit test that covers None elements and the None-array case:
   - assert ClassUtils.to_class(None) is None
   - assert ClassUtils.to_class(['Test', None, 99.0]) == [str, type(None), float]

Why these fixes solve the problem
- Explicitly checking for element is None avoids accessing attributes on None and prevents AttributeError.
- Returning the intended placeholder (type(None) or None) makes the behavior match the docstring and the test expectations.
