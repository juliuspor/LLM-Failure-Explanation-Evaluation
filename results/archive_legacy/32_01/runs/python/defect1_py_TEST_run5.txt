What happened
- The call fails because you passed a negative value for the minutes argument: DateTimeZone.for_offset_hours_minutes(-2, -15).
- In Joda-Time the method DateTimeZone.forOffsetHoursMinutes(int hours, int minutes) treats minutes as a non‑negative value between 0 and 59 and validates it. Passing -15 causes the method to reject the input and throw an IllegalArgumentException (the failure happens inside DateTimeZone.forOffsetHoursMinutes).

Why it fails (root cause)
- The method applies the sign only to the hours parameter. It expects minutes to be a magnitude (0..59). The implementation performs a range check on minutes and will fail if minutes < 0 or minutes > 59. Supplying -15 triggers that check and makes the method throw instead of producing an ID string.

Where it occurs (exact method)
- The problem happens in DateTimeZone.forOffsetHoursMinutes(...). The call site is your test line:
  zone = DateTimeZone.for_offset_hours_minutes(-2, -15);
  assert zone.id == '-02:15'
- The exception is raised inside DateTimeZone.forOffsetHoursMinutes when it validates the minutes parameter.

How to fix it (concrete steps)
1. Pass a non‑negative minutes value and put the sign on the hours:
   - Use DateTimeZone.forOffsetHoursMinutes(-2, 15)
   - This will produce a zone id of "-02:15".
2. Or construct by milliseconds if you prefer a single signed value:
   - Compute offsetMillis = -((2 * 60 + 15) * 60 * 1000)
   - Use DateTimeZone.forOffsetMillis(offsetMillis)
3. Or create by id string directly:
   - DateTimeZone.forID("-02:15")

Summary
- Don’t pass a negative minutes argument to DateTimeZone.forOffsetHoursMinutes. Use a positive minutes value (0..59) and a negative hours value to express negative offsets like -02:15.
