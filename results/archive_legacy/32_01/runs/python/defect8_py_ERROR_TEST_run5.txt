What happened
- The call LocaleUtils.to_locale("fr__POSIX") failed because the parser rejected the input string as an invalid locale format and threw ValueError: Invalid locale format: fr__POSIX.
- The failure happens inside the LocaleUtils.toLocale(String) method when it parses and validates the tokens split by underscores (language, country, variant).

Why it fails (root cause)
- LocaleUtils.toLocale splits the input on underscores and then validates each token.
  - It expects either: language, language_country, or language_country_variant.
  - When you pass "fr__POSIX" there are two underscores in a row: language = "fr", country = "" (empty), variant = "POSIX".
- The implementation contains a validation check that treats an empty country as invalid in that branch. In other words, the method does not accept a zero-length country token even when a non-empty variant follows.
- Because of that validation branch the method throws the ValueError with the message you saw.

Where in the code it happens
- Inside LocaleUtils.toLocale(String). The exception is raised in the validation branch that handles the case where there are two underscores (language + empty country + variant). Concretely, look for the code that:
  - finds the first underscore (indexOf('_')), the second underscore (indexOf('_', first+1)),
  - extracts country = s.substring(first+1, second),
  - then runs validation on country and/or variant and throws the invalid-format error when the country is an empty string.
- (If you open the method, search for the block that handles first != -1 && second != -1 — that’s the exact location to inspect.)
