What happened (short): some code called .upper() on a variable that was None. .upper() only exists on string objects; calling it on None raises AttributeError: 'NoneType' object has no attribute 'upper'.

Why it happened (root cause):
- A variable that your code expects to be a string is actually None at runtime.
- That None came from an earlier operation that returned no value (explicit None, a missing dict key, .get() with no default, a function that returned None, or a failed parse).
- Your code then uses .upper() on that variable without checking that it is a string first.

Where it occurred (how you find the exact spot):
1. Look at the exception stack trace. It identifies the file, the function/method, and the line number. Example stack trace line:

       normalized = name.upper()
   - This tells you the failing file (app.py), method (process_name) and the exact line (42).
2. Open that file and go to the indicated line. That is the call that attempted .upper() on a None value.
