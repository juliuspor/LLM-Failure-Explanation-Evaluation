  - line 408: minutes_offset = hours_in_minutes + minutes_offset
- Because of that sign/combination bug you produce the wrong minutes total and then compute offset in milliseconds on line 409. Why it failed (root cause)
- The code treats minutes_offset both as “minutes within the hour” (0–59) and then reuses the same variable to hold “total minutes” after combining with hours. When the hour part is negative the code mixes up subtracting versus adding the minute part:
  - For a negative hour offset you must subtract the minute part from the absolute total minutes (e.g. -2:30 => total minutes = -(2*60 + 30) = -150). The current code does hours_in_minutes - minutes_offset in one branch and hours_in_minutes + minutes_offset in the other, but those operations are performed on variables whose signs/meanings are inconsistent. That yields the wrong sign for minutes in some cases.
- In short: minutes_offset is validated as a 0–59 minute field, but then treated as if it already reflected the hour sign; the combination logic is inverted, producing wrong total minutes (sometimes far too large), and that triggers the overflow/ValueError later.
