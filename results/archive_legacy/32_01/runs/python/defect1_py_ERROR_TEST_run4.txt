What happened (root cause)
- The for_offset_hours_minutes call rejects -15 because the method treats the minutes parameter as an absolute 0..59 value and does not accept a negative minutes argument. The code validates "minutes in range 0..59" (or equivalently disallows minutes < 0), so passing (-2, -15) fails that check and raises ValueError: Minutes out of range: -15.
- In other words, the method expects the minutes sign to be carried only by the hours argument (or expects minutes to be positive and combined with a separately-supplied negative hours), but your test passed a negative minutes value and the validator failed.

Where it happens
- In DateTimeZone.for_offset_hours_minutes (the public factory you called). The ValueError is thrown during the minutes validation inside that method (the lines that check minutes >= 0 and minutes < 60).

Why that logic is wrong for your call
- A time zone offset like -02:15 is a single signed offset. Calling for_offset_hours_minutes(-2, -15) is a legitimate representation of that signed offset, but the method treats minutes independently and refuses negative minutes. The method therefore cannot accept a valid signed minutes representation — it only accepts either:
  - (hours = -2, minutes = 15) or
  - (hours = 2, minutes = 15, sign = negative) — but you passed (-2, -15), so the minutes check fails.

     ```
     def for_offset_hours_minutes(hours, minutes):
         if hours == 0 and minutes == 0:
             return DateTimeZone.UTC
         # allow minutes to be negative but enforce same sign as hours (or zero)
         if minutes == 0:
             sign_ok = True
         else:
             sign_ok = (hours == 0) or ((hours > 0) == (minutes > 0))
         if not sign_ok:
             raise ValueError("Hour and minute sign mismatch")
         if abs(minutes) >= 60:
             raise ValueError("Minutes out of range: %d" % minutes)
         total_minutes = hours * 60 + (1 if hours >= 0 else -1) * abs(minutes)
         return DateTimeZone.for_minutes_offset(total_minutes)
     ```
