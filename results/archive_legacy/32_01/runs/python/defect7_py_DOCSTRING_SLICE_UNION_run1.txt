What happened (where it fails)
- At line 769 the code calls array[i].upper(). If array[i] is None that call raises AttributeError because NoneType has no .upper() method. The exception therefore occurs on line 769 during the loop.

Why it happens (root cause)
- The loop assumes every element is a string (or at least has an .upper() method). The docstring says a None element should produce a None in the result, but the code never checks for None before calling .upper(). That mismatch causes the AttributeError when a None element appears.
- There is also a docstring/behavior conflict: the Raises section says AttributeError if any element is None, but the description says a None element should produce a None entry. The implementation matches the Raises text (it raises) but contradicts the intended behavior in the docstring.
