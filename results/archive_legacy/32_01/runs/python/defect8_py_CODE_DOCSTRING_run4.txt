What fails
- The method is defined as def to_locale(cls, locale_str: str) -> Optional[Locale): but it is not decorated with @classmethod (or @staticmethod). Because of that, Python does not treat the first parameter as the class automatically.
- When you call LocaleUtils.to_locale("en") Python tries to bind the function as a normal function on the class and will pass only a single argument ("en"). That makes the interpreter try to put "en" into the first parameter (cls) and leaves locale_str missing, so you get a TypeError like:
  TypeError: to_locale() missing 1 required positional argument: 'locale_str'
- If you instead call the method on an instance (instance.to_locale("en")), Python will pass the instance as the first argument and "en" as locale_str and the body will run — which hides the bug until you call it from the class.

Where it occurs
- The problem occurs at the method definition line:
    def to_locale(cls, locale_str: str) -> Optional[Locale]:
  (i.e., the first line of the code you posted). The error happens at call time, not inside the body.

Why it happens
- In Python, methods that expect the class as the first argument must be declared with @classmethod. Without that decorator, the method is a plain function object on the class; calling it on the class does not auto-bind the class to the first parameter.
- The signature uses the name cls which indicates you intended a class method, but you forgot to add the @classmethod decorator.

How to fix (concrete steps)
Pick one of these fixes and apply it at the method definition:

1) Make it a true class method (recommended if you need cls):
   - Add the decorator immediately above the def line:
       @classmethod
       def to_locale(cls, locale_str: str) -> Optional[Locale]:
   - Then calling LocaleUtils.to_locale("en") will work; cls will be the class.

2) Make it a static method (if you don't use cls inside the body):
   - Remove the cls parameter and add @staticmethod:
       @staticmethod
       def to_locale(locale_str: str) -> Optional[Locale]:
   - Or keep the signature and change to:
       @staticmethod
       def to_locale(cls, locale_str: str) -> Optional[Locale]:
     but that is confusing; better remove unused cls.

3) Make it an instance method (if you want to call via an instance):
   - Rename cls to self and call via an instance:
       def to_locale(self, locale_str: str) -> Optional[Locale]:

Which to choose
- If you intended the method to be a utility on the class (looks like a utility function), use @classmethod or @staticmethod. If the method never uses cls, @staticmethod is simplest.

One-line patch example (preferred):
    @staticmethod
    def to_locale(locale_str: str) -> Optional[Locale]:
        ...

This removes the TypeError and makes the method callable as LocaleUtils.to_locale("en") or instance.to_locale("en").
