What happened
- The test calls LocaleUtils.to_locale('fr__POSIX'). This string means language = "fr", empty country = "", variant = "POSIX" (the double underscore is the standard way to show an empty country).
- The code in LocaleUtils.to_locale reaches the check at lines 244–245:
  244 |             if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':
  245 |                 raise ValueError(f"Invalid locale format: {locale_str}")
- For 'fr__POSIX', ch3 is the third character (index 3) and is '_' (the second underscore). Since '_' is not between 'A' and 'Z', the condition is true and the method raises ValueError.

Why it fails (root cause)
- The parser assumes that ch3 and ch4 must be uppercase letters (a two-letter country code) at that position. It does not account for the valid case of an empty country, which is represented by two consecutive underscores.
- In other words, the code checks for a two-letter country unconditionally instead of first checking whether the country part is empty (double underscore) and handling that case separately. Because of that, the underscore character is treated as invalid and the method raises an error.

Where it occurs
- In LocaleUtils.to_locale, around lines 244–245 (the shown snippet). The failing validation happens before the method extracts the variant when the country is empty.

How to fix it (concrete steps)
1. Detect the empty-country form ("ll__variant") before validating country letters. In code, check if locale_str[2] == '_' and locale_str[3] == '_' (or otherwise detect a double underscore at the country boundary).
2. When a double underscore is present:
   - Set country = ''.
   - Set variant = locale_str[4:] (the remainder after the two underscores).
   - Skip the two-letter country validation entirely.
3. Otherwise (not a double underscore), validate country letters as the code currently does.
4. Alternatively (cleaner): split the string on '_' with a limit and handle empty parts explicitly:
   - parts = locale_str.split('_', 2)
   - Then interpret parts[0] as language, parts[1] as country (possibly ''), parts[2] as variant (if present). Validate language and country only when they are non-empty.
