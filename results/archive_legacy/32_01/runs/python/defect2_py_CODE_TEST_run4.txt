What happened (where and why)
- In method get_paint you clamp the incoming value into variable v:
  v = max(value, self._lower_bound)
  v = min(v, self._upper_bound)
  But the grayscale calculation that follows uses the original value, not the clamped v:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)

- Because get_paint uses value instead of v, a value below the lower bound (for example -0.5) produces a negative g. The subsequent range check:
  if g < 0 or g > 255:
      raise ValueError(...)
  triggers and the test fails.

Where it occurs in the code
- In get_paint: the line computing g uses value (the unclamped input), and the next lines raise the exception when g is outside 0–255.

Concrete reproduction
- If self._lower_bound == 0.0 and self._upper_bound == 1.0, calling get_paint(-0.5) leads to:
  g = int((-0.5 - 0.0) / (1.0 - 0.0) * 255.0) → negative g → ValueError raised.

Exact fix (step-by-step)
1. Use the clamped variable v in the grayscale calculation instead of the original value.
2. Protect against division by zero if lower and upper bounds are equal.
3. Optionally improve the ValueError message to include the out-of-range component value.

Fixed code example:
    def get_paint(self, value: float) -> Tuple[int, int, int]:
        # Clamp value to bounds
        v = max(value, self._lower_bound)
        v = min(v, self._upper_bound)
        
        # Avoid division by zero if bounds are equal
        span = self._upper_bound - self._lower_bound
        if span == 0.0:
            g = 0
        else:
            g = int((v - self._lower_bound) / span * 255.0)
        
        if g < 0 or g > 255:
            raise ValueError(f"Color parameter outside of expected range: {g}")
        
        return (g, g, g)

Why this fixes it
- Using v ensures values below the lower bound are clamped to lower_bound before computing g, so g will be within 0–255 for valid bounds.
- Checking span == 0.0 prevents a ZeroDivisionError when upper and lower bounds are equal.

That is the root cause and the exact code change needed.
