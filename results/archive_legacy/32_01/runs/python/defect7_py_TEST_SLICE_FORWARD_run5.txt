What happened (where it failed)
- The failure occurs inside ClassUtils.to_class at line 770:
  770 |             classes.append(type(array[i]))
- The code is using array[i] even though the loop variable i is the element, not an index. That makes array[i] use a string (or None or float) as an index into the list.
- For your test input ['Test', None, 99.0], that produces a runtime error like:
  TypeError: list indices must be integers or slices, not str
  (or a similar indexing error), so the method never returns the expected [str, type(None), float].

Why this is the root cause (plain language)
- In Python a common loop form is either:
  - for i in range(len(array)): i is an integer index, so array[i] is valid, or
  - for element in array: element is the list item directly, so you should use element, not array[element].
- The code mixes those styles: the loop supplies elements (e.g. 'Test'), but the body treats i as an index (array[i]). When i is a string, list indexing fails. That is exactly what line 770 does.

Exact place to fix
- Method: ClassUtils.to_class
- Line: 770 (the line with classes.append(type(array[i])))
