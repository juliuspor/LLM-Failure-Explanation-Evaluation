What happened (root cause)
- The loop in ClassUtils.to_class calls array[i].upper() unconditionally (line 769). That assumes every element is a string.
- When the loop reaches a non-string value (None or 99.0 in your test) .upper() does not exist and Python raises an AttributeError. The error happens at line 769, before you append the element’s type at line 770, so the test fails.

Where it occurred
- Method: ClassUtils.to_class
- File / snippet: the shown slice
- Exact lines: 768–770; the failure is at line 769 (array[i].upper()).

Why it fails for your test input
- Your input array ['Test', None, 99.0] contains:
  - 'Test' (str) — .upper() works
  - None (NoneType) — NoneType has no .upper(), so AttributeError is raised
  - 99.0 (float) — float has no .upper() either (but you never get to it because the loop already crashed on None)
- Because the exception occurs before classes.append(type(array[i])) runs for the non-string elements, the result is not produced and the assertion fails.

Concrete fixes (step-by-step)
Choose one of these fixes depending on intent.

1) Only call .upper() on strings (recommended if you only want to normalize string elements)
- Replace line 769 with:
    if isinstance(array[i], str):
        _ = array[i].upper()
- Keep line 770 unchanged (classes.append(type(array[i]))).
- Effect: strings get uppercased before processing; None and floats are skipped for .upper() but their types are appended.

2) Use a safe attribute check (if .upper() is optional)
- Replace line 769 with:
    upper = getattr(array[i], 'upper', None)
    if callable(upper):
        _ = upper()
- Then append the type as before.
- Effect: only objects that implement upper() get called.

3) Catch the AttributeError if .upper() is not critical
- Surround with try/except:
    try:
        _ = array[i].upper()
    except AttributeError:
        pass
- Then append the type on line 770.
- Effect: avoids crashing; use if you expect mixed types and don't care if .upper() fails.

4) If you intended to only accept strings and reject others, validate input early and raise a clear error
- At start of method:
    for i, v in enumerate(array):
        if not isinstance(v, str):
            raise TypeError(f"array[{i}] must be str, got {type(v).__name__}")
- Effect: fails fast with a clear message instead of an AttributeError deep in the loop.

Which to pick
- If the goal is to return [str, type(None), float] for the sample input, use option 1 or 2 (so non-strings are allowed and their types are appended).
- If non-strings are invalid input, use option 4 to give a clear error.

Summary
The immediate bug is that line 769 assumes every element has an .upper() method. Replace that unconditional call with a string check, a getattr/callable guard, or a try/except, or validate input early depending on desired behavior. This will prevent the AttributeError on None and floats and allow your test to pass.
