What happens (where it fails)
- Failure occurs at the check on line 76: `if g < 0 or g > 255:` (see slice lines 76–77).
- If g is not a single Python number (for example, a list/tuple of numbers or a NumPy array), the expressions `g < 0` and `g > 255` do not produce single True/False values. For a NumPy array they produce boolean arrays, and Python cannot evaluate `boolean-array or boolean-array` to a single truth value. That raises an exception like:
  - "ValueError: The truth value of an array with more than one element is ambiguous." (NumPy)
  - or "TypeError: '<' not supported between instances of 'list' and 'int'" (if g is a list)
- The f-string on line 77 (`f"Color parameter ...: Red Green Blue"`) is unrelated to the runtime error; it is just a message and does not interpolate g. The root cause is the comparison in line 76 producing non-scalar truth values.

Why this happens (root cause)
- The code assumes g is a scalar number so `g < 0` yields a boolean. If g is a sequence/array, `g < 0` yields a sequence/array of booleans. Python’s `or` operator requires its operands to be single truth values, so combining boolean arrays (or comparing lists to ints) triggers an exception. The check therefore fails when callers pass non-scalar color inputs.
