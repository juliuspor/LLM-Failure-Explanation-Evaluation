What happened (where it failed)
- The test calls ClassUtils.to_class(['Test', None, 99.0]).
- In the code shown (inside the slice_union implementation used by to_class) the loop at lines 768–770 runs:
  - line 769:  _ = array[i].upper()
  - line 770:  classes.append(type(array[i]))
- That call to array[i].upper() throws AttributeError when array[i] is None — NoneType has no upper() method — so execution stops at line 769 and you never get to append the inferred class for None or the float.

Why it failed (root cause)
- The code assumes every element in array is a string and calls .upper() unconditionally. That assumption is false: the input contains None and a float.
- Trying to call a string method (.upper()) on non-strings raises an exception; the loop does not guard against None or non-string types, so the method crashes before producing the expected [str, type(None), float].

Precise location
- Failure occurs at line 769 in slice_union (called by ClassUtils.to_class), where array[i].upper() is called on a None value.

How to fix it (concrete steps)
1. Don’t call .upper() without checking the element’s type. Replace the unconditional .upper() call with an explicit type check.
2. Use isinstance(item, str) to detect strings. Handle None explicitly with item is None or isinstance(item, type(None)).
3. For other values, append their type with type(item).
