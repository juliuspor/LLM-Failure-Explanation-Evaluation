What happened (where it fails)
- The code raises a ValueError at lines 76–77 when the variable g is outside 0..255:
  76 |         if g < 0 or g > 255:
  77 |             raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- The condition is correct: the exception is triggered whenever g < 0 or g > 255. The failure you’ll see is the ValueError raised at line 77.

Why it fails (root cause)
- The exception message is wrong and misleading. The f-string on line 77 is written as f"Color parameter outside of expected range: Red Green Blue" but contains no braces { } to insert variables. Because of that, the message prints the literal text "Red Green Blue" instead of telling you which component failed or what its value was.
- As a result you do get an exception at the correct place (g out of range), but the message does not identify "g" or its value, so you cannot tell from the message whether the failing component was red, green, or blue, nor what value triggered it.

Concrete fixes (step‑by‑step)
1. Replace the message with one that identifies the component and shows the value. For example, to report green explicitly:
   - Change line 77 to:
     raise ValueError(f"Green value out of range: {g} (expected 0..255)")
2. If this check sits among similar checks for r and b, make each message specific. For example:
   - after if r < 0 or r > 255:
       raise ValueError(f"Red value out of range: {r} (expected 0..255)")
   - after if g < 0 or g > 255:
       raise ValueError(f"Green value out of range: {g} (expected 0..255)")
   - after if b < 0 or b > 255:
       raise ValueError(f"Blue value out of range: {b} (expected 0..255)")
3. Alternatively, combine the checks and raise a single message showing which components are invalid:
   - Collect invalid components into a list, e.g. invalid = [(name, val) for name,val in (("r",r),("g",g),("b",b)) if val<0 or val>255]
   - If invalid: raise ValueError("Color parameter(s) out of range: " + ", ".join(f"{n}={v}" for n,v in invalid))
4. Test:
   - Call the method with g out of range (e.g., g = -1 or g = 300) and confirm the raised ValueError now shows "Green value out of range: -1 (expected 0..255)" or similar informative text.

Why these fixes help
- They make the exception message explicit about which parameter failed and what its value was, so you (or other developers) can immediately see the cause and location (lines 76–77) without guessing.
