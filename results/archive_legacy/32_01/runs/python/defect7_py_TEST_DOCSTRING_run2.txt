What happened
- The test calls ClassUtils.to_class(['Test', None, 99.0]) and expects [str, type(None), float].
- The implementation of to_class tries to get the element class by invoking an attribute/method on each element (for example, calling obj.getClass() or obj.__class__ without guarding for None).
- When the loop reaches the None element, the code invokes that attribute/method on None. In Python that raises AttributeError: 'NoneType' object has no attribute 'getClass' (or similar). That exception is thrown inside ClassUtils.to_class and the method fails instead of returning a list that contains type(None).

Why it happened (root cause)
- The code does not treat None as a special case before attempting to read an attribute or call a method on the element.
- The docstring for to_class contains contradictory statements:
  - "If any of these objects is None, a None element will be inserted into the array." (implies None in -> None out)
  - "Raises: AttributeError: If any element is None" (implies it will raise)
  Those contradictions indicate the implementation and the docstring/tests are not aligned. The tests expect None to be converted to type(None), not to raise.
- The direct cause is the unguarded attribute access on the element (the call that accesses the element's class) when the element is None.

Where it occurred
- Inside ClassUtils.to_class, at the line that obtains an element's class. Example failing line patterns:
  - Java-style: clazz = element.getClass();  // throws NullPointerException for null
  - Python-style: clazz = element.getClass()  // AttributeError for NoneType
  - Python simpler: clazz = element.__class__  // also fails if the code assumed a method that doesn't exist
- In short: the failure happens at the per-element class lookup inside to_class when element is None.

Concrete fixes (step-by-step)
1. Decide the desired behavior and make docstring and tests agree.
   - Option A (matches test): Convert None elements to type(None) in the returned array.
   - Option B: Insert None into the returned array (literal None) if that's what you want.
   - Option C: Explicitly raise a documented exception for None elements.

2. Implement the chosen behavior by handling None before accessing attributes:
   - If you want type(None) in the result (recommended, matches your test):
     - Python implementation sketch:
       - def to_class(array):
           if array is None:
               return None
           out = []
           for i, e in enumerate(array):
               if e is None:
                   out.append(type(None))
               else:
                   out.append(type(e))
           return out
     - Java implementation sketch:
       - public static Class<?>[] toClass(Object[] array) {
           if (array == null) return null;
           Class<?>[] result = new Class<?>[array.length];
           for (int i = 0; i < array.length; i++) {
               Object e = array[i];
               if (e == null) {
                   result[i] = Void.class; // or use Object.class, or a sentinel, or choose to return null
               } else {
                   result[i] = e.getClass();
               }
           }
           return result;
         }
       - Note: Java has no runtime type object for null — common choices are to return null in that slot or use Void.class. Choose what callers expect.

   - If you want literal None in the output:
     - Replace the None branch with out.append(None) (Python) or result[i] = null (Java).

   - If you want to raise a clear error:
     - Check e is None and raise a clear exception with a message that names the method and the array index:
       - raise AttributeError(f"ClassUtils.to_class: element at index {i} is None") (Python)
       - throw new IllegalArgumentException("ClassUtils.to_class: element at index " + i + " is null"); (Java)

3. Update the docstring to match the chosen behavior exactly:
   - If returning type(None): "If an element is None, the returned array will contain type(None) at that position."
   - If returning None slot: "If an element is None, the returned array will contain None at that position."
   - If raising: "Raises AttributeError if any element is None" (and keep implementation that raises).

4. Add unit tests for:
   - Null/None array input returns None.
   - Mixed arrays with None behave as you documented (return type(None) slot, return None slot, or raise).
   - Edge cases: empty array, array of all None, non-object elements if applicable.

Summary in one sentence
The code fails because to_class tries to get an element's class without checking for None first, causing an AttributeError on the None element; fix it by explicitly handling None in to_class and updating the docstring to match the chosen behavior.
