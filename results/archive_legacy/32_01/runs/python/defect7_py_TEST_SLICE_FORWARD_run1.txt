What happened (root cause)
- At line 770 you call type(array[i]) but the loop is iterating elements, not indices. In other words the loop variable i already is the element (the string "Test", None, 99.0), but the code treats it like an index into array.
- That mismatch is the root cause: the code uses an element as if it were an index, so it either raises an error (if an element can't be used as an index) or returns the wrong values when it does succeed.

Where it occurs
- Exact location: ClassUtils.to_class, line 770: classes.append(type(array[i])).

Why that makes the test fail
- Your test calls ClassUtils.to_class(['Test', None, 99.0]) and expects [str, type(None), float].
- If the loop is written like:
    for i in array:
        classes.append(type(array[i]))
  then on the first iteration i == 'Test', and array['Test'] is an invalid list lookup (TypeError). If the code somehow swallows that or uses elements that are valid integer indices, you will still get wrong types because you're inspecting array[array_element] instead of array_element itself.
- The correct behavior is to inspect the element's type, not to index the list again.

How to fix it (explicit steps)
1. Decide whether the loop variable should be the element or the index.
2a. If you want to iterate elements (recommended), change line 770 to use the element directly:
    - Before: classes.append(type(array[i]))
    - After: classes.append(type(i))   (assuming the loop is for i in array:)
   or better, rename the variable for clarity:
    for elem in array:
        classes.append(type(elem))
2b. If you want to iterate indices, change the loop header and keep array[i]:
    - Change the loop to: for i in range(len(array)):
        classes.append(type(array[i]))
3. Add a unit test assertion to ensure the new behavior:
    result = ClassUtils.to_class(['Test', None, 99.0])
    assert result == [str, type(None), float]
4. (Optional) Add a defensive check at the top of the method to ensure you are given a list/iterable and not None.

Short example fix (clear, exact change)
- Original (problem):
    for i in array:
        classes.append(type(array[i]))   # line 770
- Fixed (recommended):
    for elem in array:
        classes.append(type(elem))

That change makes the method return [str, type(None), float] for the input ['Test', None, 99.0], so the test will pass.
