What happens
- The method incorrectly rejects or computes offsets when minutes are negative (examples in the docstring such as (0, -15) and (-2, -15)).
- Two problems cause the failure:

1) Input validation (line with "if minutes_offset < 0 or minutes_offset > 59") rejects negative minutes outright, but the docstring permits negative minutes in some combinations (e.g. (0, -15), (-2, -15)). That line should not unconditionally forbid negative minutes.

2) The minutes-to-total-minutes calculation is wrong for negative hours with negative minutes. The code does:
   hours_in_minutes = hours_offset * 60
   if hours_in_minutes < 0:
       minutes_offset = hours_in_minutes - minutes_offset
   else:
       minutes_offset = hours_in_minutes + minutes_offset
This produces the wrong total for cases like hours_offset = -2, minutes_offset = -15:
   hours_in_minutes = -120
   minutes_offset becomes -120 - (-15) = -105  (this is -1:45), but the expected offset is -2:15 (-135 minutes).

Why that is wrong
- The code tries to combine hours and minutes by reusing the minutes_offset variable and applying a single subtraction for all negative-hour cases. But the correct combination depends on the sign of minutes:
  - For positive hours, minutes must be non-negative and you add minutes.
  - For negative hours:
    - if minutes is positive, the total is hours_in_minutes - minutes
    - if minutes is negative, the total is hours_in_minutes + minutes  (since minutes is negative this makes the total more negative)
- The current code only implements the "hours_in_minutes - minutes" branch for all negative hours, so negative minutes are handled incorrectly.

Exactly where to change (concrete steps)
1. Fix validation (current line: if minutes_offset < 0 or minutes_offset > 59:)
   - Allow negative minutes in the allowed input range. Use:
     if minutes_offset < -59 or minutes_offset > 59:
         raise ValueError(f"Minutes out of range: {minutes_offset}")

2. Fix the combination logic (current block starting with hours_in_minutes = hours_offset * 60)
   - Compute total minutes explicitly and clearly, for example:
     hours_in_minutes = hours_offset * 60
     if hours_offset < 0:
         total_minutes = hours_in_minutes - abs(minutes_offset)
     else:
         total_minutes = hours_in_minutes + minutes_offset
     (This follows the doc: negative hour + negative minute -> subtract the absolute minutes; negative hour + positive minute -> subtract minutes; positive hour + negative minute is invalid and should be rejected earlier.)

   - Alternatively, when you already validated sign combinations, a simpler and correct expression is:
     total_minutes = hours_in_minutes + minutes_offset
     but only if you ensure minutes_offset carries the correct sign relative to hours (or you normalize minutes first).

3. Enforce the prohibited sign combination (doc: positive hour with negative minute is invalid)
   - After validation and before combining, add:
     if hours_offset > 0 and minutes_offset < 0:
         raise ValueError("Positive hours with negative minutes is not allowed")

Summary of a minimal correct change

    hours_in_minutes = hours_offset * 60
    if hours_offset < 0:
