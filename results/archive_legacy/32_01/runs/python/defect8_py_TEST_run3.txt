What happened
- The test expects a Locale with language='fr', country='' and variant='POSIX'.
- LocaleUtils.to_locale returned a Locale whose variant is '_POSIX' (variant includes a leading underscore), so the equality assert fails.

Where it happened
- In LocaleUtils.to_locale, at the line that splits the input string into language/country/variant (the split call — e.g. the call to s.split('_', 1) or equivalent; you’ll find it in LocaleUtils.to_locale, near the top of that method).

Why it happened (root cause)
- The parsing code uses a split or substring that only separates the language from the rest once (maxsplit=1). For the input "fr__POSIX" that produces two parts: "fr" and "_POSIX". The code then treats the second part as the “variant” without removing the leading underscore. That leaves the variant string starting with '_', so the produced Locale is effectively Locale('fr', '', '_POSIX') while the test expects Locale('fr', '', 'POSIX').
- In short: the parser preserved the separator underscore in front of the variant when the country was empty, so the parsed variant has an extra leading underscore.

How to fix it (concrete steps)
1. Open LocaleUtils.to_locale and locate the line that splits the input (the call like s.split('_', 1) or code that finds the first underscore and takes the remainder).
2. Change the splitting so you get up to three parts (language, country, variant). Two concrete options:
   - Use a maxsplit of 2: parts = s.split('_', 2). Then assign language = parts[0], country = parts[1] if present else '', variant = parts[2] if present else ''.
   - Or keep the current split but, after extracting the “remainder” string, check for an empty country explicitly and then strip a leading underscore from the variant: if remainder.startswith('_'): country = ''; variant = remainder[1:]; else split remainder into country and variant normally.
3. Ensure you do not leave a leading underscore on variant: variant = variant.lstrip('_') is acceptable only if you confirm no valid variant should start with underscore in your system.
4. Add unit tests:
   - Existing test: LocaleUtils.to_locale('fr__POSIX') -> Locale('fr', '', 'POSIX')
   - Additional tests: LocaleUtils.to_locale('en_US') -> Locale('en','US',''), LocaleUtils.to_locale('en_US_WIN') -> Locale('en','US','WIN'), LocaleUtils.to_locale('fr') -> Locale('fr','','')
5. Run the tests and confirm the assert now passes.

Why this fix is safe
- Using split with maxsplit=2 exactly matches the expected format language[_country][_variant] and preserves empty country as an empty string while preventing the separator from becoming part of the variant.
- Explicitly stripping only the leading separator when country is empty avoids accidentally cutting valid variant characters.
