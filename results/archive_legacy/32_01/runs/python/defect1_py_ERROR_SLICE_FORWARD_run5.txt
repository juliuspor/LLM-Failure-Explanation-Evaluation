Why it failed (root cause)
- The code mixes signs and subtraction incorrectly when building the total offset in minutes from hour and minute parts.
- Specifically, the branch that sets minutes_offset uses both subtraction and addition of the minute component (lines 406 and 408). That logic produces a negative minute value for some inputs (for example when the hours part is negative but the minute part is handled the wrong way), so minutes_offset ends up being -15 instead of a valid 0..59 minute part or the correct total signed minute count.
- The downstream code expects either:
  - the minute component to be validated as a positive 0..59 before forming the total, or
  - a correctly signed total minutes value (hours*60 ± minutes),
  but the current code does neither reliably. As a result, you end up multiplying a negative minute count and later raising "Minutes out of range" (the immediate symptom).
