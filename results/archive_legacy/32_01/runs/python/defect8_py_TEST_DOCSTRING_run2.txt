What happened
- The test calls LocaleUtils.to_locale('fr__POSIX') expecting Locale('fr', '', 'POSIX').
- The implementation of LocaleUtils.to_locale rejects that input during its strict validation step because it treats an empty country field as invalid.

Where it fails (method and location)
- Fails inside LocaleUtils.to_locale(), in the validation block that parses and checks the language/country/variant parts. This is the code section that:
  - splits the input on '_' into parts,
  - checks the language is lowercase,
  - checks the country (if present) is exactly two uppercase letters,
  - and checks the separators are underscores.
  You can find the failing validation check in the to_locale method’s country-validation branch (around the parsing/validation block).

Why it fails (root cause)
- 'fr__POSIX' has three parts when splitting on underscores: ['fr', '', 'POSIX'].
- The implementation expects the country part to be either absent or exactly two uppercase letters. It does not treat an empty string between two underscores as a valid empty country.
- The code therefore treats the empty country string ('') as an invalid country and raises ValueError (or otherwise rejects the input).
- The docstring shows that an empty country with a variant should be accepted (Locale('fr', '', 'POSIX')), but the validator enforces country length > 0 when a second underscore is present — that mismatch is the bug.

How to fix it (concrete steps)
1. Parse the string with at most two splits: parts = locale_str.split('_', 2). This yields at most 3 parts: language, country (possibly empty), variant (possibly missing).
2. Validate the language part exactly as before (lowercase letters).
3. Change the country validation to accept either:
   - an empty string (meaning no country), or
   - a string of exactly two uppercase letters.
   Concretely:
   - if country == '' : accept (this indicates an explicit empty country like 'fr__POSIX')
   - else if len(country) == 2 and country.isupper() and country.isalpha(): accept
   - else: raise ValueError
4. Leave the variant validation unchanged (allow any non-empty variant when provided).
5. Ensure separators remain underscores (do not accept other separators).

Minimal code example (illustrative change)
- Replace the strict country check:
    if country and (len(country) != 2 or not country.isupper()):
        raise ValueError("Invalid country")
  with:
    # accept empty country when variant is present (e.g. "fr__POSIX")
    if country != '' and (len(country) != 2 or not country.isupper()):
        raise ValueError("Invalid country")

Why this fix works
- It preserves strictness for normal country values (must be two uppercase letters) while allowing the documented form where the country is explicitly empty between two underscores. After the change, LocaleUtils.to_locale('fr__POSIX') will parse as language='fr', country='', variant='POSIX' and return Locale('fr', '', 'POSIX').
