What happened (where and when)
- The failure happens inside ClassUtils.to_class, at the lines you showed:
  - Line 768: for i in range(len(array)):
  - Line 769:     _ = array[i].upper()
- With your test input array = ['Test', None, 99.0], when the loop reaches the second element (None) the code calls None.upper().
- NoneType has no attribute upper, so Python raises AttributeError: 'NoneType' object has no attribute 'upper'. The same problem would occur for the float 99.0.

Why it fails (root cause)
- The code assumes every element in array is a string and unconditionally calls .upper() on each element.
- That assumption is false for your input list (it contains None and a float), so the method tries to call .upper() on objects that don't provide that method.
- The root cause is an unchecked type assumption (string-only) in ClassUtils.to_class at the exact line shown (769).
