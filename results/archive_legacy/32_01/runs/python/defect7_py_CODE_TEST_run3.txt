What happened
- The test call ClassUtils.to_class(['Test', None, 99.0]) raises an exception instead of returning [str, type(None), float].
- The failure happens inside the to_class method, specifically on this line in the loop:
    _ = array[i].upper()
- The exception is an AttributeError: 'NoneType' object has no attribute 'upper' (or "'float' object has no attribute 'upper'") because the code blindly calls .upper() on every element, but not every element is a string.

Why it fails (root cause)
- The code assumes every array element is a string and calls .upper() unconditionally.
- When array[i] is None or a numeric type (e.g. 99.0), those objects don't have an upper() method, so Python raises AttributeError and stops before classes.append(type(array[i])) can run.
- In short: the unnecessary call to .upper() on non-strings is the direct cause.

Where it occurs
- Method: to_class
- Problematic statement: _ = array[i].upper() inside the for loop that iterates over array elements.

Concrete fixes (step‑by‑step)
Option A — If you never need .upper(): remove the call
1. Delete the line _ = array[i].upper().
2. Keep classes.append(type(array[i])) to collect element types.
Result: works for mixed lists (strings, None, numbers).

Option B — If you wanted to normalize strings before checking them: only call .upper() for strings
1. Replace the unconditional call with a type check, e.g.:
   if isinstance(array[i], str):
       _ = array[i].upper()
2. Then append the type as before.
3. This avoids calling .upper() on None or floats.

Option C — If you intended to treat string-like values differently (e.g. to detect classes by name): handle None explicitly and handle non-strings separately
1. For each element:
   - if element is None: append type(None)
   - elif isinstance(element, str): maybe normalize string then map name->class (implement lookup)
   - else: append type(element)
2. This makes behavior explicit and safe.

Example corrected minimal implementation (Option A):
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        elif len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for i in range(len(array)):
            classes.append(type(array[i]))
        return classes

What to expect after fixing
- With the removal or guarding of .upper(), ClassUtils.to_class(['Test', None, 99.0]) will return [str, type(None), float] as the test expects.
