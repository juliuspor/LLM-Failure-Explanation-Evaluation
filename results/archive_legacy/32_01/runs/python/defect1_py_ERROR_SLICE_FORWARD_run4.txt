- The code is producing a negative minute value (-15) and then passing that negative minute into a validation/multiplication routine that expects a 0..59 minute component, so the validator raises the "Minutes out of range: -15" error.

Why it happened (root cause)
- The code mixes the minute component (0..59) and the total minutes (hours*60 + minutes) in the wrong order/sign, and performs validation at the wrong time.
- Specifically, the branch at line 406 uses subtraction: minutes_offset = hours_in_minutes - minutes_offset. That makes the local variable minutes_offset contain a combined value that can be negative (for example, hours_in_minutes = 0, minutes_offset = 15 -> result = -15). Later logic that expects a minute-of-hour value (0..59) sees -15 and throws.
- In short: the code treats the minute field both as “minute-of-hour” and as “total minutes offset” without (a) validating the minute-of-hour first, and (b) computing a signed total in a single consistent step. This produces -15 where a minute-of-hour validator is applied.
