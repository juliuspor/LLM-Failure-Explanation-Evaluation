What happened (where it failed)
- The test calls GrayPaintScale.get_paint(-0.5).
- Inside get_paint the code computes a color component named g and then checks it at lines 76–77:
  76 |         if g < 0 or g > 255:
  77 |             raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- Because the input produces g < 0 (negative), the condition is true and the function raises the ValueError. That is why the assert color == (0, 0, 0) never runs — an exception was raised before a color could be returned.

Why it happened (root cause)
- The code assumes the gray input will produce a g value in the integer range 0..255, but the test passed -0.5 which produces a negative g. The method does not accept or clamp out-of-range inputs before checking g, so it raises an exception.
- The error message itself is unhelpful: "Color parameter outside of expected range: Red Green Blue" gives no numeric context (no offending value, no variable name, no allowed range), which makes debugging harder.

Precise place to fix
- Method: GrayPaintScale.get_paint
- Lines: 76–77 (the range check and the raise)

   - If inputs should be clamped, clamp the value so g never leaves 0..255 and return the clamped color.
