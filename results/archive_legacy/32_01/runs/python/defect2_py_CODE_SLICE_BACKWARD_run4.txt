What happened (where it failed)
- In get_paint (line 56) you clamp the input into variable v:
  - v = max(value, self._lower_bound)
  - v = min(v, self._upper_bound)
  (these are the clamps).
- But on line 73 you compute g using the original value, not the clamped v:
  - g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- Because you used value instead of v, g can still fall outside 0–255 and trigger the ValueError on line 76:
  - if g < 0 or g > 255: raise ValueError(...)
- Also, if self._upper_bound == self._lower_bound that division on line 73 will raise ZeroDivisionError.

Why it failed (root causes)
1. Unused/clamped variable: You computed and stored the clamped input in v, but never used it to compute g. That makes the clamping a no-op.
2. Division by zero risk: The expression (self._upper_bound - self._lower_bound) can be zero, causing a crash before you even reach the range check.

Concrete fixes (exact steps)
1. Use the clamped value v when computing g (replace value with v on line 73).
2. Guard against equal bounds by returning a defined color or handling the special case before the division.
