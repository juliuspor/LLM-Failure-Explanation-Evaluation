- Out-of-range indexing: line 242 — "ch3 = locale_str[3]" (also line 243 accesses locale_str[4])

Why it happened (root cause)
- The branch condition at line 236 is incorrect. The code uses "if length == 2:" but immediately inside that branch it assumes the string is much longer (accesses indices 3 and 4, slices 3:5, checks locale_str[5], etc.). That mismatch means the code takes a branch intended for longer strings only when length equals 2, so indexing blows up for any other length. In short: the conditional is inverted/wrong, so the parser tries to read characters that aren't there.
