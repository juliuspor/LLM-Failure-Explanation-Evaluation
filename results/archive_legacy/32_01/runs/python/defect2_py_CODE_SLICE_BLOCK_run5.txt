What happened (where it fails)
- At line 70–73 you correctly clamp the input into v:
  70-73: v = max(value, self._lower_bound); v = min(v, self._upper_bound)
- But at line 75 you compute g using the original value variable, not the clamped v:
  75: g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- If value is outside the bounds, that expression can produce a negative number or a number >255; the check at lines 76–77 then raises the ValueError you see:
  76-77: if g < 0 or g > 255: raise ValueError(...)

Why this is the root cause
- You intended to map the clamped value to a 0–255 grayscale but accidentally used the un-clamped value, so the clamp on lines 70–73 has no effect on the computed g.
- As a result, out-of-range inputs produce g outside 0..255 and trigger the exception at lines 76–77.

1. Use the clamped variable v when computing g:
   Replace line 75 with:
     g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
2. Protect against a zero-width range (lower == upper) to avoid division by zero:
   Example:
     if self._upper_bound == self._lower_bound:
         g = 0  # or 255 or whatever makes sense for your app
     else:
         g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
3. Optionally clamp g again after converting to int to guard against rounding:
     g = max(0, min(255, g))
