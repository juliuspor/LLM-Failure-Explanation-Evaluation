- The failure happens inside the method to_locale at runtime when a short string is passed. Specifically, at lines 242–244 the code reads:
  - ch3 = locale_str[3]
  - ch4 = locale_str[4]

Why it fails (root cause)
- The condition at line 236 is wrong. It says if length == 2, but the following code assumes the string is at least 5 characters long (it reads indices 3 and 4, later checks index 5 and slices [3:5] and [6:]).
- In short: the logic branch and the length check do not match. The branch meant to validate/parse region and variant parts is being entered for two-character inputs, causing out-of-range indexing.
