What happened (where it failed)
- The failure occurred in GrayPaintScale.get_paint at the code shown around lines 73 and 76–77.
- For value = -0.5 the code computes g at line 73:
  g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- That computation produces a negative g. The next lines (76–77) check the range and immediately raise ValueError when g < 0 or g > 255:
  if g < 0 or g > 255:
      raise ValueError(f"Color parameter outside of expected range: Red Green Blue")
- Because g < 0 for -0.5, get_paint raises ValueError instead of returning (0, 0, 0). That's why the test assert color == (0, 0, 0) fails.

Why it happened (root cause)
- get_paint assumes input value will always map into [0,255]. It does not clamp out-of-range inputs; it treats any out-of-range result as an error and raises an exception (lines 76–77).
- The test expects out-of-range inputs (like -0.5) to be clamped to the nearest valid color (black = (0,0,0)), not to raise.
- The error message itself is unhelpful: the f-string contains no runtime values, so it prints a fixed string instead of showing the offending value or g.
