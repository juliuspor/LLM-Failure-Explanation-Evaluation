What happened
- The test calls GrayPaintScale.get_paint(-0.5) and expects black (0, 0, 0).
- The method turns the input value into an index into a grayscale palette without clamping the input first.
- In Python a negative integer index picks from the end of a list. So index computed from -0.5 becomes negative and the method returns the palette's last entry (usually white) instead of the first entry (black). That makes the assertion fail.

Why it happens (root cause)
- Root cause: GrayPaintScale.get_paint does not clamp or validate inputs before converting them to a palette index.
- Specific behavior: code computes something like
    index = int(value * (len(palette) - 1))
    return palette[index]
  When value is -0.5, index is negative, so palette[index] resolves to the end of the list (white).
- This is purely an input-clamping logic bug in GrayPaintScale.get_paint — not a problem with assert or test framework.

Where it occurs
- In GrayPaintScale.get_paint, at the point where the numeric input value is converted to a list index (the lines that compute index and return palette[index]).

How to fix it (concrete steps)
1. Decide the allowed input domain (presumably 0.0..1.0). For inputs outside the domain, clamp to the nearest boundary.
2. Add an explicit clamp before computing the index, for example:
    value = max(0.0, min(1.0, value))
3. Then compute the index and return the palette entry:
    index = int(round(value * (len(palette) - 1)))
    return palette[index]
   Or, if you prefer floor behavior, use int(value * (len(palette) - 1)) and then bound the index:
    index = max(0, min(len(palette) - 1, int(value * (len(palette) - 1))))
4. Add or update unit tests:
    - assert get_paint(-0.5) == (0, 0, 0)
    - assert get_paint(1.5) == (255, 255, 255)
    - assert get_paint(0.0) == (0, 0, 0)
    - assert get_paint(1.0) == (255, 255, 255)

Minimal patch example (pseudo-code you can drop into get_paint)
- Before:
    index = int(value * (len(self.palette) - 1))
    return self.palette[index]
- After:
    value = max(0.0, min(1.0, value))        # clamp input to [0.0, 1.0]
    index = int(round(value * (len(self.palette) - 1)))
    return self.palette[index]

Why this fix works
- Clamping prevents negative values from producing negative indices, so a value < 0 maps to index 0 (black) and a value > 1 maps to the last index (white).
- This matches the test's expectation and prevents surprising behavior from Python's negative list indexing.
