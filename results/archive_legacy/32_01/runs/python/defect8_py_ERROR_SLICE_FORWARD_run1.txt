What happened (where it failed)
- The code that parses locale strings fails when given "fr__POSIX".
- The failure comes from the check at line 250: `if locale_str[5] != '_':` which leads to the ValueError raised on line 251 (`ValueError(f"Invalid locale format: {locale_str}")`).

Why it failed (root cause)
- The parser expects either:
  - a short form like "fr_FR" (language + country), handled by the return on line 248, or
  - a long form like "fr_FR_POSIX" (language + country + variant), handled by the code after the check on line 250–252.
- "fr__POSIX" has an empty country portion (there are two underscores back-to-back). That shifts the characters so the code no longer finds the underscore at index 5 where it expects the separator before the variant.
- Concretely, for "fr__POSIX":
  - indexes: 0='f', 1='r', 2='_', 3='_', 4='P', 5='O', ...
  - The code reads `locale_str[3:5]` as the country (characters at 3 and 4) -> "_P", which is wrong, and then checks `locale_str[5] != '_'` (line 250). At index 5 there is 'O', not '_', so the code treats the format as invalid and raises the ValueError.
- So the parser assumes fixed character positions for language/country/variant and does not handle an empty country field; an input with a missing country (double underscore) breaks those position-based checks.

Clear fixes (step-by-step)
1. Stop relying on fixed character indexes. Split the string into components instead:
   - Use parts = locale_str.split('_', 2) to get up to three components: language, country, variant.
2. Validate parts explicitly:
   - Ensure parts[0] (language) is present and valid length (e.g. 2).
   - If len(parts) == 1: return Locale(parts[0])
   - If len(parts) == 2: return Locale(parts[0], parts[1])  (allow empty country if you want to support it, or reject empty country with a clear error)
   - If len(parts) == 3: return Locale(parts[0], parts[1], parts[2])
3. Replace the index-based check at line 250 with the split-based logic so the parser correctly handles "fr__POSIX" if empty country is allowed, or produces a clear error message if empty country should be rejected.

Example replacement outline
- Replace the block around lines 248–252 with:

This fixes the root cause (index assumptions) and gives a clear, testable way to handle "fr__POSIX".
