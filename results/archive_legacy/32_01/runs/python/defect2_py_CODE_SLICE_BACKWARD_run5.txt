- But on line 73 you ignore that clamped v and use the original value when computing g:
  - g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
- Because you used value instead of v, a value outside the bounds can produce g < 0 or g > 255 and trigger the ValueError at line 76.

Why it fails (root cause):
