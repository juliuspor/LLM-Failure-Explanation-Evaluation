What happened (what the failure is)
- The code raises ValueError "Minutes out of range: X" at lines 400–401 when parsing an offset that includes a negative sign (for example "-01:30").
- The root cause is that the minutes field has already been given the negative sign before the range check at line 400. The check expects minutes to be in 0..59, but because the code applied the sign early the value is negative (e.g. -30) and the check fails.

Why it happens (root cause, precisely)
- The variable minutes_offset is treated in two different ways in this method:
  - At lines 400–401 it is validated as if it were the unsigned minutes component (must be between 0 and 59).
  - Later at lines 406–409 minutes_offset is reused to build the total minutes (hours_in_minutes +/- minutes_offset) after the sign is applied.
- If the sign is applied to the minutes component before the validation (so minutes_offset is negative when you reach line 400), the condition minutes_offset < 0 is true and you get the ValueError even though the logical minutes component (30 in "-01:30") is valid.
- In short: the code applies the negative sign to the minutes value too early and then validates the signed value instead of the absolute minutes magnitude.
