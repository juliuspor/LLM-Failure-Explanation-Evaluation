What happened (where it failed)
- The error came from line 769: array[i].upper().
- Python raised AttributeError: 'NoneType' object has no attribute 'upper' because array[i] was None at runtime, so None.upper() was attempted.

Why it happened (root cause)
- .upper() is a string method. It only exists on str objects.
- Some element(s) inside array are None (or possibly another non-string type), so calling .upper() on that element causes the AttributeError.
- The loop uses for i in range(len(array)): and then directly calls array[i].upper() without checking element type or presence, so the code assumes every element is a string when it is not.

Where to look
- Inspect the contents of array right before the loop (line ~768). The problematic element index can be found by checking which array[i] is None.

2) Only call .upper() on strings
- Use an isinstance check to be safe:
  for i in range(len(array)):

      else:
          # decide what to do for non-strings: skip, convert, or raise

- If non-string types are expected and should be converted: use option 3.
- If None is a bug (should never occur): trace and fix the place that appends/returns None (option 4), and keep a defensive check during debugging.

Summary
- The error occurred because array[i] was None and .upper() is only on strings.
- Either guard before calling .upper(), convert the value to a string, or fix the code that produced None. Use enumerate/print to locate the offending index. These steps will prevent the AttributeError and point to the underlying bug.
