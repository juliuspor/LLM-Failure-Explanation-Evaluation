What’s happening
- At line 768 you build a loop index range(len(array)). That range is fixed at the time the loop starts.
- At line 769 you do array[i].upper(). If the list array has been shortened (elements removed) during the loop, array[i] can become out of bounds and Python raises IndexError: list index out of range.
- Alternatively, if array[i] exists but is not a string (for example None or an int), calling .upper() raises AttributeError: 'int' object has no attribute 'upper'.

Why each error happens (where and why)
- IndexError occurs on line 769 when i >= len(array) because some code removed items from array after you computed range(len(array)). The index calculation is done once (line 768), so the loop still tries to access positions that no longer exist.
- AttributeError occurs on line 769 when array[i] is not a str. .upper() is a str method; calling it on a non-string object fails.

Concrete fixes (choose one depending on the root cause)
- If list is being mutated during the loop (IndexError):
  1. Iterate over a snapshot copy so removals don’t affect indexing:

         _ = array[i].upper()
     (or better:)

         _ = item.upper()

     for i in range(len(array) - 1, -1, -1):
         _ = array[i].upper()

     for item in list(array):   # shallow copy
         _ = item.upper()
