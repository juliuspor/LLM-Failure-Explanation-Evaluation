What went wrong (what happened and where)
- The method to_class calls array[i].upper() inside the loop (this is the line just before the classes.append call).
- That .upper() call assumes every array element is a string. If any element is not a string (for example None or already a class/type object), Python raises AttributeError: "'NoneType' object has no attribute 'upper'" or "'type' object has no attribute 'upper'".
- The stack slice you showed points at the line classes.append(type(array[i])) (line 770). The actual fault originates at the previous line where .upper() is invoked; the interpreter reports the frame where execution was active, which is why line 770 appears in the trace even though the .upper() call caused the AttributeError.

Why this is the root cause (simple explanation)
- .upper() is only defined on str. If an element is not a str, calling .upper() fails immediately. The code never checks element type or None before calling .upper(), so it crashes when a non-string element is present.
