### What Happened

The `toClass` function is supposed to convert an array of `Object` instances into an array of corresponding `Class<?>` objects. If an element in the input array is `null`, `toClass` should also return `null` for that position in the output array.

### Why It Happened

The `toClass` method, when handling a `null` object in the input array, attempts to call `getClass()` on it. This results in a `NullPointerException` because `null` does not have a `getClass()` method. The method doesn't have a specific check for `null` inputs to prevent this exception from being thrown.

The issue occurs in the following portion of the `toClass` method:

```java
for (int i = 0; i < array.length; i++) {
    classes[i] = array[i].getClass(); // This line can throw NullPointerException if array[i] is null
}
```

To fix this problem, you need to add a null check when iterating over the `array` elements:

```java
for (int i = 0; i < array.length; i++) {
    if(array[i] != null) {
```
