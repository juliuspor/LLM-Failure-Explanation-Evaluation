What happened (where it failed)
- The ValueError is raised at lines 400–401 when the code checks minutes_offset:
  400 |         if minutes_offset < 0 or minutes_offset > 59:
  401 |             raise ValueError(f"Minutes out of range: {minutes_offset}")
- That check rejects any negative minutes_offset. The code therefore fails whenever minutes_offset is negative, even when that negative value is valid in context (for example a -30 minute component of a negative timezone offset).

Why it happened (root cause)
- The code treats the minutes component as if it must always be non‑negative (0..59). That ignores the common representation where the sign of an overall offset is carried separately from a 0..59 minutes magnitude. If minutes_offset is computed from a total offset that can be negative (for example total_minutes = -90 → minutes_offset = -30), the check incorrectly treats -30 as out of range and raises.
- In short: minutes_offset can legally be negative because the sign belongs to the whole offset, not to the minutes field by itself, but the validation test assumes otherwise.

Concrete example that triggers the failure
- total_offset = -90 minutes (i.e. -1 hour 30 minutes)
- If minutes_offset is computed as -30, the condition minutes_offset < 0 is true and the ValueError is raised, even though the overall -1:30 offset is valid.

How to fix it (precise steps)
1. Inspect how minutes_offset is computed. If it comes from a signed total (e.g., total_minutes, total_seconds), derive sign and magnitude separately:
   - sign = -1 if total_minutes < 0 else 1
   - minutes = abs(total_minutes) % 60
   - hours = abs(total_minutes) // 60
   - then reapply sign to hours (or to the combined offset), not to minutes.
2. Replace the failing check with a validation of minutes magnitude, not sign. For example:
   - if minutes < 0 or minutes > 59: raise ValueError(...)
   - Or more simply: if not (0 <= minutes <= 59): raise ValueError(...)
   (where minutes is the absolute-minute value from step 1)
3. If you must keep minutes_offset variable name, normalize it before checking:
   - minutes_offset = abs(minutes_offset) % 60
   - if minutes_offset > 59: raise ValueError(...)
   But prefer computing minutes from the absolute total as in step 1.
4. Add unit tests that cover negative offsets, e.g. total offsets: -90, -30, +90, +30, to ensure minutes are handled correctly.
