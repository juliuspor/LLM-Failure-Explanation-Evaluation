What the error means (plainly)
- Python tried to call the string method .upper() on a value that was actually None. None has no .upper attribute, so Python raised AttributeError: 'NoneType' object has no attribute 'upper'.

How to find the exact place this happened
1. Read the full traceback you were shown. The topmost user-code frame lists a filename, a line number, and the function/method name where the .upper() call happened. For example the traceback will look like:
   File "/path/to/app.py", line 42, in normalize_name
     return name.upper()
   Use that filename and line number to open the file and inspect that line.

Why it happened (concrete root causes)
- The variable you call .upper() on (e.g., name) holds None, not a string. That happens when:
  - You called dict.get('key') and the key was missing (dict.get returns None by default).
  - A function returned None (either intentionally or because it reached a code path without a return).
  - You assigned the result of an operation that can return None (e.g., re.search(... ) can return None; you then used .group() or .upper()).
  - You overwrote a string variable earlier with None (e.g., name = something_that_might_be_none()).

Concrete examples and exact fixes
1) Offending code pattern (typical)
   # app.py, line 42, function normalize_name
   def normalize_name(data):
       name = data.get('name')          # returns None if no 'name' key
       return name.upper()              # AttributeError if name is None

   Fixes (choose one, depending on intended behavior):
   - Provide a default when reading from dict:
       name = data.get('name', '')     # '' instead of None
       return name.upper()
   - Validate and raise a clear error:
       name = data.get('name')
       if name is None:
           raise ValueError("missing required 'name' in data")
       return name.upper()
   - Coerce non-strings to string (only if appropriate):
       name = data.get('name')
       return (name or "").upper()

2) Offending code pattern where a helper returns None
   # app.py, line 68, function process_user
   def find_user_display(user_id):
       if missing_condition:
           return None      # later code assumes string
       return "User X"

   def process_user(user_id):
       display = find_user_display(user_id)
       return display.upper()   # AttributeError if find_user_display returned None

   Fixes:
   - Change find_user_display to always return a string, or
   - Add a check before calling .upper():
       if display is None:
           raise ValueError("find_user_display returned None for user_id=%s" % user_id)
       return display.upper()

3) Offending code pattern with regex
   # app.py, line 120, function extract_label
   m = re.search(pattern, text)
   label = m.group(1)          # AttributeError if m is None (search failed)
   return label.upper()

   Fix:
   - Check for match:
       m = re.search(pattern, text)
       if not m:
           # handle no match
       label = m.group(1)
       return label.upper()

2. Identify the variable immediately left of .upper() (e.g., name, display, label).
3. Find where that variable is assigned earlier in the same function or passed in as an argument.
4. Ask: can that assignment produce None? (dict.get without default, function that sometimes returns None, failed regex match, etc.)
5. Choose one of these safe changes:
   - Provide a default value (dict.get(key, ''), or coerce with (value or "")).
   - Add an explicit check and raise a clear error if it’s None.
   - Fix the function that returned None so it returns a proper string.
