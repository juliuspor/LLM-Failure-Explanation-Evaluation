What happened
- The call DateTimeZone.for_offset_hours_minutes(-2, -15) fails because the method rejects a negative minutes argument.
- DateTimeZone.forOffsetHoursMinutes(int hours, int minutes) expects minutes in the range 0..59 and applies the sign from the hours parameter. Passing -15 for minutes triggers an argument validation error (IllegalArgumentException) inside DateTimeZone.forOffsetHoursMinutes.

Why it fails (root cause)
- The method validates its inputs and requires minutes to be non‑negative. It does not accept a separately signed minutes value; the sign for the whole offset comes from the hours parameter only. Passing a negative minutes value violates that validation and causes the method to throw.

Where it occurs
- The exception is thrown inside DateTimeZone.forOffsetHoursMinutes(...) (in DateTimeZone.java) when the code checks the minutes argument and finds it outside the allowed 0..59 range.

How to fix it (concrete steps)
1. Use a non‑negative minutes value and put the sign on the hours parameter. For -02:15 pass:
   DateTimeZone.forOffsetHoursMinutes(-2, 15);
2. If you only have a signed minutes value, convert to the canonical (hours, positiveMinutes) pair before calling:
   - Example conversion: given totalMinutes = -135 (for -2:15),
     int hours = Math.floorDiv(totalMinutes, 60); // -> -3
     int minutes = Math.abs(totalMinutes - hours * 60); // -> 45
     // then call DateTimeZone.forOffsetHoursMinutes(hours, minutes)
   (Or compute hours = -2 and minutes = 15 if you already know them.)
3. Alternatively, use DateTimeZone.forOffsetMillis(...) if you prefer to build the offset from a signed millisecond value.

Summary
- Don’t pass a negative minutes argument. Put the sign on the hours parameter (or convert your signed total minutes into the hours + non‑negative minutes form) and the call will succeed.
