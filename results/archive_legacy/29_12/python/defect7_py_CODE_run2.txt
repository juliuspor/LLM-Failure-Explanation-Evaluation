What happened (brief)
- At runtime the method to_class() raises an AttributeError on this line inside the loop:
  _ = array[i].upper()
- The exception message will be like: "'int' object has no attribute 'upper'" or "'NoneType' object has no attribute 'upper'".
- It fails because .upper() is being called on values that are not strings (or are None).

Where it happened (exact lines / method)
- Method: to_class
- Problem line: the loop body, specifically the call to array[i].upper()
- The next line classes.append(type(array[i])) executes only if the previous line succeeds, so the AttributeError prevents the method from returning.

Why it happened (root cause, clearly)
- .upper() is a string method. If an element of array is not a str (for example an int, a dict, or None), Python raises AttributeError because that object does not implement .upper().
- The code does not check the element type before calling .upper(), so any non-string element will break the function.
- Likely intent confusion: the method name to_class suggests you want the element class/type. But calling .upper() suggests you expected strings. The code mixes those two intentions: it calls .upper() (string-only) but then appends type(array[i]) (works for any object).

Concrete fixes (exact steps and replacement code)
Pick one of these 3 fixes depending on intent:

1) If you want to convert string names to uppercase and still return their types:
- Check for string before calling .upper(), or convert to str() if that is acceptable.
- Example replacement inside loop:
    for i in range(len(array)):
        if array[i] is None:
            classes.append(type(None))
            continue
        if isinstance(array[i], str):
            s = array[i].upper()        # safe: only called for strings
            # do something with s if needed
        classes.append(type(array[i]))

2) If you simply want a list of each element's type (no .upper() needed):
- Remove the .upper() line entirely and use a clearer construct:
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        return [type(x) for x in array]

3) If you want to treat string elements as class names and convert them to actual classes:
- You must look up classes by name (import or use globals/locals) instead of .upper():
    def to_class(cls, array: Optional[List[Any]]) -> Optional[List[type]]:
        if array is None:
            return None
        if len(array) == 0:
            return ArrayUtils.EMPTY_CLASS_ARRAY.copy()
        classes = []
        for elem in array:
            if isinstance(elem, str):
                name = elem.strip()
                # example: lookup in globals() or importlib to resolve to a class
                cls_obj = globals().get(name)
                if cls_obj is None:
                    raise LookupError(f"class named {name} not found")
                classes.append(cls_obj)
            elif isinstance(elem, type):
                classes.append(elem)
            else:
                classes.append(type(elem))
        return classes

Notes
- Decide which intent you want (1 string processing + types, 2 only types, 3 name → class) and apply the corresponding fix.
- Do not call .upper() without confirming the element is a str; that is the immediate cause of the failure.
