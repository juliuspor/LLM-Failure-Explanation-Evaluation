What happened
- The failing test calls GrayPaintScale.get_paint(-0.5) and expects black (0,0,0) for negative inputs.
- The method get_paint computes an index from the input value (e.g. index = int(value * 255)) and then returns a color from a precomputed palette/list using that index.
- Because the code does not clamp the input value to the [0,1] range before making the index, a negative value becomes a negative index. In Python a negative index selects from the end of the list (e.g. -1 returns the last entry) instead of producing black. That is why get_paint(-0.5) returns the wrong color and the assert fails.

Where it occurs (specific method/line)
- In GrayPaintScale.get_paint: the line that computes the palette index from value (for example, index = int(value * 255) or index = floor(value * N)) is executed before any clamping.
- Immediately after that, the code returns self._map[index] (or self.palette[index]) using the un-clamped index.
- The root cause is the missing clamping of value in get_paint before converting it to an index.

Why this is the root cause (clear, no jargon)
- Negative numbers make the index negative.
- Python negative indices select elements from the end of a list, so a negative input returns a palette color from the end, not black.
- The test expects out-of-range inputs to be treated as boundary values (negative → 0 → black).

Precise fix (steps you can apply)
1. In GrayPaintScale.get_paint, clamp the incoming value to the [0.0, 1.0] range before computing an index. For example, add these lines before you compute index:
   - value = max(0.0, min(1.0, value))
2. Then compute the index from the clamped value, e.g.:
   - index = int(value * (len(self._map) - 1))
   - return self._map[index]
3. Add unit tests for boundary and out-of-range values:
   - assert gps.get_paint(-0.5) == (0,0,0)
   - assert gps.get_paint(0.0) == (0,0,0)
   - assert gps.get_paint(1.0) == (255,255,255) or your expected white
   - assert gps.get_paint(1.5) == (255,255,255)

Exact example patch (3 lines to add / change)
- Change from:
  - index = int(value * 255)
  - return self._map[index]
- To:
  - value = max(0.0, min(1.0, value))          # clamp input
  - index = int(value * (len(self._map) - 1))
  - return self._map[index]

Why this fix is correct
- Clamping forces negative inputs to 0 and >1 inputs to 1, so the index is always inside the palette range and negative indexing cannot happen.
- This matches the test expectation that out-of-range inputs map to the nearest boundary color.
