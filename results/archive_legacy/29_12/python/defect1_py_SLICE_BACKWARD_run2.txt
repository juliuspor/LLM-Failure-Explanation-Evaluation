What happened (where it failed)
- Failure occurs at lines 400–401 in the function that contains this code:
  400 |         if minutes_offset < 0 or minutes_offset > 59:
  401 |             raise ValueError(f"Minutes out of range: {minutes_offset}")
- The code raises ValueError whenever minutes_offset is negative or greater than 59.

Why it failed (root cause)
- minutes_offset in this code is not the minute "component" (0..59). Instead it can be a signed total-minute offset (for example, -30, -210, or 330) coming from a timezone offset calculation or from caller data.
- The check on lines 400–401 treats a negative total offset (e.g. -30 for UTC−00:30 or -210 for UTC−03:30) as invalid. That makes the function reject legitimate offsets that use a negative sign on the hours part but have a valid minute component.
- In short: the code validates the whole signed value as if it must be in [0,59], but the value being passed is a signed total number of minutes or a negative minute component — so the condition is wrong for the real meaning of minutes_offset.
