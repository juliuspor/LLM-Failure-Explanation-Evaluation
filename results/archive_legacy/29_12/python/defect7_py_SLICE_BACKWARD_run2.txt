What happened (where): At line 769 your code calls array[i].upper(). That call is the failing operation.

Why it failed (root cause): One or more elements in array are not str objects, so they do not have an upper() method. Python raises an AttributeError (or a TypeError if the value is None) when you call .upper() on a non-string. The code uses indexing on every element (for i in range(len(array))) and assumes every array[i] is a string — that assumption is false.

How you can confirm that diagnosis
- Re-run the loop and print the type and value just before the call:
  - Insert before line 769: print(i, type(array[i]), repr(array[i]))
  - You will see the non-string element(s) (for example int, NoneType, bytes).

Concrete fixes (pick one that fits your intent)
1) Convert everything to string first (if you want a string representation of every element):
   - Replace line 769 with: array[i] = str(array[i]).upper()
   - Effect: safe for ints, None, etc.; modifies array in-place.

2) Only call .upper() on true strings (if you only want to upper-case existing strings):
   - Replace line 769 with:
     if isinstance(array[i], str):
         array[i] = array[i].upper()
   - Effect: non-strings are left unchanged.

3) Filter out non-strings before iterating (if non-strings are unexpected and should be ignored):
   - Before the loop: array = [x for x in array if isinstance(x, str)]
   - Then run your loop as-is.

4) Catch and handle the error at the call site:
   - Replace lines 768–769 with:
     for i in range(len(array)):
         try:
             array[i] = array[i].upper()
         except AttributeError:
             # handle or skip non-strings
             pass

Which to choose
- Use #1 if you want a string for every element.
- Use #2 if non-strings should remain as-is.
- Use #3 if non-strings are invalid input you want removed.
- Use #4 if you want to log or handle unexpected types but continue processing.
