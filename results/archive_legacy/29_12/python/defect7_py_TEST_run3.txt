What happened
- The test calls ClassUtils.to_class(['Test', None, 99.0]) and expects [str, type(None), float].
- The assertion fails because ClassUtils.to_class does not return the runtime types of the list items. Instead it interprets the string 'Test' as a class name and returns whatever class object it resolves to (for example a user-defined Test class), or otherwise handles strings specially. That makes the first element of result a different class than builtins.str, so result != [str, type(None), float].

- For None and 99.0 the method either returns type(None) and float (or computes types correctly), so only the string case is wrong.
- In short: a semantic mismatch between what the test expects (the runtime type of each element) and what ClassUtils.to_class implements (treats strings as class names to look up).
