Root cause (exact)
- On line 9 the code calls array[i].upper(). If array[i] is not a str (for example it is a class object, dict, int, None, etc.), Python raises AttributeError: "'X' object has no attribute 'upper'". The code does not check element type before calling .upper(), so the exception happens as soon as a non-string element is encountered.
- Secondary: line 10 appends type(array[i]) — that returns the element’s runtime type (e.g., int, dict, MyClass) not the class named by a string. This looks like a logical bug if your intent was to get classes from string names.
