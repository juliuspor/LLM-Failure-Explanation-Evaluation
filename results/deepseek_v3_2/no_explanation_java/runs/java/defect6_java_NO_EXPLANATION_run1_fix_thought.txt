The bug is in the condition `Math.abs(x) >= 100`. Since `x` is a double, but we are dealing with its integer representation after casting to long, we should use `Math.abs(value)` where `value` is the long cast. This ensures correct integer absolute value comparison. Also, the while loop condition uses `Math.pow` which returns double, but the comparison with `value` (long) is safe due to integer equality after scaling. The fix changes `Math.abs(x)` to `Math.abs(value)`.
