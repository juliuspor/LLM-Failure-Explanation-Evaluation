The original function incorrectly validates locale strings of length 7 or more. It should allow any length >= 7 for the variant part, not just exactly 7. The condition `len < 7` incorrectly rejects strings like 'en_GB_xxx' (len=9). Changed to `len < 7` to `len < 8` because valid lengths are 2, 5, or >=7 (since variant must be at least one character after the second underscore). Also need to ensure variant part is extracted correctly from index 6 onward.
