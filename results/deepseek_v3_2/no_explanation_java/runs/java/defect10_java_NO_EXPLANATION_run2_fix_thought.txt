Fixed two bugs: 1) The condition `!Character.isDigit(lastChar)` incorrectly treated '.' as a type qualifier, causing numbers like "1.23" to be mis-handled. Added `&& lastChar != '.'` to exclude decimal point. 2) In the exponent extraction for non-digit lastChar, the substring end index was off by one (should be `val.length()` not `val.length() - 1`). Fixed to `val.substring(expPos + 1, val.length())`.
