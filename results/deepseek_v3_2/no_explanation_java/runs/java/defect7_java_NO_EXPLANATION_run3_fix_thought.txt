The original `toClass` method did not handle null elements in the input array correctly. The documentation states: "If any of these objects is null, a null element will be inserted into the array." However, the original code called `array[i].getClass()` for every element, which would throw a NullPointerException when `array[i]` is null. The fix adds a null check inside the loop and sets the corresponding output element to null when the input element is null.
