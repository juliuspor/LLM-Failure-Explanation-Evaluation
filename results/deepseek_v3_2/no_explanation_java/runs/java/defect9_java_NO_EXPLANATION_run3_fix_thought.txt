The original `replaceEach` method had a bug where it could cause an infinite loop when replacements create new matches that were already processed. The fix ensures that after each replacement, we continue scanning from the end of the replacement string to avoid reprocessing the same text. Also, we need to handle the case where the replacement string contains the search string, which could cause an endless loop. The method now correctly updates the start position after each replacement and uses a while loop to find the next earliest match.
