The bug is in the `translate` method's loop when handling consumed codepoints. The original code incorrectly uses `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. This recalculates the codepoint at the same `pos` each iteration, leading to incorrect advancement and potential infinite loop. Instead, we should advance `pos` by the code units of each consumed codepoint sequentially. Also, when consumed==0, we should advance by `char_count(codepoint)` not `len(c)`. The fix adjusts the loop to correctly advance `pos` based on consumed codepoints.
