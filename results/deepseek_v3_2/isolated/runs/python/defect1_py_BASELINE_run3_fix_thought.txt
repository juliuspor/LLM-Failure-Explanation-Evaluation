The bug was that minutes_offset validation incorrectly restricted to 0-59, but negative minutes are allowed when hours_offset is negative (e.g., (-2, -15) should produce -02:15). Changed validation to allow -59 to 59. The existing logic for combining hours and minutes already handles negative minutes correctly: when hours_in_minutes is negative, it subtracts minutes_offset, so if minutes_offset is negative, subtracting a negative adds its magnitude, resulting in the correct total offset in minutes.
