The bug was that -0.0 was being output as 0 because the code didn't handle negative zero specially. The existing logic for integer-like numbers (x == int(x)) treats -0.0 as equal to 0, so it outputs '0'. To fix, we check for negative zero using the existing is_negative_zero method and output '-0.0' explicitly before the integer path. This preserves the sign while keeping other behavior unchanged.
