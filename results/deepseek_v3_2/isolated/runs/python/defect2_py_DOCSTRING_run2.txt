The code fails because the `_map_to_grayscale` method does not clamp the input value to the scale's bounds before using it in the linear interpolation formula. The docstring states that values outside the bounds should be clamped, but the method directly uses `value` in `self.bounds[0] + (value - self.bounds[0]) * t`. If `value` is outside the bounds, the computed `t` will be outside 0 to 1, leading to a color component outside 0 to 255, which triggers a ValueError. The issue occurs in the line that calculates `t` and the subsequent color calculation. The fix is to clamp `value` to `self.bounds[0]` and `self.bounds[1]` before computing `t`.
