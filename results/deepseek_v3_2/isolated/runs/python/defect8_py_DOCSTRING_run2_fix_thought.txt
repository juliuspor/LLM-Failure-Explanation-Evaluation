The bug is that the function only accepts underscores as separators, but the input uses a hyphen. The fix normalizes the input by replacing hyphens with underscores before parsing. This allows both underscore and hyphen separators, which is common in locale strings. The validation logic is updated to handle variable-length language and country codes (ISO 639 allows 2-3 letters, ISO 3166 allows 2-3 letters or digits). The original code had strict length checks (2,5,7) that were too rigid; now we split and validate each part independently.
