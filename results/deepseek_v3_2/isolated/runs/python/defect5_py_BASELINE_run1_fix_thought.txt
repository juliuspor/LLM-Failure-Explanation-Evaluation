The bug is that the type checking logic in `ArrayUtils.add` incorrectly raises a TypeError when `expected_type` is provided and `inferred_type` is `object`. This mimics Java's array type casting restrictions, but in Python, lists are dynamically typed and can hold any type, so this check is unnecessary and causes the test to fail. The fix is to remove the type checking block entirely, as it's not applicable in Python. The function should simply append the element to a copy of the array (or create a new list if array is None) and return it.
