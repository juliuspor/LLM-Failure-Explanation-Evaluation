The bug is that negative zero (-0.0) is incorrectly output as "0" because the integer branch does not preserve the sign. The fix is to check for negative zero using the existing `is_negative_zero` method before entering the integer branch. If x is negative zero, we output "-0.0" (or "-0" but to be safe we output "-0.0" to indicate it's a float). This ensures the sign is preserved. The rest of the logic remains unchanged.
