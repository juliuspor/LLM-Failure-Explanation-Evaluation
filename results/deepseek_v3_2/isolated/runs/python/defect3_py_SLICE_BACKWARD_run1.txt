The function `to_utf16_code_units` returns a list of UTF-16 code units (integers). The error occurs at line 35-36 because the variable `index` is being checked against the length of `code_units`, but `index` is intended to be an index into the original string `s`, not into the `code_units` list. Since some Unicode characters (like emojis or certain symbols) are represented by two UTF-16 code units (a surrogate pair), the `code_units` list can be longer than the original string. Therefore, a valid string index can be out of range for `code_units`, causing an incorrect IndexError.
