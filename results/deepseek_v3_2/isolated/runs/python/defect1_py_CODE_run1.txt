The code fails because on line 10, when `hours_offset` is negative, the calculation incorrectly subtracts `minutes_offset` from `hours_in_minutes`. For example, if `hours_offset = -1` and `minutes_offset = 30`, `hours_in_minutes = -60`. The condition `hours_in_minutes < 0` is true, so it computes `minutes_offset = -60 - 30 = -90`. This is wrong because the total offset in minutes should be `-60 + 30 = -30`. The sign handling is reversed. The correct logic should add the two components, respecting their signs: `total_minutes = hours_in_minutes + minutes_offset` regardless of sign. The current `else` branch already does this for positive hours, but the `if` branch introduces the error.
