The bug was in the sign handling when combining hours and minutes. The original code incorrectly adjusted minutes_offset based on the sign of hours_in_minutes. For negative hours, it set minutes_offset = hours_in_minutes - minutes_offset, which would double-count the negative sign. For positive hours, it set minutes_offset = hours_in_minutes + minutes_offset, which is correct. The fix is to compute total_minutes as hours_offset * 60, then add or subtract minutes_offset based on the sign of hours_offset. Since minutes_offset is non-negative (0-59), for negative hours we subtract minutes_offset, and for positive hours we add minutes_offset. This yields the correct total minutes, which is then converted to milliseconds.
