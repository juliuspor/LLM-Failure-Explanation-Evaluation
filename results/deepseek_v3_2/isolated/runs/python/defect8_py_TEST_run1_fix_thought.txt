The bug is that `to_locale` incorrectly validates the country code when the locale string has an empty country (two underscores). The current code assumes that after the first underscore, positions 3 and 4 are uppercase letters, but in 'fr__POSIX', position 3 is '_' (the second underscore) and position 4 is 'P', which fails the uppercase check. The fix is to detect when the country part is empty (i.e., the third character is '_' and the fourth is also '_'), and then treat the country as empty string, with the variant starting after the second underscore. We need to adjust the parsing logic to handle this case.
