The bug occurs because `exp` can be `None` when there is no exponent in the input string, and `NumberUtils._is_all_zeros` is called with `None`, causing a TypeError. The fix is to check if `exp` is `None` before calling `_is_all_zeros`. In both places where `all_zeros` is computed (lines 44 and 78 in the original), we change the condition to `NumberUtils._is_all_zeros(mant) and (exp is None or NumberUtils._is_all_zeros(exp))`. This ensures that if there is no exponent, it is considered as all zeros (or at least doesn't cause an error).
