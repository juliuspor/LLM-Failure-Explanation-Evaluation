The bug is that `inferred_type` was set to `type(array)` (which is `list`), not the element type. The fix is to infer the element type from the first element of the array if it exists, otherwise default to `object`. This matches the intended behavior of checking element types for Java-like type safety. Also, the `_copy_list_grow1` method expects an element type, not the list type, so this change ensures the correct type is passed.
