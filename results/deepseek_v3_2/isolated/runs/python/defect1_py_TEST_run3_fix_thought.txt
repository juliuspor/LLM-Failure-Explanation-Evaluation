The bug was in the sign handling of minutes_offset when hours_offset is negative. The original code incorrectly adjusted minutes_offset based on the sign of hours_offset, which led to wrong total offset. The fix is to compute total_minutes as hours_offset * 60 + minutes_offset directly, which correctly handles mixed signs (e.g., -2 hours and +15 minutes yields -105 minutes). Also updated the minutes range check to allow negative minutes (down to -59).
