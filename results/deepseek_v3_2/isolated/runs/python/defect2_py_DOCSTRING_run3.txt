The code fails because the `_map_to_grayscale` method does not clamp the input value to the scale's bounds before using it in the calculation. The docstring states that values outside the bounds should be clamped, but the method directly uses `(value - self.lower) / (self.upper - self.lower)` without checking if `value` is less than `self.lower` or greater than `self.upper`. This can cause the computed `fraction` to be negative or greater than 1, leading to a `color_component` outside the range 0–255. When `color_component` is cast to an integer, it may become negative or exceed 255, triggering the ValueError because the check `if not (0 <= component <= 255)` fails. The issue occurs in the line where `fraction` is calculated, and the fix is to clamp `value` to `[self.lower, self.upper]` before computing `fraction`.
