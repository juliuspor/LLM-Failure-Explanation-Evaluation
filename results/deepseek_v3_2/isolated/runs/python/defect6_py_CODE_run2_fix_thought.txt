The bug is that the function uses `math.isinf` and `math.isnan` without importing the math module, causing a NameError. Also, the while loop can become infinite for x=0, but that case is excluded by the `abs(x) >= 100` guard. However, to be safe, we should handle the mantissa=0 case explicitly. Additionally, the check_val calculation uses integer division which works correctly for both positive and negative numbers. The fix adds the missing import and adds a guard to break the loop if mantissa becomes 0.
