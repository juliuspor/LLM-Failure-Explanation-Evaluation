The bug occurs when both `array` and `element` are `None` and an `expected_type` is provided. The current logic sets `inferred_type` to `object` in that case, which triggers a TypeError when `expected_type` is not `object`. The fix is to skip the type-checking error when `inferred_type` is `object` and `array` is `None` (i.e., we're creating a new list from scratch), because we can safely create a list with any type. We'll modify the condition to only raise the error if `array` is not `None` and `inferred_type` is `object` but `expected_type` is not `object`.
