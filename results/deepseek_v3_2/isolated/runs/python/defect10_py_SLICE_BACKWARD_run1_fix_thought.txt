The bug is in the `char_at` function, which incorrectly raises an IndexError for negative indices. In Python, negative indices are valid and access elements from the end. The condition should check if the index is out of the valid range for both positive and negative indices: index >= len(s) or index < -len(s). This ensures that indices like -1 are allowed while still raising an error for out-of-bounds indices like -len(s)-1.
