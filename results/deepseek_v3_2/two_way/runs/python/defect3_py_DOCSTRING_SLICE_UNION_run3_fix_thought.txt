The bug is that the `translate` method incorrectly handles the consumed codepoints. When `consumed > 0`, it should advance `pos` by the number of code units consumed, not by the number of codepoints. The loop `for pt in range(consumed):` incorrectly uses `char_count` on each codepoint, which could skip too many code units. Instead, we should advance `pos` by the total code units consumed, which can be computed by summing `char_count` for each consumed codepoint. However, the translator's `translate_codepoint` returns the number of codepoints consumed, not code units. So we need to convert that to code units by iterating through the consumed codepoints and adding their char counts. Also, the `translate` method should handle the case where `consumed` is zero by appending the current codepoint's code units and advancing by its char count (already done).
