The bug is in the for loop that advances `pos` after a non-zero consumption. The original code called `code_point_at(code_units, pos)` inside the loop but `pos` was not updated between iterations, leading to repeated calls on the same index. The fix is to update `pos` inside the loop by the char count of the codepoint at the current position, which changes as `pos` increments. This ensures we move forward by the correct number of code units for each consumed codepoint.
