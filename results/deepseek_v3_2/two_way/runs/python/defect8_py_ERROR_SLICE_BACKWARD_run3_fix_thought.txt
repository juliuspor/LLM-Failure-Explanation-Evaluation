The bug is that the original to_locale method only handles locale strings with exactly one underscore between language and country, and another underscore before variant. It fails on 'fr__POSIX' because there are two underscores before 'POSIX', which is a valid variant format. The fix is to make the method more flexible: allow multiple underscores in the variant part, and handle cases where country is empty but variant is present (like 'fr__POSIX'). We should parse by splitting on underscores, not by fixed positions.
