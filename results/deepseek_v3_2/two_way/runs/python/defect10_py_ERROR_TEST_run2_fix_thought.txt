The bug occurs when processing a single-character suffix like 'l' or 'f' because after removing the suffix, the resulting string is empty. The code then calls `char_at(numeric, 0)` where `numeric` is empty, causing an IndexError. The fix is to check if `numeric` is empty before accessing its first character, and raise a ValueError instead. Also, ensure that when `exp` is None, we don't try to access its length or characters incorrectly.
