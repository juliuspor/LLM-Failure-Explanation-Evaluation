The bug occurs because the original code assumes that for locale strings with length >= 5, the characters at indices 3 and 4 are uppercase letters representing the country code. However, the format language__variant (e.g., 'fr__POSIX') has an empty country code, indicated by two consecutive underscores. The fix is to detect this case by checking if the character at index 3 is an underscore. If so, treat the country as empty and parse the variant from index 4 onward.
