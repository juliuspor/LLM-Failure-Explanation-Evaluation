The bug is in the loop that processes consumed codepoints. When `translate_codepoint` returns a non-zero `consumed` (number of codepoints consumed), the code incorrectly advances `pos` by iterating over each codepoint and adding `char_count(code_point_at(...))`. However, `char_count` returns the number of UTF-16 code units for a codepoint, not the number of codepoints. This leads to misalignment when surrogate pairs are involved. Instead, we should directly advance `pos` by the number of UTF-16 code units consumed, which can be computed by summing `char_count` for each consumed codepoint. The fix is to compute the total code units consumed and add that to `pos`.
