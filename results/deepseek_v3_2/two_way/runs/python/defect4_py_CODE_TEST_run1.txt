Consequently, the `_max_middle_index` may not be updated correctly, leading to wrong results in tests like `test_max_middle_index_simple` where the expected max middle index is 1 but the bug causes it to be set incorrectly (e.g., 3). The fix is to change `self._min_middle_index` to `self._max_middle_index` on that line.
