The bug is in the for loop that advances `pos` after a translator consumes codepoints. It incorrectly uses `char_count(code_point_at(...))` inside the loop, which for a surrogate pair returns 2, but the loop iterates `consumed` times, causing double counting. Instead, we should advance `pos` by the total number of code units consumed, which can be computed by summing `char_count` for each consumed codepoint, or more simply by iterating once per consumed codepoint and incrementing `pos` by its char count. The fix is to replace the for loop with a direct calculation: iterate `consumed` times, each time adding `char_count(code_point_at(code_units, pos))` to `pos`.
