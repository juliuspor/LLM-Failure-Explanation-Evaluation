The bug is that negative zero (-0.0) is incorrectly treated as an integer because int(-0.0) == 0 and -0.0 == 0 is true. This causes the function to output '0' instead of '-0.0'. The fix is to check for negative zero before the integer path, using the existing is_negative_zero method. If it's negative zero, we output the string representation directly, which will be '-0.0'.
