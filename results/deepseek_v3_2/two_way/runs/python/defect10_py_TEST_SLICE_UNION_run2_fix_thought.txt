The bug was that for inputs like "l" or "L" alone, the method would call `char_at(numeric, 0)` with an empty string, causing an IndexError instead of a ValueError. The fix restructures the 'l'/'L' branch to check if the numeric part is empty and then raise ValueError. Also, we ensure that the numeric part is a valid integer (with optional sign) before attempting conversion. This matches the expected behavior from the test suite.
