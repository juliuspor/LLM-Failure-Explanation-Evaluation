The bug was in the sign handling for mixed-sign inputs (negative hours, positive minutes). The original code incorrectly computed minutes_offset = hours_in_minutes - minutes_offset when hours_in_minutes < 0, but hours_in_minutes is already negative, so subtracting positive minutes makes it more negative, which is correct. However, the variable name 'minutes_offset' was reused, causing confusion. The fix clarifies the logic by using a new variable 'total_minutes' and explicitly handling the sign: if hours_offset is negative, subtract minutes_offset; otherwise, add minutes_offset. This ensures that (-2, 15) yields -135 minutes total, which is correct for -02:15.
