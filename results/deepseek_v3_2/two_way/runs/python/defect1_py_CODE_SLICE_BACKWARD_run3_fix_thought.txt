The bug occurs because the variable `minutes_offset` is reused for both the input minutes component and the total offset in minutes. When `hours_offset` is negative, the calculation `minutes_offset = hours_in_minutes - minutes_offset` can produce a negative total minutes value, which would have passed the initial range check (0-59) but is invalid for the total offset. The fix is to use a separate variable (`total_minutes`) to hold the computed total offset in minutes, preserving the original `minutes_offset` for validation and clarity.
