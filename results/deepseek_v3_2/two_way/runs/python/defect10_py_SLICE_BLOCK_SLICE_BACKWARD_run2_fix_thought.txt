The bug is in the `char_at` function, which incorrectly raises an IndexError for negative indices. In Python, negative indices are valid and access characters from the end. The condition should only raise an error if the index is out of the valid range for both positive and negative indices. For a string of length n, valid indices are -n <= index < n. So we need to adjust the condition to check if index is less than -len(s) or index >= len(s).
