The bug was in the calculation of total minutes when hours_offset is negative. The original code incorrectly subtracted minutes_offset from hours_in_minutes for negative hours, leading to wrong total offset. The fix is to always add hours_in_minutes and minutes_offset (since minutes_offset can be positive or zero) and store the result in a new variable total_minutes to avoid modifying the parameter. This ensures correct sign handling for both positive and negative hour offsets.
