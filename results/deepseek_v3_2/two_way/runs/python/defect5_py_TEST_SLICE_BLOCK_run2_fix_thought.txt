The bug occurs because when both array and element are None, inferred_type becomes object, and the condition `inferred_type == object and expected_type != object:` raises a TypeError incorrectly. The fix is to skip the type check when element is None, because None is assignable to any type. Also, we should avoid raising the error when the inferred_type is object but the element is None. We'll adjust the condition to only raise if the element is not None and the types mismatch.
