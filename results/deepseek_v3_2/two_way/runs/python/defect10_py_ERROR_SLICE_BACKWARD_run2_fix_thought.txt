The bug occurs when `numeric` is an empty string (e.g., input "L") and `char_at(numeric, 0)` is called, causing an IndexError. The fix adds a length check before accessing the first character to ensure the string is not empty. Specifically, in the 'L' qualifier branch, changed the condition to `len(numeric) > 0 and char_at(numeric, 0) == "-"` to prevent the call on an empty string.
