The bug is that `type(array)` returns `list`, not the element type. To infer the element type, we should check the type of the first element if the array is non-empty, or fall back to the type of the element being added. However, since Python lists are heterogeneous, we cannot guarantee a single element type. The original Java version uses component type checks, but in Python we can't replicate that exactly. The safest fix is to remove the type inference logic and rely on Python's dynamic typing, but keep the `expected_type` check for compatibility. We'll set `inferred_type` to `object` by default, and only raise TypeError if `expected_type` is provided and not `object`.
