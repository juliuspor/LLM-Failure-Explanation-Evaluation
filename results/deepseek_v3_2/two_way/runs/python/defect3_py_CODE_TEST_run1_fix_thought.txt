The bug is in the while loop's index advancement after a translator consumes characters. The inner for loop incorrectly uses the same `pos` value each iteration, causing infinite loop or wrong advancement. The fix is to advance `pos` by the total number of code units consumed, not by iterating over codepoints. Since `translate_codepoint` returns the number of codepoints consumed, we need to convert that to code units. We can compute the total code units consumed by summing `char_count` for each consumed codepoint, but we must increment `pos` as we go. Simpler: use a separate variable to count consumed code units while processing the consumed codepoints.
