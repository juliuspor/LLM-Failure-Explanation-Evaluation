The bug is that `add_number` does not handle negative zero correctly. The method uses `x == int(x)` which is true for -0.0 (since int(-0.0) == 0) and then proceeds to output `str(value)` where `value` is 0, losing the sign. The fix is to add a special case for negative zero using the existing `is_negative_zero` static method. If the number is negative zero, we output `str(x)` directly, which in Python yields "-0.0". This preserves the sign. The rest of the logic remains unchanged.
