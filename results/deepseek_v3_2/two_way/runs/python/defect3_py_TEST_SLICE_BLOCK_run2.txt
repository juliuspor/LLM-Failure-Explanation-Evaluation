The test fails because the `code_point_at` function incorrectly handles surrogate pairs. When a high surrogate is found at position `index`, the function attempts to read the next code unit at `index + 1` to check for a low surrogate. However, the bounds check at line 35 (`if index < 0 or index >= len(code_units):`) only validates the initial `index`, not `index + 1`. If the high surrogate is at the last position of the `code_units` list (e.g., when the input string ends with a high surrogate), accessing `code_units[index + 1]` will cause an `IndexError` because `index + 1` is out of range. This occurs in the test `test_translate_surrogate_pair_with_suffix` because the emoji \U0001F630 is a surrogate pair (two UTF-16 code units), and the function incorrectly advances the index when processing it, leading to an out-of-bounds access.
