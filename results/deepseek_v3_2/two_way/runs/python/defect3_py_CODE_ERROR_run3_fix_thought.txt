The bug is in the loop that advances `pos` after a translator consumes codepoints. The loop incorrectly updates `pos` inside the iteration, causing `code_point_at` to be called on a moving index, which can lead to out-of-range errors or skipped characters. The fix is to compute the total increment based on the original position and the number of codepoints consumed, without re-evaluating `code_point_at` on the updated `pos` each time. We'll replace the for-loop with a while loop that accumulates the increment correctly.
