The code fails because the logic for handling negative hour offsets incorrectly subtracts the minutes_offset on line 383. When hours_offset is negative, the variable hours_in_minutes is negative (since it is hours_offset * 60). The code then does minutes_offset = hours_in_minutes - minutes_offset, which makes the total offset more negative than intended. For example, if hours_offset = -1 and minutes_offset = 30, hours_in_minutes becomes -60. The calculation becomes minutes_offset = -60 - 30 = -90 minutes, which is an offset of -1 hour and 30 minutes, but the correct offset should be -1 hour and +30 minutes, totaling -30 minutes. The correct operation should be addition: minutes_offset = hours_in_minutes + minutes_offset, as is correctly done for positive offsets on line 385. This error causes the resulting millisecond offset to be wrong for negative hour offsets with a non-zero minutes component.
