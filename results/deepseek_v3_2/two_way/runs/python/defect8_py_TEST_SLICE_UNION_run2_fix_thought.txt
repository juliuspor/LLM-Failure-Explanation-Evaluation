The bug is that `to_locale` incorrectly validates indices 3 and 4 as uppercase letters when the locale string contains a double underscore (empty country) and a variant (e.g., 'fr__POSIX'). The fix is to detect the double underscore pattern: if the character at index 3 is '_', then the country is empty and the variant starts at index 4. We must adjust the validation and parsing accordingly.
