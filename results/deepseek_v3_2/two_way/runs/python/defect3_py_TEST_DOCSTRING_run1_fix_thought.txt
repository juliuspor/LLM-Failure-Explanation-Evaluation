The bug was in the loop that advances the position after a translator consumes codepoints. The original code used `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`, but this incorrectly recalculates the codepoint at the same position each iteration, leading to misalignment. The fix ensures that for each consumed codepoint, we get the codepoint at the current position and advance by its char count, updating the position correctly for surrogate pairs. Also, in the 'no translation' branch, we now use `char_count(codepoint)` instead of `len(c)` to advance consistently.
