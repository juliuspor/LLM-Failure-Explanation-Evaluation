The test `test_lower_bound_clamping` expects negative values to be clamped to black (0,0,0), but the code at line 74 calculates `g` directly from the formula without clamping. When `value` is -0.5 and default bounds are 0.0 and 1.0, the calculation becomes `(-0.5 - 0.0) / (1.0 - 0.0) * 255.0 = -127.5`, which is cast to `int(-127.5) = -127`. Then, at line 77, the condition `g < 0` is true, so a `ValueError` is raised instead of clamping to 0. The same issue occurs for values above the upper bound, where `g` exceeds 255 and triggers the same error. The code fails because it validates the computed `g` instead of clamping the input `value` to the bounds before computation.
