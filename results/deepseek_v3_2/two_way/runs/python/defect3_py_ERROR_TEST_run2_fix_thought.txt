The bug is in the `translate` method's loop when handling consumed codepoints. The original code incorrectly advances `pos` by iterating over consumed codepoints and adding `char_count(code_point_at(code_units, pos))` each time, but `pos` is not updated within the loop, leading to repeated calculation of the same codepoint. Moreover, the loop should advance by the total code units corresponding to the consumed codepoints. The fix computes the total code units for all consumed codepoints before updating `pos`. Also, when `consumed == 0`, we already advance by `char_count(codepoint)`, which is correct. The fix ensures proper handling of surrogate pairs and avoids IndexError.
