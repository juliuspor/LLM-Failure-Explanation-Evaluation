The bug is that `to_locale` fails to parse locale strings with double underscores (empty country), like 'fr__POSIX'. The method incorrectly validates characters at positions 3 and 4 as uppercase letters, but in 'fr__POSIX', position 3 is '_'. The fix is to detect the double underscore pattern and treat the country as empty, then parse the variant after the second underscore. We need to adjust the validation logic to allow an underscore at position 3 when followed by another underscore, indicating an empty country.
