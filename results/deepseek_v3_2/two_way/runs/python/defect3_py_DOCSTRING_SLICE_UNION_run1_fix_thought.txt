The bug is in the `translate` method: when `consumed == 0`, it incorrectly handles surrogate pairs by using `len(c)` as the increment, but `c` is the UTF-16 representation of a single codepoint, not the number of codepoints consumed. The loop at the end also incorrectly increments `pos` by `char_count` for each consumed codepoint, but `consumed` is the count of codepoints, not code units. The fix is to track code unit consumption directly. We'll compute the number of code units consumed by iterating through the consumed codepoints and summing their `char_count`. Also, when `consumed == 0`, we should consume exactly one codepoint (which may be 1 or 2 code units).
