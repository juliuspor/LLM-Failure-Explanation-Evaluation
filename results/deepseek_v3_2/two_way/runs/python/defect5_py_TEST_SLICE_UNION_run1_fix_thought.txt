The bug occurs when both `array` and `element` are `None`. In this case, `inferred_type` is set to `object`. Then, if `expected_type` is provided (e.g., `str`), the condition `inferred_type == object and expected_type != object` triggers a `TypeError`. However, when `array` is `None`, there is no existing array to cast, so the type check should be skipped. The fix is to only raise the `TypeError` when `array` is not `None`. This ensures that the function returns `[None]` as expected when both inputs are `None` and an `expected_type` is specified.
