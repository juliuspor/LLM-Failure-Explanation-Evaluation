The bug is that `inferred_type` was incorrectly set to the container type (e.g., `list`) instead of the element type. The fix changes the logic to infer the element type: if `array` is not None and non-empty, use the type of the first element; if empty, infer from `element` or default to `object`. This ensures the type check condition `inferred_type == object` works as intended, raising a TypeError when expected_type is provided and the inferred element type is generic object (i.e., unknown).
