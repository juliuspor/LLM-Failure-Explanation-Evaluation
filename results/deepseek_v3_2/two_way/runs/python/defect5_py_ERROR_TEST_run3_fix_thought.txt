The bug is that `ArrayUtils.add` raises a `TypeError` when `array` is `None`, `element` is `None`, and `expected_type` is not `object`. This happens because `inferred_type` becomes `object` in that case, triggering the exception. The fix is to only raise the exception when `array` is not `None`, because if the array is `None`, there is no existing array to cast, so no `ClassCastException` should occur. This matches Java behavior where casting a null reference is allowed. Also, `None` (null) is assignable to any type, so no error should be raised for the element being `None`. The condition is changed to only raise when `inferred_type == object` and `expected_type != object` and `array is not None`. This ensures the test passes while preserving the simulation for non-null arrays.
