The bug is that the method `random` does not validate that `start < end`. When `start >= end`, the half-open interval is empty, leading to an invalid range. The test expects a ValueError with a message containing both 'start' and 'end'. The fix adds a check at the beginning of the method, raising a ValueError with a message that includes both words. The message format is chosen to match typical Java style, but any message containing 'start' and 'end' should satisfy the test. The rest of the method remains unchanged.
