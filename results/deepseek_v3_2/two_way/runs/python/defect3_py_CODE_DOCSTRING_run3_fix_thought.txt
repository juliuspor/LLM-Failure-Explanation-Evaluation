The bug is in the for loop that increments `pos` for each consumed codepoint. The loop uses the same `pos` value in each iteration to compute `char_count`, so it doesn't advance to the next codepoint. Instead, we should compute the codepoint at the current position, get its char count, and increment `pos` by that amount, then repeat for each consumed codepoint. Alternatively, we can directly increment `pos` by the total number of code units consumed, but the contract expects consumed codepoints, not code units. The fix is to compute the codepoint at the current position, get its char count, increment `pos` by that, and repeat for each consumed codepoint.
