The bug was in the `_update_bounds` method where the max middle index update incorrectly referenced `self._min_middle_index` instead of `self._max_middle_index`. Additionally, the original code did not handle ties properly, potentially causing the wrong index to be stored when timestamps were equal. The fix corrects the variable reference and adds tie-breaking logic to choose the smallest index for minima and the largest index for maxima when values are equal, ensuring deterministic behavior.
