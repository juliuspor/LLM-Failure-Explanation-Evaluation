The code fails because the `code_point_at` function incorrectly handles surrogate pairs when called from `CsvEscaper.translate_string`. The `translate_string` method iterates over the input string using an index `i` and calls `code_point_at` to get the Unicode code point at that index. For characters that are part of a surrogate pair (like emojis), `code_point_at` returns the full code point and expects the caller to advance the index by 2 (since a surrogate pair consists of two UTF-16 code units). However, the `translate_string` method only advances `i` by 1 after each call, causing an `IndexError` when `code_point_at` is called with an index pointing to the second half of a surrogate pair (because it checks if `index >= len(code_units)`). Specifically, in the test `test_translate_surrogate_pair_with_suffix`, the input string "\U0001F630A" contains an emoji (surrogate pair) followed by 'A'. After processing the emoji, `i` is incremented by 1, pointing to the second UTF-16 unit of the emoji, but `code_point_at` is called again with that index, which is out of bounds for the code units array (since the emoji uses two code units). The error occurs at line 36 in `code_point_at` when the index is equal to `len(code_units)`.
