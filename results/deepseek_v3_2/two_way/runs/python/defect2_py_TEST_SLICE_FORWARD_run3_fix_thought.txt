The bug was that the original code used the unclamped 'value' in the grayscale calculation instead of the clamped 'v'. Also, rounding should be used to get the nearest integer (like Java's (int) Math.round(...)). Additionally, defensive clamping ensures the result is within 0..255. The fix uses the clamped value 'v' and rounds to nearest integer, then clamps to 0-255.
