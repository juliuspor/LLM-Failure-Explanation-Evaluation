The bug occurs because `substring_java` raises an IndexError when `end > len(s)`. In the failing test, `upper` is set to 20, which exceeds the string length. The Java original handles this by adjusting `upper` to `len(text)` earlier, but the condition `if upper == -1 or upper > len(text): upper = len(text)` is already present. However, the test passes `lower=15` and `upper=20`. Since `upper > len(text)`, `upper` becomes 10. Then `if upper < lower: upper = lower` sets `upper` to 15. Now `upper` (15) is still greater than `len(text)` (10), but `upper` is not reset again. The call to `substring_java` with `end=15` triggers the IndexError. The fix is to ensure `upper` never exceeds `len(text)` after adjusting for `lower`.
