The bug is in the loop after a translator returns consumed > 0. The original code used `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. However, `code_point_at` is called with the same `pos` each iteration, which is incorrect because `pos` should advance after each codepoint. The fix is to compute the codepoint at the current position, then increment pos by its charCount, repeated for each consumed codepoint. This ensures proper handling of surrogate pairs. Also, the zero-consumption branch already uses charCount, which is correct.
