The bug occurs because the original implementation assumes a strict format with exactly one underscore between language and country, and another between country and variant. However, locale strings like 'fr__POSIX' use double underscores to indicate an empty country code. The fix is to split the string on underscores and handle the parts more flexibly: language is the first part (must be 2 letters lowercase), country is the second part if present and length 2 (or empty if missing/empty), and variant is everything after the second underscore. This matches Java's Locale parsing behavior.
