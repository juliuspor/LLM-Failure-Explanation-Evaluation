The bug occurs when processing a string ending with 'L' (or 'l') that has no digits before it, e.g., "L" or "-L". In such cases, `numeric = val[:-1]` becomes an empty string. Then `char_at(numeric, 0)` is called, causing an IndexError. The fix is to add a check for `len(numeric) > 0` before accessing `char_at(numeric, 0)`. Specifically, in the condition for the 'L' case, we need to ensure that `numeric` is not empty when checking if the first character is '-'. The condition now reads: `(len(numeric) > 0 and char_at(numeric, 0) == "-" and NumberUtils.is_digits(numeric[1:]))`. This prevents the IndexError and ensures the logic only proceeds when `numeric` has at least one character. Additionally, the condition `NumberUtils.is_digits(numeric)` already handles the empty string case (returns False), so no change needed there. The fix is minimal and focused on the specific bug.
