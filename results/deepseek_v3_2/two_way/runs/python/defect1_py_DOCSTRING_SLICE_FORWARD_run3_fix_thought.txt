The bug is in the calculation of total minutes offset when hours_offset is negative. The code incorrectly subtracts minutes_offset from hours_in_minutes, but minutes_offset is always a positive value (0-59). The sign is already captured by hours_offset. The fix is to always add minutes_offset to hours_in_minutes, regardless of sign, because minutes_offset is a magnitude. So we replace the conditional subtraction with a simple addition.
