The bug was in the while loop's handling of consumed codepoints. When consumed == 0, the code incorrectly advanced pos by len(c) (the number of code units) instead of char_count(codepoint) (the number of codepoints). This could cause misalignment for surrogate pairs. Also, the loop for consumed > 0 had an issue: it recalculated codepoints at the moving pos, which could skip code units incorrectly. Fixed by advancing pos by char_count(codepoint) in the consumed == 0 case and using a cumulative count for consumed > 0.
