The bug is that `to_locale` incorrectly handles locale strings with empty country (two underscores). The current logic assumes a single underscore between language and country, and another between country and variant. For 'fr__POSIX', after verifying the language part, we need to check if the character after the first underscore is another underscore (indicating empty country). If so, treat the part after the double underscore as variant. Also need to adjust length checks to allow this pattern.
