The bug was that minutes_offset validation only allowed 0-59, but negative minutes are needed for offsets like UTC+05:45 (positive minutes) and UTC-09:30 (negative minutes). The fix expands the validation to allow -59 to 59. Also, the calculation of total minutes needed adjustment: when hours_offset is negative, we need to add minutes_offset (which could be negative) correctly. The original code incorrectly subtracted minutes_offset when hours_in_minutes < 0, which would double-count the negative sign. Actually, we should simply add hours_in_minutes and minutes_offset, respecting their signs. So we can compute total_minutes = hours_offset * 60 + minutes_offset, then multiply by MILLIS_PER_MINUTE. This handles both positive and negative minutes correctly.
