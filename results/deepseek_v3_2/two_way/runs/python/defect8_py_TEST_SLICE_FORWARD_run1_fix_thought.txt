The original implementation assumed a fixed structure with underscores at specific positions, which failed for locales with empty country codes (double underscores). The fix uses a more robust parsing approach: split by underscore and handle the parts. We validate the language code (2 letters, lowercase). For the second part, we check if it's empty (indicating empty country) or a valid 2-letter uppercase country code. If it's neither, we treat it as a variant (for backward compatibility with non-standard inputs). The third part, if present, is the variant (unless the second part was empty, then the third part is the variant). This handles 'fr__POSIX' correctly: language='fr', country='', variant='POSIX'.
