The bug is that negative zero (-0.0) is incorrectly emitted as '0' instead of '-0.0'. The fix is to detect negative zero using the existing is_negative_zero method (which uses math.copysign) and handle it before the integer conversion branch. If x is negative zero, we output '-0.0' directly and return early. This preserves the sign and the float representation. The rest of the function remains unchanged for other numbers.
