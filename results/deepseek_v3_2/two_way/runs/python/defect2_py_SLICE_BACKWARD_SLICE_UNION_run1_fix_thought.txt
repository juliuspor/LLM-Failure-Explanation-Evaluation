The bug is that the `__init__` method already validates lower_bound < upper_bound, but the `get_paint` method uses the clamped value `v` for bounds but then uses the original `value` in the calculation. This means that even though `v` is clamped, the formula uses `value` directly, which could be outside the bounds, leading to a `g` value outside 0-255. The fix is to use the clamped value `v` in the grayscale calculation instead of the original `value`.
