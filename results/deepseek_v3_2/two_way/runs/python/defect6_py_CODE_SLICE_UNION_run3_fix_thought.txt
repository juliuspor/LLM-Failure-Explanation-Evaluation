The bug is due to incorrect indentation causing the `if exp > 2:` block to be nested inside the `if abs(x) >= 100:` block. This leads to `exp` being referenced only when that outer condition is true, but the logic intends to check `exp` regardless of the magnitude of `x`. The fix is to dedent the `if exp > 2:` block to be at the same level as the outer `if`, ensuring `exp` is always defined (initialized to 0) and the condition is evaluated correctly.
