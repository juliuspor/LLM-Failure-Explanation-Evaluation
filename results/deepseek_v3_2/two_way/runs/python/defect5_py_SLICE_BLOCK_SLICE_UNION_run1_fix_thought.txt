The bug is that `inferred_type` was set to the container type (e.g., `list`) instead of the element type. The fix is to infer the element type from the first element of the array (if exists) or from the element being added. Then, compare the element type (which could be `object`) with `expected_type`. The condition `first_element_type == object and expected_type != object` will now correctly simulate the Java array cast failure when the list is of type Object and we try to cast to a more specific type. Also, handle edge cases like empty array and None element.
