The bug is that the loop incorrectly advances `pos` when `consumed > 0`. The original code used `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. However, `code_point_at` expects an index into `code_units`, but after the first iteration, `pos` has already been incremented, so the second `code_point_at` call may be at an index that is not the start of a codepoint (e.g., index 1 of a surrogate pair). This leads to incorrect behavior and potential IndexError. The fix is to compute the total code units consumed by iterating from the original position without modifying `pos` until the end. Also, when `consumed == 0`, we already advance by `char_count(codepoint)`, which is correct. The fixed code uses a temporary variable `i` to accumulate the code unit count for the consumed codepoints, then adds it to `pos`.
