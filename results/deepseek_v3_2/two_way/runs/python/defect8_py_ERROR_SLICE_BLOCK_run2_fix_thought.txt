The original validation incorrectly assumed the third character (index 2) is an underscore and the fourth and fifth are uppercase letters, but that fails for locales like "fr__POSIX" where the country part is empty. The fix uses split('_', 2) to separate language, country, and variant, then validates each part appropriately. Language must be 2 lowercase letters. Country, if present, must be 2 uppercase letters. Variant can be any string. This handles empty country parts correctly.
