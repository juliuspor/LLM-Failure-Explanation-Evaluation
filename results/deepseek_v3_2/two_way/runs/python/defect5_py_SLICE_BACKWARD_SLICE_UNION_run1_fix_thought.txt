The bug occurs when both `array` and `element` are `None`, causing `inferred_type` to be set to `object`. If `expected_type` is provided and is not `object`, a `TypeError` is raised incorrectly. The fix is to skip the type check in this specific case, as adding `None` to an empty list should be allowed regardless of the expected type. We add a condition to bypass the type check when both `array` and `element` are `None`.
