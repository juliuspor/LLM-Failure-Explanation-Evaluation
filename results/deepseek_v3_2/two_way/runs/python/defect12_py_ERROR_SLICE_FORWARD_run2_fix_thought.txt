The bug is that substring_java raises IndexError when start or end > len(s). In abbreviate, after adjusting upper, lower may still be > len(text) (e.g., if text is short). Also, after adjusting lower, upper might become < lower. We need to clamp lower to len(text) and ensure upper >= lower. Additionally, the call to substring_java with start=0 and end=upper is safe because upper is now <= len(text). The index search uses lower, which is now clamped, so it won't cause out-of-range errors. The fix adds bounds checks for lower and re-adjusts upper if needed.
