The bug was in the calculation of total minutes for negative hour offsets. The original code incorrectly subtracted minutes_offset from hours_in_minutes when hours_offset was negative, leading to an offset that was more negative than intended. The fix is to always add hours_in_minutes and minutes_offset, regardless of sign, because minutes_offset is always a positive component (0-59). This yields the correct total minutes offset. For example, hours_offset=-1, minutes_offset=30 should give total_minutes = -60 + 30 = -30 minutes, which corresponds to UTC-00:30. The corrected code uses a single variable total_minutes to store the sum and then multiplies by MILLIS_PER_MINUTE.
