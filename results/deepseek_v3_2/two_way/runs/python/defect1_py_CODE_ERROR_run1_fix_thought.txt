The bug was that the code incorrectly handled negative hour offsets by subtracting the minute offset instead of adding it. The variable `minutes_offset` was being reused to store the total offset in minutes, causing confusion. The fix is to compute the total minutes as `hours_in_minutes + minutes_offset` (which works for both positive and negative hours) and store it in a new variable `total_minutes`. This ensures the correct total offset in minutes is calculated before converting to milliseconds.
