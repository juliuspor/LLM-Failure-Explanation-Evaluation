The bug is in the loop after consuming >0 codepoints. The original code uses a for loop that calls code_point_at at the same pos each iteration, which can cause an index out of range because pos doesn't advance within the loop. Instead, we need to advance pos by the code units for each consumed codepoint. The fix is to replace the for loop with a while loop that advances pos based on char_count of each codepoint, or simply accumulate the total code units consumed. Since consumed is the number of codepoints, we can loop consumed times, each time getting the codepoint at current pos and adding its char count to pos. This ensures pos moves correctly and doesn't exceed length.
