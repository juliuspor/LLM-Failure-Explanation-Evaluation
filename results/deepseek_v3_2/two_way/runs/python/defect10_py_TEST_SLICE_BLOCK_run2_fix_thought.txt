The issue is that `char_at` raises an IndexError for out-of-bounds indices, but the test expects a ValueError for invalid inputs like 'l'. In `create_number`, when processing a string ending with 'L', the code calls `char_at(numeric, 0)` where `numeric` might be empty (e.g., for input 'l', numeric is ''). This triggers an IndexError. The fix is to replace the `char_at` call with a safe check for the first character, and ensure that all error conditions raise ValueError as expected.
