The bug is in the `substring_java` function, which incorrectly raises an IndexError for backward slices (start > end). In Python, slicing with start > end returns an empty string, which is safe. The condition 'start > end' should be removed. Also, the error message should reference the actual out-of-range index (either start or end). The fix is to adjust the bounds check to only raise when start or end exceed the string length, and remove the start > end check.
