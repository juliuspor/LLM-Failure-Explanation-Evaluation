The test fails because the `code_point_at` function incorrectly handles surrogate pairs when the index points to the high surrogate. However, the caller `translate_string` (not shown in the slice) likely increments the index by 1 after calling `code_point_at`, but for a surrogate pair, the index should advance by 2 because the pair represents a single character.
