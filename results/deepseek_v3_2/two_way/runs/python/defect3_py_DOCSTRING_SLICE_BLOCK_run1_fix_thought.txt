The bug is in the while loop of `translate`. When `consumed` is non-zero, the loop increments `pos` by the sum of `char_count` for each consumed codepoint, but `code_point_at` is called with the same `pos` each iteration, leading to an infinite loop and eventually an out-of-bounds index. Instead, we should advance `pos` by the total number of code units consumed, not by recalculating char counts. We can compute the total code units consumed by iterating over the consumed codepoints and summing their char counts, or more simply, we can directly advance `pos` by the number of code units that correspond to the consumed codepoints. Since `translate_codepoint` returns the number of codepoints consumed, we need to convert that to code units. We'll add a helper to compute code units from codepoints consumed.
