The bug is that line 352 incorrectly sets `self._min_start_index = index` inside the outer `if` block, causing the minimum start index to be overwritten on every insertion. The fix is to move that assignment to an `else` clause, so it only runs when `self._min_start_index < 0` (i.e., first insertion). The same pattern applies to all other min/max indices (start, middle, end).
