The test fails because the GrayPaintScale.get_paint method does not clamp input values to the expected range before computing the grayscale color. The error message 'Color parameter outside of expected range: Red Green Blue' indicates that a color component (red, green, or blue) was computed as a value outside the valid 0-255 range. This occurs when the input value is outside the lower and upper bounds set for the scale. For example, in test_lower_bound_clamping, the input -0.5 with default bounds (0.0, 1.0) should be clamped to 0.0, producing (0,0,0). Instead, the method likely computes a negative color component (e.g., -0.5 * 255 = -127.5), which triggers the ValueError. The same issue happens in test_upper_bound_clamping with input 1.5, which should be clamped to 1.0 but may produce a value >255. The fix is to clamp the input value to the [lower, upper] range before interpolation.
