The bug arises from using `x == int(x)` to check if a float is an integer, which fails for large floats due to precision loss. Additionally, the loop to factor trailing zeros uses a check that may also suffer from floating-point inaccuracies. The fix replaces the integer check with `x.is_integer()` which correctly identifies integer-valued floats. For integer values, we convert to int (with overflow handling) and then factor trailing zeros using integer arithmetic to avoid precision issues. For non-integer floats, we fall back to the string representation. We also handle inf and nan separately to avoid errors in `is_integer()`.
