The bug was in the calculation of total minutes offset when hours_offset is negative. The original code incorrectly subtracted minutes_offset from hours_in_minutes when hours_in_minutes was negative, which flipped the sign of the minutes component. The fix is to always add minutes_offset to hours_in_minutes, regardless of sign, because minutes_offset is always non-negative (0-59). This yields the correct total minutes offset. For example, hours_offset=-1, minutes_offset=30 should give total minutes = -60 + 30 = -30, representing -0.5 hours, not -90 minutes.
