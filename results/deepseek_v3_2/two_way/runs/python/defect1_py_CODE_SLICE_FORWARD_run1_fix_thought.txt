The bug is in the calculation of total minutes when hours_offset is negative. The original code incorrectly subtracts minutes when hours_offset is negative, but minutes should always be added regardless of sign. The correct formula is total_minutes = hours_offset * 60 + minutes_offset, where minutes_offset is always positive (0-59). The sign is already accounted for in hours_offset. So we just need to compute total_minutes = hours_offset * 60 + minutes_offset, then multiply by 60,000 to get milliseconds. Also need to handle the sign correctly for the total offset.
