The bug is that `int(x)` can raise TypeError if `x` is not a numeric type (e.g., string, None). Also, `x == int(x)` can raise OverflowError for huge floats. We should first check if `x` is a numeric type (int or float) and handle conversion safely. Use `isinstance(x, (int, float))` and then check if it's finite and integral. For non-numeric inputs, fall back to `str(x)`.
