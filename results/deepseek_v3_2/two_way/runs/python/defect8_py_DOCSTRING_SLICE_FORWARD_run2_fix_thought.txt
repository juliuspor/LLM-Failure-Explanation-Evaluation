The original function used fixed indices and length checks, which failed for strings like 'en' because it tried to access indices beyond the string length. The fix splits the locale string by underscores and validates each part separately. This handles all valid cases: language only (e.g., 'en'), language and country (e.g., 'en_GB'), and language, country, and variant (e.g., 'en_GB_xxx'). It also validates that language is lowercase and country is uppercase, matching the original intent.
