The bug is in the loop that advances `pos` when `consumed` > 0. The original code calls `code_point_at(code_units, pos)` inside the for-loop, but `pos` is not updated until after the call, causing repeated reads from the same position. This leads to incorrect advancement and potential infinite loops or IndexError. The fix is to compute the codepoint once per iteration and update `pos` accordingly, ensuring proper traversal through the code units.
