The bug is in the `translate` method's loop when `consumed != 0`. The `for` loop incorrectly increments `pos` by calling `char_count(code_point_at(code_units, pos))` multiple times with the same `pos` value, leading to incorrect advancement. Instead, we should advance `pos` by the total number of code units consumed, which is the sum of `char_count` for each codepoint consumed. Since `consumed` is the number of codepoints consumed, we can iterate through each codepoint, get its codepoint value, and add its `char_count` to `pos`.
