The bug was in the calculation of total minutes. The original code incorrectly adjusted minutes_offset, potentially making it negative. The fix is to compute total_minutes directly as hours_offset * 60 + minutes_offset, which correctly handles both positive and negative hours_offset. This ensures total_minutes can be negative (if hours_offset is negative) but that's fine for the multiplication. The validation on minutes_offset ensures it's 0-59, and hours_offset ensures -23 to +23, so total_minutes will be within -23*60-59 to +23*60+59, which is safe for multiplication with MILLIS_PER_MINUTE.
