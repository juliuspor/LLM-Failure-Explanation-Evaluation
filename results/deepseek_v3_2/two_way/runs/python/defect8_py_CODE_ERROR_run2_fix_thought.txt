The original code assumed a strict format with exactly one underscore between language and country, and another underscore before variant. It failed for double underscores (e.g., 'fr__POSIX') because it expected uppercase letters at positions 3 and 4. The fix is to parse the locale string more flexibly: split by underscore into up to three parts, then determine if the second part is a country code (two uppercase letters) or a variant (or empty). This handles double underscores and other valid locale strings.
