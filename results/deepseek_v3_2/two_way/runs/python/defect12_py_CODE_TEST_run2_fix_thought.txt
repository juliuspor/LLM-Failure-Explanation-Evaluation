The bug occurs when lower > len(text). After adjusting upper to len(text) and then setting upper = lower, upper becomes larger than len(text). The fix is to clamp upper to len(text) after the adjustment. This ensures substring_java is called with a valid end index. Also, when lower is beyond length, the space search will return -1, and we will return the full string (since upper will be len(text) after clamping) without appending the suffix because upper == len(text).
