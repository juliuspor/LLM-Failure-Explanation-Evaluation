The bug occurs when `char_at` is called with an index that is out of bounds. In `create_number`, the call `char_at(numeric, 0)` is used to check if the first character is '-', but `numeric` could be an empty string (e.g., when `val` is 'L' or 'l' alone). This leads to an IndexError. The fix is to ensure `numeric` is not empty before calling `char_at`. We can check the length of `numeric` directly.
