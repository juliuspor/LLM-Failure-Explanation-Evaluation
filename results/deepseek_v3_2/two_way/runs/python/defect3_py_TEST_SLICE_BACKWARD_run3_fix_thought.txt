The bug is in the loop that advances `pos` after consuming codepoints. The `consumed` variable is the number of codepoints consumed, but the loop incorrectly uses `char_count` on each codepoint to advance `pos`. This can cause `pos` to overshoot the length of `code_units` when surrogate pairs are involved. Instead, we should directly advance `pos` by the number of code units corresponding to the consumed codepoints. We'll compute the total code units consumed by iterating through the consumed codepoints and summing their `char_count`. This ensures `pos` stays within bounds.
