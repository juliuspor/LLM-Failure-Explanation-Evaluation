The bug was in the index advancement logic in the `translate` method. When `translate_codepoint` returns a non-zero `consumed` (number of codepoints consumed), the loop incorrectly advanced `pos` by calling `char_count(code_point_at(...))` for each codepoint, but `pos` was not updated between iterations, leading to repeated processing of the same codepoint. The fix updates `pos` correctly within the loop, ensuring each codepoint's code units are skipped. Also, when `consumed == 0`, the method now uses `char_count(codepoint)` to advance by the correct number of code units (1 for BMP, 2 for surrogate pairs). This ensures proper handling of surrogate pairs and prevents IndexError or skipped characters.
