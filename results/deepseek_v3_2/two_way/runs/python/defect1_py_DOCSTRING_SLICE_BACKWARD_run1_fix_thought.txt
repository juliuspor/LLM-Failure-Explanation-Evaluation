The bug is in the validation of minutes_offset. The condition `minutes_offset > 59` incorrectly rejects 59, which is a valid minute value (0-59 inclusive). Changed it to `minutes_offset >= 60` to properly reject invalid values. Also note that the condition `minutes_offset < 0` already handles negative values, so the combined condition is `minutes_offset < 0 or minutes_offset >= 60`. This ensures minutes_offset must be between 0 and 59 inclusive.
