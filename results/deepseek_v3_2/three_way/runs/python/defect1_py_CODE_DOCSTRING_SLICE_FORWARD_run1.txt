The code fails because it incorrectly calculates the total offset minutes when the hours offset is negative. On line 383, when `hours_in_minutes` is negative, the code subtracts `minutes_offset` from it, which makes the total offset more negative than intended. For example, if `hours_offset = -1` and `minutes_offset = 30`, then `hours_in_minutes = -60`. The code computes `minutes_offset = -60 - 30 = -90`, but the correct total offset should be `-60 + 30 = -30`. The sign of the minutes component should be preserved, not inverted. This error occurs only for negative hour offsets and leads to an incorrect millisecond offset being passed to `cls.for_offset_millis`.
