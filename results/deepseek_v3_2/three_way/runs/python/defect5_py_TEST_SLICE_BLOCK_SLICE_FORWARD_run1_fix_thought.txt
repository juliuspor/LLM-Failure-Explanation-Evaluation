The bug occurs when both array and element are None, and expected_type is provided. The condition `inferred_type == object and expected_type != object` triggers a TypeError, but None should be allowed for any type. The fix is to only raise the TypeError when element is not None. If element is None, we should allow the addition regardless of expected_type, because None is a valid value in Python for any type. This matches the test expectation and Python's dynamic typing.
