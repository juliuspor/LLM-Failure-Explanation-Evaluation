The bug is in the calculation of total minutes when hours_offset is negative. The minutes_offset parameter is always a non-negative value (0-59) representing the absolute minutes component. The sign of the minutes component should follow the sign of the hours. So for negative hours, we subtract minutes_offset from hours_in_minutes (which is negative), yielding a more negative total. For positive hours, we add minutes_offset. The original code incorrectly handled negative minutes_offset (which shouldn't occur due to validation). The fix ensures minutes_offset is treated as an absolute magnitude, and the sign is determined by hours_offset.
