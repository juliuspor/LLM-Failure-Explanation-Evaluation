The bug is in the `substring_java` function, which incorrectly raises an IndexError when `end > len(s)`. Python slicing handles this safely, so we should remove that check. The fix is to allow `end` to be greater than the string length, as the slice will just go to the end. Also adjust the condition to not raise when `end > len(s)` but `start <= end` and `start <= len(s)`.
