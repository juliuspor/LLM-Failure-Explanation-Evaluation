The bug is that the calculation of the grayscale component 'g' uses the original 'value' instead of the clamped 'v'. This can cause 'g' to be out of the 0-255 range even after clamping. The fix is to replace 'value' with 'v' in the formula. Also, the error message could be improved, but the diagnosis only requires fixing the variable usage. I'll update the code accordingly.
