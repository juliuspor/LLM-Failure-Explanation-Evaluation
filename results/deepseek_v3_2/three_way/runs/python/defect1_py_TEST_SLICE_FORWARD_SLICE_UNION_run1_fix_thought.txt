The bug was in the calculation of total minutes when hours_offset is negative. The original code incorrectly handled the sign of minutes_offset by conditionally subtracting or adding. The fix simplifies the calculation to directly compute total_minutes as hours_offset * 60 + minutes_offset, which correctly handles all sign combinations (positive, negative, mixed). This ensures that both (-2, 15) and (-2, -15) produce the same offset of -135 minutes (-02:15).
