The bug occurs when both `array` and `element` are None, causing `inferred_type` to be set to `object`. This triggers an unnecessary TypeError when `expected_type` is provided and not `object`. The fix is to set `inferred_type` to `None` in that case, and then skip the type check when `inferred_type` is `None`. This ensures that appending None to an empty list (or treating None as an empty list) is allowed without raising an error, while preserving the type check for other cases.
