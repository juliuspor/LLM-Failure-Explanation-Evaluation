The bug occurs when both array and element are None, and expected_type is provided (e.g., str). The original code sets inferred_type to object and then raises a TypeError because inferred_type (object) != expected_type (str). However, when array is None, there is no existing array to cast, so the type check should be skipped. The fix is to only raise the TypeError when array is not None (i.e., there is an actual array that might have a type mismatch). If array is None, we treat it as an empty list and can append element (even if None) without type constraints. Therefore, we add a condition to only raise if array is not None.
