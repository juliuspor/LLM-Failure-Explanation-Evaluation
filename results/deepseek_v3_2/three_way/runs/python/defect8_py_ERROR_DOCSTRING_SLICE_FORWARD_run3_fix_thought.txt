The original function had a rigid structure expecting exactly one underscore between language and country, and another between country and variant. It failed for double underscores (empty country) and didn't handle numeric UN M.49 country codes. The fix uses split('_') to handle multiple underscores and empty parts. It validates language as two letters, country as two letters or three digits, and variant as the rest. This matches Java's Locale parsing behavior for strings like "fr__POSIX" (language=fr, country empty, variant=POSIX).
