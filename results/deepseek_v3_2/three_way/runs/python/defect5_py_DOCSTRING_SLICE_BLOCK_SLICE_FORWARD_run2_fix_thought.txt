The bug is that `inferred_type` is set to `type(array)` which returns the class of the list (e.g., `list`), not the type of its elements. This causes the condition `inferred_type == object` to always be false (since `list` != `object`), and the TypeError is never raised when it should be. The fix is to infer the element type, not the container type. We should infer the type of the first element if the array is non-empty, otherwise fall back to the type of the element being added or object. This matches the intended Java-style type checking.
