The bug is that `add_number` incorrectly outputs "0" for `-0.0` because it converts to integer, losing the sign. We need to detect negative zero before entering the integer branch and output "-0.0" (or "-0") to preserve the sign. The method `is_negative_zero` is already available, so we can use it. If `x` is negative zero, we should output a string that keeps the negative sign, e.g., "-0.0". Since the integer branch would output "0", we can treat negative zero as a special case and call `self.add("-0.0")` (or `self.add(str(x))` which yields "-0.0"). However, note that `str(-0.0)` returns "-0.0" in Python. So we can simply handle negative zero before the integer branch and output `str(x)`.
