The bug is that minutes_offset validation incorrectly restricts to 0-59, but negative minutes should be allowed when hours are negative. The fix is to change the validation range to -59 to 59. Also, the calculation of total minutes needs to handle negative minutes correctly. The existing calculation already handles negative hours by checking hours_in_minutes < 0 and subtracting minutes_offset (which could be negative). However, note that minutes_offset can be negative independently of hours_offset. The current logic: if hours_in_minutes < 0: minutes_offset = hours_in_minutes - minutes_offset else: minutes_offset = hours_in_minutes + minutes_offset. This works because if minutes_offset is negative, subtracting a negative adds it. For example, hours_offset = -2, minutes_offset = -15: hours_in_minutes = -120, hours_in_minutes < 0, so minutes_offset = -120 - (-15) = -105, which is correct total minutes. For hours_offset = 2, minutes_offset = -15: hours_in_minutes = 120, so minutes_offset = 120 + (-15) = 105. So the calculation is correct. Only validation needs to change.
