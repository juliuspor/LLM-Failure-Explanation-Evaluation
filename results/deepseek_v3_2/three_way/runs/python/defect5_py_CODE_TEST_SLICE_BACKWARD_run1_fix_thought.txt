The bug is in the type inference logic. When array is not None, we should not set inferred_type to type(array) (which is list). Instead, we should treat it as object for the purpose of the type check, because we cannot know the element type of a list in Python. Also, when array is None and element is None, inferred_type should be object, which is already the case. The fix changes the logic to set inferred_type = object when array is not None, and keep the rest the same. This ensures the type check works as intended: if expected_type is provided and not object, and inferred_type is object, raise TypeError. This matches the Java behavior where you cannot cast an Object[] to a more specific type array.
