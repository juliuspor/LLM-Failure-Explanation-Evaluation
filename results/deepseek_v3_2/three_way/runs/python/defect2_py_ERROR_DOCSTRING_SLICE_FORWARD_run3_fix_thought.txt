The bug is that the original code uses the original `value` (not the clamped `v`) to compute `g`. This can cause `g` to be out of range when `value` is outside the bounds, even though the docstring says values outside bounds are clamped before conversion. The fix is to use the clamped value `v` in the calculation. Additionally, we add a final clamp of `g` to 0..255 to guard against floating-point rounding errors.
