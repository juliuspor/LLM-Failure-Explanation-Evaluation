The bug was in the calculation of total minutes when hours are negative. The original code incorrectly used `minutes_offset = hours_in_minutes - minutes_offset` for negative hours, which could produce a more negative value than intended. The fix ensures that the total minutes are computed correctly: total_minutes = hours_offset * 60 ± minutes_offset, where minutes_offset is always non-negative (0-59). For negative hours, we subtract minutes_offset (since hours_offset * 60 is negative, subtracting positive minutes makes it more negative). For positive hours, we add minutes_offset. This matches the expected behavior: (-2, 15) yields -2 hours and 15 minutes, i.e., -135 minutes total, which corresponds to offset -02:15.
