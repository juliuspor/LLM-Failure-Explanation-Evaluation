The bug occurs because when both `array` and `element` are None, `inferred_type` is set to `object`. This triggers a TypeError when `expected_type` is not `object`. The fix is to skip the type-checking condition when `inferred_type` is `object` and `array` is None (i.e., we are creating a new list from scratch). In that case, we should not raise a TypeError because we are not casting an existing object list. We can simply return the new list with the element appended.
