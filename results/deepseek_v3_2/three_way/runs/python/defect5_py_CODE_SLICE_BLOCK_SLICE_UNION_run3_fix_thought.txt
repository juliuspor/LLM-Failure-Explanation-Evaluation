The bug is that `inferred_type` was set to `type(array)` (which is `list`) instead of the element type. To fix, we need to infer the element type from the first element of the array (if non-empty) or from the element being added. If array is empty and element is None, default to `object`. This matches the intended Java-like type checking. The condition `inferred_type == object` will now correctly compare the element type with `object`.
