The bug is in the loop that advances `pos` when `consumed > 0`. The original code called `code_point_at(code_units, pos)` inside the loop but `pos` wasn't updated between iterations, causing the same codepoint to be read each time. The fix is to compute the codepoint at the current `pos` each iteration and increment `pos` by the appropriate number of code units (1 or 2). This ensures `pos` moves forward correctly for each consumed codepoint.
