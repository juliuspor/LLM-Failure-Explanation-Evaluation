The bug is in the `while` loop's advancement when `consumed > 0`. The `for` loop incorrectly assumes each iteration corresponds to one code unit, but `consumed` is the number of codepoints handled. We need to advance `pos` by the total UTF-16 code units for those codepoints. We can compute this by iterating through the codepoints and summing their `char_count`. This ensures correct advancement for surrogate pairs.
