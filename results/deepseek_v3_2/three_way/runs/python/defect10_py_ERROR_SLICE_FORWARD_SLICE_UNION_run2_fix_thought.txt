The bug occurs when `char_at` is called with an empty string and index 0, which is out of range. In `create_number`, `char_at(numeric, 0)` is called when processing a trailing 'L' or 'l'. If `numeric` is empty (e.g., input "L"), this triggers the error. The fix is to check if `numeric` is empty before calling `char_at`. If empty, it's invalid, so raise ValueError. Also, ensure `numeric` is not empty when checking the first character.
