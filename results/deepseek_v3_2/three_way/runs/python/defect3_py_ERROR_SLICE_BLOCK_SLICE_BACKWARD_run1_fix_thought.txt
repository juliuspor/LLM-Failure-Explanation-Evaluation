The bug was in the loop after a translator consumed codepoints. The original code incorrectly used `char_count(code_point_at(code_units, pos))` for each consumed codepoint, which could cause an IndexError when `pos` advanced beyond valid indices. The fix replaces the for-loop with a simple increment of `pos` by `consumed`, since `translate_codepoint` already returns the number of codepoints consumed, not code units. This ensures `pos` stays within bounds.
