The bug occurs when both `array` is None and `element` is None, causing `inferred_type` to be set to `object`. If the caller provides an `expected_type` that is not `object`, the condition triggers a TypeError. This is undesirable because appending None to an empty list should not raise a cast error. The fix is to only raise the TypeError when `inferred_type` is `object` and `expected_type` is not `object` AND the `array` is not None (i.e., we are actually trying to cast an existing object list). When `array` is None, we are creating a new list, so the cast simulation should be skipped.
