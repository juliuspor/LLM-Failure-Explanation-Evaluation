However, the function does not check if the next index (`index + 1`) is within the bounds of the `code_units` list before accessing it. In the failing test, the input string is a surrogate pair (emoji) followed by 'A'. The `translate` method likely processes the string by iterating and calling `code_point_at`. At that index, the code unit is a low surrogate (between 0xDC00 and 0xDFFF). The function then tries to read `code_units[index + 1]` to check for a following high surrogate, but `index + 1` is now out of bounds because the list ends after the 'A' character.
