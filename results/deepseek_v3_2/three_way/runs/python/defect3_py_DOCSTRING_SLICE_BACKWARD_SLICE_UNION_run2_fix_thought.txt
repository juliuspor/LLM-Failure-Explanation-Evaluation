The bug is in the loop that advances `pos` after a translator consumes codepoints. The original code used `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. However, inside the loop, `code_point_at` is called with the current `pos`, but `pos` is updated each iteration, leading to incorrect indexing when `consumed > 1`. The fix is to compute the advance correctly: either by summing the char counts of each consumed codepoint, or by using a temporary variable. Also, note that when `consumed == 0`, we already advance by `char_count(codepoint)`. The fixed code uses a while loop to process each consumed codepoint, updating `pos` accordingly.
