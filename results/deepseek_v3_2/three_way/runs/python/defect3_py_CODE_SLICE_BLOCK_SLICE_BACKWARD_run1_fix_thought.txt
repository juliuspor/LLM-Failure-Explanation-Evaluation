The bug is in the for loop that advances `pos` after a translator consumes codepoints. The loop uses `code_point_at(code_units, pos)` but `pos` doesn't change between iterations, causing infinite loop. Instead, we should advance `pos` by the total number of code units consumed, not by counting codepoints. Since `translate_codepoint` returns the number of codepoints consumed, we need to convert that to code units by iterating through each consumed codepoint and adding its charCount. We'll do that in a separate loop.
