The bug is in the while loop's handling of consumed codepoints. After translate_codepoint returns consumed > 0, the loop incorrectly uses code_point_at with the same pos multiple times, causing index errors. The fix is to update pos inside the for loop by the charCount of each consumed codepoint, moving pos forward correctly. Also, in the consumed == 0 branch, we should advance by charCount(codepoint) instead of len(c) (which is the same, but charCount is more explicit).
