The bug was in the calculation of total minutes offset. The original code incorrectly subtracted the minutes component when hours_offset was negative, leading to wrong offsets (e.g., -5 hours and 30 minutes gave -330 minutes instead of -270). The fix simplifies the calculation by directly computing total_minutes = hours_offset * 60 + minutes_offset, which correctly handles both positive and negative hour offsets. This ensures the offset in milliseconds is accurate.
