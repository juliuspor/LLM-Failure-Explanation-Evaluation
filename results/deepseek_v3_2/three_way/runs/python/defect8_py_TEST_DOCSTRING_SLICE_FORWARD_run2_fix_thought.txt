The original function incorrectly assumed that after the first underscore there would be a two-letter uppercase country code. However, the locale format can have a double underscore (e.g., "fr__POSIX") where the country is empty and the variant follows. The fix modifies the parsing logic to detect a double underscore pattern and handle it appropriately. We also need to adjust length checks to allow for variable lengths. The updated function now: 1) validates the first two characters as lowercase letters, 2) checks for a single underscore, 3) if the next character is another underscore, treat country as empty and extract variant (if any), 4) otherwise, expect a two-letter uppercase country code and proceed as before. This ensures compatibility with both standard (language_country_variant) and empty country (language__variant) formats.
