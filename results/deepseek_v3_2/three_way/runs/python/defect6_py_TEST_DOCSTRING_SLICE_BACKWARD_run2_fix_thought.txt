The bug is that negative zero (-0.0) is incorrectly formatted as "0" because the condition `x == int(x)` holds true (since -0 == 0) and the sign is lost. The fix is to add a special case for negative zero using the existing `is_negative_zero` static method, which correctly identifies negative zero via `math.copysign`. If negative zero is detected, output "-0.0" and return early, bypassing the integer formatting logic. This ensures the sign is preserved.
