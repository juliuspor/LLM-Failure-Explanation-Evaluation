The bug is that negative zero (-0.0) is incorrectly output as "0" because the integer branch converts it to integer 0, losing the sign. The fix is to check for negative zero using the existing `is_negative_zero` method before entering the integer branch, and output "-0.0" directly. This ensures the sign is preserved. Also, the condition `x < 0` for adding a space before negative numbers already handles negative zero (since -0.0 < 0 is false), but we still need to output the negative sign. So we add a check for negative zero and output "-0.0" and return early.
