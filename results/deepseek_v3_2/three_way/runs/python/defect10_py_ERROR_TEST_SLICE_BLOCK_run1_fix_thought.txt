The bug occurs when processing strings like 'l' or 'L' where numeric = val[:-1] becomes an empty string. In the condition for 'L' handling, char_at(numeric, 0) is called without checking if numeric is empty, causing IndexError. The fix adds a length check (len(numeric) > 0) before accessing char_at(numeric, 0). This ensures that empty numeric strings are handled correctly, raising ValueError as expected.
