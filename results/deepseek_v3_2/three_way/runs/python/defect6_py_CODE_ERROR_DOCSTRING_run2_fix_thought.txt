The bug is that negative zero (-0.0) is incorrectly formatted as "0" because the condition x == int(x) holds (since int(-0.0) == 0) and the integer branch loses the sign. The fix is to add a special case for negative zero using the existing is_negative_zero method. If x is negative zero, we output "-0.0" (or "-0") and return early, bypassing the integer branch. This preserves the sign. Also note that the condition x < 0 and prev == '-' already adds a space for negative numbers, but for negative zero, we still need to output the negative sign. The early return ensures that the sign is not lost.
