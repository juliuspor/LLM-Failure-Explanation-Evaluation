The bug was that the sign of the offset was lost when hours were negative and minutes positive. The original code computed hours_in_minutes = hours_offset * 60, then if hours_in_minutes < 0, set minutes_offset = hours_in_minutes - minutes_offset. This incorrectly used minutes_offset as both the parameter and the result, and the subtraction could flip the sign. The fix is to compute total_minutes = hours_offset * 60, then adjust by minutes_offset while preserving the sign: if total_minutes < 0, subtract minutes_offset; otherwise add minutes_offset. This ensures the total minutes retain the correct sign and magnitude.
