The bug occurs because -0.0 is treated as an integer (since int(-0.0) == 0 and -0.0 == 0 is true), causing the sign to be lost. The fix is to check for negative zero using the existing `is_negative_zero` method before entering the integer path, and if true, output '-0.0' directly. This preserves the sign while keeping the rest of the integer formatting logic unchanged.
