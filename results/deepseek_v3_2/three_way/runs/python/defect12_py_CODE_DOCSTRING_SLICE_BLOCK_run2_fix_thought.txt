The bug occurs when `lower` is greater than the length of the text. After setting `upper = max(upper, lower)`, `upper` can exceed the text length, causing `substring_java` to raise an IndexError. The fix is to clamp `upper` to the text length before calling `substring_java`. Also, we should ensure that `lower` is not greater than the text length when searching for a space. The Java original handles this by using `Math.min` for the space search. We'll adjust the space search to be within bounds and clamp `upper` to `len(text)`.
