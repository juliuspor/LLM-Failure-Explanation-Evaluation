If value is below self._lower_bound (e.g., -0.5 with default bounds 0.0 and 1.0), the numerator becomes negative, leading to a negative g. Since g is not clamped, it falls below 0, triggering the ValueError at line 78. Similarly, if value is above self._upper_bound, g can exceed 255, also raising the ValueError. The tests expect clamping to black (0,0,0) for values below the lower bound and white (255,255,255) for values above the upper bound, but the current implementation instead raises an exception.
