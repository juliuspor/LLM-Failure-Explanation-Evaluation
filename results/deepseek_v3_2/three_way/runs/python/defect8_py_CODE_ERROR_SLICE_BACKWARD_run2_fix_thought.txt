The original function had rigid length checks and assumed a specific underscore pattern. It failed for POSIX locales like 'fr__POSIX' which have two underscores separating language and variant with an empty country. The fix uses split('_') to handle arbitrary underscores, validates language code, and then processes remaining parts: if second part is empty (POSIX), treat next as variant; else if second part is a 2-letter uppercase country code or 3-digit numeric code, treat as country; else treat as variant. Extra parts are joined into variant. This matches Java's Locale parsing behavior.
