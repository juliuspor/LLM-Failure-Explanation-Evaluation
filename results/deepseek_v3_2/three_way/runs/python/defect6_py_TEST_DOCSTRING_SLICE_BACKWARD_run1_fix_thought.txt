The bug is that negative zero (-0.0) is incorrectly treated as an integer because -0.0 == int(-0.0) (0) is True. The fix is to check for negative zero using the existing is_negative_zero method before the integer check, and output "-0.0" directly. This preserves the sign. Also, we need to ensure that the space insertion for negative numbers still works correctly for negative zero (the condition x < 0 is false for -0.0 in Python, so we don't add a space, which is fine).
