The bug occurs when the input string ends with a non-digit qualifier (like 'l', 'f', 'd') and stripping that qualifier results in an empty string (e.g., input 'l'). The code then tries to access numeric[0] via char_at, causing an IndexError. The fix adds a check after numeric = val[:-1] to ensure numeric is not empty, raising ValueError if it is, which aligns with the expected behavior for invalid inputs.
