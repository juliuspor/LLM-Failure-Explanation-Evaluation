The bug was in the while loop of the translate method. After translate_codepoint returns a positive consumed (number of code units processed), the code incorrectly used a for loop to increment pos by char_count of each codepoint, which could cause pos to exceed the length of code_units. The fix is to simply increment pos by consumed directly, as translate_codepoint already handled the appropriate number of code units. Also removed the unnecessary for loop and char_count calls.
