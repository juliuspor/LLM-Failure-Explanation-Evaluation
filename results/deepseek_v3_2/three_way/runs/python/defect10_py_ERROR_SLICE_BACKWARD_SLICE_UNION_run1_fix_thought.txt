The bug occurs when processing a string ending with 'L' (or 'l') where the numeric part is empty (e.g., "L"). The original code calls char_at(numeric, 0) without checking if numeric is empty. Since numeric = val[:-1], for "L" it becomes an empty string, causing IndexError. The fix adds a length check before calling char_at to ensure the string is not empty, preventing the out-of-range access.
