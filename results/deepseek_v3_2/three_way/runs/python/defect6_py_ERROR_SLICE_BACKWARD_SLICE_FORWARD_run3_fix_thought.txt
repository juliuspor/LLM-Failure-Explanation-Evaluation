The bug is that -0.0 is treated as an integer because -0.0 == int(-0.0) (0) and is not infinite or NaN, so it goes into the integer branch and loses the negative sign. The fix is to treat zero values specially, preserving the sign. We can check if x is zero (positive or negative) using math.isclose(x, 0.0) or x == 0.0, and then output the original float representation to keep the sign. Alternatively, we can use repr(x) which for -0.0 gives '-0.0'. However, we must also handle the case where x is negative and prev is '-', which is already covered by the early space insertion. So we can adjust the integer branch to skip for negative zero, or handle zero separately before the integer branch.
