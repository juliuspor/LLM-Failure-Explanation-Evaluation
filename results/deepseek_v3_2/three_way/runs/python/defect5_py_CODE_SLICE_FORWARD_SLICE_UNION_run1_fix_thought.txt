The bug is that `type(array)` returns `list`, not the element type. We need to infer the element type from the first element of the array if it exists, otherwise default to `object`. Also, we should handle the case where `array` is None but `element` is not None to infer type from the element. The fix: if array is not None and has at least one element, set `inferred_type = type(array[0])`; else if element is not None, set `inferred_type = type(element)`; else default to `object`.
