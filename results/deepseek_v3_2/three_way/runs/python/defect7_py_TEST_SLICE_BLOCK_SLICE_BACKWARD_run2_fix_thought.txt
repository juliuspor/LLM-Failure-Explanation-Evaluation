The bug is in the `to_class` method where `array[i].upper()` is called on each element. This fails when an element is `None` because `None` has no `upper` method. The method should simply get the type of each element. The fix is to remove the erroneous line `_ = array[i].upper()` and keep only `classes.append(type(array[i]))`. This ensures that `None` elements are handled correctly (their type is `NoneType`).
