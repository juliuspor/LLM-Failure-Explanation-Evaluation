The bug is that negative zero (-0.0) is incorrectly output as "0" because the integer path converts it to int(0) and then to string. The fix is to check for negative zero using the existing is_negative_zero method before the integer path, and output "-0.0" directly. This preserves the sign. Also, the condition x < 0 and prev == '-' already adds a space for negative numbers, but for negative zero we still need to handle the sign. The fix adds a check right after the space-adding logic, and returns early to avoid the integer path.
