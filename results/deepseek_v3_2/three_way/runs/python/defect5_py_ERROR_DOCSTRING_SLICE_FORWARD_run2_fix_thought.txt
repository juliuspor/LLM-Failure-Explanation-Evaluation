The bug is that the type check only considered the inferred_type (which is the type of the list, e.g., list) and compared it to object. Instead, we need to check that each element in the existing array (if any) and the new element are instances of the expected_type. The fix is to iterate over the array and the element, verifying they are instances of expected_type. If any element is not an instance, raise a TypeError with the same error message.
