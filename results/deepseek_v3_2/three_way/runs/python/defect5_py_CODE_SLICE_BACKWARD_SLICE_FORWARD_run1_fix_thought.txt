The bug is that `inferred_type` incorrectly captures the container type (`list`) instead of the element type. We need to infer the element type from the first element of the array (if present) or from the element being added. Also, the comparison `inferred_type == object` should compare the element type, not the container type. The fix: compute `element_type` based on the first element of the array (if array is non-empty) or the element being added, defaulting to `object` if both are None. Then use `element_type` for the type safety check.
