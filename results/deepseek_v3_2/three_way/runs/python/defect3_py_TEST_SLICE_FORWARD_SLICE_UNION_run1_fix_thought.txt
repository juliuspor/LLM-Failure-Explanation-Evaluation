The bug is in the `translate` method's loop when handling consumed codepoints. The original code incorrectly advances `pos` by iterating over each consumed codepoint and adding `char_count(code_point_at(code_units, pos))` but `pos` is not updated within the loop, leading to repeated calls to `code_point_at` at the same position. Moreover, the loop for consumed codepoints should advance `pos` by the total number of code units corresponding to the consumed codepoints. The fix: when `consumed > 0`, compute the total code units consumed by iterating over each codepoint, using a temporary offset, and then advance `pos` by that total. Also, in the `consumed == 0` branch, we already correctly advance by `char_count(codepoint)`. This ensures proper handling of surrogate pairs and single code units.
