The bug is that after a translator consumes characters, the loop incorrectly increments `pos` by calling `char_count(code_point_at(...))` inside a for-loop, which uses the updated `pos` each iteration, leading to index out of bounds. The fix is to compute the total code units consumed by iterating over the consumed codepoints from the current position, summing their charCounts, and then incrementing `pos` by that total. This ensures `pos` moves correctly without recalculating based on a moving index.
