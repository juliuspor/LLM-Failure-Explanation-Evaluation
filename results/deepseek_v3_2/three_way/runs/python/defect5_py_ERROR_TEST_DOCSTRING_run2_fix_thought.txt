The bug is that the type check in `ArrayUtils.add` incorrectly raises a `TypeError` when `array` is `None` and `element` is `None`, even though there is no existing array to cast. The fix is to only perform the type check when `array` is not `None`. Additionally, the type check logic should infer the element type from the first element of the array (if available) rather than using the type of the list itself. If the array is empty, we default to `object`. If the inferred element type is `object` and the `expected_type` is not `object`, we raise the `TypeError` to simulate the Java array cast failure. This ensures that the test `test_add_null_to_null_with_expected_type` passes because when `array` is `None`, the type check is skipped.
