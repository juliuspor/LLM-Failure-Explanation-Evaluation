The bug is that `inferred_type` is set to `type(array)` which returns `list`, not the element type. Instead, we should infer the element type by checking the first element of the array (if any) or default to `object`. Also, we need to handle the case where `array` is empty but not None, where element type should be inferred from the element being added. The fix involves: 1) Determine element type from the first element of the array if array is non-empty, else from the element being added, else default to object. 2) Use that element type for the type-checking logic.
