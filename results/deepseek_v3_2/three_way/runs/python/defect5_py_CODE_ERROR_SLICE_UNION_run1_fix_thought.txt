The bug is that `inferred_type` was set to `type(array)` (which is `list`) instead of the element type. We need to infer the element type from the array's contents. The fix: when `array` is not None, iterate through its elements to find the first non-None element and use its type. If all elements are None, default to `object`. This matches the Java behavior where an array of nulls is considered an Object array. Also, we keep the same logic for when array is None: use element's type if not None, else object. The check `inferred_type == object` will now correctly compare the element type.
