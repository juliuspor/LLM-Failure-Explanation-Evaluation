The original code assumed a strict format: language (2 letters) + underscore + country (2 uppercase) + underscore + variant. However, the Java Locale allows double underscore to indicate an empty country with a variant (e.g., "fr__POSIX"). The fix is to relax the validation: after the language and underscore, we check if the next two characters are uppercase letters; if so, treat them as country; otherwise, treat everything after the first underscore as variant (with possible leading underscore). This matches Java's Locale.forLanguageTag and Locale.toString behavior.
