The bug is that the type check only considered the inferred type of the list (which is always 'list' in Python) and the element type, but didn't validate that all elements in the list are compatible with the expected_type. The fix is to change the type checking logic: instead of checking if inferred_type == object, we should check each element's type against expected_type when expected_type is provided. This ensures that if a list contains heterogeneous types, an error is raised before attempting a cast. Also, we need to check the new element's type. This matches the Java behavior where adding an element of wrong type to a typed array would cause an ArrayStoreException. The fix modifies the condition to validate each element individually.
