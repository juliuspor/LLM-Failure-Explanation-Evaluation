The bug is that int(-0.0) yields 0, losing the sign. We need to treat negative zero specially. The class already has a helper method is_negative_zero. We can use it to detect -0.0 and output "-0.0" directly, bypassing the integer conversion path. This preserves the sign. Also, we must ensure that the space insertion for negative numbers still works (it does because x < 0 is true for -0.0). The fix adds a check for negative zero before the integer conversion, and returns early with the string "-0.0".
