The bug is that `ArrayUtils.add` incorrectly handles the `expected_type` parameter when both `array` and `element` are None. The test expects that adding null to null with expected_type=str should return [None] without raising a TypeError. The current code infers the type as `object` when both are None, and then raises a TypeError because expected_type (str) != object. The fix is to treat None array and None element as a special case: if array is None and element is None, we should not raise a TypeError regardless of expected_type, because the resulting list is just [None] and type checking is irrelevant. Also, the inferred_type logic should be adjusted to avoid unnecessary type errors when expected_type is provided but not matching.
