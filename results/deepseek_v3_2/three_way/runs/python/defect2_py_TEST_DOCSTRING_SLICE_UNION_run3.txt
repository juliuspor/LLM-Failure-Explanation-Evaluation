The code fails because the `get_paint` method does not clamp values outside the bounds before computing the grayscale value. In line 74, the formula `g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)` is applied directly to the input `value`. For example, if `value` is -0.5 with default bounds (0.0, 1.0), the calculation becomes `int((-0.5 - 0.0) / (1.0 - 0.0) * 255.0) = int(-0.5 * 255.0) = -127`. This results in `g = -127`, which triggers the `ValueError` on line 78 because `g < 0`. The same issue occurs for values above the upper bound, where `g` can exceed 255. The docstring states that values outside bounds should be clamped, but the implementation lacks clamping logic before the interpolation step.
