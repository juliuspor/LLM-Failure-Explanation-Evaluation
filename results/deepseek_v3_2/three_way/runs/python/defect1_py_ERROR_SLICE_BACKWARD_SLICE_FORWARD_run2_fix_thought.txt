The bug is that the validation for minutes_offset incorrectly assumes it's always non-negative. However, minutes_offset can be negative when the overall offset is negative (e.g., -05:30). The fix is to allow minutes_offset in the range -59 to 59, and compute total_minutes correctly by adding hours_offset*60 and minutes_offset. Also removed the problematic logic that was modifying minutes_offset incorrectly.
