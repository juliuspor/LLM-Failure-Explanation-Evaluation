The bug is that negative zero (-0.0) is incorrectly output as '0' because the condition x == int(x) is true for -0.0 (since int(-0.0) is 0) and the sign is lost. The fix is to check for negative zero using the existing is_negative_zero method before the integer handling branch, and output '-0.0' directly. This preserves the sign while still handling other numbers as before.
