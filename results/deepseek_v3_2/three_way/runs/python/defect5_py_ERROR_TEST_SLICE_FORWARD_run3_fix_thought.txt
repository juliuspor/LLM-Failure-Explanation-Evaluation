The bug is that when `expected_type` is provided and both `array` and `element` are None, the function incorrectly infers the type as `object` and raises a TypeError. The test expects it to return [None] without error. The fix is to prioritize `expected_type` for type inference when `array` is None and `element` is None, and only raise the TypeError when there is a real type mismatch (i.e., when `array` is not None and its inferred type is `object` but `expected_type` is a specific type). Also, the `inferred_type` variable should be set to `expected_type` when `expected_type` is not None and `array` is None and `element` is None.
