The bug is that negative zero (-0.0) is output as "0" because Python's str(-0.0) returns "0". The fix is to add a special case using the existing is_negative_zero method to output "-0.0" explicitly. This check should be placed before the integer handling branch, because -0.0 equals int(-0.0) (0) and would incorrectly go into the integer branch. The condition x == int(x) holds for -0.0 (since int(-0.0) is 0), so we need to handle negative zero first.
