The bug was in the calculation of total minutes when hours_offset is negative. The original code incorrectly subtracted minutes_offset from hours_in_minutes when hours_offset was negative, leading to wrong total minutes. The fix is to simply add hours_in_minutes (which already includes the sign) to minutes_offset, as hours_in_minutes = hours_offset * 60. This yields the correct total minutes, e.g., for hours_offset=-1 and minutes_offset=30, total_minutes = -60 + 30 = -30, which corresponds to UTC-1:30. The positive case already worked because hours_in_minutes is positive. So we replace the conditional logic with a single addition.
