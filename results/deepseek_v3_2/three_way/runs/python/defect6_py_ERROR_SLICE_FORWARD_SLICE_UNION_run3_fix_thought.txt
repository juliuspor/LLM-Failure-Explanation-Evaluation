The bug is that negative zero (-0.0) loses its sign because the integer branch converts it to int(0) and outputs '0'. We need to treat zero specially to preserve the sign. Since -0.0 == 0 is True, we can check for zero using x == 0 and then output str(x) to keep the sign (str(-0.0) gives '-0.0'). Alternatively, we can check for negative zero using is_negative_zero method and output '-0' or '-0.0'. The simplest fix is to handle zero before the integer branch: if x == 0, output str(x) directly.
