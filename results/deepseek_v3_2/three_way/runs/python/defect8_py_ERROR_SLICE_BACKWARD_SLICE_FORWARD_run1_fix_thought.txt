The original code assumed a strict format with exactly one underscore between language and country, and another before variant. It failed on strings like "fr__POSIX" with double underscores. The fix handles flexible separators: after the initial language and underscore, it checks if the next two characters are uppercase letters (a country code). If not, it treats everything after the first underscore as variant. This accommodates locale strings with missing country codes (e.g., "fr_POSIX" or "fr__POSIX") while preserving validation for standard formats.
