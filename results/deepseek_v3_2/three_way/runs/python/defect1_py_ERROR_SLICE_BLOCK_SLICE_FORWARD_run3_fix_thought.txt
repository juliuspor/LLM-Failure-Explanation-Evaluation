The bug occurs because the variable `minutes_offset` is reassigned to a negative value when `hours_in_minutes` is negative and we subtract the original minutes offset. The validation after that reassignment then fails. The fix is to compute the total offset in minutes without mutating the original `minutes_offset` variable, and then validate that the total offset is within the allowed range (-23:59 to +23:59). We should also keep the original validation for the input parameters, but the total offset validation should be done after the calculation.
