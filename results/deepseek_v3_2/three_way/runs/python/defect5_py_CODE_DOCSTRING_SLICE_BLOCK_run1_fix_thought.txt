The bug is in the type inference logic of `ArrayUtils.add`. The method incorrectly uses `type(array)` which returns `list` (the container type) instead of the element type. The intended check is to simulate Java's array type safety, but Python's type system is different. The fix is to remove the flawed type inference and the unnecessary `expected_type` check, as it doesn't align with Python's dynamic typing. The method should simply append the element to a copy of the list (or create a new list if None).
