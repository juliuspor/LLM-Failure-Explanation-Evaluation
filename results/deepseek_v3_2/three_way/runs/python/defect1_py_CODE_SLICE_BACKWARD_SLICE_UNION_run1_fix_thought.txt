The bug is that the early return condition `if hours_offset == 0 and minutes_offset == 0:` incorrectly returns UTC when `hours_offset` is zero but `minutes_offset` is non‑zero. For example, `for_offset_hours_minutes(0, 30)` should create a +30‑minute offset, but instead returns UTC. The fix is to keep the early return only when both components are zero, and otherwise compute the total offset correctly. Additionally, to avoid confusion, the variable that holds the total minutes is renamed to `total_minutes` instead of reusing the parameter `minutes_offset`.
