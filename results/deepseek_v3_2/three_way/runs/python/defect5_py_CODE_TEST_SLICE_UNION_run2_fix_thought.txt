The bug occurs when both `array` and `element` are None, causing `inferred_type` to default to `object`. This triggers an unnecessary TypeError when `expected_type` is provided. The fix is to prioritize `expected_type` for type inference when both inputs are None, and also to avoid the cast error when `expected_type` is provided but the inferred type is object. We should only raise TypeError if there's a genuine type mismatch, not when we have no information. The simplest fix is to set `inferred_type` to `expected_type` if both array and element are None and expected_type is provided, and to skip the cast check when inferred_type is object (since it's a generic fallback).
