The code fails because the GrayPaintScale.get_paint method does not clamp values outside the lower and upper bounds. In line 74, the formula `g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)` computes a gray level `g` that can be less than 0 or greater than 255 when `value` is outside the bounds. Then, lines 77-78 check if `g` is out of range and raise a ValueError instead of clamping. The test expects clamping: for example, test_lower_bound_clamping passes `-0.5` with default bounds (0.0, 1.0), which would make `g` negative, triggering the exception. The fix is to clamp `g` to [0, 255] or clamp `value` to [lower_bound, upper_bound] before the calculation.
