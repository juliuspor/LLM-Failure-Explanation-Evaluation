The bug occurs when `lower` is beyond the string length. The original code sets `upper` to `len(text)`, then if `upper < lower`, sets `upper = lower`, causing `upper` to exceed the string length. This leads to unnecessary appending of `append_to_end` when `upper != len(text)`. The fix is to detect when `lower >= len(text)` and return the original string immediately, without any modification or appending. This matches the Java Commons Lang behavior and passes the failing test.
