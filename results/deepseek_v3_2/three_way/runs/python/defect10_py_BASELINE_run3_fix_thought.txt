The bug occurs when `last_char` is 'l' or 'L' and `numeric` (which is `val[:-1]`) is an empty string. The call to `char_at(numeric, 0)` will raise an IndexError because the string length is 0. The fix is to check if `numeric` is empty before calling `char_at`. If empty, the condition `char_at(numeric, 0) == "-"` should be false, and we should proceed to check `NumberUtils.is_digits(numeric)`, which will be false for an empty string, leading to the expected ValueError. We'll add a guard to avoid calling `char_at` on an empty string.
