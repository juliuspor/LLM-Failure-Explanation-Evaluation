The original implementation assumed fixed positions and lengths, which failed for locale strings with empty country codes (double underscore). The fix uses splitting by underscore to handle variable parts. After splitting, we validate each part appropriately: language must be two lowercase letters; country must be either empty (for double underscore) or two uppercase letters; variant can be any string. This correctly parses cases like "fr__POSIX" into Locale("fr", "", "POSIX").
