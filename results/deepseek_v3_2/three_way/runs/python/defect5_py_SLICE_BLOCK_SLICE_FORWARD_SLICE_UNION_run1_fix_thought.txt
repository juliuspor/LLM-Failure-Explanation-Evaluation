The bug is that `inferred_type` was incorrectly set to the type of the container (list) or the element itself, rather than the element type of the list. The fix is to infer the element type from the first element of the array (if non-empty), otherwise fall back to the element's type or object. This matches the intended logic: if the inferred element type is the generic `object` and the expected type is more specific, raise a TypeError to simulate Java's array cast failure.
