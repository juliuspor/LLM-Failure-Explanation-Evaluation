The bug is that `to_locale` incorrectly validates the country code when the locale string has a double underscore (empty country) before the variant. The function should detect that the characters at positions 3-4 are both underscores, treat the country as empty, and extract the variant from position 6 onward. The fix modifies the validation logic to allow underscores in the country positions when the variant is present, and adjust the slicing accordingly.
