The bug is that negative zero (-0.0) is incorrectly output as '0' because int(-0.0) == 0 and the condition x == int(x) is true. To fix, we need to treat zero as a special case before the integer branch. We check if x is zero (x == 0.0). If it is, we check the sign using math.copysign. If negative, output '-0.0'; else output '0'. This ensures the sign is preserved. For non-zero numbers, the original logic works.
