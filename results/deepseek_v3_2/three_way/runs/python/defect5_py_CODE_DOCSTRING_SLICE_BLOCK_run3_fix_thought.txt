The bug is in the type inference logic. The original code sets inferred_type to type(array) which returns list, not the element type. We need to infer the element type from the first element of the array (if array is non-empty). If array is empty, we cannot infer and treat as object. If array is None, we infer from element type, else object. Then the comparison inferred_type == object will be correct (object vs a specific type like int). Also, we should keep the same error raising logic as before but now with correct inferred_type.
