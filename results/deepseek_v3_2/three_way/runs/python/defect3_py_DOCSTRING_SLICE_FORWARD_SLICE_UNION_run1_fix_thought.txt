The bug is in the `translate` method's loop when `consumed == 0`. The original code incorrectly used `len(c)` to advance `pos`, but `c` is the list of UTF-16 code units for the codepoint, not the number of codepoints consumed. Since `consumed` is 0, we need to advance by the number of UTF-16 code units that represent the current codepoint, which is given by `char_count(codepoint)`. The fix replaces `pos += len(c)` with `pos += char_count(codepoint)` to correctly handle surrogate pairs (2 code units) and BMP characters (1 code unit).
