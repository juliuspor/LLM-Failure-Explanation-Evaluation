However, `code_point_at` reads from `code_units` at the current `pos`, which changes each iteration, leading to an IndexError when `pos` exceeds the length of `code_units`. Specifically, after the first increment, `pos` may be out of bounds for `code_point_at` (line 35-36). The issue is that `pos` should be incremented by the total consumed code units, not by iterating and recalculating.
