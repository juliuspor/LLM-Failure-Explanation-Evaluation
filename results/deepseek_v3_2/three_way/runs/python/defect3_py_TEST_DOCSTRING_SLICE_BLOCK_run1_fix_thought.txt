The bug is in the loop where `pos` is incremented after a translator consumes codepoints. The original code incorrectly used `char_count(code_point_at(code_units, pos))` for each consumed codepoint, which could misalign `pos` because `consumed` already represents the number of codepoints consumed, not code units. The fix changes the increment to `pos += consumed`, ensuring `pos` advances by the number of codepoints consumed, and updates the fallback case to use `char_count(codepoint)` for consistency. This aligns with the contract that `translate_codepoint` returns codepoints consumed.
