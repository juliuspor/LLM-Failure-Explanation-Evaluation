The bug is that `add_number` incorrectly outputs "0" for `-0.0` because the integer conversion discards the sign. The fix is to treat zero values specially: if `x` is zero (positive or negative), we should output the original float representation to preserve the sign, using `str(x)` which yields "-0.0" for negative zero. We can detect zero with `x == 0` and then use `str(x)`. This also handles the case where `x` is negative zero but not exactly equal to integer zero (due to floating-point precision).
