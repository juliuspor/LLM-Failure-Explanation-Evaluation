The bug was in the for loop that updates `pos` when `consumed > 0`. The original code called `code_point_at(code_units, pos)` inside the loop but `pos` wasn't updated until after the call, so it always read the same starting codepoint. This caused incorrect advancement and potential infinite loops. The fix is to recompute the codepoint at the current `pos` each iteration and increment `pos` by the correct number of code units for that codepoint.
