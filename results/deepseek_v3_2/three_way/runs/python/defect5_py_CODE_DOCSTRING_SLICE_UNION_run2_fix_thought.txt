The bug is that `inferred_type` is set to `type(array)`, which returns `list` (the container type) instead of the element type. To fix, we need to infer the element type when possible. If `array` is not None and not empty, we can get the type of its first element. Otherwise, fall back to `type(element)` or `object`. This ensures the type check works correctly when `expected_type` is provided.
