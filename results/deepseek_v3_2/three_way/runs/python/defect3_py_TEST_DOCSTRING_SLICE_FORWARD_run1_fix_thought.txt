The bug is in the `translate` method's loop when handling surrogate pairs. After `translate_codepoint` returns a non-zero `consumed` (e.g., 2 for a surrogate pair), the loop incorrectly advances `pos` by iterating over `consumed` and adding `char_count` for each codepoint. This double-counts because `char_count` returns 2 for a surrogate pair, leading to an over-advance of `pos`. Instead, `pos` should advance by the number of UTF-16 code units consumed, not by the sum of `char_count` of each codepoint. The fix is to directly add `consumed` to `pos`, since `translate_codepoint` already returns the number of code units consumed.
