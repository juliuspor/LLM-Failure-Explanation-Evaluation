The bug was in the offset calculation: when hours_offset is negative, the minutes_offset should be subtracted (not added) to maintain the correct sign. The original code incorrectly assigned minutes_offset = hours_in_minutes - minutes_offset, which could lead to wrong offset magnitude. The fix ensures total_minutes is computed correctly: total_minutes = hours_offset * 60, then adjust by minutes_offset with proper sign. If total_minutes is negative, subtract minutes_offset (since minutes_offset is positive magnitude). If total_minutes is positive, add minutes_offset. This yields the correct total minutes offset, which is then converted to milliseconds.
