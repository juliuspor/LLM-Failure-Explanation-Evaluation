The bug was in the calculation of total minutes. The original code incorrectly reassigned the `minutes_offset` variable, potentially making it negative and causing confusion. The fix simplifies the calculation by directly computing `total_minutes = hours_offset * 60 + minutes_offset`. This correctly handles both positive and negative hour offsets while keeping minutes_offset non‑negative as validated. The validation remains unchanged because minutes_offset must still be 0‑59.
