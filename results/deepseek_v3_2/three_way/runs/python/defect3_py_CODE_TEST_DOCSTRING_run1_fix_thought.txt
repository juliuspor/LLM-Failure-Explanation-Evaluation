The bug is in the for-loop that advances `pos` after a non-zero `consumed`. The loop incorrectly calls `code_point_at` with the updated `pos` each iteration, leading to out-of-bounds or skipped characters. Instead, we should advance `pos` by the total number of code units consumed, which can be computed by summing `char_count` for each consumed codepoint, but we must not read from `code_units` with a moving index. Simpler: treat `consumed` as the number of codepoints processed, and advance `pos` by the total code units for those codepoints. We can compute this by iterating through the codepoints from the original `code_units` list, using a temporary index to avoid side effects.
