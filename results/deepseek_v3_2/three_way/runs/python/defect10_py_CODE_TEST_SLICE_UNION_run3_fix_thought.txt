The bug is that `char_at` is called but not defined in the function's scope. The function `char_at` is defined earlier in the code (outside the class), but the `create_number` method doesn't have access to it because it's a static method and `char_at` is not in its namespace. The fix is to replace the call to `char_at(numeric, 0)` with direct indexing (`numeric[0]`), which is safe because the condition ensures `numeric` is non-empty (since `val` has at least one non-digit character). This matches the original intent of checking the first character for a minus sign.
