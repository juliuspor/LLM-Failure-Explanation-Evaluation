The bug occurs because when both `array` and `element` are None, `inferred_type` is set to `object`. Then, if `expected_type` is provided (e.g., `str`), the type-checking condition incorrectly raises a TypeError. The fix is to prioritize `expected_type` for type inference when it's provided and no other type can be inferred. Specifically, if `expected_type` is not None, use it as the inferred type when both inputs are None, preventing the cast error.
