The bug is in the loop that advances `pos` after a translator consumes characters. The original code calls `char_count(code_point_at(code_units, pos))` inside the loop, but `pos` is updated each iteration, causing misalignment for surrogate pairs. The fix is to compute the codepoint at the current `pos` before advancing, and then advance by the correct number of code units. This ensures that for each consumed codepoint, we advance by its char count (1 for BMP, 2 for surrogate pairs). The loop now correctly handles surrogate pairs when `consumed` is 1.
