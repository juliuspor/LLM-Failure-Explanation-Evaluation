The bug is that `inferred_type` incorrectly uses `type(array)` (which returns `list`) when `array` is not None. Instead, we need to infer the element type from the first element of the list, if available. If the list is empty, we cannot infer, so set to `object`. This matches the Java behavior where the component type of an empty array is `Object`. Also, when `array` is None and `element` is None, `inferred_type` should be `object`. The fix updates the type inference logic accordingly.
