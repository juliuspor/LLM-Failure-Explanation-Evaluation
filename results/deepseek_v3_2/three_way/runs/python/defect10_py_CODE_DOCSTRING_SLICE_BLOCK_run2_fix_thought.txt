The bug is that `char_at` is not defined in the code, causing a NameError. The function `char_at` was used to get the first character of a string with bounds checking, but it's unnecessary because Python string indexing is safe for valid indices. In the context, `numeric` is guaranteed to have at least one character (since `val` has a non-digit last character and `numeric = val[:-1]`). Therefore, we can directly access `numeric[0]` without a helper function. The fix replaces `char_at(numeric, 0)` with `numeric[0]`.
