The bug was in the loop after a translator consumed codepoints: it used `len(c)` to advance `pos`, but `c` is the list of code units for a single codepoint, not the number of codepoints consumed. This could cause `pos` to overshoot or undershoot, leading to index errors in `code_point_at`. The fix changes `pos += len(c)` to `pos += char_count(codepoint)`, ensuring `pos` advances by the correct number of codepoints (1 for BMP, 2 for surrogate pair). This aligns with the contract that translators return consumed codepoint count, not code unit count.
