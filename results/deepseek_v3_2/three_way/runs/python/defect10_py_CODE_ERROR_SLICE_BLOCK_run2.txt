In create_number, at line 48, there is a call to char_at(numeric, 0) when numeric might be an empty string. This happens when val ends with 'l' or 'L' and numeric is val[:-1] (the whole string except the last character). If val is just "L" (length 1), then numeric becomes an empty string. Then char_at(numeric, 0) is called, which tries to access index 0 of an empty string, causing the IndexError.
