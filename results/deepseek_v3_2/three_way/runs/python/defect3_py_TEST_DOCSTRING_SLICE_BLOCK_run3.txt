The code fails because the `code_point_at` function incorrectly handles surrogate pairs when the index points to a high surrogate. The function checks if the index is within bounds (lines 35-36) but then directly returns the code unit at that index as a code point. For a surrogate pair (like an emoji), the high surrogate and low surrogate together form a single code point. The function should check if the code unit at the index is a high surrogate (0xD800 to 0xDBFF) and if the next index is within bounds and contains a low surrogate (0xDC00 to 0xDFFF). If so, it should combine them into a single code point. Otherwise, it should return the code unit as is. The bug causes the index to advance by only one unit when processing a surrogate pair, leading to an IndexError in `translate` because the next iteration may exceed the string length or misalign characters.
