The bug is that the original to_locale method assumed a strict length-based format, which failed for locale strings with empty country (double underscore). The fix is to parse the locale string by splitting on underscores and handling the cases: language only, language_country, language__variant (empty country), and language_country_variant. We validate language as two lowercase letters, country as two uppercase letters (or empty), and variant as any string. This matches Java's Locale parsing behavior.
