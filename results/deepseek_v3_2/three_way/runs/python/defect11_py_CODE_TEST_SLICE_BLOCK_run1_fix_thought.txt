The bug is that when `letters=False`, `numbers=False`, `chars=None`, and `start` and `end` are equal (so the range is empty), the Java implementation raises an IllegalArgumentException with a message that includes 'start' and 'end'. The current code does not validate this condition and proceeds to compute a bound of 0, which later causes an error in `_next_int_java` with a different message. The fix is to add a check in `_random_internal` for an empty range when `chars` is None and both `letters` and `numbers` are False, raising a ValueError with a message that mentions 'start' and 'end'.
