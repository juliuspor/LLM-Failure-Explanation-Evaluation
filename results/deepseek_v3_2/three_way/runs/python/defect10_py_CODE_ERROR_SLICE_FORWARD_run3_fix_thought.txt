The bug occurs when processing a trailing 'L' and the numeric part becomes empty (e.g., val="L") or just a minus sign (val="-L"). The condition `char_at(numeric, 0) == "-"` will raise an IndexError if numeric is empty. The fix is to check that `numeric` has length > 0 before accessing index 0. Also, the condition `NumberUtils.is_digits(numeric[1:])` will fail if numeric is exactly "-" (since slicing gives empty string, which is_digits returns False). That's fine. So we add a length check for numeric before char_at. Additionally, we should ensure that the condition `NumberUtils.is_digits(numeric)` is only evaluated when numeric is not empty (but is_digits already returns False for empty string). However, the condition `char_at(numeric, 0) == "-"` must be guarded. The fix is to change the condition to `(len(numeric) > 0 and char_at(numeric, 0) == "-" and NumberUtils.is_digits(numeric[1:]))`. This ensures we only call char_at when numeric is non-empty. The rest of the function remains unchanged.
