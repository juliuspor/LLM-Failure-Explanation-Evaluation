The bug is that `to_locale` does not handle the double underscore format (language__variant) where country is empty. The current code assumes that after the first underscore there are exactly two uppercase letters for the country. For 'fr__POSIX', indices 3 and 4 are underscores, not uppercase letters, causing a ValueError. The fix is to detect when the character after the first underscore is another underscore, indicating an empty country, and then parse the variant directly.
