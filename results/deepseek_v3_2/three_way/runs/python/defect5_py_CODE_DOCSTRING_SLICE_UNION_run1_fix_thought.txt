The bug is that `type(array)` returns the container type (list) instead of the element type. We need to infer the element type from the first element of the array (if non-empty) or from the element being added. The fix: when array is not None, check if it has elements; if yes, use `type(array[0])`; else, infer from element (or default to object). This ensures inferred_type is the element type, not the list type.
