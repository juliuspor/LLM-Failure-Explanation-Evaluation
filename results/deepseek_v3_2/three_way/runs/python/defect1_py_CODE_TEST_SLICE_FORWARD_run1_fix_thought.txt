The bug is in the calculation of total minutes when hours_offset is negative. The current code subtracts minutes_offset from hours_in_minutes, which is correct when minutes_offset is positive but wrong when minutes_offset is negative (since subtracting a negative adds). The fix is to treat minutes_offset as a signed component and simply add it to hours_in_minutes, because hours_in_minutes already has the correct sign. So we can replace the conditional logic with a simple addition: total_minutes = hours_in_minutes + minutes_offset. This works for all sign combinations.
