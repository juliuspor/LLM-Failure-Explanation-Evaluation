The bug is in the loop that advances `pos` after a translator consumes codepoints. The original code incorrectly assumed that `consumed` is the number of code units, but it's the number of codepoints. For each consumed codepoint, we need to advance `pos` by the number of UTF-16 code units that represent that codepoint (1 for BMP, 2 for supplementary). The fix is to compute the codepoint at the current position and add its char count, iterating for each consumed codepoint. This ensures surrogate pairs are skipped as a whole.
