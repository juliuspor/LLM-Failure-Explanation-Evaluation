The bug is that -0.0 is incorrectly treated as an integer because int(-0.0) == 0 and -0.0 == 0 is True. This causes the negative sign to be lost. The fix is to check for negative zero using the existing is_negative_zero method before entering the integer branch. If it's negative zero, we output the string representation directly, which will be '-0.0'. This ensures the negative sign is preserved.
