The bug is that when both array and element are None, and expected_type is provided (e.g., str), the method incorrectly raises a TypeError because inferred_type is object. However, the test expects no error and returns [None]. The fix is to only raise the TypeError when we are trying to cast an existing non-None array of type object to a more specific expected_type. If array is None, we are creating a new list, and there is no cast involved, so no error should be raised. Therefore, we modify the condition to check if array is not None before raising the error.
