The bug occurs when both `array` and `element` are None, causing `inferred_type` to be set to `object`. This triggers a spurious TypeError when `expected_type` is provided and not `object`. The fix is to set `inferred_type` to `None` in that case, and then skip the type check when `inferred_type` is `None`. This allows the operation to proceed as intended (appending None to an empty list).
