The bug occurs because the original implementation assumes a strict format of language (2 letters) + '_' + country (2 letters) + optional '_' + variant. However, locale strings like 'fr__POSIX' have an empty country code (two underscores) which should be parsed as language='fr', country='', variant='POSIX'. The fix is to handle the case where after the first underscore, we might have another underscore (empty country) or a country code. We need to check if the character after the first underscore is another underscore, indicating an empty country, and then parse accordingly.
