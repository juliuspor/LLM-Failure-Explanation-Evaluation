The bug is that negative zero (-0.0) is incorrectly handled because the integer branch discards the sign. We need to treat negative zero as a float, not an integer. The fix is to add a special case for negative zero before the integer branch, using the existing is_negative_zero method. This ensures that -0.0 is output as "-0.0" (or "-0" if str(-0.0) yields "-0.0").
