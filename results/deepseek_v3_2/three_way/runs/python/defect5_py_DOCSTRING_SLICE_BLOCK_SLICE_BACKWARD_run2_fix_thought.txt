The bug occurs when both array and element are None, causing inferred_type to be set to object, which incorrectly triggers a TypeError when expected_type is not object. The fix is to set inferred_type to None in that case, indicating no specific type inference, and then adjust the condition to only raise TypeError when inferred_type is exactly object and expected_type is not object. This ensures that when both inputs are None, no cast failure is simulated.
