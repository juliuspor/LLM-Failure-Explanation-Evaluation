The bug was in the validation of `upper` and its conversion to a valid end index. The original code set `upper = len(text)` only if `upper == -1 or upper > len(text)`, but then later called `substring_java` with `end = upper` without ensuring `upper` is within bounds. The fix ensures `upper` is clamped to the string length after handling the -1 case, and also ensures `upper` is not less than `lower`. This prevents the IndexError when `upper` exceeds the string length.
