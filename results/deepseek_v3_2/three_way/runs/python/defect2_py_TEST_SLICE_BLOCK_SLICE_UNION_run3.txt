The test `test_lower_bound_clamping` fails because the `get_paint` method does not clamp values below the lower bound. At line 74, `g` is computed as `int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)`. For `value = -0.5` and default bounds (0.0, 1.0), this yields `int((-0.5 - 0.0) / (1.0 - 0.0) * 255.0) = int(-127.5) = -128`. Since `g = -128`, the check at line 77 (`if g < 0 or g > 255:`) triggers a `ValueError` instead of clamping the value to 0 (black). The same issue occurs for values above the upper bound, causing `test_upper_bound_clamping` to also fail. The method lacks clamping logic before computing `g`.
