The bug occurs when `char_at` is called with an empty string `numeric` (e.g., when `val` is a single character like 'L'). The fix adds a length check before calling `char_at` to ensure the string is not empty, preventing the IndexError. Specifically, in the 'L' case, we check `len(numeric) > 0` before accessing `char_at(numeric, 0)`. This aligns with the Java implementation's behavior and avoids the out-of-bounds error.
