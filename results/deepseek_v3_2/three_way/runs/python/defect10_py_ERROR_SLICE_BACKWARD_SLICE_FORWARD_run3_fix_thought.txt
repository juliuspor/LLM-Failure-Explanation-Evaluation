The bug occurs when the input string ends with 'L' (or 'l') and the numeric part is empty (e.g., 'L' or '-L'). In the branch for 'L' handling, the code calls char_at(numeric, 0) where numeric is val[:-1] (which could be empty). For example, val='L' -> numeric='' (empty). char_at('', 0) raises IndexError because the string is empty. The fix is to check that numeric is not empty before accessing char_at. Specifically, we need to ensure numeric has at least one character when checking if the first character is '-'. The condition should be: (len(numeric) > 0 and char_at(numeric, 0) == '-' and NumberUtils.is_digits(numeric[1:])). Also, note that the condition for 'L' already requires dec and exp to be None, but numeric could be empty. The fix adds a length check.
