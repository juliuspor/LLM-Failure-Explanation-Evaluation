The bug is that `inferred_type` was set to the type of the list container (`type(array)`) instead of the element type. To fix, we need to infer the element type from the first element of the list (if available). If the list is empty, we default to `object`. If the array is None but element is not None, we use the type of the element. Otherwise, default to `object`. This ensures that the type check works correctly for simulating Java array casting.
