The bug occurs when both array and element are None, and expected_type is provided (e.g., str). The original code sets inferred_type = object, which triggers the type check and raises a TypeError incorrectly. The fix is to set inferred_type to expected_type when both array and element are None and expected_type is provided; otherwise keep object as default. This ensures that when creating a new list from scratch with a None element and an expected type, the type check passes because inferred_type will match expected_type (if provided).
