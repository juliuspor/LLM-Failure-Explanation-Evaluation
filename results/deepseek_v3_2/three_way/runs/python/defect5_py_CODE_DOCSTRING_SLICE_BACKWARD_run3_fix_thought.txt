The bug is that `inferred_type` is set to `type(array)` which returns the list class, not the element type. We need to infer the element type from the first element of the list (if available) or from the element being added. The fix: when `array` is not None, check if it's non-empty; if so, get the type of the first element. If empty, fall back to the type of the element (or object). This ensures `inferred_type` is the element type, not the list type, allowing the type-safety check to work correctly.
