The bug is that `add_number` incorrectly converts -0.0 to integer 0, losing the negative sign. The fix is to treat -0.0 as a special case: if x is negative zero, output '-0.0' directly. Also, we must ensure that for other negative numbers, the sign is preserved. The existing check for negative numbers before adding a space is fine, but we need to handle -0.0 before the integer conversion path.
