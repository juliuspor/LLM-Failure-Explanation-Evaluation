The original function had rigid length checks and character validation that failed for locale strings with double underscores (like 'fr__POSIX') or non-standard country codes. The fix uses a more flexible approach: split the string by underscores into up to three parts. Validate the language code (2 or 3 letters). For the second part, check if it's a valid 2-letter uppercase country code or a 3-digit numeric area code; if not, treat it as part of the variant. Handle empty country part (double underscore) by moving remaining parts to variant. This accommodates POSIX and other special locales while maintaining compatibility with standard formats.
