In `code_point_at`, when `index` is 0 (the start of the string), it checks if the unit at index is a high surrogate and if the next unit is a low surrogate. If so, it returns the combined code point. However, the bug is that after processing a surrogate pair, the calling code (likely in `translate_string`) advances the index by 1, but it should advance by 2 because a surrogate pair consists of two code units. When processing index 0, it correctly combines the surrogate pair, but then the index increments to 1 (pointing to 0xDE30, the low surrogate). The function `code_point_at` at index 1 will see a low surrogate (0xDE30) which is not a high surrogate, so it returns 0xDE30 as a single code point, which is invalid.
