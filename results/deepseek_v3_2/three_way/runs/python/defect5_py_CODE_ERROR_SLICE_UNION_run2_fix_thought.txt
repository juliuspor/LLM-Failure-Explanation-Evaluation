The bug is that `inferred_type` incorrectly captures the list's type instead of the element type. When `array` is not None, we need to infer the element type from the first element (if any) or default to `object`. When `array` is None, we should infer from `element` or default to `object`. The fix adjusts the logic to set `inferred_type` to the element type, not the list type.
