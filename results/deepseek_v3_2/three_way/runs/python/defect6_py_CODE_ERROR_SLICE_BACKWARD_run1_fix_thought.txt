The bug occurs because `x == int(x)` is true for `-0.0` (since `int(-0.0)` is `0` and `-0.0 == 0` is true). This leads to the integer branch, which loses the negative sign. The fix is to first check for negative zero using `is_negative_zero(x)`. If true, output `"-0.0"`. Otherwise, proceed with the existing logic.
