The bug was in the for loop that incorrectly advanced `pos` by calling `char_count` multiple times, potentially overshooting the length of `code_units`. The fix is to remove the for loop and simply advance `pos` by `consumed`, which is the number of code units already processed by `translate_codepoint`. This matches the contract that `translate_codepoint` returns the number of code units consumed from the input. Also, when `consumed == 0`, we already advance `pos` by `char_count(codepoint)`. The corrected loop ensures `pos` stays within bounds.
