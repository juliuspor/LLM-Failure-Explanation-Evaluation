The bug occurs when both array and element are None, and expected_type is provided (e.g., str). The inferred_type becomes object, and the condition `inferred_type == object and expected_type != object` triggers a TypeError, which is incorrect because there is no existing array to cast. The fix is to only raise the TypeError when array is not None (i.e., there is an existing list that could be of type object). If array is None, we are creating a new list, and the expected_type is just a hint; we should not raise an error. Therefore, add a check `if array is not None:` before raising the TypeError.
