The bug is in the for loop that advances pos when consumed > 0. The original loop recalculates the codepoint at the current pos each iteration, but pos is updated inside the loop, causing it to skip extra characters. The fix is to compute the new position by iterating over the consumed codepoints using a temporary index, then assign pos = temp_pos after the loop. This ensures pos advances by the total code units of the consumed codepoints, not by the sum of lengths of consecutive codepoints starting from the original pos.
