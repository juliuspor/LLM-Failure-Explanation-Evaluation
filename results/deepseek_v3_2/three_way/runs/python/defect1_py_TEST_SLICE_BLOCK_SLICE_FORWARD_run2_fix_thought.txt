The bug is in the sign handling when hours_offset is negative and minutes_offset is positive. The current code incorrectly adjusts minutes_offset based on the sign of hours_offset, but the logic is flawed. For negative hours, we need to subtract the positive minutes from the negative total, not add them. The fix is to compute total minutes as hours_offset * 60 + minutes_offset, preserving the sign of each component, then convert to milliseconds.
