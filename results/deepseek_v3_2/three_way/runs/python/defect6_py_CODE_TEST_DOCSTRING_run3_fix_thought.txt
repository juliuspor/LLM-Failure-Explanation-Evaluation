The bug is that negative zero (-0.0) is incorrectly handled. The condition `x == int(x)` is true for -0.0, causing it to be treated as an integer and output as "0". To fix, we add a special case for negative zero using the existing `is_negative_zero` method. If the number is negative zero, we output "-0.0" and return early, bypassing the integer branch. This ensures the sign is preserved.
