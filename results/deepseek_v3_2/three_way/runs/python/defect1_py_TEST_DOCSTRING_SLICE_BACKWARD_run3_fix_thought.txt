The bug was in the calculation of total minutes. The original code incorrectly used a variable named `minutes_offset` to store the total minutes after adjusting for sign, which caused confusion. The fix correctly computes total minutes by adding or subtracting the minute component based on the sign of the hour component. For negative hours, minutes should be subtracted (making the offset more negative). For positive hours, minutes should be added. This ensures that (-2, 15) yields -2 hours - 15 minutes = -135 minutes total, which matches the expected ID "-02:15".
