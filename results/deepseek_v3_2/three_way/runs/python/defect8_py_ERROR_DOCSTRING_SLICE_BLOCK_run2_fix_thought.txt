The original function assumed a strict format with single underscores separating language, country, and variant, and validated positions based on fixed indices. This failed for strings like "fr__POSIX" where the country is empty. The fix uses a more flexible parsing approach: split by underscore, then interpret parts according to locale conventions. Language must be lowercase alphabetic. Country is optional and can be two uppercase letters or three digits. Variant is any remaining part. Empty country (double underscore) is allowed. This matches Java's Locale semantics and handles the problematic case.
