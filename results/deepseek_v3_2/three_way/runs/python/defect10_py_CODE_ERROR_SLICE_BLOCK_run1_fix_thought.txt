The bug occurs when processing a single-character qualifier like 'L', where `numeric` becomes an empty string and `char_at(numeric, 0)` raises an IndexError. The fix replaces the `char_at` call with a direct length check and indexing to safely handle empty strings, ensuring the condition only proceeds if `numeric` is non-empty and the first character is '-'. This maintains the original logic while preventing the out-of-range access.
