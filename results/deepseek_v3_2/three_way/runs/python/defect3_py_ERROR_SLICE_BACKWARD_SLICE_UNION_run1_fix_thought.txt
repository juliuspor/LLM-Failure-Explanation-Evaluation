The bug was in the loop's update logic after a translator consumed codepoints. The original code incorrectly used `char_count(code_point_at(...))` inside a loop, which could skip too many code units and cause an IndexError. The fix ensures `pos` is incremented by the exact number of codepoints consumed (`consumed`), as the translator's return value already accounts for surrogate pairs. This matches the contract that `translate_codepoint` returns the count of codepoints consumed, not code units.
