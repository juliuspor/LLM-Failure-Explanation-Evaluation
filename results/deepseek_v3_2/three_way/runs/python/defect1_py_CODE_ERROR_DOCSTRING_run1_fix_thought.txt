The bug was that the validation for minutes_offset incorrectly restricted it to 0-59, while negative minutes (e.g., -15 for offset -01:15) should be allowed. Additionally, the calculation of total minutes was flawed: it reused the minutes_offset variable and incorrectly subtracted when hours_offset was negative. The fix expands the validation to allow -59 to 59, and computes total_minutes as hours_offset * 60 + minutes_offset directly, which correctly handles both positive and negative components.
