The bug was in the calculation of total minutes when hours_offset is negative. The original code subtracted minutes_offset from hours_in_minutes when hours_in_minutes was negative, which incorrectly doubled the negative contribution of minutes. Since minutes_offset is always non-negative (0-59), the correct total minutes is simply hours_in_minutes + minutes_offset regardless of the sign of hours_offset. The fix replaces the conditional subtraction with a single addition line.
