The original to_locale method used fixed character positions and assumed a specific format, which failed for locale strings with empty country codes (e.g., 'fr__POSIX'). The fix replaces the rigid parsing with a split on underscore, handling 1, 2, or 3+ parts. It validates language codes (lowercase letters) and country codes (uppercase letters, if present). It also allows empty country strings and treats the second part as variant if it's not a two-letter uppercase country code. This matches the expected behavior for Java's Locale parsing.
