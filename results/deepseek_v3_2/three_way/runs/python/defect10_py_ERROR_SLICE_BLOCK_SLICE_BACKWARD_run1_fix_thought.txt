The bug occurs when `char_at` is called with an empty string for `numeric` (which happens when `val` ends with 'L' and `numeric` is empty after stripping the suffix). The fix is to ensure that `numeric` is not empty before calling `char_at`. In the Java original, `String.charAt(0)` on an empty string would throw `StringIndexOutOfBoundsException`. We should mimic that by checking if `numeric` is empty and raising a `ValueError` accordingly, because an empty numeric part is invalid for a number with an 'L' suffix.
