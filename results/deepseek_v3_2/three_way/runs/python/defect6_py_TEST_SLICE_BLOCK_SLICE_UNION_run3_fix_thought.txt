The bug is that negative zero (-0.0) is incorrectly printed as '0' instead of '-0.0'. This happens because -0.0 passes the integer check (x == int(x)) and enters the integer branch, where int(x) yields 0, losing the sign. The fix is to treat negative zero as a special case before the integer branch, using the existing is_negative_zero method to detect it and output '-0.0'.
