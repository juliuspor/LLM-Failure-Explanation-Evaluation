The bug was that minutes_offset validation incorrectly restricted to 0-59, but the function should allow negative minutes (e.g., -15) to represent offsets like -02:15. The fix expands the validation range to -59 to 59. The logic for combining hours and minutes already handles negative minutes correctly: when hours_in_minutes is negative, it subtracts minutes_offset, which works for both positive and negative minutes_offset values. For example, (-2, -15) => hours_in_minutes = -120, minutes_offset = -120 - (-15) = -105, which equals -2h -15m = -135m. For (-2, 15) => minutes_offset = -120 - 15 = -135, also correct. So only the validation needs to change.
