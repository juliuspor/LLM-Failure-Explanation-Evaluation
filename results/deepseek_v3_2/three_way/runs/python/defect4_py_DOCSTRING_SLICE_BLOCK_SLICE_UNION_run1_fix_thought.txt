The bug was in the conditional logic for updating min/max indices. The original code had an else block that only executed when the index was -1, but the update condition inside the if block was not sufficient. The fix restructures each update into a clear pattern: if the index is -1, set it; otherwise, compare with the current candidate and update if the new value is better (less for min, greater for max). Also, there was a typo in the max middle index section where it referenced self._min_middle_index instead of self._max_middle_index. That is corrected.
