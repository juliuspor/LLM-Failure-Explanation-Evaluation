The bug was in the validation and calculation of the offset when hours_offset is negative. The original code incorrectly handled the sign of minutes_offset and didn't enforce consistency between the signs of hours and minutes. The fix ensures that when hours_offset is negative, minutes_offset must be non-positive (0 or negative) and within -59 to 0. When hours_offset is positive, minutes_offset must be non-negative (0 or positive) and within 0 to 59. The calculation is simplified to total_minutes = hours_offset * 60 + minutes_offset, which correctly combines the signs. This matches the expected behavior: (-2, 15) raises ValueError (since minutes positive with negative hours), (-2, -15) yields -2h15m, (2, -15) raises ValueError, and (2, 15) yields +2h15m.
