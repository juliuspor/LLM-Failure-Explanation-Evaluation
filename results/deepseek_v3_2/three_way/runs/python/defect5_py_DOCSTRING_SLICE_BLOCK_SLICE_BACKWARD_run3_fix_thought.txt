The bug occurs when both `array` and `element` are None, causing `inferred_type` to be `object`. The subsequent type check incorrectly raises a TypeError if `expected_type` is not `object`, even though there's no actual list to cast. The fix is to only raise the TypeError when `array` is not None, because the casting error simulation only makes sense when there is an existing list to cast. This ensures that creating a list from None inputs with a non-`object` `expected_type` is allowed.
