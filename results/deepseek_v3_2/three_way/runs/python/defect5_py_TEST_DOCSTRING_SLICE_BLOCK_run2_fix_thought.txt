The bug occurs because when `array` is None, `inferred_type` is set to `object`, which triggers the TypeError when `expected_type` is not `object`. However, the test expects that when `array` is None, the function should simply return a list with the element appended, ignoring the `expected_type` check. The fix is to only perform the `expected_type` check when `array` is not None, because the type inference is only meaningful when there's an existing array. If `array` is None, we should treat it as an empty list and not enforce the type check.
