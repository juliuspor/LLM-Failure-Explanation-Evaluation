The bug is that `to_locale` fails to parse locale strings with an empty country code (e.g., 'fr__POSIX'). The original code assumes that after the first underscore, positions 3 and 4 are uppercase letters for a country code. However, when the country is empty, the second character after the language is another underscore. The fix is to detect this pattern: if the character at index 2 is '_' and the character at index 3 is also '_', then the country is empty and the variant starts at index 4. We must adjust the validation and parsing accordingly.
