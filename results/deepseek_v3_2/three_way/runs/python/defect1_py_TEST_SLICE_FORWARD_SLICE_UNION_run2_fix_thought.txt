The bug was in the sign handling when combining hours and minutes. The original code incorrectly used a variable named `minutes_offset` to store the total minutes, which caused confusion. The fix clarifies the logic: compute total minutes by adding or subtracting the minutes component based on the sign of hours. For negative hours, we subtract positive minutes (making total more negative). For positive hours, we add minutes. This ensures that (-2, 15) yields -135 minutes (i.e., -02:15).
