The bug is in the while loop's update logic when consumed > 0. The original code used a for loop that iterated consumed times, each time advancing pos by char_count(code_point_at(code_units, pos)). This could fail to advance pos if char_count returned 0, causing an infinite loop. Moreover, it incorrectly treated consumed as a count of characters rather than code units. The fix is to simply increment pos by consumed directly, as translate_codepoint returns the number of code units consumed. Also, when consumed == 0, we need to advance pos by the number of code units for the current codepoint, which is char_count(codepoint). The original code used len(c) which is the same as char_count(codepoint) (since to_chars returns a list of code units). However, for consistency, we should use char_count(codepoint).
