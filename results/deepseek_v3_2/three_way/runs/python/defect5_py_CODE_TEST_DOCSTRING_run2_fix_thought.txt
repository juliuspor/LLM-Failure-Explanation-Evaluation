The bug occurs when both array and element are None, causing inferred_type to be set to object, which triggers an unnecessary TypeError when expected_type is provided (e.g., str). The fix is to set inferred_type to None when both inputs are None, and only raise the TypeError when inferred_type is explicitly object (i.e., when array is a list of objects). Additionally, we should skip the type check when inferred_type is None because we cannot determine the type. This ensures that adding None to None with an expected_type does not raise an exception.
