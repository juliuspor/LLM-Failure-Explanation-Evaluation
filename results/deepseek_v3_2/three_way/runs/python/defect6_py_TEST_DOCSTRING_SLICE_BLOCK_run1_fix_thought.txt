The bug is that `add_number` incorrectly handles negative zero. When `x` is -0.0, `int(x)` returns 0, losing the sign. The fix is to treat negative zero as a special case: if `x` is negative zero, output `"-0.0"` directly. Also, ensure that for other negative numbers, the sign is preserved. The existing logic for adding a space before a negative number after a '-' character already works, but we must avoid converting negative zero to integer.
