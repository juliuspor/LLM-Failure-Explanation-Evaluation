The bug is in the for loop that advances pos after a translator consumes multiple codepoints. The original code called char_count(code_point_at(code_units, pos)) but pos wasn't updated between iterations, leading to repeated evaluation of the same codepoint and incorrect advancement. The fix is to update pos inside the loop by the char_count of the current codepoint, moving to the next codepoint each iteration. Also, the zero-consumption branch should use char_count(codepoint) instead of len(c) (they are equivalent but char_count is more explicit).
