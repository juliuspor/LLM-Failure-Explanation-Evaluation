The bug was that the loop incorrectly advanced `pos` by the number of code units per codepoint (`char_count`) for each consumed codepoint, but `consumed` is supposed to be the number of code units already consumed by the translator. The fix is to treat `consumed` as the number of UTF-16 code units consumed, and simply add `consumed` to `pos`. This matches the contract that `translate_codepoint` returns the number of code units it processed.
