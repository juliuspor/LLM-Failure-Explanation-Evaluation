The bug is in the type inference logic. `type(array)` returns `list`, not the element type. We should infer element type from the first element of the array (if present) or from the element being added. Since Python doesn't have static typing like Java, the `expected_type` check is only for simulation. The fix: when array is not None, check if it's non-empty and get type of first element; otherwise infer from element. Also, compare `inferred_type` with `expected_type` directly, not with `object` class.
