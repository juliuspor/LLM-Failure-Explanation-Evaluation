The bug is that when both array and element are None, the inferred_type is set to object, which triggers a TypeError if expected_type is not object. However, the test expects no error because there is no actual array to cast. The fix is to only perform the type check when array is not None, because the type safety check should only apply when we have an existing array with a component type. If array is None, we are creating a new list, and there is no existing type to cast. Therefore, we modify the condition to raise TypeError only when array is not None and inferred_type is object and expected_type is not object.
