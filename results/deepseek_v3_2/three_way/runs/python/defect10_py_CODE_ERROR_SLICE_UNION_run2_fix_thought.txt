The bug occurs when `val` is a single non-digit character (e.g., 'f'), leading to an empty `numeric` string. The call to `char_at(numeric, 0)` then raises an IndexError. The fix is to check if `numeric` is empty before calling `char_at`. If empty, it's not a valid number, so raise ValueError. This matches the Java behavior where a lone 'f' is invalid.
