The original code assumed a fixed structure with exactly one underscore between language and country, and another between country and variant. It failed on double underscores (e.g., 'fr__POSIX'). The fix generalizes parsing: after validating the two‑letter language, we find the next underscore. If none, the remainder is the country (with relaxed validation). If there is a second underscore, we split into country and variant parts, allowing empty country (double underscore case). This matches Java's Locale semantics where double underscore indicates a variant without a country.
