The code fails because the `get_paint` method does not clamp input values before computing the grayscale component. If `value` is below `self._lower_bound` (e.g., -0.5 when lower_bound is 0.0), the numerator becomes negative, resulting in a negative `g`. Since `g` is then checked on line 77 (`if g < 0 or g > 255`), a `ValueError` is raised because `g` is outside the 0..255 range. The test expects negative values to be clamped to black (0,0,0) instead of raising an exception. The issue occurs because the clamping logic is missing before the interpolation calculation.
