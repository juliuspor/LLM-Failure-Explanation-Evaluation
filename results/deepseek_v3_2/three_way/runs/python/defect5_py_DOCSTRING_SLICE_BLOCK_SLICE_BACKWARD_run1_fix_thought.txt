The bug occurs when both `array` and `element` are None, causing `inferred_type` to be set to `object`. Then, if `expected_type` is provided and is not `object`, a TypeError is raised unnecessarily. The fix is to only perform the type check when `array` is not None, because if the array is None, we are creating a new list and there is no existing list to cast. This prevents the false positive when both inputs are None.
