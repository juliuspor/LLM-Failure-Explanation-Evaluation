The bug is in the `translate` method's loop when handling consumed codepoints. The original code incorrectly assumed that `consumed` is the number of code units, but it's the number of codepoints consumed. The fix is to advance `pos` by the total number of code units corresponding to the consumed codepoints. In the case where `consumed == 0`, we already correctly advance by `char_count(codepoint)`. In the `consumed > 0` branch, we need to iterate over each consumed codepoint and advance `pos` by the char count for each. This ensures proper handling of surrogate pairs.
