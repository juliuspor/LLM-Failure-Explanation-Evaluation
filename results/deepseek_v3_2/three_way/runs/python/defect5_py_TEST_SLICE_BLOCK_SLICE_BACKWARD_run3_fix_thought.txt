The bug occurs because when both `array` and `element` are None, `inferred_type` is set to `object`. Then, if `expected_type` is provided and not `object`, a TypeError is raised incorrectly. The fix is to skip the type check when `inferred_type` is `object` and the `element` is None, because there's no actual type conflict. We can adjust the condition to only raise if the element is not None and types mismatch.
