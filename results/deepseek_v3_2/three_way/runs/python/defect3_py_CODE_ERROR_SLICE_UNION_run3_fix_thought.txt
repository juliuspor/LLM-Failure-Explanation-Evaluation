The bug is in the while loop's update of `pos` after a non-zero `consumed`. The original code used a for loop that called `char_count(code_point_at(...))` repeatedly, which incorrectly handled surrogate pairs and could cause index errors. Since `consumed` already represents the number of UTF-16 code units processed by `translate_codepoint`, we should simply add `consumed` to `pos`. This ensures proper advancement through the code_units list.
