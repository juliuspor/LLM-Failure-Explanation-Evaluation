The issue is that code_point_at reads the UTF-16 code unit at the current pos, but if the code unit is part of a surrogate pair (e.g., a high surrogate), code_point_at will return the full Unicode codepoint by reading two code units (pos and pos+1). Then, char_count returns 2 for a surrogate pair. But the for loop iterates consumed times, and each iteration increments pos by char_count(...).
