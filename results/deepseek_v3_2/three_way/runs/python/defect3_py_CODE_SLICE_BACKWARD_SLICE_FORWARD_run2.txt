The code fails because the while loop can become infinite. In the loop, `pos` is only advanced if `consumed == 0`. When `consumed > 0`, the loop attempts to advance `pos` by iterating `for pt in range(consumed)`. Inside that loop, `pos` is updated with `pos += char_count(code_point_at(code_units, pos))`. However, `code_point_at` is called with the current `pos` each iteration, but `pos` is not incremented between calls to `code_point_at`. This means `code_point_at(code_units, pos)` will return the same codepoint every time, and `char_count` will return the same value (1 or 2). Thus, `pos` increases by the same amount repeatedly, but the loop runs `consumed` times, causing `pos` to jump too far, possibly exceeding `length`. More critically, if `consumed` is incorrectly large or `char_count` returns 0, `pos` might not increase at all, leading to an infinite loop. The issue is in the logic for advancing `pos` when `consumed > 0`.
