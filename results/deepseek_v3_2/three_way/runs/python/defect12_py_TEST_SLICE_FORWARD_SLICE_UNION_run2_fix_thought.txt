The bug occurs when the lower bound exceeds the string length. In the original Java implementation, if the lower bound is beyond the string length, the method returns the original string (or the substring up to the upper bound if within limits). The fix is to adjust the logic: if lower > len(text), then we should not attempt to find a space, and instead return the substring from 0 to min(upper, len(text)) with the suffix if truncated. Also, the helper function substring_java is too strict; we should avoid calling it with invalid indices. Instead, we can use Python's safe slicing directly after adjusting bounds.
