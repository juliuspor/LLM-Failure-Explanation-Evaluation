The bug is that `add_number` does not handle negative zero correctly. In Python, -0.0 == 0.0 is True, so the condition `x == int(x)` holds (since int(-0.0) is 0) and the integer path is taken, outputting "0" instead of "-0.0". We need to check for negative zero before any other processing and output "-0.0" explicitly. The method already has a helper `is_negative_zero`, so we can use it. If x is negative zero, we should output "-0.0" directly, bypassing the integer formatting logic.
