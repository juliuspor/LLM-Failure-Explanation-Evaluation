The original function assumed a strict structure: either 2 chars (language), 5 chars (language + '_' + 2-letter country), or at least 7 chars (language + '_' + country + '_' + variant). It didn't account for the case where the country code is missing (empty) and a variant is present, e.g., "fr__POSIX". The fix is to parse the locale string by splitting on underscores, allowing empty country codes. We split into at most 3 parts. The first part is the language (must be 2 letters, lowercase). The second part, if present, could be a 2-letter uppercase country code or, if not a valid country code (or empty), it's treated as the variant (or empty country). The third part, if present, is the variant (or part of variant if second part is empty). This approach handles all valid locale strings: "en", "en_US", "en_US_POSIX", "en__POSIX".
