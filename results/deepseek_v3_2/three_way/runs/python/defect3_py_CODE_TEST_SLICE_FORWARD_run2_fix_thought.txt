The bug is in the for loop that advances pos when consumed > 0. The original code called char_count(code_point_at(code_units, pos)) inside the loop, but pos was not updated between iterations, causing double counting for surrogate pairs. The fix is to compute the codepoint at the current pos, then increment pos by its char count, ensuring pos moves forward correctly. Also, note that the loop variable pt is not used, so renamed to _ for clarity.
