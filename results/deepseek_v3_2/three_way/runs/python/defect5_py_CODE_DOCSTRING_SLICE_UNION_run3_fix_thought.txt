The bug is that `type(array)` returns the list class, not the element type. To infer the element type, we should look at the first element of the array if it exists and is non-empty. If the array is None or empty, we fall back to the type of the element (if not None) or object. This matches the intended behavior: `inferred_type` should be the element type, not the container type. Also, we need to handle the case where array is not None but empty; in that case, we cannot infer from array[0], so we should fall back to element type. The fix changes the logic to check if array is not None and has at least one element, then use type(array[0]); otherwise, proceed as before. This ensures inferred_type is an element type (like int, str) or object.
