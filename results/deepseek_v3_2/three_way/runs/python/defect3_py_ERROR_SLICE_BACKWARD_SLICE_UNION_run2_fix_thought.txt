The bug is in the loop after a translator consumes codepoints. The original code incorrectly used `char_count(code_point_at(code_units, pos))` for each consumed codepoint, which could advance `pos` incorrectly when dealing with surrogate pairs. Instead, `consumed` already represents the number of codepoints consumed, so `pos` should be incremented by `consumed` directly. This ensures proper index progression regardless of whether the consumed codepoints are single code units or surrogate pairs.
