The original code had a bug: when `lower` is greater than or equal to the length of the text, `StringUtils.index_of` returns -1, and then the code proceeds to abbreviate at `upper`. However, the Java version (Apache Commons Lang) returns the original string unchanged in that case. The fix adds a check: if `lower >= len(text)`, return the original text. This matches the Java behavior and prevents an unnecessary abbreviation when the lower bound is beyond the string.
