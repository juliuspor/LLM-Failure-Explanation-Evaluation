The bug is in the loop that advances `pos` after a translator consumes codepoints. The original code incorrectly used `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. This calls `code_point_at` each iteration, which may read a surrogate pair as a single codepoint, but `pos` is not updated between calls, leading to reading the same codepoint multiple times and advancing incorrectly. The fix is to compute the advance correctly: for each consumed codepoint, get the codepoint at current `pos`, then add its charCount. This ensures proper handling of surrogate pairs.
