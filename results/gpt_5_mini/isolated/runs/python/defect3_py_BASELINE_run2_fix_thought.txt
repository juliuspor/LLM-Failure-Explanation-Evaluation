Avoid calling code_point_at while advancing pos after a positive 'consumed'. Use the number of UTF-16 code units for each codepoint consumed: for each of the 'consumed' codepoints, determine its codepoint value using code_point_at at the current pos, compute char_count for that codepoint, and advance pos by that char_count. But to avoid IndexError, we must check pos < length before calling code_point_at. The consumed count represents codepoints consumed by translator; translator should not have consumed beyond available; so loop until consumed times but if pos>=length break. Implement safer loop: for _ in range(consumed): cp = code_point_at(code_units, pos); pos += char_count(cp). Also handle case consumed <0 by raising ValueError.
