Instead of using the inferred_type as the list type sentinel and raising when it's object, inspect elements to determine a common element type. If expected_type is provided, attempt to ensure each element is an instance of expected_type or None; if not, try to coerce by calling expected_type(element) when safe, otherwise raise TypeError. When array is None, create a new list with the single element. This avoids rejecting when inferred_type == object but elements are compatible with expected_type.
