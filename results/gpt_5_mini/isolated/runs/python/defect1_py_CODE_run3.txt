The code fails because of an integer sign bug in the minutes calculation: it overwrites the input minutes_offset variable with a value that mixes minutes and sign incorrectly. Specifically at lines where minutes_offset is reassigned: when hours_in_minutes < 0 it does minutes_offset = hours_in_minutes - minutes_offset, else minutes_offset = hours_in_minutes + minutes_offset. For negative hours_offset this subtracts minutes_offset from hours_in_minutes, producing a more negative total minutes than intended. Example: hours_offset = -1, minutes_offset = 30 -> hours_in_minutes = -60, then minutes_offset becomes -60 - 30 = -90 (should be -30). This wrong arithmetic leads to a too-large magnitude passed to FieldUtils.safe_multiply and eventually causes an ArithmeticError or incorrect offset. The root cause is reusing the minutes_offset variable for combined total minutes and using subtraction in the negative-hours case instead of adding the signed minutes part. The fix: compute total_minutes = hours_in_minutes + (hours_offset < 0 ? -minutes_offset : minutes_offset) or use total_minutes = hours_in_minutes + Math.copySign(minutes_offset, hours_offset) (or explicit branch) and then multiply that by MILLIS_PER_MINUTE. Also avoid catching overly broad ArithmeticError; validate ranges early to prevent overflow.
