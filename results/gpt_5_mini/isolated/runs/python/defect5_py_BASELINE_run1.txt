The code fails because inferred_type is set to type(array) when array is non-None; for a Python list this yields <class 'list'>, not the element type. Then later the code checks if inferred_type == object to decide whether to allow casting to expected_type (lines 332-337 and 343-347). When both array and element are None, inferred_type becomes object, so the code raises TypeError if expected_type is provided. In the failing test, ArrayUtils.add(None, None, expected_type=str) sets inferred_type = object and expected_type != object, so the check at line 343 triggers the TypeError with the message about casting Object[] to str[].
