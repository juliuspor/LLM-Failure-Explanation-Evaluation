The code fails because get_paint computes the grayscale component as int((value - lower) / (upper - lower) * 255.0) without clamping the input value before scaling. When value < lower or value > upper (e.g. -0.5 or 1.5), the computed component becomes negative or >255, causing ValueError: "Color parameter outside of expected range: Red Green Blue" where the color creation/check enforces 0..255. This happens in GrayPaintScale.get_paint (around the normalization/scale calculation). The root cause is missing clamping of value to the [lower, upper] interval prior to mapping to 0..255, and/or using int() that truncates 127.5 to 127 (expected) but not the problem. Fix: clamp value = max(lower, min(upper, value)) before computing fraction; compute fraction = (value - lower)/(upper - lower) guarding divide-by-zero; then component = int(fraction * 255.0 + 0.5) or use round and ensure result in 0..255.
