The bug is in translate(): advancing pos after a translator consumes N code points incorrectly recalculates character widths by repeatedly calling code_point_at at the advancing pos. Simpler and correct approach: when consumed>0, advance pos by the total number of UTF-16 code units that correspond to the consumed code points. Do this by calling code_point_at once per consumed codepoint starting at the original position, accumulating char_count for each, but ensure we read from a copy of the starting index. Implement by using a local variable i = 0..consumed-1 and index = pos and sum units using char_count(code_point_at(code_units, index)); increment index by char_count(...). Finally set pos = index.
