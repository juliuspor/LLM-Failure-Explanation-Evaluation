The code fails because the function to_utf16_code_units builds a list named code_units but never shows how it's filled in here; however the specific bug is in the slice_forward logic: at line 43 the calculation for a surrogate pair is incorrect because it omits adding the lower 10 bits properly — the formula should be 0x10000 + ((high - 0xD800) << 10) + (low - 0xDC00) which is present, so perhaps the real issue is that return high at line 44 returns the code unit (UTF-16) value, not a Unicode code point. When index points to a low surrogate (0xDC00–0xDFFF) that isn't preceded by a high surrogate, the function returns the low surrogate value instead of raising or handling it, causing invalid code points to be produced. Also index bounds check at line 35 raises IndexError if index out of range. The root cause: translate_codepoint expects a Unicode code point but slice_forward returns UTF-16 code units (sometimes a low surrogate) leading to invalid codepoints passed to translate_codepoint and downstream errors.
