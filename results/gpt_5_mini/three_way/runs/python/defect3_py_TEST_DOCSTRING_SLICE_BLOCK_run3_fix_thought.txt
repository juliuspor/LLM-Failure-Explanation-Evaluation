When a translator consumes codepoints, the loop must advance the position by the number of UTF-16 code units that represent those codepoints. The previous code used a loop that incremented pos by char_count(code_point_at(code_units, pos)) inside a for-range, which incorrectly recalculated and could step only one unit for surrogate pairs or read out-of-range. Fix by using the 'consumed' value to compute how many UTF-16 code units were consumed: iterate over the consumed codepoints, summing char_count of each starting codepoint, but update safely by checking bounds. Simpler: when consumed > 0, compute units = 0; for i in range(consumed): units += char_count(code_point_at(code_units, pos + units)); then pos += units. Also ensure access to code_units[pos] is always guarded by pos < length.
