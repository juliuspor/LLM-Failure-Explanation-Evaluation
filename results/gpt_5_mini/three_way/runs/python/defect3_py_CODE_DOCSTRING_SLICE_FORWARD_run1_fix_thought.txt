Fix the pos advancement after translators consume codepoints. Instead of looping and re-evaluating char_count at the same pos repeatedly, compute the number of UTF-16 code units to advance by summing char_count of each consumed codepoint starting from the original pos, advancing a temporary index for each. Alternatively, use a loop that for consumed times reads code_point_at at current temp index, adds char_count of that codepoint to pos_temp, then set pos = pos_temp. This avoids over-advancement.
