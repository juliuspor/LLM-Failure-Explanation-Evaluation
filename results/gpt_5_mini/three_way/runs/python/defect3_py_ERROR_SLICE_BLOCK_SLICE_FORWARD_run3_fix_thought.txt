The bug arises because translate() increments pos incorrectly when translators claim to have consumed N codepoints: it loops and calls code_point_at with pos which may equal length, causing IndexError. Fix by advancing pos based on consumed using code_point_at on the starting positions iteratively but ensure bounds: compute cp = code_point_at(... ) inside try/except and use char_count safely. Simpler: when consumed>0, advance pos by sum of char_count for each codepoint consumed, but by reading code units without calling code_point_at beyond length: iterate a temp_pos and for _ in range(consumed): cp = code_point_at(code_units, temp_pos); temp_pos += char_count(cp). Finally set pos = temp_pos. This avoids reading at pos==length.
