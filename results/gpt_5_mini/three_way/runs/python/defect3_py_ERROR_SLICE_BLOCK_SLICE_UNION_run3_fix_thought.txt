The IndexError arises when code_point_at is called with an index equal to len(code_units). In translate(), after a translator consumes 'consumed' > 0 codepoints, the loop advances pos by iterating 'consumed' times and calling code_point_at(code_units, pos). If pos has become equal to length (e.g., translator consumed at the end), code_point_at raises. Fix: advance pos by summing char_count(code_point_at(...)) but ensure we check pos < length before calling code_point_at. If pos >= length, break out of loop. Also simplify increment by using a for loop that checks bounds; if pos >= length, return. Implement guard to avoid IndexError and to progress pos when translators consume codepoints at end.
