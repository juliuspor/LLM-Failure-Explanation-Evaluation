The code fails because when computing max_middle it fetches the period using the wrong index: it uses self._min_middle_index at line where max_middle_period is assigned instead of self._max_middle_index. As a result, the current max_middle comparison compares middle against the wrong stored period (the minimum-middle period), so _max_middle_index can be set incorrectly (or not updated) and ends up with the wrong index (3 instead of expected 1). The bug occurs in the block that begins with "if self._max_middle_index >= 0:"; change get_data_item(self._min_middle_index) to get_data_item(self._max_middle_index) to fix it.
