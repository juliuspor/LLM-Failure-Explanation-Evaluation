The code raises IndexError when substring_java calls its slice helper with an end index beyond the string length. In abbreviate(), when upper is -1 it's set to len(text) which is fine, but elsewhere upper can be left larger than len(text) before the check. The critical path: index = StringUtils.index_of(text, " ", lower) may return -1 or a value >=0. Later, when index == -1, it does result = substring_java(text, 0, upper). If upper was set to len(text) earlier that's okay, but when index != -1 and index > upper branch returns substring_java(text, 0, upper) too. The failing case is when upper > len(text) because the earlier check only corrected upper when upper == -1 or upper > len(text) before index computation; but later code sets upper = lower when upper < lower which can increase upper to a value greater than len(text) if lower > len(text). Specifically if lower > len(text) and upper < lower, the line upper = lower sets upper to lower which may be > len(text). Then substring_java(text, 0, upper) is called with end > len(text), causing the IndexError from the slice guard (lines 32-34). The root cause: the check that clamps upper to string length happens before possibly overwriting upper when upper < lower. Move the clamp (if upper == -1 or upper > len(text): upper = len(text)) to after the possible upper = lower adjustment, or clamp upper again after that. Also validate that lower is not greater than len(text) before using it as an index. This ensures substring_java is never called with end > len(text).
