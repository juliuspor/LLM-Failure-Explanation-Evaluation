translate() advanced pos incorrectly when translate_codepoint consumes codepoints-count rather than code unit count. Fix by interpreting consumed as number of UTF-16 code units consumed (not codepoints), and advance pos by consumed directly. Also ensure when consumed==0 we advance by char_count(codepoint) as before. Handle negative or invalid consumed by raising ValueError. This keeps indexing in UTF-16 code unit space and prevents out-of-range accesses.
