The bug is caused by translate() iterating using UTF-16 code unit positions but translate_codepoint may return counts in code points, causing pos to advance incorrectly and eventually equal len(code_units). Fix by ensuring that when translate_codepoint returns 0, we advance pos by the number of code units for the codepoint; when it returns >0 (consumed codepoints), advance pos by the sum of UTF-16 code units for each consumed codepoint. Also guard against translate_codepoint being called when pos>=length by breaking. Essentially ensure pos remains < length and avoid calling code_point_at with out-of-range index.
