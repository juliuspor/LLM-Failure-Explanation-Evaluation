Avoid treating unknown type as a blocking cast failure. Determine inferred_type more concretely by checking array elements when array is provided and contains non-None items; fall back to element type if present; only raise TypeError when expected_type is provided and inferred_type is not object and incompatible. In other words, change the check to raise only if inferred_type is object and expected_type is not object? Actually previous behavior raised when inferred_type == object and expected_type != object. That caused failure when both array and element are None. Better to relax and not raise TypeError when inferred_type is object (unknown) — allow creation. Only raise if inferred_type is concrete and not compatible with expected_type.
