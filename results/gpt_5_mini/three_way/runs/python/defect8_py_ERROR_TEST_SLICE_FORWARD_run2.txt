The failure occurs because to_locale improperly assumes a country code is always present when the input length is >=5. In the slice shown (lines 233-237):
- Line 233 returns Locale(locale_str[0:2], locale_str[3:5]) for the 5-character case (e.g. "en_US").
- Lines 235-237 handle longer strings by checking locale_str[5] != '_' and then returning Locale(locale_str[0:2], locale_str[3:5], locale_str[6:]).

When the input is "fr__POSIX" (7+ chars), locale_str[3:5] yields "_" (characters at positions 3 and 4 are "__" so slice 3:5 is "_"). The code treats that as the country code but then enforces that locale_str[5] must be '_' (line 235). For "fr__POSIX" locale_str[5] is 'P', so the check fails and raises ValueError("Invalid locale format: fr__POSIX").

Root cause: the implementation does not handle an empty country field (language__variant) and assumes the two characters at positions 3 and 4 are a valid uppercase country code. It then checks the character after that for '_' to separate the variant, which is incorrect when the country is empty. This leads to mis-parsing and the ValueError.

Where it occurs: in to_locale, around lines 233 and 235–237, specifically the slices locale_str[3:5] and the check locale_str[5] != '_'.

How to fix (concrete steps):
1. Detect the double-underscore pattern immediately after the language: check if locale_str[2:4] == "__" (i.e. positions 2 and 3 are underscores). If so, treat country as "" and set variant = locale_str[4:]. Return Locale(locale, "", variant).
2. Otherwise, continue existing parsing: if length == 2 return language only; if length == 5 validate country slice locale_str[3:5] is two uppercase letters and return Locale(language, country); if longer and locale_str[5] == '_' validate country and return Locale(language, country, locale_str[6:]).
3. Add unit tests for the "language__variant" case (already present) and adjust the index checks to avoid reading past string end.

These changes avoid treating an empty country as an invalid uppercase country and prevent the false ValueError for inputs like "fr__POSIX".
