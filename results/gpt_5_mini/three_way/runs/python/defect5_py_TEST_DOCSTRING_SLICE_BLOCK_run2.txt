In Python, when both array and element are None (ArrayUtils.add(None, None, expected_type=str)), inferred_type will be object (the fallback), expected_type is str, so the condition triggers and raises TypeError. This is incorrect because adding two Nones with an expected_type should produce a list of that expected_type containing None (i.e., [None]) rather than raising. The check should instead consider whether the element actually violates expected_type (e.g., element is not None and not isinstance(element, expected_type)), or when array components determine a concrete element type different from expected_type.
