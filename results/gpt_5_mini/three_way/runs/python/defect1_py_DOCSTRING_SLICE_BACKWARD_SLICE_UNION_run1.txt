The code fails because when computing the fixed-offset milliseconds the sign handling for negative hours with minutes is wrong. Specifically, in the branch for hours_offset < 0 (around the lines that compute hours_in_minutes and adjust minutes_offset), the code sets minutes_offset = hours_in_minutes - minutes_offset (line ~383) which produces a more-negative minute total than intended. For example if hours_offset = -1 and minutes_offset = 30, hours_in_minutes = -60, then minutes_offset becomes -60 - 30 = -90 minutes (should be -30 minutes total). Similarly, the other branch uses minutes_offset = hours_in_minutes + minutes_offset causing incorrect totals for positive offsets with minutes. This incorrect arithmetic then multiplies minutes_offset by MILLIS_PER_MINUTE to produce the wrong offset value passed to for_offset_millis, and can also produce values outside allowed range causing the "Offset is too large" ValueError (line ~388). The root cause is mixing absolute and signed components instead of converting the full offset to a single signed total: compute total_minutes = hours_offset * 60 + (hours_offset < 0 ? -minutes_offset : minutes_offset), then multiply by MILLIS_PER_MINUTE. Fix by using that formula and validating ranges before constructing the DateTimeZone.
