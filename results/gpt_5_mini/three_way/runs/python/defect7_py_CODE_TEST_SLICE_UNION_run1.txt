The code fails because line 761 calls array[i].upper() without checking for None. When array contains None (e.g., test_array = ["Test", None, 99.0]), None has no attribute 'upper', causing an AttributeError. Specifically, in to_class at lines 760-762: the loop uses _ = array[i].upper() which assumes elements are strings; this both raises on None and is unnecessary for computing types. As a result, when a None element is encountered the function crashes before appending type(None). The root cause is the unguarded call to the string method .upper() on array elements. Suggested fix: remove that call or check if array[i] is not None and has upper attribute before calling, then append type(array[i]) (use type(None) for None).
