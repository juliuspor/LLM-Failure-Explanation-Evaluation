When translate_codepoint returns a count of code points consumed, the loop must advance pos by the number of UTF-16 code units those code points occupy without calling code_point_at out of range. I compute consumed_units by iterating up to consumed times, each time calling code_point_at at a safe index (pos + consumed_units) and adding char_count of that codepoint. This ensures code_point_at is never called with index >= length and advances correctly across surrogate pairs.
