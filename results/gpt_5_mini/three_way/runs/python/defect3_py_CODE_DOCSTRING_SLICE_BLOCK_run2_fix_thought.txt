The bug is in advancing pos after translators consume codepoints. The loop uses code_point_at(code_units, pos) while incrementing pos, which can call code_point_at with pos==length and raise IndexError (see code_point_at's index check at lines 35-36). Fix by iterating consumed times but computing the codepoint at the current pos before incrementing, then advance pos by char_count of that codepoint. Also avoid calling code_point_at when pos >= length. Use a for-loop that first gets cp = code_point_at(code_units, pos) then pos += char_count(cp).
