The bug comes from using code_units variable earlier in helper functions; but actual issue in translate is that when a translator consumes n codepoints it should advance pos by n codepoints, not by summing char_count of code_point_at for each pt; current loop recalculates code_point_at at same pos repeated causing IndexError if translator consumed surrogate pair and pos doesn't advance. Fix by advancing pos using code_point_at at current pos once per consumed iteration: for _ in range(consumed): cp = code_point_at(code_units, pos); pos += char_count(cp). Also guard consumed>0. This prevents out-of-range access.
