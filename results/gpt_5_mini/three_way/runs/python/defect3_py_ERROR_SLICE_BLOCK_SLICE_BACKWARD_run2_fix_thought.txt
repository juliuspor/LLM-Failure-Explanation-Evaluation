The bug arises from advancing pos incorrectly when translators consume codepoints. The translate method used pos += len(c) when no translation, but advanced by char_count(code_point_at(...)) inside consumed loop which caused index to jump beyond available code units because code_point_at raises IndexError if pos equals length. Fix by ensuring we check bounds before calling code_point_at and by advancing pos by char_count of the consumed codepoints safely. Simplest fix: when consumed>0, loop pt from 0 to consumed-1 and for each, get codepoint = code_point_at(code_units, pos) but catch IndexError and break; then advance pos by char_count(codepoint). Also ensure translate_codepoint returns number of codepoints (not code units), so counting is correct. Implement try/except around code_point_at to avoid IndexError and stop if out of range.
