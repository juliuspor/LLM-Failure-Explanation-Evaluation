The IndexError occurs when translate_codepoint is called with index equal to len(code_units). The caller loop in translate increments pos using char_count on code_point_at(code_units, pos) which raises when pos==len. The fix is to ensure pos remains < length before calling translate_codepoint and before computing char_count; in the post-consumed loop, increment pos by char_count of the codepoint at original positions, but avoid calling code_point_at when pos >= length. Simpler: after consumed !=0, increment pos by sum of char_count of codepoints consumed by iterating consumed times but using code_point_at only if pos < length. If pos >= length break.
