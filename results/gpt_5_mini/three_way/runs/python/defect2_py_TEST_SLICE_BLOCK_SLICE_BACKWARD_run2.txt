The code raises a ValueError when g is outside 0..255 (lines 77-78). This happens because g is computed with integer conversion of the normalized value without clamping (line 74): g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0). For inputs below lower_bound (e.g. -0.5 with default bounds) the normalized expression becomes negative, producing a negative g; for inputs above upper_bound it becomes >255. The subsequent check then raises instead of clamping. The failing tests expect clamping to 0 or 255 (see test_lower_bound_clamping and test_upper_bound_clamping). The root cause is missing clamping of ‘g’ after calculation (and dividing by zero should also be guarded if upper==lower). The error occurs in GrayPaintScale.get_paint at lines 74 and 77-78.
