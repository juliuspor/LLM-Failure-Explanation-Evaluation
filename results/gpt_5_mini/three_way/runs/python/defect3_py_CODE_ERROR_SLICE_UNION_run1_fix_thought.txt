Avoid calling code_point_at when pos may be out of bounds. After a translator consumes some codepoints, advance pos by summing the UTF-16 code unit counts of the consumed codepoints by reading code units directly rather than calling code_point_at at a possibly out-of-range index. Specifically, use a loop that examines code_units starting at original pos and advances by char_count(code_point_at(...)) but ensure we don't call code_point_at with index >= length; instead, iterate consumed times and increment pos by 1 for each consumed code unit unless a surrogate pair is found, in which case increment appropriately. Simpler: advance pos by the number of UTF-16 code units consumed, which equals the number of code units spanned by the consumed codepoints; since translate_codepoint contract is to return count of codepoints consumed, compute code units advanced by summing char_count(code_point_at(code_units, start_index_of_each_codepoint)). To avoid out-of-range, maintain a temp index and only call code_point_at while temp < length.
