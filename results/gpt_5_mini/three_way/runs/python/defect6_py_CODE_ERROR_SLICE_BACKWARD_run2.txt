The code treats integers specially when x == int(x), converting x to int(value). For negative zero (-0.0) in floating point, x == int(x) is true because int(-0.0) == 0, and math.isinf/math.isnan checks pass. The code then sets value = int(x), which loses the sign: int(-0.0) becomes 0. Later the code serializes value (or mantissa) with str(value), producing '0' instead of '-0.0'. The root cause is converting the float to an int too early and relying on int() which cannot represent signed zero; the sign information of -0.0 is lost before serialization (at line where value = int(x) right after the if on line 185).
