The reported issue is an IndexError from char_at when an invalid index is passed. The create_number function uses char_at(numeric, 0) to check first char of numeric; numeric can be empty in some flows which leads to IndexError. Fix: ensure numeric is non-empty before calling char_at; adjust condition to check numeric and use is_digits appropriately. Update create_number to avoid calling char_at on empty string and to handle signs correctly when numeric is empty.
