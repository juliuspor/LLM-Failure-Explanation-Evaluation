The code fails because of a logic bug in how minutes_offset is combined with hours_in_minutes when hours are negative. Specifically, in the slice around lines 383 and 385: the branch intended to handle negative hours incorrectly uses both subtraction and addition depending on flow, leading to wrong sign for minutes. The code checks minutes_offset range at lines 377-378, then for negative hours it computes hours_in_minutes - minutes_offset (line 383) in one path and hours_in_minutes + minutes_offset (line 385) in another. This causes cases like (-2, -15) and (-2, 15) to be handled inconsistently: one path subtracts a negative minutes_offset (turning it into addition) and the other adds it, producing incorrect total minutes_offset. The result is an incorrect offset value passed to FieldUtils.safe_multiply at line 386 and then to for_offset_millis, so tests expecting -02:15 (i.e. -(2*60 + 15) minutes) may get + or wrong magnitudes instead. In short, the root cause is inconsistent sign handling of minutes_offset with negative hours due to wrong use of '+' vs '-' at lines 383 and 385.
