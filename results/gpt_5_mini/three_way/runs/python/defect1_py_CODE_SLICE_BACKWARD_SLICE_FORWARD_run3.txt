The code mistakenly reassigns the parameter minutes_offset with a combined minutes value causing variable reuse and sign errors. Specifically: at line with hours_in_minutes = hours_offset * 60, then if hours_in_minutes < 0: minutes_offset = hours_in_minutes - minutes_offset else minutes_offset = hours_in_minutes + minutes_offset. This mutates minutes_offset (originally validated as 0..59) into total minutes, but the subtraction branch uses wrong sign when hours_offset is negative: for e.g. hours_offset = -1, minutes_offset = 30 -> hours_in_minutes = -60 -> minutes_offset becomes -60 - 30 = -90 but expected total -30. This is why offsets are wrong and can lead to large magnitude values causing FieldUtils.safe_multiply to overflow; ArithmeticError is then raised or incorrect offsets produced. The root cause is reusing the minutes_offset parameter and using minus instead of adding/subtracting the minutes correctly. Fix by computing total_minutes = hours_in_minutes + ( -minutes_offset if hours_offset < 0 else minutes_offset ) or use math.copysign/explicit sign handling, and avoid mutating the input parameter.
