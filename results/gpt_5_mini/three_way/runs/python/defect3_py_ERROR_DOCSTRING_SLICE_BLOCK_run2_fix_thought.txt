The bug is an IndexError caused by advancing pos using code point counts while code_units length is in UTF-16 code units. Fix by ensuring pos is advanced by UTF-16 code unit count. change loop: when consumed==0 advance by char_count(codepoint) already done; but when consumed>0, translators return number of codepoints consumed — we must advance pos by the number of UTF-16 code units corresponding to each consumed codepoint. So iterate consumed times, each time get codepoint at current pos and advance pos by char_count(codepoint). Also ensure code_point_at is not called with pos==len(code_units) by checking pos < length in loop. Add guard: if pos >= length: break.
