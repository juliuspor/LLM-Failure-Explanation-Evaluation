Allow adding None when array and element are both None with an expected_type specified. Only raise TypeError when inferred_type is object because of mixed unknown types from an existing array that is not None (i.e., array provided but its element type is generic object). Change condition to raise only when array is not None and inferred_type is object and expected_type is not object. This preserves simulation of Java cast failure for non-empty object-typed arrays while allowing add(None,None,expected_type=...) to succeed producing [None].
