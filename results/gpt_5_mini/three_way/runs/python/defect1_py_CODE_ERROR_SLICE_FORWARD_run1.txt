The code fails because the function modifies the minutes_offset parameter before using it and does not validate the resulting combined minute value. Specifically: - In lines 381-385 the code computes hours_in_minutes = hours_offset * 60 and then overwrites minutes_offset with either hours_in_minutes - minutes_offset (for negative hours_in_minutes) or hours_in_minutes + minutes_offset (for non-negative hours_in_minutes). This changes minutes_offset to a combined total minutes value, which may become negative even when the original minutes_offset argument was in the valid 0..59 range. - The input that caused the error was minutes_offset = -15, which passed the earlier validation (minutes_offset < 0 or minutes_offset > 59) only if minutes_offset was originally non-negative; however the ValueError indicates that the earlier check saw minutes_offset as -15. This means the validation caught the original invalid input, but the real bug is the sign handling logic: when hours_offset is negative, the code subtracts minutes_offset (line 383) instead of adding its absolute value, turning a small positive minutes_offset into a negative combined minutes_offset. - As a result, after the rewrite, the combined minutes_offset can fall outside 0..59 and produce incorrect exceptions or wrong offsets. The root cause is misusing minutes_offset both as an input and as a working variable, and incorrect arithmetic when hours_offset is negative. Fixes: 1) Do not mutate the minutes_offset parameter; use a new variable total_minutes. 2) When hours_offset is negative, compute total_minutes = hours_in_minutes - minutes_offset only if minutes_offset is positive? More clearly: compute total_minutes = hours_in_minutes + (minutes_offset if hours_offset >= 0 else -minutes_offset). 3) Keep the initial validation of minutes_offset (0..59) and adjust the arithmetic accordingly. 4) Replace FieldUtils.safe_multiply(minutes_offset, ...) with FieldUtils.safe_multiply(total_minutes, ...). Also update the early validation to accept negative minutes only when hours_offset is negative and handle that explicitly.
