The bug is in how pos advances when translators consume codepoints. Instead of looping consumed times and recalculating char_count from code_point_at at each step (which can double-count surrogate lengths), compute a local count of code units consumed by summing char_count for each codepoint starting at the original pos, advancing a temporary index to move through code_units consumed times, and finally set pos = temp_pos. This avoids calling code_point_at with out-of-range index and double-counting. Implement by using temp_pos and for _ in range(consumed): cp = code_point_at(code_units, temp_pos); temp_pos += char_count(cp); then pos = temp_pos.
