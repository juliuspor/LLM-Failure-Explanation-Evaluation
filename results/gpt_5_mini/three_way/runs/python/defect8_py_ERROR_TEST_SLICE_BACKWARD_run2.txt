The code fails because to_locale incorrectly assumes there are two uppercase country letters at positions 3 and 4 whenever length >= 5, and it unconditionally checks ch3 and ch4 without handling the special case of an empty country surrounded by double underscores (language__variant). Specifically, in to_locale at lines around 227-230 it sets ch3 = locale_str[3] and ch4 = locale_str[4] and then rejects when they are not uppercase A-Z. For input "fr__POSIX", locale_str[3] is '_' (not an uppercase letter) so the condition triggers and raises ValueError: "Invalid locale format: fr__POSIX". The root cause is missing logic to allow an underscore at position 3 when position 2 is also an underscore (i.e., language__variant where country is empty). The fix is to detect the double underscore pattern and parse variant from substring after the second underscore instead of requiring country letters at positions 3-4.
