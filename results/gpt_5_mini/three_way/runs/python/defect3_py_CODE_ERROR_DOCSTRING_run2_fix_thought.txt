Advance pos by the number of UTF-16 code units consumed as reported by translate_codepoint, not by recomputing char_count at the (moving) pos. When consumed>0, increase pos by the sum of char_count for each codepoint handled, but avoid reading beyond bounds. The translator reports consumed in codepoints, so to convert to code units, iterate over the next 'consumed' codepoints using code_point_at on a saved position and advance by char_count of that codepoint, but ensure we don't call code_point_at with an out-of-range index. Instead, compute code units advanced by checking code units directly: if current code unit is a high surrogate and next exists and is low surrogate, advance by 2 and count one codepoint; otherwise advance by 1. Repeat consumed times. This avoids out-of-range reads.
