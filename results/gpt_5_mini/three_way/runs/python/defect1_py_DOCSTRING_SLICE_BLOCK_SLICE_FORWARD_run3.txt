The code fails because minutes_offset is reassigned incorrectly when hours_offset is negative. In the negative branch (lines 383 and 385), minutes_offset is set by subtracting or adding from hours_in_minutes, which is the total minutes for the hour sign, instead of computing the remaining minutes to combine with the hour offset. This causes minutes_offset to become a large value (e.g., 60 - minutes) and then multiplied by MILLIS_PER_MINUTE, producing an offset larger than allowed, triggering the "Offset is too large" ValueError on line 388. Specifically, at line 383 the code does: minutes_offset = hours_in_minutes - minutes_offset, and at line 385: minutes_offset = hours_in_minutes + minutes_offset. Both use hours_in_minutes (which equals abs(hours_offset) * 60) rather than computing sign-correct total minutes like total_minutes = hours_offset * 60 + (hours_offset < 0 ? -minutes_offset : minutes_offset). As a result, for negative hours_offset the sign of the minutes is handled incorrectly and results in an out-of-range offset, causing the exception.
