The bug is caused by advancing pos by inspecting codepoints at the current pos after a translator consumed codepoints. The translator returns the number of codepoints consumed, but translate should advance pos by the number of UTF-16 code units corresponding to those consumed codepoints. Fix by computing the number of code units consumed by iterating consumed times and using char_count on code_point_at while ensuring bounds, or simpler: advance pos by summing char_count of each consumed codepoint, but avoid calling code_point_at when pos >= length. We'll compute units_consumed = 0 and loop while units_consumed < consumed and pos < length: cp = code_point_at(code_units, pos); units = char_count(cp); pos += units; units_consumed += 1. That prevents IndexError.
