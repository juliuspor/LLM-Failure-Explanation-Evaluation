The code fails because it assigns to the input parameter minutes_offset when computing the combined minute offset, which masks the original minutes and can produce incorrect negative/positive math when hours_offset is negative; specifically at line 383 onward it does: if hours_in_minutes < 0: minutes_offset = hours_in_minutes - minutes_offset else: minutes_offset = hours_in_minutes + minutes_offset. This mutates minutes_offset and then passes that value to FieldUtils.safe_multiply. If hours_offset is negative, e.g., hours_offset = -1 and minutes_offset = 30, hours_in_minutes = -60 and minutes_offset becomes -60 - 30 = -90, giving an offset of -90 minutes (correct) but the earlier validation required minutes_offset be 0..59; however the real failure indicated by ArithmeticError catch likely arises because hours_in_minutes is an int and FieldUtils.safe_multiply raises ArithmeticError for overflow when multiplying large minute values. The root bug, shown at lines 378-383, is that variable reuse and sign handling are incorrect: for negative hours the code subtracts minutes_offset instead of adding the magnitude, and it mutates minutes_offset rather than using a new variable, causing wrong offsets or overflow. Use a separate total_minutes variable: total_minutes = hours_in_minutes + ( -minutes_offset if hours_in_minutes < 0 else minutes_offset ) and then multiply safely.
