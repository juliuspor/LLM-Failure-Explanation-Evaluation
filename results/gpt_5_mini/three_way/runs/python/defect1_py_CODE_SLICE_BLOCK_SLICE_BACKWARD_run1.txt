The code incorrectly updates the minutes_offset variable when combining hours and minutes. On lines after computing hours_in_minutes, it uses:
    if hours_in_minutes < 0:
        minutes_offset = hours_in_minutes - minutes_offset
    else:
        minutes_offset = hours_in_minutes + minutes_offset
This is wrong for negative hour offsets. For example, hours_offset = -1 and minutes_offset = 30 should produce total minutes -30 (-(1*60) - 30 = -90? Wait). The intended math is: total_minutes = hours_in_minutes + (minutes_offset with sign matching hours_offset). Currently for negative hours it subtracts minutes_offset instead of adding the signed minutes (should be hours_in_minutes - minutes_offset but only if minutes_offset represents positive minutes to subtract; the real bug is more subtle)
Also variable misuse: reassigning minutes_offset loses the original minutes sign, and the check earlier only allows minutes_offset 0..59, so negative minute offsets aren't representable. The ArithmeticError catch is not appropriate for Python since OverflowError is rarely raised here. The likely root cause is wrong sign handling combining hours and minutes and misnaming leading to incorrect offset calculations and wrong results for negative offsets.
