The bug is that the loop advancing pos after a translator consumed codepoints double-counts surrogate pairs because it calls code_point_at and char_count inside a loop for 'consumed' times. The simplest fix: treat consumed as number of UTF-16 code units consumed and advance pos by consumed directly. Modify translate() to increment pos += consumed when consumed > 0, instead of the existing for-loop. Keep existing behavior when consumed == 0 unchanged.
