The function get_paint clamps input to bounds into local variable v but then calculates g using the original parameter value instead of the clamped v. If value is outside [self._lower_bound, self._upper_bound] this causes the fraction (value - lower)/(upper - lower) to be <0 or >1, so g can be <0 or >255, triggering the ValueError on lines 74 and 77.
