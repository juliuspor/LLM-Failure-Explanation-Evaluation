The code fails because it mistakenly reassigns the input parameter minutes_offset when computing the total offset in minutes. At lines 383-385 the variable minutes_offset is overwritten with the combined minutes value (hours_in_minutes ± minutes_offset), losing the original minutes sign information and causing wrong arithmetic and potential negative/positive sign errors. Specifically, when hours_in_minutes is negative the code does: minutes_offset = hours_in_minutes - minutes_offset, but minutes_offset was the original positive minutes offset; this yields an incorrect total minutes (should be hours_in_minutes - original_minutes, but reusing the same variable causes confusion). Moreover, using minutes_offset as both input minutes and as the combinedMinutes value then passes that to FieldUtils.safe_multiply, and because the variable names are misused the subtraction/addition logic is reversed for negative hour offsets. The root cause occurs in the block starting at line 383 where minutes_offset is reused instead of using a separate variable (e.g., total_minutes). This leads to wrong offset calculation and can trigger ArithmeticError or produce incorrect offsets returned by for_offset_millis.
