The code fails because it uses type(array) where it intended to get the element type: type(list) returns 'list', not the element type. As a result inferred_type is often 'list' or 'object' rather than the element class, causing ArrayUtils._copy_list_grow1 to allocate an array with the wrong component type and later the cast check comparing inferred_type==object to behave incorrectly. Also, inferred_type is set to object when both array and element are None, but the condition uses 'inferred_type == object' comparing a type object to the built-in 'object' which may be correct but the earlier mistake of type(array) is the root cause. The problematic lines are 332-335 (inferred_type assignment) and 343-347 (the cast check and exception). Use the element's type (type(element)) or inspect array's element type, not type(array).
