Remove the incorrect strict type check that raises TypeError when inferred_type is object. Instead, compute inferred element type properly: when array is not None, infer element type from first non-None element if possible; otherwise fall back to object. Then only raise TypeError if expected_type is provided and inferred_type is not object and not subclass of expected_type (i.e., incompatible). This avoids rejecting appending to a generic object list into a specific expected_type list.
