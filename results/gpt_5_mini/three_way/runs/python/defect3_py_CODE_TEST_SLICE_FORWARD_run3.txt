The code fails because the loop that advances pos after a successful translation uses char_count(code_point_at(code_units, pos)), which calls code_point_at with the current pos that may point to a high surrogate. char_count expects a codepoint integer but is passed a codepoint computed from possibly a surrogate pair; moreover, calling code_point_at does not advance the index but returns the full codepoint, and then char_count on that codepoint returns 2 for supplementary characters. The for-loop then runs 'consumed' times and each iteration does pos += 2, advancing pos twice per surrogate pair (total +4) when consumed==1 for a surrogate pair, skipping characters and causing IndexError. Specifically: in translate(), lines around the for pt in range(consumed): pos += char_count(code_point_at(code_units, pos)) (the loop after the comment) wrongly multiplies surrogate advancement. The bug occurs when translate_codepoint returned 1 (consumed==1) for a surrogate pair at pos; code_point_at(pos) returns a supplementary codepoint (lines 38-44) and char_count(...) returns 2, so the loop increments pos by 2 once, which is correct; but because the loop repeats 'consumed' times (which already counts codepoints consumed), if consumed represents code units rather than codepoints or is misinterpreted, the loop may increment too much. Root cause: mismatch between what consumed represents (number of code units consumed) and how pos is advanced (using code_point_at+char_count which double-counts surrogate length). The failing case in test_translate_surrogate_pair_with_suffix shows this: after processing the surrogate pair, pos is moved past the following 'A', leading to IndexError or missing characters. Fix: advance pos by consumed if consumed is in code units, or if consumed is number of codepoints advance by 1, or simply set pos += consumed when consumed is code unit count. Alternatively, replace the for-loop with pos += sum(char_count(code_point_at(code_units, i)) for i in range(consumed_start, consumed_end)) or better: have translate_codepoint return the number of UTF-16 code units consumed and then do pos += consumed. Also ensure code_point_at is only called when index is valid. Location: translate() function, lines handling 'consumed' and the for-loop, and code_point_at implementation (lines 38-44) which returns full codepoint for surrogate pairs.
