Fix loop index advancement: when translate_codepoint returns 0 we compute codepoint and append its UTF-16 code units; advance pos by the number of UTF-16 code units (char_count). When translate_codepoint returns >0 it's the number of codepoints consumed; we must advance pos by the total UTF-16 code units corresponding to those codepoints. So compute for consumed codepoints the sum of char_count(code_point_at(...)) iterating consumed times, incrementing pos accordingly. Also ensure we don't call code_point_at with index out of range by checking pos < length in loop.
