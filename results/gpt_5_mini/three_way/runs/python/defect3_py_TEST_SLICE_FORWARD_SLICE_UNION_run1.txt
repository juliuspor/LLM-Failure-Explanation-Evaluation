The code fails because to_utf16_code_units/code_point_at uses 'high' variable (code_units[index]) and returns 'high' when no valid surrogate pair is found; but 'high' is a 16-bit code unit value, not a Unicode code point for BMP characters when returned directly. However the real failure in tests is IndexError when index advances incorrectly: code_point_at checks index+1 < len(code_units) and uses code_units list derived from to_utf16_code_units; the translator advances index by 1 even when encountering a surrogate pair, causing subsequent call to code_point_at with index pointing to the low surrogate, which is in range but not a high surrogate; because index advanced incorrectly, later code may call code_point_at with index equal to len(code_units) (past end) leading to IndexError at line 35. In short: the translator's loop increments index by 1 for surrogate pairs instead of 2, so when a surrogate pair is at the end or followed by a char, the next index can exceed length causing IndexError at line 35 in code_point_at.
