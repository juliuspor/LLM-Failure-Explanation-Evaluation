Simplify advancing pos after a translator consumes codepoints. When translate_codepoint returns consumed N (number of codepoints), we should advance pos by the number of UTF-16 code units representing those N codepoints. The previous implementation tried to increment per codepoint calling code_point_at each time, which can call code_point_at at pos==length and raise IndexError. Fixed by iterating consumed times and advancing pos by char_count(code_point_at(code_units, pos)) but only if pos < length; safer: compute advance_units = 0; for i in range(consumed): advance_units += char_count(code_point_at(code_units, pos + advance_units)); then pos += advance_units. Alternatively simpler: count units by iterating code_units index: since consumed is count of codepoints, not code units, loop consumed times and increment pos by 1 or 2 depending on whether current unit is high surrogate followed by low surrogate. Implemented: for _ in range(consumed): if 0xD800 <= code_units[pos] <= 0xDBFF and pos+1<length and 0xDC00<=code_units[pos+1]<=0xDFFF: pos+=2 else: pos+=1. This avoids calling code_point_at out of range.
