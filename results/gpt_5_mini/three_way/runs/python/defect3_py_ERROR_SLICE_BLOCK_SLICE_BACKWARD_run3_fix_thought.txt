The IndexError arises because translate_codepoint returns 2 to indicate two codepoints consumed for a surrogate pair, but caller loop advances pos by char_count of code_point_at at the current pos repeatedly; moreover if translate_codepoint consumes a surrogate pair it should return the number of UTF-16 code units consumed, not codepoints. Fix: change contract so translate_codepoint returns number of code units consumed. Update translate loop to advance pos by returned consumed when consumed>0. Keep existing behavior for consumed==0. This avoids calling code_point_at with out-of-range index.
