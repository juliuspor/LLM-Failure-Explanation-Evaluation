The code rejects locale strings where the country part is empty (e.g. "fr__POSIX"). In to_locale, after checking length and initial lowercase letters, it requires locale_str[2] == '_' and then insists characters locale_str[3] and [4] are uppercase A-Z. For inputs like "fr__POSIX", locale_str[3] is '_' not an uppercase letter, so the check on lines assigning ch3/ch4 (lines ~227-230) fails and raises ValueError. Root cause: the implementation does not allow an empty country element (two consecutive underscores) which is valid in formats like language__variant. To fix: allow locale_str[3] to be '_' and handle empty country by using "" for country, or adjust the length/branching logic to accept length >= 6 with '_' at pos2 and treat an empty country when pos3 == '_'.
