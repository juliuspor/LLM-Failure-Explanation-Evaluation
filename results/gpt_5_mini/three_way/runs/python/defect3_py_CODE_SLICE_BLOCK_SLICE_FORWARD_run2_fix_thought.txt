The bug is advancing pos incorrectly after a translator consumes codepoints. Rather than repeatedly calling code_point_at which can raise IndexError when pos reaches length, advance pos by the number of UTF-16 code units consumed. Use pos += consumed. If consumed represents codepoints (not code units) but translators in this code return counts in code units (CsvEscaper returns 2 for surrogate pair), so pos += consumed is correct.
