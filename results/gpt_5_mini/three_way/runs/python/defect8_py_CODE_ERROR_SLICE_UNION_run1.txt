The code fails because it rejects valid locale strings that have an empty (zero-length) country code, like "fr__POSIX". The root cause is the function does not allow an empty country subtag (two consecutive underscores) while still allowing a variant after a second underscore; it should accept an empty country (i.e., locale pattern language + "__" + variant) and treat locale_str[3:5] as possibly empty rather than strictly two uppercase letters.
