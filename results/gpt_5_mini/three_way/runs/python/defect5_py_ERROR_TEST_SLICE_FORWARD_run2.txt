The failure is caused by creating or returning a list/array whose runtime element type is Object[] when the caller expects a more specific element type (e.g., String[]). Specifically, ArrayUtils.add constructs a new array using a generic or raw array creation path (such as new Object[length] or list(obj_array).toArray()) and returns it without matching the expected_type parameter or preserving the input array's element type. When the test calls ArrayUtils.add(None, None, expected_type=str) the implementation returns an Object[] (list of Python objects mapped to Java Object[] in the bridge), and the test harness attempts to cast that Object[] to String[], causing ClassCastException: [Ljava.lang.Object; cannot be cast to [Ljava.lang.String;. Root causes: (1) when both array and element are None, the code falls back to creating an Object[] instead of creating an array of the expected_type; (2) when adding to an existing array, the new array is created with element type Object rather than the original array's component type. Where it occurs: inside ArrayUtils.add (and potentially ArrayUtils.add_at_index), in the branch that handles null or mixed types and in the array-copy/creation lines that call toArray or Arrays.copyOf without specifying the component type. Steps to fix: - If expected_type is provided and array is None, create a new array of that component type with length+1 (e.g., (T[]) Array.newInstance(expected_type, newLength) in Java). - If array is non-null, determine component type via array.getClass().getComponentType() and create the new array using Arrays.copyOf(original, newLength, componentType). - Avoid using toArray() without a typed target array; instead use toArray((T[]) Array.newInstance(componentType, size)). - When element is null and expected_type is not provided, choose the existing array's component type if array is non-null; if both are null and expected_type missing, default to Object[] but do not later cast to String[]. These changes ensure returned arrays have the correct runtime component type and prevent ClassCastException.
