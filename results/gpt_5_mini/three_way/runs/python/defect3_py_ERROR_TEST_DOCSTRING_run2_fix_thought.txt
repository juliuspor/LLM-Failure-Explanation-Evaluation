Advance pos by the number of UTF-16 code units consumed rather than by 1. Use char_count on the consumed codepoint(s) or compute units from translate_codepoint’s consumed return. When consumed>0, increment pos by the sum of char_count for each codepoint consumed by repeatedly calling code_point_at at current pos before advancing. This avoids treating low surrogates as standalone. Also ensure when consumed==0 we still advance by the correct number of code units for the single codepoint appended.
