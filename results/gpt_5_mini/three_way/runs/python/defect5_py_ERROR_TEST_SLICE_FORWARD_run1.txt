The code fails because ArrayUtils.add constructs or returns a plain Object[] (Python: list of generic objects) when adding to a null array or when element types are mixed, and the caller expects a String[] (i.e. list[str]). In the Java-backed test harness this causes a ClassCastException: an Object[] cannot be cast to String[] when the test calls ArrayUtils.add(None, None, expected_type=str) or otherwise assumes a String[] result. Root cause: type inference/array creation logic in ArrayUtils.add uses Object.class (or no component type) when input array is null or when element is None, so the new array is created with component type Object instead of the requested expected_type (or the element's runtime type). Location: ArrayUtils.add handling of (array == null) or when all elements are null — code paths that call Array.newInstance(Object.class, ...) or new Object[n] produce Object[]. When caller performs a cast to String[] (or test expects list[str]), the JVM throws ClassCastException: [Ljava.lang.Object; cannot be cast to [Ljava.lang.String;. How it happened step-by-step: test calls ArrayUtils.add(None, None, expected_type=str) -> implementation sees null array and null element, chooses default component type Object (or derives from element but element is null) -> creates Object[] with one slot and returns it -> caller tries to treat it as String[] -> ClassCastException. Fixes (concrete steps): 1) When an expected_type parameter is provided, use it as the component type when creating a new array even if input array and element are null. 2) When array is null and expected_type is not provided, if element is non-null use element.getClass() as component type; if element is null, default to Object but allow caller to specify expected_type to avoid cast failure. 3) In Java code paths, replace new Object[...] or Array.newInstance(Object.class, ...) with Array.newInstance(componentType, ...), where componentType is determined as above. 4) Add unit tests for add(null, null, expected_type=String.class) and for sequences of adds mixing nulls that ensure resulting array has correct component type. Lines to inspect: ArrayUtils.add method beginning where it checks for null array and chooses componentType (look for code branching array == null or element == null).
