When translate_codepoint consumes N codepoints, we must advance pos by the number of UTF-16 code units those codepoints occupy. The previous code attempted to recompute sizes using code_point_at repeatedly, which can read past end for a trailing high surrogate. Fix: when consumed>0, advance pos by summing char_count of each codepoint starting at current pos but ensuring safe access: if a high surrogate is at pos and no low surrogate follows, treat it as one code unit. Also simpler: advance pos by consumed in codepoints converted to code units by inspecting code_units: iterate up to consumed times, for each increment add 2 if surrogate pair present and next in range and is low surrogate, else add 1. This avoids calling code_point_at beyond bounds.
