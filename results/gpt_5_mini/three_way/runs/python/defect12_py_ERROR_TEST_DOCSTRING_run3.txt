IndexError occurs because the code attempts to access text[upper] or another character index when upper (15) is beyond the length of text (10). Specifically, in WordUtils.abbreviate the logic searches for a space between indices 'lower' and 'upper' and uses text[i] or text[upper] without checking whether upper is -1 (no limit) or greater than len(text)-1. In the failing test, lower=15 > len(text), so any loop or direct indexing that starts at lower or uses upper as an index triggers 'String index out of range: 15'. The root cause: lack of bounds checks on 'lower' and 'upper' before using them as string indices. Fix: at function start, handle None/empty, then clamp upper to len(text)-1 when upper!=-1, and if lower >= len(text) return text (possibly with append_to_end only if abbreviation needed). Also ensure loops use min(upper, len(text)-1) and do not index at positions >= len(text).
