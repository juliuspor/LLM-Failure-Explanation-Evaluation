When both array and element are None but an expected_type is provided, we should not treat inferred_type==object as an error. Instead derive inferred_type from expected_type when array is None and element is None and expected_type is provided. Adjust logic: if array is not None -> inferred_type = type(array); elif element is not None -> inferred_type = type(element); elif expected_type is not None -> inferred_type = expected_type; else inferred_type = object. Remove previous exception condition or keep but it will now not trigger for expected_type provided. This allows add(None,None,expected_type=...) to return [None] without raising.
