When a translator consumes codepoints, we must advance pos by the number of UTF-16 code units those consumed codepoints occupy. The previous loop called code_point_at repeatedly and could read past the end or double-count surrogate pairs. Fix: compute units_consumed by summing char_count(code_point_at(code_units, pos)) once per consumed codepoint, but guarding index bounds; replace the existing loop with a for loop that checks pos < length before calling code_point_at, and increments pos by char_count for each consumed codepoint. This avoids IndexError and incorrect skipping.
