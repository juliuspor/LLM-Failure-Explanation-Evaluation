The code incorrectly mutates minutes_offset when computing total minutes. At line: if hours_in_minutes < 0: minutes_offset = hours_in_minutes - minutes_offset else: minutes_offset = hours_in_minutes + minutes_offset This reassigns minutes_offset (the input minutes component) to be total minutes, but for negative hours offsets it subtracts minutes instead of adding the negative minutes, producing wrong sign. Example: hours_offset=-1, minutes_offset=30 -> hours_in_minutes=-60, branch sets minutes_offset = -60 - 30 = -90 (should be -30). This leads to an incorrect total minutes and therefore wrong millisecond offset, and can cause overflow or wrong zone. Also the variable name reuse reduces clarity and causes the arithmetic bug; the subtraction should be hours_in_minutes - minutes_offset only when minutes_offset should be subtracted from negative hours, but correct logic is to compute total_minutes = hours_in_minutes - minutes_offset if hours_offset < 0 else hours_in_minutes + minutes_offset, or better use sign = -1 if hours_offset<0 else 1; total_minutes = hours_in_minutes + sign * minutes_offset. The error occurs in the try block when computing minutes_offset and then passed to FieldUtils.safe_multiply at the offset assignment; fix by using a new variable total_minutes and not overwriting minutes_offset.
