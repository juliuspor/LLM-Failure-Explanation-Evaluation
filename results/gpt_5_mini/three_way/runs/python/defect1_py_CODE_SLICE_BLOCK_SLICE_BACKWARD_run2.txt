The code fails because when hours_offset is negative the code subtracts minutes_offset from hours_in_minutes and assigns that back to minutes_offset, but minutes_offset has been validated to be between 0 and 59, so subtracting increases magnitude and can make minutes_offset negative beyond expected, leading to incorrect sign handling. Specifically: lines ~370-379, variable minutes_offset is reused to hold total minutes instead of using a separate variable. For negative hours_offset the line minutes_offset = hours_in_minutes - minutes_offset computes e.g. hours_in_minutes = -60, minutes_offset=30 => result -90 which is correct total minutes but later FieldUtils.safe_multiply is called; however the main bug is hours sign logic should combine as hours_in_minutes - minutes_offset for negative offsets but the code's validation forbids negative minutes_offset earlier meaning ambiguity; also possible overflow leading to ArithmeticError caught and rethrown as ValueError.
