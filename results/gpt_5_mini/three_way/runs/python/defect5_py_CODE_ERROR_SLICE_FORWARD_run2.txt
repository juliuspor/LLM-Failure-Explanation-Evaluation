The code fails because inferred_type is set to type(array) instead of the element type: type(array) returns the list type (e.g. builtins.list) not the component type, so inferred_type becomes object when array is None or becomes list when array provided. ArrayUtils._copy_list_grow1 likely creates a new list with component type inferred_type; when inferred_type is object and expected_type is str, the check raises TypeError. The root cause: using type(array) rather than inspecting elements or expected_type; also the equality check uses inferred_type == object which is true when both are the builtin 'object' type, but inferred_type may be list leading to wrong behavior. Also when expected_type is str the error message constructs Java-style array class name with expected_type.__name__ ('str') producing '[Ljava.lang.str;' which is incorrect.
