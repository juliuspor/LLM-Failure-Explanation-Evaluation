The root cause is that RandomStringUtils.random forwards start and end to _random_internal without validating that end > start. When end <= start, _next_int_java is called with bound = end - start <= 0, raising ValueError. Fix by validating the range up front and raising a clearer ValueError when end <= start. Implement RandomStringUtils.random to check that if end != 0 or start != 0 then end > start; otherwise allow defaults. Return _random_internal as before.
