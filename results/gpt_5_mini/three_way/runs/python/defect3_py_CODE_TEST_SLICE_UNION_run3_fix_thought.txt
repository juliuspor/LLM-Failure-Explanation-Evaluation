Avoid re-reading codepoint at pos which may be out-of-range. Use UTF-16 code unit increments: when a translator returns consumed N codepoints, advance pos by the total number of UTF-16 code units those N codepoints occupied. Do this by repeatedly checking the code unit at pos and if it's a high surrogate and next exists and is low surrogate, advance by 2, else advance by 1. This prevents calling code_point_at out of range.
