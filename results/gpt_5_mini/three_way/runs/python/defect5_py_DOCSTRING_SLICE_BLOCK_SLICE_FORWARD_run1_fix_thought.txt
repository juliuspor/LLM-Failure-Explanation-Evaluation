Relax the overly-strict TypeError: only raise when expected_type is provided and we can conclude a real mismatch. If array was None or elements gave no specific component type (inferred_type is object) but expected_type provided, do not treat that as cast failure. Instead, only raise when inferred_type is not object and expected_type differs (real mismatch). Also handle when array present: infer element type by scanning elements to find a concrete type, not type(array). Implement inference: if array is not None, iterate elements to find first non-None element type; if none found, inferred_type = object. Then only raise if inferred_type is not object and expected_type is not object and inferred_type != expected_type.
