Fix the advancement after a translator consumes code points by advancing pos in terms of UTF-16 code units per consumed code point without re-calling code_point_at on a mutated pos. For each consumed code point, inspect the code unit at the current pos: if it's a high surrogate and next is low surrogate then advance by 2, else advance by 1. This avoids invalid code_point_at calls and correctly moves past surrogate pairs.
