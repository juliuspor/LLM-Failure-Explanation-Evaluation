The bug is the advancement of pos after a translator consumes codepoints. Instead of looping and using char_count(code_point_at(...)) which can read beyond the end, simply increment pos by the number of UTF-16 code units consumed. Since 'consumed' is defined as number of codepoints consumed, but translators return counts in codepoints; however in this implementation translators should return number of codepoints consumed. To advance over UTF-16 code units, we should for each consumed codepoint compute char_count at the starting pos, but do it without re-reading after incrementing in loop: iterate consumed times and add char_count(code_point_at(code_units, pos)) but ensure code_point_at is safe; simpler: keep a local variable for advance = 0; for _ in range(consumed): advance += char_count(code_point_at(code_units, pos + advance)); then pos += advance. This avoids index errors.
