The code fails because minutes_offset is reused as both input minutes and later as total minutes; when hours_offset is negative the calculation at line 383 (minutes_offset = hours_in_minutes - minutes_offset) subtracts a positive minutes_offset from a negative hours_in_minutes, producing an incorrect sign. More importantly, the code mutates minutes_offset then passes it to FieldUtils.safe_multiply which expects a total minutes value; but because minutes_offset variable may start negative or positive this leads to wrong results and potential integer overflow. Also the initial check for hours_offset==0 and minutes_offset==0 returns cls.UTC but doesn't validate combined sign consistency (e.g., negative hours with positive minutes). The root cause is incorrect handling of sign when combining hours and minutes: the code should compute total_minutes = hours_offset*60 + (hours_offset<0 ? -minutes_offset : minutes_offset) instead of reusing minutes_offset and subtracting in the wrong direction. The exception occurs at the multiplication step when the combined minutes exceed safe range, leading to ArithmeticError and ValueError("Offset is too large").
