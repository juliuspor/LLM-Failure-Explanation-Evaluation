The code fails because it reassigns the input parameter minutes_offset to hold a total minutes value, then later uses FieldUtils.safe_multiply with that value. If hours_in_minutes is negative, the code does: minutes_offset = hours_in_minutes - minutes_offset which subtracts minutes_offset from hours_in_minutes (wrong sign). The root cause: variable name reuse and sign error. Specifically, at lines 383 and 385 the code modifies minutes_offset instead of computing a new total_minutes variable. When hours_in_minutes < 0 it should subtract the minutes_offset magnitude (i.e., total_minutes = hours_in_minutes - minutes_offset is correct only if minutes_offset is positive and hours_in_minutes already negative representing hours*60, but the intended logic is to add the sign of hours to minutes). The real bug: for negative hour offsets the minutes are applied with the wrong sign, producing incorrect total minutes or double-negation. This leads to wrong offsets like -1 hour 30 minutes becoming -90 or -30 incorrectly. Also mutating the parameter loses the original minutes range check and can produce values outside safe range, causing FieldUtils.safe_multiply to throw ArithmeticError and then ValueError("Offset is too large"). Fix: compute total_minutes = hours_in_minutes + ( -minutes_offset if hours_offset<0 else minutes_offset ) or better: total_minutes = hours_in_minutes + (minutes_offset if hours_offset>=0 else -minutes_offset). Do not overwrite minutes_offset; use a new variable and keep checks. Replace lines 383/385 with total_minutes calculation and then multiply that.
