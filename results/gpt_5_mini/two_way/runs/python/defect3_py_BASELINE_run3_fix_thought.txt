The bug is that when a translator consumes codepoints, pos is incremented inside the loop and code_point_at is called with pos that may reach length, causing IndexError. Fix by using a temporary index to calculate how many UTF-16 code units to advance: set temp = pos; for pt in range(consumed): cp = code_point_at(code_units, temp); temp += char_count(cp); pos = temp. This avoids calling code_point_at with pos past the end.
