The code fails because the loop advancing pos after a translator consumes codepoints increments pos incorrectly, causing pos to advance past the end of code_units and triggering an index error when code_point_at(code_units, pos) is called. Specifically: in the while loop, after consumed = self.translate_codepoint(...), when consumed > 0 the code enters the for pt in range(consumed) loop. Inside that loop it calls code_point_at(code_units, pos) and then char_count(...) and increments pos by that value. However, code_point_at expects pos to be a valid index into code_units. On the first iteration of the for loop pos is valid, but the loop uses pos (which is updated each iteration) to read successive code points; if consumed is the number of code points consumed by translate_codepoint, the loop should increment pos by the number of code units that made up each consumed codepoint (e.g., 1 for BMP, 2 for surrogate pairs). The bug is that char_count is being called with code_point_at(code_units, pos) which itself can raise IndexError when pos equals length (past the last index), because the loop iterates consumed times even when translate_codepoint already advanced over the end or when consumed does not match remaining code units. In short: the logic assumes it is safe to call code_point_at(code_units, pos) consumed times, but when consumed is incorrect or when pos is at the last index and a surrogate pair is expected, code_point_at reads beyond the available indices and throws "String index out of range: 2". The failing line is the call to code_point_at(code_units, pos) inside the for pt in range(consumed) loop; cause: incorrect pos management and not validating that pos < length before calling code_point_at. Fixes: 1) Change the inner loop to advance pos by the number of code units consumed by translate_codepoint directly, e.g. pos += consumed (if consumed already counts code units) or have translate_codepoint return the number of code units consumed rather than codepoints. 2) Or, before calling code_point_at, check if pos >= length and break or raise a clear error. 3) Alternatively, rewrite translate_codepoint contract so it returns the number of code units consumed and update pos accordingly without calling code_point_at in the caller.
