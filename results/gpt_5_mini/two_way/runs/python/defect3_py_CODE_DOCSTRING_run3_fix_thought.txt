When consumed>0, need to advance pos by the total UTF-16 code units corresponding to the consumed code points. Compute total_units by iterating consumed times reading codepoint at a local cursor (not modifying pos) and summing char_count; then add total_units to pos. This avoids re-reading at mutated pos and prevents out-of-range calls. Also handle safety: if remaining code units run out while summing, break to avoid IndexError.
