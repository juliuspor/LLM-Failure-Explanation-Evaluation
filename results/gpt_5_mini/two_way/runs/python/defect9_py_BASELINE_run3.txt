The code fails because it calls len() on None when computing 'greater' in the buffer-size guess loop. Specifically at slice lines 264–266: for i in range(search_length): greater = len(replacement_list[i]) - len(search_list[i]). If either replacement_list[i] or search_list[i] is None, len() raises TypeError. Earlier the code skips None/empty entries when searching for matches (lines ~243 and ~285), but it does not skip them in this sizing loop, so None values in replacement_list or search_list cause the crash.
