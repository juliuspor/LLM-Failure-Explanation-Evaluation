Bug: translate() increments pos incorrectly after a translator consumes codepoints. It loops consumed times but within loop calls code_point_at with same pos, causing repeated same value and wrong increment when consumed>1. Fix by advancing pos based on codepoints consumed: for i in range(consumed): get cp = code_point_at(code_units, pos); pos += char_count(cp). Alternatively, compute pos += sum of char_count for each consumed codepoint. Implement loop that processes each consumed codepoint properly.
