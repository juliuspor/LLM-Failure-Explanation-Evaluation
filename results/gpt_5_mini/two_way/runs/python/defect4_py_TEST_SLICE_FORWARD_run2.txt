Specifically, in the loop that computes max_middle_index the code uses 'if middle > max_middle:' but then sets 'max_index = i' (or uses max_middle_index in the comparison but assigns to max_index); likewise min/max start/end index code sometimes compares using one index variable and assigns a different one. The root cause is a variable-name bug: using max_index instead of max_middle_index (and min_index vs min_middle_index) when updating. Fix by consistently using the dedicated index variables (max_middle_index/min_middle_index/max_start_index/etc.) both for comparisons and assignments, and ensure the variables are initialized from the corresponding values (e.g. set max_middle = middle of index 0 and max_middle_index = 0).
