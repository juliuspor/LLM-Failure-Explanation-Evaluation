The bug arises from using inferred_type = type(array) which for a list gives 'list' not element type; then expected_type check mistakenly raises when inferred_type==object only. Fix by inferring element type by examining existing elements: if array provided and non-empty, infer element_type = type(array[0]) if all elements have same type else object. When array None, infer from element. Use element_type variable for expected_type check: if expected_type provided and element_type == object and expected_type != object then raise. Also no need to pass new_list_element_type to _copy_list_grow1 beyond signature; keep behavior but use element_type for comparison.
