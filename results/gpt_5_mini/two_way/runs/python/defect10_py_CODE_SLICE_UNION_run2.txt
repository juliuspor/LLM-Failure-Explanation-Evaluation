The code fails because NumberUtils._is_all_zeros is called with mant or exp being None in some branches, causing a TypeError. Specifically, at lines where all_zeros is computed: first occurrence around the block handling non-digit last_char (after "numeric = val[:-1]") uses mant and exp even though earlier when dec is None or exp is None they can be set to None. For example, if the input ends with a suffix like 'F' and the string has no decimal point, dec is None and exp may be None; then NumberUtils._is_all_zeros(mant) is fine but NumberUtils._is_all_zeros(exp) will be called with exp==None leading to a failure. Also slice helper char_at is shown in SLICE_UNION: its implementation raises IndexError when index out of range; calling char_at(numeric,0) without checking numeric may raise IndexError for empty numeric, causing the ValueError path to be bypassed and an exception to be raised instead. Root causes: calling functions expecting strings with None values (mant/exp) and not guarding empty numeric before char_at. Location: computation of all_zeros and use of char_at in the branch handling last_char in ("l","L","f","F","d","D").
