The code fails because minutes_offset is reassigned incorrectly when adjusting for negative hours, causing an incorrect calculation of the total offset. Specifically, at line 383 the code does: minutes_offset = hours_in_minutes - minutes_offset which subtracts the minute component from the converted hours (hours_in_minutes = hours_offset * 60), then at line 385 it does minutes_offset = hours_in_minutes + minutes_offset which adds them back in a way that double-applies the minute value or uses the wrong sign. This leads to an incorrect minutes_offset passed into FieldUtils.safe_multiply at line 386 and ultimately an offset value that can exceed supported range, triggering the ValueError at line 388. The root cause is mixing up variable names and signs when composing hours and minutes into a single minute total; the fix is to compute total_minutes = hours_in_minutes + signed_minutes (apply the sign of hours_offset to minutes_offset once) and then use total_minutes when converting to milliseconds and validating range.
