The bug is due to advancing pos using code_point_at which can raise IndexError when at end. Simpler fix: when consumed > 0, advance pos by the number of UTF-16 code units consumed, not by re-evaluating code_point_at. Assume translate_codepoint returns number of code units consumed (as implemented in CsvEscaper). So replace the for-loop with pos += consumed. This avoids out-of-bounds access.
