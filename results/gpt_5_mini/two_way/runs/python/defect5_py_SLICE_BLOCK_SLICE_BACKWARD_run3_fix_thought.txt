The bug arises because inferred_type defaults to object when neither array nor element provide a specific type, causing a spurious TypeError when expected_type is provided. Fix by improving type inference: if array is not None, infer element type from its elements (first non-None element). If array is None, but element is not None, use type(element). Only treat inferred_type as object if truly no type info exists. Also change the TypeError check to compare element type (not list type) to expected_type: only raise if inferred element type is object and expected_type is more specific and there's no element type information. This avoids false positives and allows adding None elements. Implement by inspecting array contents for first non-None element.
