The code fails because substring_java is called with an end index greater than the string length in the branch that handles 'no space found'. Specifically, when upper == -1 or upper > len(text) the code sets upper = len(text). However earlier the function checks if upper < lower then upper = lower. In the test case lower=15 and upper=20 for a 10-char string: after setting upper = len(text)=10, the next check upper < lower (10 < 15) sets upper = lower (15). That leaves upper=15 while text length is 10. Later substring_java(text, 0, upper) is called with end=15, causing the IndexError because end > len(text). The problematic lines are the order of adjustments (the block setting upper = len(text) should occur after ensuring upper >= lower or the lower bound should be adjusted differently). Also index_of(..., lower) uses lower unchanged (15) but the main crash is substring_java called with end beyond len(text) in the 'no space found' or other branches.
