The bug is in advancing pos after a translator consumed codepoints. translate_codepoint returns the number of codepoints consumed; we should advance pos by the number of UTF-16 code units that those codepoints occupy. To avoid calling code_point_at which can raise IndexError when pos is at end, compute code units advanced by summing char_count for each codepoint starting from current pos but only while pos < length. Simpler: iterate consumed times, but before calling code_point_at check pos < length then get cp and advance by char_count(cp); if pos >= length break. This prevents IndexError and ensures pos moves correctly.
