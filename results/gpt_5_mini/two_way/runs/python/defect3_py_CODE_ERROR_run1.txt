The code fails due to an IndexError caused by advancing pos incorrectly in the loop that handles 'consumed' code units. Inside the for pt in range(consumed): loop, it calls code_point_at(code_units, pos) then char_count(...) and increments pos by that. But code_point_at expects a valid index; after the first increment pos may go past length because consumed already indicated how many code units were consumed by translate_codepoint. The loop should instead advance pos by 1 for each consumed UTF-16 code unit, or better, advance pos by the number of code units that make up the consumed codepoint (which is either 1 or 2) using the same value consumed, not by re-calling code_point_at with the updated pos. In particular, if consumed > 1 or surrogate pairs exist, the inner call to code_point_at(code_units, pos) can be called with pos == length (out of range), producing "String index out of range: 2". The root cause is mixing two different meanings: 'consumed' is already the number of UTF-16 code units handled, but the loop advances pos by char_count(code_point_at(...)) which can attempt to read beyond the end. Fixes: replace the for-loop body with pos += 1 (advance per consumed unit), or compute the codepoint once before the loop and use its char_count, or simply do pos += consumed. Also add bounds checks before calling code_point_at.
