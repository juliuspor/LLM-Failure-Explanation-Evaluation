The code fails because minutes_offset variable is reused and mutated when combining hours and minutes: in the block starting at line where hours_in_minutes is computed, code assigns minutes_offset = hours_in_minutes - minutes_offset for negative hours, or + minutes_offset otherwise. This overwrites the original minutes_offset and leads to incorrect sign handling and potential integer overflow before multiplying by MILLIS_PER_MINUTE. Also, the condition hours_offset == 0 and minutes_offset == 0 returns cls.UTC early, which is fine. The real bug is on lines after 378: using minutes_offset to store total minutes causes wrong results when hours_offset is negative because it subtracts minutes_offset instead of adding signed minutes; for example hours_offset=-1, minutes_offset=30 becomes hours_in_minutes=-60 then minutes_offset = -60 - 30 = -90 (should be -30). This yields incorrect offset and may trigger overflow in FieldUtils.safe_multiply.
