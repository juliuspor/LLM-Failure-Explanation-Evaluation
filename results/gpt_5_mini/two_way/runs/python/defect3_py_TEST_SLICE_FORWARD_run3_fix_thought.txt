Change translate to advance pos according to number of UTF-16 code units consumed by translate_codepoint. When translate_codepoint returns n>0, it indicates number of code points consumed; we must sum the corresponding UTF-16 code unit counts by calling code_point_at and char_count, but be careful: code_point_at may currently return raw surrogate values; to avoid relying on it, instead inspect code_units at pos directly: if it's a high surrogate and next is low, treat as one code point consuming 2 units; otherwise one unit.
