The tests fail because StringUtils.replace_each raises a TypeError when a replacement list contains None. Specifically, in the function it attempts to call len() on a value that can be None (the code uses len(replace) or len(replacements[i]) without checking for None). This occurs when replace_each iterates replacement strings from the 'replacements' array and accesses their length to adjust offsets or build the result; if replacements contains None (e.g., replacements[0] is None), len(None) raises TypeError: object of type 'NoneType' has no len(). The failure happens inside replace_each when handling inputs like replacements = [None] or ['c', None] (see tests test_null_safety_inside_arrays_single and test_null_safety_inside_arrays_mixed). To fix: before calling len() or treating a replacement as a string, check for None and treat it as an empty string (or skip replacing) according to desired semantics. Concretely, in replace_each, replace occurrences where code does rep_len = len(replacements[i]) with rep = replacements[i]; if rep is None: rep_len = 0; rep = '' (and use rep in concatenation). Also ensure any indexing like replacements[i] is checked for None before calling methods like replace or iterating characters. This avoids calling len(None) and matches the intended null-safety in the tests.
