Fixed a bug in translate loop: when a translator consumes N codepoints it should advance pos by the number of UTF-16 code units corresponding to the consumed codepoints, not repeatedly call char_count on potentially advancing pos. The original loop advanced pos inside for-loop incorrectly using code_point_at with changing pos; simplified to iterate consumed times, getting codepoint at current pos then incrementing pos by char_count. Also handle case consumed < 0 by raising ValueError. Preserved rest of behavior.
