The bug is in the loop that advances the position after translation. When a translator consumes codepoints, the code incorrectly uses `Character.codePointAt(input, pos)` (which always returns the codepoint at the current `pos`) inside the loop, causing incorrect position advancement, especially for multi-char codepoints or multiple codepoints. Fixed by advancing `pos` by the total `consumed` chars directly, matching the fallback case.
