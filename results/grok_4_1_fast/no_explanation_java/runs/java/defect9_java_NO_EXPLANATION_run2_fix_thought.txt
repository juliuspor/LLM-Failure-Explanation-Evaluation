The `replaceEach` method has a critical bug in the repeated replacement logic. When `repeat=true`, it recursively calls itself with `timeToLive - 1`, but `timeToLive` is initialized to `searchList.length`, which can lead to infinite recursion or stack overflow if replacements create new matches. The fix ensures proper termination by checking `timeToLive <= 0` early and only recursing when `timeToLive > 0`. Also added null safety for arrays and fixed the recursive call condition.
