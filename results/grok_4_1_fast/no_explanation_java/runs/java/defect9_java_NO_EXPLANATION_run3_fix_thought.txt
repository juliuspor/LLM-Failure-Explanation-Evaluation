The `replaceEach` method has a critical bug in the repeated replacement logic. When `repeat=true`, it calls itself recursively with `timeToLive - 1`, but `timeToLive` is initialized to `searchList.length`, which can lead to infinite recursion if replacements create new matches. The fix ensures proper termination by checking `timeToLive <= 0` at the start and only recursing when `timeToLive > 0`. Also added null safety for arrays and fixed minor logic issues.
