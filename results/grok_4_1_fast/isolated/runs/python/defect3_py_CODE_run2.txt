The code fails in the loop after `consumed = self.translate_codepoint(code_units, pos, out)` when `consumed > 0`. The for loop `for pt in range(consumed):` repeatedly calls `code_point_at(code_units, pos)` using the same `pos` value each time because `pos` is not updated inside the loop body before the next iteration. On the second iteration (pt=1), `pos` still points to the start of the already-processed surrogate pair, causing `code_point_at` to return an invalid high surrogate codepoint (0xD800-0xDBFF). Then `char_count(invalid_high_surrogate)` likely returns 0, so `pos += 0` does not advance `pos`. This creates an infinite loop at the surrogate pair position, as `pos` never increases enough to exit `while pos < length`.
