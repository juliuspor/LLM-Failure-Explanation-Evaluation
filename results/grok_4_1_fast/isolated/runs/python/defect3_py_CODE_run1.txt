The code fails in the loop after `consumed = self.translate_codepoint(code_units, pos, out)` when `consumed > 0`. The for loop `for pt in range(consumed):` incorrectly calls `code_point_at(code_units, pos)` inside the loop body each time. Since `pos` is not updated inside the loop, `code_point_at` always returns the same codepoint. The loop then repeatedly calls `char_count` on this unchanged codepoint `consumed` times, adding the same `char_count` value to `pos` multiple times. This causes `pos` to overshoot the correct position by `(char_count - 1) * consumed`. For example, if `consumed == 2` and `char_count == 2` (a surrogate pair), `pos` advances by 4 instead of the correct 2, skipping code units and corrupting the translation.
