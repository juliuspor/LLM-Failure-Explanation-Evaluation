The bug diagnosis is incorrect - code_point_at does not initialize an empty code_units list. The real bug is in CharSequenceTranslator.translate: after a translator consumes codepoints, the loop incorrectly uses char_count(code_point_at(code_units, pos)) which re-looks up the codepoint at the current pos (already consumed), instead of using the consumed count directly. Fix by advancing pos by consumed code units.
