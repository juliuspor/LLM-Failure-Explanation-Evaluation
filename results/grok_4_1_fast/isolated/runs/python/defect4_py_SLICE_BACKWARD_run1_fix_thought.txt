The bug is in the _update_bounds method where index updates for min/max bounds are incorrectly placed outside the if/else blocks. For the first item (when index == -1), the code skips the comparison and always sets the index, which is correct. However, for subsequent items, if the new value doesn't beat the current min/max, the index isn't updated to the new index (it should remain the current min/max index). The fix moves all index assignments inside the proper if/else branches and fixes a variable reference bug in _max_middle_index calculation.
