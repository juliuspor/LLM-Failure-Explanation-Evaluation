The code fails with 'IndexError: String index out of range: 2' in the test case with input "\U0001F630A" (emoji followed by 'A'). Here's what happens:

1. `to_utf16_code_units(input_seq)` creates `[55357, 56896, 65]` (3 code units: high surrogate, low surrogate, 'A').
2. At `pos=0`, `self.translate_codepoint(code_units, 0, out)` returns `consumed=2` (handles the surrogate pair).
3. The loop then executes `for pt in range(2):` (pt=0, pt=1).
4. For `pt=0`: `code_point_at(code_units, pos)` where `pos=0` → returns codepoint for surrogates.
5. `char_count()` for a surrogate pair returns 2, so `pos += 2` → `pos=2`.
6. For `pt=1`: `code_point_at(code_units, 2)` → `index=2` equals `len(code_units)=3`, but `code_point_at` checks `index >= len(code_units)` and raises IndexError.

Root cause: After `translate_codepoint` consumes 2 code units, the loop incorrectly calls `code_point_at(code_units, pos)` *twice* (once per consumed unit), but `pos` is still at the start of the surrogate pair on the second iteration, causing an out-of-bounds access at index 2.
