The bug occurs because `text.find(search_list[i], start)` searches the original full `text` from `start`, including already-processed prefix areas, instead of only the unprocessed suffix `text[start:]`. Fix by changing all `text.find(..., start)` calls inside the main loop to `text.find(..., start)` on the remaining suffix by slicing `text[start:]` and adjusting indices back by `start`.
