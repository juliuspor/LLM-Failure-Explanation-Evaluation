The code fails because the clamping logic doesn't work correctly when using custom bounds. In test_custom_bounds(), GrayPaintScale(10.0, 20.0) sets self._lower_bound=10.0 and self._upper_bound=20.0. For value=5.0:

1. v = max(5.0, 10.0) = 10.0
2. v = min(10.0, 20.0) = 10.0 (clamping happens but v is unused)
3. g = int((5.0 - 10.0) / (20.0 - 10.0) * 255.0) = int((-5.0 / 10.0) * 255.0) = int(-127.5) = -128
4. if g < 0 triggers ValueError

The formula on line 74 uses the original unclamped 'value' instead of the clamped 'v', so g becomes negative for values below lower_bound. The test expects (0,0,0) but gets the exception.
