The code fails with a syntax error on this line:

    check_val = (mantissa // 10) * (10 ** (exp + 1))

The problem occurs because `mantissa` is an integer (from `value = int(x)` and `mantissa = value`), but the comment above the function mentions preventing misparsing of `x- -4`. If `x` is negative (like x = -100), then `value = -100` and `mantissa = -100`. When the while loop runs, `mantissa % 10 == 0` is true (-100 % 10 == 0), so it enters the loop. Then `mantissa // 10` becomes -10. Later, when this code is output as a string in a larger expression like `x- -10`, Python parses it as `x--10` (post-decrement, invalid syntax) instead of `x - -10`. The space-adding logic at the top only handles the immediate previous character being '-', not this deferred negative mantissa case.
