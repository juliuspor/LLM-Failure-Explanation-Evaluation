The test `test_null_safety_inside_arrays_mixed` fails because `StringUtils.replace_each("aba", ["a", "b"], ["c", None])` returns "cba" instead of the expected "cbc". The replacement arrays have different types: search array contains strings ["a", "b"], replacement array contains a string "c" and Python's `None`. When the code processes the input "aba", it replaces the first 'a' with "c" (position 0), then the second 'a' with `None` (position 2, since replacement index matches search index 0), resulting in "cba". However, the test expects "cbc", which assumes the second 'a' should be replaced by the replacement at search index 1 ("null"/None), incorrectly applying replacement index 1 to search index 0's matches. This reveals a logic error in the implementation's replacement index selection for repeated search terms with mixed-type arrays.
