The bug is in the translate method's advancement logic after translate_codepoint returns 0. When code_point_at detects a surrogate pair, it returns the full codepoint but the advancement uses len(to_chars(codepoint)) which is always 2 for surrogate pairs, but pos += len(c) only advances by 1 initially. The fix is to use char_count(codepoint) to correctly advance by 2 code units for surrogate pairs.
