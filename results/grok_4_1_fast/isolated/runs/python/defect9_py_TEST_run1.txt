The test `test_null_safety_inside_arrays_mixed` fails because `StringUtils.replace_each("aba", ["a", "b"], ["c", None])` returns "cba" instead of the expected "cbc". The replacement logic incorrectly applies the null replacement for 'b' as an empty string (producing "ca" after first replacing 'a' to 'c'), but then fails to preserve the original 'b' character when the replacement array has None at index 1. This violates the expected null-safety behavior where null replacements should leave the matching substring unchanged, as shown in the Java equivalent and the single-null test case.
