The `to_locale` method fails to handle the `language__variant` format (double underscore with empty country). After validating the 2-letter language, it must check for a double underscore (`__`) at position 2-3 to set empty country and parse the variant. Added explicit handling for this case before the single underscore check, ensuring `fr__POSIX` correctly parses as `Locale('fr', '', 'POSIX')`.
