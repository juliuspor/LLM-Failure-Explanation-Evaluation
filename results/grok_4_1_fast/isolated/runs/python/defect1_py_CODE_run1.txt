The code fails due to incorrect logic in calculating the total minutes offset on these lines:

if hours_in_minutes < 0:
    minutes_offset = hours_in_minutes - minutes_offset
else:
    minutes_offset = hours_in_minutes + minutes_offset

When hours_offset is negative (hours_in_minutes < 0), it subtracts minutes_offset from hours_in_minutes. This is wrong because minutes_offset should always be added to represent the total offset. For example, with hours_offset = -1 and minutes_offset = 30, hours_in_minutes = -60, so minutes_offset becomes -60 - 30 = -90 minutes. But -1:30 should be -90 minutes total, not -60 + 30 = -30 minutes as intended.

This error causes wrong timezone offsets for negative hour offsets with positive minutes. The subtraction should be replaced with addition in both cases: minutes_offset = hours_in_minutes + minutes_offset.
