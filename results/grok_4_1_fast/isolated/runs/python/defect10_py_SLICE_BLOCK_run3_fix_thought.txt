The bug is in the `char_at` helper function used by `create_number`. The condition `index < 0 or index >= len(s)` triggers unnecessary bool-to-int coercion when the first operand is True, violating PEP 8 and causing potential issues. Fix by using parentheses: `(index < 0) or (index >= len(s))` to prevent coercion. Only `char_at` needs fixing; `create_number` remains unchanged.
