The code fails in the test `test_abbreviate_lower_beyond_length` when calling `WordUtils.abbreviate("0123456789", 15, 20, None)`. The input string has length 10, but `lower=15` and `upper=20` are both beyond the string length.

The function does not clamp `lower` to the string length, so `StringUtils.index_of(text, " ", lower)` is called with `lower=15`. This searches for a space starting at index 15, which exceeds `len(text)=10`.

The `substring_java` function used internally raises `IndexError: String index out of range: 15` because its validation checks `start > len(s)` (line 32 in slice validation), where `start=15` and `len(s)=10`.

This occurs in the first `if index == -1` block, where `substring_java(text, 0, upper)` is called with `upper=20` (after adjustment to `len(text)=10`), but the error happens earlier during the `index_of` call.
