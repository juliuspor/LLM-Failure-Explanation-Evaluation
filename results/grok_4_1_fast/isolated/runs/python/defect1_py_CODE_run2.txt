The code fails due to incorrect logic in calculating the total minutes offset on these lines:

hours_in_minutes = hours_offset * 60
if hours_in_minutes < 0:
    minutes_offset = hours_in_minutes - minutes_offset
else:
    minutes_offset = hours_in_minutes + minutes_offset

When hours_offset is negative (e.g., -1), hours_in_minutes becomes negative (e.g., -60). The if condition triggers, and minutes_offset gets reassigned to hours_in_minutes - minutes_offset (e.g., -60 - 30 = -90). Later, offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE) uses this negative value (-90 minutes), creating a negative total offset in milliseconds. However, the method cls.for_offset_millis(offset) likely expects only positive offsets and fails with such negative values, even though they represent valid time zones like UTC-01:30.

The root cause is the subtraction (-) instead of addition (+) in the negative hours case, which should always be minutes_offset = hours_in_minutes + minutes_offset to correctly compute the total minutes.
