The bug occurs because `substring_java` throws IndexError when `upper > len(text)`, even though the code checks `upper == -1 or upper > len(text)` and sets `upper = len(text)`. However, if `len(text) < lower`, the subsequent `upper = lower` can make `upper > len(text)` again, causing the error in `substring_java(text, 0, upper)`. Fix by adding an additional bounds check to ensure `upper <= len(text)` after all adjustments.
