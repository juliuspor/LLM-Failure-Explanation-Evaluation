The test `test_negative_zero` in `TestCodeConsumerAddNumber` fails because `self.cc.add_number(-0.0)` calls the `add_number` method in the real `CodeConsumer` class, which does not preserve the negative zero sign. Python's `-0.0` prints as `'0.0'` (positive zero) when formatted by `str(-0.0)` or `repr(-0.0)`, but the test expects `'-0.0'` to match the Java behavior that preserves the sign. The `MockCodeConsumer` only overrides `append`, `get_code`, and `reset` - it inherits `add_number` from the buggy parent `CodeConsumer`, so `add_number(-0.0)` appends `'0.0'` instead of `'-0.0'`, causing `self.assertEqual(result, "-0.0")` to fail with the actual result `'0.0'`.
