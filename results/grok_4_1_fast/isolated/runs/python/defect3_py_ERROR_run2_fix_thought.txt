The bug is in the translate method's loop when a translator consumes characters. After translate_codepoint returns consumed > 0, the code incorrectly iterates 'consumed' times and calls code_point_at(pos) each time without advancing pos, causing repeated calls at the same position. For surrogate pairs (consumed=2), it calls code_point_at(pos) twice, and the second call uses pos+char_count which exceeds bounds when near end. Fixed by advancing pos by total code units consumed once after translator handling.
