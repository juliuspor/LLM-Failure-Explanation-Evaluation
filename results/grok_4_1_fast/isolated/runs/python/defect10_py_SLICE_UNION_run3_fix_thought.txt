The bug is in the `char_at` helper function used by `create_number`. Python's `or` short-circuits and returns the first truthy value. When `index < 0` is True (negative index is truthy), it returns the negative index instead of evaluating the full condition, causing invalid IndexError raises for valid negative indices like -1. Fix by using `if not (0 <= index < len(s)):` to properly check bounds without short-circuit issues.
