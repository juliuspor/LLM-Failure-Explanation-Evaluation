The bug occurs because after `translate_codepoint()` consumes a surrogate pair and returns `consumed == 2`, the for loop calls `code_point_at(code_units, pos)` with the current `pos` (still pointing to the start of the surrogate pair), but `code_point_at()` tries to access `index + 1` which is out of bounds when `pos + 1 == length`. Fixed by advancing `pos` by `consumed` code units first, then using the new `pos` in `code_point_at()` to get the correct codepoint for `char_count()`.
