The test `test_null_safety_inside_arrays_mixed` fails because `StringUtils.replace_each("aba", ["a", "b"], ["c", None])` returns "cca" instead of the expected "cbc". The replacement logic incorrectly applies the first replacement rule ("a" -> "c") to both the first 'a' and the second 'a' position, rather than applying the second rule ("b" -> null, which should remove it). This happens because the implementation does not properly match the longest applicable search string at each position and fails to handle null replacements correctly, causing subsequent matches to use the wrong replacement index.
