The code fails in the loop after `consumed = self.translate_codepoint(code_units, pos, out)` when `consumed > 0`. The for loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` incorrectly advances `pos` multiple times using the `char_count` of codepoints starting at the same `pos` position each iteration. This happens because `pos` is not updated inside the loop body before the next `code_point_at(code_units, pos)` call. As a result, it repeatedly processes the same position, causing either an infinite loop (if `char_count` returns 0) or incorrect position advancement (skipping or duplicating code units). The bug occurs in the last for loop at the end of the while loop.
