The diagnosis misidentifies the bug location. The real issue is in `CharSequenceTranslator.translate`: after `translate_codepoint` returns `consumed > 0`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` incorrectly advances `pos` multiple times from the same starting position, causing double-skipping of surrogate pairs. Fixed by advancing `pos` by the exact number of code units consumed using `char_count` calls with proper positioning.
