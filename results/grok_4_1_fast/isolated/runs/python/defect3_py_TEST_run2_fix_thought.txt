The bug is in the translate method's fallback path. When no translator consumes a codepoint, it uses to_chars(codepoint) and advances pos by len(c), but len(c) is the number of code units for that codepoint, not the number of codepoints consumed (always 1). This causes incorrect advancement for surrogate pairs (advances by 2 instead of 1 codepoint). Fixed by always advancing by 1 codepoint using char_count(code_point_at(code_units, pos)) in the fallback path, matching the translator contract.
