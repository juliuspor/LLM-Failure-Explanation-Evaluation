The bug diagnosis incorrectly identifies the issue in to_utf16_code_units(). The real bug is in CharSequenceTranslator.translate(): after converting input_seq to code_units with to_utf16_code_units(), the loop uses char_count(code_point_at(code_units, pos)) which calls code_point_at(pos) with the wrong index. It should increment pos by consumed code units (not codepoints) since translate_codepoint returns code units consumed.
