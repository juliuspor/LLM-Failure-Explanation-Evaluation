The bug occurs because `substring_java` throws IndexError when `upper > len(text)`, even after the code attempts to clamp `upper`. The clamping logic has a flaw: when `upper == -1`, it sets `upper = len(text)`, but later checks `if upper != len(text)` which fails. Additionally, `StringUtils.index_of` can return -1, leading to invalid `substring_java` calls. Fix by using safe Python slicing with min(upper, len(text)) and handling edge cases properly.
