The bug is in the scientific notation logic for integers >= 100. The while loop condition `check_val == value` fails because `check_val` uses the original `value` while `mantissa` is being divided, causing premature loop exit. Fixed by restructuring the loop to properly compute `mantissa` and `exp` by repeated division by 10, and using 'e' instead of 'E' for standard Python float notation. Also simplified the condition to handle only finite integers correctly.
