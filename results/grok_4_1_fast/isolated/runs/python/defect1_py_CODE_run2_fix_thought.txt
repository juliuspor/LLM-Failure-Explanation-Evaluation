The bug is in the offset calculation logic. When hours_offset is negative, the code incorrectly subtracts minutes_offset instead of adding it, causing incorrect total minutes (e.g., -60 - 30 = -90 instead of -60 + 30 = -90). The fix is to always add the minutes component to the hours-in-minutes value regardless of sign, as total_minutes = hours_offset * 60 + minutes_offset.
