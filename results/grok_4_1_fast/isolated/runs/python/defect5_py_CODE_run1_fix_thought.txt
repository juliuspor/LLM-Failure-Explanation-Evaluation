The bug occurs because `inferred_type = type(array)` sets the type to `list` instead of the element type `T`. The `_copy_list_grow1` method expects an element type, not the list type. Fixed by inferring the element type correctly: use `expected_type` if provided, otherwise infer from first non-None element in `array` or from `element`. This ensures proper type tracking and triggers the cast error when `inferred_type == object`.
