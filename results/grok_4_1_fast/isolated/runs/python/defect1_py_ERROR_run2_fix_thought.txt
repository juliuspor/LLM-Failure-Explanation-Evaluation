The bug is in the minutes calculation logic. When hours_offset is negative (e.g., -1), hours_in_minutes becomes negative (-60), and the code incorrectly sets minutes_offset = hours_in_minutes - minutes_offset (-60 - 15 = -75), resulting in negative minutes which fails the range check. The fix is to always compute total minutes as hours_offset * 60 + minutes_offset, then convert to millis, eliminating the flawed conditional logic.
