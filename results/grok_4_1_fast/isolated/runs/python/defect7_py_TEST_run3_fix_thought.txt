The `to_class` method fails when encountering `None` elements in the input array because it calls `type(array[i])` without checking if `array[i]` is `None`. In Python, `type(None)` returns `type(None)` which is correct, but the unnecessary `array[i].upper()` call before it causes an `AttributeError` since `None` has no `upper()` method. Remove the unused `upper()` call and add explicit `None` handling for clarity and safety.
