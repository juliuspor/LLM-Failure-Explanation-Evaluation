The code fails in the final for loop when `translate_codepoint()` consumes a surrogate pair (returning `consumed == 2`) for input "\U0001F630A". After processing the emoji at positions 0-1, `pos` starts at 2. The loop `for pt in range(consumed):` (range(2)) executes twice. First iteration: `code_point_at(code_units, 2)` succeeds ('A' at position 2), `char_count()` returns 1. Second iteration: `code_point_at(code_units, 2)` is called again (pos still 2), but then `pos += 1` makes `pos = 3`. On the next `code_point_at(code_units, 3)`, index 3 is out of range (length=3), causing IndexError: String index out of range: 3.

Root cause: The loop re-checks `code_point_at(code_units, pos)` each iteration but advances `pos` inside the loop using the result from the current position, causing pos to overshoot on the second iteration for consumed=2.
