The bug is in `substring_java` which throws IndexError when `end > len(s)`. In `abbreviate`, `upper` is capped at `len(text)` but `index` from `index_of` can return -1, causing `substring_java(text, 0, -1)` which fails the `end < 0` check. Fixed by handling `index == -1` case before calling `substring_java` and ensuring valid bounds.
