The bug occurs because when `chars=None` and `letters=False, numbers=False`, the code sets `start=0, end=127`, resulting in `gap=127`. However, `_next_int_java(rnd, 127)` can return 0-126, so `code_point = 0 + 0 = 0`, and `chr(0)` is generated. Later, when `chars` from `random_from_string(chars=None)` is passed to another `random`, it sets `end=len(chars)=0`, making `gap=0`, causing `_next_int_java(rnd, 0)` to raise ValueError. Fix by ensuring a positive gap by setting `start=32, end=127` (printable ASCII) when no specific constraints are given.
