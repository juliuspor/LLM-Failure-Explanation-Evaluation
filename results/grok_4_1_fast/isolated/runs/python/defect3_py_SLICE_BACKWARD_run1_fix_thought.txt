The bug diagnosis incorrectly identifies the problem. The actual issue is in the `translate` method's loop: after `translate_codepoint` consumes characters, `pos` is advanced using `char_count(code_point_at(code_units, pos))` starting from the already-consumed position, causing double-skipping and missing characters. Fix by advancing `pos` by the exact number of code units consumed (`consumed`), since `translate_codepoint` returns code units consumed.
