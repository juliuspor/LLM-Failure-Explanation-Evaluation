The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in the `translate` method of `CsvEscaper`. The input "\U0001F630A" (😰A) contains a surrogate pair (😰) followed by 'A'. When `translate_codepoint()` correctly consumes the 2-character surrogate pair using `to_utf16_code_units`, it advances the index `i` by 2 positions. However, the `translate` loop then increments `i` by 1 more (`i += 1`), causing `i` to become 3, which exceeds the string length of 3. On the next iteration, `code_point_at(input_seq, i)` receives an out-of-bounds index, raising IndexError. This bug occurs because the index advancement double-counts the characters consumed by surrogate pairs: `translate_codepoint()` already advances past them, but the loop adds an extra increment.
