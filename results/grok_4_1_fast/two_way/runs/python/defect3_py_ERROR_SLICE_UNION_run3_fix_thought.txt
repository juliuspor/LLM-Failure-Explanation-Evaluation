The bug diagnosis misidentifies the issue. The IndexError occurs in the translate method's loop when pos advances incorrectly after consuming surrogate pairs. When translate_codepoint returns 2 (for a surrogate pair), the loop incorrectly advances pos by 2 * char_count(code_point_at(pos)), which calls code_point_at(pos) with an index already past the pair, causing IndexError. Fix by advancing pos by consumed code units directly instead of using char_count.
