The bug occurs because `translate_codepoint` can return `consumed > 0` even when near the end of `code_units`, potentially leaving `pos` at or beyond `length`. The subsequent `code_point_at(code_units, pos)` call raises IndexError. Fix by adding a bounds check before calling `code_point_at` in the `consumed == 0` case, and advance `pos` by 1 if out of bounds to safely exit the loop.
