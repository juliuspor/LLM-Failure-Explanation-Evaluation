The bug occurs because the country code validation (A-Z check) runs for all lengths >2 without distinguishing empty country cases like 'fr__POSIX'. Added check after first underscore: if positions 3-4 are both '_', treat as empty country + variant (language__variant). Otherwise, validate as normal language_country[_variant]. This handles 'fr__POSIX' -> Locale('fr', '', 'POSIX') while preserving existing validation.
