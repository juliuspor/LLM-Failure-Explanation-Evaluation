The bug occurs because `create_number` calls `char_at(numeric, 0)` without checking if `numeric` is empty. When `val` ends with a type qualifier like 'L' and the preceding part is empty, `numeric = val[:-1]` becomes an empty string, causing `char_at('', 0)` to raise IndexError. Fix by adding an early check after computing `numeric` to raise ValueError if it's empty.
