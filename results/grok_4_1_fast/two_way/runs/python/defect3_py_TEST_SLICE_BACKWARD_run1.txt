The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `to_utf16_code_units` at line 35-36. When processing input_str = '\U0001F630A' (emoji 😰 followed by 'A'), the surrogate pair 😰 consumes 2 UTF-16 code units at indices 0-1. After handling the pair, the index variable advances to 2 (pointing to 'A'). However, the code incorrectly checks `if index < 0 or index >= len(code_units)` where code_units has length 3 (code units for 😰 and 'A'). Since index=2 is valid for accessing code_units[2], the check passes, but a later operation (likely code_units[index+1] or slicing) attempts to access beyond index 2, causing the IndexError 'String index out of range: {index}'. This reveals a bug in index advancement logic after surrogate pair processing in CsvEscaper.translate_string, which calls to_utf16_code_units.
