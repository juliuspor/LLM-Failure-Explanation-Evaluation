The code fails the test `test_locale_with_empty_country_variant` for input "fr__POSIX". After validating length (7 characters) and language chars "fr", it checks `locale_str[2] == '_'` (which passes). Then it extracts `ch3 = locale_str[3] = '_'` and `ch4 = locale_str[4] = 'P'`, and checks if `ch3` is uppercase A-Z. Since '_' (ASCII 95) is not between 'A' (65) and 'Z' (90), it raises ValueError "Invalid locale format: fr__POSIX". The code assumes positions 3-4 are always country code letters when length > 2, but doesn't handle the empty country case where position 3 is the second '_'.
