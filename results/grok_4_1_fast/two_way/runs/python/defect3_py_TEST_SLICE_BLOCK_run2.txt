The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError at line 36 in `code_point_at`: `raise IndexError(f"String index out of range: {index}")`. The input string "\U0001F630A" (😰A) contains a surrogate pair (2 UTF-16 code units) followed by 'A'. When `CharSequenceTranslator.translate` calls `code_point_at(index)` with `index=1` (the low surrogate), the check `if index < 0 or index >= len(code_units)` fails because `code_units` is the UTF-16 representation with length 3 ([55357, 56896, 65]). However, `code_point_at(1)` doesn't recognize this as part of a surrogate pair and attempts to access `code_units[1+1]` (index 2, which exists) but then the surrogate decoding logic produces an invalid code point, causing the method to return an error signal that triggers the bounds check to incorrectly raise IndexError as if index was out of range.
