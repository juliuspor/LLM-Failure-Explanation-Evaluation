The bug is in `code_point_at`: when a valid surrogate pair is detected, it returns the decoded codepoint but doesn't advance the index by 2. In `translate`, `pos += len(c)` where `len(c)==2` for surrogates fixes advancement, but `pos += char_count(code_point_at(...))` calls `code_point_at` which doesn't advance internally, causing infinite loop or incorrect advancement. Fix by making `code_point_at` return both codepoint and consumed code units count.
