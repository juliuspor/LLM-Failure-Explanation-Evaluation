The test suite fails because the GrayPaintScale class implementation in hit02_grayscale.py does not handle the custom constructor GrayPaintScale(10.0, 20.0) correctly. In test_custom_bounds(), gps = GrayPaintScale(10.0, 20.0) passes custom lower=10.0 and upper=20.0 bounds, but the class likely only uses the default bounds of 0.0-1.0. This causes gps.get_paint(15.0) to incorrectly treat 15.0 as above the default upper bound 1.0, returning (255,255,255) white instead of the expected middle gray (127,127,127). Similarly, gps.get_paint(5.0) and gps.get_paint(25.0) fail clamping relative to the custom bounds. The get_paint() method must use instance variables for lower/upper bounds set in the constructor, not hardcoded defaults.
