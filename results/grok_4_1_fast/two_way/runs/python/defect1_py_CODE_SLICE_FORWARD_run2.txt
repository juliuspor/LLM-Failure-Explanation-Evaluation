The code fails due to incorrect logic for handling negative hours_offset. On line 383, when hours_in_minutes is negative (hours_offset < 0), it sets minutes_offset = hours_in_minutes - minutes_offset. This subtracts the original positive minutes_offset (0-59) from a negative hours_in_minutes, making minutes_offset more negative than intended. For example, hours_offset=-1, minutes_offset=30 gives hours_in_minutes=-60, then minutes_offset=-60 - 30 = -90. Later on line 386, this negative minutes_offset is passed to FieldUtils.safe_multiply, which throws ArithmeticError for overflow. The except block on line 388 catches it and raises ValueError('Offset is too large'). The bug occurs because the code subtracts minutes_offset instead of adding it for negative offsets; it should always add the absolute contribution: minutes_offset = hours_in_minutes + minutes_offset.
