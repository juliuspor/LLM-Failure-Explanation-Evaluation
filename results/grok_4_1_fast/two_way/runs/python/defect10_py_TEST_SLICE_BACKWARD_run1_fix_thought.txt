The IndexError occurs because `char_at(numeric, 0)` is called on `numeric = 'bobL'[:-1] = 'bob'` when processing the 'L' suffix, but later logic expects `numeric` to be a valid signed number for `is_digits(numeric[1:])`. For invalid inputs like 'bobL', `numeric[1:]` is invalid. Fix by using safer string access with `numeric[0:1]` instead of `char_at`, preventing IndexError and allowing proper ValueError for malformed inputs.
