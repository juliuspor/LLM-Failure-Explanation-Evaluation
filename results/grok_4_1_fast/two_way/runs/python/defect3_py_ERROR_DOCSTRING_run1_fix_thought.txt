The bug is in the translate method's loop after translate_codepoint returns consumed > 0. It incorrectly advances pos by summing char_count(code_point_at(code_units, pos)) for each of the consumed codepoints, but pos isn't updated between calls, causing code_point_at to be called at the same index repeatedly and eventually exceed bounds. Fix by advancing pos by the total UTF-16 code units consumed (consumed * average char size isn't correct; instead calculate total units by summing char_count for each consumed codepoint starting from current pos, updating pos in the loop).
