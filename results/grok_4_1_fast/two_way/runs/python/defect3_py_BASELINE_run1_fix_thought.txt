The bug occurs because after `translate_codepoint` consumes a surrogate pair (returns 2), the for loop iterates twice calling `code_point_at(code_units, pos)` without advancing `pos` between iterations. The second call uses the same `pos` (pointing to the high surrogate), but `char_count` expects the position to advance past each codepoint. Fix by tracking the current position with a local index that advances by `char_count` for each consumed codepoint, preventing repeated calls at the same position and ensuring proper bounds.
