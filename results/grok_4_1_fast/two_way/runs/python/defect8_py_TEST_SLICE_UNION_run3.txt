The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The bug occurs at lines 227-230 in `to_locale`: when `length == 5` (for "fr__P"), it extracts `ch3 = locale_str[3] = '_'` and `ch4 = locale_str[4] = 'P'`. Line 229 then checks `if ch3 < 'A' or ch3 > 'Z'`, which fails because `'_' < 'A'` is true, raising `ValueError("Invalid locale format: fr__POSIX")`. The code never reaches line 233 to handle the empty country case (`fr__POSIX`) correctly, as the length-5 check incorrectly rejects the double underscore before checking for variant.
