The test `test_to_class_with_none_element` fails because the `ClassUtils.to_class` method does not handle `None` elements in the input array. On line 761 in the implementation (`for i in range(len(array)): _ = array[i].upper()`), the code calls `.upper()` on every element of the array, including `None`. When `array[i]` is `None` (the second element of `["Test", None, 99.0]`), `None.upper()` raises an `AttributeError: 'NoneType' object has no attribute 'upper'`. This crashes the loop before it can complete, so the method never returns the expected `[str, type(None), float]`.
