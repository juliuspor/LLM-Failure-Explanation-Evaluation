The core bug is that `no_more_matches_for_repl_index[i]` is never reset to False after being set to True. Once a search string has no matches from one start position, it stays permanently skipped even from later start positions where matches could exist. Fix by resetting the array to False at the start of each search iteration. Also remove the `replacement_list[i] is None` skip condition since null replacements should replace with empty string (handled correctly by appending None which becomes '').
