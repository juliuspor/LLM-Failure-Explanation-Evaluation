The IndexError occurs in the `char_at` function called within `create_number` when `numeric[:-1]` produces an empty string (len=0) and `char_at(numeric, 0)` is invoked for 'L' suffix validation. This happens with inputs like 'L' or '-L'. Fixed by adding early validation to reject invalid suffix cases (non-digit last_char with empty numeric part) before calling `char_at`.
