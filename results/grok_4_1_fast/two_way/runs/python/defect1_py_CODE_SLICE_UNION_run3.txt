The code fails due to integer overflow in the calculation of `minutes_offset` on lines 383 and 385. When `hours_offset` is -23, `hours_in_minutes` becomes -1380. On line 383, `minutes_offset = -1380 - minutes_offset` results in values like -1380 (if minutes_offset=0) or -1439 (if minutes_offset=59), which are large negative numbers. The `FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)` call on line 386 then detects this overflow and raises ArithmeticError, triggering the `raise ValueError("Offset is too large")` on line 388. This happens even for valid inputs like hours_offset=-23 and minutes_offset=0 because the intermediate `minutes_offset` becomes too large in magnitude before multiplication, despite the hours validation on line 376 allowing -23 <= hours_offset <= 23.
