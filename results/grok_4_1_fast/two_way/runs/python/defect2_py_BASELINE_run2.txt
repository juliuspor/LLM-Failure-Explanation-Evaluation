The code fails because the clamping logic doesn't work correctly when using custom bounds. The variable `v` is clamped with `max(value, self._lower_bound)` and `min(v, self._upper_bound)`, but then the calculation `g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)` still uses the original unclamped `value`, not the clamped `v`. In the test `test_custom_bounds`, when `gps = GrayPaintScale(10.0, 20.0)` and `value=5.0`, the formula computes `(5.0 - 10.0) / (20.0 - 10.0) * 255.0 = -127.5`, so `g = -128`. The check `if g < 0 or g > 255` on line 77 then raises ValueError, failing the test which expects `(0, 0, 0). The `v` clamping on lines 2-3 is computed but never used.
