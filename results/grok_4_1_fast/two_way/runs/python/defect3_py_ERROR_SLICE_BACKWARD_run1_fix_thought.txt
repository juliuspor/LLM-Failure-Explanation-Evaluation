The bug diagnosis incorrectly identifies the error location. The actual bug is in CharSequenceTranslator.translate: after translate_codepoint returns consumed > 0, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls code_point_at with the current pos (unchanged), which raises IndexError if pos >= len(code_units). Fix by advancing pos correctly using consumed code units, not codepoints.
