The bug occurs because `_random_internal` calls `_next_int_java(rnd, gap)` where `gap = end - start`. When `start=5, end=5`, `gap=0`, triggering `ValueError('bound must be positive')` in `_next_int_java`. The test expects an error message mentioning 'start' and 'end'. The fix adds validation in `random` to check if `start >= end` before calling `_random_internal`, raising a descriptive `ValueError` that satisfies the test assertions.
