The bug occurs because `char_at(numeric, 0)` is called on 'bob' (from 'bobL') when checking 'L' suffix, but the condition `NumberUtils.is_digits(numeric)` passes incorrectly since it only checks if all characters are digits without length validation. For non-numeric 'bob', it enters the branch and calls `char_at` with index 0 on empty or invalid string contexts, causing IndexError. Fix by adding early validation that the entire numeric part contains only valid digits before calling `char_at`.
