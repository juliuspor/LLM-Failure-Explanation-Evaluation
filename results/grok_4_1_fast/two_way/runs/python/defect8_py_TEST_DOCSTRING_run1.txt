The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` does not return the expected `Locale("fr", "", "POSIX")`. The implementation likely splits the input string on single underscores only, treating "fr__POSIX" as two parts ("fr" and "__POSIX") instead of recognizing the double underscore pattern for empty country. This causes either a ValueError (invalid format) or an incorrect Locale object where the variant is not properly extracted as "POSIX" with empty country code. The Java equivalent handles "language__variant" explicitly, but the Python code lacks this empty country code parsing logic.
