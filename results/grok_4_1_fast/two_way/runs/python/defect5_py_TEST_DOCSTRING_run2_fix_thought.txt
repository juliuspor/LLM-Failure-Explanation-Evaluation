The bug occurs because when both `array` and `element` are None, `inferred_type` defaults to `object`. The type check only raises TypeError if `inferred_type == object` AND `expected_type != object`, but when `element=None` with `expected_type=str`, it should fail the Java-style cast simulation. Fix by checking if `expected_type` is provided and `element` doesn't match it (None != str), raising TypeError before returning the list.
