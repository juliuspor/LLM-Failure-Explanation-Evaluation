The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. At lines 229-230, the code checks `if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z'`, expecting uppercase letters for country code positions. For "fr__POSIX", `ch3` is the first `_` (ASCII 95) and `ch4` is the second `_` (ASCII 95), both outside 'A'-'Z' range, triggering the `ValueError(f"Invalid locale format: {locale_str}")`. The parser assumes positions 3-4 must always be uppercase letters for a valid country code but doesn't handle the empty country case with double underscore (`language__variant`) as required by the test.
