The bug is in the translate method's surrogate pair handling after a translator consumes characters. When a translator returns consumed > 0, the loop incorrectly advances pos by summing char_count(code_point_at(code_units, pos)) for each consumed codepoint, but pos is still the original position, causing code_point_at to read invalid surrogate pairs (like treating 'A' as a trailing low surrogate). Fix by advancing pos by the consumed count directly, as translators handle surrogate pairs correctly and return the exact number of code units consumed.
