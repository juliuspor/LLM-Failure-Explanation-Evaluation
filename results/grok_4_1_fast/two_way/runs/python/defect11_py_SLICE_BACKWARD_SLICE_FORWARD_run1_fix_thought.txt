The bug occurs because when `chars=None` and `letters=False, numbers=False` in `random_letters_numbers`, `_random_internal` sets `gap = 127 - 0 = 127`, but if `chars` is provided later in the call chain with `len(chars) == 0`, `gap = len(chars) - 0 = 0`, causing `_next_int_java(rnd, 0)` to raise ValueError. The fix adds validation in `random` to prevent zero-length effective ranges by raising a clear error when `end <= start`.
