The bug occurs because `translate_codepoint` implementations in subclasses (like CsvEscaper) access `code_units[index + 1]` without checking if `index + 1 < len(code_units)`, causing IndexError when near the end of input. The fix adds bounds checking in the main `translate` loop before calling `translate_codepoint`, skipping translation and copying the character as-is when insufficient code units remain for potential surrogate pairs.
