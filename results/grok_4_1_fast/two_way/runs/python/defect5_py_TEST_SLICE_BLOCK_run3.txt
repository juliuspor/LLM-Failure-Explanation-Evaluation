The test `test_add_null_to_null_with_expected_type` fails because ArrayUtils.add(None, None, expected_type=str) infers the array type as `object` when both elements are None, but the caller specifies `expected_type=str`. On lines 343-348 of the implementation, the code checks `if inferred_type == object and expected_type != object:` and raises a TypeError with the message mimicking Java's ClassCastException. This explicit check prevents returning a list that could be used as str[], causing the test to fail instead of passing with result == [None] as expected.
