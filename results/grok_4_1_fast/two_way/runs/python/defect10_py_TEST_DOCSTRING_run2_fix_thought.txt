The bug occurs because when `last_char` is not a digit (e.g., 'l', 'F', 'junk'), the code processes type qualifiers but falls through without raising ValueError for invalid suffixes like single 'l' or non-numeric 'junk'. After the non-digit block, it attempts integer parsing which fails silently, then float/double parsing which also fails, ultimately returning a BigDecimal or not raising. Fix by adding an early return False check after hex handling, and ensuring all invalid paths explicitly raise ValueError, preventing fall-through to numeric parsing for invalid inputs.
