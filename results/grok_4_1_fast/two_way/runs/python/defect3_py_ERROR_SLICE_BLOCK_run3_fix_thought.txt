The bug occurs because after `translate_codepoint` returns a non-zero `consumed` value, the loop incorrectly advances `pos` using `code_point_at(code_units, pos)` which can raise IndexError if `pos` is out of bounds. The fix replaces this with a simple `pos += consumed` since `translate_codepoint` returns the exact number of code units consumed, matching Java's char[] indexing behavior.
