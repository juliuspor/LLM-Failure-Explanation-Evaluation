The bug occurs because the interpolation computes a float value (e.g., 127.5) which is cast to int (127) for the return tuple, but the error check uses the original float 'g' before casting, causing ValueError when it exceeds 0-255 bounds due to floating-point precision. Fix by computing the clamped, scaled value as float first, then convert to int once for the return value, eliminating the erroneous range check on the intermediate float.
