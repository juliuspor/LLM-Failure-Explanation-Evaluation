The code fails the test `test_negative_hours_negative_minutes` for input (-2, -15). In the calculation block:

1. `hours_in_minutes = hours_offset * 60` sets `hours_in_minutes = -120`
2. Since `hours_in_minutes < 0`, it executes `minutes_offset = hours_in_minutes - minutes_offset` = `-120 - (-15)` = `-105`
3. `offset = FieldUtils.safe_multiply(-105, DateTimeConstants.MILLIS_PER_MINUTE)` = `-105 * 60000` = `-6,300,000`

This produces -01:45 (-105 minutes), but the test expects -02:15 (-135 minutes = -8100000 millis). The subtraction `hours_in_minutes - minutes_offset` incorrectly reduces the magnitude when both are negative, instead of adding them as |hours| + |minutes| for negative offsets.
