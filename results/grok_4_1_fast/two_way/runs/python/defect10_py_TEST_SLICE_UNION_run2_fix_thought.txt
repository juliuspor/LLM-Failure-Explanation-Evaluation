The bug occurs because `create_number('bobL')` passes the invalid numeric part 'bob' to `is_digits` after stripping the 'L' suffix, and `is_digits('bob')` returns False, but the code only raises ValueError if the numeric part is a valid signed digits string. For invalid strings like 'bob', it silently falls through without raising. Fix by adding an early validation check using `is_number` to ensure the entire string (before suffix) represents a valid number format, rejecting malformed inputs like 'bobL'.
