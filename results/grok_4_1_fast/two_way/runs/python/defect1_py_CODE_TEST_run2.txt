The code fails the test `test_negative_hours_negative_minutes` for input (-2, -15). The test expects offset -8100000 (-2h -15m), but the code produces +8100000 (+2h +15m). In the calculation block:

1. `hours_in_minutes = hours_offset * 60` sets `hours_in_minutes = -120` (correct)
2. Since `hours_in_minutes < 0`, it executes `minutes_offset = hours_in_minutes - minutes_offset`
3. `minutes_offset` starts as -15, so `-120 - (-15) = -120 + 15 = -105` (incorrect magnitude)
4. `offset = FieldUtils.safe_multiply(-105, DateTimeConstants.MILLIS_PER_MINUTE)` = -105 * 60000 = -6300000

However, the test passes because `get_id()` likely normalizes -01:45 to "-02:15" (equivalent offsets), but `get_offset(0)` returns -6300000 instead of expected -8100000, causing the assertion to fail with message "Offset should be sum of negative parts".

Root cause: The line `minutes_offset = hours_in_minutes - minutes_offset` incorrectly subtracts a negative `minutes_offset`, reducing the total magnitude instead of adding it.
