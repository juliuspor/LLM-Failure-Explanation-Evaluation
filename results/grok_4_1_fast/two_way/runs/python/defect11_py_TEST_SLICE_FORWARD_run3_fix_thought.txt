The bug occurs because when start == end (e.g., random(3,5,5,...)), gap becomes 0, and _next_int_java(rnd, 0) raises ValueError('bound must be positive') which doesn't mention 'start' and 'end'. The Java version raises IllegalArgumentException specifically mentioning 'start' and 'end' when no valid chars exist. Fix by adding explicit validation in random() before calling _random_internal.
