The bug is in the incorrect handling of negative hours_offset. When hours_in_minutes < 0, the code subtracts minutes_offset instead of adding it, causing incorrect total minutes (e.g., -60 - 30 = -90 instead of -60 + 30 = -30). The fix is to always add the absolute minutes_offset to hours_in_minutes regardless of sign, as the sign is already handled by hours_offset.
