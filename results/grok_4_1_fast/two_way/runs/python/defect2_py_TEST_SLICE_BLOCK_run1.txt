The test suite fails because the `GrayPaintScale.get_paint` method returns a tuple of three identical gray values (R, G, B) for grayscale colors, such as (127, 127, 127) for input 0.5. However, on line 78, the code incorrectly unpacks this tuple into separate Red, Green, and Blue variables and checks if `g` (the Green value) is outside 0-255, which triggers `ValueError(f"Color parameter outside of expected range: Red Green Blue")`. This happens for all test cases since every returned color has three components, causing the unpacking and validation to fail even for valid grayscales like (0,0,0), (127,127,127), and (255,255,255). The root cause is mismatched assumptions: `get_paint` outputs RGB tuples, but the validation code expects single-channel values.
