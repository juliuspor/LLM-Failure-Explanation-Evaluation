The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at`. When processing input '\U0001F630A' (😰A), the surrogate pair 😰 occupies code_units[0] and code_units[1]. After processing this pair at index=0, `code_point_at` returns successfully, but the translate loop advances index by only 1 (to index=1). The next call to `code_point_at(1)` sees code_units[1] (low surrogate 0xDF30) which is not a high surrogate (0xD800-0xDBFF), so line 44 returns just the low surrogate value. However, the translate logic then incorrectly advances index by 1 more (to index=2), which equals len(code_units). The subsequent call to `code_point_at(2)` on line 38 accesses code_units[2] which doesn't exist, causing IndexError. The root cause is the translate loop not advancing index by 2 after surrogate pairs are detected in `code_point_at`.
