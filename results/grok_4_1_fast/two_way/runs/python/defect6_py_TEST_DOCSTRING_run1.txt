The test `test_negative_zero` in `TestCodeConsumerAddNumber` fails because `CodeConsumer.add_number(-0.0)` outputs "0" instead of the expected "-0.0". Python's `str(-0.0)` and `repr(-0.0)` both return "0.0", losing the negative sign that must be preserved for JavaScript compatibility where -0.0 is distinct from +0.0. The `is_negative_zero` utility correctly detects negative zero (passing `TestNegativeZeroUtility`), but `add_number` doesn't use it to emit the "-0.0" literal. This matches the original Java bug where "var x=-0.0" printed as "var x=0".
