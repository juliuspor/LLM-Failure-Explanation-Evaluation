The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` does not return the expected `Locale("fr", "", "POSIX")`. The code at line 233 assumes a standard language_country format and returns `Locale(locale_str[0:2], locale_str[3:5])`, which extracts "fr" as language and "_P" as country (locale_str[3:5] = characters at positions 3-4). This creates `Locale("fr", "_P")` instead of recognizing the empty country (double underscore "fr__POSIX") and handling the variant "POSIX". The parsing logic never reaches lines 235-237 to check for the variant separator '_' at position 5, so it incorrectly treats the malformed "_P" as a country code.
