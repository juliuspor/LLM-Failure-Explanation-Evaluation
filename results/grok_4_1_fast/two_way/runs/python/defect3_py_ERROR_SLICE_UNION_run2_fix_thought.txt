The bug diagnosis is incorrect - the IndexError occurs in `code_point_at` when `index` equals the list length, which passes the bounds check but fails on `code_units[index]`. The real issue is in `translate`: after `consumed > 0`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls `code_point_at` with an invalid `pos` value equal to `length`, causing the error. Fix by using the pre-calculated `consumed` count directly: `pos += consumed`.
