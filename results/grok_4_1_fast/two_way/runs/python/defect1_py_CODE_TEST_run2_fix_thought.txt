The bug occurs when hours_offset is negative and minutes_offset is negative. The line `minutes_offset = hours_in_minutes - minutes_offset` incorrectly computes the magnitude because subtracting a negative minutes_offset reduces the total magnitude. Fixed by always computing total_minutes = |hours_offset| * 60 + |minutes_offset|, then applying the sign of hours_offset (or UTC if zero). This correctly handles all sign combinations.
