The infinite loop occurs when `translate_codepoint` returns 0 for the last single code unit (often a lone surrogate). `code_point_at` returns the raw code unit, `to_chars` produces an empty list for surrogate values (0xD800-0xDFFF), and `pos += 0` fails to advance. Fix by always advancing `pos` by 1 when no translation occurs, treating each code unit individually as fallback.
