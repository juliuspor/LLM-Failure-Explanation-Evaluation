The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. At lines 229-230, the code checks `if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z'`. For "fr__POSIX", `ch3` is the first '_' (ASCII 95) and `ch4` is the second '_', both of which fail the uppercase letter check (`'A'` to `'Z'`). This incorrectly assumes positions 3-4 must always be uppercase letters for a country code, but the language__variant format (empty country) has '_' characters there, causing the validation to reject valid input.
