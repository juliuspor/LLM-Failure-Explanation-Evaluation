The bug occurs because `translate_codepoint` implementations (like CsvEscaper) only consume surrogate pairs if starting at the high surrogate (index 0), returning 0 for low surrogates (index 1). This causes the main loop to call `code_point_at(1)`, which doesn't recognize lone low surrogates as valid and fails. The fix ensures `translate_codepoint` is only called at codepoint boundaries by tracking codepoint positions instead of code unit positions, consuming full codepoints each iteration.
