The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code assumes a 5-character string at lines 227-229 has an uppercase country code at positions 3-4 (ch3=locale_str[3], ch4=locale_str[4]). For "fr__POSIX", ch3='_' (ASCII 95) and ch4='P', so line 229's check `ch3 < 'A' or ch3 > 'Z'` fails since '_' < 'A', raising `ValueError("Invalid locale format: fr__POSIX")`. The code doesn't handle the language__variant format (double underscore with empty country) and incorrectly validates the separator '_' as if it were a country code letter.
