The code fails due to a logic error in calculating the total minutes offset on lines 383 and 385. When hours_offset is negative (hours_in_minutes < 0), the code sets minutes_offset = hours_in_minutes - minutes_offset (line 383). This subtracts the original positive minutes_offset (0-59) from the negative hours_in_minutes, making minutes_offset more negative than intended. For example, hours_offset=-1, minutes_offset=30 gives hours_in_minutes=-60, then minutes_offset=-60 - 30 = -90. Later, on line 386, this incorrect -90 is multiplied by MILLIS_PER_MINUTE to compute offset, which exceeds the valid timezone offset range and triggers ArithmeticError in FieldUtils.safe_multiply, caught on line 388. The correct calculation should be minutes_offset = hours_in_minutes + minutes_offset in both positive and negative cases to properly add the minutes component to the hours component.
