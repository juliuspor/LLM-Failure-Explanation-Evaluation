The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. At lines 229-230, the code checks `if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z'`. For "fr__POSIX", `ch3` is the first `_` (position 3) and `ch4` is the second `_` (position 4). Since `_` is not an uppercase letter ('A'-'Z'), the condition triggers and raises `ValueError("Invalid locale format: fr__POSIX")`. The code incorrectly assumes positions 3-4 must always be uppercase letters for a country code, but fails to handle the valid `language__variant` format where country is empty.
