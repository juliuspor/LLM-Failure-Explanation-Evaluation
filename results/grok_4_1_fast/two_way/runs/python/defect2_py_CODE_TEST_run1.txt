The code fails the test_custom_bounds.test case. After clamping `v` to bounds in lines 3-4, the calculation `g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)` on line 6 still uses the original unclamped `value`. For `GrayPaintScale(10.0, 20.0)` and `gps.get_paint(5.0)`, `value=5.0` is clamped to `v=10.0` but `g` computes `(5.0-10.0)/(20.0-10.0)*255.0 = -255`, so `g=-255`. The check `if g < 0` on line 9 raises ValueError, preventing return of expected (0,0,0). Similarly fails for values above upper bound.
