The bug occurs in `_random_internal` when `letters=True` or `numbers=True` with `start==0`. The code sets `gap = end - start` which can be 0, but `_next_int_java` rejects non-positive bounds. Fixed by ensuring a valid gap >=1 when letters/numbers are specified by setting appropriate start/end bounds for letters (97-123) and numbers (48-58), falling back to ASCII range otherwise.
