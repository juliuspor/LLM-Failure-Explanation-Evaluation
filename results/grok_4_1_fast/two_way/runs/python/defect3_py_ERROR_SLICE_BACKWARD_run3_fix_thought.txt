The bug diagnosis incorrectly identifies the error location. The actual issue is in `CharSequenceTranslator.translate`: after `translate_codepoint` returns `consumed > 0`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls `code_point_at(code_units, pos)` where `pos` is the current index, but `code_point_at` raises IndexError if `pos >= len(code_units)`. This happens when `translate_codepoint` consumes characters near the end. Fix by advancing `pos` by the correct number of code units directly using `consumed * char_count(code_point_at(code_units, index))` once before the loop, then replace the loop with `pos += consumed`.
