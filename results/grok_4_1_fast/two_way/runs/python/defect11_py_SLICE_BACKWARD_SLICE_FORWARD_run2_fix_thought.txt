The bug occurs because when `chars=None` and `letters=False, numbers=False` (default case), `gap = end - start = 127 - 0 = 127`, but `_next_int_java(rnd, gap)` receives `bound=127` which is positive. The diagnosis suggests `bound <= 0` is true, which can only happen if `gap <= 0`. This occurs when `end <= start`, specifically in `random_letters_numbers` when `letters=False, numbers=False` because `start=0, end=0` sets `gap=0`. Add explicit validation in `random` to raise a clear error before calling `_random_internal` when `start >= end`.
