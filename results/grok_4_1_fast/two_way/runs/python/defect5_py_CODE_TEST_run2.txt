The code fails the test `test_add_null_to_null_with_expected_type` because the type check logic doesn't catch the invalid case. When `array` is None and `element` is None, `inferred_type` is set to `object` on line `if array is not None: ... else: inferred_type = object`. Then `ArrayUtils._copy_list_grow1(None, object)` creates a list of `object` type. The check `if expected_type is not None: if inferred_type == object and expected_type != object:` on lines after doesn't raise TypeError because `expected_type=str` != `object` is True, but `inferred_type == object` uses Python's `==` which compares class identity, and `type([])` (list) != `object`, so the condition fails silently. The test expects `[None]` without error, but in a real Java-like system this would cause a ClassCastException when casting object[] to String[].
