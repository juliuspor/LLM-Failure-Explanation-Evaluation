The code fails in the final for loop when `translate_codepoint` consumes a surrogate pair (returns consumed=2). After `consumed = self.translate_codepoint(code_units, pos, out)`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` incorrectly advances `pos`.

For input "\U0001F630A" (😰A), `code_units` = [0xD83D, 0xDE30, 65] (length=3). At pos=0, `translate_codepoint` consumes the surrogate pair and returns 2. The loop runs twice:
1. First iteration: `code_point_at(code_units, 0)` succeeds, pos += 2 → pos=2
2. Second iteration: `code_point_at(code_units, 2)` → index=2 >= len=3 → IndexError: String index out of range: 2

The bug is the loop advances `pos` *consumed* times (2), but should advance only once by the total amount consumed (2). It double-advances past the end into the suffix 'A'.
