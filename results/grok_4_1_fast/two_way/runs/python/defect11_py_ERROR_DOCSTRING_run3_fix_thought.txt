The bug occurs in `_random_internal` when `letters` or `numbers` is True and `chars` is None. The code sets `start=ord(' ')` (32) and `end=ord('z')+1` (122+1=123), so `gap=123-32=91` which is positive and valid. However, the diagnosis indicates a 'bound must be positive' error with `start=0`. The issue is in `random_from_chars` where `start=0` and `end=len(chars)`; if `len(chars)==0` after validation, or if `chars` contains only surrogate/private surrogate characters that get rejected in the loop, `gap=0` causes `_next_int_java(rnd, 0)` to fail. Fix by ensuring `gap > 0` before generating characters, and handle empty effective character sets gracefully.
