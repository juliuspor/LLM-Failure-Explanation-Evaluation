The code fails because it searches for matches in the **original** `text` string using `text.find(search_list[i], start)`, instead of the **modified** text being built in `parts`. This causes incorrect replacements when later matches overlap with or are affected by earlier replacements.

**Example:**
text = "aaa"
search_list = ["aa", "a"]
replacement_list = ["x", "y"]

- First match: "aa" at index 0 → parts = ["", "x"] , start=2
- Second match: searches original text.find("a", 2) → finds "a" at index 2 → parts = ["", "x", "y", ""] → result="xy"
- **Wrong!** Should be "xa" (replace first "aa"→"x", then remaining "a"→"y", but code wrongly replaces original position 2).

**Where:** In the `while text_index != -1:` loop, the line `temp_index = text.find(search_list[i], start)` uses unmodified `text` instead of the current state after replacements.
