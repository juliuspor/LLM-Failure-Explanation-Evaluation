The IndexError occurs because substring_java raises an error when end > len(s), even though Python slicing handles it safely. In abbreviate, when upper == len(text), substring_java(text, 0, upper) calls s[0:len(s)] which is safe in Python but the bounds check end > len(s) incorrectly triggers the error since it uses > instead of >=. Fix by changing the condition to end >= len(s) + 1.
