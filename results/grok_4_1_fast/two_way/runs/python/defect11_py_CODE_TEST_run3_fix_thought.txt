The bug is in _random_internal where gap = end - start = 0 is not validated before calling _next_int_java(rnd, 0), which raises 'bound must be positive' instead of a custom error mentioning 'start' and 'end'. Add validation after computing gap to raise ValueError('Invalid range: start {} >= end {}') if gap <= 0, matching Java LANG-807 behavior. Fix applies to all calls including random(3,5,5,False,False).
