The code fails due to incorrect logic in calculating the total minutes offset on these lines:

hours_in_minutes = hours_offset * 60
if hours_in_minutes < 0:
    minutes_offset = hours_in_minutes - minutes_offset
else:
    minutes_offset = hours_in_minutes + minutes_offset

When hours_offset is negative (e.g., -1), hours_in_minutes becomes negative (e.g., -60). The if condition triggers, and minutes_offset (e.g., 30) is overwritten as hours_in_minutes - minutes_offset (-60 - 30 = -90). This produces wrong total minutes (-90 instead of correct -30). Later, offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE) uses the incorrect value, creating a wrong time zone offset. The subtraction should be addition in both cases: total_minutes = hours_offset * 60 + minutes_offset.
