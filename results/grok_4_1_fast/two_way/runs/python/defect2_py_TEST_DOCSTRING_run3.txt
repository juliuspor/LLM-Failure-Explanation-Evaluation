The test suite fails because the `GrayPaintScale.get_paint` method raises a `ValueError` when computed color components are exactly 127.5 (like for input 0.5), as this falls outside the integer range 0..255. The docstring explicitly states it raises `ValueError: If the computed color components fall outside 0..255`. In `test_mid_value_interpolation`, the calculation `(0.5 - 0.0) / (1.0 - 0.0) * 255.0` equals 127.5, which triggers the exception before returning the expected `(127, 127, 127)`. This violates the test expectation that values inside bounds are interpolated and clamped to valid integers without raising errors.
