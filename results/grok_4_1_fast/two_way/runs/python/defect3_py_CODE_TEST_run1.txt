The code fails in the final for loop of the translate method when self.translate_codepoint() consumes a surrogate pair (consumed > 0). After translate_codepoint() processes the surrogate pair, the loop 'for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))' incorrectly advances pos. For each consumed unit, it calls code_point_at(code_units, pos) which reads the codepoint starting at the current pos, then adds char_count (2 for surrogate pairs). This double-counts the advancement: translate_codepoint already moved past the surrogate pair, but the loop re-advances pos again by the surrogate pair length, causing pos to skip characters. In test_translate_surrogate_pair_with_suffix(), when translating '😰A' (emoji + 'A'), consumed=2 after the emoji, the loop skips the 'A', leading to IndexError when accessing beyond the string end.
