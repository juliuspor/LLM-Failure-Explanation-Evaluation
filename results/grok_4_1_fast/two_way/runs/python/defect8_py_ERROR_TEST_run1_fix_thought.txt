The bug occurs because `to_locale` doesn't handle the `language__variant` format (double underscore with empty country). After the first underscore at position 2, it expects uppercase letters at positions 3-4 for country code, but finds another underscore instead. Fix by checking for a second underscore at position 5 to detect empty country, then parse variant from position 6. Also adjust initial length check to allow length >=7 and fix variant extraction index.
