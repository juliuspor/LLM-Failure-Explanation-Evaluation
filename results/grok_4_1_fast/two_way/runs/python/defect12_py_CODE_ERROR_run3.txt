The code fails with 'IndexError: String index out of range: 15' in the `substring_java(text, 0, upper)` call when `upper` exceeds the length of `text`. This happens because the initial bounds check `if upper == -1 or upper > len(text): upper = len(text)` is incorrect—it sets `upper = len(text)` only when `upper > len(text)`, but Python slicing and substring functions require the end index to be <= string length. If `upper` is exactly `len(text)`, `substring_java(text, 0, upper)` attempts to access index `len(text)`, which is out of range. The error occurs in these lines:
- Line with `result = substring_java(text, 0, upper)` (first branch when no space found)
- Line with `return substring_java(text, 0, upper) + ...` (second branch when `index > upper`)
For example, if `len(text) = 15` and `upper = 15`, it tries to slice up to index 15, causing the IndexError.
