The code fails in the loop after `translate_codepoint` returns a positive `consumed` value. It enters a `for pt in range(consumed):` loop that repeatedly calls `code_point_at(code_units, pos)` and `char_count` on the same `pos` value without updating it inside the loop. This causes infinite re-processing of the same position. The comment mentions handling surrogate pairs, but `pos` is only incremented after the loop using an incorrect formula: `pos += char_count(code_point_at(code_units, pos))` which advances by only 1 or 2 code units instead of the full `consumed` amount that `translate_codepoint` already processed.
