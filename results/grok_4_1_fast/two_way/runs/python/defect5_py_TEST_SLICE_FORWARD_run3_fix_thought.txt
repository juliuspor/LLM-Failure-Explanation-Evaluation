The bug is caused by a mismatch between test expectations and the method signature. Most tests call add() with 2 arguments (array, element), but one test calls it with 3 arguments (array, None, expected_type=str). The current implementation requires expected_type to be optional with a default, but tests pass None explicitly as the element. Fix by making expected_type truly optional with default=None and adjusting type inference to handle cases where both array and element are None.
