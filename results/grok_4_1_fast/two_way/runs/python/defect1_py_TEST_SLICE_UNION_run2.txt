The test `test_mixed_signs_invalid` fails because the implementation at lines 383-385 incorrectly handles the case of negative hours with positive minutes. For `DateTimeZone.for_offset_hours_minutes(-2, 15)`, the code enters the `if hours_offset < 0:` block. It then sets `hours_in_minutes = 120` (from `abs(-2) * 60`) and executes `minutes_offset = hours_in_minutes - minutes_offset` (120 - 15 = 105). This makes `minutes_offset = 105`, which exceeds 59. The check at line 377 (`if minutes_offset < 0 or minutes_offset > 59`) then raises `ValueError("Minutes out of range: 105")`. However, the test expects this call to succeed and return a zone with ID "-02:15". The formula `hours_in_minutes - minutes_offset` for negative hours subtracts positive minutes instead of adding their magnitude to create the expected total offset of -2h15m (-8100000 millis).
