The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` does not return the expected `Locale("fr", "", "POSIX")`. The implementation likely splits the input string on single underscores only, treating "fr__POSIX" (with double underscore) as invalid format instead of recognizing it as language + empty country + variant. This causes a `ValueError` to be raised (or an incorrect `Locale` object returned) instead of properly parsing the empty country case, exactly matching the documented Java bug where position 3 being '_' (not uppercase letter) triggers validation failure.
