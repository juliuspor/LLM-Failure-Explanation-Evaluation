The bug is in the post-translation advancement loop. `consumed` represents the number of *codepoints* consumed by `translate_codepoint`, but `pos` indexes UTF-16 *code units*. The loop incorrectly calls `code_point_at(code_units, pos)` repeatedly from the same `pos` initially. Fix by advancing `pos` by `char_count(code_point_at(code_units, pos))` exactly `consumed` times, properly skipping the consumed code units.
