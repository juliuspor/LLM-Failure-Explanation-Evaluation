The test suite fails because the `CodeConsumer.add_number(-0.0)` method does not preserve the negative zero sign. In the `test_negative_zero` test, it expects the output to be "-0.0" but receives "0" instead (as shown in the Java failure example where expected:<var x=[-0.]0> but was:<var x=[]0>). This happens because standard string formatting like `str(-0.0)` or `repr(-0.0)` in Python converts -0.0 to "0.0", losing the negative sign. The `is_negative_zero` utility correctly detects -0.0 (as verified by `TestNegativeZeroUtility`), but the `add_number` method in hit06_codeconsumer.py fails to use this detection to output "-0.0". All other tests pass, confirming the issue is isolated to negative zero handling in `add_number`.
