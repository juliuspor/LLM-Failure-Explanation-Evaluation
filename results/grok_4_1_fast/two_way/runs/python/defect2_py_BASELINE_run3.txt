The code fails because the clamping logic doesn't work correctly when using custom bounds. In test_custom_bounds, GrayPaintScale(10.0, 20.0) sets self._lower_bound=10.0 and self._upper_bound=20.0. For value=5.0, the first clamp sets v=max(5.0, 10.0)=10.0, but then g is computed using the original unclamped value: g=int((5.0 - 10.0) / (20.0 - 10.0) * 255.0)=int(-255.0)=-255. The check if g < 0 raises ValueError. The bug is on line 74: g uses 'value' instead of the clamped 'v', so clamping has no effect on the computation.
