The code fails the test_null_safety_inside_arrays_mixed test case. When processing input text='aba', search_list=['a', 'b'], replacement_list=['c', None], it incorrectly produces 'aba' instead of the expected 'cbc'.

The root cause is at line 275 where parts.append(replacement_list[replace_index]) unconditionally appends the replacement without checking if it's None. When the second match 'b' is found at position 1, replace_index=1, so it appends replacement_list[1] which is None. This inserts the string 'None' (str(None)) into parts instead of skipping the replacement and advancing past the match.

Consequently, parts becomes ['c', 'None', 'a'], which joins to 'cNonea', failing the assertion self.assertEqual('cbc', result).

The null safety check at lines 286-288 (replacement_list[i] is None) only skips searching for future matches of that repl_index but does not prevent appending None during an actual match.
