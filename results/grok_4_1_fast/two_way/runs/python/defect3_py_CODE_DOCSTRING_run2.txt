The code fails in the loop after `consumed = self.translate_codepoint(code_units, pos, out)` when `consumed > 0`. The for loop `for pt in range(consumed):` increments `pos` by calling `char_count(code_point_at(code_units, pos))` for each consumed unit. However, `code_point_at` extracts a single codepoint from the current `pos` (a surrogate pair codepoint if present), and `char_count` returns 2 for surrogate pairs or 1 for BMP codepoints. This undercounts the total units consumed by `translate_codepoint`, which handles multiple codepoints (each potentially 1-2 units). As a result, `pos` advances too little, causing an infinite loop since `pos < length` remains true.
