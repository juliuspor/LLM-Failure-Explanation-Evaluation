The bug is that `substring_java(text, 0, upper)` excludes the character at position `upper`, but Java's `substring(0, upper)` includes characters up to index `upper-1`. To match Java behavior, change all `substring_java(text, 0, upper)` calls to `substring_java(text, 0, upper + 1)` so Python slicing `s[0:upper+1]` includes the correct range.
