The code fails the test `test_abbreviate_lower_beyond_length` where `WordUtils.abbreviate("0123456789", 15, 20, None)` should return "0123456789" but returns "" instead.

The root cause is in these lines:

```python
if upper == -1 or upper > len(text):
    upper = len(text)
if upper < lower:
    upper = lower
```

When `text = "0123456789"` (length 10), `lower = 15`, `upper = 20`:
1. `upper > len(text)` is true, so `upper` becomes 10
2. `upper < lower` (10 < 15) is true, so `upper` becomes 15

Now `upper = 15` and `lower = 15`. The code then does `index = StringUtils.index_of(text, " ", 15)` which searches for a space starting at position 15 (beyond the string's length of 10), so `index = -1`.

It takes the first branch: `substring_java(text, 0, upper)` where `upper = 15`. Since the string length is only 10, `substring_java(text, 0, 15)` returns an empty string "" (Java's `substring(0, n)` returns empty when `n > length`).

The condition `upper != len(text)` (15 != 10) is true, so it appends `StringUtils.default_string(None)` (likely ""), resulting in "".

The code never checks if `lower > len(text)` before calling `index_of` or `substring`, causing it to silently truncate to empty when bounds exceed the string length.
