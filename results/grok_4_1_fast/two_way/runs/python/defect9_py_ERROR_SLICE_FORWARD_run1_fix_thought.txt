The bug occurs because `len(search_list[i])` is called when `search_list[i]` is None, despite the `or` short-circuit check. The condition must explicitly check if `search_list[i]` is not None before calling `len()`. Fixed by adding `search_list[i] is None` check before `len(search_list[i]) == 0` in both the initial match finding loop and the subsequent next match finding loop.
