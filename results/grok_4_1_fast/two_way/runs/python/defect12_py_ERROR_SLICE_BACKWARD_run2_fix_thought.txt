The bug is in substring_java where end > len(s) triggers IndexError even when start <= len(s). In abbreviate, when upper == len(text), substring_java(text, 0, upper) fails if upper > len(text) due to the overly strict end > len(s) check. Fix by changing to end > len(s) and start >= len(s) to allow valid slices where end exceeds length (Python slicing handles this safely).
