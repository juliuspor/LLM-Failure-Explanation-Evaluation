The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code splits the input on underscores into ['fr', '', 'POSIX'] and detects 3 parts (length=7). After extracting language='fr', it reaches lines 227-230 where ch3=locale_str[3] which is the first character of the empty country ('') - this is '_'. The check `if ch3 < 'A' or ch3 > 'Z'` fails because '_' (ASCII 95) is not an uppercase letter A-Z, so it raises `ValueError("Invalid locale format: fr__POSIX")`. The code assumes position 3 is always the start of a country code and must be uppercase A-Z, but with an empty country, position 3 is still '_' from the double underscore.
