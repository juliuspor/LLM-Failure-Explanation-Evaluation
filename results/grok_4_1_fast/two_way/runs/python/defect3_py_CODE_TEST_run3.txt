The code fails in the final for loop when `translate_codepoint` consumes a surrogate pair (consumed > 0). After `consumed = self.translate_codepoint(code_units, pos, out)`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` incorrectly advances `pos` multiple times from the **same position**. For a surrogate pair (2 UTF-16 code units), if `consumed == 2`, it calls `code_point_at(code_units, pos)` twice at the unchanged `pos`, getting the same codepoint both times. This causes `pos` to advance too far (e.g., 4+ units instead of 2), leading to an IndexError in `test_translate_surrogate_pair_with_suffix` when processing "\U0001F630A" (😰A), as it skips past the trailing 'A' character.
