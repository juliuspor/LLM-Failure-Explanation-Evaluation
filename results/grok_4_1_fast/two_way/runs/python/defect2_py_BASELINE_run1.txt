The code clamps `value` to `self._lower_bound` and `self._upper_bound` and stores it in `v`, but then computes `g` using the original unclamped `value` instead of `v`. This causes the test to fail on `test_lower_bound_clamping` with `value = -0.5`: `g = int((-0.5 - 0.0) / (1.0 - 0.0) * 255.0) = int(-127.5) = -128`, which triggers `ValueError` at line 77 since `g < 0`. The same issue affects `test_upper_bound_clamping` with `value = 1.5`: `g = int((1.5 - 0.0) / (1.0 - 0.0) * 255.0) = int(382.5) = 382 > 255`. Change line 74 to use `v` instead of `value`.
