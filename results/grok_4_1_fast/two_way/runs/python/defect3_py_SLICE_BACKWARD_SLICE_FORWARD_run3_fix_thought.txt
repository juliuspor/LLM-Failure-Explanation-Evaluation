The bug diagnosis incorrectly identifies the issue. The `code_point_at` function is correct. The real bug is in `CharSequenceTranslator.translate`: after `consumed = self.translate_codepoint(...)` returns a non-zero value, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls `code_point_at(code_units, pos)` with the same `pos` each iteration, causing it to always consume only 1 code unit regardless of `consumed` value. Fixed by advancing `pos` by `char_count(code_point_at(code_units, pos))` once per consumed codepoint, using a temporary index.
