The bug diagnosis incorrectly identifies `to_utf16_code_units` as the problem. The actual issue is in `CharSequenceTranslator.translate`: after `consumed == 0`, `pos += len(c)` uses code unit count instead of codepoint count, causing surrogate pairs (2 units) to advance `pos` by 2 but consume only 1 codepoint, leading to infinite loops. Fix by using `char_count(codepoint)` to advance correctly by codepoint width.
