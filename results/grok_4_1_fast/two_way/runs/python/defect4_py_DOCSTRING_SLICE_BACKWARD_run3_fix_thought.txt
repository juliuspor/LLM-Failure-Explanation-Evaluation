The bug is in the min_start_index logic: when self._min_start_index >= 0, the code correctly compares start < min_start but then unconditionally sets self._min_start_index = index on line 352, overwriting the true minimum with the current index. The fix is to move this assignment inside the if start < min_start block, so it only updates when the new period actually has a smaller start time. The same pattern applies correctly to other bounds checks.
