The infinite loop occurs when encountering a lone high surrogate. When `translate_codepoint` returns 0, `code_point_at` returns the high surrogate value (not 0), `to_chars` returns `[high_surrogate]` (len=1), and `pos += 1` correctly advances past it. However, `CsvEscaper.translate_codepoint` has a bug: it returns 2 for valid surrogate pairs but doesn't append the low surrogate. Fix by appending both code units and returning 2 only for valid pairs. For lone high surrogates, return 0 to let the main loop handle copying it as-is.
