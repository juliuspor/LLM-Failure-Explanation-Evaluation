The code fails because it uses the original `text` object for all `text.find()` calls instead of the modified text after replacements. This causes incorrect replacement positions after the first match.

**What happens:**
1. First match found at `text_index` (e.g., text='abcde', search_list=['b'], finds at index 1).
2. `parts` gets `['a', 'replacement']`, `start` becomes `1 + 1 = 2`.
3. Next loop searches `text.find(search_list[i], start=2)` on **original** `text='abcde'`, finds 'b' at index 1.
4. But `temp_index=1 < start=2` is false, so `text_index` stays -1, loop exits.
5. Final result: `'a' + 'replacement' + text[2:] = 'areplacementcde'` (misses replacing 'b').

**Why:** After first replacement, `text` is unchanged but `start` advances. Subsequent `text.find(..., start)` on original text skips matches before `start` but finds nothing new since prior matches are before `start`.

**Where:** In the while loop, lines `temp_index = text.find(search_list[i], start)` and `temp_index = text.find(search_list[i])` (first iteration) always use original `text`, not rebuilt text from `parts`.
