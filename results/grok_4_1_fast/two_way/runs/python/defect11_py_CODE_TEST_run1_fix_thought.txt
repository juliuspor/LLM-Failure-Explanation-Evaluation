The bug is in `_random_internal` (called by `random`), which doesn't validate that `start < end` when `chars is None`. When `start == end` (like 5,5), `gap=0` causes `_next_int_java(rnd, 0)` to raise ValueError('bound must be positive'), but the message doesn't mention 'start' and 'end'. Add validation in `_random_internal` to raise ValueError mentioning 'start' and 'end' when no characters are available.
