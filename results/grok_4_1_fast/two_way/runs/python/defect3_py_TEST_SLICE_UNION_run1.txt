The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at` at line 35. When processing input_str = '\U0001F630A' (😰A), `to_utf16_code_units` creates code_units = [55358, 56672, 65] where index=65 (ASCII 'A'). The CsvEscaper.translate_string calls code_point_at(code_units, 1) to get the code point after the surrogate pair 😰 (which spans indices 0-1). Since 1 < len(code_units)=3, it doesn't raise IndexError there. However, inside code_point_at at line 39, it checks if index+1 < len(code_units) (2 < 3 is true), reads low = code_units[2] = 65, but 65 is NOT in surrogate low range 0xDC00-0xDFFF. It falls through to return high = code_units[1] = 56672. Later code in translate_string incorrectly advances the index by 2 positions (expecting a surrogate pair), setting index = 1 + 2 = 3. On the next iteration, code_point_at(code_units, 3) triggers the IndexError at line 35 because 3 >= len(code_units)=3.
