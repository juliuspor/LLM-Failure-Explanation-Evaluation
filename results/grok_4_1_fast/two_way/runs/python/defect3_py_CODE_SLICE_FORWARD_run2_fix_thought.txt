The bug occurs because when translate_codepoint returns consumed=2 for a surrogate pair, the translate method's loop calls code_point_at(code_units, pos) twice with the same pos, causing the second call to incorrectly return just the high surrogate (1 code unit) instead of recognizing the full pair (2 code units). Fix by advancing pos by the correct number of code units directly when consumed > 0, using char_count only on the first codepoint to get the total units consumed.
