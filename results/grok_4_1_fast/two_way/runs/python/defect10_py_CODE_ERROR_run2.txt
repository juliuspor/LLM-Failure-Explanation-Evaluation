The code fails with 'IndexError: String index out of range: 0' when processing a string like 'l' or 'L' (single character). Here's what happens:

1. For val = 'l', last_char = 'l' which is not a digit, so it enters the 'if not last_char.isdigit()' block.
2. numeric = val[:-1] = '' (empty string).
3. It reaches the 'l'/'L' case and calls char_at(numeric, 0), where numeric is empty.
4. char_at(numeric, 0) tries to access numeric[0] on an empty string, causing IndexError.

This occurs because the code doesn't check if numeric is empty before calling char_at(numeric, 0) in the condition:
if char_at(numeric, 0) == "-" and NumberUtils.is_digits(numeric[1:])

The error happens specifically in the 'l'/'L' handling branch when last_char is 'l' or 'L' and val has length 1.
