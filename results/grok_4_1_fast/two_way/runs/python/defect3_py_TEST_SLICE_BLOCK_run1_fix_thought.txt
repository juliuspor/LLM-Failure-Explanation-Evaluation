The bug occurs because after `translate_codepoint` consumes characters, the loop incorrectly advances `pos` by calling `char_count(code_point_at(code_units, pos))` repeatedly for `consumed` times, but `code_point_at` is called with the already-advanced `pos`, causing index out-of-bounds. The fix is to advance `pos` by `consumed` code units directly since `translate_codepoint` returns the number of code units consumed.
