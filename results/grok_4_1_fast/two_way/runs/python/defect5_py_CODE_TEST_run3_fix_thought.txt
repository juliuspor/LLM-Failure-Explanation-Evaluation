The bug occurs because `inferred_type == object` compares the Python `object` type class to itself, which always succeeds when `array`/`element` are None, but the check should only trigger when creating a new list from None (simulating Java array type mismatch). Fix by restructuring the type check to happen before `_copy_list_grow1` call, raising TypeError if `expected_type` is provided and `array` is None (null-to-typed array creation failure).
