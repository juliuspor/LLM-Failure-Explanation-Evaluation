The test suite fails because the `GrayPaintScale.get_paint` method raises a `ValueError` when computed color components are exactly 127.5 (like for input 0.5), as this falls outside the integer range 0..255 before rounding or truncation. The docstring explicitly states it raises `ValueError: If the computed color components fall outside 0..255`. In `test_mid_value_interpolation`, the line `color = gps.get_paint(0.5)` triggers this exception since (0.5-0.0)/(1.0-0.0)*255 = 127.5, causing the test to fail before the `assertEqual` check. Similarly, `test_custom_bounds` fails at `gps.get_paint(15.0)` because (15.0-10.0)/(20.0-10.0)*255 = 127.5.
