The bug is in the translate method's fallback path. When no translator consumes a codepoint, it calls code_point_at(pos) to get the codepoint, then extends out with to_chars(codepoint) and advances pos by len(c). However, len(c) is the number of code units output, not the number of code units consumed from input. For surrogate pairs, this advances pos by only 1 instead of 2, causing the next iteration to start at the low surrogate (index=1), which triggers IndexError in code_point_at when checking index+1. Fix by advancing pos by char_count(codepoint), which correctly returns 2 for surrogate pairs.
