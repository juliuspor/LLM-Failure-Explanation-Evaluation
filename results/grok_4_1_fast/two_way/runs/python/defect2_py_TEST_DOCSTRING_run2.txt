The test suite fails because the `GrayPaintScale.get_paint` method raises a `ValueError` when computed color components are exactly 127.5 (like for input 0.5), as this falls outside the integer range 0..255 per the docstring's `Raises` clause. In `test_mid_value_interpolation`, `gps.get_paint(0.5)` computes `(127.5, 127.5, 127.5)` from the linear interpolation formula `int((value - lower) / (upper - lower) * 255)`, triggering the exception before returning the expected `(127, 127, 127). Similarly, `test_custom_bounds` fails at `gps.get_paint(15.0)` for the same reason (middle value 15.0 between 10.0-20.0 also yields 127.5). The clamping tests may pass if implemented with `max(min(...))`, but interpolation tests crash due to this floating-point precision issue in the method's error checking.
