The code fails the test `test_negative_hours_negative_minutes` for input (-2, -15). The test expects offset -8100000 (-2h -15m = -8100000ms), but the code produces +8100000ms (+02:15). This happens because when hours_offset=-2, hours_in_minutes=-120. The condition `if hours_in_minutes < 0:` executes, setting minutes_offset = -120 - (-15) = -105. Then offset = -105 * 60000 = -6300000ms (-01:30), not the expected -8100000ms. The logic incorrectly subtracts the negative minutes_offset instead of adding its magnitude to create the total negative offset.
