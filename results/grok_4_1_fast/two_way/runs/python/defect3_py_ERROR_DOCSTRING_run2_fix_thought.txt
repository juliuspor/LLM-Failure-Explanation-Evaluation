The bug occurs because after `translate_codepoint` consumes characters, the loop incorrectly advances `pos` by calling `char_count(code_point_at(code_units, pos))` which can raise IndexError if `pos` is at or near the end of `code_units`. The fix is to advance `pos` by exactly the `consumed` value (number of code units consumed), as that's what `translate_codepoint` returns.
