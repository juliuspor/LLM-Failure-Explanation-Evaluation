The code fails the test_custom_bounds test case. When GrayPaintScale is initialized with custom bounds (lower_bound=10.0, upper_bound=20.0) and get_paint(15.0) is called:

1. The clamping lines 'v = max(value, self._lower_bound)' and 'v = min(v, self._upper_bound)' set v=15.0 (since 15.0 is within [10.0, 20.0]).

2. However, the next line 'g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)' uses the original unclamped 'value=15.0' instead of the clamped 'v', which is correct: (15.0-10.0)/(20.0-10.0)*255.0 = 127.5, so g=127.

3. The return statement (g, g, g) returns (127, 127, 127), which passes.

But for gps.get_paint(5.0):

1. Clamping sets v=10.0 (max(5.0, 10.0)).

2. The g calculation uses original value=5.0: (5.0-10.0)/(20.0-10.0)*255.0 = (-5.0/10.0)*255.0 = -127.5, so g=int(-127.5)=-128.

3. The validation 'if g < 0 or g > 255' detects g=-128 < 0 and raises ValueError, causing the test to fail instead of returning clamped (0,0,0).

Root cause: The g calculation (line with 'g = int((value - ...') uses 'value' instead of the clamped 'v', so clamped inputs still produce out-of-range g values and trigger the exception.
