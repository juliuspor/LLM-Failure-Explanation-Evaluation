The bug occurs because after `translate_codepoint` consumes characters, the loop incorrectly advances `pos` using `code_point_at(code_units, pos)` where `pos` may already be at or beyond `len(code_units)`, causing IndexError in `code_point_at`. Fixed by advancing `pos` directly by the number of code units consumed (`consumed`), since `translate_codepoint` returns the count of code units processed.
