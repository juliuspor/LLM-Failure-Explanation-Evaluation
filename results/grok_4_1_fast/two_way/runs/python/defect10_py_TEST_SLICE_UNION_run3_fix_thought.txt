The bug occurs because single-letter inputs like 'l', 'L', 'f', 'F' pass through the non-digit last_char block but fail the subsequent validation checks (e.g., is_digits check) without raising ValueError, falling through silently. Invalid strings like 'junk', 'bobL' also don't raise. Added early validation after None/empty checks to raise ValueError for invalid formats before processing. Fixed _is_all_zeros call for exp=None case and ensured all invalid paths raise ValueError.
