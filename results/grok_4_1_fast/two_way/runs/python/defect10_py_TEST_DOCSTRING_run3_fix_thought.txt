The bug occurs because invalid inputs like 'l', 'L', 'f', 'F', 'junk', 'bobL' are not properly validated before attempting parsing. When last_char is not a digit, the code processes type qualifiers but doesn't validate that the numeric part is actually digits (only checks is_digits but doesn't raise ValueError on failure). Single letter inputs skip validation and fall through to integer parsing attempts which fail silently. Added early validation using is_number() and explicit ValueError raises when numeric part validation fails for type qualifiers.
