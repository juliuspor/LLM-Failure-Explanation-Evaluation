The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError at line 36 in `code_point_at`: `raise IndexError(f"String index out of range: {index}")`. The input string "\U0001F630A" (😰A) contains a surrogate pair (2 UTF-16 code units) followed by 'A'. The `CharSequenceTranslator.translate` method (in `CsvEscaper.translate_string`) incorrectly advances the `index` variable in the UTF-16 `code_units` array. When processing the surrogate pair at index 0-1, the index is advanced by only 1 position instead of 2, landing on index 1 (the trailing surrogate). The next call to `code_point_at(1)` checks `if index < 0 or index >= len(code_units)` (line 35), finds index 1 is valid, but then fails internally when trying to decode the incomplete trailing surrogate as a full code point, causing the index to be advanced beyond `len(code_units)`, triggering the IndexError on the next `code_point_at` call.
