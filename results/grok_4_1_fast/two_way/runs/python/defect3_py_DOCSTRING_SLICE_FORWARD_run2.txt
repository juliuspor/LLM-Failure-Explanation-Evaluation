The code fails because it does not advance the index after consuming a UTF-16 surrogate pair. On line 39, it checks if `index + 1 < len(code_units)` to access `low = code_units[index + 1]` on line 40. If `high` (on line 38) is a high surrogate (0xD800-0xDBFF) and `low` is a low surrogate (0xDC00-0xDFFF), it combines them into a single code point on line 43. However, the function returns without incrementing `index`, so the next call to this function (likely in a loop) will reuse the same `high` value at the same `index`. This causes the high surrogate to be decoded repeatedly as a standalone BMP character (on line 44 in subsequent calls), corrupting the output and skipping the low surrogate.
