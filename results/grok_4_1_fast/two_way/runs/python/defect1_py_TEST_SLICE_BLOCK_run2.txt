The test `test_mixed_signs_invalid` fails because it expects `DateTimeZone.for_offset_hours_minutes(-2, 15)` to return a zone with id "-02:15", but the implementation at line 377 raises `ValueError("Minutes out of range: 15")`. The validation `if minutes_offset < 0 or minutes_offset > 59:` incorrectly rejects positive minutes (15) as out of range since 15 >= 0 and 15 <= 59. According to the docstring, negative hours with positive minutes should produce a negative offset like "-02:15", but the absolute value check on `minutes_offset` prevents this valid case from working.
