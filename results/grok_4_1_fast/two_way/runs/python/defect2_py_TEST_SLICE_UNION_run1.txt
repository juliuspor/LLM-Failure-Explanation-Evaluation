The test suite fails because the `get_paint` method in `GrayPaintScale` raises a `ValueError` instead of returning clamped RGB values for inputs outside the bounds. Specifically:

- In `test_lower_bound_clamping`, `gps.get_paint(-0.5)` calls line 74: `g = int((-0.5 - 0.0) / (1.0 - 0.0) * 255.0)` which computes `g = int(-127.5) = -128`.
- Line 77 detects `g < 0` and line 78 raises `ValueError("Color parameter outside of expected range: Red Green Blue")`.
- Similarly, `test_upper_bound_clamping` with `gps.get_paint(1.5)` computes `g = int(382.5) = 382 > 255`, triggering the same `ValueError`.

The tests expect `(0,0,0)` and `(255,255,255)` respectively, but the exception crashes the test before any return value. The `custom_bounds` test also fails for the same reason with values 5.0 and 25.0 outside [10.0, 20.0]. Only `test_mid_value_interpolation` passes since 0.0, 0.5, and 1.0 produce g values of 0, 127, and 255 which don't trigger the error.
