The condition `index < 0 or index >= len(s)` becomes `index < 0 or index >= 0`. Since `index` is negative, the first part `index < 0` is True, triggering the `IndexError`. This happens during backward slicing logic that checks string bounds before accessing `s[index]`, but it doesn't handle the empty string case where no valid negative index exists.
