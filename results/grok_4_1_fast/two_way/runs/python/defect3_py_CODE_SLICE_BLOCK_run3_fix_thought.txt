The bug occurs because the loop condition `while pos < length` allows pos to equal length-1, where code_point_at(pos) succeeds but to_chars(codepoint) may produce 2 code units, causing pos += 2 to overshoot beyond length. On the next iteration, pos == length but code_point_at is called before checking consumed==0, raising IndexError. Fix by adding `if pos >= length: break` after incrementing pos to prevent overshooting and unnecessary calls.
