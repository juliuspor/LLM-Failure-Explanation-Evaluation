The bug occurs because `char_at(numeric, 0)` is called on 'bob' (from 'bobL') without bounds checking in the context of invalid input. The `is_digits` check fails due to non-digits, but before raising ValueError, `char_at` throws IndexError if index computation is invalid. Fix by adding early validation: check if the numeric part (before type suffix) contains only valid number characters using `is_number(numeric[:-1])` or similar before accessing `char_at`. For 'bobL', since 'bob' fails `is_number`, raise ValueError immediately without char_at call.
