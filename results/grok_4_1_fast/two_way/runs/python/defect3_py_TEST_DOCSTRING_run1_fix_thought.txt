The bug is in the translate method's loop. When translate_codepoint returns a non-zero consumed count, the code incorrectly advances pos by summing char_count for each of the consumed codepoints starting from the current pos, causing double-counting of surrogate pairs. Instead, it should simply advance pos by the number of code units consumed (consumed value), as translate_codepoint already handles surrogate pairs correctly and returns the code unit count it consumed.
