The code fails in the loop after `consumed = self.translate_codepoint(code_units, pos, out)` when `consumed > 0`. It enters the for loop `for pt in range(consumed):` which iterates `consumed` times. Each iteration calls `code_point_at(code_units, pos)` to get the codepoint at the **current** `pos`, then adds `char_count` of that codepoint to compute how much to advance `pos`. However, `pos` is **not updated inside the loop**, so every iteration uses the **same `pos`**. This causes `pos` to be advanced by `consumed * char_count(code_point_at(code_units, pos))`, massively overshooting the intended position past `length`. The loop condition `while pos < length` then fails immediately, leaving surrogate pairs or sequences unprocessed and the output `out` incomplete.
