The bug occurs because the logic for updating min/max indices is inverted. When an index is -1 (indicating no previous extremum), the code skips the comparison and doesn't set the new index. The fix reverses the logic: always set the index to the new value when it's -1, and only compare/update when a previous extremum exists (>=0). Also fix a bug where _max_middle_index uses _min_middle_index variable.
