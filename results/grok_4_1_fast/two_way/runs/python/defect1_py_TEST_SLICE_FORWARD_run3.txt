The test `test_negative_hours_negative_minutes` fails because the `DateTimeZone.for_offset_hours_minutes(-2, -15)` implementation at line 383 incorrectly calculates the offset. When hours are negative, the code executes `minutes_offset = hours_in_minutes - minutes_offset` where `hours_in_minutes = -120` (from -2 hours) and `minutes_offset = -15`, resulting in `minutes_offset = -120 - (-15) = -105`. Then at line 385, it adds `minutes_offset = -105 + (-15) = -120`, making the total offset -120 minutes (-02:00) instead of the expected -135 minutes (-02:15). The logic wrongly subtracts negative minutes (making them positive) before adding them back, canceling their contribution. The test expects `expected_millis = -(2 * 3600000 + 15 * 60000) = -8100000` but gets `-120 * 60000 = -7200000`, causing the `assertEqual(zone.get_offset(0), expected_millis)` to fail.
