The bug occurs because `replace_index` remains -1 when the earliest match has a None replacement (skipped in the loop), but the while loop still executes and accesses lists with index -1, causing IndexError. Fix by adding a check after finding matches: if `text_index != -1` but `replace_index == -1`, skip that match by setting `text_index = -1`. Apply this check both after the initial match-finding loop and inside the while loop.
