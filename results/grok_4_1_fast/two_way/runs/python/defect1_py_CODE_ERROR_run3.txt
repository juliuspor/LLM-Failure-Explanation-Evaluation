The code fails because the validation for minutes_offset occurs before combining it with hours_offset, allowing invalid negative values to pass through initially. Specifically, when hours_offset = -1 (valid, within -23 to 23) and minutes_offset = -15 (invalid, < 0), the minutes_offset check 'if minutes_offset < 0 or minutes_offset > 59:' raises ValueError('Minutes out of range: -15'). This happens on the line checking minutes_offset range, before the later logic in the try block where negative hours_in_minutes = -60 is subtracted from minutes_offset = -15 to produce minutes_offset = -75. The validation should occur after computing the total minutes to catch the true out-of-range total offset.
