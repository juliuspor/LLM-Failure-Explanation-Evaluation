The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at`. The input "\U0001F630A" (😰A) produces UTF-16 code units [0xD83D, 0xDE30, 0x0041]. When `code_point_at` is called at index=0 in the `translate` loop:

- Line 38: `high = code_units[0]` gets 0xD83D (valid high surrogate)
- Line 39: Condition passes, checks `index + 1 < len(code_units)` (1 < 3, true)
- Line 40: `low = code_units[1]` gets 0xDE30 (valid low surrogate)
- Line 41: Condition passes, returns the combined code point

The `translate` method then advances `index += 2` (to index=2). Next iteration calls `code_point_at(index=2)`:

- Line 38: `high = code_units[2]` gets 0x0041 ('A')
- Line 39: `0xD800 <= 0x0041 <= 0xDBFF` is false, skips if block
- Line 44: Returns `high` (0x0041)

`translate` advances `index += 1` to index=3. Now `index == len(code_units)`, but the loop condition `index < len(code_units)` in `translate` uses `<` instead of `<=`, so it doesn't process the final character and exits prematurely, producing incomplete output instead of the expected "\U0001F630A".
