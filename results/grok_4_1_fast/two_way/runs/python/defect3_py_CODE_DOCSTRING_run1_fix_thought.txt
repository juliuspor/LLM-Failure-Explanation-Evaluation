The bug is in the post-translation advancement logic. After `translate_codepoint` returns `consumed > 0`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` repeatedly calls `code_point_at` on the same unchanged `pos`, causing an infinite loop. The fix is to simply advance `pos` by the total number of code units consumed: `pos += consumed`. This correctly skips the already-processed code units.
