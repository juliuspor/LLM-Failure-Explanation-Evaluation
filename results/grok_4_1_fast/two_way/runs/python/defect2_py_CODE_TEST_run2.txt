The code fails the test_custom_bounds test case. In the get_paint method, the clamping lines (lines 3-4) correctly set v to be between self._lower_bound and self._upper_bound. However, the calculation for g on line 6 uses the original unclamped 'value' instead of the clamped 'v': g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0). For custom bounds like lower=10.0 and upper=20.0, when value=5.0 is passed, v becomes 10.0 but g is still calculated using value=5.0, resulting in g = int((5.0-10.0)/(20.0-10.0)*255.0) = int(-127.5) = -128. The validation check on line 9 detects g < 0 and raises ValueError, causing the test to fail instead of returning (0,0,0). The same issue occurs for values above the upper bound.
