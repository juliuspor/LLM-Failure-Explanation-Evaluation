The bug is in the negative hours case where minutes_offset is incorrectly subtracted instead of added. For negative hours_offset (e.g., -1 hour = -60 minutes), the total should be hours_in_minutes + minutes_offset (-60 + 30 = -30 minutes total). Currently it subtracts (-60 - 30 = -90), producing wrong offsets. Fix by always adding: total_minutes = hours_offset * 60 + minutes_offset.
