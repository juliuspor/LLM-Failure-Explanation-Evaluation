The bug is in the `char_at` function used by `create_number`. It incorrectly raises IndexError when `index == len(s)` (e.g., index=0 on empty string), but Python slicing/indexing handles this gracefully. Fix by changing the bounds check to `index < 0 or index > len(s)` to allow `index == len(s)`. No changes needed to `create_number` itself.
