The bug is in `code_point_at`: when a valid surrogate pair is detected, it returns the combined codepoint but the caller incorrectly advances by `char_count(codepoint)` which is always 1 for the returned supplementary codepoint (>0xFFFF). It should advance by 2 code units. Fix by changing the advancement logic in `translate` to always advance by the number of code units consumed (1 or 2), not codepoints.
