The code fails due to incorrect logic in calculating the total minutes offset on these lines:

hours_in_minutes = hours_offset * 60
if hours_in_minutes < 0:
    minutes_offset = hours_in_minutes - minutes_offset
else:
    minutes_offset = hours_in_minutes + minutes_offset

When hours_offset is negative (e.g., -1), hours_in_minutes becomes negative (e.g., -60). The if condition triggers, and minutes_offset gets reassigned as hours_in_minutes - minutes_offset (e.g., -60 - 30 = -90). Later, offset = FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE) uses this wrong value (-90 minutes instead of the correct -90 minutes total), producing incorrect results.

The root cause is the subtraction in the negative case: it should add the absolute minutes component (hours_in_minutes + minutes_offset, e.g., -60 + 30 = -30), but subtracts instead, flipping the minutes_offset sign and yielding wrong totals like -90 instead of -30.
