The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The input string has length 7. At line 227, `ch3 = locale_str[3]` gets `'_'` and `ch4 = locale_str[4]` gets `'P'`. Then at line 229, the condition `ch3 < 'A' or ch3 > 'Z'` is true since `'_'` (ASCII 95) is greater than `'Z'` (ASCII 90), so it raises `ValueError(f"Invalid locale format: {locale_str}")` with message "Invalid locale format: fr__POSIX". The code incorrectly assumes position 3 must be an uppercase letter for country code, but in `language__variant` format, position 3 is `'_'` indicating empty country, which should be handled before checking for uppercase letters.
