The test suite fails because the `get_paint` method in `GrayPaintScale` raises a `ValueError` instead of returning clamped RGB values for inputs outside the bounds. Specifically:

1. In `test_lower_bound_clamping`, `gps.get_paint(-0.5)` calls line 74: `g = int((-0.5 - 0.0) / (1.0 - 0.0) * 255.0)` which equals `int(-127.5) = -128`.

2. Line 77 detects `g < 0` and line 78 raises `ValueError("Color parameter outside of expected range: Red Green Blue")`.

3. Similarly, `test_upper_bound_clamping` with `gps.get_paint(1.5)` computes `g = int((1.5 - 0.0) / 1.0 * 255.0) = int(382.5) = 382`, which triggers `g > 255` and the same `ValueError`.

4. The tests expect `(0,0,0)` and `(255,255,255)` respectively, but the exception crashes the tests.

The root cause is that lines 77-78 raise an exception instead of clamping `g` to the range `[0, 255]` before returning the tuple.
