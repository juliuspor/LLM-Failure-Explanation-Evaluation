The IndexError in code_point_at() occurs because the translate() method incorrectly advances the position using char_count(code_point_at(code_units, pos)) in a loop, where code_point_at(pos) fails if pos is out of bounds. The fix replaces this with a direct code unit count: pos += consumed, since translate_codepoint() returns the number of code units consumed, matching the UTF-16 code_units indexing.
