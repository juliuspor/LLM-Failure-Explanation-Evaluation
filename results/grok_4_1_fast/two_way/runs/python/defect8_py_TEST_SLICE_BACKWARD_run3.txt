The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code assumes a 5-character string "fr__P" has a 2-character uppercase country code at positions 3-4 (lines 227-228: `ch3 = locale_str[3]` and `ch4 = locale_str[4]`). For "fr__POSIX", `ch3` is '_' (ASCII 95) and `ch4` is 'P'. Line 229 checks `if ch3 < 'A' or ch3 > 'Z'`, which fails because '_' < 'A', raising `ValueError(f"Invalid locale format: {locale_str}")` at line 230. The code doesn't recognize the language__variant format (double underscore with empty country) and incorrectly validates the first '_' as an invalid country code character.
