This causes it to always compare against the minimum middle period item instead of the maximum, leading to incorrect updates of `self._max_middle_index`. When `self._max_middle_index >= 0`, it fetches the wrong data item using the min index, computes `max_middle` from that min period (which is too small), and then incorrectly sets `self._max_middle_index = index` even when the new `middle` is not actually larger.
