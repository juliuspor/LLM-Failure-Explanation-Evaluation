The bug diagnosis is incorrect - the error is not in `code_point_at` where `code_units` is empty, but in the `translate` method's loop. After `translate_codepoint` returns `consumed > 0`, the loop incorrectly calls `code_point_at(code_units, pos)` where `pos` is the unchanged original position, causing an IndexError when `pos` equals `length`. The fix updates `pos` incrementally using `char_count` within the loop for each consumed codepoint.
