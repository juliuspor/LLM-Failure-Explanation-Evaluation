The bug is in the offset calculation logic. The code incorrectly uses `hours_in_minutes - minutes_offset` for negative hours, which subtracts the minutes offset instead of adding it. The correct approach is to always compute total minutes as `hours_offset * 60 + minutes_offset`, then convert to millis. This ensures negative hours work correctly (e.g., -1:30 becomes -90 minutes). Removed unnecessary try-except since `FieldUtils.safe_multiply` already handles overflow.
