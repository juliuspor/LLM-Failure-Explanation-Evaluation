The bug is in the offset calculation logic. The code incorrectly uses `hours_in_minutes - minutes_offset` for negative hours, which subtracts the minutes offset instead of adding it. The correct approach is to always compute total minutes as `hours_offset * 60 + minutes_offset`, then convert to millis. This fixes the calculation for negative offsets (e.g., -2:30 should be -9000000 millis).
