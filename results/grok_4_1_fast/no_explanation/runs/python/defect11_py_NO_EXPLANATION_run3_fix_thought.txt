The `random` method exists but has incorrect surrogate pair handling that doesn't match Java's behavior. Java's RandomStringUtils doesn't generate surrogate pairs from the code point range; it only handles them correctly if they appear in the input `chars` array. The current implementation generates invalid surrogate pairs when code points fall in surrogate ranges, which breaks the algorithm. Fixed by rejecting surrogate code points entirely when using code point range (chars=None), preserving Java compatibility.
