The bug occurs because `random_letters_numbers` calls `_random_internal` with `start=0, end=0` when both `letters=True` and `numbers=True`, but `_random_internal` doesn't handle this case correctly for alphanumeric. Fix by setting `start=48` (ASCII '0') and `end=123` (ASCII '{') to cover digits 0-9 and letters A-Z a-z, matching Java's alphanumeric behavior.
