The bug is in the _update_bounds method where indices are unconditionally set to the new index when no previous minimum/maximum exists (_index < 0), but for max bounds, there's a wrong reference to _min_middle_index instead of _max_middle_index. Fixed by: 1) only setting indices when _index < 0 at the beginning, 2) always comparing new values against current min/max, 3) correcting the _max_middle_index reference to use self._max_middle_index.
