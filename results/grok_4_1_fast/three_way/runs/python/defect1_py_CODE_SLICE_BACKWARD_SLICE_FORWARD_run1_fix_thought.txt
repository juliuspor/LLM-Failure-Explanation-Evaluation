The bug is in the offset calculation logic for negative hours_offset. The code incorrectly subtracts minutes_offset from hours_in_minutes when hours are negative, making the total more negative (e.g., -60 - 30 = -90). Instead, it should always add the components: total_minutes = hours_offset * 60 + minutes_offset, regardless of sign. Remove the conditional subtraction and simplify to a single addition.
