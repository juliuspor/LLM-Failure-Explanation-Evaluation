The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at`. When processing input "\U0001F630A" (😰A), the UTF-16 code units are [0xD83D, 0xDE30, 0x0041]. At index=0 in `code_point_at`, it detects the high surrogate (0xD83D) at `code_units[0]`, checks `index + 1 < len(code_units)` (1 < 3, true), reads low surrogate `code_units[1]` (0xDE30), and correctly returns the code point. However, `CsvEscaper.translate_string` then calls `code_point_at` again at index=2 for character 'A'. The bounds check `if index < 0 or index >= len(code_units)` (2 >= 3, true) raises IndexError on line 36 because index=2 is out of range for len(code_units)=3. The root cause is `translate_string` advancing the index by 2 after surrogate pairs (to skip both code units) but the test expects no escaping so index should advance by 1 per code point, causing it to overshoot the string length.
