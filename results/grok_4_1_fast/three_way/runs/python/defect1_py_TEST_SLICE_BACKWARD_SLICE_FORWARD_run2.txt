The test `test_negative_hours_negative_minutes` fails because the implementation at lines 383-386 incorrectly handles the case of negative hours (-2) and negative minutes (-15). When hours_offset is negative and minutes_offset is negative, the code enters the `elif hours_offset < 0:` block. It then executes `minutes_offset = hours_in_minutes - minutes_offset` (line 383), where `hours_in_minutes = 120` (2*60) and `minutes_offset = -15`, resulting in `minutes_offset = 120 - (-15) = 135`. This 135 exceeds the valid minute range (0-59), triggering the check at line 377 (`if minutes_offset < 0 or minutes_offset > 59`) which raises `ValueError("Minutes out of range: 135")`. Thus, `DateTimeZone.for_offset_hours_minutes(-2, -15)` raises ValueError instead of creating the expected -02:15 zone with offset -8100000 millis.
