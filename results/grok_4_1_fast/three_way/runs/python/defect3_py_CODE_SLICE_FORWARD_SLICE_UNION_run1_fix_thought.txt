The infinite loop occurs because when `translate_codepoint` returns 0 for a surrogate pair, `code_point_at` correctly decodes it to a supplementary codepoint, `to_chars` produces 2 code units, and `pos += len(c)` (== 2) advances exactly past the pair. However, `CsvEscaper.translate_codepoint` mishandles surrogate pairs by returning 2 (consuming 2 code units) when it should return 1 (consuming 1 codepoint). Fix by changing `CsvEscaper.translate_codepoint` to always consume 1 codepoint using `code_point_at`, regardless of surrogate pair structure.
