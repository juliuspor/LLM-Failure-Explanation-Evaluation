The bug diagnosis is incorrect - the IndexError occurs in `code_point_at()` called from `translate()`, not from an empty list on line 12/35. The real issue is in the `translate()` method's main loop: after `translate_codepoint()` returns `consumed > 0`, the loop incorrectly calls `code_point_at(code_units, pos)` where `pos` is the original index, but then advances `pos` by `char_count()` of that same position repeatedly, causing `pos` to jump ahead incorrectly and potentially call `code_point_at()` with an unchanged `pos` in the next iteration if `consumed` doesn't match the advancement. Fix by advancing `pos` directly by `consumed` code units instead of using codepoints.
