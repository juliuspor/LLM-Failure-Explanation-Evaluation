The code fails in test_abbreviate_lower_beyond_length() when calling WordUtils.abbreviate("0123456789", 15, 20, None). The input text has length 10. The code does not clamp the 'lower' parameter (15) to the text length, so lower=15 > len(text)=10. After setting upper=min(20,10)=10, the condition upper < lower becomes true (10 < 15), so upper is set to lower (upper=15). Then StringUtils.index_of(text, " ", lower=15) is called, but index_of with start index 15 on a 10-character string throws IndexError: String index out of range: 15.
