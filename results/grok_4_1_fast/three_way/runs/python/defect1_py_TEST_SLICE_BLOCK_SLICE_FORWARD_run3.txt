The test `test_negative_hours_negative_minutes` fails because the implementation at lines 383-386 incorrectly calculates the offset for negative hours and negative minutes. When hours_offset = -2 and minutes_offset = -15, the code enters the negative hours branch. It sets `hours_in_minutes = hours_offset * 60` (-120), then executes `minutes_offset = hours_in_minutes - minutes_offset` (-120 - (-15) = -105). This produces offset = -105 * 60000 = -6,300,000 (-1h45m), but the test expects -8,100,000 (-2h15m). The subtraction `hours_in_minutes - minutes_offset` wrongly reduces the magnitude when both are negative, instead of adding their absolute contributions.
