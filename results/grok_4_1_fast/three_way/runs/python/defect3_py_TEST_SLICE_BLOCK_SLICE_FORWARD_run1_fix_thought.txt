The bug occurs because when `translate_codepoint` returns 0 for a surrogate pair, `translate` incorrectly advances by `len(c)` (always 1 or 2 based on the first code unit) instead of the actual number of code units consumed (2 for surrogate pairs). Fix by using `char_count(codepoint)` to advance the correct number of code units (2 for supplementary characters, 1 for BMP).
