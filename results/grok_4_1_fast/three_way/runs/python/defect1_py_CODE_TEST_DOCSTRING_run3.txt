The code fails the test `test_negative_hours_negative_minutes` because of incorrect logic in the offset calculation when `hours_offset` is negative. Specifically, in the line `if hours_in_minutes < 0: minutes_offset = hours_in_minutes - minutes_offset`, when `hours_offset = -2` and `minutes_offset = -15`, it computes `hours_in_minutes = -120`, then `minutes_offset = -120 - (-15) = -105`. This produces an incorrect total offset of -105 minutes (-02:45) instead of the expected -135 minutes (-02:15). The subtraction `hours_in_minutes - minutes_offset` wrongly reduces the magnitude when both are negative, but the test expects additive behavior where both negative inputs increase the total negative offset magnitude.
