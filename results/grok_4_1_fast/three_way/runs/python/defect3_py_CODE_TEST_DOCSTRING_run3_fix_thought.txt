The bug occurs because the for loop incorrectly advances `pos` multiple times by full codepoint widths for each consumed unit, skipping unprocessed content. When `consumed > 1` (e.g., surrogate pair), it should advance `pos` once by the total width of all consumed codepoints. Fix by calculating total width with `consumed * char_count(code_point_at(code_units, pos))` or simply `sum(char_count(code_point_at(code_units, pos + i)) for i in range(consumed))`, but since translators consume whole codepoints, use `consumed * char_count(code_point_at(code_units, pos))`.
