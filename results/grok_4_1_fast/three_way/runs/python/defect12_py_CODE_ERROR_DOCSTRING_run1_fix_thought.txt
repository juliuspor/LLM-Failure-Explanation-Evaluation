The bug occurs because `upper` adjustment happens after the empty string check, but when `len(text) > 0` and `upper > len(text)`, `substring_java(text, 0, upper)` is called with `upper > len(text)`, triggering IndexError since `substring_java` enforces Java substring bounds. Move the `upper` adjustment immediately after null/empty checks to ensure it's always <= `len(text)` before any substring calls.
