The code fails due to incorrect logic for handling negative hours_offset. On line 383, when hours_in_minutes is negative (e.g., hours_offset = -1, so hours_in_minutes = -60), it executes 'minutes_offset = hours_in_minutes - minutes_offset'. For a valid input like hours_offset=-1 and minutes_offset=30, this sets minutes_offset = -60 - 30 = -90. Then on line 386, offset = FieldUtils.safe_multiply(-90, DateTimeConstants.MILLIS_PER_MINUTE), which is -5400000 millis (-90 minutes). This triggers ArithmeticError in safe_multiply because -90 minutes exceeds the supported offset range, causing the code to raise ValueError('Offset is too large') on line 388. However, -1:30 is a valid offset (-90 minutes total). The bug is that for negative hours, it should add the absolute hour minutes and minutes_offset (i.e., minutes_offset = abs(hours_in_minutes) + minutes_offset, then apply the negative sign), but the subtraction makes the total minutes too large in magnitude.
