The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError on line 36 in `code_point_at`. The input "\U0001F630A" (😰A) converts to UTF-16 code units [55357, 56360, 65] via `to_utf16_code_units`. When `code_point_at` processes index=1 (low surrogate 56360), it checks `index + 1 < len(code_units)` (2 < 3, true), then accesses `code_units[2]` (65, ASCII 'A'). Since 65 is not a low surrogate (0xDC00-0xDFFF), it returns the high surrogate value 55357. The bug occurs because `CsvEscaper.translate_string` incorrectly advances the index by 2 positions after every surrogate pair detection, even when it's not a complete pair. After processing index=0 (complete pair 😰), index advances to 2. Then at index=2 ('A'), it detects high surrogate range incorrectly due to UTF-16 logic mismatch, tries index+1=3 (out of bounds), and raises IndexError on line 35 when checking `index + 1 < len(code_units)`.
