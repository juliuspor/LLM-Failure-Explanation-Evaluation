The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code at lines 227-230 checks characters at fixed positions 3 and 4 (`ch3 = locale_str[3]`, `ch4 = locale_str[4]`). For "fr__POSIX", position 3 is '_' and position 4 is 'P'. The condition `if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':` (line 229) evaluates to true because '_' (ASCII 95) is greater than 'Z' (ASCII 90), triggering `raise ValueError(f"Invalid locale format: {locale_str}")` (line 230). This check incorrectly rejects the valid language__variant format where the country code is empty.
