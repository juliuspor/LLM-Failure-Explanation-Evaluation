The bug is in the incorrect logic for negative hours_offset. Instead of overwriting minutes_offset with 'hours_in_minutes - minutes_offset' (which produces wrong results like -60 - 30 = -90), compute total minutes as hours_offset * 60 + minutes_offset directly. This correctly handles both positive and negative cases (e.g., -1*60 + 30 = -30). Remove the problematic conditional and use safe_multiply on the total minutes.
