The code fails with 'IndexError: String index out of range: 2' in the final for loop when `consumed > 0` after processing a surrogate pair. Here's what happens:

1. For input "\U0001F630A" (😰A), `code_units` = [55357, 56360, 65] (surrogate pair + 'A')
2. At `pos=0`, `self.translate_codepoint()` recognizes the surrogate pair and returns `consumed=2`
3. The loop `for pt in range(consumed):` (pt=0,1) executes
4. Inside loop: `code_point_at(code_units, pos)` where `pos` is still 0 calls `code_units[0+2]=code_units[2]`
5. `code_units[2]=65` ('A'), then `char_count(65)=1`
6. `pos += 1` → pos becomes 1 after first iteration
7. Second iteration (pt=1): `code_point_at(code_units, 1)` calls `code_units[1+2]=code_units[3]`
8. Index 3 doesn't exist (length=3), causing IndexError

Root cause: When `consumed > 0`, the loop incorrectly uses the unchanged `pos=0` instead of advancing position by `consumed`. It should be `pos += consumed` immediately after the if-block, before the loop.
