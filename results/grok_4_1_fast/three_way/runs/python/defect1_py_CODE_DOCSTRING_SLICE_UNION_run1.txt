The code fails due to incorrect handling of negative hours_offset in the minutes calculation. On line 383, when hours_in_minutes is negative (e.g., hours_offset = -1, so hours_in_minutes = -60), it executes 'minutes_offset = hours_in_minutes - minutes_offset'. For a valid input like minutes_offset = 30, this sets minutes_offset = -60 - 30 = -90. Then on line 386, FieldUtils.safe_multiply(-90, DateTimeConstants.MILLIS_PER_MINUTE) triggers ArithmeticError because the total offset exceeds supported limits, even though -1:30 is a valid timezone offset (-90 minutes total). The subtraction wrongly makes the minutes more negative instead of less negative: it should be hours_in_minutes + minutes_offset (-60 + 30 = -30) to correctly compute the total minutes.
