The bug diagnosis misidentifies the issue - `char_at` correctly checks bounds and `s` is never None in `create_number` calls. The real bug is in `create_number`: when `exp_pos > -1` but no decimal point, `mant` excludes the exponent part, causing `_is_all_zeros(mant)` to fail for cases like '1e0L' (mant='1', all_zeros=False despite being integral). Fix by computing `all_zeros` only when needed for float checks, using full `numeric` string, and simplifying logic to match Java's behavior.
