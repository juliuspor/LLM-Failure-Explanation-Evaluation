The bug occurs because the loop condition `pos < length` allows `pos` to equal `length - 1`, and if `translate_codepoint` consumes characters that advance `pos` to `length`, then `code_point_at(code_units, pos)` is called with `pos == length`, causing IndexError. Fixed by changing the loop condition to `while pos <= length:` and adding an early return in the `consumed == 0` branch when `pos >= length`. Also fixed incorrect `pos` advancement in the consumed branch by using the correct starting index for `code_point_at`.
