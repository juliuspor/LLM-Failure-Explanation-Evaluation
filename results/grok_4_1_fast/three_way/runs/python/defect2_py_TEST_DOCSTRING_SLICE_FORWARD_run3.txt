The test suite fails because the `GrayPaintScale.get_paint` method raises a `ValueError` as documented in its docstring when computed color components fall outside `0..255`, but the tests expect clamping without exceptions. Specifically:

- `test_lower_bound_clamping`: `gps.get_paint(-0.5)` computes a negative value before clamping, triggering `ValueError` instead of returning `(0,0,0)`.
- `test_upper_bound_clamping`: `gps.get_paint(1.5)` computes a value >255 before clamping, triggering `ValueError` instead of returning `(255,255,255)`.
- `test_custom_bounds`: `gps.get_paint(5.0)` (below lower bound 10.0) and `gps.get_paint(25.0)` (above upper bound 20.0) trigger `ValueError` instead of returning `(0,0,0)` and `(255,255,255)`.

The `mid_value_interpolation` test may pass if internal clamping occurs before the final check, but the docstring-mandated `ValueError` for out-of-bounds components violates the test expectations for silent clamping on all boundary cases.
