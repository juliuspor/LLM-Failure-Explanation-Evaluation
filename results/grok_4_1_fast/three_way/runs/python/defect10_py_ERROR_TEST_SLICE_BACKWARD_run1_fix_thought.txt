The bug occurs because `char_at(numeric, 0)` is called when `numeric` is empty (e.g., for input 'l', numeric=''), passing index=0 to a 1-char string 'l' wait no—for 'l', numeric='' (empty), len('')=0, so index=0 >=0 triggers IndexError. Fix by adding early check for empty numeric before non-digit last_char handling, preventing invalid char_at call.
