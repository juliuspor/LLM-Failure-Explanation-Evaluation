The bug is in the translate method's index advancement logic after translate_codepoint returns a non-zero consumed count. The loop incorrectly advances pos by calling char_count(code_point_at(code_units, pos)) consumed times, but pos hasn't been updated yet and code_point_at(pos) returns the wrong codepoint. Instead, it should advance pos by the consumed count directly since translate_codepoint already consumed that many code units.
