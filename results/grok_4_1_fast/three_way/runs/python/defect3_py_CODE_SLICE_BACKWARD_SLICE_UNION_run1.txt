The code fails in the final for loop due to an infinite loop or incorrect position advancement. After `self.translate_codepoint` consumes `consumed` code units (e.g., 2 for a surrogate pair), the loop `for pt in range(consumed):` iterates over each consumed unit. Inside, `code_point_at(code_units, pos)` is called with the **current** `pos` (not advanced yet), and `char_count` returns the number of code units for that codepoint (1 for a normal char, 2 for a surrogate pair). Then `pos += char_count(...)` advances `pos`, but on the next iteration, `pos` is used again for `code_point_at`, which now points to the **next** codepoint after previous advancements. This skips codepoints incorrectly and can cause `pos` to overshoot `length` or loop indefinitely if `consumed` doesn't match the actual structure. The comment 'they just took care of a surrogate pair' suggests `consumed` is 2, but re-parsing with `code_point_at` at shifting `pos` values breaks the logic.
