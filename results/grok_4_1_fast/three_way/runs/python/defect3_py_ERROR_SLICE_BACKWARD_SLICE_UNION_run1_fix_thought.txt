The bug diagnosis is incorrect. The IndexError occurs in `translate_string` when manually decoding surrogate pairs from `out`, not in `code_point_at`. The issue is that `code_point_at` is never called with an empty `code_units` list in normal usage. However, the diagnosis mentions line 35, which corresponds to the surrogate decoding loop in `translate_string`. The real bug in `translate` is that when a translator consumes characters, `pos` is advanced incorrectly by calling `code_point_at(code_units, pos)` where `pos` has already advanced past the consumed characters, causing IndexError. Fixed by advancing `pos` directly by `consumed` code units instead of using codepoints.
