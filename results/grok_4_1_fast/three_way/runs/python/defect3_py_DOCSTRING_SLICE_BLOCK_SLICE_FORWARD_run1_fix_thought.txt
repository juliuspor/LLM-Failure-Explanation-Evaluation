The bug is in `code_point_at` where indexing `code_units` (a list of ints) returns an int, not a str, so surrogate pair detection works correctly. The diagnosis confuses this with string indexing. The real issue is in `translate` method: after `consumed > 0`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` uses the wrong starting `pos` (already advanced) and redundant calls. Fix by advancing `pos` correctly using `char_count` on the original consumed codepoints.
