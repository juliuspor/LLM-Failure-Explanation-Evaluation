The code fails in the `substring_java` calls when `upper` equals `len(text)`. The function `substring_java(text, 0, upper)` passes `end = len(text)`, but according to `SLICE_UNION`, if `end > len(s)` it raises `IndexError`. This happens in two places:

1. When no space is found (`index == -1`) and `upper == len(text)` (after `upper = len(text)` adjustment), so `substring_java(text, 0, len(text))` raises the error.

2. When a space is found after `upper` (`index > upper`) and `upper == len(text)`, so `substring_java(text, 0, upper)` again passes `end = len(text)` and raises the error.

The docstring expects the full string when no suitable space is found or `upper` has no limit, but `substring_java` rejects `end == len(s)` even though Python slicing allows `s[0:len(s)]`.
