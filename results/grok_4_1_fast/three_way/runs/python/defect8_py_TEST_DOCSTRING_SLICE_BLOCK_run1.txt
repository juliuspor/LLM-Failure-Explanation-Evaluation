The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code at lines 229-230 checks characters at positions 3 and 4 (`ch3` and `ch4`) expecting uppercase letters A-Z for a country code. In "fr__POSIX", position 3 is the first underscore ('_') and position 4 is the second underscore ('_'), which fail the condition `ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z'`, causing the `ValueError("Invalid locale format: fr__POSIX")`. The implementation does not handle the empty country case with double underscore (language__variant format) as required by the test.
