When `start == end` (like 5,5), `_random_internal` computes `gap=0` and calls `_next_int_java(rnd, 0)` which raises `ValueError('bound must be positive')`. This message doesn't mention 'start' or 'end', failing the test. Add validation in `random` to raise a `ValueError` with a message mentioning both 'start' and 'end' before calling `_random_internal`.
