The code fails with an IndexError in the final for loop when `translate_codepoint` consumes characters (consumed > 0). After `translate_codepoint` processes some code units starting at the original `pos`, the loop does `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. On the first iteration, `code_point_at(code_units, pos)` calls with the current `pos` (after `translate_codepoint`), but if `pos` equals `length`, it triggers the IndexError check at line 36: `if index < 0 or index >= len(code_units): raise IndexError(...)`. This happens because `translate_codepoint` can consume up to the end of `code_units`, leaving `pos == length`, and the loop still executes `range(consumed)` times, attempting to access `code_units[length]`.
