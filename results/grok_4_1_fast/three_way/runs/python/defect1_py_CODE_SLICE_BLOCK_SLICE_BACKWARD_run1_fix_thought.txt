The bug occurs because negative hours_offset creates negative hours_in_minutes, and the incorrect branch logic (hours_in_minutes - minutes_offset) produces excessively negative minutes_offset values, causing safe_multiply to overflow. Fixed by always computing total_minutes = hours_offset * 60 + minutes_offset (preserving sign of hours), then converting to millis. This correctly handles negative offsets like -1:30 -> -90 minutes -> -5400000 millis.
