The diagnosis is incorrect - `code_units` is populated in `translate()` via `to_utf16_code_units(input_seq)`, not in `code_point_at()`. The real bug is in `translate()`: after `translate_codepoint()` returns `consumed > 0`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls `code_point_at(code_units, pos)` with the same `pos` each iteration (since `pos` isn't updated until after the loop), causing infinite recursion when surrogate pairs are handled. Fix by advancing `pos` incrementally within the loop using `char_count()`.
