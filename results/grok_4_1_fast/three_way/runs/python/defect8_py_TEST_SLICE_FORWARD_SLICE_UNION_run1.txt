The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` does not return the expected `Locale("fr", "", "POSIX")`. 

In the `to_locale` method, when processing a 5+ character locale string like "fr__POSIX":
- At line 227-230, it checks `ch3 = locale_str[3] = '_'` and `ch4 = locale_str[4] = 'P'`. 
- The condition `ch3 < 'A' or ch3 > 'Z'` is true because '_' (ASCII 95) is outside A-Z range.
- This raises `ValueError("Invalid locale format: fr__POSIX")` before reaching lines 233-237 that would handle the empty country case (language__variant format).

The code incorrectly validates the country code position (chars 3-4) as uppercase letters even when it's empty (double underscore), blocking the intended `language__variant` parsing logic at line 233+ from executing.
