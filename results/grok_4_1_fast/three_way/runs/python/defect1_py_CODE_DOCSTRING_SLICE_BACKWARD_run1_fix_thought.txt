The bug occurs because negative hours_offset makes hours_in_minutes negative, and the subtraction logic `minutes_offset = hours_in_minutes - minutes_offset` creates a negative total minutes value (e.g., -60 - 30 = -90). This negative value causes FieldUtils.safe_multiply to throw ArithmeticError. The fix is to always compute total minutes as `hours_offset * 60 + minutes_offset` (preserving the sign of hours_offset), then multiply by MILLIS_PER_MINUTE to get the correct signed offset in milliseconds.
