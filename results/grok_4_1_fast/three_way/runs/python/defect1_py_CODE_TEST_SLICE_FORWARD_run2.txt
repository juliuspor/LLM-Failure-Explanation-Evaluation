The code fails the test `test_negative_hours_negative_minutes` for input (-2, -15). At line 383, when hours_offset is negative (-2), it executes `minutes_offset = hours_in_minutes - minutes_offset`. Here, hours_in_minutes = -120 and minutes_offset = -15, so minutes_offset becomes -120 - (-15) = -105. Then at line 386, offset = FieldUtils.safe_multiply(-105, DateTimeConstants.MILLIS_PER_MINUTE) = -105 * 60000 = -6300000. This equals -1:45 (-105 minutes), but the test expects -8100000 (-2:15). The subtraction incorrectly reduces the magnitude of the negative offset instead of adding the minute components.
