The bug is not in `translate` but the diagnosis points to index handling issues with surrogate pairs transitioning to single code units. The `translate` method has a critical bug in the loop: after `translate_codepoint` consumes characters, it incorrectly advances `pos` by calling `char_count(code_point_at(code_units, pos))` repeatedly from the same `pos`, causing infinite loops or incorrect advancement. Fixed by advancing `pos` directly by `consumed` code units, as `translate_codepoint` returns the number of UTF-16 code units consumed.
