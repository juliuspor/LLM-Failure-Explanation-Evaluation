The bug is in `RandomStringUtils.random` where custom range `[start, end)` is passed with `letters`/`numbers` flags to `_random_internal`, conflicting with the docstring's restriction logic. Fix by using `letters`/`numbers` to select predefined ranges (letters: 'a'-'z'/'A'-'Z'; numbers: '0'-'9') when either is True, ignoring custom range; otherwise use custom range without restrictions.
