The code fails on input "l" with an IndexError from `char_at(numeric, 0)` when `numeric = val[:-1] = ""` (empty string). This happens because:
1. `val = "l"`, `last_char = "l"`, `len(val) > 0`, so it passes initial checks.
2. No decimal or exponent found, so `mant = "l"`, `dec = None`.
3. `last_char in ("l", "L")` is true, so it enters the long-parsing block.
4. `numeric = val[:-1] = ""` (empty string).
5. `char_at(numeric, 0)` (line referenced in SLICE_BLOCK) tries to access index 0 of empty string, causing `IndexError: String index out of range: 0`.

The condition `(char_at(numeric, 0) == "-" and NumberUtils.is_digits(numeric[1:]))` assumes `numeric` has length >=1, but empty `numeric` breaks it before checking `NumberUtils.is_digits(numeric)`. Same issue for "L", "f", "F".
