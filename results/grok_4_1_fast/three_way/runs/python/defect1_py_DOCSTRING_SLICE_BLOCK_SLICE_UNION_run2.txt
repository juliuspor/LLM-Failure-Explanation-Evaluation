The code fails because `minutes_offset` can become negative after line 383 (`minutes_offset = hours_in_minutes - minutes_offset`), but the range check on lines 377-378 (`if minutes_offset < 0 or minutes_offset > 59`) only runs before this adjustment. A negative `minutes_offset` then causes `FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)` on line 386 to produce a negative `offset`, which triggers the "Offset is too large" `ValueError` on line 388 since fixed offsets must be non-negative.
