The code fails due to incorrect logic for handling negative hours_offset on line 383. When hours_offset is negative (e.g., -1), hours_in_minutes becomes negative (e.g., -60). The condition 'if hours_in_minutes < 0' is true, so it executes 'minutes_offset = hours_in_minutes - minutes_offset' (e.g., -60 - 30 = -90). This overwrites the original positive minutes_offset with a large negative value. Later on line 386, FieldUtils.safe_multiply(-90, DateTimeConstants.MILLIS_PER_MINUTE) triggers ArithmeticError due to the excessive magnitude, which is caught and raises ValueError('Offset is too large'). This happens even for valid inputs like hours_offset=-1, minutes_offset=30, because the subtraction produces an invalid total minutes value instead of correctly computing -30 total minutes.
