The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code at lines 229-230 checks the 3rd and 4th characters (`ch3` and `ch4`) of the locale string, expecting both to be uppercase letters A-Z for a country code. In "fr__POSIX", the 3rd character `ch3` is `'_'` (ASCII 95), which fails the condition `ch3 < 'A' or ch3 > 'Z'`, triggering the `ValueError("Invalid locale format: fr__POSIX")`. This validation incorrectly rejects the language__variant format (empty country with double underscore) that the test expects to handle, matching the Java bug description where position 3 is '_' instead of an uppercase letter.
