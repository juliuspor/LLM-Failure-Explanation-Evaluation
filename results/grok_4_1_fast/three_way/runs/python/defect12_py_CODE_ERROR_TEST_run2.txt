The code fails in test_abbreviate_lower_beyond_length when calling WordUtils.abbreviate("0123456789", 15, 20, None). The input text has length 10. The code adjusts upper to 10 since 20 > len(text). Then StringUtils.index_of(text, " ", 15) is called with start index 15, which exceeds the string length of 10. This causes IndexError: String index out of range: 15 in the index_of method. The code never clamps the lower parameter to be <= len(text), so when lower > len(text), index_of receives an invalid start index.
