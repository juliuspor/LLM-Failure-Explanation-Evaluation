The bug is in the `translate` method's surrogate pair handling logic. When `translate_codepoint` consumes a surrogate pair (returns 2), the position advancement loop incorrectly uses `code_point_at(code_units, pos)` which checks `index + 1` bounds, causing IndexError when a surrogate pair is followed by a single code unit at string end. Fix by advancing `pos` directly by the number of code units consumed (`consumed`), since `translate_codepoint` already handles surrogate pairs correctly.
