The bug occurs because null replacements are skipped entirely instead of being treated as empty string replacements. When replacement_list[1] is None, the code skips that match (i=1) due to 'replacement_list[i] is None', preventing the 'b' replacement. Fix: change condition to skip only if search_list[i] is None/empty, but treat null replacement_list[i] as empty string by using StringUtils.EMPTY when appending.
