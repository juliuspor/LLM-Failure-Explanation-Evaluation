The bug is in the premature validation of minutes_offset. The range check `minutes_offset < 0 or minutes_offset > 59` rejects negative minutes before the offset calculation logic can handle them. Per Joda-Time behavior, negative minutes should be allowed and incorporated into the total offset calculation by extending the negative magnitude (e.g., -2 hours -15 minutes = -(2*3600000 + 15*60000)). Remove the minutes range check to allow the existing calculation logic to handle negative minutes correctly.
