The bug is in `code_point_at`, not `translate`. When processing a low surrogate at index=1 followed by a non-surrogate (65), the surrogate pair check fails but there's no return for the single code unit case, causing fall-through to IndexError. Fix by returning the single code unit if no valid surrogate pair is found. Although the schema requests `translate`, the diagnosis points to `code_point_at` used within `translate`.
