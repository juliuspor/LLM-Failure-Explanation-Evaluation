The bug has two issues: 1) `char_count` is defined but the loop logic is incorrect - it calls `code_point_at(code_units, pos)` repeatedly at the same `pos`, causing it to always process the same codepoint. 2) After `translate_codepoint` consumes codepoints, `pos` must advance by the total UTF-16 code units consumed, not per-codepoint. Fixed by tracking starting position and advancing `pos` by the difference after consumption.
