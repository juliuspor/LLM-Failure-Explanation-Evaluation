The bug occurs because after `translate_codepoint` returns `consumed > 0`, the loop `for pt in range(consumed):` calls `code_point_at(code_units, pos)` repeatedly using the unchanged `pos`, causing IndexError when `pos` is near the end. The fix updates `pos` incrementally inside the loop using `char_count(code_point_at(...))` to properly skip over the consumed codepoints.
