The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in the `translate` method of `CharSequenceTranslator`. The bug occurs in the surrogate pair handling logic around lines 38-44 in `code_point_at`. When processing the input "\U0001F630A" (emoji 😰 followed by 'A'), the code correctly identifies the surrogate pair at `index` (high surrogate at `code_units[index]`, low at `code_units[index+1]`). After consuming the pair in `translate_codepoint()`, the `translate` method advances the index by only 1 position instead of 2, so the new `index` now points to the low surrogate (0xDC00-0xDFFF range). On the next iteration, line 39 checks `if 0xD800 <= high <= 0xDBFF`, but a low surrogate fails this check. The code then falls through without advancing `index`, causing an infinite loop that eventually hits `index >= len(code_units)` and raises IndexError.
