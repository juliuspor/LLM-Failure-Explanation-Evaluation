The code fails in test_translate_surrogate_pair_with_suffix() when processing input "\U0001F630A" (😰A). Here's what happens:

1. code_units = to_utf16_code_units("\U0001F630A") creates [0xD83D, 0xDE30, 0x0041] (3 UTF-16 code units: surrogate pair for 😰 + 'A')
2. At pos=0, self.translate_codepoint(code_units, 0, out) returns consumed=2 (handles surrogate pair)
3. The loop then executes "for pt in range(consumed):" (pt=0, pt=1)
4. First iteration (pt=0): code_point_at(code_units, pos=0) works, char_count()=2, pos += 2 → pos=2
5. Second iteration (pt=1): code_point_at(code_units, pos=2) → accesses code_units[2] which doesn't exist (only indices 0,1 exist)

**Root cause**: After consumed=2, the code incorrectly calls code_point_at(code_units, pos) *twice* in the for loop (range(2)), but pos=2 is out of bounds for a 3-element list.
