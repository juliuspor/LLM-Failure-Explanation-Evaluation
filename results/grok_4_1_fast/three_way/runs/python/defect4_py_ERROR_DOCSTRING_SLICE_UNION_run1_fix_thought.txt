The bug is in the _update_bounds method where the else clauses (lines 352, 356, 362, 366, 372, 376) unconditionally set the min/max indices to the current index, overriding previous minimum/maximum values. This causes the indices to always point to the most recently added item instead of the true min/max. The fix removes these unconditional assignments so indices are only updated when the new period actually establishes a new min/max.
