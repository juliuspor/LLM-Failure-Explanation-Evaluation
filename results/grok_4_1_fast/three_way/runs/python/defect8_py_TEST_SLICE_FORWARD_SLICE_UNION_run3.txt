The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` does not return the expected `Locale("fr", "", "POSIX")`. 

In the `to_locale` method, when processing a 7+ character locale string like "fr__POSIX":
- At line 227-230, it checks `locale_str[3]` ('_') and `locale_str[4]` ('P'). Since '_' < 'A' fails the uppercase country check, it raises `ValueError("Invalid locale format: fr__POSIX")`.
- The code never reaches lines 233-237 to handle the empty country (language__variant) case with double underscore.

The root cause is the country validation at lines 227-230 incorrectly rejects '_' as an invalid country character before checking for the empty country pattern at line 235.
