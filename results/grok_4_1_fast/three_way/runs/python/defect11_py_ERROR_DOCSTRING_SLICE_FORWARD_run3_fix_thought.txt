The bug occurs because `random.randrange` (via `_next_int_java`) requires a positive bound, but `gap = end - start` can be negative when `start > end`, causing `ValueError: bound must be positive`. Add validation in `random` to ensure `start < end` before calling `_random_internal`, raising a clear error mentioning 'start'. This enforces the docstring's `[start, end)` half-open interval assumption.
