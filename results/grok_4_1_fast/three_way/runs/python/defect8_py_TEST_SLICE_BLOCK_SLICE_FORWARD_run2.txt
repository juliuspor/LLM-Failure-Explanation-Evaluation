The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The input string has length 8 with underscores at positions 2 and 4 (0-based indexing: 'f','r','_','_','P','O','S','I','X'). The code logic incorrectly treats the second underscore as the country code start at position 3. When it checks lines 229-230 (`if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z'`), `ch3` is `'_'` (not A-Z), so it raises `ValueError("Invalid locale format: fr__POSIX")`. The parsing logic doesn't handle the `language__variant` format (empty country with double underscore) and assumes position 3 must always be an uppercase letter for a country code.
