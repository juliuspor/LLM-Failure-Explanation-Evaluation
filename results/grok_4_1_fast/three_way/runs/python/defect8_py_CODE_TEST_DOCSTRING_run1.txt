The code fails the test `test_locale_with_empty_country_variant` for input "fr__POSIX". After validating length==7 and first two chars "fr" as lowercase letters, it checks `locale_str[2] != '_'` which passes (position 2 is '_'). Then it extracts `ch3 = locale_str[3]` which is also '_', and checks `if ch3 < 'A' or ch3 > 'Z'`. Since '_' (ASCII 95) is greater than 'Z' (ASCII 90), this condition is true, raising ValueError "Invalid locale format: fr__POSIX". The code assumes position 3 is always the first country code character (uppercase letter), but in language__variant format, position 3 is another '_', not an uppercase letter.
