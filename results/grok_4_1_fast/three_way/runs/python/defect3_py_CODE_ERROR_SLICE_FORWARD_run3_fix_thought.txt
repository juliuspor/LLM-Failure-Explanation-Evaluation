The infinite loop occurs when translate_codepoint consistently returns 0 for the same position. When consumed == 0, we copy the current codepoint and must advance pos by the correct number of code units (1 or 2 for surrogates), but currently use len(c) which is always 1 for single codepoints. Fix by using char_count(code_point_at(code_units, pos)) to properly advance pos. This prevents revisiting the same position and eliminates the infinite loop that leads to the IndexError.
