The code fails in the while loop's fallback branch when `translate_codepoint` returns 0. After `out.extend(c)` where `c = to_chars(codepoint)`, it does `pos += len(c)`. Here `len(c)` is the number of UTF-16 code units for the codepoint (1 or 2), but `code_units` is already a list of UTF-16 code units. This advances `pos` by 1 or 2 code units, which is correct for single surrogates but skips the second code unit when handling a surrogate pair (two code units). Later, if `translate_codepoint` consumes something, the for loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls `code_point_at(code_units, pos)` with an incorrect `pos` that has already been advanced, causing `IndexError` from slice 35-36 in `code_point_at` since `pos` is out of range (negative or >= len(code_units)).
