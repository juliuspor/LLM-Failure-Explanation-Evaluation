The bug occurs because `random(3, 5, 5, False, False)` passes start=5, end=5 to `_random_internal`, creating a zero-length range (gap=0) that triggers `ValueError('bound must be positive')` in `_next_int_java`. The test expects an error message mentioning both 'start' and 'end'. The fix adds validation in `random` to catch zero/negative range before calling `_random_internal` and raises a descriptive ValueError.
