The bug diagnosis misidentifies the error location. The actual issue is in `translate()` method: after `translate_codepoint()` returns `consumed > 0`, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls `code_point_at(code_units, pos)` with an invalid `pos` that exceeds `len(code_units)`. This happens because `pos` is advanced incorrectly - it should simply advance by the number of code units consumed, not re-calculate codepoint boundaries. Fixed by changing the advancement logic to `pos += consumed`.
