The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at` at line 36. When processing input_str = '\U0001F630A' (😰A), `to_utf16_code_units` correctly generates UTF-16 code units [55358, 56608, 65] for the surrogate pair (high=55358=D980, low=56608=DE30) followed by 'A'. In `CharSequenceTranslator.translate`, `translate_codepoint()` calls `code_point_at(0)`, which at line 39 detects the surrogate pair and returns the code point 128528. However, the translator's index is NOT advanced by 2 to skip both surrogate code units. It advances only by 1, so the next call to `code_point_at(1)` occurs. At line 35, `code_units[1]` (56608=DE30) is accessed successfully, but line 39 then checks `index + 1 < len(code_units)` (1 + 1 = 2 >= 3), finds it false, and returns just the low surrogate 56608 as a code point. Later, when this low surrogate reaches the end of processing, an out-of-bounds access occurs in `code_point_at`, triggering the IndexError at line 36 because the index has not been properly advanced past the full surrogate pair.
