The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError on line 36 in `code_point_at`. The input "\U0001F630A" (😰A) converts to UTF-16 code units [55357, 56896, 65] via `to_utf16_code_units`. When `code_point_at` processes index=1 (low surrogate 56896), the check `index + 1 < len(code_units)` (line 39) is false because index+1=2 equals len=3. It returns the raw low surrogate value instead of combining it into the proper code point. Later, when the translator advances the index past this malformed code point, it attempts to access `code_units[3]` (line 35), which is out of range (len=3), triggering `IndexError: String index out of range: 3`. The root cause is `code_point_at` not recognizing the surrogate pair at the end of the code units array.
