The `GrayPaintScale.get_paint` method fails because it raises a `ValueError` when color components are computed outside the 0-255 range, instead of clamping them as required. The test cases show inputs like `-0.5` and `1.5` (outside default 0-1 bounds) and `5.0`/`25.0` (outside custom 10-20 bounds) should clamp to `(0,0,0)` or `(255,255,255)`, but the method throws `ValueError: Color parameter outside of expected range: Red Green Blue`. The docstring confirms clamping is expected before conversion, but the implementation incorrectly validates after computation, causing the error on out-of-bounds inputs.
