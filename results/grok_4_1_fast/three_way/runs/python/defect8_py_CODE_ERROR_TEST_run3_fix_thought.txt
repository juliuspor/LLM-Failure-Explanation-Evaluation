The bug occurs because the code assumes that after the first '_' (at position 2), positions 3-4 must be uppercase country letters, but fails to handle the empty country case like 'fr__POSIX' where position 3 is another '_'. Fixed by restructuring the parsing logic to first split by '_' and validate each part's length and casing, then handle the three cases: language-only, language_country, and language_country_variant (including empty country).
