The test `test_mixed_signs_invalid` fails because the code at lines 377-378 in `DateTimeZone.for_offset_hours_minutes` raises `ValueError` for negative `minutes_offset` values:

```
if minutes_offset < 0 or minutes_offset > 59:
    raise ValueError(f"Minutes out of range: {minutes_offset}")
```

This validation rejects `minutes_offset = 15` in the call `DateTimeZone.for_offset_hours_minutes(-2, 15)`, even though 15 is within the valid range -59 to +59. The check incorrectly treats all negative minutes as invalid regardless of the hours sign, causing the assertion `self.assertEqual(zone.get_id(), "-02:15")` to never execute and fail with a `ValueError` instead.
