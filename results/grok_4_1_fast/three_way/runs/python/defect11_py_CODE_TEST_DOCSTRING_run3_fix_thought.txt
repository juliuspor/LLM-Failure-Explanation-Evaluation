The bug occurs because `RandomStringUtils.random` calls `_random_internal` without validating if `start == end`, which creates an empty range `gap=0`. When `_next_int_java` is called with `bound=0`, it raises `ValueError('bound must be positive')` missing the required 'start' and 'end' mention. Add validation in `random` to catch `start == end` early and raise a descriptive error mentioning both parameters.
