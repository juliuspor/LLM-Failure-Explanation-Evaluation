The bug is in the `char_at` function used by `create_number`. The bounds check uses 'or' instead of 'and', so `index < 0 or index >= len(s)` incorrectly fails for valid index=0 (0 < 0 is False, but if len(s)>0 then 0 >= len(s) is False, yet logic error allows reaching raise). Fix by changing to 'and'. Only replace the `char_at` function since diagnosis confirms it's the source.
