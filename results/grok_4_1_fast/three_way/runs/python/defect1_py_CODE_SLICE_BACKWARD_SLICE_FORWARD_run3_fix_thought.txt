The bug is in the offset calculation logic for negative hours_offset. When hours_in_minutes < 0, the code incorrectly subtracts minutes_offset (minutes_offset = hours_in_minutes - minutes_offset), making the total more negative (e.g., -60 - 30 = -90). Instead, it should always add the absolute minutes component: total_minutes = hours_in_minutes + minutes_offset, preserving the sign of hours_in_minutes.
