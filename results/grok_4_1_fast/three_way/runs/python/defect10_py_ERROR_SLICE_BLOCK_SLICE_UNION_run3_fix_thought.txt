The bug is in the `char_at` helper function, which incorrectly raises IndexError for index=0 on empty strings since `0 >= len('')` is true. Fix by changing the bounds check to `index < 0 or index > len(s) - 1` to properly allow index=0 on non-empty strings while handling empty strings correctly. This resolves the IndexError in `create_number` when `char_at(numeric, 0)` is called on empty `numeric`.
