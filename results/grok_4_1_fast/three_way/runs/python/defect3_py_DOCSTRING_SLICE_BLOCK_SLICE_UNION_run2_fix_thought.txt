The diagnosis misidentifies the bug location. `to_utf16_code_units()` correctly populates its list and doesn't call itself recursively. The real bug is in `CharSequenceTranslator.translate()`: after `consumed == 0`, it calls `code_point_at(code_units, pos)` where `pos` is a code unit index, but if `pos` points to a low surrogate, `code_point_at()` may access `code_units[pos+1]` out of bounds. Fixed by using `char_count(codepoint)` consistently instead of `len(c)`.
