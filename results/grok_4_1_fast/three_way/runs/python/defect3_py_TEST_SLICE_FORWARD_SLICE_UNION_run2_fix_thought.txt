The bug occurs because after `translate_codepoint` consumes a surrogate pair (returns 2), the main loop advances `pos` by `consumed` code units instead of by the correct number of UTF-16 code units for those codepoints. For a surrogate pair, `consumed=2` but each codepoint should advance by `char_count(codepoint)=2`, so it needs to advance by 4 total. The fix iterates over the consumed count using `char_count(code_point_at(pos))` to properly skip the UTF-16 code units consumed.
