The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. At line 227, `ch3 = locale_str[3]` sets `ch3 = '_'`. Then at lines 229-230, the check `if ch3 < 'A' or ch3 > 'Z'` is true (since '_' is ASCII 95, between 'Z'=90 and 'a'=97), so it raises `ValueError(f"Invalid locale format: {locale_str}")`. The code never reaches line 233's `return Locale(locale_str[0:2], locale_str[3:5])` which would have created `Locale("fr", "_")` (also wrong). The bug is that lines 229-230 incorrectly validate `locale_str[3:5]` as uppercase letters when it should handle the empty country case `language__variant` where positions 3-4 are `_` followed by variant.
