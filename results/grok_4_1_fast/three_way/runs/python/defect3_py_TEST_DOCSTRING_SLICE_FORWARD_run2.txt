The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in the `translate` method of `CharSequenceTranslator`. The bug occurs in the surrogate pair handling logic around lines 38-44 in `code_point_at`. When processing the input "\U0001F630A" (emoji 😰 followed by 'A'), the code correctly identifies the surrogate pair at `index` (high surrogate at `code_units[index]`, low at `code_units[index+1]`). After consuming the pair in `translate_codepoint()`, it advances the index by 2 positions. However, the next iteration starts processing 'A' (a single UTF-16 code unit) at the new `index`, but the surrogate pair check on line 39 (`if 0xD800 <= high <= 0xDBFF and index + 1 < len(code_units)`) incorrectly looks one position too far (`index + 1`), causing an out-of-bounds access since 'A' is the last character and there is no `code_units[index + 1]`. This IndexError happens specifically when a surrogate pair is followed by a single code unit at the end of the string.
