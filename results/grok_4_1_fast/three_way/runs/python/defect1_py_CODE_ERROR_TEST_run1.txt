The code fails in the test `test_zero_hours_negative_minutes` when calling `DateTimeZone.for_offset_hours_minutes(0, -15)`. The error 'Minutes out of range: -15' occurs because the validation `if minutes_offset < 0 or minutes_offset > 59:` executes before the offset calculation. For `hours_offset=0` (not negative) and `minutes_offset=-15` (negative), this check incorrectly rejects the valid negative minutes input, even though the code later intends to handle it correctly in the calculation block where `if hours_in_minutes < 0:` is false and `minutes_offset = hours_in_minutes + minutes_offset` would produce the expected negative total offset of -15 minutes.
