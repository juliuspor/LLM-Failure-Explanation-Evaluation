The bug is in the final for loop of translate(). When translate_codepoint() returns consumed > 0, the loop incorrectly calls code_point_at(code_units, pos) repeatedly from the same pos position, causing pos to advance incorrectly (e.g., for consumed=2, advances by 4 instead of 2). Fix by tracking the starting position and advancing pos by char_count(code_point_at(code_units, start_pos)) for each consumed codepoint.
