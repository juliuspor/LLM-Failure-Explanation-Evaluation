The code fails in the `if self._max_middle_index >= 0:` block. It incorrectly uses `self.get_data_item(self._min_middle_index)` instead of `self.get_data_item(self._max_middle_index)` to fetch the period for comparison. This means it always compares the new period's middle value against the minimum middle period (from `_min_middle_index`) rather than the maximum middle period (from `_max_middle_index`).

In `test_max_middle_index_simple()`, after adding periods with middles 50 (idx0), 150 (idx1), 25 (idx2), 75 (idx3):
- After idx0: `_max_middle_index = 0`
- After idx1 (middle=150): compares against idx0 (middle=50), correctly sets `_max_middle_index = 1`
- After idx2 (middle=25): `_min_middle_index` gets set to 2
- After idx3 (middle=75): fetches period from `_min_middle_index=2` (middle=25), compares 75>25, incorrectly sets `_max_middle_index = 3`

This causes `get_max_middle_index()` to return 3 instead of the correct 1.
