The code fails in TimePeriodValues._update_bounds because it incorrectly updates max_middle_index when adding the 4th period (SimpleTimePeriod(50,100), index=3). After adding periods with middle points 50(idx0), 150(idx1), 25(idx2), the max middle should remain at index 1 (150). However, when processing index=3 (middle=75):

1. It correctly compares: if 75 > current max_middle (150) → False, so skips updating
2. But then unconditionally executes self._max_middle_index = index (line ~352 equivalent), setting it to 3

This overwrites the correct index 1 with 3, failing the test (Expected 1, got 3).

The same bug exists for _max_start_index, _max_end_index - they update to the newest index regardless of whether the new value is actually larger, due to missing proper conditional logic after the comparison.
