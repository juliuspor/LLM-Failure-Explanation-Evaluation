The code fails in the `if self._max_middle_index >= 0:` block. It incorrectly uses `self.get_data_item(self._min_middle_index)` instead of `self.get_data_item(self._max_middle_index)` to fetch the period for comparison. This means it always compares the new period's middle value against the minimum middle period (from `_min_middle_index`) rather than the maximum middle period (from `_max_middle_index`).

As a result, when a new period has a middle value larger than the current maximum, the comparison `middle > max_middle` fails because `max_middle` is actually the minimum middle value, so `_max_middle_index` is never updated correctly.

This bug is triggered in `test_max_middle_index_simple()` where after adding periods with middles 50, 150, 25, 75, the expected max middle index is 1 (middle=150) but the code fails to update it properly due to the wrong index being used.
