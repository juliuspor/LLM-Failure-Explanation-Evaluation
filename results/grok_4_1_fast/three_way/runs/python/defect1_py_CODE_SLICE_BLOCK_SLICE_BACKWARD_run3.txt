The code fails due to incorrect handling of negative hours_offset in the calculation of total minutes. On lines 384-385, when hours_offset is negative (e.g., -1), hours_in_minutes becomes negative (e.g., -60). The if hours_in_minutes < 0 branch then sets minutes_offset = hours_in_minutes - minutes_offset (e.g., -60 - 30 = -90). This assigns a negative value to minutes_offset after the range check on line 377 has already passed, since the original minutes_offset was valid (0-59). Later, FieldUtils.safe_multiply(-90, DateTimeConstants.MILLIS_PER_MINUTE) on line 386 throws ArithmeticError due to the large negative value, which is caught and re-raised as ValueError('Offset is too large'). This happens for valid timezone offsets like -1:30, where hours_offset=-1 and minutes_offset=30.
