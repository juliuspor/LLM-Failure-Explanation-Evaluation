The test `test_mixed_signs_invalid` fails because the implementation of `DateTimeZone.for_offset_hours_minutes` does not match the test expectations. Specifically:

1. For `DateTimeZone.for_offset_hours_minutes(-2, 15)`, the test expects `get_id()` to return "-02:15", but the code's logic at lines 383-385 incorrectly handles negative hours with positive minutes:
   - It enters the `if hours_offset < 0:` block
   - Then `if minutes_offset < 0:` is false, so it executes `minutes_offset = hours_in_minutes + minutes_offset` (60 + 15 = 75)
   - This creates offset = 75 * 60000 = 4,500,000 ms (+01:15), not the expected -02:15

2. The minutes validation at line 377 (`if minutes_offset < 0 or minutes_offset > 59`) rejects `DateTimeZone.for_offset_hours_minutes(2, -15)` with ValueError as expected, so that part passes.

The root cause is the flawed conditional logic in lines 383-385 which doesn't correctly compute the total offset magnitude for negative hours regardless of minutes sign, as implied by the docstring behavior.
