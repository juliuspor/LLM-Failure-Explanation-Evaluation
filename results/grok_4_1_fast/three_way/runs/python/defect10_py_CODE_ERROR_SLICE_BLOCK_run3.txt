The code fails with 'IndexError: String index out of range: 0' when processing an empty string after stripping the last character. Here's what happens:

1. For input val = '0l' (length 2), len(val) == 0 check passes.
2. last_char = 'l' (not a digit), so it enters the 'not last_char.isdigit()' block.
3. exp_pos = -1, so exp = None.
4. numeric = val[:-1] = '0' (length 1).
5. last_char = 'l', so it enters the 'if last_char in ("l", "L")' block.
6. Calls char_at(numeric, 0), where numeric = '0'.
7. Inside char_at (line 33), index=0 is valid for len=1, but the condition char_at(numeric, 0) == '-' fails since '0' != '-'. 
8. Then NumberUtils.is_digits(numeric) is called with numeric='0', which succeeds.

However, the real failure occurs for val = 'l' (length 1):
1. len(val) == 0 check passes.
2. last_char = 'l' (not digit).
3. numeric = val[:-1] = '' (empty string, length 0).
4. In 'if last_char in ("l", "L")' block:
   - char_at(numeric, 0) with numeric='' and index=0: 0 >= len('')=0, so char_at raises IndexError at line 33.

Root cause: numeric can be empty string when val length=1 and last_char is a suffix letter (l,L,f,F,d,D). char_at(numeric, 0) doesn't check for empty string before accessing index 0.
