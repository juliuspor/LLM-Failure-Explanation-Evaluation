The contract comment says translators handle surrogate pairs, so `pos` should be advanced by `consumed` directly: `pos += consumed`. Instead, the code enters the for loop, calls `code_point_at(code_units, pos)` where `pos` is still the original position (0), and when `pos` gets advanced incorrectly later, subsequent `code_point_at` calls (like in processing the 'A' at position 2) use an out-of-bounds index, causing IndexError: String index out of range: 2 in `code_point_at` at the bounds check `if index < 0 or index >= len(code_units)`.
