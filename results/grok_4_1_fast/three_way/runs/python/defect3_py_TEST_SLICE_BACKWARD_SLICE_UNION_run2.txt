The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at` at line 36. When processing input_str = '\U0001F630A' (😰A), `to_utf16_code_units` creates code_units = [0xD83D, 0xDE30, 65] for the surrogate pair 😰 followed by 'A'. At index=0, `code_point_at` detects the surrogate pair (lines 38-41), correctly returns the code point, but the translate loop advances index by only 1 instead of 2. On the next iteration, index=1 points to the low surrogate (0xDE30). Line 39 checks index+1=2 < 3 (true), but line 40 reads low = code_units[2] = 65, which is NOT a low surrogate (0xDC00-0xDFFF). The if condition fails, so line 44 returns high = 0xDE30 (invalid lone surrogate). Then at line 35, when translate tries index+1=2 which equals len(code_units), it raises IndexError("String index out of range: 2"). The root cause is the translate method's index increment logic incorrectly advances by 1 after surrogate pairs instead of skipping both code units.
