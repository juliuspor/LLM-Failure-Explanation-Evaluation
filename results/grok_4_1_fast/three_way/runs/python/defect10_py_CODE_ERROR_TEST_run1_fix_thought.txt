The bug occurs because `char_at(numeric, 0)` is called when `numeric` is an empty string (for input 'l'), causing IndexError. Fix by adding a length check before calling `char_at`: if `len(numeric) > 0 and char_at(numeric, 0) == '-'` for the negative case, and directly check `numeric == '-'` for the single dash case. This prevents the bounds error while preserving the logic.
