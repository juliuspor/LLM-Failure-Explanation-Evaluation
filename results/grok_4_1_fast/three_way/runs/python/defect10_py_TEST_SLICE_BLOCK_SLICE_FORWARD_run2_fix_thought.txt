The bug occurs because `char_at(numeric, 0)` is called with `numeric='bob'` where index=0 is valid, so it returns 'b' instead of raising IndexError. This allows invalid input 'bobL' to pass the digit check since 'b'.isdigit() is False but the condition doesn't properly validate. Fix by using safer indexing with `numeric[0] if numeric else None` and improving the digit validation logic to ensure the entire numeric part consists only of valid digits (after sign).
