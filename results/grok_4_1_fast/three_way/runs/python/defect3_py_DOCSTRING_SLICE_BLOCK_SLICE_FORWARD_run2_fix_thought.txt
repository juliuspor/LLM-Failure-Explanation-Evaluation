The bug is in the `translate` method where `code_units` is correctly converted to a list of UTF-16 code units, but the diagnosis incorrectly assumes it's a string. The surrogate pair handling in the main loop (lines 38-44 don't exist; diagnosis refers to `code_point_at`) already works correctly since `code_units` is a list of ints. The real issue is in the consumed handling loop: `pos += char_count(code_point_at(code_units, pos))` incorrectly advances by code unit count instead of codepoint count, and redundantly calls `code_point_at`. Fix by advancing `pos` directly by `consumed` code units.
