The code fails in the `to_utf16_code_units` function with `IndexError: String index out of range: 2` during the `test_translate_surrogate_pair_with_suffix` test. The input string `\U0001F630A` (emoji 😰 followed by 'A') generates exactly 3 UTF-16 code units: [55357, 56896, 65]. When `code_point_at(code_units, 2)` is called to process the 'A' character at index 2, the check `if index < 0 or index >= len(code_units)` on line 35 incorrectly evaluates `2 >= 3` as True because `len(code_units)` is 3. This raises the IndexError even though index 2 is valid (valid indices are 0, 1, 2). The bug is in the off-by-one error in the bounds check on line 35, which should be `index > len(code_units) - 1` or `index >= len(code_units)` corrected to allow the last valid index.
