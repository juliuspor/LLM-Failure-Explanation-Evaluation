The bug is in get_paint(): the clamping sets 'v' correctly but the calculation of 'g' uses the original unclamped 'value' instead of 'v'. This causes negative or >255 values for out-of-bounds inputs, triggering the ValueError. Fix by using 'v' in the calculation. Since clamping ensures g is always 0-255, the range check becomes redundant but is kept for safety.
