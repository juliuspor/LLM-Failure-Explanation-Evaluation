The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError at line 36 in the `code_point_at` function (slice block shown). The input string '\U0001F630A' contains a surrogate pair 😰 (UTF-16 code units at indices 0 and 1) followed by 'A' at index 2. When `CharSequenceTranslator.translate` calls `code_point_at(index)` with index=2 to process the 'A', the `code_units` slice returned by `to_utf16_code_units()[index:index+1]` has length 0 because index=2 is the valid end of the UTF-16 array. The check `if index < 0 or index >= len(code_units)` on line 35 then raises IndexError since 2 >= 0 (empty slice length), even though index=2 is valid for the full string. This happens because `code_point_at` incorrectly uses the sliced `code_units` length instead of the full string length to validate the index.
