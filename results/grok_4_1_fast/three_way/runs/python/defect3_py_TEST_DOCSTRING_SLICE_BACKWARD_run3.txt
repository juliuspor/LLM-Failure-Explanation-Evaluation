The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError from `code_point_at` at line 35-36. The input "\U0001F630A" (😰A) contains a surrogate pair (2 UTF-16 code units) followed by 'A'. In `CharSequenceTranslator.translate`, the loop calls `self.translate_codepoint(code_point_at(input_seq, index))` then advances `index` by 1 (single code unit). When `index` points to the second code unit of the surrogate pair 😰, `code_point_at` sees an isolated trailing surrogate (U+DF30), which is invalid. The `code_point_at` function attempts to access `code_units[index]` where `index` equals `len(code_units)` from `to_utf16_code_units`, triggering the IndexError at line 35: `if index < 0 or index >= len(code_units)`. The root cause is `translate` advances `index` by 1 code unit regardless of whether `translate_codepoint` consumed 1 or 2 code units for astral characters.
