The code fails on `LocaleUtils.to_locale("fr__POSIX")` because it doesn't handle the empty country case with double underscore (`language__variant` format). At line 227, `ch3 = locale_str[3]` gets the second underscore `'_'`. Then at line 229, the check `if ch3 < 'A' or ch3 > 'Z'` fails because `'_'` (ASCII 95) is not an uppercase letter A-Z (ASCII 65-90), raising `ValueError("Invalid locale format: fr__POSIX")`. The code assumes position 3 is always the start of a 2-letter uppercase country code, but in `fr__POSIX`, position 3 is `'_'` indicating empty country, so it never reaches line 233 to create `Locale("fr", "", "POSIX")`.
