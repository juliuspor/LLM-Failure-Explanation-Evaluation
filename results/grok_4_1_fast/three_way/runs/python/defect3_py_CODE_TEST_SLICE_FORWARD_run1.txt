The code fails with an IndexError in test_translate_surrogate_pair_with_suffix() when processing input '\U0001F630A' (emoji 😰 followed by 'A'). Here's what happens:

1. At pos=0, code_units = [0xD83D, 0xDE30, 0x41] (surrogate pair + 'A')
2. self.translate_codepoint(code_units, 0, out) returns consumed=2 (handles surrogate pair correctly)
3. The loop then executes: for pt in range(2): (pt=0, pt=1)
4. First iteration (pt=0): code_point_at(code_units, 0) returns 0x1F630, char_count(0x1F630)=2, pos += 2 → pos=2
5. Second iteration (pt=1): code_point_at(code_units, 2) → index=2 is valid, but slice_forward shows it checks index+1=3 which is out of bounds (len=3)
6. code_point_at() crashes with IndexError on line 40 when accessing code_units[3]

Root cause: After consumed=2, the for pt in range(consumed) loop calls code_point_at(pos) *twice*, but pos=2 only has 1 code unit left. The comment 'they just took care of a surrogate pair' is wrong - translate_codepoint already advanced past it, so this loop double-advances and accesses invalid indices.
