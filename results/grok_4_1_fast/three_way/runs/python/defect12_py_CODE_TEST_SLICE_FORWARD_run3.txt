The code fails the test `test_abbreviate_lower_beyond_length` where `WordUtils.abbreviate("0123456789", 15, 20, None)` should return "0123456789" but returns "" instead. The root cause is in the bounds adjustment at the start: `if upper < lower: upper = lower` sets `upper = 15` since `20 > 15`. Then `index = StringUtils.index_of(text, " ", 15)` returns `-1` because position 15 is beyond the string length of 10 and no space is found from there. This takes the first branch: `substring_java(text, 0, upper)` calls `slice_forward(text, 0, 15)`, but since `15 > len(text)`, `slice_forward` returns an empty string per `[SLICE_FORWARD]` behavior. The append logic does not trigger because `upper == len(text)` is false (15 != 10), so it returns "" instead of the full string.
