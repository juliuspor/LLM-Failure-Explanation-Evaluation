The test `test_add_null_to_null_with_expected_type` fails because `ArrayUtils.add(None, None, expected_type=str)` sets `inferred_type` to `object` (line 337 in the else branch when both array and element are None). Then at line 343, the condition `if inferred_type == object and expected_type != object` is true since `expected_type=str`, triggering the `TypeError` raise at line 344-347. This simulates the Java `ClassCastException` for the LANG-35 bug, but the test expects the call to succeed and return `[None]` without raising any error.
