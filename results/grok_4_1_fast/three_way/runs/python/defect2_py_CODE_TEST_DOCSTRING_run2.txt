The code fails because the clamping logic does not work correctly for custom bounds. In `get_paint`, `v` is clamped with `max(value, self._lower_bound)` and `min(v, self._upper_bound)`, but the calculation `g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)` uses the original unclamped `value`, not `v`. For `GrayPaintScale(10.0, 20.0)` and `value=5.0`, `v=10.0` but `g = int((5.0 - 10.0) / (20.0 - 10.0) * 255.0) = int(-127.5) = -128`. Then `if g < 0` raises `ValueError`, failing `test_custom_bounds` since `(0,0,0)` is never returned.
