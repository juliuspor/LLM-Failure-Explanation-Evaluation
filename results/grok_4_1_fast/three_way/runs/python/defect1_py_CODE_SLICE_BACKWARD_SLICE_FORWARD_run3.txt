The code fails due to incorrect offset calculation logic starting at line 383. When hours_offset is negative (hours_in_minutes < 0), it sets minutes_offset = hours_in_minutes - minutes_offset. For example, hours_offset = -1 and valid minutes_offset = 30 results in hours_in_minutes = -60, then minutes_offset = -60 - 30 = -90. This negative value is then multiplied at line 386 by DateTimeConstants.MILLIS_PER_MINUTE, likely causing an ArithmeticError that triggers the except block at line 388, raising 'Offset is too large'. The subtraction wrongly makes the total offset more negative instead of combining the offsets correctly (should be hours_in_minutes + minutes_offset even for negative hours).
