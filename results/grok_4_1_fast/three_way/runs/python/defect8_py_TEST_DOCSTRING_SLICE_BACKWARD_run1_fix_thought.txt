The bug occurs because the code doesn't handle the empty country case (language__variant format like 'fr__POSIX'). After the first underscore at position 2, it immediately checks positions 3-4 for uppercase letters without verifying if there's actually a country code. The fix adds a check after the first underscore: if position 5 is also an underscore (indicating empty country), parse language from 0:2 and variant from 6:, else proceed with the existing country+variant logic.
