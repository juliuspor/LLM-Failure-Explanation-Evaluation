The code fails because the minutes_offset validation occurs before the offset calculation, but the calculation modifies minutes_offset in a way that can make it negative, which then fails a later test case. Specifically, when hours_offset = -1 (valid) and minutes_offset = 45 (valid), the check 'if minutes_offset < 0 or minutes_offset > 59' passes. Then hours_in_minutes = -1 * 60 = -60. Since hours_in_minutes < 0, line 383 executes: minutes_offset = -60 - 45 = -105. Later, a test checks if minutes_offset < 0 or > 59 and raises 'ValueError: Minutes out of range: -15' (or -105 in this case), even though the input was valid. The root cause is that minutes_offset validation (lines before offset calculation) doesn't account for the subtraction on line 383 that produces negative values from valid inputs.
