The code fails because it does not validate that the total offset stays within one hour. When hours_offset is negative (like -1) and minutes_offset is positive (like 15), the hours validation passes since -1 is between -23 and 23, and minutes validation passes since 15 is between 0 and 59. However, in the calculation block, hours_in_minutes = -1 * 60 = -60, which is < 0, so minutes_offset gets reassigned to -60 - 15 = -75. Later, the code checks if minutes_offset < 0 or > 59, which -75 fails, raising ValueError: Minutes out of range: -15 (the test input). The root cause is the premature validation of minutes_offset before the calculation overwrites it with a negative total value.
