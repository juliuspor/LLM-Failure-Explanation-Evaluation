The bug is in the `translate` method's loop logic. When `translate_codepoint` returns a non-zero `consumed` value, the loop incorrectly advances `pos` by calling `code_point_at(code_units, pos)` (which can raise IndexError if `pos` is out of bounds) inside `char_count`. This violates the contract that `translate_codepoint` already handled advancement. Fix by simply advancing `pos` by `consumed` code units directly, since translators work at the code unit level.
