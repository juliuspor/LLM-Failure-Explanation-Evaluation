The code fails the test `test_negative_hours_negative_minutes` because of incorrect offset calculation logic on lines 383-385. For input (-2, -15):

1. `hours_in_minutes = -2 * 60 = -120`
2. Since `hours_in_minutes < 0`, line 383 executes: `minutes_offset = -120 - (-15) = -120 + 15 = -105`
3. Line 386 multiplies: `offset = -105 * 60000 = -6300000` (which is -01:45)

But the test expects -02:15 (-8100000 millis). The bug is that when hours are negative, the code does `hours_in_minutes - minutes_offset`, which for negative minutes becomes `negative_hours - (negative_minutes) = negative_hours + positive_minutes`, reducing the total magnitude instead of increasing it.

The same logic incorrectly passes `test_mixed_signs_invalid` for (-2, 15), producing the same wrong result (-01:45) that coincidentally matches the expected ID format, but fails the other test's offset value check.
