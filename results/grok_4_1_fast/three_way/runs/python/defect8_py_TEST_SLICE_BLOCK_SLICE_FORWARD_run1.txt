The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code incorrectly parses the format as language_country (5 characters) before checking for language__variant (double underscore). At lines 229-230, it extracts `ch3 = '_'` (position 2) and `ch4 = 'P'` (position 3), then checks if `ch3` is an uppercase letter ('A'-'Z'). Since `'_' < 'A'`, it raises `ValueError("Invalid locale format: fr__POSIX")`. The logic assumes position 3 is always a country code start, but for `language__variant` format, position 3 is the variant start and position 2 is the second underscore, causing the uppercase letter validation to fail prematurely.
