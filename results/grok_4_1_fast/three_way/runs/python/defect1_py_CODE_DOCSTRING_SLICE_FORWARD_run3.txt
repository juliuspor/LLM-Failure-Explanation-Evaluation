The code fails due to incorrect handling of negative hours_offset in the calculation of total minutes. On line 383, when hours_in_minutes is negative (hours_offset < 0), it sets minutes_offset = hours_in_minutes - minutes_offset. This subtracts a positive minutes_offset (0-59) from a negative hours_in_minutes, making minutes_offset more negative than needed. For example, hours_offset=-1, minutes_offset=30 gives hours_in_minutes=-60, then minutes_offset=-60 - 30 = -90. Later on line 386, offset = FieldUtils.safe_multiply(-90, MILLIS_PER_MINUTE), which creates a total offset of -90 minutes instead of the correct -30 minutes (-1 hour -30 minutes). The correct calculation should be minutes_offset = hours_in_minutes + minutes_offset for both positive and negative cases to properly add the minutes component to the hours component.
