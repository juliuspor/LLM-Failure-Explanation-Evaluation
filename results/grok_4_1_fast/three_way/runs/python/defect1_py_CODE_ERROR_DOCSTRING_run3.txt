The code fails because it does not validate negative minutes_offset values before checking their range. The test passes hours_offset=-1 and minutes_offset=-15. The hours range check passes since -1 is within -23 to 23. Then the minutes range check triggers: if minutes_offset < 0 or minutes_offset > 59, which is true for -15 < 0, raising ValueError('Minutes out of range: -15'). This happens before the offset calculation logic, which could have handled the negative minutes correctly by subtracting them from the negative hours component.
