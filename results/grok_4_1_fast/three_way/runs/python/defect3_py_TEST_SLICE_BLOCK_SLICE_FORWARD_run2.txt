The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at` at line 36. When processing input_str = '\U0001F630A' (😰A), `to_utf16_code_units` produces [55357, 56384, 65] (high surrogate 0xD8D8, low surrogate 0xDDF0, ASCII 'A' 0x41). The `CsvEscaper.translate_string` method calls `code_point_at(0)` which correctly returns code point for 😰 and advances index to 2. Then it calls `code_point_at(2)` where `code_units[2] = 65` (not a high surrogate), so line 39 condition fails and it returns `high = 65`. However, the translate loop then advances index by 2 (index += 2), making index = 4. Now `code_point_at(4)` is called but len(code_units) = 3, so line 35 raises IndexError: "String index out of range: 4". The root cause is the translate loop incorrectly advances index by 2 for single code unit characters (ASCII), instead of always advancing by 1 and letting `code_point_at` handle surrogate pair detection.
