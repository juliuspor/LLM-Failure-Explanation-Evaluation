The bug occurs because `random` passes `start` and `end` directly to `_random_internal`, which computes `gap = end - start` and calls `_next_int_java(rnd, gap)`. When `start=0` and `end<=0`, `gap <= 0` triggers ValueError('bound must be positive'), but tests expect an error mentioning 'start'. The fix adds validation in `random` to raise ValueError('start must be less than end') before calling `_random_internal`.
