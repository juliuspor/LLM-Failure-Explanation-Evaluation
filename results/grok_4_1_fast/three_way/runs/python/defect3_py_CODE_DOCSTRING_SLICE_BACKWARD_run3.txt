The code fails in the while loop's fallback branch when `translate_codepoint` returns 0. After appending `c = to_chars(codepoint)` to `out` and advancing `pos += len(c)`, the loop continues to the next iteration. However, `pos` now points to the same position where the previous codepoint started, because `len(c)` equals the number of UTF-16 code units for that codepoint (1 for BMP, 2 for astral). This causes an infinite loop: `translate_codepoint` is called repeatedly on the same code units, returns 0 each time, `to_chars` and `out.extend(c)` execute again, but `pos` doesn't advance past the codepoint, so `pos < length` remains true forever. The issue occurs in the `while pos < length:` loop, specifically after `pos += len(c)`.
