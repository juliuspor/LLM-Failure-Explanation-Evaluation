The code fails the test `test_null_safety_inside_arrays_mixed` because it does not handle `None` values in `replacement_list` correctly. In the test, `StringUtils.replace_each("aba", ["a", "b"], ["c", None])` expects "cbc", where the first 'a' is replaced by "c", the 'b' by `None` (empty string), and the second 'a' by "c" again. However, at line 275, `parts.append(replacement_list[replace_index])` appends `None` directly when `replace_index=1` (for "b"), causing `parts` to contain `None`. Later, when joining `parts` into the result string, Python's `''.join(["c", None, "c"])` produces "cNonec" instead of "cbc". The docstring and Java equivalent confirm `None` should be treated as an empty string for replacement, but the implementation appends it literally.
