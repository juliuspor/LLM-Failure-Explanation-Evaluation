The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError from `code_point_at` at line 35 in `to_utf16_code_units`. The input "\U0001F630A" (😰A) contains a surrogate pair 😰 followed by 'A'. When `CharSequenceTranslator.translate` processes the surrogate pair using `code_point_at(index)`, it correctly advances the index by 2 positions (past the pair). However, the translator's index advancement logic then incorrectly increments the index again by 1, causing `index` to become 3. This exceeds the string length of 3 characters (len("\U0001F630A") == 3), triggering the IndexError check `if index < 0 or index >= len(code_units)` at line 35. The root cause is double-advancement of the index after surrogate pairs: once inside `code_point_at` and again in the main translate loop.
