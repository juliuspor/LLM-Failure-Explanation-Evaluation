The code fails the test `test_negative_hours_negative_minutes` and `test_mixed_signs_invalid` for (-2, -15). The bug is on line 383 in the `if hours_in_minutes < 0:` block. When hours_offset is negative (-2), `hours_in_minutes` is negative (-120), so it executes `minutes_offset = hours_in_minutes - minutes_offset` (-120 - (-15) = -105). This incorrectly subtracts the negative minutes_offset, making the total magnitude too large (-105 minutes instead of -135 minutes). The correct logic should always add the signed minutes_offset to the signed hours_in_minutes: `minutes_offset = hours_in_minutes + minutes_offset` (-120 + (-15) = -135). The current code only works correctly when minutes_offset is positive with negative hours, but fails when minutes_offset is also negative.
