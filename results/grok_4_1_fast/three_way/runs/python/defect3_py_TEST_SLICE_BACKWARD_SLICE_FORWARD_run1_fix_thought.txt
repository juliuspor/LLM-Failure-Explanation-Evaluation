The bug occurs because after `translate_codepoint` consumes a surrogate pair (returns 2), the advancement loop incorrectly calls `code_point_at(pos)` where `pos` points to the low surrogate, then advances by `char_count` of that invalid position. Fixed by advancing `pos` directly by the number of code units consumed (`consumed`), since `translate_codepoint` returns the code unit count, not codepoint count.
