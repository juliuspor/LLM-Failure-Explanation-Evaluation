The code fails in test_zero_hours_negative_minutes() when calling DateTimeZone.for_offset_hours_minutes(0, -15). At line 385, the code executes 'minutes_offset = hours_in_minutes + minutes_offset'. Here hours_in_minutes is 0 (from 0 hours * 60) and minutes_offset is -15, resulting in minutes_offset = 0 + (-15) = -15. Then at line 386, this -15 is passed to FieldUtils.safe_multiply(-15, DateTimeConstants.MILLIS_PER_MINUTE), which raises ValueError('Minutes out of range: -15') because safe_multiply rejects negative minute values outside the range 0-59.
