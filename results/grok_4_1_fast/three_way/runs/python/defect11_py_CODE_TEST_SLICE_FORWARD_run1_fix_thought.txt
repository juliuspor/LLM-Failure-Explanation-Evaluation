The bug occurs because `RandomStringUtils.random(3, 5, 5, False, False)` passes start=5, end=5 to `_random_internal`, causing gap=0. When `_next_int_java(rnd, 0)` is called, it raises ValueError('bound must be positive') which doesn't mention 'start' or 'end'. Add validation in `random` method to catch start >= end case and raise a ValueError mentioning both parameters, matching Java LANG-807 behavior.
