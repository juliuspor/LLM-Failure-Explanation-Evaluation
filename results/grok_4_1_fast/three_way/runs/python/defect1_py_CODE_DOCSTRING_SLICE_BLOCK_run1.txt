The code fails due to incorrect handling of negative hours_offset values in the calculation of total minutes. On line ~384 (`minutes_offset = hours_in_minutes - minutes_offset` when `hours_in_minutes < 0`), it subtracts the original positive `minutes_offset` (0-59) from the negative `hours_in_minutes`. This produces incorrect results:

The real failure is that after recalculation, `minutes_offset` can become a large negative value (e.g., hours_offset=-23, minutes_offset=0 → minutes_offset=-1380), but there's no range check on this new `minutes_offset` value before `FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE)` on line ~386. If `minutes_offset` exceeds safe integer bounds for multiplication, it throws `ArithmeticError`, which is caught and raises `ValueError("Offset is too large")`. However, the docstring promises raises only for 'out of range' inputs (-23 to +23 hours, 0-59 minutes), not for valid inputs like hours_offset=-23, minutes_offset=0 that overflow internally.
