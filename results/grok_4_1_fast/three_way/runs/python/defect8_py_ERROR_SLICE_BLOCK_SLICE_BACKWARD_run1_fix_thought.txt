The bug is in the control flow of `to_locale`. After handling `length == 2`, the code unconditionally accesses `locale_str[3]` and `[4]` for all longer strings without checking `length == 5` first, causing IndexError on strings like 'fr_' (length 3). Fixed by restructuring with proper elif chain: handle length==2, then require '_' at [2] and length>=5, then handle length==5, else require second '_' at [5] and parse variant.
