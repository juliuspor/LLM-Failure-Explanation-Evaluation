The bug occurs because the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` calls `code_point_at(code_units, pos)` without ensuring `pos + 1 < len(code_units)` when checking for surrogate pairs, causing IndexError when `pos` is at the second-to-last position. Fix by advancing `pos` by `consumed` code units directly, since `translate_codepoint` already consumed those units and returned the count.
