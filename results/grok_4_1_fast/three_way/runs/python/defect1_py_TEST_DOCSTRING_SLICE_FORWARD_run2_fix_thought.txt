The bug is in the offset calculation logic for mixed sign cases. For negative hours + positive minutes (e.g., -2, 15), minutes should add to the negative magnitude (resulting in -2:15). For positive hours + negative minutes (e.g., 2, -15), raise ValueError as invalid. Fix by always computing total_minutes = hours_offset * 60 + minutes_offset, then validate that sign(total_minutes) matches sign(hours_offset) when hours_offset != 0, and ensure minutes_offset has correct sign relative to hours_offset.
