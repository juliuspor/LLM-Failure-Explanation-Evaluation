The code fails due to incorrect handling of negative hours_offset in the calculation of total minutes. On line 383, when hours_in_minutes is negative (hours_offset < 0), it sets minutes_offset = hours_in_minutes - minutes_offset. This subtracts a positive minutes_offset (0-59) from a negative hours_in_minutes, making minutes_offset more negative than intended. For example, hours_offset=-1 (hours_in_minutes=-60) and minutes_offset=30 results in minutes_offset = -60 - 30 = -90. Then on line 386, this negative minutes_offset is multiplied by MILLIS_PER_MINUTE (positive), producing a correct negative offset. However, the docstring and validation only expect minutes_offset as a positive 'minutes component (within a single hour)', so callers may pass negative values expecting special handling, but the code doesn't validate or adjust for this, leading to inconsistent behavior between positive and negative offsets.
