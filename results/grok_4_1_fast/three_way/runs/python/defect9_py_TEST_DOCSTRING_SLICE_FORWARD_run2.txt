The code fails the test_null_safety_inside_arrays_single test. When search_list=["a"] and replacement_list=[None], the replacement_list[replace_index] (None) is appended to parts list at line 275. In Python, appending None to a list of strings and later joining with ''.join(parts) inserts the string representation of None ("None") into the result, producing "NbNa" instead of the expected "aba". The Java equivalent works because new String[]{null} appends an empty string "" for null replacements, but the Python implementation does not handle None in replacement_list as an empty string.
