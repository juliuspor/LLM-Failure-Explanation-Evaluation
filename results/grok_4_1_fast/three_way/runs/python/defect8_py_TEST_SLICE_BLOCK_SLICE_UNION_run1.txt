The test `test_locale_with_empty_country_variant` fails because the code in `LocaleUtils.to_locale` incorrectly validates the format "fr__POSIX". When length == 5, it checks `ch3 = locale_str[3]` ('_') and `ch4 = locale_str[4]` ('P') at lines 227-229. The condition `if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z':` (line 229) evaluates to true since ch3='_' is not an uppercase letter A-Z, raising `ValueError` at line 230 before reaching the `Locale` constructor. This rejects valid language__variant formats with empty country codes, but the test expects `Locale("fr", "", "POSIX")`.
