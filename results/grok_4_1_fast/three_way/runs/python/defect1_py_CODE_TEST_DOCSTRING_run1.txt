The code fails the test `test_negative_hours_negative_minutes` because of incorrect logic in the offset calculation when `hours_offset` is negative. Specifically, in the line `if hours_in_minutes < 0: minutes_offset = hours_in_minutes - minutes_offset`, when `hours_offset = -2` and `minutes_offset = -15`, it calculates `hours_in_minutes = -120`, then `minutes_offset = -120 - (-15) = -105`. This produces an incorrect total offset of `-105 * 60000 = -6,300,000` milliseconds (-105 minutes), but the test expects `-02:15` (`-(2*3600000 + 15*60000) = -8,100,000` milliseconds). The subtraction `hours_in_minutes - minutes_offset` wrongly reduces the magnitude when both are negative, instead of adding their absolute values to get the expected -2 hours -15 minutes.
