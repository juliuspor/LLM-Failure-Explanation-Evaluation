The test suite fails because the `GrayPaintScale.get_paint` method raises a `ValueError` when input values are outside the bounds (like -0.5 or 1.5), violating the docstring's guarantee that values are clamped before conversion. The docstring states 'Values outside the bounds are clamped before conversion' and returns an `(r, g, b)` tuple with components in `0..255`, but the `Raises: ValueError: If the computed color components fall outside 0..255` clause is triggered instead of clamping. Specifically, `test_lower_bound_clamping` expects `gps.get_paint(-0.5)` to return `(0, 0, 0)` but gets a `ValueError`; similarly, `test_upper_bound_clamping` fails for `gps.get_paint(1.5)`. The `get_paint` method lacks clamping logic for values below the lower bound or above the upper bound before computing the grayscale value.
