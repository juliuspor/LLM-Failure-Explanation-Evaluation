The `substring_java` function incorrectly raises IndexError when `end > len(s)`, but Python slicing handles this gracefully by clamping to `len(s)`. The fix removes the `end > len(s)` check from `substring_java` to match Python slicing behavior while preserving Java-like bounds checking for invalid cases like negative indices or `start > end`. This prevents the IndexError in `abbreviate` when `upper` exceeds string length.
