The bug is in the _update_bounds method where the else clauses (lines ~352, etc.) unconditionally set the min/max indices to the current index even when existing bounds exist, overwriting previous minimums. The fix indents these assignments inside their respective if conditions so they only execute when no previous bound exists (index == -1). This affects all six bound trackers (min/max start, middle, end). A similar bug exists in max_middle where wrong index is used.
