The bug occurs because `random(3, 5, 5, False, False)` passes `end=5` to `_random_internal`, resulting in `gap = end - start = 0`. The `_next_int_java` call then receives `bound=0` and raises `ValueError('bound must be positive')`, but the test expects a message mentioning 'start' and 'end'. Fix by adding validation in `random` method to check if `end <= start` and raise a descriptive error before calling `_random_internal`.
