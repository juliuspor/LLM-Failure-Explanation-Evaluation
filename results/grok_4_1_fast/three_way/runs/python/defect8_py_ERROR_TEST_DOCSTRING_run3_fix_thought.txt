The bug occurs because `to_locale` doesn't handle the `language__variant` format (double underscore with empty country). Added logic to detect consecutive underscores after language (length > 2 and locale_str[2:5] == '__'), validating language letters then extracting variant from position 5. For other cases, preserved existing validation while fixing index access to use `or ''` for safety. This correctly parses 'fr__POSIX' as Locale('fr', '', 'POSIX').
