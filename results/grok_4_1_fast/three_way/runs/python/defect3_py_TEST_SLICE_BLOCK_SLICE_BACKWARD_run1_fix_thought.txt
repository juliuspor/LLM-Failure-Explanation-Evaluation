The bug is in `code_point_at`: when checking for surrogate pairs, it accesses `code_units[index + 1]` without verifying `index + 1 < len(code_units)` before accessing `low = code_units[index + 1]`. For single code units like 'A' at index 2, it tries to access index 3 (out of bounds), triggering IndexError. Fix by adding the bounds check before accessing the low surrogate.
