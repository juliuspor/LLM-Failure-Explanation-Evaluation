The code fails because `minutes_offset` can become larger than 59 after line 383 where it is reassigned as `hours_in_minutes - minutes_offset`. This happens when the original `minutes_offset` is negative (allowed temporarily before validation). For example, if `hours_in_minutes` is 120 and `minutes_offset` is -10, then `minutes_offset` becomes 130 on line 383. Later on line 386, this large value is multiplied by `DateTimeConstants.MILLIS_PER_MINUTE`, creating an `offset` that exceeds the allowed limit, triggering the `raise ValueError("Offset is too large")` on line 388. The initial range check on line 377 (0-59) does not catch this because the value changes afterward.
