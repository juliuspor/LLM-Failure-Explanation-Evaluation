The code fails the test `test_abbreviate_lower_beyond_length` where `WordUtils.abbreviate("0123456789", 15, 20, None)` should return "0123456789" but returns "" instead.

The root cause is in these lines:

```python
if upper == -1 or upper > len(text):
    upper = len(text)
```

When `text` has length 10, `upper=20 > 10`, so `upper` is set to 10. Then `if upper < lower` (10 < 15) is true, so `upper = lower` sets `upper=15`.

Now `index = StringUtils.index_of(text, " ", 15)` searches for a space starting at index 15, but since `len(text)=10`, this returns -1.

The code then executes `result = substring_java(text, 0, upper)` where `upper=15`. Since `substring_java` follows Java slicing rules (exclusive end index), `substring_java(text, 0, 15)` on a 10-character string returns an empty string (no characters from index 0 to 15 exist).

Thus `result=""`, which fails the assertion expecting "0123456789".
