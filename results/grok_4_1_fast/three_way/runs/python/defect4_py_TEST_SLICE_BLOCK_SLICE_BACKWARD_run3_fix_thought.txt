The bug is in the unconditional assignments after each comparison block in _update_bounds(). When an index is already set (>=0), the code compares but always overwrites with the new index regardless of comparison result. Fixed by only updating indices when the new period actually sets a new min/max, and only setting to new index on first item (index == -1). Also fixed bug where max_middle used _min_middle_index instead of _max_middle_index.
