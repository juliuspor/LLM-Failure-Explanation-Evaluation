The bug occurs because `x == int(x)` returns True for negative zero (-0.0) since it equals 0 and `int(-0.0)` is 0, routing it to the integer branch which outputs '0' instead of '-0.0'. Fix by adding an explicit check for negative zero using `is_negative_zero` before the `x == int(x)` check, and handle it in the float branch to preserve the sign.
