The bug occurs because the parser assumes position 3 is always the first country letter for lengths >2, but fails on 'fr__POSIX' where position 3 is '_'. Fixed by splitting on '_' first to handle empty country case, then validating each part's length/format: language (2 lowercase), optional country (2 uppercase), optional variant (non-empty). Adjusted length check to allow >=2 and proper part counts.
