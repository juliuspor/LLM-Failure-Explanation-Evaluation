The code fails because it does not validate `minutes_offset` correctly when `hours_offset` is negative. The test passes `hours_offset = -1` and `minutes_offset = 15`, which are both valid inputs (equivalent to -00:15). The initial check on line 377 passes since `minutes_offset` (15) is between 0 and 59. However, inside the `try` block on line 383, `hours_in_minutes` is -60 (from `-1 * 60`), which is negative, so it executes `minutes_offset = hours_in_minutes - minutes_offset` (-60 - 15 = -75). Then on line 378, the check `minutes_offset < 0` (now -75) triggers and raises ValueError: Minutes out of range: -15. The bug is that the validation on line 377 only checks the input `minutes_offset`, but after line 383 modifies `minutes_offset` to a negative value, the same validation runs again and fails incorrectly for valid timezone offsets.
