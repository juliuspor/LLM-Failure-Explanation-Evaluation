The code fails due to incorrect handling of negative hours_offset values on lines 385-386. When hours_offset is negative (e.g., -1), hours_in_minutes becomes negative (e.g., -60). The if hours_in_minutes < 0 branch then sets minutes_offset = hours_in_minutes - minutes_offset (e.g., -60 - 30 = -90). This makes minutes_offset negative, but the validation on line 378 (if minutes_offset < 0) has already passed since the original minutes_offset was valid (0-59). The negative minutes_offset then causes FieldUtils.safe_multiply on line 388 to throw ArithmeticError, which is caught and re-raised as ValueError('Offset is too large'). This incorrectly rejects valid offsets like hours_offset=-1, minutes_offset=30, which should produce -90 minutes total offset.
