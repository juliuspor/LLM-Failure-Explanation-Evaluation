The code fails due to incorrect logic when computing the total minutes offset for negative hour offsets. On line 383, when hours_in_minutes is negative (e.g., hours_offset = -1 gives hours_in_minutes = -60), the code sets minutes_offset = hours_in_minutes - minutes_offset. For a valid minutes_offset like 30, this becomes -60 - 30 = -90. This negative value then passes the range check on line 377 (which only ran before modification), but causes FieldUtils.safe_multiply on line 386 to throw an ArithmeticError due to the large negative magnitude, triggering the ValueError on line 388. The subtraction incorrectly computes the offset instead of addition; it should always be hours_in_minutes + minutes_offset to get the correct total (e.g., -60 + 30 = -30 minutes).
