The code fails the test `test_null_safety_inside_arrays_mixed` because it does not handle `None` values in `replacement_list` correctly. In the test, `StringUtils.replace_each("aba", ["a", "b"], ["c", None])` expects "cbc", where the first 'a' is replaced by "c", the 'b' by `None` (empty string), and the second 'a' by "c" again. However, at line 275, `parts.append(replacement_list[replace_index])` crashes with TypeError when `replacement_list[1]` is `None` because `None` cannot be appended to a list of strings. The null safety check at line 288 (`replacement_list[i] is None`) skips searching for that replacer but does not prevent appending `None` during a match with an earlier replacer (like index 0 for the first 'a'). This causes the runtime error instead of producing the expected "cbc".
