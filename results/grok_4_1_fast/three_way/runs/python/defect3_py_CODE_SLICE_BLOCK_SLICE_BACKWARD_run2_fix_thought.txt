The bug is in `to_utf16_code_units` returning empty list for non-empty strings due to incorrect surrogate pair logic. The condition `if codepoint > 0xFFFF` is wrong - it should check `>= 0x10000`. Fixed by changing `> 0xFFFF` to `>= 0x10000` in `to_utf16_code_units`. Also fixed the fallback block in `translate` which incorrectly used `len(c)` instead of `char_count(codepoint)`.
