The bug occurs because substring_java raises IndexError when end == len(s), treating it as out of range, even though Python slicing handles end == len(s) safely. The adjustment 'upper = len(text)' still triggers this. Fix by using safe Python slicing s[0:upper] instead of substring_java in abbreviate, since we already clamp upper <= len(text). This avoids the strict Java bounds check while preserving logic.
