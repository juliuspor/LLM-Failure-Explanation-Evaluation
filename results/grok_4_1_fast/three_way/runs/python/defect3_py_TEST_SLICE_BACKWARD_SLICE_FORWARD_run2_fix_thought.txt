The bug is in the index advancement logic after translate_codepoint consumes codepoints. When consumed > 0, the loop `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))` incorrectly calls code_point_at with the current pos (which may be invalid after previous advancements) and doesn't advance pos within the loop, causing pos to reference invalid indices. Fix by calculating total code units to skip once using code_point_at on the starting index, then advancing pos by that amount.
