The code fails because it incorrectly modifies `minutes_offset` before validating its range. When `hours_offset` is negative (e.g., -1), `hours_in_minutes` becomes negative (-60). The code then executes `minutes_offset = hours_in_minutes - minutes_offset` on line 383. For `minutes_offset = 15`, this computes -60 - 15 = -75. Later, the range check `if minutes_offset < 0` on the original line detects -75 < 0 and raises ValueError: Minutes out of range: -15 (showing the corrupted value).
