The diagnosis misidentifies the issue - `char_at` is used correctly in the code for safe indexing. The real bug is in `_is_all_zeros`: it returns True for None (per docstring) but the all_zeros check `not (f == 0.0 and not all_zeros)` incorrectly treats None as non-all-zeros, causing valid zero inputs like '0.0' to fail float parsing and raise ValueError. Fix by handling None explicitly in all_zeros checks.
