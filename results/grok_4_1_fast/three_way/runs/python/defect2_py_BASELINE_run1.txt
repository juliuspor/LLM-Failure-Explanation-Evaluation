The code fails because the clamping logic doesn't work correctly when using custom bounds. In test_custom_bounds(), GrayPaintScale(10.0, 20.0) sets self._lower_bound=10.0 and self._upper_bound=20.0. For value=5.0:

1. v = max(5.0, 10.0) = 10.0
2. v = min(10.0, 20.0) = 10.0 (clamping works)
3. But g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)
   uses the original unclamped value=5.0, not v:
   g = int((5.0 - 10.0) / (20.0 - 10.0) * 255.0) = int(-0.5 * 255.0) = int(-127.5) = -128

4. Line 77: if g < 0 triggers ValueError: "Color parameter outside of expected range: Red Green Blue"

The calculation on line 74 uses 'value' instead of the clamped 'v', so clamped inputs still produce out-of-range g values.
