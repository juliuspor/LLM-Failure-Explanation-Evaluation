The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at`. The input "\U0001F630A" (😰A) produces UTF-16 code units [55357, 56384, 65] via `to_utf16_code_units`. When `code_point_at` processes index=1 (low surrogate 56384), it checks `index + 1 < len(code_units)` (2 < 3, true), then accesses `code_units[2]` (65, ASCII 'A'). Since 65 is not a low surrogate (0xDC00-0xDFFF), it falls through without returning. The function lacks a `return` statement for non-surrogate cases at index=1, causing it to raise the IndexError check at lines 35-36 instead of returning the code point for 'A' (65). This bug occurs because `code_point_at` doesn't handle trailing non-surrogate code units after surrogate pairs.
