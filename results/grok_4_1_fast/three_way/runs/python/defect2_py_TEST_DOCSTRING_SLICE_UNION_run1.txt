The code fails the test_custom_bounds test case. When GrayPaintScale is initialized with custom bounds (lower_bound=10.0, upper_bound=20.0) and get_paint(15.0) is called, the normalized value is computed as (15.0 - 10.0) / (20.0 - 10.0) = 0.5. This maps to g = int(0.5 * 255.0) = 127, producing color (127, 127, 127). However, after line 77's check (if g < 0 or g > 255), the code raises ValueError on line 78 because the implementation incorrectly validates the final RGB values instead of the input value. The docstring specifies this ValueError should only raise if computed components fall outside 0..255, but 127 is valid, so the raise condition is wrongly triggered for all inputs.
