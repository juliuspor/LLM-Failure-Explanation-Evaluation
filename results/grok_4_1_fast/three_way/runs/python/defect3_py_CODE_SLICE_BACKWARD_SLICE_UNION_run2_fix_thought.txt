The bug occurs because after `translate_codepoint` returns `consumed > 0`, the loop doesn't advance `pos` before entering the `for pt in range(consumed):` loop. Inside this loop, `code_point_at(code_units, pos)` is called with the unadvanced `pos`, which can equal `len(code_units)` after consuming all elements, triggering IndexError. The fix advances `pos` by the total UTF-16 code units consumed (using `char_count`) immediately after `translate_codepoint` returns `consumed > 0`, then removes the unnecessary inner loop.
