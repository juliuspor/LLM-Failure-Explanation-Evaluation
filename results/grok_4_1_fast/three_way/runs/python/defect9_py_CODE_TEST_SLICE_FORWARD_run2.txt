The code fails the test `test_null_safety_inside_arrays_mixed` where `StringUtils.replace_each("aba", ["a", "b"], ["c", None])` should return "cbc" but doesn't. The issue occurs in the replacement loop around lines 275-277. When the first match is found ("a" at position 0 with replacement "c"), `parts.append(replacement_list[replace_index])` adds "c" and `start` is set to `0 + len("a") = 1`. Then, searching from `start=1` for the next match, "b" is found at position 2. However, `text.find("b", 1)` returns 2, but the code replaces the "b" at position 2 with `None`, causing `parts.append(None)` which becomes the string "None" in the final join. This produces "cNonea" instead of "cbc". The root cause is that null replacements are skipped only during match *searching* (line 288: `replacement_list[i] is None`), but not during *replacement* (line 275), so nulls are incorrectly appended as strings.
