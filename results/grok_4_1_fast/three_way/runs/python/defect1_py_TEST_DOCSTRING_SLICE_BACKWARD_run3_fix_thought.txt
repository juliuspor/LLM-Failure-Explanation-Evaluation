The bug is in the offset calculation logic for negative hours and positive minutes. The code incorrectly sets `minutes_offset = hours_in_minutes - minutes_offset` when `hours_in_minutes < 0`, which subtracts a positive minutes_offset from a negative value, making it more negative. For -2:15, this computes hours_in_minutes=-120, then minutes_offset=-120-15=-135, which fails the earlier range check (minutes_offset < 0). The fix is to always compute total minutes as hours_offset * 60 + minutes_offset, regardless of sign, then convert to millis.
