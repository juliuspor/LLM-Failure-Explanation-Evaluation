The code fails due to incorrect handling of negative hours_offset values when calculating the total minutes offset. On line 383, when hours_in_minutes is negative (hours_offset < 0), the code sets minutes_offset = hours_in_minutes - minutes_offset. This subtracts a positive minutes_offset (0-59) from a negative hours_in_minutes value, making minutes_offset more negative (e.g., hours_offset=-1, minutes_offset=30 becomes -90). Then on line 386, FieldUtils.safe_multiply(minutes_offset, DateTimeConstants.MILLIS_PER_MINUTE) throws ArithmeticError for this large negative value, which is caught and re-raised as ValueError('Offset is too large') on line 388. This happens even for valid timezone offsets like -1 hour 30 minutes, because the subtraction logic on line 383 wrongly increases the magnitude instead of combining the offsets correctly.
