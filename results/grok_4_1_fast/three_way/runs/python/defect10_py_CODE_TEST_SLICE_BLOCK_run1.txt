The code fails on input "l" with an IndexError from `char_at(numeric, 0)` when `numeric = val[:-1] = ""` (empty string). This happens because `len(val) == 1` passes the empty check, `last_char = 'l'`, `exp_pos = -1`, so `exp = None`. Then `last_char in ("l", "L")` is true, but `numeric[1:]` (when checking `char_at(numeric, 0)`) causes `index=0` to be out of range for empty `numeric`. The slice block confirms IndexError at index 0 on empty string.
