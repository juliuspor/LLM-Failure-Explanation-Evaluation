The code fails in test_add_null_to_null_with_expected_type because ArrayUtils.add(None, None, expected_type=str) returns a list of objects ([None] as Object[] equivalent) instead of a list of strings (String[] equivalent). When the test does self.assertEqual(result, [None]), Python tries to cast the returned object list to str list for comparison, causing TypeError: Cannot cast object list to str list (ClassCastException: [Ljava.lang.Object; cannot be cast to [Ljava.lang.str;).

Root cause: Around line 333-337, type inference sets inferred_type = object when both array and element are None. Later at line 343-344, even though expected_type=str is provided, the code doesn't override the object type and create a str-typed list, so it returns the wrong type. The expected_type parameter is not properly used to enforce String[] creation when both inputs are null.
