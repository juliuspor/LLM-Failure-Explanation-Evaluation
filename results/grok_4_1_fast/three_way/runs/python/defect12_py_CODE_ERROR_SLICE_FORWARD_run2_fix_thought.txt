The bug occurs because `upper` is not properly clamped to `len(text)` when it's a positive integer exceeding the string length. The existing check `if upper == -1 or upper > len(text): upper = len(text)` only triggers for -1 or values > len(text), but fails if upper is set to a value like 20 when len(text)=15. Move the clamping logic to always ensure upper <= len(text) after the initial adjustments.
