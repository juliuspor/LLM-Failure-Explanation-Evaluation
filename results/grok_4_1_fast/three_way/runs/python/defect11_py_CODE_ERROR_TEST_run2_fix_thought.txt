The bug occurs because `random(3, 5, 5, False, False)` passes `start=5` and `end=5` to `_random_internal`, resulting in `gap=0`. The `_next_int_java` function raises 'bound must be positive', but the test expects the error message to mention 'start'. Fix by adding validation in `random` method to check if `end <= start` and raise a descriptive ValueError mentioning 'start' and 'end' before calling `_random_internal`.
