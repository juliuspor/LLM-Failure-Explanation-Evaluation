The diagnosis incorrectly assumes `to_utf16_code_units` fails to encode astral characters - it actually works correctly by producing surrogate pairs. The real bug is in the main `translate` loop: after `consumed == 0`, it calls `code_point_at(code_units, pos)` (which succeeds), then incorrectly advances `pos += len(c)` using code unit count instead of codepoint length via `char_count(codepoint)`. This undershoots for surrogate pairs (advancing 1 instead of 2), causing repeated 0-consumption and eventual IndexError. Fix by using `pos += char_count(codepoint)`.
