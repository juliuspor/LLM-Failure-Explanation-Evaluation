The test `test_mixed_signs_invalid` fails because it expects `DateTimeZone.for_offset_hours_minutes(-2, 15)` to return a zone with id "-02:15", but the implementation raises ValueError. The code checks `if minutes_offset < 0 or minutes_offset > 59:` on lines 377-378, which incorrectly rejects positive minutes (15 >= 0). This validation prevents negative hours with positive minutes, even though the docstring specifies (-2, 15) should produce -02:15. The minutes range check should allow -59 to +59, not 0 to 59.
