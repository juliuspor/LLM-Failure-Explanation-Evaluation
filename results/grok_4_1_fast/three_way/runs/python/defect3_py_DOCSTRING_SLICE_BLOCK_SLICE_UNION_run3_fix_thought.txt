The bug diagnosis is incorrect - `to_utf16_code_units()` has no recursion. The real issue is in `translate()`: when `consumed == 0`, it calls `code_point_at(code_units, pos)` which raises IndexError if `pos` equals `length`, but the loop condition `pos < length` allows this. Fix by adding `if pos >= length: break` before processing. Also fix surrogate pair advancement in the consumed case by using `pos += consumed` since `translate_codepoint()` returns codepoint count, not code unit count.
