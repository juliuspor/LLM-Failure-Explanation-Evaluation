The code fails the test_custom_bounds test case. When GrayPaintScale is initialized with custom bounds (lower_bound=10.0, upper_bound=20.0) and get_paint(15.0) is called, the normalized value is computed as (15.0 - 10.0) / (20.0 - 10.0) = 0.5. Then g = int(0.5 * 255.0) = 127, which is within 0..255, so no ValueError is raised there. However, for gps.get_paint(5.0), the normalized value is (5.0 - 10.0) / (20.0 - 10.0) = -0.5, so g = int(-0.5 * 255.0) = int(-127.5) = -128. At line 77, if g < 0 or g > 255 is true (g=-128 < 0), so it raises ValueError at line 78. The test expects clamping to (0,0,0) without raising an exception, but the implementation raises ValueError instead of clamping values outside bounds before the int() conversion at line 74.
