The code fails on input "l" (and similar single-character suffixes like "L", "f", "F") due to an undefined function `char_at` called on line where `char_at(numeric, 0) == "-"` is checked. For input "l", `last_char` is "l", `numeric = val[:-1] = ""` (empty string), then in the `if last_char in ("l", "L"):` block, `char_at("", 0)` triggers `SLICE_UNION` which raises `IndexError` because index 0 is out of range for empty string. This happens before the `ValueError` can be raised, causing the test `test_lang457_bad_inputs_raise_value_error` to fail since `IndexError` (not `ValueError`) is raised.
