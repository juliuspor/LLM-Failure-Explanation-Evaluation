The bug occurs because for plain numeric strings like '123', the code skips suffix handling (correctly) but then tries create_integer first, which rejects plain decimal strings without hex/octal prefixes due to its Java-like strict parsing. The fix adds a try-except for plain int(val, 10) parsing before attempting create_integer, create_long, and create_big_integer, ensuring simple decimal strings parse correctly while preserving hex/octal support.
