The bug occurs because the condition `if self._min_start_index >= 0:` is false for the first item (when initialized to -1), so the unconditional assignment `self._min_start_index = index` is never reached for the first item, leaving it as -1. This affects all bound indices. Fix by changing the condition to `if self._min_start_index < 0 or ...` so the first item initializes the bounds and subsequent items update if they exceed the current extremes. Applied consistently to all six bound calculations.
