The code fails due to a logic error in handling negative minutes_offset values on lines 383-385. When minutes_offset is negative (e.g., -30), the condition 'minutes_offset < 0' is true, so it executes 'minutes_offset = hours_in_minutes - minutes_offset' where hours_in_minutes is 60. This changes minutes_offset from -30 to 90. Then, on line 386, it computes offset = 90 * 60000 = 5,400,000 ms (90 minutes). Later, the check 'if abs(offset) > MAX_OFFSET' detects this as too large and raises ValueError('Offset is too large') on line 388. However, a negative minutes_offset like -30 should represent a valid -30 minute offset, not be converted to +90 minutes. The incorrect adjustment for negative values produces an invalid oversized offset, causing the raise even for supported inputs like hours_offset=0, minutes_offset=-30.
