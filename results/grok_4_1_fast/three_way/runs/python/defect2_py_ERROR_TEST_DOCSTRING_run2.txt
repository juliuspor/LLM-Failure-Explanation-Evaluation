The `GrayPaintScale.get_paint` method fails because it raises a `ValueError: Color parameter outside of expected range: Red Green Blue` when the input value is outside the expected bounds (like -0.5 or 1.5 in `test_lower_bound_clamping` and `test_upper_bound_clamping`). The docstring states that values outside bounds should be clamped before conversion, but the implementation throws this error instead of clamping to (0,0,0) or (255,255,255). This happens because the code likely computes the color components without clamping the normalized value to [0.0, 1.0] first, resulting in invalid RGB values outside 0..255 during the conversion process.
