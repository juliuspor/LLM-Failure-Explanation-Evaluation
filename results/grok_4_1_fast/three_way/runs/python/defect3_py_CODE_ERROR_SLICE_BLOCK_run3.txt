The code fails with IndexError in code_point_at when index >= len(code_units). In the while loop, after self.translate_codepoint returns consumed > 0, the loop for pt in range(consumed): calls code_point_at(code_units, pos) each iteration. However, pos is not updated inside this loop, so every iteration uses the same pos value. char_count is called but its return value is ignored, so pos never advances. On the second iteration (pt=1), if pos points to a surrogate pair or multi-unit sequence, code_point_at(pos) accesses code_units[pos+2] (or higher), exceeding the array length and triggering the error at SLICE_BLOCK line 36.
