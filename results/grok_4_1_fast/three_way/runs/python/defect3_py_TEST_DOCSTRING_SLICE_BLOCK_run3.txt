The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError at line 36 in the `code_point_at` function (slice block shown). The input "\U0001F630A" (😰A) contains a surrogate pair 😰 followed by 'A'. When `CharSequenceTranslator.translate` processes the surrogate pair using `code_point_at(code_units, index)`, it correctly advances the index past both surrogate code units (2 positions in UTF-16). However, the translator's main loop then increments the index again, causing `index >= len(code_units)`. The `code_point_at` function at line 35 checks `if index < 0 or index >= len(code_units)` and raises IndexError with message "String index out of range: {index}". This double-advancement bug occurs because `translate_codepoint()` consumes the surrogate pair but the loop doesn't account for multi-unit characters.
