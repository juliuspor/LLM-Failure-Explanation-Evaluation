The code fails the test `test_abbreviate_lower_beyond_length` where `WordUtils.abbreviate("0123456789", 15, 20, None)` should return "0123456789" but returns "" instead.

The root cause is in these lines:

```python
if upper == -1 or upper > len(text):
    upper = len(text)
```

When `text` has length 10, `upper=20 > 10`, so `upper` is set to 10. Then:

```python
if upper < lower:
    upper = lower
```

`10 < 15` is true, so `upper` becomes 15.

Now `index = StringUtils.index_of(text, " ", 15)` searches for a space starting at index 15 in a 10-character string, which returns -1 (not found).

The code then executes:

```python
result = substring_java(text, 0, upper)
```

`substring_java(text, 0, 15)` on a 10-character string returns an empty string (equivalent to Java's `substring(0,15)` behavior when 15 > length), causing the test to fail.
