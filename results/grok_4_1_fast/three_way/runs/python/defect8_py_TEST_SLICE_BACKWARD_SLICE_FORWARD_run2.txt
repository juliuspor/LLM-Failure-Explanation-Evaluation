The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The bug occurs at lines 227-230 in `to_locale`: after detecting the double underscore at position 2 (locale_str[2] == '_'), the code accesses `ch3 = locale_str[3]` and `ch4 = locale_str[4]`. For "fr__POSIX", these are '_' (ASCII 95) and 'P' respectively. Line 229 then checks `if ch3 < 'A' or ch3 > 'Z'`, where ch3='_' fails since 95 > 90 ('Z'), raising `ValueError("Invalid locale format: fr__POSIX")`. This incorrectly assumes position 3 must always be an uppercase letter for a country code, but in language__variant format (double underscore), position 3 is another '_' and country should be empty.
