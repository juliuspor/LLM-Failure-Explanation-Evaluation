The code fails because `substring_java(text, 0, upper)` passes `upper` as the end index to `SLICE_FORWARD` at line 34 (`return s[start:end]`), but when `upper == len(text)`, this returns an empty string since Python slicing excludes the end index. This happens in two places:
1. When no space is found (`index == -1`) and `upper != len(text)` is false (i.e., no truncation occurred), it returns empty instead of the full text.
2. When `index > upper`, it returns `substring_java(text, 0, upper) + ...` which is empty + suffix when `upper == len(text)`.

The docstring expects the full text returned when no suitable space is found before `upper` (adjusted to `len(text)`), but `SLICE_FORWARD` with `end == len(s)` returns "" due to Python's exclusive end slicing.
