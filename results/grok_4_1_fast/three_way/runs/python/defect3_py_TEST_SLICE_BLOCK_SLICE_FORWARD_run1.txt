The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at` at line 36. When processing input_str = '\U0001F630A' (😰A), the UTF-16 code units are [0xD83D, 0xDE30, 0x0041]. After correctly decoding the surrogate pair at index 0 (high=0xD83D at line 38, low=0xDE30 at line 40) and returning code point 0x1F630 at line 43, `CharSequenceTranslator.translate` advances the index by only 1 position instead of 2. On the next iteration, index=1 points to low surrogate 0xDE30. At line 39, 0xDE30 (0xDE30) is not in high surrogate range 0xD800-0xDBFF, so it falls through to line 44 and returns high=0xDE30 as a code point. Later, when advancing past this invalid single low surrogate, the index becomes 2, which equals len(code_units)=3, triggering the IndexError check at line 35 since index >= len(code_units). The root cause is `translate` not advancing by 2 after surrogate pairs are detected in `code_point_at`.
