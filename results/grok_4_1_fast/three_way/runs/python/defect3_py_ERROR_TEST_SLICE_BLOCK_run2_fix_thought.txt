The bug is in the translate method's loop. When translate_codepoint returns consumed > 0, the code incorrectly advances pos by summing char_count(code_point_at(code_units, pos)) for each consumed codepoint, but pos isn't updated between calls, causing code_point_at to be called with the same pos repeatedly and potentially invalid indices. Fixed by advancing pos by consumed code units directly, then using code_point_at with the updated pos only if needed for char_count.
