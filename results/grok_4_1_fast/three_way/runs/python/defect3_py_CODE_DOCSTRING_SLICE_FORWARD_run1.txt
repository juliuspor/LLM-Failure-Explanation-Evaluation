The code fails in the loop after `translate_codepoint` returns a non-zero `consumed` value. It enters a `for pt in range(consumed):` loop that calls `code_point_at(code_units, pos)` and `char_count` for each consumed unit. However, `pos` is not updated inside this loop, so `code_point_at` always examines the same position. This causes infinite re-processing of the same surrogate pair or codepoint that `translate_codepoint` already handled. The comment 'they just took care of a surrogate pair' confirms this intent, but the logic is broken because `pos` stays static during the `for` loop iterations.
