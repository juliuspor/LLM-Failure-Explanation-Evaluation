The bug occurs in `_next_int_java` when `gap = end - start <= 0`, causing `bound <= 0` and raising ValueError. This happens when `chars` is provided with `len(chars) == 0` (already validated) or when default range derivation sets invalid `start`/`end`. Fix by adding validation in `_random_internal` to ensure `gap > 0` before generating characters, matching Java's behavior.
