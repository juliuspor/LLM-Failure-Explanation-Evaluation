The code fails in the `if self._max_middle_index >= 0:` block. It incorrectly uses `self.get_data_item(self._min_middle_index)` instead of `self.get_data_item(self._max_middle_index)` to fetch the period for comparison. This means it always compares the new `middle` value against the middle time of the current minimum middle period (from `_min_middle_index`), not the maximum middle period (from `_max_middle_index`).

In `test_max_middle_index_simple()`, after adding periods with middles 50 (idx 0), 150 (idx 1), 25 (idx 2), and 75 (idx 3):
- When processing idx 1 (middle=150), `_max_middle_index` is set to 1 correctly.
- When processing idx 3 (middle=75), it fetches the period from `_min_middle_index=2` (middle=25), compares 75 > 25 (true), and incorrectly sets `_max_middle_index=3`.

The test expects `_max_middle_index=1` (middle=150) but gets 3, causing the failure.
