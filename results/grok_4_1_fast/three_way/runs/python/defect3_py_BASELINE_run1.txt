The code fails in the final for loop when `translate_codepoint` consumes a surrogate pair (2 code units) but returns `consumed == 2`. For input "\U0001F630A" (😰A), the UTF-16 code units are [55357, 56384, 65] (length=3). When `pos=0`, `translate_codepoint` consumes 2 code units. The loop then executes `for pt in range(2): pos += char_count(code_point_at(code_units, pos))`. First iteration: `code_point_at(code_units, 0)` returns the full emoji codepoint, `char_count` returns 2, `pos = 0 + 2 = 2`. Second iteration: `code_point_at(code_units, 2)` accesses index 2, but `len(code_units)=3`, so index 2 is valid and returns 65 ('A'). However, the error "String index out of range: 2" occurs because `code_point_at` at `pos=2` checks `index + 1 < len(code_units)` (line 39), finds `2 + 1 = 3 == len(code_units)` is false, but the IndexError slice shows it's triggered at the bounds check (line 35-36). The bug is that after consuming 2 code units, `pos` should be advanced by exactly 2 once, not by calling `char_count` twice in a loop which over-advances `pos` beyond the string length.
