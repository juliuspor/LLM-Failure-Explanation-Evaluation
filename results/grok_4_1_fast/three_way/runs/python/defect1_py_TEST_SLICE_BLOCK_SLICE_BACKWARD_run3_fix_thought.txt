The bug is in the offset calculation logic. The code incorrectly modifies `minutes_offset` based on the sign of `hours_in_minutes`, but `minutes_offset` should remain unchanged. The correct approach is to compute total minutes as `hours_offset * 60 + minutes_offset` directly, then convert to millis. This handles mixed signs properly (e.g., -2 hours + 15 minutes = -105 minutes = -02:15). Validation already correctly rejects invalid `minutes_offset` values.
