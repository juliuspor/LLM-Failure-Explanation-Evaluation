The bug is in `code_point_at` function where `high = code_units[index]` accesses the list before the bounds check `if index < 0 or index >= len(code_units)`. This causes IndexError when index=2 but len(code_units)<2. Fix by moving the bounds check to the very beginning of `code_point_at`. The `translate` method itself doesn't need changes as it properly manages indices.
