The bug occurs because when `translate_codepoint` returns 0 for the first code unit of a surrogate pair, the code incorrectly advances `pos` by `len(c)` (which is 2) but then in the consumed==0 fallback path, it calls `code_point_at(pos)` which consumes 2 code units internally without advancing `pos` by 2. Instead, it should use `code_unit_at` that only consumes 1 code unit when surrogates are involved. Fixed by using single code unit access in the fallback path and advancing by 1.
