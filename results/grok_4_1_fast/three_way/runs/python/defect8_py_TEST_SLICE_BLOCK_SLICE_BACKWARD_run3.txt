The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The code at lines 227-230 checks characters at fixed positions 3 and 4 (`ch3 = locale_str[3]`, `ch4 = locale_str[4]`). For "fr__POSIX", position 3 is the first '_' (ASCII 95) and position 4 is the second '_', both of which fail the uppercase letter check `if ch3 < 'A' or ch3 > 'Z' or ch4 < 'A' or ch4 > 'Z'`, triggering `raise ValueError(f"Invalid locale format: {locale_str}")` at line 230. This rigid position-based validation doesn't handle the language__variant format (empty country with double underscore) correctly.
