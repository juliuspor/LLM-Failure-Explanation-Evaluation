The bug occurs when translate_codepoint consumes a partial surrogate pair (high surrogate only), leaving pos at length-1. The for loop then calls code_point_at(pos) which returns the lone high surrogate, and char_count receives an invalid half-pair value (0xD800-0xDBFF). Fix by tracking code units consumed instead of codepoints, and advancing pos by the exact number of code units consumed by translate_codepoint, ensuring pos never points mid-surrogate.
