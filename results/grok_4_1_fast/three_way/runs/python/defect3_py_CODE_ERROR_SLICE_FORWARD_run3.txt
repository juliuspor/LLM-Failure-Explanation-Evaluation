for pt in range(consumed):

When `consumed == 0`, the loop doesn't run, so `pos` stays unchanged. Then the outer `while pos < length` loop repeats, calling `self.translate_codepoint(code_units, pos, out)` again with the same `pos`. If it returns `consumed == 0` repeatedly (e.g., for unhandled codepoints), it gets stuck in an infinite loop at the same position. Eventually, `code_point_at(code_units, pos)` is called from the `if consumed == 0` block, and if `pos == len(code_units) - 1` (last UTF-16 code unit), `index + 1` becomes `len(code_units)`, causing the index error at line 40.
