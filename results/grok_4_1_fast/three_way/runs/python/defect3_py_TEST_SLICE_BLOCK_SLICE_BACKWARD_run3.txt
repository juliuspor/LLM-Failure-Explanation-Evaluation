The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError at line 36 in `code_point_at`: `raise IndexError(f"String index out of range: {index}")`. This happens because `CsvEscaper.translate_string` calls `code_point_at(index)` with an invalid `index` value that exceeds the length of the `code_units` list returned by `to_utf16_code_units(input_str)`.

When processing `input_str = "\U0001F630A"` (😰A), the surrogate pair 😰 consumes 2 UTF-16 code units (positions 0 and 1). After handling this pair, the code incorrectly advances `index` to 2, but `len(code_units)` is only 3 (😰 uses 2 units + 'A' uses 1). The next `code_point_at(2)` call passes `index=2` where `2 < 3` is true, but the internal logic of `code_point_at` (visible in SLICE_BACKWARD at line 35) checks `index >= len(code_units)` and raises IndexError.

Root cause: After processing the surrogate pair at index 0, the translator advances `index` by only 1 instead of 2, or miscalculates the next position, causing `code_point_at` to receive an out-of-bounds index during the suffix 'A' processing.
