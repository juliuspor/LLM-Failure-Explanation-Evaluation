The bug occurs because `to_locale` doesn't handle the `language__variant` format (double underscore with empty country). The current logic assumes a fixed-position check at index 5 for the second underscore, which fails for 'fr__POSIX'. Fix by splitting on underscores first, then validating based on the number of parts: 1 part=language only, 2 parts=language+country, 3 parts=language+country+variant (country can be empty). This handles double underscores correctly while preserving original validation rules.
