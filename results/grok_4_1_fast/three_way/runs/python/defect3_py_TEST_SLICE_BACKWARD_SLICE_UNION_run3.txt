The test `test_translate_surrogate_pair_with_suffix` fails with an IndexError in `code_point_at` at line 36. When processing input_str = "\U0001F630A" (😰A), `to_utf16_code_units` creates code_units = [0xD83D, 0xDE30, 65] for the surrogate pair (0xD83D, 0xDE30) followed by 'A' (65). At index=0, `code_point_at` correctly detects the surrogate pair at lines 38-41 and returns the combined code point. However, the translate loop in CsvEscaper.translate_string then advances the index by only 1 position (index += 1) instead of 2, landing on index=1 (the low surrogate 0xDE30). At index=1, line 35 checks if 1 >= len(code_units)=3 (false), but line 39 checks if index+1=2 < 3 (true), then line 40 reads low = code_units[2]=65. Since 65 is not in 0xDC00-0xDFFF range, it falls through, but the code likely mishandles this invalid lone low surrogate state, or a subsequent bounds check fails when trying to access beyond the pair, triggering the IndexError at line 36 in a later iteration.
