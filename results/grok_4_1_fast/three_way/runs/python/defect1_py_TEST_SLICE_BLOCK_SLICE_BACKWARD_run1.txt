The test `test_mixed_signs_invalid` fails because the code at lines 377-378 in `DateTimeZone.for_offset_hours_minutes` rejects negative minutes with `if minutes_offset < 0 or minutes_offset > 59: raise ValueError(...)`. This causes `DateTimeZone.for_offset_hours_minutes(2, -15)` to raise ValueError as expected, but `DateTimeZone.for_offset_hours_minutes(-2, 15)` also fails because `minutes_offset` (15) passes the range check, yet the test expects it to succeed and return a zone with id '-02:15'. The implementation lacks logic to handle negative hours with positive minutes by treating minutes as additive magnitude (e.g., calculating total offset as -(hours * 3600000 + minutes * 60000)), causing the assertion `self.assertEqual(zone.get_id(), "-02:15")` to fail since no zone object is created.
