The bug is caused by unconditional assignments to the min/max indices after the comparison blocks (e.g., line 352 for min_start_index). These always overwrite the indices with the current index regardless of whether the new period actually sets a new min/max. The fix removes these unconditional assignments for all six index updates (min_start, max_start, min_middle, max_middle, min_end, max_end), ensuring indices are only updated when the new period actually improves the extremum.
