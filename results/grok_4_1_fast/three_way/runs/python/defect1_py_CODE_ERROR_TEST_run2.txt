The code fails in test_negative_hours_negative_minutes when calling DateTimeZone.for_offset_hours_minutes(-2, -15). The minutes validation `if minutes_offset < 0 or minutes_offset > 59:` executes first and raises ValueError('Minutes out of range: -15') because minutes_offset = -15 < 0.
