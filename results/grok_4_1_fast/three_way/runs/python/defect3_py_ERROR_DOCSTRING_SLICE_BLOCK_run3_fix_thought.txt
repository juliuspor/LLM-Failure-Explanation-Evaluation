The bug occurs because after `translate_codepoint()` consumes characters, the loop incorrectly calculates advancement using `char_count(code_point_at(code_units, pos))` where `pos` may already be advanced past valid indices, causing `code_point_at()` to raise IndexError. Fixed by advancing `pos` directly by the number of code units consumed (`consumed`), since `translate_codepoint()` returns the count of code units (not codepoints) it processed.
