The bug is in `char_at` being called with `numeric` that can be empty when `val` ends with a type qualifier like 'L' and has no digits before it (e.g., '-L'). Added empty string check in `char_at` before index access to prevent IndexError. Also fixed inconsistent `all_zeros` logic for non-digit suffix cases by computing it after `numeric = val[:-1]` but before suffix handling.
