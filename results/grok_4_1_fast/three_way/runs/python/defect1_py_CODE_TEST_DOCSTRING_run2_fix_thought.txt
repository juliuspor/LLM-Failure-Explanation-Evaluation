The bug is in the conditional logic for negative hours. When hours_offset is negative, minutes_offset should be added (not subtracted) to hours_in_minutes to correctly accumulate the negative offset. The fix changes `minutes_offset = hours_in_minutes - minutes_offset` to `minutes_offset = hours_in_minutes + minutes_offset` for the negative case, ensuring -2 hours -15 minutes correctly computes as -135 total minutes.
