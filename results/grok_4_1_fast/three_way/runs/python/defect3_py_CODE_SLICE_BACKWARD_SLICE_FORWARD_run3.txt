The code fails in the final for loop due to an infinite loop. After `self.translate_codepoint` consumes some code units (consumed > 0), the loop does `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. On the first iteration, `code_point_at(code_units, pos)` (using lines 35-44) detects a surrogate pair at the current `pos`, consuming 2 code units and returning `char_count=2`. However, `pos` is then incremented by 2, so the next iteration of the for loop calls `code_point_at(code_units, pos)` at the same position again (because pos was already advanced past those units by `translate_codepoint`). This repeatedly processes the same surrogate pair, never advancing `pos` beyond it, causing an infinite loop.
