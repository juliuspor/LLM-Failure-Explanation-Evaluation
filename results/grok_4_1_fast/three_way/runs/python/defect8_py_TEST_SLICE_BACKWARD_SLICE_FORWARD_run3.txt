The test `test_locale_with_empty_country_variant` fails because `LocaleUtils.to_locale("fr__POSIX")` raises a `ValueError` instead of returning `Locale("fr", "", "POSIX")`. The bug occurs at lines 227-230 in `to_locale`: after detecting the double underscore at positions 2-3 (`fr__`), the code accesses `ch3 = locale_str[3]` ('_') and `ch4 = locale_str[4]` ('P'), then checks if `ch3` is an uppercase letter (`'_' < 'A'` is false, but `'_' > 'Z'` is true). This incorrectly raises `ValueError("Invalid locale format: fr__POSIX")` because the empty country case (`language__variant`) is not handled separately before the country validation logic assumes positions 3-4 must be uppercase letters.
