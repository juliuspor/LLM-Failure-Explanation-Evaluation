The bug is in the post-translation advancement logic. When `translate_codepoint` returns a non-zero `consumed` value, the `for pt in range(consumed):` loop calls `code_point_at(code_units, pos)` repeatedly without updating `pos`, causing infinite re-processing of the same position. The fix moves `pos += char_count(...)` outside the loop and updates `pos` incrementally for each consumed codepoint by calculating the correct advancement using `code_point_at` at the current position.
