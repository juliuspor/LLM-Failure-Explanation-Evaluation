The bug occurs because after a translator consumes a surrogate pair (2 code units), the loop incorrectly advances `pos` using `char_count(code_point_at(code_units, pos))` where `pos` is already at the end of the array, causing `code_point_at` to raise IndexError. The fix replaces the incorrect advancement logic with a simple `pos += consumed`, since `translate_codepoint` already returns the exact number of code units consumed.
