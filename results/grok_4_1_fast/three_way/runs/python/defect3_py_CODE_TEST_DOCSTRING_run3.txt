The code fails with an IndexError in `test_translate_surrogate_pair_with_suffix` when processing the input "\U0001F630A" (emoji 😰 followed by 'A'). Here's what happens:

1. `code_units` = [55357, 56384, 65] (UTF-16 for 😰A)
2. At `pos=0`, `translate_codepoint()` returns `consumed=2` (handles surrogate pair)
3. The loop then executes `for pt in range(2):` (pt=0, pt=1)
4. **First iteration (pt=0)**: `code_point_at(code_units, pos=0)` returns the emoji codepoint, `char_count()` returns 2, `pos += 2` → `pos=2`
5. **Second iteration (pt=1)**: `code_point_at(code_units, pos=2)` → `pos=2` is index 65 ('A'), `char_count()` returns 1, `pos += 1` → `pos=3`
6. `length=3`, so now `pos=3 >= length=3`, loop exits
7. **Problem**: The 'A' at position 2 was never processed by `translate_codepoint()` or the fallback path - it was skipped entirely during the `pos += char_count()` advances in the for loop.

The bug is in these lines after `if consumed == 0:`:
```python
for pt in range(consumed):
    pos += char_count(code_point_at(code_units, pos))
```
This incorrectly advances `pos` by full codepoint widths for each consumed unit, skipping unprocessed content when `consumed > 1`.
