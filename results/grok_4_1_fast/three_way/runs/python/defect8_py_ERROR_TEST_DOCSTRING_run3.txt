The `LocaleUtils.to_locale("fr__POSIX")` function fails with `ValueError: Invalid locale format: fr__POSIX` because the parsing logic does not correctly handle the `language__variant` format (double underscore with empty country code). 

The test case expects `Locale("fr", "", "POSIX")` but the code incorrectly validates the format. When it encounters the double underscore `fr__POSIX`, it likely:
1. Splits on single underscores, getting `["fr", "", "POSIX"]`
2. Interprets the empty string `""` as the country code position
3. Rejects it because the country code validation expects either an empty string (for language-only) OR exactly 2 uppercase letters, but fails to recognize this as the special empty-country-with-variant case

The root cause is missing logic to detect and handle consecutive underscores (`language__variant`) as a valid pattern mapping to `Locale(language, "", variant)`, similar to how Java's `Locale.forLanguageTag()` accepts this format.
