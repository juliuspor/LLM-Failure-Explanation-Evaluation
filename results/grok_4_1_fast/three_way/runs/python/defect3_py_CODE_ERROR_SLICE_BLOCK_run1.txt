The code fails with IndexError in code_point_at when index >= len(code_units). In the while loop, after self.translate_codepoint returns consumed > 0, the code enters this block:

for pt in range(consumed):
    pos += char_count(code_point_at(code_units, pos))

Here, code_point_at(code_units, pos) is called with the current pos value each iteration. If pos is already near or at length, it raises IndexError immediately. Even if pos < length initially, char_count returns 1 or 2, but subsequent iterations call code_point_at with the unchanged pos (since pos += ... happens after the call), causing repeated access to the same invalid index. This occurs because pos is not incremented before calling code_point_at inside the loop, violating the slice check at line 36 in code_point_at.
