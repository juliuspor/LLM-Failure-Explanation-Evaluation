The bug is in the final for loop after translate_codepoint consumes code units. When consumed == 2 (surrogate pair), the loop calls char_count(code_point_at(code_units, pos)) twice, but pos changes after the first call, causing code_point_at(2) to fail bounds check even though index 2 is valid. Fix by advancing pos by exactly 'consumed' code units once, since translate_codepoint already consumed that many code units.
