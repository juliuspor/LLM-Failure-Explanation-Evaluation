The code fails in the while loop's final line due to an infinite loop. After `self.translate_codepoint(code_units, pos, out)` returns `consumed > 0`, the loop executes `for pt in range(consumed): pos += char_count(code_point_at(code_units, pos))`. This increments `pos` by the number of UTF-16 code units for the codepoint at the *current* `pos` (which has already been processed), but `pos` is never advanced past the consumed range. On the next iteration, `pos` remains at or before the previously consumed position, so `translate_codepoint` is called again on the same surrogate pair or codepoint, returning `consumed > 0` repeatedly, causing an infinite loop. This occurs because the loop lacks a final `pos += consumed` to skip the already-translated code units.
