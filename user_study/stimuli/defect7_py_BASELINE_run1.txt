What happened (where & why)
- Location: method to_class, loop starting at line 765 (for i in range(len(array))).
- Faulty line: line 766 does this: _ = array[i].upper()
- Root cause: that line assumes each element array[i] is a string and therefore has an upper() method. If array contains None (a valid input per your tests and docstring), array[i] is None and None.upper() raises AttributeError: 'NoneType' object has no attribute 'upper'. That is exactly the exception reported by the test.
- Effect: the method never reaches the append on line 767 for None elements and crashes instead of returning type(None).

Why that line is present (and why it's wrong)
- It looks like the call to .upper() is only being used to touch/inspect the element (_ = …) but it is not used to compute the result. The method should return the element types (type(array[i])), not rely on string methods. Calling .upper() is unnecessary and unsafe for non-string elements (including None).

Concrete fixes (clear steps)
1. Remove the unnecessary .upper() call.
2. Handle None explicitly so you append type(None) for None elements.

Example replacement loop (drop-in fix):
    classes = []
    for i in range(len(array)):
        if array[i] is None:
            classes.append(type(None))
        else:
            classes.append(type(array[i]))
    return classes

Short alternative, more Pythonic:
    return [type(x) for x in array] if array is not None else None

Notes
- The empty-array/None input handling you already have is fine (lines before the loop). Ensure ArrayUtils.EMPTY_CLASS_ARRAY.copy() yields a plain list as expected.
- After these changes the method will satisfy tests that include None elements (e.g. ["Test", None, 99.0]) and will no longer raise AttributeError.
