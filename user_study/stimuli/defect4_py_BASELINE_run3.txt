What happened
- In the "max middle" block you compare the new period's middle against the wrong existing period. Instead of retrieving the period at self._max_middle_index, the code retrieves the period at self._min_middle_index.

Where it occurs (exact spot)
- Inside the block beginning with:
  if self._max_middle_index >= 0:
the code calls:
  max_middle_period = self.get_data_item(self._min_middle_index).get_period()
This is the incorrect variable. Because of that typo, max_middle_period is actually the period for the current minimum-middle index, not the current maximum-middle index.

Why that causes the failing test
- For each new item you want to check whether its middle > current maximum middle. But you compare against the middle of the minimum item (min_middle) instead of the actual maximum. That produces wrong decisions and yields the wrong self._max_middle_index (the test expected 1 but got 3).
