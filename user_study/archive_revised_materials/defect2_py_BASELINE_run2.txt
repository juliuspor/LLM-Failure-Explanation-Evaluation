The code clamps the input value to v but then computes g using the original value variable instead of the clamped v. As a result, when value is outside the bounds (e.g. -0.5 or 1.5), g can be <0 or >255 and triggers the ValueError at lines 74 and 77. Specifically: line 54 defines get_paint and lines 70–75 compute g; at line 74 g = int((value - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0) uses value rather than v. The fix is to use the clamped v in that expression (g = int((v - self._lower_bound) / (self._upper_bound - self._lower_bound) * 255.0)) so clamped inputs produce g in 0..255.
